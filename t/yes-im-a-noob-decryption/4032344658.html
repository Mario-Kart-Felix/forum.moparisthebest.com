Mopman August 1, 2016, 1:04pm #29

You’re basically right, except that BCrypt is also built on a strong encryption, whereas MD5 as a hash algorithm is known to be broken. It has collision flaws coming out the wazoo.

But yes, you could do that if you really wanted to. Every language in the world has a BCrypt library though, so why go to the effort when a very secure oneliner is there for you already?

Also, even 100x MD5 is still much faster than normal BCrypt implementation defaults, so you’d have to do a lot more than that.
