Pretty good, just a few things that I figured I’d add.

  1. I usually don’t specify the implementation on the left side unless I know that it won’t be changed. Take the following code
final ArrayList<String> names = new ArrayList<String>();

If an ArrayList is the suitable implementation of the List interface that I need, and I either a) know that the implementation should never change or b) not allow an outside source (end-user) to change the implementation, than why should it matter?

  1. For things like a group of players, where each is unique, than a Set would be a more suitable solution, as it does not allow duplicate entries.
