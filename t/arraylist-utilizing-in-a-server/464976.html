I will be using JDK 6 for this tutorial. One of the first things I learned about Java was utilizing arrays for a bot I fooled around with back in 2007-08ish. If you are new to Java, specifically ArrayList, this tutorial is for you.

So first thing is first, why would we use an ArrayList in a rsps? Well we can use an ArrayList to add players to a specific minigame or possibly a clan chat system is the first thing that comes to mind. For the sake of the tutorial I will be doing a custom minigame.

To start, we need to create the ArrayList by importing java.util.ArrayList. We will use a “String” data type as we will be using player names.

import java.util.ArrayList;

//two different teams :)
//forget the static, read up on it later.
	public static ArrayList<String> mgBlue = new ArrayList<String>();
	public static ArrayList<String> mgRed = new ArrayList<String>();

The ArrayList is created, now we want to add/remove players to the ArrayList. I provided the links to the api so you guys can read up on the methods provided. In a private server, you are going to want an npc or game object/item whatever to add the player to the ArrayList. Minigames and games in general are just an illusion to the player but yet simple tasks in the backend of the game.

In the scenario that players join our mini game, Let’s add players to both teams.

add(E e) Appends the specified element to the end of this list.
		mgBlue.add("Rodgerwilco");
		mgRed.add("Dds2hit");

For testing purposes, lets see how many players are on each team. Again, in a rsps we would want the world to tell everyone how many are on each team, or maybe specific players. Since we are not working with a game source and we are working with the basics I will use System.out.println();

		System.out.println("Amt of players on blue team : " + mgBlue.size());
		System.out.println("Amt of players on red team : " + mgRed.size());

It should spit out one on each team

Now since we have players being added to our array, we need a way to remove them, if they leave the area they obviously are not playing, so you we will use the remove method.

remove(Object o) Removes the first occurrence of the specified element from this list, if it is present.

		mgBlue.remove("Rodgerwilco");

And for the sake of testing, let’s spit out the amount on each team.

System.out.println("Amt of players on blue team : " + mgBlue.size()); System.out.println("Amt of players on red team : " + mgRed.size());

If you are starting from scratch, your code should look something like this.

Now let’s progress. We have a way to add and remove players, but what if the minigame ends? We don’t want these players in the teams arraylist anymore… so let’s remove them.

clear() Removes all of the elements from this list.
                mgRed.clear();
		System.out.println("Amt of players on red team : " + mgRed.size());

Awesome, now we can clear the minigame arrays when we feel like it/when the minigame is over. What if a minigame is getting crowded? Let’s say we only want 3 people on each team… I created a new method for this and I will walk you through it.

We can check the size of an array by using the size() method.

int size() Returns the number of elements in this list.

private static void addPlayerToBlueMG(String s) { if(mgBlue.size() < 3) { mgBlue.add(s); } else { System.out.println("srry"); } }

A simple size check, if the size > 3… sorry… else add the player. Simple eh?

We learned how to :
Add a player to an arraylist.
Remove a player.
Clear all players.
Check for the size.

Those are pretty much the basics and should get you started, have any questions… feel free to ask.

public class ArrayTut {

public static ArrayList<String> mgRed = new ArrayList<String>();

	mgBlue.add("Rodgerwilco");

	mgBlue.remove("Rodgerwilco");
	
	System.out.println("Amt of players on blue team : " + mgBlue.size());
	System.out.println("Amt of players on red team : " + mgRed.size());
	
	mgRed.clear();

	System.out.println("Amt of players on red team : " + mgRed.size());
	
	addPlayerToBlueMG("Hey");
	addPlayerToBlueMG("How");
	addPlayerToBlueMG("Are");
	addPlayerToBlueMG("You");
	
	System.out.println("Amt of players on blue team : " + mgBlue.size());
	
	
}

	if(mgBlue.size() < 3) {
		mgBlue.add(s);
	} else {
		System.out.println("srry");
	}
}

}

Btw it is good practice to use the least specific type on the left side of a declaration - e.g. here:

1
private static HashMap<Class<? extends RSProtocol>, RSClientFactory> factories = new HashMap<Class<? extends RSProtocol>, RSClientFactory>();
you would change it to this:

1
private static Map<Class<? extends RSProtocol>, RSClientFactory> factories = new HashMap<Class<? extends RSProtocol>, RSClientFactory>();
This makes it easier if you ever want to change the actual implementation, as all the other code referring to this variable, copying it, etc. will be using thta least-specific Map type - you only have to change HashMap to e.g. TreeMap in one place, rather than everywhere you use it.
