Many-Many Relationships in Android

General Programming
zyle1992 August 3, 2016, 12:30pm #1

I’m working on a project that involves many-many relationships between object. My only problem is determining the best way to model them.

A little background on the project, it’s going to be a kind of specialized knowledge base, namely the ingredients and potions in Skyrim.

Each ingredient can be part of 4 potions, and each potion has 4 possible ingredients that can make it.

I would like it to be as loosely coupled as possible, but if it boils down do it I’ll make each Potion/Ingredient it’s own class.

The simplest way I could see is to create a couple of tables for the ingredients and potions using a SQLite database on the phone.

For those of you that have worked with the Android API, what are your opinions on how to approach this?

        results.add(p);

EDIT: I now see why I’ve been having difficulty. I’ve been trying to be too broad, as well as too narrow at the same time.

I think what I’ll do instead is create more of a wikipedia-type app, so that I can add pages, and add the content there.
