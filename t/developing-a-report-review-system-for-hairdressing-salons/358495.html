General Discussion
Ollie July 31, 2016, 7:05pm #1

Hi, my next door neighbour owns a hair salon and has come to me with an idea, their current system “Shortcuts” is a booking program which allows them to assign appointments to staff, for the value, what was purchased etc. this is then put into the staff profile and is stored.

it generates reports, however they are hard to read. What he asked me is would it be possible to create an application which took the information from shortcuts (it outputs an XML document) and implement it into an bespoke application for us to use that information and generate our own reports. In our own reports we would have colour coded items if the staff are meeting their targets, based on hours worked and their previous salon experience.

Currently they have a excel spreadsheet which they use for this, but its not very good as they have lots and lots of excel documents and have to input all the data manually, data intensive.

The reports need to be able to be done daily, weekly and per period. there are 13 periods in a year, in each period is 4 weeks and in each week is 6 working days.

The staff would each have their own targets for each day and the end-user would be able to modify these if need be, different targets would be number of clients, total revenue, customer satisfaction, and a few others… these would be compared to the imported data from the shortcuts output file to generate a percentage and finally display some information which they can look at and they can then review that person and say “yes your doing great” or “wtf is up with this shit” and can show them with colours what they have been doing in certain areas.

I said this was all possible through create, I proposed I would create a Java application which was a node based system and integrated with a MySQL server to store/grab the data from.

However I would like your opinion of different ways of going about it, as I am still quite unsure in my own head of how it will all work.

If there are any things i have missed out just ask.
