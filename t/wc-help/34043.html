Ok, I think I fixed it. I’ll go through and explain what I changed.

First, there’s something you need to understand about if statements. Say you have an if statement like this:

[php]if (ThisIsTrue) then[/php]

As a rule for if statements, if that returns true, it will execute the next line, and only the next line. So if you want multiple things to happen if the if statement returns true, then you have to put a begin as the next line, which will tell the script to execute anything between that begin and its end;. So that if statement would look like this:

[php]if (ThisIsTrue) then
begin
DoThis;
AlsoDoThis;
AndThis;
end;[/php]

That would execute the three procedures ‘DoThis;’, ‘AlsoDoThis;’, and ‘AndThis;’ only if the if statement returns true. Now, if you leave out the begin and end, this is what it would look like:

[php]if (ThisIsTrue) then
DoThis;
AlsoDoThis;
AndThis;[/php]

Now, if the if statement returns true, then the procedure ‘DoThis;’ would be executed, and that would be the end of the if statement, and then the script would carry on to execute ‘AlsoDoThis;’ and ‘AndThis;’. So even if the if statement returns false, those two procedures would be executed.

But, if you only need one thing to be executed off of an if statement, then you could use something like this:

[php]if (ThisIsTrue) then
DoThis;[/php]

Now, in your RunAway; procedure, you had that if statement problem. You ran this if statement:

[php]If (FindFight) then
Runawaydirection(‘N’)
Wait(25000+Random(5000))
RunBack;[/php]

Now, hopefully you can see how this would be a problem. If the if statement returns true, then RunAwayDirection(‘N’); would be executed, and then the Wait() statement, and the RunBack; procedure. But even if the if statement returns false, those last two procedures will execute anyways. The fixed version of that is this:

[php]if (FindFight) then
begin
RunAwayDirection(‘N’);
Wait(25000+Random(5000));
RunBack;
end;[/php]

I believe that’s the thing that was causing the problem, because after the ChopWillows procedure is called, it goes to the drop; procedure, and then the RunAway; procedure, which would’ve previously executed the Wait(); statement and the RunBack; statement.

Also, take a look at the ChopWillows; procedure in the script above, you had the repeat in the wrong spot, I think I put it where you meant it to be, just a small mistake, but take a look at it to see what I did.

Those were the only things I fixed, but please take a look at the SCAR Scripting Standards, because it makes it a lot easier for other people to read your script, and when you start using a lot of nested if staments and loops, it’ll make things way easier.

I hope this fixes your problem,
