sppt July 30, 2016, 6:53am #7

a few comments:

  1. you used a really OUTDATED way of using superglobals, like post, get, server etc($HTTP_SERVER_VARS[‘PHP_SELF’], should be $_SERVER[‘PHP_SELF’]), use the newer ones instead, as i don’t think alot of people are still running like, php 3.

  2. you used addslashes to escape input for the mysql, but addslashes is NOT to be used with mysql, use mysql_real_escape_string() if there’s a connection with the DB.

  3. shorttags (<?), never use them, because some servers don’t support them, use the normal <?php ones.

  4. SQL Queries, i saw some which could be improved, like these:

"select username from users where username = '$username'"

i prefer to put SQL keywords(select, from, where etc) in UPPERcase, this is more for read-ability, and, always put table names and field names in between and, this again for readability, but also for when potential user input is expected.
i would make it like this:

"SELECT `username` FROM `users` WHERE `username`='$username'"

  1. assign the mysql connection to a variable is kinda useless, because you can do without, use it like this, using the STFU-operator(@):
    [php]@mysql_connect($host, $user, $pass) or die(mysql_error());
    @mysql_select_db($db) or die(mysql_error());[/php]
    what the STFU-operator basicly does is surpress error messages and pipe it to the “or …” part.
    also, everywhere where it would say the name of the connection variable, you can leave it out, so you’ll get like:
    [php]mysql_query(…);[/php]

  2. you used multiple unset()'s after eachother, while you can add everything to one unset(), like so:
    [php]unset($var1, $var2, $etc);[/php]

  3. when checking wether a mysql_query has succeeded, i prefer to check it with mysql_affected_rows(), NOT mysql_num_rows!
    [php]if(mysql_affected_rows() == 1) { code } //only when doing stuff like adding / deleting users! haven’t tried anything else![/php]
    also see this

  4. when a required form isn’t filled in, DON’T let it die, return a value, as this would still continue the script!

  5. ever noticed that mysql_numrows() doesn’t exist? => use mysql_num_rows(), or like i said in point 7, mysql_affected_rows()

  6. are you putting the result of the query in a SESSION variable? :S
    this is not very safe to do, as someone with an evil SESSION stealer could then get the pass hash if he’s that desperate

  7. don’t make a wrapper around EVERYthing, it’s just not needed when it’s only called once in the whole script.

  8. in the pages that include others, put in this:
    [php]define(“in_site”, true, false);[/php]

then, on each included page, use this:
[php]if(!defined(“in_site”)) { die(); } //or w/e[/php]

13.(my lucky number ;))
don’t include the mysql connection when it’s just not needed, put it in a wrapper or something

i know, i can bitch about alot, but this is it(so far :P)
i haven’t actually tried this, because i can see from the script how it works
good thing that you kept an eye on HTML standards, BUT!(here we go again…)
you should always put a DTD(doctype declaration) in the first line of HTML, like this:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

and put in the xmlns attribute in the html tag, like so:

<html xmlns="http://www.w3.org/1999/xhtml">

also, you should specify the charset to use, like this:

<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
