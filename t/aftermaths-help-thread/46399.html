There isn’t an exception being thrown for using an item on an NPC. What is happening is this:

1: The server reads the byte for the user using an item on an NPC.
2: The server discards the next 4 bytes, knowing that they are part of the information that the client sends
3: The client sends more than 4 bytes, but the server thinks that the next thing it reads is a whole new cycle
4: The server reads a byte that is part of the current cycle, and thinks that the user is trying to attack a player
5: The server reads the next byte as the playerID of the player the user is trying to attack
6: Since the next byte is probably larger than the number of players on, the server crashes with an ArrayIndexOutOfBoundsException when it tries to do step 5

Du Thelduin:

Okay, you prolly dunno anything about this (as you probably don’t have the same router as me), but I’m having troubles with my router. It’s not major, but I just don’t know where to put ANYTHING since it is different than all the other help tuts about routers.

Ok, go to start, and select run. In run, type ‘cmd’, without the ’ (quotation marks). Once the command prompt window comes up, type ‘ipconfig’, once again, don’t include the ’ quotation marks.

Once you type that, it will come up with something that looks like this:

Windows IP Configuration
Ethernet adapter Local Area Connection:
Connection-specific DNS Suffix:
IP Address… :192.168.1.10
Subnet Mask… :255.255.255.0
Default Gateway… :192.168.1.1

Now when it comes up with that, you need the number that appears next to Default Gateway - That is the internal IP address of your router. Once you get that number, open either Firefox or Internet Explorer, and put it in the address bar - then hit enter.

From there, you should be in your router’s configuration area. Grab your router’s manual and go from there.

Firstly, declare this:

public int plantTimer = 0;

Then search for boolean process() {
