Hello Everyone!

I’m working on developing a Minimum-Viable-Product (MVP) for a Super-Resolution Image/Video Service. To make this post educational, I’m going to explain what exactly a MVP is, and how you can use it to determine how interested customers may be in a potential product.

Now, no one wants to spend a month developing a website that no one ends up wanting, that’s just depressing (and a huge waste of time, especially for a business) So, to minimize this risk, you employ a technique called Deploy First, Code Later. This is where we enter the potential website HDify.com

HDify.com if the idea for a product that offers a service allowing users to upload low-resolution images and videos and increase their resolution while minimizing the loss in quality. The idea is to provide a service that allows access to modern Super-Resolution methods to average users. If you ever want to increase the playable resolution of a video practically everyone gets a very pixelated image. Fixing this is no small task, and there is no guarantee that anyone will use it (which means it’d be hard to make money off it.)

So, instead of spending potentially months on creating a completely automated system, why not create a very small interface and just monitor website visits and attempts to use the product. This would give you a pretty good estimate if anyone will actually use it should you create it.

Because there are implementations of Super-Resolution applications around (just not web) a better MVP would be setup to where users can upload an image/video along with their email address. Then the website would send an email to me along with the image/video, and i would work frantically to increase the resolution (with a desktop app/etc) and send it back to them via email. If I receive a lot of requests in a two week period, I can reliably invest the extra time to implement an automated version of the website, while knowing that there is a chance of monetization. (This is what i currently have)

Anyway, I figured I’d share this technique and a project using it on. Also, if you think the website is a good idea for a product, let me know.
