Ok you can do this [4096]=max number of triangles
the [2000]=max number of triangles with a priority, this is for when priority values are given in the file and not just using the flag value. NEED TO CHANGE THIS TO YOUR NEW VALUE

Don’t worry too much about the [1500] this is for model extents, means the maximum x,y,z value is +/-1024… sqrt(1024²*1024²) = 1448.1546 (so they use 1500 to make sure)

You changed the gzip buffer limit as well? will need to…

Now the really boring bit…
Be realistic in the new value you set, 30k polygons is stupid, the client uses fixed point arithmatic which basically means any vertices less than 4 units in distance will end up at the same point on the screen. That means the client would do lots of calculations and end up not drawing half the triangles. 8k polygons is probably more than you’ll need without having the client waste its time calculating stuff that won’t be drawn.
