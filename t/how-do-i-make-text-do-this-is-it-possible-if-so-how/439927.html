zyle1992 August 3, 2016, 2:17am #17

That’s still not what it’s doing. If you watch the video, it’s essentially replacing the current letter with either a space or the previous/next character (depending on which direction you’re moving). Which simply CANNOT happen with the standard Java libraries. In fact, even in C/C++ you would need to use ncurses, to have more control of what’s actually being displayed, which depends on the context being used (tty, terminal, a terminal emulator, etc).
Here is a bit of the expected results
