Requirements: An application that handles shell scripts (i.e. Terminal.app located in your utilities folder.)

We’re going start this tutorial off with an explanation of shell script.
Shell is the equivalent of the command prompt for mac. It’s remotely related to java, in that it has variables, if-then/if-then-else statements, do-while statements, and input/output.
This script will contain variables, input reading, output, and if-then-else statement.

Scripts are saved with the .sh file extension and can be edited outside of terminal, or whatever your using.

So finally, we’ll get started with the actual tutorial.
So open up TextEdit, or anything that can edit shell scripts, and paste this:

[code]#!/bin/sh

Compiler.sh

Created by TheChubbyBunny @ Moparscape forums

printf “Please specify your server’s path:”

read MYPATH

echo “your server is located in $MYPATH” #just basically a test to see if it’s reading the input

cd $MYPATH

printf “Please specify an action: ‘compile’, ‘run’, or ‘both’:”

read ACTION

if [ “$ACTION” = “compile” ] ; then

javac *.java

echo "compiling server..."

elif [ “$ACTION” = “run” ] ; then

java -Xmx256m server

echo "running server..."

elif [ “$ACTION” == “both” ] ; then

javac *.java
java -Xmx256m server

echo "compiling and running server..."

else

echo "Error in specifying an action, please re-execute the script"

fi[/code]

Save it as compiler.sh, and [U]SAVE IT IN YOUR HOME FOLDER, OTHERWISE IT WILL NOT WORK.[/U]
Well, just make sure the script is in your home folder.

Now open up Terminal or whatever, and type this:

chmod +x compiler.sh

This makes the script executable
Now type this:

./compiler.sh

That command will run the script.
