sk8rdude461 August 4, 2016, 2:53pm #3

I’d do it server sided and screw updating the client.
A “cheap” way you could do this, without changing the value of the integer to a long or etc, would be to add a second integer that would hold all your “billions”. (not that it matters if you changed it to a long…)

[code=java]package com;

import java.text.NumberFormat;

public class MoneyTest {
private static int money = 999999999;
private static int billions = 0;

public static void main(String[] args) {
	addMoney(1);
	addMoney(1000);
	addMoney(1999999999);
	addMoney(Integer.MAX_VALUE);
	System.out.println("Resetting money to 0");
	billions = money = 0;
	addMoney(10000);
}

public static void addMoney(int moneyToAdd) {
	if (moneyToAdd < 1)// If no money to add, don't continue in the method
		return;
	while (moneyToAdd > 999999999) {// if money to add is greater than
									// 999.999m
		moneyToAdd -= 1000000000;
		billions++;
	}
	money += moneyToAdd;
	while (money > 999999999) {// if money to add is greater than 999.999m
		money -= 1000000000;
		billions++;
	}
	System.out.println("Cash: " + getCash());
}

private static String getCash() {
	if(billions > 0)
		return billions + "B and " + format(money) + " coins";
	else
		return format(money) + " coins";
}

public static String format(int num) {
	return NumberFormat.getInstance().format(num);
}

}[/code]
The output from above:

Cash: 1B and 0 coins Cash: 1B and 1,000 coins Cash: 3B and 999 coins Cash: 5B and 147,484,646 coins Resetting money to 0 Cash: 10,000 coins

Above is an example of what I’m talking about.
With this method, you technically can carry up to 2147 billion billions (so 2,147,000,000 * 2,147,000,000).
