Hey guys and girls,

It’s been a while since I’ve posted in this board or anything RSPS related. I think my last source release was sometime around 2008, anyhow myself and Justin Bieber have started a server development which we’ve been working on for a few months now. The development speed has been slow but steady and now we’re looking to up the work completion rate and move towards creating a fun and playable server.

We’re looking for an active developer who has a good working knowledge of how the game client and server operate. You’ll be expected to write content from scratch, integrate this content with other programs running outside of the RSPS, and understand systems that are already operating with the server (the design patterns they follow and how to add to them). The developer then needs a good working knowledge of Java, PHP, and SQL, must be able to use git to commit changes, and must be experienced enough to use SSH and Ubuntu server (includes setting up your RSA key etc). Then there are other requirements such as security competence and credibility that can be discussed.

There is no extensive quiz or anything, you’ll simply join the IRC for a nice discussion with a Smux representative (you’ll be pmed on here first after posting your interest). Now onto the actual server and where we stand in development.

[center][size=24pt]Smux - Server Progress[/size][/center]

[ol][li]Uses Hyperion as a base, fixed any performance or memory issues that may have been present. There is always the potential for more but none we’re currently aware off. [/li]
[li]Full SQL integration with our web side, this includes full integration with our SMF forum:[/li]
[li]Fully functioning report system, when a player is reported ingame a report is made through the SMF report centre.[/li]
[li]Fully functioning membership system based off SMF groups, this includes automatic membership expiration, automatic payment handling, and a membership management function both ingame and on the forum for the player.[/li]
[li]Completely and comprehensive NSA style logging system, this comes with integration into the staff channel where all player communications and actions are streamed. Staff can then override and perform ingame actions from the same channel. This makes all staff and player actions completely transparent and up for discussion in real time. This allows for real time filtering and analysis among other things. [/li]
[li]Membergroups integrated into SMF forum, moderator, admin etc.[/li]
[li]Fully functioning client control centre from SMF, this allows the server administration to change client settings in real time (for example: server ip, port, server name, client info, buttons etc) where a lot of server need to provide updates for small changes we have full dynamic control over a lot of our client from our server. This also helps with preventing rule breakers etc[/li]
[li]Full updating system for the client, each client is assigned a version and has auto updating. Like the above this allows us to keep our playerbase uptodate with the server and our content without causing any fuss.[/li]
[li]All of this along with a few other items come together to form one big database that acts as our command and control centre for the Smux userbase[/li][/ol]

Content -

[ol][li]Basic skills including combat, woodcutting, firemaking, magic, fishing, thieving [/li]
[li]Skills have compactly designed areas with multiple spots, for example with fishing there is an introduction point next to where new players spawn and where our home is, once done there you can speak to one of the NPC fishermen who are also fishing there and gain access to the fishing guild. The idea is to make some form of interaction.[/li]
[li]Fully functioning random events, these are still in experimental stages, they range from typical RS events to sending interfaces (like shaking, or colour masking interfaces) that are designed to disrupt colour based or bitmap bots. This also includes creating local objects on the server that only the target player can see as a form of bait. We’re generally open to a lot of solutions and counter measures that are to be discussed. [/li]
[li]Lowered the threshold of playing the server, things such as magic levels to teleport, and fishing your own food etc have been replaced with shops.[/li]
[li]Minigames started such as castlewars, right now we have fully working portals and teams, we have a game timer and game updating and a fully navigable castlewars map.[/li][/ol]

Myself and Justin aren’t the best content designers you’re gonna meet, that’s why we want to bring another developer to the table, we want somebody who is familiar with the game and can design content and quests and help create unique content that can set this server aside from the rest.

Interested? If so feel free to post on here and I’ll shoot you a PM. Please bear in mind you’ll be expected to use the IRC and be able to figure out how to talk on it. Any discussion regarding payment or how Smux operates can be had over the IRC. Thanks for reading and I look forward to speaking to you.

PS I’m not posting pictures of me cutting down trees and running around castle wars so deal with it.
