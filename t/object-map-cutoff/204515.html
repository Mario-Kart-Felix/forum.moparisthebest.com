One is able to load the static objects without too much trouble. What one will not be able to do is accurately load stateful objects. (i.e. mining rocks, doors, gates, trees, etc) These objects exist in a default state in the cache but get updated as one moves within range. (i.e. whether a door is opened or closed, whether a rock has ore in it, whether a tree is cut down or available) The problem one will find is that the server controls when one will get this updated information. As a simple example, one may see the rocks in the chasm mining area, but one will not know if they have coal/iron/etc in them. Also, one will not see the players/npcs beyond the set range the server has defined. The ‘fog-of-war’ is server-defined. There is no way around it that I have found. Of course, none of this matters if all one wants to do is load static object data…
