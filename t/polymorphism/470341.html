zyle1992 August 4, 2016, 1:25am #2

You can take a look at my code here.

Basically, you have three pillars in Object-oriented Programming; encapsulation, inheritance, and polymorphism.

Encapsulation is basically hiding all data from other objects (an instance of a class). More specifically, if an object doesn’t need to know about a field of another object, it shouldn’t be able to access it. wL-based servers are notorious for violating every aspect of OOP, especially encapsulation (client.skillXP[skills.id], client.running, etc)

Inheritance is, as the name suggests, inheriting behaviors (methods) and traits (fields) from it’s parent (super) class. In the code example, Cat and Dog share the common parent Animal, and therefore inherit the makeSound method.

Polymorphism (which is what this thread is really about) is basically a result of inheritance. As you see in Main.java, an array of Animal objects is created. However, you can’t tell whether the Animal that’s making a sound is a Cat, or a Dog, and the program doesn’t need to know. It just knows that it is an Animal, and can make a noise.

Here is probably a more accurate explanation if you’d like to take a look.
