Err. Okay…
So trenta (not a big fan of them, helped a friend who used em) should have a section for you to get your website’s IP address. It’ll be in the control panel of the side (typically…). However, you can always just ping your website.
Open up the start menu, type in ‘cmd’. Right click the cmd program, hit run as admin (This gives us access to features we need).
Now type:
ping www.yourwebsite.com

And when you run that command, it’ll send 3/4 pings to your site. Here’s an example of the text you’ll see from the ping:

Microsoft Windows [Version 6.1.7601] Copyright (c) 2009 Microsoft Corporation. All rights reserved.

C:\Windows\system32>ping www.google.com

Pinging www.google.com [74.125.207.147] with 32 bytes of data:
Reply from 74.125.207.147: bytes=32 time=20ms TTL=47
Reply from 74.125.207.147: bytes=32 time=23ms TTL=47
Reply from 74.125.207.147: bytes=32 time=46ms TTL=47
Reply from 74.125.207.147: bytes=32 time=47ms TTL=47

Ping statistics for 74.125.207.147:
Packets: Sent = 4, Received = 4, Lost = 0 (0% loss),
Approximate round trip times in milli-seconds:
Minimum = 20ms, Maximum = 47ms, Average = 34ms

C:\Windows\system32>

notice how I pinged google, and it got a reply from 74.125.207.147.
www.google.com is a domain, hence what DNS’s are for. Domain Name Servers (DNS) take the domain (such as google) and route it to the IP, so end users like yourself don’t have to. Because it’d be a pain to remember every website by IP address (esp. when sites have multiple servers).

Now that we’ve got your IP, you can replace the ‘localhost’ you put in the server (as your database isn’t local, it’s on whatever server they put your website on).
