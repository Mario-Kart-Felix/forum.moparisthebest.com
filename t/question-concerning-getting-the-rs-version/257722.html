moparisthebest July 31, 2016, 4:57am #2

It has to do with how an ArrayList works. When you run through the ArrayList to remove the bad values, the indexes change, so you may end up checking a certain value more than once or not at all.

I will try to explain it better using examples, say you have an ArrayList and the underlying array looks like this:

arr[0] = 322
arr[1] = 863
arr[2] = 123
arr[3] = 536

Now, with your loop the first value you check is arr[0], which is less than 500, so you remove it. Now the array looks like this:

arr[0] = 863
arr[1] = 123
arr[2] = 536

Now your program moves onto index 1, arr[1] which is 123, notice you left 863 in there which is incorrect. Do you see the flaw in the method now?

An easy way to go about it would be to not modify the original ArrayList, and just push the values that are possible onto a new one.
