I’ve been working on this project for 3 days out of sheer boredom, and am surprised at how far I’ve got. Basically, RuneC is a form of XML markup parsed using C# to simulate user input, and preform basic programming functions. It doesn’t do a lot right now, but so far I have a few basic features fully working for this semi-language.
-Printing messages to the standard output.
-Moving the mouse to a specified x,y coordinate set.
-Sending keyboard strokes to the currently focused window.
-Pausing execution for the specified amount of milliseconds.
-Variables. No types, but would be possible to add in the future.
Right now, I’m working on if statements, and loops.
Here is an example program that would output “Hello World!”

<program name="Untitled1" date="8/9/2007 8:41:43 PM" author="Owner">
<start />
<print>Hello World!</print>
<end />
</program>

Here is one that sends the value of the variable “message” to the currently focused window.

<program name="Untitled1" date="8/9/2007 8:41:43 PM" author="Owner">
<start />
<var name="message">Hello,from SendKeys!</var>
<sendKeys msg="_message" />
<end />
</program>

That example would send “Hello,from SendKeys!” as if typed from the keyboard.
XML Tag list:

//Program info, always the very first element.
//End of the program, always the last element
//Starting point for all code execution, always the second element
//Ending point for all code execution, always the second last element
{ Variable value } //Initializes a new variable with the specified name and value. The variable is always accessed using the _ prefix, to differentiate a variable from a string.
//Pauses program execution for the specified amount of milliseconds
{ Value to print } //Prints the specified value to the standard output stream.
//moves the mouse to the specified screen coordinates.
//Sends the specified value to the currently focused application as if typed from the keyboard.

Notes:
This isn’t just an idea, all of this actually WORKS.
Script files are saved at compile time as programName.rcs.
Hoping to incorporate syntax highlighting along with many other standard IDE features into the built in editor.
I’m not hoping to replace SCAR or anything, I’m making this because I have nothing better to do.
RuneC is written in Visual C# 2005, using Visual Studio 2005 Express C# Edition.
Source code may, or may not be released. How many people here know C# anyway…

Exception July 30, 2016, 1:32pm #9
xxlegitxx:

Don’t variables have the underscore in front? If so, why does ‘if’ have the underscore in front and why doesn’t lol1 or lol2 have an underscore?

Variables have an underscore prefix when they are referenced, not when they are initialized. The _ isn’t actually part of the name, just a means to tell the compiler that it is a variable name, not a value. The reason if statements are prefixed, is that C# wouldn’t allow me to make a boolean named “if”, so I just prefixed it. In PHP, _ represents super globals, like $_GET and $_POST, while __ is used in things like __construct() which is the constructor method for a class. I guess I just copied that ;p
