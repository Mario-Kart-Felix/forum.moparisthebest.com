system July 30, 2016, 11:32pm #6

You’re wrong there TeleNubby. When you declare the package-name for a certain class, you don’t write the fully-qualified package-name… You remove the class-name from it.

So, what I am saying is, change the following:

package game.utils.Sprite;

To this:

package game.utils;

When importing a certain class, however, you need to use the fully-qualified package-name. For instance:

import game.utils.Sprite;

The class-name, which is “Sprite” with an upper-case ‘S’ (first letter), is just the standard naming-convention, which preferrably should be followed (however, is not a requirement). But, if you by any reason name the class with a lower-case first letter, then you need to import it with a lower-case first letter (it is also called case-sensitivity).

I guess you maybe already know this, pizza pie, but it is good for people that does not know it, who also looks in this thread.
