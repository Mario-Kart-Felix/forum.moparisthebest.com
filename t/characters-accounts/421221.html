What’s your thoughts on multiple characters per website/forum account?

Originally it seemed like a good idea being able to share your bank account but I see it as a big complication now. For example, if a user signs up with the username ‘Luke’ and manages to register the website name but the in-game character name is already taken, they don’t read the error message and see that they’re logged into the website as ‘Luke’. When they try to login in-game and get an invalid login it’s confusing. They then go back to the account management page and change the password only to go through the same thing again.

I’ve tried to make it as clear as I possibly can but still it confuses new players not used to the system so on our new server we’re going to have just one account. If you want another in-game character you will have to register another website account. I think this makes a lot more sense but I also think not being able to share bank accounts will be a huge turn off to a lot of players.

What’s your thoughts?

I’m opting for just a ‘user’ table that uses the same username in-game as it does on the website/forum. Its one table. No complications having to remember multiple passwords or whatever. No worries of banning one character only for them to create another on the same website account to retrieve their items from the bank, etc.

Both methods have their pros and cons but what would you opt for?
