So the “for” can be really helpful sometimes, with it we can “loop through” array, enums, or just loop x times.
An example could be that we wanted to print numbers from 1 to 20,
Why print it 20 times?

System.out.println("1"); System.out.println("2"); System.out.println("3"); System.out.println("4"); System.out.println("5"); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println(""); System.out.println("20");
When we can just do something like this:

for(int i = 0; i < 20; i++) { System.out.println(i); }
That little piece of code will do the same thing as the shown before cluster love. Not only it’s easier & faster to write, it is also more efficient.

Now to write a for there are 3 conditions that we want to know.

[ul][li]Initializing a object.[/li]
[li]The condition.[/li]
[li]The counter.[/li][/ul]

Okay so when we initialize the object we do for(int i = 0;
That means that the variable i has a value of 0, now the condition for the “for” to continue
for(int i = 0; i < 10;
This means that the code inside the for, will repeat as long as i is less than 10
Now if we leave it like that, well we cant but lets pretend that we could, it would be executed infinite times because we are missing the counter
for(int i = 0; i < 10; i++)
the i++ means that the value of i will be increased by one.

That is a way of doing it but what if our variable is bigger than the counter?
It doesn’t matter because we can still do this
for(int i = 20; i > 10; i–)
so our variable has a value of 20, and it will be executed as long as our variable is greater than 10, and the counter will decrease.

Now to declare an array we can do this.

int anArray[]; int[] anArrayTwo;
It doesn’t matter which way you do, either ways work, now lets pretend that we want to add data in our array, this is how we would do.

so here we have an array of type integer with a length of 5, but the index only gets to 4, because Java starts counting and 0, so if we wanted to get the value of 1 we wouldn’t to
anArray[1] because it will be getting the value of 2, if we wanted to get the first value we would do anArray[0].

int[] objectIds = {409, 10638, 44};

        if(objectId == objectIds[i]) {
            deleteItem(boneData[i], 1);

Common errors:
Array out of index exception:
This error comes when we are trying to access the index of an array but the array doesn’t have that length.
For example we have an array with a length of 10 but some how in our for the variable gets to 11
Lets pretend the the variable is i.
If we did array[i], an index exception will come, because we can’t access something that doesn’t even exist.
