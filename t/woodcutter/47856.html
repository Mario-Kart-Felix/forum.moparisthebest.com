Dizzy July 30, 2016, 4:04am #4

Hmm, it looks like you’re using MoveMouseSmooth. I suggest you use MMouse.

How to use:

MMouse(x, y, randomx, randomy);

The randomx and randomy are used so the mouse wont EXACTLY land on the target, but go to a co-ordinate near the specified target, depending on how far the randomx and randomy ranges are.

Example:

MMouse(197, 230, 5, 5);

So now the mouse will not go to 197,230 but a co-ordinate near that. I’ve set the random x and y ranges to 5. So the mouse might land on X:197-202 and Y:230-235. The first 5 is the x and the second is the y.

It’s very simple and highly recommended to use.
There’s also dead’s mouse procedure, but it currently has some bugs in it. (Have tested and does not work at the moment.)

Anyways, other than the mouse procedures, the script looks good.

~Dizzy
