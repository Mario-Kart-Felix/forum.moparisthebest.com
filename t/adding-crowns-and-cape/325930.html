When loading a custom crown, you need to make sure you’re telling the client that it’s from the Sprites folder, if they are in the sprites folder. Look for this:

for(int l4 = 0; l4 < 2; l4++) modIcons[l4] = new Background(streamLoader_2, "mod_icons", l4);
Change the “new Background” to “new Sprite”, like this:

modIcons[l4] = new Sprite("mod_icons");

Then, wherever you see “modIcons = new Background”, change it to “modIcons = new Sprite(“mod_icons”);”
Also, when adding new mod icons, you’ll need to set them to be a sprite, and not a background, as so:

modIcons = new Background[2];

Change to:

modIcons = new Sprite[2];

Change the “2” in the array to however many mod icons you wish to load/use. When you have the mod icons as the above code, it’s saying it’s a sprite as well as an array, but you are not declaring the array or else it won’t take the array. Look for this:

private final Background[] modIcons;

Change that to this:

private final Sprite[] modIcons;

Now, wherever you find this code here:

modIcons[1].method361(k1, l - 12);

Change it to this:

modIcons[1].drawSprite(k1, l - 12);

The codes above are for refactored clients, and seeing how you have a non-refactored client, use this:
http://uppit.com/v/GWBEI3B7
Download it and you’ll see that wherever “Background” is, it might say something like, “Class30_Sub2_Sub1_Sub2”, I think. About the models now. Not really sure; maybe someone else can help you here. Haven’t done modeling in months.
