This has been done to death and I’m sure I’m repeating myself but here we go again with a reflection based canvas hack…

Set the Canvas’s peer to an instance of a light weight peer. That way it will paint on the parent’s component.
Dispose the old heavyweight peer so it doesn’t cover what you want to draw.
Create an instance of JComponent (or similar) and override the getGraphics() method in a similar way to bytecode editing bots (double buffered, etc…).
Make the client a child of the component that you just created.

There will probably be some issues with focus, mouse and keyboard but you get the idea.
