[quote=“stCky, post:21, topic:550705”]For the retardeds, like me, what’s the tl;dr aim of the game ultimately?[/quote]Lol, well it’s a Minecraft-like game in the sense that you gather resources, build stuff, and shape the world around you. That’s about where the similarities end though. You will travel around planets, collecting unique elements and fusing together materials you need to survive and thrive in other environments. The plants and creatures/enemies will run off of a DNA system, and just about everything else will run off of a logical circuitry system. That’s really summarizing things but the main thing to take from it is this game will be cool and allow endless creativity and possibilities.

So here’s an update on what I’ve been working on. I’ve (mostly) fixed light updates to be optimal now, which is great, and so elements and materials have been started. Here’s a couple examples of some elements I whipped up.






There are literally infinite unique elemental possibilities. Each element has 46 visual traits that determine the texture (I whipped all those up in a short time) and two different blending modes, which completely change how the colors blend. All of the textures are seamless (except the lava one, which I messed up early on) and can be rendered with unlimited detail. 512x512 is just the max I went here.

Then there are over 30 in-game properties that make up the physical, chemical, and electrical behaviors of an element. So if materials are going to be combined elements, you can quickly see how unique and endless this part of the game will be.

So while this may look like a small update, here are elements with working behaviors.



Getting the proper element to be chosen as a texture to use was very annoying. Due to how marching cubes works and the fact that points are shared in the world, choosing the right element wasn’t simple. The bottom left point, or the “center” point, could’ve been air but other mesh could’ve been dirt. The “average” couldn’t be taken like lighting, and if the main point was air but other points were different elements, just any point couldn’t be chosen. The solution was to store two elements - one for appearance, and one for behavior. I won’t explain why but this is the most elegant and proper solution.

The problem could’ve been avoided if integer interpolation was a thing in DirectX or graphics cards in general, but it’s not, and texture indexes couldn’t be passed and interpolated as floats. I’m very annoyed with how long it took to solve.
