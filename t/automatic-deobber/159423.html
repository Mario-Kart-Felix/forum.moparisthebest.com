BAsh_CntLgn July 30, 2016, 5:03pm #14

@yakman: It set an array that described the action under the mouse. For example you hovered your mouse over a NPC and it would say actionArray = {“Attack NPC”,123(the id for “fight”),some other crap…} - obviously its not a String[]. Then it just simulated a click somewhere in the client screen.

RS ran like this: {grab data from server, update internal data to reflect server, figure out where user had clicked} and then it did {send user actions to server}. By injecting between those two it was able to click anywhere. Since the action was “forced” (the array was set), it didn’t matter what point you clicked on because the game hadn’t had a chance to update the current action under the mouse. I haven’t had a good look at the code but this is the general idea. I will send you a copy of Aryan (whatever version i can dig up), when someone un-moderates me.
