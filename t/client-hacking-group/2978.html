Mind telling me which VM you used? Suns SCSL Source?
And this guy might be right that it is simpler to properly recode the VM such that it easily allows us code/data inspection and code injection. It would be easier than always bother with new restructured client versions. If done properly, universal signatures can be used to trace data we seek. I have made some thoughts on this and came up with a couple of ideas, yet no time to implement and try those. One approach might be to monitor the “temporal evolution” (this could be done in a rather controlled way to reduce any influences of randomness, though, if done right, shouldn’t be a critical point) of each variable (be it global or local), method invokation, interelations between those objects and the like and create some reasonable data out of it on basis of a statistical analysis. This data should be feasible identify fields and methods of our interest.
But the main advantage is, that there’s very little chance for jagex (unless they do a system scan which can be detected and prohibited easily ) to detect a VM that was being tampered with.
AFAIK, jegax randomly checks whether the client has been tampered with using reflection, and it requires quite some care to properly handle this. IHMO it is pointless to waste any more time on bots based on modified java clients. They can add new nuiscances as we speak here. One should not stick with outdated concepts, one should go with the time and eventually try something new. It is in our nature to advance (well, at least in mine and certainly some other people too).
That said, I have to note that in order to realize such a project you’d have to be a very skilled and experienced coder (otherwise you’re not even able to understand how the VM works), of course you should be familiar with java/bytecode and also ASM knowledge is of advantage for the JIT engine. Plus you need to have a lot spare time to burn and your ambition shouldn’t be any less.
I doubt anyone here falls into the class of fulfilling all the requirements.
So this probably remains a vision :o
