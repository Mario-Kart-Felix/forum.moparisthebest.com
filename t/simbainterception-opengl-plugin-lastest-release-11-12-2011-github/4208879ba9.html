You have to do array, length for procedures [or otherwise pass the length] (regarding Pascal to C++)… this is the nature of C++. An array is not an object with a length property as in Java, but just a pointer with convenient offsetting in possibly multiple dimensions. The only ways you know the length of an array or data starting at a pointer is either 1) passing the length as a parameter explicitly 2) passing it just before the pointer, as Wizzup mentioned 3) terminating the data with the NUL character, as in the case of zero-terminated strings.

If you look into the various implementations of strings in C++ libraries, they use either 2), 3), or a combination of the two. 1) is used for example in main for a standalone program - the argument count is passed to you. This is the nature of C/++.
