As a generic solution…

Put the applet in a container. Hack Canvas so its getGraphics() returns the container’s (not the parent’s eg. applet’s) graphics, transformed as appropriate. Then subclass the container you put the applet so the getGraphics call returns your BufferedImage graphics.

More complex but you don’t have to worry about the crap of multiple Canvases that replace themselves at different times.

What I was thinking of:

BotContainer.getGraphics:
Render the bot information on buffered image.
Get the parent graphics and draw buffered image on it.
Return the graphics of the buffered image.

Canvas.getGraphics:
Walk up the container tree until we hit BotContainer.
Get graphics on the bot container and transform coordinates based on the canvas’s relative position.
Return it.

Canvas.
Dispose the peer.

What I think you’re thinking of:
applet.getComponentAt(…).getGraphics().drawText(…);
