moparisthebest July 31, 2016, 12:18pm #44

phpBB, in my experience, is not nearly as good as SMF, for multiple reasons.

  1. phpBB has always had more bugs (or ‘exploits’) than SMF and they have always been of a more serious nature. In the years since I have been using SMF (since 2006) there has never once been any serious bugs in the software that allowed anyone to gain any staff permissions or anything that affected the forum at all. phpBB has had these kind of exploits.

  2. Last time I checked (granted it has been awhile, it likely has changed since then) phpBB uses md5 as the hashing algorithm for the passwords (fairly easy to crack, using rainbow tables), and sends the passwords in clear text from the browser to the server. SMF uses sha1 hashes, and actually hashes your password with javascript in your browser and only sends the hash over the network, meaning that even if someone is sniffing your network, or even if the server administrator steals your hash, there is no way they can ever get your actual password.

  3. Simply by the structure of phpBB, it is more easily exploitable and prone to bugs than SMF. With SMF, everything is handled through index.php. This means that the same code is used each time to check permissions, input, login status, login and everything else. Also index.php is the single and only entry point into the software, if you want to do something extra, for example make people agree to a TOS, you just edit this one page. However, phpBB (along with vBulletin, same flaw) uses multiple entry points into the application (just a few I could find, search.php, community.php, index.php, post.php, view.php, etc, etc) This means duplicate code that doesn’t have to be there, and more of a likelihood of a bug in this code, AND if there is a bug with anything, it has to be fixed in every single one of those files, making more of a chance it won’t get fixed correctly or even allowing a new bug to be introduced. And good luck if you are going to add a TOS or some other check to your forum, because you have to edit every single one of these files.

I’m sure there are other reasons as well, those are just a few that made me go with SMF (before it was cool to go with SMF ;)) Other things such as look and layout are just preference, I HATE the way phpBB looks, and LOVE SMF, but that doesn’t make one better than the other.
