Several things come to mind immediately:

  1. Running the client as an application is believed to trigger a trap. It did in the first engine and I don’t see why they would remove a way of catching careless cheaters. However, I haven’t ever looked for the actual code.

  2. The client can retrieve reflection objects out of that weird object-tracking thing they’ve got. I don’t remember what it does with them, but you probably know that you can get field/method names and class names out of any reflection object.

  3. IIRC: If you look at the exception code (which calls a “reporterror” method), you’ll see that it essentially transmits a stack trace (containing method/class names, as you probably know) plus method arguments, your character’s name, and your client version to a script on the Jagex web server.

So I think renaming is the big one. I don’t know about access levels. The “original” names will not compile, by the way - some duplicate identifiers end up giving you compiler-stopping numbers of errors.
