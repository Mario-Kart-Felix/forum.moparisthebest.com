chassy13 August 4, 2016, 3:16pm #2

Let’s say you set that rarity value to 9.

NPC_DROPS is a two-dimensional array.

if (npcs[i].npcType == Config.NPC_DROPS[npc][0]) { if (Misc.random(Config.NPC_DROPS[npc][3]) == 0) {

If the npcType is the same as the npcType listed in [npc][0], proceed. [npc][0] in this case is 1265. [row][column]

Now here is what you’re looking for in this next if-statement. [npc][3] is that RARITY value you’re looking at. misc.random(value) is going to return an integer between 0 and value. So if you have misc.random(9), you are going to return a random number between 0 and 9. If you have Misc.random(Config.NPC_DROPS[npc][3]), you’ll return a number between 0 and RARITY.

if (Misc.random(Config.NPC_DROPS[npc][3]) == 0) is saying that if the RANDOMLY CHOSEN NUMBER from 0-RARITY is ZERO, drop the item id in [npc][1] and the amount of [npc][2]. This means a RARITY of 0 will always drop because a random number between 0 and 0 is always 0.

So if you have a rarity value of 9, it is a 1/10 chance the item will be dropped. (because 0-9 is ten numbers. the item will ONLY be dropped if misc.random returns 0).

With this system, it is possible to receive all of the NPC’s drops at one time, although depending on numbers, it will be unlikely. It’s going to use misc.random on every single drop the NPC has because of the for-loop. It’s going to loop through every row of NPC_Drops --> if the dead npc has the same id as [npcs][0], do the misc.random --> if the misc.random is 0, drop the item --> move onto the next row).
