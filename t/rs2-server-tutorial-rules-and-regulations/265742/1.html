Be sure to read these and this before continuing.

New times call for new rules. The following first set of rules are now guidelines that are stressed to be followed. We are going to be considerably more lenient toward new tutorials and considerably more strict toward replies. We want to center our attention on encouraging optimism and squashing out any existing pessimism. Be open-minded about everything that goes on in this section, as it is now a day of innovating that which needs dire innovations.

When Posting a New Tutorial
When you are posting a new tutorial, be sure that it covers all of the following:

[ul][li]It shows some effort put into it – it’s your work, show some pride![/li]
[li]It’s a tutorial relevant directly to RuneScape 2 Private Servers and nothing else.[/li]
[li]Any code snippets provided have been thoroughly explained with careful detail.[/li]
[li]Make sure that the work you post is only yours, unless you have permission to post someone else’s work and accredit them appropriately. Stealing work is not tolerated here.[/li]
[li]You may post a tutorial that has already been created if and only if it surpasses any existing tutorials of its type.[/li][/ul]

When Replying to a Tutorial
When you are replying to a tutorial, bear in mind all of the following:

[ul][li]Accusations regarding theft must be proven. Any arguments created over theft will result in some punishments if they do not have any proof.[/li]
[li]Treat others how you wish to be treated. This must be included in every situation.[/li]
[li]Arrogance and cockiness are no longer welcome here.[/li]
[li]Keep an open mind while reading other tutorials. You may be surprised to learn something new from someone else’s tutorials.[/li]
[li]When posting, be sure to provide constructive criticism in a non-discussion, or by minimum a comment or neutrality. Any negative criticism or flaming will result in punishments.[/li][/ul]

All of the above rules will be enforced at the discretion of a moderator. The first set of rules will be enforced behind-the-scenes using the new filter system. The second set of rules will be enforced on-the-scene using visual warnings. The level of visual warning that is given out depends on the opinion of the moderator.

When you are actually posting a new tutorial, bear in mind that the approval of the tutorial may take several days to complete. If the topic you posted is not spam, you will receive a private message if your topic was denied. The message should include some instructions on how you can improve your tutorial so that it meets our standards. If you receive no private message and your topic was certainly a tutorial that followed all of the rules relevant to it, then it should appear within the Tutorial section at some point.

You may post below if you think any of the above rules can be improved, changed, etc. If you want to suggest new rules or wish to argue the removal of any existing rules, post below. If you request clarification of certain unnamed rules, also post below. Any questions that come to mind regarding any of the above, please post them below. Questions only please, as nothing else is needed.

On a side note: If you want to post any small code snippets or any tutorials you don’t want to explain, post them here as snippets in a pastebin. Posting them as actual topics will lead to them being denied.

-The MoparScape Moderation Team
