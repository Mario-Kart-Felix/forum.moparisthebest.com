So, I know there are already a few tutorials. I’ll explain some of the basics, and I’ll show you how to add Ring of Stone turning you into a Rock, wielding an AGS only if you are a Donator…

Step 1:

Open your client/clientHandler class, and search for:

When you found it, you will see something like this:

if(wearID == ****) {

Okay so :

if(wearID == ****) { npcId = ****; isNpc = true; updateRequired = true; appearanceUpdateRequired = true; }

wearID is the ID of the item you want to use to do something.

npcId, is the ID of the NPC you want to become by wearing an item.

Confirms that you’re an NPC

this confirms that an update|appearance update is required.

updateRequired = true; appearanceUpdateRequired = true;

So, for example. You would like that a ring of stone to turn you into a rock?
It would look like this:

if(wearID == 6583) {//Ring of Stone ID npcId = 2626;//Rock NPC ID isNpc = true; updateRequired = true; appearanceUpdateRequired = true; sendMessage("As you put on the ring you turn into a Rock!");//going to send you a message that by wearing the item you turned into an NPC }

And this is what you will get as Result.

So, now, if you would remove the Ring of Stone you would still stay a Rock. So this is what you have to do now.

in your clientHandler class search for:

add this somewhere in it:

if(wearID == 6583) { npcId = 0;//NPC -> Person isNpc = false;//Confirms that you are not an NPC anymore updateRequired = true; appearanceUpdateRequired = true; }

[size=2em]WEARING ITEMS WITH PLAYER RIGHTS, DONATOR ONLY…[/size]

Example: You want that only Donators can wear an Armadyl Godsword. So it would look like this.
Still in your public boolean wear, add this somewhere in it:

if(wearID == 11694 && (donator != 1)) { //AGS ID sendMessage("You Need To Be A Donator To Wear This Item.");//Sends a message that you have to be Donor to wear it return false;//Wearing Denied }

Pretty easy.
