Hello, I was wondering if it is possible to access libraries without having them be in a jar file. Say I have the source and binaries and I want to be able to edit that code continuously yet still be able to use the library. Is that possible?

Protosstribe August 4, 2016, 1:27pm #2

Yes, if you’re using an IDE like Eclipse, for instance, and you have a project with the code of the library, then you can add your library project to the class-path of your other project.

In Eclipse (Kepler):

  1. Select your project tree and right-click it.
  2. Select “Build Path” and “Configure Build Path…”.
  3. In the “Java Build Path” section of the tree, select the “Projects” tab and click “Add…”.
  4. Then you can select the project you wish to add.
