lothy July 31, 2016, 11:58pm #9

Just a tip: It’s not possible to have a server that requires absolutely no work on the part of the person running it when it comes to being non-specific version wise.
The best you can do is make the packet handling code two-tiered – packet comes in -> sent to packet converter (which is version specific) for decoding into usable variables -> usable variables are sent to packet handlers that are NOT version specific (this is where the functionality can remain static despite the version being played).

So in short, each version will require a type of PacketConverter class which passes arguments to the PacketHandler class. You can distribute version-specific PacketConverter classes in JAR files for any given version, along with codec class you need. The PacketHandlers and and scripts stay the same of course.
