[anchor=classic]Getting Started[/anchor]
So you’re probably wondering, “why do I need this tutorial when there’s sources out there which can give me the information I need?” and that’s good. However, I want to lead and guide whomever reads this into the right direction when they begin their project. A resource of what you need to know, what you need to consider, and some important tips to remember when you begin so that you can confidently begin your project knowing you can tackle the workload.

[anchor=sucks]Things you’ll need to know before you start[/anchor]
So what do you consider when you start a project? Well you really need to lay out some guidelines and in the world of RSPS there are two approaches you can branch from. The first is writing your server from scratch. A popular approach when trying to learn about server-client communication and JAVA as a whole, this approach is taken by a variety of members who in some shape or form have contributed their time to the private server scene. Many servers which started with this approach were not developed alone and had multiple members contributing to the project at one time. When beginning a project from scratch, you need to consider:

[ul][li]Goals you’re trying to reach[/li]
[li]Can you do it alone[/li]
[li]Will you drop the project because of the workload[/li][/ul]

If you cannot do it alone, consider your options with creating a development team by creating a topic here

I will elaborate more on how to approach creating a scratch server in a later [iurl=#league]section[/iurl].

The second option you have when beginning a project is taking an existing source and using it to begin your server. This is the easiest and most common route taken by many servers hosted today and a small portion of the most popular servers created have come from pre-existing sources. Now, you might be wondering, “why should I consider anything when there’s already servers ready to be hosted?”

The thing with taking on a project RSPS is that you do not want to be a copy of another person’s server, otherwise you will lose credibility as a hoster/developer and likely lose players. You cannot simply assume that you can download a server, run it, advertise it, and expect your server to succeed either. With this is mind, you’re probably expected to run into various errors or troublesome bugs because no code or script is completely perfect (unless drubrkletern programs it).

[anchor=at]Essential knowledge to developing a server[/anchor]
The first thing you’re going to want to do is experiment with JAVA (most common language the server is programmed in). There are a handful of websites and sources online and “try it yourself” tutorials floating around the internet. The most common can be found here at oracle.

We also have a giant pack of language learning documents (found in the sticky) and members willing to help with your questions in General Programming.
Don’t forget to check out GameDev if your project doesn’t pertain to RSPS.

Now, you’re got some basic java knowledge in and you can modify basic settings or copy & paste snippets correctly, what else will you need to know? For website related content such as home pages, forums, etc, you or someone on your team will need to familiarize themselves with HTML, PHP, XML, and various other languages in order to integrate systems like hiscores, forums, and customer support. Again, a resource for these languages can be found in the General Programming board and the Web Design/Dev board.
