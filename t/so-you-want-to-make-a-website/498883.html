This is a tutorial on creating your very own website from scratch and you can change it to your own needs and how you’d like it to look, this just gets you started. This tutorial will first talk you through the entire chain of thought of creating your own website and then I will write the most common CSS styling codes you will use and the general rules for them.

So, where do I start my website?
You start by deciding what you want to make your site for and who you’re going to be aiming that site at. So lets say since this community does revolve around RuneScape private servers that we’re making a site for one of those. You’d start by thinking about the main backing of your game, which is medieval type times. Now you can start brain-storming certain things about medieval times. Here’s a few that come to my mind:

[ul][li]Maces[/li]
[li]Brick walls(dark)[/li]
[li]Flames/Torches[/li]
[li]Catapults[/li]
[li]Castles[/li][/ul]

Along-side those a dark-themed site is in need since those times were also known as the dark ages etc. You can make a light themed RSPS site all you want, this is just how I would make it.

The next thing you want to think of is the colour-scheme. Now, a common downfall of many new website designers is they have a large range of colours which clash and don’t work well together, this is a big no-no. The first thing a lot of new users see of your server is the website and the website is the epicenter of the entire server, not the game. Without the website you wouldn’t have a web-client and without a web-client you’d be missing a large part of your player base and those users who use the downloadable client wouldn’t be able to get the link very easily. What I’m getting is that if you don’t have a visually appealing and user-friendly site you’re gonna be struggling a lot more than you usually would. Back to the colour scheme: You’re going to want to choose two base-colours. If I’m thinking of a dark-themed site I would then think of a colour that fits well with black, which is actually a lot of colours. Lets take gold. Black and gold are two very good colours that compliment each other, especially if you’re focusing around an older site, or at least I think so. So you’ve now picked your two primary base colours, your other colours should now be variations of those. So I’ll say a light-ish grey and a more darker coloured gold.

Now you have your colours you can start planning out the layout of your site and the images you’re going to be using inside it. Now, layout is incredibly important, I can’t express this point enough. Layout is key. If you don’t prioritise what you want your users to see first and how you’d want to show them then you’re going to be displaying a lot of pointless things on the homepage. Now, I personally usually add the banner of the site to the middle of the top of the page, add a simple navigation bar, nothing too hard to figure out. How do I put these thoughts into action?
This is where the code comes in. Now, I see people who have went off on their own accord and created their own site and code daily but the problem is it’s almost always full of deprecated code. I’m going to teach you the latest and greatest website design techniques, or at least with the code-side. I’ll provide you with a bare-bones HTML document and then some snippets of CSS code to get you started. Obviously I’ll link you to the CSS documentation too so you can see how you’d use different code for different things and ultimately you can become a competent website developer. Now, just remember HTML is for semantics and CSS is for all styling. Okay, so here’s the barebones HTML document:

<!DOCTYPE html>
<html>
   <head>
      <title>YOUR SITE TITLE</title>
      <link href="css/main.css" rel="stylesheet" />
   </head>
   <body>
   </body>
</html>

Okay, so now you have that you can start building on top of it; save this file as “index.html”. I personally would now start with the CSS file. Before we go any further I’m going to show you a folder structure for an effectively organised website. Here’s how yours should look:

[center]

[/center]

All CSS files go in the css folder, all images associated with the website are included in the img folder and all JavaScript which is used in the creation of the website is saved to. This allows you to easily navigate your folders and find the files you need to edit easily. You should also keep the names in all of your folders lowercase, this allows for easy navigation in the through URL’s. If you are going to include spaces or two words in your folders you should also use a hyphen or also known as a dash (-) between words.

Okay, now, create a file called “main.css” inside the css folder. Here is the link to the CSS documentation by Mozilla. You can also view the CSS Tutorial for beginners by Mozilla.

To style your background of your site you would like to do something along these lines:

body {
  width:100%;
  font-family: 'SomeFont.tff' font type;
  background-image: url(./path/to/your/image.jpg);
}

The above code would your site stretch to the size of even the largest screen resolution, since the width is declared as 100%. You’re then setting your font that you wish to use for the most part of your website. You’re finishing up with setting the image you want set as the background of your site, I’ll explain this code a little further below with the selectors part.

A very crucial part of your site is that you want to aim for multi-resolution support alongside multi-browseral support. The only browser you really have to worry about is Internet Explorer but that in it’s own is a whole topic worth, if you feel you’re struggling with getting your site compatible with IE and Google can’t explain it well enough then feel free to request me to make a tutorial on it and I will, it’s a fairly simple fix but there’s no point in giving you a fix if you don’t understand the mechanics behind it, that way you’re learning and if you merely get handed the fix you won’t be learning, you’ll just be relying on others and not be becoming a competent website designer. Okay, so getting back to multi-resolutions. Obviously not everyone has just a single screen resolution and this can really, really make you lose your sites “umph”. Now, the first and easiest way to create a multi-resolutional site is to just create what’s known as a container. For this you’d use something similar to this:

.container {
  width: 990px;
  margin: auto;
}

Okay, so what this code is doing is declaring that when the div class container is called in the HTML file you’ll have a file width of 990 pixels. This is then centered using the margin-left to be auto and margin-right to be auto. I’ll talk you a little through the construction of this code:

container is known as a selector.
width is known as a property.
990px; is known as the value.

The above stands for all CSS code, replace container with whatever you wanted your selector to be, replace width with whatever property you were using, for a full list of properties take a look at the CSS documentation I linked to above. If you replaced the 990px; value with something else it would still work. Values for sizes can also be called in percentages which is very handy for styling the body feature of your site.

You can see the semi-colon after each value, this is to separate the values from each the next property. Because this starts with a “.” it’s known as a class selector. If the selector began with a #(number sign/hash) it’s known as an id selector, and if it was just the words container it would be known as a tag selector. These are called into CSS like this:

<div class="container">
     This is for the class selector container
</div>

<div id="container">
     This would call the id selector container, in other words if it started with a #. 
</div>

<container>
     This would call the tag selector container, if it didn't have a . or # in front of it.
</container>
Mozilla Doc:

If more than one rule applies to an element and specifies the same property, then CSS gives priority to the rule that has the more specific selector. An ID selector is more specific than a class selector, which in turn is more specific than a tag selector.

You can read more on selectors here.

Pseudo-classes
Just a quick run-through on pseudo classes. These may be a little too advanced for a very beginner, but you must use them eventually if you wish to create a decent website. Basically a pseudo-class is a keyword added to a selector that gives a special state to the element this selector is called to. For example, :hover or :visited are needed to make a decent nav bar. If you don’t use both of these your browser will colour visited links to their default colour, usually a variation of purple. This will not flow with your site.

Here’s the general syntax:

.selector:pseudo-class {
  property: value;
}

Useful and most commonly used pseudo-classes:

[ul][li]:link [/li]
[li] :visited [/li]
[li] :active [/li]
[li] :hover [/li]
[li] :focus [/li]
[li] :first-child [/li]
[li] :nth-child[/li][/ul]

        The box model
        The box model is a very important concept to any competent designer and should be brought in and pumped through you from a very early stage. Not a lot of beginners understand this but it is rather important so try reading over it a few times if you don’t catch on. The box-model is used when people are discussing a CSS based layout, basically any HTML element can be considered a box and this means that the box model applies to all elements. The box model is what specifies the box and its attributes relate to each other. In simple terms, if an element is defined as to have a width of 50px and a height of 50px it should be drawn so. However, more can be added to a box - padding, margins to name but a few. I found this image online to explain it further:

        So, let me talk you through this. A box is made up of 4 main features, you have the outside (margin) which is completely invisible, this has no effect on the elements behind it. You have the next step in (border) this basically gives definition to the part of the element which is on display. We take a further step in to find the padding, this defines the space between the content of the box and the border. We then finally have the content. Keeping up so far? Good! When you’re defining the content of a box in CSS you only actually define the size of the content part, you must define the padding, border and margin separately. Then and only then can the entire size of your box be calculated by the browser. As you can see, the browsers do a lot of work unseen by you. For every element on your page, the browser itself must run this formula:

        box width = content width + left padding + right padding + left border + right border + left margin + right margin

        If you haven’t caught on, re-read above since that’s mostly the important parts of it. Post any queries below.
