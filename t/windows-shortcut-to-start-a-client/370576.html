xryz August 1, 2016, 1:12am #1

ok i didnt think there was a section for this so thats why i put it here. today i was like why hasnt anyone made a windows shortcut to start a client? so i did. the only reason i would think of, why they wouldnt is for debugging. b/c after the client closes so does the command prompt so if u had an error you wouldnt be able to read what it was. so if you want to know how to do it instead of using a batch file this is how you do it.

  1. create a windows shortcut from either copying a shortcut from the desktop or creating a new one from the right click menu.

2.a if you copied the shortcut then rename it to “run” or w/e you want it to be called. open the properties of the shortcut from the right click menu. in the text box for the “target” change its contents to match what works with your computer and client:“C:\Path\to\your\jre” main class of the client the arguments for the main class an example would look like: (“C:\Program Files (x86)\Java\jre6\bin\java.exe” -Xmx1500m Gui 30 0 highmem members 32) After you have that information in the target box make sure the text box for “start in” is empty. then make sure the shortcut is in the same directory as the main class. [edit](Note: if you want to have the shortcut in another directory, then put the absolute path of the client class files directory in the text box “start in.”)

2.b if you made a new shortcut, then follow the wizard to create the shortcut. then follow the steps of 2.a.

3rd. that should be it and you can use this method with other java application.
