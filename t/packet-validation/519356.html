two things that aren’t checked in packet processing are appearance and objects.

appearance: is the appearance a valid appearance (allows for users to go completely invisible/custom appearances)
objects: use any object from anywhere (prayer in wild (also heals you on most servers), banking, and majority of skills, portals for teleporting etc.)
region object loading: spawn objects on the game per region, not globally (reloading the region will erase the object though)

some resource skills can be used to spawn items anywhere (woodcutting, mining)
skills can be used anywhere + with autotypers/clickers (runecrafting, fishing, thieving)
objects can be abused in pvp (prayer healing, quick teleports, prayer switching, magic switching, etc)
though these can be checked and validated, switching interfaces can still be done client sided. so you will need to do some
checks upon activating prayers/spells and checking which book is currently being used.
appearance can be glitched to show you as another player (chat packet even shows their name), or custom appearances (invisible,
minecarts, objects, npcs, etc.)

as well with the majority of packets not being properly validated, the content is horribly written. the combat contains dupes, the skills contain dupes, the minigames contain dupes, and the server is a memory-eater.

all-in-all, if you are only in the scene to make money, don’t expect to be punishing abusers (i do love to go on random servers and do this. i have a custom client, but i can also just download yours and readd my commands)

so yeah. if you’re gonna host a server, either take the time to properly check things, or don’t complain when players do it. when someone finds a bug/dupe/glitch, it gets leaked extremely fast
