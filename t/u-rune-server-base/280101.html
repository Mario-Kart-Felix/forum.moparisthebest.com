Added class files

  • Combat class. (attacking NPCs, attacking players, special attacks, weapon delays, etc.)

  • ClickNPC class. (NPC clicking: talk to, trade with, etc.)
  • Dialogue class. (NPC dialogues: chatbox.)
  • Command class. (commands.)
  • Save class. (player saving and loading.)
  • ItemOnItem class. (item on item, fletching, etc.)
  • Bank class. (banking methods.)
  • Run class. (incomplete.)
  • ObjectClick class. (clicking objects, bank booths, doors, etc.)
  • EventManager, EventContainer and Event class. (see here: info)
  • ItemRandom class. (used for NPC drops, clue scrolls, etc.)
  • Special class. (handles the special bar graphics, reduces class Client quite a bit.)

Misc.

  • Class Client is 193kb.

In-game features

  • Created a range base.
  • Beautiful PVP.
  • NPCs no longer walk away during combat.
  • Weapon timers. (eating food will add a 1000ms delay to your next attack.)
  • Character screen on first login.
  • Few Lumbridge NPCs
  • Basic teleports
  • Player cap of 50 (see below on how to change)
  • Plenty debugging for easy editing.
  • Anti-null added.

Changing the player cap
In your class Client, find:

/*Max players allowed online*/ public int max_players_accepted = 50;

Change the ‘50’ to your desired max player amount.

How to remove the set client
In your class Client, search for:

if (junk != expectedUid) {

and remove all of this:

if (junk != expectedUid) { returnCode = 11; disconnected = true; nullCheck(); println(playerName + ": Invalid client. - Player disconnected."); return; }

To add NPC drops
If you’ve ever used Project Czar, then you’ll see that you add drops the exact same way, just using the ItemRandom class.
