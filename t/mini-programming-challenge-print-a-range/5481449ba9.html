z:

Is it possible for a function to have its own stack frame? If that question makes any sense.

Yes and no. You cannot run the first iteration of the recursive function in its own thread, but you can create a new Thread for the rest of iterations and set a name for the thread so it doesn’t make new Threads (By checking the name of the thread you are in).

Pure’s solution “rewritten” to avoid problems with stack position, as long as no one else make a thread named “t4schallenge” and run f(n:Int) in it at the wrong place in the stack. But here you have a problem if you run f(X) twice in a row. I guess you can block two threads from accessing f(n:Int) at the same time, I just have to find out how to do it in Scala.. Fixed.

	def f(n:Int):Void = {
			this.synchronized{
				if(Thread.currentThread().getName() == "t4schallenge"){
					if (Thread.currentThread().getStackTrace().length == n*2 + 2){
						return null;
					}
					if (Thread.currentThread().getStackTrace().length < n + 3) { // print downwards
						println(Thread.currentThread().getStackTrace().length - 2);
					} else { // print upwards
						println(n - (Thread.currentThread().getStackTrace().length - 2 - n));
					}
					f(n);
				}else{
					val t = new Thread("t4schallenge"){
						override def run = {
								f(n);
						}
					};
					t.start();


				}
				return null;
			}
	}

void f(int x)
{
        if (x > 0)
        {      
                x = -x;
                printf("1\n");
                f(-2);
                printf("1\n");
        }
        else
        {
                if (PREVIOUS_FRAME(&x, 1) <= x)
                {
                        printf("%i\n", -x);
                        if (PREVIOUS_FRAME(&x, 1) != x)
                        {
                                f(PREVIOUS_FRAME(&x, 1));
                                printf("%i\n", -x);
                        }
                }
                else
                {
                        printf("%i\n", -(PREVIOUS_FRAME(&x, 1) - 1));

                        {
                                f(PREVIOUS_FRAME(&x, 1) - 2);

        }
