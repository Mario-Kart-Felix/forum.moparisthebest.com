Not sure what to call this but I was recently messing around with the client and moparclassics ability to switch servers which led to this idea.

Your username, password and stats will be the same on all servers. You actually have the ability to choose which server you want to train on. Instead of servers where they battle for players at the same time ripping the community apart… players will have the ability to choose and not lose their stats or bank be screwed over by people hosting.

Now there is a flaw to this and that is people skilling localhost with a 1000x exp rate and basically loading up. I am sure we can do something where we isolate the core server and just allow plugins that the community approves of. Anyone can run the server and choose which plugins they allow on their server.

Why would you become a host?
Maybe if you provide a feature another server doesn’t you can “charge” players game gold to play. Hosters can use that wealth to purchase new plugins from developers, spend it on items or whatever you can think of.

Rodgerwilco August 4, 2016, 11:30am #3

Lothy, I think it would be much more simple than that with a plugin system as the control system. It would work like this…

Anybody can download the “server” and when they run it it will ask them or load from a file their information (ip, port number) and let them know the steps taken to become a hoster. The owner can then choose appropriate plugins that fit what they are looking for. Plugins would range from mini games, monster locations, portals… content, whatever. Everything is predefined so you can’t freely change the exp rate or what items monsters drop unless it’s approved.

The problem and only problem I can see is someone hosting a server and people connect with bot clients or some form of macro and they basically free load. I am sure there is some way to battle this.

Tbh, I believe this is the solution to ‘fixing’ the community. It’s for the players and only benefits them. There will always be competition between servers and a while back RSCE, DQ, RSCA would allow you to transfer your stats from one server to the other. Players won’t feel obligated to train every few months because their characters will still be intact without the server drama.

lothy August 4, 2016, 11:30am #4

They don’t even need to freeload. The server just needs to tell the central server that however many NPCs have been killed by player X. Basically a shard in this could tell the central server that player X has gained 50k experience in the last half hour, and who is the central server to say otherwise?

The same with item drops. Say they can’t change the drops. However they could change the server such that NPCs frequently drop a known rare item. Again, it’s the same situation. There’s no actual way for the central server to prove or disprove the authenticity of the item.

The only way to know for sure that item drops and experience are authentic is if there’s constant back and forth between the central server (which is the authority over all shard servers) and the shard server.
