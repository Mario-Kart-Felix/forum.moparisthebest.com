I’ve always wanted to replicate RSC’s sleeping screen but there’s literally no way to 100% replicate their captcha by generating it yourself as there’s thousands (maybe even millions) of words and each word can be generated in different fonts and styles meaning there’s countless combinations. It then came to me that I could collect the RSC captcha images and send them as an actual image rather than generating from scratch.

I set about collecting RSC captcha images (I’m at about 15,000+ now and counting) and have saved them as PNG images. The file name is the actual word itself. To check the theory worked well they were all stored as separate files and the server would randomly pick one when someone sleeps. If the word sent back matches the file name then it’s a match and they have successfully slept.

The images are now being stored in a MySQL database, one entry per image. I’ve always liked this method of binary storage. The structure is simply:

CREATE TABLE `RSCEmulation Tools`.`captcha` ( `captcha` BLOB NOT NULL, `string` VARCHAR(20) NOT NULL ) ENGINE = INNODB;

For those of you that don’t know what the BLOB data type is, it’s short for “Binary Large Object” – it’s designed for storing binary data.

We can then either loop through them all at once upon server start-up or run one simple query every time a sleep word is requested. Each query takes 0.006 seconds to execute.

I intend on adding hundreds of thousands of images to make it as random as possible. I know there’s numerous OCR’s for RSC, I’m clearly not trying to make it difficult for bots in this case but merely continue replicating RSC. The sleeping screen was a big part of RSC and it’s always annoyed me that on every single private server to date it was a tacky screen. Regardless of that, it’s an improvement on our current captcha.
