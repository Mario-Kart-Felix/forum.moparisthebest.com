Who would like to help out in porting the RSC client onto the Android mobile platform? i have a small amount of experience with android development and after creating an android project and importing all the RSC code there, obviously there’s many errors, but going through them it seems like it’s not so far fetched to get something going.

the only external libraries the client uses is Xstream/XPP3 for XML loading. XmlPullParser (XPP3) is available in the android api and Xstream have an android version.

a large amount of the client will function completely fine once the main libraries have been converted to android ones.

That’s just to get the project to compile, after that the entire input system would need to be rewritten for touch events. game would need to scale for different screen densities etc.

I can see some great potential for having an Android Client, it would boost the RSC pserver scene allowing people from anywhere on any android device to log on and play. i wouldn’t expect people to be PKing on it, but theirs nothing like sitting on a train and mining/fishing on your phone/tablet.

It would also be open source, people can fork it and modify it to their needs for a connection to their own servers.

anyway, if enough interest is generated in this thread ill throw it up on github for everyone and myself to contribute towards.
