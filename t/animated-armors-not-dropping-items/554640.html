X-J-K-X August 4, 2016, 5:17pm #2

Could be a few things.

In the standard server, npc drops and constant drops are handled in different files. So make sure you’re checking both. Also make sure you have the correct id for the armours, or of course they won’t drop any items.

The “bones and coins” could be a default drop in your server for something that isn’t defined.

Make sure that there is no exception being thrown when dropping the item, this can cause no drops.

The ids that I used were 4278-4284

sk8rdude461 August 4, 2016, 5:17pm #3

Don’t use your drop system to add their drops.
Add in code to where the death handler is for the npcs.
The reason I say not to use your drop system is because they actually aren’t supposed to have any drops… Not even bones (It’s animated armour… how would it have a skeletal system?).
Another reason why is because there’s also supposed to be a chance you lose a part, or all of, the armour you used in the animator.

(A poor) Example:

if(npcs[i].npcType == animated_amour_id) { Server.itemHandler.createGroundItem(the_player, armour_id, etc, etc...); }
