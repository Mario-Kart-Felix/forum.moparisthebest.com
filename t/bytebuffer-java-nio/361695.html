mod_taharok August 1, 2016, 12:08am #4

It’s basically the same concept. The stream class allocates one array of bytes and one pointer, used for both reading and writing. ByteBuffer does the same thing, with some additional functionality. The reason a ByteBuffer can’t practically replace the stream class is because of the special datatypes which are sent when using the stream class. There are ugly ways around this, but generally it’s no good. As Miss Silabsoft said, you’d most likely use a ByteBuffer at the lower levels of a NIO server or when using Apache MINA. If you use Netty, you’d probably be working with a ChannelBuffer, which is even cooler in my opinion.

Hope this helps.
