Krichevskoy July 29, 2016, 10:06pm #7

The page you’re talking about is a copy of Kaitnieks’s autorune history site. Packet editing programs (like AutoRune) are possible, but it was such a successful cheat back during Classic that Jagex put a lot of effort into killing it specifically.

First, there’s encryption (this didn’t exist when Kaitnieks wrote the first version of AutoRune).
To interpret game commands, you have to have a special random number generator synchronized with the one the server is using (the packets come to your client scrambled). In order to synchronize your client’s number generators, you have to have the session keys. You can only get the session keys in two ways:

  1. Intercept the login packet and crack RSA encryption.
  2. Hack the client to output session keys.

The first option is not practical. Although evo did maintain AutoRune for a long time and thus had to figure out a way around all this stuff, he was being paid lots of money to do so. No one has that kind of motivation at the moment. I think he modified a client to send the session keys to AutoRune rather than cracking the RSA, but that brings us to the next option.
My point here is that if you’re already hacking the client, why bother with packets?

Second, Jagex changes the protocol slightly with every update - they randomly change the packet IDs and sometimes add bogus packets. The result is that the protocol works the same, but looks different. It’s possible to update a program to use new packet IDs. I guess it would be no more of a pain than writing an autoupdater for a modern java bot, but for a long time it would have been 10x harder than writing a java bot. So it became one of those “just isn’t done” things.

popcorn89 July 29, 2016, 10:38pm #16

It’s not that simple. They switch between little-endian and big-endian ints and scramble packet info, and other stuff like that. For example, let’s say in revision 423 the packet that sets the chats looks like this:

public chat (big-endian) private chat (little-endian) trades (little-endian)

The next revision, it might look something like this:

trades (little-endian) public chat(little-endian) private chat(big-endian)

So you’d need to do some extensive updating every time there’s an update. You’re a lot better off just making a normal bot.
