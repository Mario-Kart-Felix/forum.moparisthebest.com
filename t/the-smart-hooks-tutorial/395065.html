The reflection library is almost completely based off of hooks returning the values of fields from the RuneScape client. In order to be able to make your own reflection functions, you need to first understand how to retrieve field data and use it in your script. This tutorial will cover that, and hopefully make it easier for some to understand reflection.

[CENTER][SIZE=24pt]The Hooks[/SIZE][/CENTER]

So, what are they? Every one of these PascalScript constants represents a field or method in the RuneScape client. The string of letters you see them assigned to are the obfusicated names of the fields. Jagex obfusicates their client so that bots are a pain in the ass to create and maintain, but that is why we have hook updaters. I won’t go further into what fields are in this tutorial, if you want to know more you can read my Introduction to Client Hacking for the Java Impaired.

These hooks aren’t of much use to you if you don’t know what java data type they represent, so you will have to find out The RuneScape client has data types that we don’t commonly use in scripting, so here’s a quick summary of each data type from the Java Tutorials.

[QUOTE] byte: The byte data type is an 8-bit signed two’s complement integer. It has a minimum value of -128 and a maximum value of 127 (inclusive). The byte data type can be useful for saving memory in large arrays, where the memory savings actually matters. They can also be used in place of int where their limits help to clarify your code; the fact that a variable’s range is limited can serve as a form of documentation.

short: The short data type is a 16-bit signed two's complement integer. It has a minimum value of -32,768 and a maximum value of 32,767 (inclusive). As with byte, the same guidelines apply: you can use a short to save memory in large arrays, in situations where the memory savings actually matters.

int: The int data type is a 32-bit signed two's complement integer. It has a minimum value of -2,147,483,648 and a maximum value of 2,147,483,647 (inclusive). For integral values, this data type is generally the default choice unless there is a reason (like the above) to choose something else. This data type will most likely be large enough for the numbers your program will use, but if you need a wider range of values, use long instead.

long: The long data type is a 64-bit signed two's complement integer. It has a minimum value of -9,223,372,036,854,775,808 and a maximum value of 9,223,372,036,854,775,807 (inclusive). Use this data type when you need a range of values wider than those provided by int.

float: The float data type is a single-precision 32-bit IEEE 754 floating point. Its range of values is beyond the scope of this discussion, but is specified in section 4.2.3 of the Java Language Specification. As with the recommendations for byte and short, use a float (instead of double) if you need to save memory in large arrays of floating point numbers. This data type should never be used for precise values, such as currency. For that, you will need to use the java.math.BigDecimal class instead. Numbers and Strings covers BigDecimal and other useful classes provided by the Java platform.

double: The double data type is a double-precision 64-bit IEEE 754 floating point. Its range of values is beyond the scope of this discussion, but is specified in section 4.2.3 of the Java Language Specification. For decimal values, this data type is generally the default choice. As mentioned above, this data type should never be used for precise values, such as currency.

boolean: The boolean data type has only two possible values: true and false. Use this data type for simple flags that track true/false conditions. This data type represents one bit of information, but its "size" isn't something that's precisely defined.

char: The char data type is a single 16-bit Unicode character. It has a minimum value of '\u0000' (or 0) and a maximum value of '\uffff' (or 65,535 inclusive).

[/QUOTE]

Now that we have that out of the way, we need to find that field! Here’s a step by step process on finding the variable you need and its data type.

Step 1: Download the runescape client jar from here. You also need a java decompiler, you can get one here.

Step 2: Look in hooks.simba and find the class of the field you are looking for. I will be looking for the hook “GetX”.

{Class ObjectInterface} class_ObjectInterface = 'sha'; // The class containing my hook hook_interface_GetActions = 'I'; hook_interface_GetX = 'v'; // My hook hook_interface_GetY = 'mc'; hook_interface_GetWidth = 'rd'; hook_interface_GetHeight = 'Ac'; hook_interface_GetText = 'dd'; hook_interface_GetTextureID = 'x'; hook_interface_GetTextColor = 'fd'; hook_interface_GetModelID = 'r'; hook_interface_GetModelZoom = 'Tc'; hook_interface_GetName = 'nc'; hook_interface_GetChildren = 'h'; hook_interface_GetID = 'tb'; hook_interface_GetStack = 'W'; hook_interface_GetParentID = 'X'; hook_interface_IsHidden = 'Yb';
If your hook was static, then the letters preceding the dot are the class name, and the letters after are the field name.

Step 3: Now open up the java decompiler you downloaded. Click File -> OpenFile, and find where you saved the client (runescape.jar). Open that up.

Step 4: Scroll down the left panel until you see the class containing your field, then click it to open it.

Step 5: Now that you have your class open, find the field associated with your hook. Here’s mine:

Step 6: The variable assigned to your field is right before the variable name. In this case, mine is an int.

Now, according to the data type of your hook, you will have to use the function that matches up with it. Since “GetX” was an int, I will have to use SmartGetFieldInt. If your variable had a another class’ name as its type, then you will use SmartGetFieldObject. There are also functions for arrays and multidimensional arrays up to 3D, so every variable type is covered.

So let’s say I want to write a function that will return an interface’s X coordinate using reflection. I have the hook from earlier, so here is what I would do.

function GetInterfaceX(parentIndex, childIndex): Integer; var i: Integer; begin i := GetInterface(parentIndex, childIndex); Result := SmartGetFieldInt(i, hook_interface_GetX); SmartFreeObject(i); end;

Don’t pay attention to the parentIndex and childIndex I’m using to grab the interface, the point I am trying to demonstrate with this snippet is the meaning of the parameters of the “SmartGetField” functions. The parent in this case is the interface ID, the parent is what you are trying to grab the data from. Also take note of the “SmartFreeObject” I have at the end of the function. If you don’t include this, you will have created a memory leak and SMART will crash.
