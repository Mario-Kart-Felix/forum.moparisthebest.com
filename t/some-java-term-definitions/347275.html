[size=12pt]Chapter 1 [/size]
• Java has a free-form layout, (doesn’t matter how many spaces or lines you make your statements). However, it is case sensitive. (lower-case needs to be in lower-case, upper case needs to be in uppercase).
• Classes are the fundamental building blocks of Java programs. (more on classes, method and objects in chapter 2)
• Each class contains definitions of methods. Each method contains a sequence of instructions. (more on definitions later)
• Every Java application contains a class with a main method. When the application starts, the instructions in the main method are executed.
o For example: look at this statement System.out.println(“Hello”); with the method println.
o What this statement does is print out “Hello” in the console display window.
o The display window or System.out is the object.
 “out” is an object, in this case, the display window.
 “System” is a class file that describes where the object out is found.
 “println” is the method name.
 “Hello” is an input of the method println, it is called a parameter of the method “println”.
o A method is called by specifying an object, the method name, and the parameters.
• A syntax error is a violation of the rules of the programming language. The compiler detects syntax errors.
• A logic error causes a program to take an action that the programmer did not intend.
o Programs called debuggers help find logic errors or bugs.
• An editor is a program for entering and modifying text, such as a Java program.
o Notepad, Textpad and word can all be used for this task.
• A Java compiler translates source code into class files that contain instructions for the Java virtual machine.
o The compiler translates the statements that you wrote in an editor (aka the source code) into class files.
o Class files have the .class extension and consist of virtual machine instructions and other information that is required for execution.
o The class files have only the instructions that you wrote. Often times, that alone is not enough to run your program. Your class files will often need instructions from other class files written by someone else. These class files compose Java’s huge library.
 A library is a collection of code that has been programmed and translated by someone else, ready for you to use in your program.
o The Java Virtual Machine loads program instructions from class files and library files.

[size=14pt]Chapter 2[/size]
2.1Value types.
• -In java, every value has a type.
o “hello” is a value that has type string.
 A string is a sequence of characters contained in quotation marks.
o The integer 100 is a value that has type int.
o It is an error to store a value whose class does not match the type of variable.
 E.g. String variablename =”Hello”; is correct
 E.g. String variablename = 100; is incorrect, because the value 100 is type int.
• You use variables to store values that you want to use at a later time.
o Variables in Java have to be defined and initialized first before you can use it in your program.
 Define the variable: int thetimeofday;
 Initialize the variable: thetimeofday = 50;
 You can define and initialize a variable like this: int thetimeofday =50;
• The name of the variable is called an identifier.
o E.g. String variablename =”Hello”; variablename is an identifier.
• An identifier is the name of a variable, method, or class.
o The rules for identifiers are the following.
 Identifiers can be made up of letters, digits, and the underscore (_) and dollar sign ($) characters.
 They cannot start with a digit.
 Other symbols are not allowed such as ?, % ! etc…
 No spaces
 No reserved words can be used for identifiers, such as public, because they hold special meaning in Java.
 Identifiers are case sensitive. The identifiers, Variablename, variablename, VarIaBlename are all different identifiers and can store different variables.
 But this doesn’t mean that you should. Good programming notations dictate that you should make variable and method names start with a lowercase letter and class names start with an uppercase letter.
• Use the assignment operator (=) to change the value of a variable.
2.2 Objects
• An object is an entity that you can manipulate in your program by calling methods.
o System.out.println(“Hello”);
 You manipulate the object System.out, by calling the method println. This tells the Java Virtual machine to print the word Hello in the system output (aka, the display window).
• A class defines the methods that you can apply to its objects.
o “Hello” is an object that belongs to the class string.
o System.out is an object that belongs to the class PrintStream.
o String greeting= “Hello”;
o int n= greeting.length();
o For the object "Hello”, method such as greeting.length() can be applied. This method get the length of the string greeting. Which is the length of Hello. So n= 5.
o Another method that can be applied to objects that belong to the String class is toUpperCase().
o The statement:
 String greeting= “Hello”;
 String biggreeting= greeting.toUpperCase();
o This makes the variable biggreeting “HELLO”.
o The methods for the class strings cannot be applied to the class int or the class PrintStream and vice versa.
o The class of an object defines what methods can be applied to it!
• The method forms the public interface of the class.
o This tells you what you can do with the objects.
• A class also defines a private implementation, describing the data inside its objects and the instructions for its methods. Those details are hidden from the programmers who use objects and call methods.

2.3 Method Parameters and Return Values.
• Some methods require inputs that give details about the work it needs to do.
o A parameter is an input to a method.
• Implicit vs explicit parameters:
o Considering the statement: System.out.println(greeting);
 The greeting parameter is an explicit parameter of the println method.
 The object System.out (aka, the display window), is an implicit parameter of the method println.
 The implicit parameter of the method call is the object on which the method is invoked.
 Some methods need many explicit parameters, other needs no explicit parameters.
• The return value of a method is a result that the method has computed for use by the code that called it.
o For example, the method “length” returns the value of the length of characters in a string.
o The statement: “int n = greeting.length();” , where the number of characters in greeting or the return value of the method length is stored in the integer n.
o Not all method has a return value.
o Does the print.ln method have a return value? No, because print.ln only interacts with the console window and gives instructions to display texts, it does not have a return value.
o You can use the return value of a method, (aka, the results of a method call) as a parameter for another method.
 For example: System.out.println(greeting.length());
 greeting.length() gives the value 5, 5 is the return value of the method greeting.length and also the explicit parameter of the method println.
• When a method is defined in a class, the definition specifies the types of the explicit parameters and the return value.
o For example, the String class defines the length method as public int length()
• The type of the implicit parameter is the class that defines the method. (more on methods and method definitions later)
• When a method returns no value, the return type is declared with the word void.
• For example the method println, which have no return value is defined by PrintStream class as “public void println(int output) or public void println(String output).
o The class PrintStream defines 2 methods with the same method name (println). One can print string types, the other integer types.
o Method names that refer to 2 or more methods are called overloaded.
 The method name println is over loaded.
Number types:
• Numbers in java can be integer types which are whole numbers or floating-point numbers, which contain fractions.
• The double type denotes the floating-point numbers that can have fractional parts.
o double fractionarecool= 1.4353
o int integerarethebomb= 5
• int, double are primitive types, not classes. Numbers are not objects, number types are not classes.
• Numbers can be combined by arithmetic operators such as +, -, *, /
2.4 Constructing Objects:
• Use the new operator, followed by a class name and parameters, to construct new objects.
o E.g. Rectangle box = new Rectangle (5, 10 ,20, 30);
o Rectangle is the name of the class from the java library. (you needed to define the type of variables before you can use them.)
o box is the variable where with the type class.
o new is an operator and makes a Rectangle object. (note, not an actual rectangle, but just a set of numbers that describe a rectangle.)
o (5, 10, 20, 30,) are construction parameters , they describe a rectangle at coordinates x= 5, y =10, that is 20 pixels in width and 30 pixels in height.
• Constructor are not like methods: the following will result in a syntax error:
o Box.Rectangle(20,35,20,30);
o You have to make a new one “box= new Rectangle(20, 35, 20, 30);”
• Accessor vs Mutator methods
o An accessor method does not change the state of its implicit parameter. A mutator method changes the state.
2.5 Test programs:
• Determining the expected result in advance is an important part of testing.
• Java classes are grouped into packages. Use the import statement to use classes that are defined in other packages.
o Example: you need to import the rectangle class to be able to construct a rectangle.
o To do that write: “import packageName.ClassName;” in this case: “import java.awt.Rectangle;”
2.6 API documentation
• The Application Programming Interface documentation lists the classes and methods of the Java library.
2.7 Object References:
• A variable whose type is a class does not actually hold an object, it hold the memory location of an object.
• Object reference denotes the memory location of an object.
• When a variable that contains the memory location of an object, we say that it refers to an object.
o For example, in the statement “Rectangle box = new Rectangle (5, 10 ,20, 30);”
o The new operator returned a reference of the new object and that reference is stored in the variable box.
 Hint: the new object is (Rectangle (5, 10 ,20, 30))
• Multiple object variables can contain references to the same object.
• Number variables store numbers. Object variables store references.
o What this means is that if you make a copy of an object variable, the 2 variables refer to the same object. So if you change the object, both variables are also changed, since they refered to the same object. But number variables will actually store numbers.
• Object variable are references because objects takes up more memory than numbers, using object variables to store memory location takes up less memory than objects stored in variables.

• a black box is a device, system or object which can be viewed solely in terms of its input, output and transfer characteristics without any knowledge of its internal workings.
• An object has an internal structure-perhaps, sometimes composed of other objects. For runescape players, their black boxes are objects which made your client. For beginning developers, black boxes might be the base used to construct the server.
• For example, a computer user uses a computer program which are constructed by objects (black boxes to him or her) designed by application programmers. In turn, the application programmers uses object designed by other application programmers. And in turn, those objects may be designed by system programmers.
• In object oriented programming, you will design objects that you use to construct other objects and you those to develop your applications. In other words, you will be the application programmer and also the system programmer at the same time.

3.1 Specifying the public interface of a class
• In order to implement a class, you first need to know which methods are required.
• A method definition contains an access specifier (usually public), a return type, a method name, parameters, and the method body.
o Example: public void eatpie (double pieces) {method body}
o The method name is eatpie and it is an accessor method. Therefore, the The return type is “void”, which means no return value.
o Public is the access specifier
o (), whatever are in brackets are parameters of the method eatpie.
 A parameter of a method definition must have a name and a type. In this case, double is the type and pieces is the name of the parameter.
• The constructors contain instructions to initialize objects. The constructor name is always the same as the class name.
o A constructor is similar to a method, except for 2 differences. Constructors have no return type, (not even void) and the name of the constructor is always the same name as the name of the class.
• Just like methods, a constructor also has a body, which are a sequence of statements that is executed when a new object is constructed.
• Constructors can take on different parameters, that is how they are differentiated since all constructors in the same class have the same name. (more on constructors later)
• The public methods and constructors of a class makes up its public interface.
3.2 Commenting the public interface
• A documentation comment is placed before the class or method definition that is being documented. It starts with a /**, a special comment delimeter used by the javadoc utility. Then you describe the method’s purpose. Then, for each method parameter, you supply a line that starts with @param, followed by the parameter name and a short explanation. Finally you supply a line that starts with @return, describing the return value. You omit the @param tag for methods that have no parameters, and you omit the @return tag for methods whose return type is void.
• The javadoc utility copies the first sentence of each comment to a summary table in the HTML documentation.
• Use documentation comments to describe the classes and public methods of your program.
• Provide documentation comments for every class, every method, every parameter, and every return value.
3.3 Instance fields
• An object stores its data in instance fields.
• A field is the storage location inside a block of memory.
• An instance of a class is an object of the class.
• The class declaration specifies the instance fields. An instance field declaration consists of:
o An access specifier (usually private)
 Private means that the method can only be accessed by methods of the same class.
o The type of the instance field (such as double)
o The name of the instance field (such as balance)
• By making instance fields private, all data access must occur through public methods.
• They are effectively hidden from the programmer who uses a class.
• The process of hiding the data and providing methods for data access is called encapsulation.
3.4 Implementing Constructors and Methods
• Constructors contain instructions to initialize the instance fields of an object.
o Often times you will need 2 constructors, one that sets all fields to a default and one that sets them to user-supplied values.
• Use the return statement to specify the value that a method returns to its caller.
o The return statement instructs the method to terminate and return an output to the statement that called the method.
