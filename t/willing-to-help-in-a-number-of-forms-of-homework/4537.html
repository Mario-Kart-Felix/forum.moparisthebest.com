Let n be a 5-digit number, and let q and r be the quotient and remainder respictively, when n is divided by 100. For how many values of n is q + r divisble by 11?

Wow interesting problem… First thing I think to do is create algebric equations.
Our number n can also be represented in terms of its seperate digits: ab,cde
If we were to divide ab,cde by 100, q would equal abc and r would equal de.
q = abc = a100+b10+c
r = de = d10+e
We can get this by looking at the individual place holders
q+r = a100+b10+d10+c+e = a100+(b+d)10+(c+e)1
Now that we have some concept of what we’ll be getting from q and r we should look at mulitples of 11.
11, 22, 33, 44, 55, 66, 77, 88, 99
121, 132, 143, 154, 165, 176, 187, 198, 209, 220, 231, 242, 253, 264, 275, 286, 297, 308, 319, 330, 341, 352, 363, 374, 385, 396, 407, 418, 429, 440, 451, 462, 473, 484, 495, 506, 517, 528, 539, 550, 561, 572, 583, 594, 605, 616, 627, 638, 649, 660, 671, 682, 693, 704, 715, 726, 737, 748, 759, 770, 781, 792, 803, 814, 825, 836, 847, 858, 869, 880, 891, 902, 913, 924, 935, 946, 957, 968, 979, 990
Since n is a 5 digit number, a must be greater than 0 and less than 10, thus the first 9 multiples of 11 need not be considered and we may stop at 990.
After realizing this, we must look at all combinations of a,b,c,d and e that will cause the expression a100+(b+d)10+c+e to equal any of the above numbers.
To do so, look at each digit individually and consider the possibilities. For example:
121 = 1100+210+1; a = 1; b+d = 2; c+e=1
There is only one possibility for the first equation:
1) a = 1
b+d = 2 can occur in 3 ways:
1) b = 1 and d = 1; 2) b = 0; d = 2; 3) b = 2; d = 0;
c+e can also occur in 2 ways:
1) c = 0; e = 1; 2) c = 1; e = 0;
Thus there are 6 possibilities: 32 = 6 (the total number of pairings possible for each digit).
To calculate the final answer, you would have to go through each number and do this. Once you’ve looked at the possible pairings for each digit, it will become quick.
You know that for 1 there are two possibilities when considering 2 numbers, 3 for 2 and you can easily figure out the rest.
I hope that helped.

Edit: After giving it a little thought I realized that the number of combinations that can be used to form any number from the sum of two others numbers is simply n+1. For example, there are 6 different ways to have a sum of 5 from only two numbers if order matters:
0,5; 5,0; 1,4; 4,1; 2,3; 3,2.
Since I have so much time on my hands, I went ahead and made a java code to count it up for you:

class Solved{
	public static void main(String[] args){
		int totalcombos = 0;
		int tensdigit, onesdigit;
		for (int i = 11; i<=90; i++){        //Going from 121 to 990
			tensdigit = ((11*i)%100)/10; //Finds tens digit
			onesdigit = 11*i%10;	     //Finds ones digit
			totalcombos = totalcombos + (tensdigit+1)*(onesdigit+1);
		} //Keeps track of the number of combos^
		System.out.println(totalcombos);
	}
}
