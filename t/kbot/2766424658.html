BAsh July 31, 2016, 7:48am #33

I assume that you load, run, display and the human interacts in such a way that it is not detectable. The uid file is taken care of.

I don’t see how pure reflection is detectable. You are just using the getDeclaredField/setAccessible/get methods which I can’t see as being an issue.

One potential issue is holding a reference to an object which should be garbage collected. The call to the finalization method is unpredictable although a System.gc() could free it. Its been a while but I don’t see any finalization methods that we should be worried about. Even then use a WeakReference and you’ll be fine.

A non-issue is the setAccessible method. You actually get a copy of the Field object and setAccessible only modifies the current object so isAccessible isn’t going to return anything meaningful.

Final issue is breaking the sandbox with getFrames, findClass(“Bot”) and so forth but these aren’t related to reflection at all.

So how is using reflection detectable?
