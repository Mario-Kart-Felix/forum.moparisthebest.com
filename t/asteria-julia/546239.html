[size=16pt]Introduction:[/size]
About the previous Julia:
Some might have passed by Julia, the server for beginners I made.
That was more of an experiment that in the end turned out pretty good.
Though it was still buggy, mainly due to the shitty deathlypvpz base I used.

About this julia version:
This time I’m basing on Asteria which is in my opinion a pretty decent base, and I’m sure many will agree with that.

Don’t expect just a remake on a different base, this is going to be a much more advanced version.
While the other version was buggy and was still not simple enough for real beginners (based on my inbox),
that will not be the case in this version.
While code was also just reworked to fit with the system but shitty as fudge, I will not let that happen with this version.
I will do my best on writing decent code, and keep Asteria in honor.

So what do you mean with more advanced?
First of all, ability to edit more stuff and customize the base much more to the user’s likings.
Second, a lot better client support.
And third, possibility for community input by an addon/patch system.
An example would be, that I have a few default themes for the panels, and people could create an own theme which
then can easily be installed by others.

[size=16pt]Progress:[/size]
I wont go for the “green is finished, …” stuff but just add what is DONE with a piece of media.

[ul][li]
[size=14pt]Client sided[/size]
[spoiler=View client progress]
First of all, before anything else launches the properties are loaded.
This way we know the last theme the user used and his server name.
e.g. The control panel will say “TheServerName - Control Panel”

The control panel of the client which gives the ability to modify… probably everything somebody wants to modify in a client.
As you can see there’s also support for distributing, zipping the cache and jarring the client.
You can also toggle whether flames should be shown on the client login screen or not.

Choosing a new client background works via a file chooser dialog.
Size doesn’t matter! As the image is scaled upon loading the client.

The distribution folder where distributions are written to.
The jar and zipped cache will automatically be named to the server name that is used.
So it’s not always Julia and Julia_cache but the choosen server name.

The cache downloader.

An example of a different theme being used and the changed background.

And here’s the loaded client itself, the background image is automatically scaled.

Unless somebody gives me a good idea, the client is pretty much finished.
[/li][/ul]

[/spoiler]

[ul][li]
[size=14pt]Server sided[/size]
[spoiler=View server progress]

Crowns importing.
Any image can be used as it will [B]automatically[/B] rescale when [B]written[/B] to the cache folder.

Rights editor, add your own custom rights with their importance.
You can assign a crown from the imported crowns to rights in case you want to.
considered player means that e.g. a moderator can play like normal players but an admin
would not be considered a player and can’t trade etc…
