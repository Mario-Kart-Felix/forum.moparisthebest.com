Web management tool or Desktop Application

General Programming
zyle1992 August 4, 2016, 2:20pm #1

So, for a game I’m working on, I need to make a content management tool. It would allow you to modify most data aspects of the game (where Npcs are placed, color palettes, other resources, etc). So, I’ve been developing it in Swing (old habits you know). But, it’s really been a pain in the ass. Someone I work with suggested to make it a web app (MVC probably) and just have a web server host it. This sounded like a good idea, but because of the way this game is being written (pluggable, thing of how minecraft works with mods), people would be able to download the game server and host it themselves. By adding a web interface, I’m adding another dependency (web app + web server) to the process of hosting and modifying the server (each plugin could also specify their own editor for plugin-specific changes).

So the question is this; should I write it as a desktop application (in java because all plugins/mods would be written in java) because they would already have the runtime environment, or add another dependency, for the sake of convenience? Is there some middle-ground?

Protosstribe August 4, 2016, 2:20pm #5

At work we use Swing for our software. But, as have been said, Swing is a real pain in the (excuse my language! :)) ass, to create flexible GUI-based applications. We need to provide different layouts for our different customers. We also need to provide different “themes” for the same features. And, not to mention, we need to support different screen resolutions, for different monitors. When we continue, we have thought about updating to JavaFX 8. I looked at JavaFX at the time we started developing this application in 2012, but it wasn’t close to as mature as it is today. It wasn’t even an option at that time. But today it really shines. So I’d suggest that you use it, rather than Swing, if you’re starting today. I know we would. We also plan to replace our Swing-based GUI with it at some point. I hope this information can help you, if not entirely, at least somewhat.
