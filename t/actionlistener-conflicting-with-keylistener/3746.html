Nice tougher problem

Before I get to the solution, a couple of pointers…you don’t need so many ‘this’ calls. You’re extending the JFrame class, and therefore all the methods are automatically a part of ‘this’ object. It makes no difference really, but its just code aesthetics.

Second, the show() method is deprecated and replaced by setVisible (thats why you’re getting the deprecation warnings). I would recommend you either just use the default setVisible() and call the pack() method in the constructor, or use this method instead:

    public void setVisible(boolean b) {
        pack();
        super.setVisible(b);
    }

Now for the problem - the button hogs the focus in the application. So what you need to so is allow your GameComponent object to be focusable and then call focus to it. First, change this.addKeyListener(this) to gc.addKeyListener(this) as you want your GameComponent to use the keyboard listener (the JFrame isn’t technically focused). Then stick this code after you add gc to the content pane:

        gc.setFocusable(true);
        SwingUtilities.invokeLater(
            new Runnable() {
                public void run() {
                    gc.requestFocus();
                }
            }
        );

Basically all that does is allow the GameComponent to be focusable (which a component is not by default) and then just requests focus when swing allows it.

Also, I would recommend you stick a enter.setFocusable(false) in there as well, so that if the user click the enter button the focus won’t stay on the button.

Hope that helps
