lothy August 4, 2016, 10:12am #6

I wasn’t talking about loading a site within the client so much as introducing some new packets that would allow for pagination and searching my username.

So you’d basically have these client to server packets:

  1. Search by player name.
  2. Search by skill, page number. (Opcode, skill id, page number).
  3. Search by rank (Opcode, skill id, rank).

The search by player name would simply return the stats for that player, or a ‘not found’ code.
The search by skill would involve sending a valid skill id to order by that skill, or a negative skill id to indicate ordering by overall level. It would also send the page number (zero to begin with) so we know which page to send back.

Then it’s just a matter of using LIMITed queries to return a subset of the highscores. The queries would be no different to those used to populate a web page.

Clicking a player’s name from within the current page view would issue the search by player packet.

Finally, the server response would be the information requested or a response indicative of throttling taking place (abuse prevention, because we are performing IO for every request after all).
