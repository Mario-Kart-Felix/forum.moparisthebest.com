What a shitty game minecraft is, with a script as simple as this you can run, jump, mine, build, and do everything else much faster, even on multiplayer servers:

#!/bin/bash

# fix time on kill
trap "ntpdate pool.ntp.org" INT

# defaults are to sleep 2 seconds, set clock forward 4 seconds
# which oddly seems to work better than 1:2
sleep_for=2
skip=4
while [ 1 ]
do
	for hour in {0..23}
	do
		for minute in {0..59}
		do
			# for second in {0..59..4}
			for (( second=0; second<=59; second=$(($second + $skip)) ))
			do
				#echo "Setting time to: $hour:$minute:$second"
				date +%T -s "$hour:$minute:$second"
				#sleep 2
				read -n1 -t$sleep_for input
				if [ "$input" != "" ]; then
					# make sure it is a number
					echo $input | egrep "^[0-9]+$"
					if [ $? == 0 ]; then
						echo "sleep_for: $input"
						sleep_for=$input
						read -n1 -t$sleep_for input
						# make sure it is a number
						echo $input | egrep "^[0-9]+$"
						if [ $? == 0 ]; then
							echo "skip: $input"
							skip=$input
						fi
					fi
				fi
			done
		done
	done
done

It will need to be ran with sudo or root privileges in order to be able to set the system time, and to avoid a big lag in minecraft when you start the script, you will want to start it before you open the minecraft client.

As you can see, by default it sleeps 2 seconds and sets the clock forward 4, but you can change those during runtime just by pressing the number of seconds to sleep, and then the number of seconds to skip, so to return to regular time just press 11, to go back to default press 24, you can try anything, like 23, etc etc. Let me know what times you find work best for you.

In case it’s not obvious, just stop the script by killing it or pressing ctrl+c, you could make it stop by pressing ‘q’ or the like if you want, but I didn’t. Also the bash code will work on a mac, but the date command is most likely different, and the read command might be different, let me know if you port this to mac.

That’s about it, have fun. I have a feeling the people over at MoparCraft.net won’t like this much.
