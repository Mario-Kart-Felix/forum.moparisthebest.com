silabsoft August 1, 2016, 12:33am #3

It would be a real memory hog to keep all the values available all the time. you also don’t really need to ‘decompress’ all the object definitions just the map file. Look at a 317 client how sending the walking queue works. it checks a map array to see if the location is reachable and then sends the queue if it is. one could simply use the same method in the server. However If you were to implement this only have the data available for regions ‘in use’ by players. the other regions should just hold nothing until they become ‘in use’. When an new region becomes in use you should take the time to load the necessary information before telling the client that entered the region that everything is fully loaded. (Runescape 317 had a packet for this I would assume the new version do as well)

‘in use region’ - when a client is in the region all other regions should be considered not in use with the exception of the wilderness which should be the clients regions and surrounding 8 blocks as many of the players will be running around quickly and a region loading delay could cause a unfair death to a player.

You also need to understand about different object types.

Type 22 is walkable except if it’s on a sky height (1, 2, 3, 5, 6, 7, 9, 10, 11, ect).

Type 0 is walkable but cannot move further until the entity they are following goes in a specific direction. If you were to at a wall facing West towards your follower, the follower needs to be north, south, east, north east, south east in order for you to move.

It’s a very long process which requires a lot of logic.

Implement the A* pathfinding algorithm, keep track of un-walkable tiles/object locations cached into region objects. Load the data for the region when a player enters that region, and release the reference to the region when it isn’t being used. This way, you save memory as you only have what you need in memory. Take the initial point and the last point that the client sends you in a walking packet as point A and point B. Plug it into your pathfinder, generate the path, and make the player walk it.
