Helping you with your server. Not spoonfeeding you every little piece of information and EXACTLY where to put it. If you have a “specific” problem, we can help push you in the right direction, and if you provide enough information about the problem, someone may have a snippet of code that you can tweak slightly to fit your needs. But posing a questions such as “How do I validate packets” is not going to be constructive. The short answer is “Determine all valid data points and ensure that the packet is within those points”. But the same solution does not usually apply across different packets.

For example, it is not valid to receive an object action packet for an object 15 tiles away, but it IS valid for a “use spell on” packet. If you’d like, you can ask a question such as “What are valid conditions used in an object action packet” which some answers might be if the object is actually loaded on the server, if the player is within range of that object, if they are not currently using an object, etc. Just as some valid conditions to check for on a button click packet might be if the player is currently on the interface, and if the id sent to the server is valid, and if depending on the type of interface, the player is not currently using another interface of that type.

These are all things that we cannot tell you, because every packet is different, and requires different levels of validation. Just like in the rules, ask specific questions, so that we may help you better.
