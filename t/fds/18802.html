The reason this is happening is because you are trying to use a lightweight component (JMenuBar) with a heavyweight component (Applet). Lightweight components will NEVER have a z-order greater than any heavyweight component. The solution is to use a MenuBar with your Applet. That way you are using Heavyweight with Heavyweight.

A good rule of thumb is that if you are using a UI component with a name starting with “J”, only use other components starting with “J”.

@lawl: I just took a look at my old source and saw that I had an additional call that I forgot about:

I just tested it out myself now (man I hate notepad, which is the only thing I can use on these college computers) and it worked perfectly fine without the above line.

import java.applet.Applet;
import javax.swing.*;

public class TestFrame {
    public static void main(String[] args) {
        new TestFrame(new HelloWorld());
    }

    public TestFrame(Applet a) {
        JPopupMenu.setDefaultLightWeightPopupEnabled(false);
        //ToolTipManager.sharedInstance().setLightWeightPopupEnabled(false);

        JFrame f = new JFrame("Test Frame");
        f.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        f.getContentPane().add(a);

        JMenuBar menuBar = new JMenuBar();
        JMenu menu = new JMenu("Menu");
        menu.add(new JMenuItem("Menu Item"));
        menuBar.add(menu);
        f.setJMenuBar(menuBar);

        f.setSize(250,250);
        f.setVisible(true);
    }

    private static class HelloWorld extends Applet {
        public void paint(java.awt.Graphics g) {
            g.drawString("Hello World!",100, 100);
            System.out.println("Hello World!");
        }
    }
}
