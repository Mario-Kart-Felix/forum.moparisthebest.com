With RSBot, since the main script starts back from the beginning every loop, it just checks the anti-randoms there and if there is a random, then it completes the random script and continues on with the regular script.

If your main script is in a while(true) loop like iBot’s is, the only thing I can think of off the top of my head is to checkevery 5 or so seconds, or make sure the user puts as the first line in their while loop checkForRandoms(); I’m sure there’s better ways though lol.

moparisthebest July 31, 2016, 5:18am #3

You pretty much have 2 options. Either check for random events every X seconds (or after your script has done it’s loop) or have random events spawn events to your program, which will make your program pause it’s execution and handle the event. Depending on the type of randoms (how many are there now? I don’t even know) you may find it best to do a combination of these techniques.
