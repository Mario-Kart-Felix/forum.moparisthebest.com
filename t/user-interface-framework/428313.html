_s11 August 2, 2016, 3:07pm #4

making GUIs programmatically is a chore for anyone, since trying to represent something as visually oriented as a GUI in a language that’s meant for computation is just gay gay gay.

the hardest part of writing a gui library is making the code easily readable, easily editable (one change doesn’t break the whole thing) and expressive of the actual gui itself. a good counterexample of my latter point is code like

// pseudocode button1 = new Button("Hello"); button2 = new Button("Click me for fun!"); window = new Frame(...); window.setLayout(BlahLayout); window.add(button2, CENTER); window.add(button1, LEFT);

what the hell does that tell you? enough, but it’s shit and unexpressive. you could use anonymous inner classes in the window.add(Component) method but since anonymous inner classes are crufty syntax that tries to be anonymous functions but fail like a fatass on a diet, writing a library to naturally express gui creation is hard with a language like Java. (double brace initialisers make it a bit more tolerable but still not great)

so i propose Clojure for frontends instead. create a little DSL using the macro system for naturally expressing GUIs, with an outcome like

(with-frame [frame]
  (with-blahlayout
    :left (button "Hello") :center (button "Click me for fun!")))

how much simpler is that!!
now the cool thing with Clojure is that it runs on the JVM, so you can interface with all the Java libraries ever. so if you like writing Java, but want a nice way of writing your GUI, use Clojure for the application frontend and Java for the backend. yay!

for more complex GUIs you could play with this concept to be extremely accurate to what you want the final representation to be. let’s give ourselves the task of creating a GUI that asks a user for their name, it then clears the frame says hello to them!

[code=clojure]
(with-frame [frame]
(with-complex-layout
(transition
((:center (textbox tx1 “Enter your name”) (button “Next” :onclick (next-transition))))

  ((:top (label (string "Hello " (contents tx1))))
   (:center (button "Exit" :quit))))))[/code]

this include a bit of logic, but you could always replace that with the Java backend logic if you wanted to do some special stuff. it might also be better to make tx1 a binding so that there’s no scoping issues

this is unfortunately hypothetical, but Clojure is great for this because of macros. you can create a DSL to almost flawlessly express a GUI!!
