yakman July 30, 2016, 6:31pm #6

perhaps you should wait a bit
images in java are a little bit odd if you’re new to them,
when you call the getImage(), it returns immediatly, even though the image is not loaded completely, when more of the image becomes avaliable, the methods in the ImageProducer interface are invoked,
a similar story when you draw an image, the method drawImage() returns immediatly, and when more of the image becomes avaliable, the methods in ImageConsumer get invoked.

when you pass ‘this’ to drawImage(), thats an instance of ImageConsumer

I might have gotten a few details wrong, but that basic idea is correct.
it looks to me like you’re doing everything right, I just suggest you wait a little bit, to give the image a chance to load a little.
or comment out the drawing of an Oval and try, but i dont think thats causing anything bad
also, check if some of the image loading methods return null.
