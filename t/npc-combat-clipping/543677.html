ohokay August 4, 2016, 1:13pm #2

I don’t think most clipping versions take into account the size of the mob or npc, which means it could be looking as if it’s running through certain objects, when actually it’s not. The actually path it’s taking is correct, but the npc is too large and would seem it’s going through objects if there is one nearby.

Also, it could be that you’re missing some map data for that certain area, which means the npc won’t be clipped as it’s not registering an object in its path.
