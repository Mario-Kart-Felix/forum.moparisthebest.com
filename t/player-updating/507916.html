Lin August 4, 2016, 5:47am #5

I’d recommend just making a really shitty system that only handles one player to start, so you get an idea of how it works. Then expand the same shitty system to support multiple players. Next, limit the searching of the players you need to add by implementing a region system. Finally, only add/remove players when you walk/run. Implement packet caching when finished and you should have a pretty fast system.
