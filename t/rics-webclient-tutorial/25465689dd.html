Simple, Lucky for you i have also used Ip.board before (both nulled, unlicensed and i have also used a license i found online for 2 different versions at times.) Now do the same steps for your HTML web page. Just upload the client to your website that you have your ip.board on.

upload all the files to a directory in your webhosting. So create a directory named something (whatever you want) so if i named mine “rsps” then i would have to add the cache.zip to that folder and all the files to that folder. Once done you edit your files for your rsps to where the cache.zip is located for downloading. So it would be something like “http://yoursite.tld/rsps/cache.zip” and thats what downloads and extracts to there hard drive.

Now it should just load the way it should but it has to look for all the files in the same directory (not hard when it searches directories within its directory). That should work. And if you cant upload your files there then open a new account elsewhere on another host upload the files there make a fake website there to make it look legit and use that domain as the downloads section for your rsps.

It should work just fine. But if you do it and host the files remotely then the rsps has to be able to find the files remotely in order to run so its just easier to have them on your currently existing file server.
