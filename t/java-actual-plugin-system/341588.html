Since Java’s ClassLoader system doesn’t let you define a same class twice, nor redefine a class (unless you use the Instrumentation API), an easy get-around is to create your own custom ClassLoader and re-instantiate it every time you want to reload the classes loaded by it. Note, however, that re-instantiating it will cause all classes loaded by that ClassLoader to lose definition since it will have been collected by the garbage collector, until of course, you reload all classes into that new ClassLoader instance.

Here’s the custom ClassLoader I wrote:

[code=java]
/* Author: abraham2119 */

import java.io.ByteArrayOutputStream;
import java.io.File;
import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.io.IOException;
import java.io.InputStream;
import java.util.zip.ZipEntry;
import java.util.zip.ZipFile;

public class ReloadableClassLoader extends ClassLoader {

public ReloadableClassLoader(String[] validNames) {
	super(ReloadableClassLoader.class.getClassLoader());
	this.validNames = validNames;
}

private InputStream getClassInputStream(String className) throws FileNotFoundException {
	String[] classPath = System.getProperty("java.class.path").split(File.pathSeparator);
	className = className.replaceAll("\\.", "/") + ".class";
	for (String s : classPath) {
		File f = new File(s);
		if (f.isDirectory()) {
			File file = new File(f.getPath() + File.separator + className);
			if (file.exists()) {
				return new FileInputStream(file);
			}
		} else if (f.exists()) {
			try {
				ZipFile archive = new ZipFile(f);
				ZipEntry entry = archive.getEntry(className);
				if (entry != null) {
					return archive.getInputStream(entry);
				}
			} catch(IOException e) {
				continue;
			}
		}
	}
	return null;
}

private boolean isValidPlugin(String name) {
	boolean validPlugin = false;
	for (String s : validNames) {
		if (name.equals(s)) {
			validPlugin = true;
		}
	}
	return validPlugin;
}
	
public Class<?> loadClass(String name) throws ClassNotFoundException {
	try {
		if (!isValidPlugin(name)) {
			return super.loadClass(name);
		}
		InputStream stream = getClassInputStream(name);
		if (stream == null) {
			throw new ClassNotFoundException(name + " was not found within the classpath.");
		}
		ByteArrayOutputStream buff = new ByteArrayOutputStream();
		byte[] buf = new byte[1024];
		int read;
		while((read = stream.read(buf)) != -1) {
			buff.write(buf, 0, read);
		}
		stream.close();
		byte[] classData = buff.toByteArray();
		return super.defineClass(null, classData, 0, classData.length);
	} catch(IOException e) {
		e.printStackTrace();
	}
	return null;
}

private String[] validNames;

}[/code]
The class takes an array of names that are valid to be reloaded by it as a parameter in its constructor. So assuming you’d only want to reload the classes: “Test1.class”, “Test2.class” and “Test3.class”, you’d pass args as so:

new ReloadableClass(new String[] { "Test1", "Test2", "Test3" });

In theory, it should work as a regular system ClassLoader works - by looking for the specific class in the classpath. It iterates through the different paths specified by the classpath and searches within the directory/JAR for the class.

Here’s a usage example of how you would load and reload “Test.class”:

[code=java]class Main {
public static void main(String[] args) {
try {
ReloadableClassLoader classLoader = new ReloadableClassLoader(new String[] { “Test” });
Object test = classLoader.loadClass(“Test”).newInstance();

		/*do whatever you want here*/

                    /*Have previous 'classLoader' instance be collected by the gc, now instantiate new one..*/
		classLoader = new ReloadableClassLoader(new String[] { "Test" } );
		test = classLoader.loadClass("Test").newInstance();
	} catch (Exception e) {
		e.printStackTrace();
	}
}

}[/code]
Enjoy.

PeeHPee:

creating a new class loader each time you want to reload a class isn’t really practical. wouldn’t this mean you’d have to discard all classes which were loaded by that classloader too?

[quote=“abraham2119, post:1, topic:341588”]Note, however, that re-instantiating it will cause all classes loaded by that ClassLoader to lose definition since it will have been collected by the garbage collector, until of course, you reload all classes into that new ClassLoader instance.[/quote]It’s the only actual way to bypass it. But it isn’t actually impractical since an instance of ‘ReloadableClassLoader’ should only load classes which are meant to be reloaded. But yes, if you want to reload one class, you’d have to reload all classes loaded by that specific classloader.

edit: Just to add on to that, if for whatever reason you don’t want to reload all “plugins” once you have to reload one, you can instantiate a new instance of “ReloadableClassLoader”-per-plugin; this, obviously, isn’t very memory efficient though.
