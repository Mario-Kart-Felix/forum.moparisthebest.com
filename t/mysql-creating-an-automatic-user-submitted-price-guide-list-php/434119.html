There’s an every-changing economy in the real world, as well as on runescape private servers, I’m new to moparscape but i’ve been doing website design professionally for the past 5 years, this tutorial is going to show you how to easily create an automatic price guide for your runescape private server (works with any server version), users will input data for the price guide and the server will automatically do some mathematical equations to get the average of the recent price changes for that item. Giving a fairly accurate price of the item each time. There’s not much pre-hand information that you will need to know to do this, just pay attention and be able to copy and paste and follow simple directions.

I. Requirements
There are a few requirements that you will need to setup this automatic price guide, I’ll list them below and where to get them.

  • MySQL
    MySQL is a program used for storing databases, tables, and information for websites and other programs. If you have highscores, you’ve more than likely used MySQL already for creating them, if you don’t have it, You can download it from the MySQL official website located at http://www.mysql.com/downloads/cluster/, But there is an easier way to get this and other necessary programs through a program called XAMPP, which i will go over in a later part of the tutorial.

  • Apache
    Apache is a web hosting application that is used for displaying a personal website to the world, sometimes not personal though, its pretty much just used for hosting your website 24-7 and directing the web browser correctly to have the website run correctly, Once again like MySQL, XAMPP will help you install Apache.

  • Php
    Php is a programming/coding language that is used with almost every website for creating basic functions that may not seem so simple, You will also be able to install php with XAMPP automatically, what i will go over later.

  • Ability to Copy and Paste
    This tutorial really isn’t that hard, all you have to do is concentrate, copy, and paste. And if you have any problems with the set-up of this, you can just post a comment below and i will help you with it ASAP

IV. Graphical User Interface and Php Program Functions
The users aren’t going to be able to put in any information without being able to have a form to enter data into, you’re now going to be needing to upload some files to your website host or wherever your website is hosted from, i would recommend creating a new directory on your website’s root index called, ‘prices’ and then uploading these files to them, below i’m going to be going over the files and what each of them do.

  • index.php
    This is the main page that will be displayed to users, .php is a file extension telling the web browser that its an php file, so that it can read it correctly. This page will have the actual price guide on it, for everyone to see (with the optional form to have the form below it)

  • insert.php
    This file is for the actual form that’s going to be inputting the information into the MySQL database and saving it there (After going through insert_ac.php)

  • insert_ac.php
    This file has the information that allows the web browser to connect to the database, if it has the correct information to do so, this is also the page that will give a success page if everything goes well.

There is the top row that you are going to need to change, you are going to need to change this line of code:

Usually, “localhost” stays the same unless your database is hosted somewhere other than your own server or webhost, sometimes it can be an ip address such as, “123.45.67.89” In that case you’d change “localhost” to “123.45.67.89”. Secondly, you need to change “DATABASE_USERNAME” to your own database username, which generally is “root” and you also need to change “DATABASE_PASSWORD” to your database password, once again, generally is “password” or “root” (In rare cases).

VII. Common Errors and Questions
More Errors and Questions will be added as people have different problems, i’m going to go over the programmed error messages though below.

Error Message:

There is an error connecting to the MySQL server

Meaning: This Error message is broadcasted to the Viewer when the page is loading and the Browser cannot connect to the MySQL Server, meaning you have connection information wrong somewhere, i would advise you to go over this line of code located in index.php:

$connect = mysql_connect("localhost", "DATABASE_USERNAME", "DATABASE_PASSWORD") or

Error Message: cannot connect
Meaning: This means that the Web browser cannot connect to the MySQL database, this happens when trying to input new information in the table, after you click, ‘Submit’. To fix this, you’ll need to re-check these three lines in insert_ac.php

$host="localhost"; $username="DATABASE_USERNAME"; $password="DATABASE_PASSWORD";

Error Message: cannot select DB
Meaning: This happens when you click the ‘Submit’ and the Web Browser tried to connect to the MySQL and succeeds, but then it cannot connect to the database, or the table with-in the database, this usually happens because you type the Database name or Table Name wrong in the code, or they have not been created. Revise these two lines in insert_ac.php

$db_name="DATABASE_NAME"; $tbl_name="items";
Also Note: Do not get “DATABASE_USERNAME” confused with “DATABASE_NAME”. “USERNAME” is what connects to the MySQL, “NAME” is the actual Database name that is contained with-in the MySQL.
