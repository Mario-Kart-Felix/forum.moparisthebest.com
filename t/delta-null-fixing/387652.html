Where is the null occurring? From what I could discern from your post, I’m assuming someone is flooding your server with invalid login requests. The easiest way to stop this is to implement counter measures (is the client version correct, username contains only valid characters, etc). Also, an easy, effective, but possibly problematic way is to restrict client connections to a specific UID (a random number either generated at start up, or a random number saved in the cache directory.). Then, if someone connects, and the UID that was sent during login doesn’t match the one you specify, reject the connection. The problem is that if someone is using a different client and wants to connect to your server, if their UID doesn’t match yours, they can’t login.
