[size=20pt]Intro[/size]
I’m writing this tutorial due my disappointment in a person
and all other people wanting to program instead of just moving npc’s around.
This guide is aimed on learning you the basics of programming
and will be explained like I’d have to learn it my mom, for the real noob.

We will be creating a basic skill throughout the tutorial.

[size=18pt]Let’s get started.[/size]
[size=15pt]Step 1: Making a class[/size]
First of all we will be creating a new class, a .java file where we will be programming in we call a class.
Our compiler converts these .java files to .class files so they can be read and executed by our computer.

If you are using a PI, add a new java file in your skills folder,
if you are using Delta just add it with the other files.
Name it “Crafting”, if you already have a crafting file remove the other one.

In the new created file, we will first be opening our class and closing it.
Therefor add this on top:

public class Crafting {
}

{ opens the class
} closes it

All the upcoming things we will do have to be added between the “{” and “}”

You now have successfully created a new class!

[size=15pt]Step 2: Creating a method[/size]
We will be creating a void for our crafting,
first of all you need to set up the constructor, the same way as making a class we open and close it:

public void cutGem() {
}

You successfully created a method!

[size=15pt]Step 3: Continuing your method[/size]
Now we want if we execute the method that our player will craft.
First of all we will need the data for it:

  • We know we need a chisel and uncut gems, so go lookup codes on a website like this: http://rspscodes.synthasite.com/
    and find the chisel code and some uncut gems codes.
    looked those up for you, chisel = 1755, gems are: 1617, 1619, 1621, 1623 (diamond, ruby, emerald, sapphire)

[size=15pt]Step 4: IF statement[/size]
Now to start off we will use an easy way of doing it to learn, the better way will follow later.
In your method so after the opening bracket, enter a line down and add this:

if (item1 == 1755 && item2 == 1617 || item1 == 1617 && item2 == 1755) { }

You successfully created an IF-statement!

If item one is the chiselID and item2 is the uncutID;
then it will look what to do after the { opening bracket.

item1 and 2 are what the player clicks in his inventory, if he does “use” on “another item”
then use will be item1 and the other item item2.

that’s why we added “|| item1 == 1617 && item2 == 1755”
this basically means “OR” if item1 is the gem and item2 the chisel (if the player used the gem on the chisel instead)

Now you are thinking, where does item1 and item2 come from?
In your constructur “public void cutGem()”
between (), add: “int item1, int item2”
so it looks like this:

public void cutGem(int item1, int item2) {

We will be adding this method in our packets first to continue the explanation,

[size=15pt]Step 5: Adding to our packet[/size]
Now since this is cutting a gem (use chisel on uncut)
we want this method to be executed when a player uses a chisel on an uncut.
Therefor we will need to add our method inside the item-on-item packet.

Hoping for you that your server is documented and tell which number is which packet,
else search for

in your client.java (delta)
for PI go to your packets folder and it should easily be named “itemonitem.java”

You should see some lines like this:

int usedWithSlot = c.inStream.readUnsignedWord();
int itemUsedSlot = c.inStream.readUnsignedWordA();
int useWith = c.playerItems[usedWithSlot] - 1;
int itemUsed = c.playerItems[itemUsedSlot] - 1;

and some other lines like our method cutGem();
If not, make sure to add after the lines shown above.
We will be adding this correctly later on, but for now just put
cutGem(useWith, itemUsed);
I need this to explain you some things farder on.

Now you see we added useWith and itemUsed, where item1 and item2 is,
and if you inspect the code you see useWith and itemUsed come from some weird lines like instream().getBigendiandzdez();

Those send what the player does ingame,
so if the player clicks use on his chisel it will send the chisel id to the packet,
Here’s an image to explain:

So since our packet is active is it will read everything in it.
The “instream.getBigWordEndian();” and similar things are the data we get.
We save those to an int:
int firstitem = Instream.dekferg(); int secondItem = instream.dfere();

The server scans through everything added in the packet and then…
it reaches our method “cutGem(firstItem, secondItem);” and executes it.

Our integers we declared in our void will now have the value of the clicked items ingame.
So basically our method now changed to this;

public void cutGem(1755, 1617) { if (1755 == 1755 && 1617 == 1617 || 1755 == 1617 && 1617 == 1755)
if we clicked the gem first, everything would be the opposite.

[size=15pt]Step 6: Continuing our method[/size]
Now we have everything required to get out things working.
So a short lookback:
[ul][li]We click on our chisel ingame and then on our gem.[/li]
[li]The client sends the data to our server.[/li]
[li]Server sees that you clicked “use” and then clicked another item.[/li]
[li]It makes the item on item packet run.[/li]
[li]The data of the client (chiselID and gemID) get added to the integers (useItem = c.instream…).[/li]
[li]It reads the methods in the packets and finds ours and executes it.[/li]
[li]Because we did this “cutGetm(usedItem, otherItem)”, item1 and item2 in our void constructor get those values.[/li]
[li]Our void opens {[/li]
[li]Reads our if statement - if (item1 == # && item2 == # || …)[/li][/ul]

If we clicked ingame on other items than the 2 id’s in the if-statement it will do nothing, because the id’s are not right.
If it is right it will open {
and read what we put there, we didn’t put anything yet so we will do that now.

We want to add a gem and remove the uncut one.
so behind the {
We add b[/b]:

c.addItem(1607, 1);  //1607  is the cut gem
c.deleteItem(1617, 1); //1617 is the uncut gem

PI:

c.getItems().addItem(1607, 1);
c.getItems().deleteItem(1617, 1);

But wait, the one who is really understanding the tutorial will now see that there’s something not right.
Where does that c come from? And the additem where’s that?

The addItem method is located in our client.java class (delta) itemmanager(pi).
We can’t just simply reach a method in another class so we have to call the class itself first which is c.
But the server wont know what c is, it’s just a character being there so we have to show the server where it comes from.

So on top of your class under your opening {
add this: (capital C if using PI)

client c;

public Crafting(client cl) {
c = cl;
}

Now the server will know that c comes from the client class and that is has to be called in the crafting class.

Now you see how we call things, you now where the c is coming from,
Now you should realize that we added our cutGem in our packets without calling it, so it can’t be reached to.
You are right it can’t so we will fix that up now.

First of all go to your client class and add this under the opening { of the class

Crafting crafting = new Crafting(this);
public Crafting getCrafting() {
return crafting;
}

For PI or packaged Delta’s don’t forget to import your Crafting class!
If you forgot how to import, scroll up to where your imported your client class in your crafting class.

Now in your item on item packet where you added cutGem(
add infront of it “getCrafting().”
so it’ll look like this:

getCrafting().cutGem(useWith, itemUsed);

Now it can reach to our cutgem method!

For PI or packaged delta’s we will also need to add a package declaration and imports.
On top above your class constructor we need to add this.

package server.model.players.skills;

Locates our crafting file, its in that ^directory. (adjust to your server’s packaging)

The path starts where the packaging started, so if you go to your Server.java you’ll see on top “package server;”
This is the start and not src like some people think, src is just a folder.

Now we also have to import the packages where classes that we call are located, so we have to import our client class.
Under your package declaration add the package of your client class like this:

import server.model.players;

If you now save and compile, go ingame and click the chisel on the diamond you’ll see it will work.
If it doesn’t you must have done something wrong.
If it is working, continue the tutorial.

[size=15pt]Step 7: Continuing our method[/size]
Now we want that something happens for all the uncuts.
One way is using easy if-else statements:

public void cutGem(item1, item2) { if (item1 == 1755 && item2 == 1617 || item1 == 1617 && 1755) { c.addItem(1607, 1); c.deleteItem(1617, 1); } else if (item1 == 1755 && item2 == 1619 || item1 == 1619 && item1 == 1755) { c.addItem(1609, 1); c.deleteItem(1619, 1); }

So basically what this does now is looking to our if statement, and if it is not right it will go to our “else if” statement.
This way you can add all your gems and they will all work.

[size=15pt]Step 8: 2D Arrays[/size]
Now a better way of doing this is using an array or a 2D array.
We will store all our data in a 2D array.
On top of your class outside the method, add this and add the id’s in the array.

private static final int[][] GEM_DATA = {
{1617, 1607}, //uncut, cut
{1619, 1609},
};

Now remove everything INSIDE your void and add this instead:

for (int i = 0; i < GEM_DATA.length; i ++) {
}

This makes us loop through our array we just created.

Now,

for (int i = 0; i < GEM_DATA.length; i ++) {
if (item1 == 1755 && item2 == GEM_DATA[i][0] || item1 == GEM_DATA[i][0] && item2 == 1755) {
c.addItem(GEM_DATA[i][1], 1);
c.deleteItem(GEM_DATA[i][0], 1);
}

Don’t mind the i, its from our loop, what you need to know is the numbers [0]
That’s the spot in the array

1619, is spot 0 or [0]
1609, is spot 1 or [1]

So -> IF item1 == chiselID or FIRST_SPOT_OF_ARRAY (uncut gem)
then
additem(SECOND_SPOT_OF_ARRAY, 1); (cut gem)
deleteItem(FIRST_SPOT_OF_ARRAY, 1); (uncut gem)

[size=15pt]Step 8: Variables.[/size]
Now to give you a thought about variables,
we will be checking how many times the player made the method execute. (used a chisel on an uncut)

On top of your class but still in the class, add this:

private int count;

You just have declared a variable! (A primitive data type to be specific)

The variable we declared is an int, this holds values of decimal numbers.
By default the value it holds is 0.

You can read up about different sorts of variables here: http://docs.oracle.com/javase/tutorial/java/nutsandbolts/datatypes.html

count ++; //increases by one
count += 1; //increased by the given value
count --; //decreased by one
count -= 1; //decreases by the given value
count *= 1; //multiplies by the given value
count /= 1; //devides by the given value

Now in our method we make the variable increase by one.
So our method should now look like this:

public void cutGem(int item1, int item2) {
   for (int i = 0; i < GEM_DATA.length; i ++) {
      if (item1 == 1755 && item2 == GEM_DATA[i][0] || item1 == GEM_DATA[i][0] && item2 == 1755) {
         c.addItem(GEM_DATA[i][1], 1);
         c.deleteItem(GEM_DATA[i][0], 1);
         count ++;
      }
   }
}

Now we want the server to tell our player the amount and we add this to it:

c.sM("You have cut "+count+" gems!"); //Delta
c.sendMessage("You have cut "+count+" gems!"); //PI

The reason we are putting our variable between: “++” is because what we outprint is a String,
if we just add count in it, it will outprint “count” and not the value of our variable.
Therefor we put our variable between: “++”

Now every time the player cuts a gem he will also get the amount of gems cut send in his chatbox.
