semaphore July 30, 2016, 10:49am #6

The most likely reason the server does not generate the initial path is simple: cycles. They save cpu cycles by delegating that to the client. In a game that is already strapped for resources, it makes sense to delegate everything you can to the client. If you look at the game code, the client generates the path and sends the movement requests node by node. The server responds with an update-client-location or a set-client-destination-to-zero (or one of a couple other packets) depending on whether the node is accessible or not. Accessibility is determined by bit comparison with a two-dimensional array.
