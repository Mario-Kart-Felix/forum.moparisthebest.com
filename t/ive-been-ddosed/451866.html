OT: In short, DDoSing is when someone attacks your server by flooding your Internet connection with traffic. If successful, nobody can connect to your server over that Internet connection because it’s busy being overwhelmed by junk packets. Unfortunately, I was unable to find a good graphic to represent what I’m trying to explain.

However, even if you were being DDoS’d, you would still be able to connect to your server from your own computer, because that kind of connection doesn’t use the Internet line that’s being attacked. Also, solo DoS attacks are rarely successful, unless your Internet is really slow.

Therefore, you are most likely not actually being DDoS’d. My best guess is your external IP changed. If you are using a DNS service (such as No-IP or DynDNs), update the IP your host redirects to. If you are not using a DNS service, use http://whatismyip.com/ and tell others to connect to that IP.

If you are using the same computer to run and connect to the server, change your client’s IP to “localhost” (no quotes). However, if you want other people to be able to connect over the Internet, that’s a whole new issue. I’m sure there’s a tutorial for it somewhere, this post is already long enough.
