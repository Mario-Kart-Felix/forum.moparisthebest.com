BAsh_CntLgn July 30, 2016, 10:28pm #12

Speed and memory aren’t the issue. The only thing flush does is clear out the (small) caches in the network stack. They aren’t deallocated and they’re small anyway.

The issue is when the server wants the last few bytes of the file or messages your sending. For example with a script reading lines from across the network it might skip the last few as they are held in the client’s cache and not actually sent to the server. By using flush you make sure the server gets all the information you wrote.
