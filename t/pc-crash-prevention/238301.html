Well, in Linux, if the application makes a memory allocation request greater than that which the system can fulfill, it dies with a segmentation fault (SIGSEG). I would assume Windows should have something like that, and I doubt running low on memory was your issue (Windows also tries to expand the pagefile to accommodate memory requests when running low on memory, virtual or otherwise), your processor probably started to overheat so your motherboard shut your computer down or something of the sort to protect it.
