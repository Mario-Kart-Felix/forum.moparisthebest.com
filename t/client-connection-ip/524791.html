Well, to start off, I don’t have a VPS and I’m in the early stages of development for my RSPS. We don’t have an official client yet and we’d all like to connect to the server to test dupes, bugs, and glitches. I can connect fine with localhost, but I’d have to open my ports etc. for my team members to connect and then when they’re hosting they’d have to do the same thing and we’d have to change client IP’s to connect to them. So I was wondering this: would it be possible to make a domain as the server IP? We could keep the IP the same forever inside of the client and just change where the domain points. Is this possible? If it is, what do I have to do? Is there another way to solve this problem?

A VPS is out of the question for right now as we’re all scrounging up money at the moment. Our budget is like $30 and we want a good VPS, we don’t want to waste it on a cheap one now and not have the money later or buy a good one now and not have money when it’s time to release.

Shorty626 August 4, 2016, 9:44am #2

All you need to do is set the IP inside the client to a DNS name. Like example.com then change the DNS to point to the team member you have hosting the project. Tbh though if you are working with multiple people, you’d be better off just creating a repository and having one person compile and run the server when changes are made rather than decide who will host and what parts of your code have changed. Then you eliminate all of those problems in the first place.
