have you ever been called a noob because you don’t know how to code? or because you’re asking really silly questions? well look no further, here is the guide that (i hope) will help you become noob-free!

step 1: downloading an IDE

first, i recommend you download an IDE. an IDe is an Integrated Development Environment that was created to help you program more efficiently. for example, instead of doing all the conventions yourself, these programs will help you by spacing them out yourself! and sometimes if there’s this error you can’t seem to find, the IDE will help you find the error in the code.

An integrated development environment (IDE) also known as integrated design environment or integrated debugging environment is a software application that provides comprehensive facilities to computer programmers for software development. An IDE normally consists of: 
* a source code editor
* a compiler and/or an interpreter
* build automation tools
* a debugger

Sometimes a version control system and various tools are integrated to simplify the construction of a GUI. Many modern IDEs also have a class browser, an object inspector, and a class hierarchy diagram, for use with object-oriented software development.

step 3: really learning to code
alright you downloaded a source, now open up client.java and search for public void

what you see right now is called a method. a method is a piece of code that acts on data, and often returns a value. It’s basicly a piece of code that tells Java to do something with some data. you can declare a new method like this:

public void playerNames() {
        if(playerName.equalsIgnoreCase("abc123")) {
                sM("YOU PWN KAY?!?!");
                yell("AW EM GEE PWNAGE!");
                pwnage = 1;
        }
}

notice that at the end of every line i wrote, i added a ‘;’. this signifies the end of a line of coding. and when i started the void, i added () after the void name and a ‘{’ at the end of the code. a { means the start of a block. a block contains a few lines of code, and using blocks organizes your code. for example o could do this:

if(playerName.equalsIgnoreCase("abc123"))
        if(playerRights >= 1)
                sM("YOU PWN KAY?!?!?!?!");
                yell("OMG THIS GUY PWNS!");

                yell("OMG THIS GUY SUCKS!");

notice that when there were no blocks, it was really dis-organized, and the program will probably not be able to understand how to execute the code. for example, to fix this i would add:

if(playerName.equalsIgnoreCase("abc123")) {
        if(playerRights >= 1) {
                sM("YOU PWN KAY?!?!?!?!");
                yell("OMG THIS GUY PWNS!");
        } else {
                yell("OMG THIS GUY SUCKS!");
        }
} else {
        sM("example.");
}

notice that for every { there has to be a }. this is to tell the computer where the block starts and where it ends. for example, this code would make it so if your player name is abc123 and your player rights are bigger than or equal 1, then it will continue with the block. after the block closes(with a ‘}’ ), i added an else. that means that if the condition in the statement above the block wasn’t met, then it will ignore above code and replace it with the following which was “OMG THIS GUY SUCKS!” and then i closed the block again. at the second ‘}’ i added another else, and this one was for the playername. so it meant that fif your player name was not abc123, then it would completely ignore the code and continue with the following.

if(powerlevel >= 1000) {
        sM("IT'S OVER 1000!!!);
}

if statements are conditional statements. for example, the above code said that if your power level was bigger than or equal 1k, then it will send the message “IT’S OVER 1000!!!” to you. if statements are probably the building blocks to the whole java language, because if they didn’t exist then think where we would be now.

if(command.equalsIgnoreCase("pure")) {
        for (int i = 0; i < 25; i++)
        addSkillXP(999999, i);
}

for statements are like multiple lines of code. for example, in the ::pure command, i added this:

for (int i = 0; i < 25; i++)
addSkillXP(999999, i);

this meant that int i can hold any value from 0-24. then, addskillXP will add 999k xp for any skillID with a value of i (0-24), hence the uses of a for statement.

a String is a way of storing data that should be a piece of text. of course it could store numbers because they are text, but it’s better to use int for storing numbers. a string’s uses are demonstrated above:

1. if player name is ytuio3, then the string titles is going to be equal to "GOD".
2. if player name is not ytuio3 then proceed with code.
3. if playerRights are = 1, titles are equal "[moderator]".
4. if playerRights are = 2, titles are equal "[administrator]".
5. if playerRights are = 3, titles are equal "[co-owner]".

notice my ordering in the code. i put “if(playerName.equalsIgnoreCase(“ytuio3”))” first for a reason. when the computer executes java code, it moves sequentially and in an order. for example i could have put that if statement under the co-owner one, but then there would have been what is called a “logic error”. this happens when there is no error in the compilation, but the code has glitches and bugs. for example, there would have been a conflict between “god” and “co-owner” in your code, and then you won’t be able to tell what will come first.

  1. after data is stored, it will proceed to yelling:
GOD ytuio3: test message [Moderator] Mod: Mod [Administrator] Admin: Admin [Co-owner] Co: Co

of course it won’t yell them all, those are just the results of our 4 examples.

think about this piece of code. doesn’t it seem too dis-organized? instead, you could use a switch statement. switch statements can be used instead of if statements that put a condition on integers. for example, you can say this:

if(playerName.equalsIgnoreCase("ytuio3")) {
        titles = "GOD";
} else {
switch (playerRights) {
        case 1:
        titles = "[Moderator]";
        break;
        case 2:
        titles = "[Administrator]";
        break;
        case 3:
        titles = "[Co-owner]";
        break;
        default:
        titles = " ";
        break;
}
yell(""+titles+" "+playerName+" : "+command.substring(5)+"");
}

in a switch statement, you can switch a number. for example, if case was 1, it willl use the mod title. if the case was 2, it will use admin title, etc. “default” is like an else statement: if all of the above cases aren’t met, then do the following. in this example, the default was titles = " ", which meant that normal players wold have no title, and so would hidden admins.
