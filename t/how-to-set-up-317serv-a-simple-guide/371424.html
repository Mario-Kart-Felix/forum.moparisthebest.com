Ok, so Mendorphin’ s post on setting up Hyperion prompted me to write a guide on 317Serv. Nobody has really used it, and I think that is because the source itself came with no means of running or even compiling the source. I will provide you with these means ;D

Difficulty: 1/10
Requirements: An IDE with Apache Ant support, the client I will upload for you, Common Sense

Strangely, the server has problems running in an IDE (Integrated Development Environment), but for this tutorial you will need one in order to compile the source. I will be using an XML script (Apache Ant) to compile the source files. You will not need 317Serv.iml, 317Serv.ipr, or 317Serv.iws, just delete them. They are used for IntelliJ IDEA, a different IDE that I will not be including in this tutorial. You can find the original thread on this server here.

Required Files
317Serv- http://www.megaupload.com/?d=CD8GLDLB
317 Deob Client- http://www.mediafire.com/?nunqoedllu5p590
An IDE- http://www.eclipse.org/downloads/ , http://netbeans.org/downloads/

Eclipse, if you already don’ t know, must be extracted from a .zip file while Netbeans installs from an executable. Choose either one, it all depends on your preference. For this tutorial, I will be using Netbeans. To set up the server in Eclipse, you can use Mendorphin’ s tutorial on setting up Hyperion using the files I give you.

Running and Compiling the Server

This is the batch file you will use to run the server. Save it as “run.bat”.

@echo off cd out\production\317Serv start javaw Main exit

this is the XML file you will use to build the server. Save it as “build.xml”.

<?xml version="1.0"?> <project name="317Serv" default="build" basedir="."> <path id="classes"> <fileset dir="out\production\317Serv" includes="**.class"/> </path> <target name="init"> <mkdir dir="out\production\317Serv"/> </target> <target name="build" depends="init"> <javac srcdir="src" classpath="${toString:libs}" destdir="out\production\317Serv"/> </target> </project>

The main question that sparked my mind when I wrote the run and compile files for this base was, “Why doesn’ t the server have some sort of library folder?”. I was wrong, the libraries in this source aren’ t jarred. They are in their .class form, so don’ t let that confuse you. Also if you look, the run.bat is set to exit out of the command prompt after it is executed. This is because Yz85Racer included a server control panel, so you don’ t need the command prompt while running the server. Heres a pic of the control panel.

Alright moving on.

Eclipse
You can follow Mendorphin’ s tutorial on setting up a server with Eclipse, found here. Remember that the server cannot be run in an IDE, so you will have to go into the files manually and execute the batch file yourself. Otherwise you will get errors. So just follow his tutorial up to “So you now have Hyperion uploaded to Eclipse.”.

Netbeans
Ok open up Netbeans and click File, New Project.

An interface will show up asking you what kind of project you would like to start. Select Java, Java Free-Form Project. This type of project allows you to use your own means of building the project, just like it says in the description. Now click next.

Q84To.png725×497

The next part is pretty self explanatory. You select where the pre existing files are (the server that you downloaded), where your build script is (it should be in the same file as the server), and where you would like the project data to be stored. The location of the project data doesn’ t really matter, I just keep it with the server files as well so that they are all in one place. After selecting the location of your files, all of the other options will be configured for you automatically by Netbeans.

Jgsqv.png723×518

Then all you have to do is click Next, Next, Finish. To start looking editing files, you have to go to the upper left hand corner of the screen and change the view to Files instead of Projects.

Now that the project is set up, it is time to test out the build script. You can compile the project by going into your files, then right click build.xml, click Run Target, then click Build.

Now you are ready to run the server! Simply go to the folder that you stored the files in and click “run.bat”. It might take a while at first to start, don’ t be alarmed, the control panel eventually comes up on the screen and binds to port 43594.

Logging In
Alright if you have gotten this far, your almost done! Using the client I put up for download, you can log into the server. Keep in mind that you can change the automatic login credentials (mopar, bob) in client.java. Another thing to note is that if you have a longer password than your username, you will get errors when you try to log in. I have no idea why, that’ s just how this source works in it’ s raw form.

bA43z.png766×527

Conclusion
Now that you have finally logged in, you can start working on the source! The server is pretty blank and needs quite a bit of work in order to function and entertain a community of players, but this will help you get started. Enjoy.
