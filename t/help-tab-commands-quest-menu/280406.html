Deadly_Alibi July 31, 2016, 7:56am #4

okay so i am just going to give you an example
for case 169 (which is one of the emotes i forget which orginally i have this code:

case 169: if(teleblock == true){ //made by Deadly sendMessage("A magical force stops you from teleporting."); } else { customCommand("kqueen"); }

this is saying that when you click that emote (case 169) unless you are teleblocked it is going to intiate my custom command “kqueen” which looks like this:

else if (command.equalsIgnoreCase("kqueen")) { if(teleblock == true){ sendMessage("A magical force stops you from teleporting."); //made by Musicmad441 } else { teleportToX = 3485; teleportToY = 9483; sendMessage("You teleport to the kalphite area."); }

so it is saying all in all that case 169 (i forget which emote it is) if not teleblocked will intiate custom command “kqueen” which teleports you to X Y
so you can either do it that way or just make it teleport you without the custom command step which would look like this:

case 169: if(teleblock == true){ //made by Deadly sendMessage("A magical force stops you from teleporting."); } { teleportToX = 3485; teleportToY = 9483; sendMessage("You teleport to the kalphite area."); }

and if you want to change what the emote says you go into your texthandler.class file and hit ctrl+f and type in case 169 (for example) and it should come up with something like this:

p.sendFrame126("bow",169);

and to change that you change “bow” to whatever you want example:

p.sendFrame126("kqueen",169);

or if you want it to be colored you can add in @red or @blu or @whi or @gre example:

p.sendFrame126(@red"kqueen",169);

and i just made the bow emote teleport you to the kalphite queen and instead of the emote tab saying bow it says kqueen (in red) all of this is just an example though so do not try to input these codes just follow their example!
