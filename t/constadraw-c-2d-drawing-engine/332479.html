ConstaDraw - C# 2D Drawing Engine

General Programming
Lin July 31, 2016, 3:13pm #1

This is a small drawing engine I started writing last night to draw Gameboy-like games (Which would explain the default window size) in C# without the use of DirectX, OpenGL, or something else, as these are all highly incompatible. Instead, it uses the GDI+ to produce the best results possible, or run just like the Gameboy. Of course, I don’t believe this is completely optimized, but it sure makes one hell of a difference.

So here’s how it works. There’s two buffers. One is a background, which is used for drawing maps or just something in the background, which really speeds it up later on. This by default does not support transparency. Anyway, it’s drawn unmodified and doesn’t get cleared after redrawing. However, the other buffer, the surface, does. This one DOES support transparency (Magenta. It does not use an alpha channel as this would conflict with the back buffer) and after every redraw, it’s cleared.

After both buffers are ready to be drawn, a regular .NET Bitmap is created. Unsafe code allows the use of directly modifying its data, so it just copies the back buffer to it. Then, the surface area buffer is copied, and magenta-colored pixels are skipped. Since the graphics object for our window will never change (Unless of course, you make it), the one created on start-up is used and it draws the final image.

Download Full Source with Executable

The testing code is a little sloppy, but everything is self-explanatory. Don’t question how to use this, and if you do, it’s clearly not for you. Later, I’ll be expanding this and creating a full game engine, but for now, you can just play with this. The frame rate depends on the interval of the first timer, which is set to 30 by default.
