Unhandled July 30, 2016, 8:05pm #4

I used to try making an amazing theme, then code something up (like my blog), and it inevitably ended up sucking. I tried designing a REALLY simple theme, (like this) for my CMS/templating system, and so far it’s be working out really well. I’d design a nice looking theme, but don’t try curved corners, or drop shadows etc, just make it look as simple as possible. But you must design your code so it can be easily themeable when your done coding (templating, MVC). When you have your code working, start designing a cool theme, get your theme working, then apply the theme to your code. I personally like the PEAR HTML_Template_IT package, because it’s very simple to use, and all you have to do is go {SomeTemplateVar} in your markup.
Good luck.
