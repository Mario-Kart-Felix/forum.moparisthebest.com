Delete the session cookie which is probably keeping it alive. session_destroy() doesn’t do this for you.

Also, just some suggestions:

[ul][li]session_register() is actually deprecated, so just use the good ol’ $_SESSION superglobal array to set that username value.[/li]
[li]After unsetting the session in your logout script, it is generally a good idea to set the $_SESSION array to an empty one,
[/li][/ul]

Remove session_register() and instead set a new value in the $_SESSION array.

You did set a session cookie when you called session_start() at the beginning of each script, which means a session cookie was automatically created in the browser. It is even outlined over at php.net:

In order to kill the session altogether, like to log the user out, the session id must also be unset. If a cookie is used to propagate the session id (default behavior), then the session cookie must be deleted. setcookie() may be used for that.
[code=php]setcookie(session_name(), '', time() - 3600, '/');[/code]
