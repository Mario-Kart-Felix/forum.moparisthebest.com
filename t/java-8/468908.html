I’m kinda disappointed in Oracle. The only good feature in Java 8’s release (which is believed to be in September 2013) will be lambda expressions. I will give them credit for just that though. Lambda expressions are awesome, in the sense that they remove a lot of verbosity that is often associated with programming in java.

I’m sure there are probably other minor updates that the average programmer won’t notice, such as changes to the garbage collector, or other JVM tweaks.

Discuss your opinions.

Here’s an example of lambda expressions in Java 8.

[code=java]public static void main(String[] args) {
List names = Arrays.asList(“hikilaka”, “misterfap”, “kermit”, “eddie”, “fenex”);
List sortedNames = names.sorted((a, b) -> a.length() - b.length()).into(new ArrayList());
forEachReverse(sortedNames, System.out::println);
}

private static void forEachReverse(List list, Block block) {
Objects.requireNonNull(list, “List may not be null!”);
Objects.requireNonNull(block, “Block may not be null!”);
for (int i = list.size(); i > 0; --i) {
block.apply(list.get(i - 1));
}
}[/code]
