[quote=“tyb97, post:28, topic:468908”]I don’t even understand how lambda expressions work… I know it’s something to do with -> but, what it does, I have no idea.[/quote]say we have an array of some sort

and we want to iterate it and print out the shit within. traditionally and imperatively you’d do this

for (int i = 0; i < array.length; i++) { System.out.println(array[i]); }

however, that’s quite verbose and we’re explicitly stating the order in which the array is being iterated through with an ugly temporary variable used simply as a record for us to know which array element is being accessed (i)

enter: a foreach function, which is a function that takes a function and applies it to all elements of the list. think of it something like (in the case of an int[]) the following pseudocode

int[] foreach(function func, int[] array) { for (int i = 0; i < array.length; i++) func(array[i]); }

now the function bit is a bit of wizardry, because we’ve now encapsulated the explicit looping with its own function and now we can reuse the foreach function without having to write a new loop with a new temporary i variable

we can now write

foreach(System.out::println, array); to print out all the elements of our array (the :: bit is just some syntax ignore it)

anonymous functions/lambdas become useful here because what if we want to print out our array with an exclamation mark at the end of every element instead? well, without lambdas we’d write up a new function

void printNumber(int n) { System.out.println(n + "!"); }

and make the foreach call as

however, that requires explicitly naming a function, which can be inconvenient and irritating if the function is only used once. basically it’s a waste. so enter functions without a name that can be used once and never again, these are basically lambdas. with our new found syntax, we can basically do

and now we’re not wasting the name & space on a function that we only use once, as we’ve now made it anonymous.
