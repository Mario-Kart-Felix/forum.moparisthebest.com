Logging in/out stack error fixed
Holy crap. This would have been the bug to end it all. I was considering dropping Asteria because of how simple this was to fix. What was happening was, when the player experienced a packet/buffer issue, it would call the disconnect() method which would call the logout() method, which would call the logout packet! and what happened? everything just repeated itself: disconnect() -> logout() -> logout packet -> disconnect() -> logout() -> logout packet ‚Ä¶ and so on, until a StackOverflowError was thrown. For some reason this caused everyone online to disconnect and the server to completely go into an ‚Äúlost‚Äù state and stop accepting any connections! I fixed this by making sure that if a player is disconnected from a packet/buffer issue, that no more packets are sent to the player and they just disconnect without the logout packet, but instead with the packet/buffer issue!
