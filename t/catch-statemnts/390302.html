Basically, try-catch block is used when handling exceptions. An exception is an event that is triggered when something does not behave as it should. Take the following code for example:

try {
    FileInputStream input = new FileInputStream("derp.txt");
    System.out.println("apple");
}
catch (FileNotFoundException e) {
    System.out.println("File derp.txt was not found.");
}

In this example, a FileInputStream object is created for derp.txt. If there was no file named derp.txt, then the program would throw an Exception that said so. So in practice, if there was no file by that name, “apple” wouldn’t be printed to the screen, and the code inside the catch statement would be executed instead.

Another way (and probably the most accepted way) to think of it is as a software interrupt. When the exception is “thrown” it interrupts the currently executing code, and “catches” the exception.

public final class Item {
    private int id;
    private int amount;

    public Item(int id) throws IllegalArgumentException {
        this(id, 1);
    }

    public Item(int id, int amount) throws IllegalArgumentException {
        if (id < 0 || id > 7956) {
            throw new IllegalArgumentException(String.format(

Just use the throws when you don’t want to handle the exception within that method AND if one exception gets called the whole code should stop, if it doesn’t matter about a single exception you should use the try-catch block. (See examples).

Example1;

import java.io.IOException;

/**
 * Created by IntelliJ IDEA.
 * User: Steven
 * Date: 11-Apr-2011
 * Time: 02:35:04
 */
public class Example {
    public static void main(String[] args) {
        Example ex = new Example();

        ex.throwException();
        ex.throwSecondException();
    }

    public void throwException() {
        try {
            throw new IOException("My Fake Exception, lul.");
        } catch (IOException e) {
            System.out.println(e);
        }
    }
             
    public void throwSecondException() {
        try {
            throw new IOException("My Second Fake Exception, lul.");
        } catch (IOException e) {
            System.out.println(e);
        }
    }
}

In this example both methods are called as each exception is handled seperetly.

Example 2;

import java.io.IOException;

/**
 * Created by IntelliJ IDEA.
 * User: Steven
 * Date: 11-Apr-2011
 * Time: 02:35:04
 */
public class Example {
    public static void main(String[] args) {
        Example ex = new Example();
        try {
            ex.throwException();
            ex.throwSecondException();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }

    public void throwException() throws IOException {
        throw new IOException("My Fake Exception, lul.");
    }

    public void throwSecondException() throws IOException {
        throw new IOException("My Second Fake Exception, lul.");
    }
}

Whereas in this example both exceptions are handled within the same try-catch block, only the first method will get called because then the code skips to the catch section skipping the second method.

So yes, there are some differences.

NOTE: Even when using the throws keyword… You’ll have to catch the exception eventually, unless you just keep throwing the exception round the whole program untill it gets back to the main method.
