[quote=".Fusion, post:1, topic:162520"]Yeah I started learning C++ the other day but I keep hearing from people it’s best if you learn java first and if you want to, go ahead and learn C++.

If you want to learn a language, learn that language. C++ is indeed larger, and hence, harder to master, than Java, but if you want to learn C++, just go for it. C++ and Java share a similar syntax (as Java took its syntax from C++), though Java has had many features of C++ removed, which makes Java easier to learn but less powerful.

Regarding speed, Java could theoretically be as fast as C++. But in reality, that is not necessarily true. I and everyone else on this forums have our own biased opinions on the matter, and our own favourite languages, so you probably shouldn’t take our opinion too seriously. After all, we don’t lose or win anything by giving a right or wrong one.

However, software companies do have much money at stake, and qualified proffessionals, so these are probably more trustable sources.

If you investigate what these companies use, you will probably find out that the most used general-purpose languages are C++, Java, and .NET, depending on the activity of the company. If they develop non time-critical applications for Windows, you will very often find that they use .NET, because it is relatively fast to get a stable program going, and it isn’t hard to find decent .NET programmers. If they develop web-related applications, you will find that very often they will use Java. And finally, if they develop applications which are especially complex, or which need to serve as a framework, or which need to be as efficient and fast as possible, they use C++.

An example of such a time-critical and complex application is, simply, a game. Basically every game which doesn’t have especial requirements (such as being able to run on mobile phones, or on browsers) is made in C++. Note that this is not my opinion, it is a fact, which you can check yourself. Every single Blizzard game, including WoW and its server, is made in C++ (http://www.blizzard.com). Same for Sony Online Entertainment (Everquest2, Vanguard, SWG…). Same for ID (Quake 4, Quake 3), Valve (Half Life 2), Electronic Arts, etc.

(Note that, some older games, when C++ compilers were slower, were done in C).

Draw your own conclussions

(Note2: Programs which require a VM, such as Java, are, and probably always will be, less memory efficient than compiled programs. Though that might not seem particularly important for small programs, it can make a huge difference in performance for the programs that require a medium to high ammount of resources (again, such as games).
