solarwind July 30, 2016, 5:49pm #43

Java:

  • Easier to learn at first.
  • Does not have pointers.
  • Has transparent memory management (user does not have to worry about the creation/deletion of objects to prevent memory leaks).
  • Runs in a VM and CAN use “just in time” compilation to compile to machine code.
  • Generally slower than C++, but this should only be a worry for time critical applications (games, engines, physics simulators, particle dynamics and so on).
  • Requires wrappers or modules written in C/C++ to work with system devices. (See JNI, SWIG).
  • No header files required, easier concept to understand at first.

C++:

  • Almost the exact same syntax as Java (Java’s syntax is derived from C++'s syntax).
  • Runs natively (very fast).
  • Has pointers (they may seem confusing, but not really if you read up on them).
  • User has to worry about memory management in order to make an efficient program that doesn’t leak memory.
  • Can do everything Java can and possibly more.
  • Can directly interact with system devices (usb ports, parallel ports, serial ports and so on).
  • Uses a preprocessor (another thing on your mind, but advantageous if you know what you’re doing.
  • Can use header files.
  • YOU DO NOT NEED TO USE AN OOP STYLE PROGRAMMING WITH C++!!!

Both are statically typed, highly structured languages and are very similar as far as object oriented programming goes.[br][br][size=1]Posted on: December 17, 2007, 03:59:45 PM[/size][hr]Personally, I use both.

On linux, if I need to program my PIC micro controller and create an interface to easily program it, I use C and C++. There’s no way in hell I’m going to use Java for that since it can’t do that natively.

On the other hand, if I want to create applications to be portable, easy to use, have GUIs and so on, I use Java.

It depends on what you want to do. If you tell us what you want to accomplish by learning one of these languages, we can point you in the right way.

Personally, I recommend that you learn Java FIRST, as it forces you to learn good OOP programming structure that will help you when you learn C++.
