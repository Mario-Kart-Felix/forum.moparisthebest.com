My initial observations were that he is trying to utilize his own session handler, but it seems to be unbeknownst to him that there is a session_set_save_handler function in PHP in which you can define which functions handle each aspect of sessions.

Here is a good example of a session class (I found this somewhere, but modified it extensively). Sessions are handled through MySQL in this example, which I highly recommend for anyone who needs to utilize sessions in their website, as having the session data stored on MySQL is better than having it stored on the file system.

[code]<?php
require_once(‘class.db.php’);

class Session {
var $table_name = ‘tbl_Session’;
var $lifetime;
var $sess_save_path;

function __construct()
{
	// Read the maxlifetime setting from PHP
	$this->lifetime = get_cfg_var("session.gc_maxlifetime");

	// Register this object as the session handler
	session_set_save_handler (
		array ( $this, "open"    ),
		array ( $this, "close"   ),
		array ( $this, "read"    ),
		array ( $this, "write"   ),
		array ( $this, "destroy" ),
		array ( $this, "gc"      )
	);

}

function open($save_path, $session_name)
{
	$this->sess_save_path = $save_path;

	return true;
}

function close()
{
	return true;
}

function read($session_id)
{
	$time = time();
	$db = new db();

	$sql = "SELECT Data FROM " . $this->table_name . " WHERE SessionID = '" . $db->escape($session_id) . "' AND Expiration > $time";

	$result = $db->get_row($sql); 

	return $result->Data;
}

function write($session_id, $data)
{
	$db = new db();

	$time = time() + $this->lifetime;

	$sql = "REPLACE INTO " . $this->table_name . " (SessionID,Data,Expiration) VALUES ('" . $db->escape($session_id) . "', '" . $db->escape($data) . "', '$time')";

	return $db->query($sql);
}

function destroy($session_id)
{
	$db = new db();

	$sql = "DELETE FROM " . $this->table_name . " WHERE SessionID = '" . $db->escape($session_id) . "'";

	return $db->query($sql);
}

function gc()
{
	$time = time();
	$db = new db();

	$sql = "DELETE FROM " . $this->table_name . " WHERE Expiration < '$time'";

	return $db->query($sql);
}

}
?>[/code]

Here is the SQL for the session handler:

CREATE TABLE IF NOT EXISTS `tbl_Session` ( `SessionID` varchar(32) NOT NULL, `Data` text NOT NULL, `Expiration` int(11) NOT NULL, PRIMARY KEY (`SessionID`) ) ENGINE=MyISAM DEFAULT CHARSET=utf8;

I’ll only comment on the password. Firstly, MD5 (and SHA1) is a hash, not encryption, and there is a big difference. If you encrypt something, that means you can decrypt it, and get the original back. If you hash something, you should never able to get back the original, that is the purpose of a hash, and whenever you ask for passwords you should always hash them.

Secondly, you should never just hash a password and call it secure. MD5 can already be bruteforced fairly easily, almost instantly with pre-computed rainbow tables, and sha1 isn’t far behind. You should always, always use a different per-user salt everytime you hash a password. SMF for instance uses something like SHA1(username+SHA1(password)), which is perfectly acceptable. You should do something similar.
