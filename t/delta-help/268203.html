Classes Modified: Client.java Difficulty: 1/10 Required Knowledge: Reading and pasting

Step 1:
We want to open client.java and search for

if (inTrade && (System.currentTimeMillis() - lastButton > 1000)) {

You should see:

if (inTrade && (System.currentTimeMillis() - lastButton > 1000)) { lastButton = System.currentTimeMillis(); getClient(trade_reqId).sM("Other player has declined the trade."); declineTrade(); }

Ok what we want to do now is remove the time interval (in this case lastbutton)

What will take effect when we remove it would be that if you press accept and decline within 1000 milli seconds then the second button click won’t “count” as a click. But even though it does not count the close window proceadure is activated and the trade window closes for the player who did this. But the players are still in the trade.

Which allows the dupe glitch

if we remove the above code (what you shall remove is alittle below this)
when we click accept and decline fast it will still decline the trade since the second click counts and the glitch is fixed!

Here’s what it should look like after you have removed the timemilli.

if (inTrade) { getClient(trade_reqId).sM("Other player has declined the trade."); declineTrade(); }

So basicly we just remove everything that has anything to do with “last button” and “timemillis”

if we remove this, it will not have a timer on for when you can decline so as soon as others decline the trade, the interface goes away.
