Games

tyb97 August 4, 2016, 3:04pm #1

Alright, so I had an idea the other day. What if we constructed a server browser client, that networked out into a ton of other servers? Basically it would be a full screen application (default preference), that allowed you to browse a seemingly unlimited amount of servers, and forcing the servers on this list to utilize “project omega’s” (name that just sounded cool) API to send current players online, and tags and such along with it, you could browse all kinds of different private servers. Filtering by tags, players, etc. etc… Have favorite lists, blacklists, maybe even friends that show what server’s they’re playing (?)… When you chose a server, the client info and cache would be downloaded by the server browser (to avoid stupidity in some client’s cache downloading setup’s), and then the client would start in-frame in the full screen server browser (full-screen to support server’s with full-screen client’s) and you could play regularly.

Thought’s on this idea? Would it work? Would it be useful? Could it make a difference? That brings some other questions in like: If this were made, would it need to be moderated with quality control, or would it just be freedom to whatever developer or idiot wants to have his server on the list?
