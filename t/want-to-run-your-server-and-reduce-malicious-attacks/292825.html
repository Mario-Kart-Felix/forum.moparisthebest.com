darscape July 31, 2016, 10:03am #9

Caching is your best bet for web servers. Wordpress users have a mod for caching their posts when Digg puts them on it’s front page. PHP accelerators work pretty well in this case too. E-accelerator is my personal favorite. I can set it to cache a couple hundred megs of PHP scripts in RAM and that helps reduce the pressure a ton. You also would need to look at your MySQL server. Most default installs only are set for at most 20 connections, others more. You can use a firewall to prevent connections to MySQL from external IP addresses and whitelist only allowed ones. A custom installation of MySQL can also be set so that it can accept upwards of about 1000 connections. I am unsure if a forum only uses a single connection to a database, same with the RSCD server, but if you find your forum being pounded by an attack, you could also set it to use a persistent connection so there will not be a flood of SYN-ACK where it connects and disconnects very rapidly which overwhelms MySQL and crashes it, along with your RSCD server.[br][br][size=1]Posted on: August 18, 2009, 07:58:26 pm[/size][hr]
