	for (int i = 0; i < 500; i++) {
		for (int j = 0; j < numbers.length; j++) {

				works[j] = true;

[code=Java]static double[] numbers = { 7104, 3087, 6208, 3496 };
static boolean[] works = new boolean[numbers.length];

public static void main(String[] args)
{
	for (int i = 0; i <= 710500; i++) {
		double f = (i / 100.0);
		for (int n = 0; n < numbers.length; n++) {
			if (numbers[n] % f == 0)
				works[n] = true;
		}
		if (works[0] && works[1] && works[2] && works[3])
			System.out.println("The number " + f + " works!");
		for (int j = 0; j < works.length; j++)
			works[j] = false;
	}
}[/code]

Output:

The number 0.25 works! The number 0.5 works! The number 1.0 works!

I divided by 100 to prevent this: http://stackoverflow.com/questions/5257166/java-floats-and-doubles-how-to-avoid-that-0-0-0-1-0-1-0-9000001
