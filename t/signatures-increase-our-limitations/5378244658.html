Your theories sound alright, limit of a single, non-gif image of a particular (small) size. In reality though, you have to ask yourself how you can make this happen.

  1. GIFs: You can disallow remote images to end in .gif, doesn’t matter, you make a remote .jpg, firefox sees it’s really a .gif and animates it anyway. You download the remote .gif and check, they change it later.
  2. Size: You add width/height to the image tag, the browser still has to download a 20mb 40 megapixel image and just resize it, page load times crawl.

The only way to implement any of these checks in a sane way is similar to how avatars work, you have to write code to allow people to upload images, then you can see if they are gifs and resize them accordingly. That’s a lot of code I’m certainly not going to write, however, if you want this feature, have at it:
https://github.com/moparisthebest/moparisthebest.com-smf
Also make an individual option to disable viewing images in signatures so people who don’t want to see them don’t have to, and I’ll merge it in.

Sound fair?
