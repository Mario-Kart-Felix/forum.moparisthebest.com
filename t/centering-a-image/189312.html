Following Taharoks idea:

	private Point getScreenCenter()
	{
		Dimension screenXY = Toolkit.getDefaultToolkit().getScreenSize();
		int centerX = ((int)screenXY.getWidth() - (int)this.getSize().getWidth()) / 2;
		int centerY = ((int)screenXY.getHeight() - (int)this.getSize().getHeight()) / 2;
		return new Point(centerX, centerY);
	}

I created that for my current program. It’ll return a Point that will “center your component.” But the point is not exactly in the center of the screen, its a little north-west from there to fit with your component’s size, making sure the component is centered.

mod_taharok February 18, 2019, 1:28pm #8

If you divide an odd integer in half, it is rounded up one. If 3 is divided by 2, the result will be 2. And since we are dealing with pixels, one single pixel out of the ordinary will make no difference.

And plus, the painting method parameters are normal integers, not floating-point integers.
