lothy August 1, 2016, 8:18am #9

Long story short, NTFS uses a paging file which references every file on your system. When you delete a file, you’re merely removing the reference from the paging file.

In short, you should be able to use a file recovery application to find the deleted file, and then remove it properly from there by doing several passes over the area of disk that the file is stored on (ie, writing 0s).

moparisthebest August 1, 2016, 8:21am #12
nade:

Open that file, replace all data with other characters - save. Is decent enough. I don’t think any commercial data recovery software will be able to show you the previous data.

Firstly, there is no guarantee that when you do that the file editor isn’t just saving it as a new file, deleting, and renaming, in fact, many do that, so it would be no better than deleting. (perhaps worse, because you don’t know if it just deleted it)

Secondly, I think fancy commercial data recovery like that available to law enforcement forensic labs and such will most certainly be able to recover at least something, since, even if a file is over-written, sometimes there are magnetic remains left behind to gather some information from, this is why ‘secure deletion’ utilities generally write over that area of the disk hundreds or thousands of times with random data, hoping all traces are deleted.
