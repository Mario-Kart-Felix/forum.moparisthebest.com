Different RuneTek engines include different schemes of client set ups, such as the string utility that started in 3 and then was removed in 4. RuneTek 5 always had bloom in it, I think RuneTek 4 did too I’d have to check my clients, but then it wasn’t added in until much later into RuneTek 5. The start of RuneTek 3 had the old cache system and then towards 414, they starting moving to the most modern JS5 system. In the middle of RuneTek 3 they changed their software rendering a bit and how the layout for that looked. There are a lot of changes but the most common thing I see when Runescape declares a new ‘RuneTek’ engine is the set up of how each class file is declared. It’s really easy to see that they script most of their client, and I think as they come out with new versions of their scripting languages (possibly Jagex Script 5, dunno) they release new engine versions for their client.
