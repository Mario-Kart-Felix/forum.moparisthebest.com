I have decided (very wisely) to make a CVS for Hybridscape
Now for the questions

WTF is a CVS?

A CVS stands for Concurrent Versioning System. CVS is an open source version control and collaboration system.

ZOMG?! WTF?! I still don’t understand

Ok, remember SVN from Aryan? Well, CVS is similar but actually quite different
First of all, only selected people can access, and commit(upload) to the CVS (And those ‘select people’ are going to be the Devteam)

Whenever the Devs commit to CVS the CVS database tries to merge the changes with the current version in the CVS. (To commit you need a program like Tortoise CVS(not SVN))

For example lets say the only code(not exactly code… :P) in the CVS is this:

Whenever you see Joe hit him.

And then someone uploads their own version:

Whenever you see Joe eat him.

It will merge them to:

Whenever you see Joe hit him and eat him.

Now, obviously this is going to have some flaws. And that’s the reason it is not openly accessible.

So, after about 1week, (or half a week) MissSilabSoft will download the whole CVS, check it for bugs, and post it in a topic on the Forums; and then let people beta test it!
