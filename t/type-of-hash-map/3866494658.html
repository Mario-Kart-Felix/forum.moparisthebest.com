Even if you were right (and trust me, you aren’t - everyone else is), the use for it that you listed (player -> object mappings) is redundant.

The only time you need to map x to y, at the end of the day, is when you don’t control the implementation of x.
If you control x then you can simply add a field in x such that a HasA relationship is established between x and y. In other words, x HasA y. You then no longer need a mapping of x -> y.

So as an example, let’s say you have two classes: Player, and Inventory.
You could a) Map<Player, Inventory>
but because you have control over the implementation details of both classes you can
b)

class Player {
  Inventory inventory;
}

class Inventory {
  ...
}
