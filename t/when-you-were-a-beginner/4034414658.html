[quote=“T4_, post:20, topic:403441”]Other than slight convention changes and obviously different libraries, C# is the about the same as Java.[/quote]In some aspects, better, since it has a lot of the features that Java programmers have been wanting for a while; e.g. operator overloading, strings in switch statements, etc etc

Personally, I like that I don’t have to create an entire object just to have a callback, I can just supply a method’s name. e.g.:

Java:

public interface MessageHandler {
    public void handleMessage(String s);
}

.... random codons ...
MessageHandler handler = new MyMessageHandler();
handler.handleMessage("what?");


.... another file ...

public final class MyMessageHandler implements MessageHandler {
    public void handleMessage(String s) {
        System.out.println(s);
    }
}

Whereas in C#:

public delegate void MessageHandler(string s);
public event MessageHandler MessageReceived;

private void MyHandleMessage(string s) { Console.WriteLine(s); }

... random codons ...
MessageReceived += MyHandleMessage; // Attaches this message handler to the event.

MessageReceived("What?"); // Calls all the message handlers attached to the event.
