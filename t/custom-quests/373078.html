the_wanderer August 1, 2016, 1:31am #2

Matters the complexity of your quest you want to make. Also this is in wrong section? Anyways. Say you want a custom boss. You would have to set some texture skins and some vertex skins. Then make some priorities. Then you want him to attack right? So you would upload it into RsPoser and make some animations. Then say you want a custom item for killing that boss. You would have to model that and animate that. Say the weapon is held weird and there are no standard animations for it. Once again you would have to open RsPoser, import a character model with animation data, with the weapon on him for reference, and create some walk, stand, run, attack, and block emotions. It would be a great graphical process. Coding? Meh just calling interfaces and text. Unless you want it more complicated. Like I said in my very first sentence.

RealmofTerror August 1, 2016, 1:32am #4

It can be various, but the main ones consist of: client.java, autospawn.cfg, and texthandler.java.

Client.java: Anything from the dialogue to the character quest tokens
Autospawn.cfg: Where you spawn the npcs for your quest
Texthandler.java: To change the text in your quest tab

Like the wanderer said it can be various based on the complexity of the quest.
