General Programming

I’ve only noticed this recently but swing uses char[] for passwords. I’ve read online that it’s better for security then using Strings because Strings are immutable and that the password would be present in memory where with an array you can completely get rid of it.

Strings are also plain text which is easier for an attacker to read.

Simple example:

pass = "test"; print(pass);

pass = "test".toCharArray(); print(pass);

The first code would print the password where as the second code would print something @[2945}[

What are your opinions?
Are there any better methods for passwords?

xx_matty_xx August 4, 2016, 1:12pm #5

I don’t think the memory thing is a valid justification. The point is that you’re less likely to accidentally dump passwords to logs if they’re in an array. Writing nulls to a char array doesn’t guarantee that the memory is completely erased either, and even if it did it’s still only a slight security improvement. If your system is vulnerable to memory dumps then you have more important things to worry about.

However, here's the caveat: Objects of type String are immutable, i.e., there are no methods defined that allow you to change (overwrite) or zero out the contents of a String after usage. lothy August 4, 2016, 1:12pm #13

C# has the same problem. They’ve provided a SecureString class to attempt to rectify it, but various APIs aren’t compatible with it so ultimately you end up converting it back to a string anyway.

Also, for your solution to have any value, you can’t start from a string literal (i.e., “Test”) and then convert it to a character array because by that stage you’ve already stored it as a String in memory.
