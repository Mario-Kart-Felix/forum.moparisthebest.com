Javacisnotrecognized July 30, 2016, 5:32am #11

AWT delegates all the drawing/component/w/e stuff to the host platforms GUI system. Each call to show() or hide() is just a wrapper for ShowWindow or what have you. Swing puts more of the processing into Java code so that you can set the flags, L & F, and other things at runtime. In this case, some graphics program that will probably be rendered by Protostribes own code, there is really no need to involve swing when all you need is a graphics surface to draw on.
