[quote=“RdSoxFan618, post:12, topic:4600”]But anyway, I agree, Notepad. Not worth wasting $299 for Dreamweaver, although it is excellent!

First of all, noone requires you to pay $299 to obtain Dreamweaver. I have Dreamweaver 8 and I like it a million times better than notepad. Notepad is good for quick editting tiny things, but large scripts are barely readable. The syntax colouring is definately required when working with large scripts.

Secondly, Dreamweaver does not work like Frontpage AT ALL. That’s just a complete lie. Frontpage completely stinks, Dreamweaver does not.

Frontpage creates very ugly code, and Microsoft probably never heard of W3C standards. Try opening a blank HTML file in Frontpage and you’ll notice that just the opening action will create code in the file!! Just opening it! It will infect the whole file with its stupid Generator META’s. That’s because everyone must know that it was Frontpage who created the file.

Furthermore, Frontpage has these extremely weird rollover codes. Suppose you want to make a link change color when hovered over with your mouse cursor. There is a simple CSS parameter that does the trick. But no, Frontpage must create weird Javascripts to get the job done. Note that the CSS consists of only 1 line, where the Frontpage ‘Dynamic Effects’ consist of entire Javascripts with a lot more lines of code.

Honestly, I once made a piece of code for someone, and he editted it once in FP. I gave him the advice to ditch FrontPage, since the simple edit code litteraly DOUBLED the filesize.

I am a very experienced webdev, and I can tell you that Dreamweaver is the best, by FAR! Notepad is a good alternative, but Dreamweaver’s functions like Syntax Colouring are what makes it much better. On top of that, Dreamweaver handles PHP excellently, while FrontPage has no support for it whatsoever.

I must also note that completely learning HTML is impossible. I am very experienced in HTML and occasionally, I still forget how a certain tag looks, or how it is spelled/used. Dreamweaver fixes that problem, by popping up a list of the tags that start with the characters you already typed. I can then say ‘Oh yeah, that’s the one I was looking for’ and just click it and it’ll complete the tag automatically.

Also, Dreamweaver places no extra code in your files. It just adds some lines that are required to make the document W3C proof. You’d have to add them anyway, so Dreamweaver does that for you. I think that Dreamweaver has the best solutions to achieve certain things. It automatically adds CSS when you do layout work, where Frontpage would use tags. (which are deprecated)

You can completely leave the ‘design view’ alone, and just use the ‘code view’ to directly type the code, as you would in Notepad. That’s what I do and I like it.

My advice, buy/download Dreamweaver. The entire Studio is even better. Fireworks and Flash rock for making leet graphics for your site.

Regardless of what you’ll choose, good luck with your websites!
