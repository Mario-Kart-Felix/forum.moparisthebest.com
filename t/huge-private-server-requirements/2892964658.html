andrew4444:

For the main questions i have i am wondering if the batch file that recognizes connections is enough to process so many connections.

The batch file itself doesn’t process connections, the client handling code is in the .java files, which are then compiled and executed using the batch file.

andrew4444:

Especially HD clients.

HD clients aren’t intensive on the server side, the graphic upgrades are handled on the client side.

andrew4444:

What do you think? Are there any other server specs i should get that may be a major factor before hosting with this company?

The bandwidth/disk space/RAM you have is suitable to handle 300 clients. You have more than enough disk space, and you likely won’t be getting anywhere near that amount of bandwidth usage in a month. More RAM would help, but since it’s a dedicated server and you won’t be running any of your own apps, it should be fine. The only thing you need to check out is how many connections you can have open.

Also, in regard to hosting from home vs. hosting on a dedicated server, hosting on a dedicated server is preferrable since internet failures / power outages happen much more often locally. You also get the ability to turn your computer off without disrupting your server, saving you a significant amount of money for power usage.
