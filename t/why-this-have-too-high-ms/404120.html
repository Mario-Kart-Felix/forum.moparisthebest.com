zyle1992 August 1, 2016, 1:11pm #5

It’s because PI is a piss poor piece of shit.

More than half the shit in there can be fixed, or improved, just by changing the data type.

Example:

public int inWilderness = 0;

Could be improved to be

public boolean inWilderness = false;

Another big problem I’ve noticed (even in Hyperion) is that PacketHandlers are created for each player. In other words, everytime a packet is received, the player handles a new instance of the packet handler, specific to them. It shouldn’t be like that. Instead, you should implement the Singleton pattern with the packet handlers (or static methods if you prefer) and just send the player as an argument.

Although, there are many more problems than I am willing to explain here.
