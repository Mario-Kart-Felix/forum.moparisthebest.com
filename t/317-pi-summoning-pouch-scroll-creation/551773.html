There are much better ways you can check items and etc, but that is just an example.

One way you could do this is store all the summoning data into something like an enum. Store how many pouches are needed, charms, and etc.
Then, in your summoning class, create an array of integers that are used for the dialogue ids…
Now, create a method that requires the dialogue id and the action button id as parameters.
First, you need to check if the dialogue id is the correct. so create a for loop to check that.
Second, you will need to grab the summoning data based on the action button clicked and dialogue id… This is going to be handled in a second method that returns the summoning object (or null if none).
Third, you need to check if the player has all the items… It’d be similar to how I showed above, but instead of specifying specific IDs, you’d just use the values from the summoning data.
Fourth, check the player level.
If all that’s good, delete the items, give the player the items, and give them exp.
