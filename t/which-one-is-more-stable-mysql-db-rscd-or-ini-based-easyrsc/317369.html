CodeForFame July 31, 2016, 12:53pm #9

Reading ini files is slow in java, especially the way it’s set up. Php would most likely be slow as well, especially for things such as high scores.

On the other hand, databases are quick, and easy to manage, but require more overhead. When getting a lot of data that requires conditional selection (such as high scores, getting player data, ect), you will definitely be outperformed by a database.

As for why people use EasyRSC over RSCD, the only reason I would use the former would be if I didn’t know how to set up a database, and didn’t feel like learning.
