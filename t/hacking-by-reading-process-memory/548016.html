Using C++ (I’m sure you can use other non-VM languages to, but I’ve only done it in C++) you can read the memory area of other programs. Everything you need should be available with #include . However to get the values you need I would recommend using Cheat Engine since it is a bit tricky to find what you need on your own. Once you get the memory address and a way to get them, since the location change every the the game starts, you can use it for what you want to do.

Since you’re most used to Java, I’m sure there you be some simple way of transferring the info from a simple C++ to a Java app that actually uses the information. Sockets are the first thing that comes to my mind, but I’m sure there is other alternatives.

You should however note that LoL might think you’re cheating (It might be looking for cheating programs like Cheat Engine), so do not do use that program with your real account.

To extend on to Bowser jr’s suggestion on using C/C++ using the Windows API, the steps look sort of like this:

  1. Since the process ID of the game won’t be constant, you can enumerate through your processes until you find the LoL process. You call CreateToolhelp32Snapshot() to create a snapshot of your current processes and enumerate through them using Process32First() and Process32Next(). An example can be found here. The important part here is how they get access to each process’ name and ID. Save the process ID for the next step.

  2. Use OpenProcess() to get a handle to the LoL process; just pass the process ID and the kind of access you want to the process (read, write, etc.). You can see an example using OpenProcess() in the above example. You might need to enable a sort of “debugging” privilege to do this but I can’t remember this step very well.

  3. Finally to read the memory of the process you can use ReadProcessMemory() to read the contents of LoL’s memory.

  4. Don’t forget to close all handles.

Sorry if this isn’t very specific, the Windows API requires alot of intermediate steps to get this right. :s
