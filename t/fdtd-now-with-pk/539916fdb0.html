fuzzout August 4, 2016, 11:52am #64

Made some updates

Reduced input lag to to near optimal levels.
Created two extra player processes that run on their own threads (less lag, more stuff!).
Adjusted login procedure to better interact with the extra processes (issues during login procedure where too many packets would cause disconnection for those living further away from the server).
Adjusted login procedure to allow multiple logins at once.
Created a separate movement thread per player (no lag! wheeee!).
Removed many pieces of old code and re-structured the layout of processes.
Changed process priority in the server.
Added looting from 1 square away.
Adjusted Special Attack bar to update as frequently as possible without lag implications.
Adjusted the initialization of a new client (you, when you connect) to no longer “mis-understand” the server as well as proceed with the login PROPERLY.
Split packer parser into 3 parts (will be more in the future) to prevent block size limitations.
Added an attack speed display in the attack-type tab.
Re-structured weapon animation codes for Stand animations, Run animations, Attack animations and walk animations.
Created a server connection backlog, now no connections should be denied or bottlenecked through.
