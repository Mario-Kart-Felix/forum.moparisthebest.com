Oh, about API calls

“They aren’t SCAR’s really…
Api calls are calls to functions precoded into the os, the win32 api is what SCAR uses to work, all programs do…
The problem with the api is that the functions are very specific, they each have their own special function and to create for example a simple macro with a findcolor functions you will need several api calls…
These are pretty useless in SCAR though, for the reason that 1 they aren’t standardly enabled (which is good)…
2 They’re alot slower when called from SCAR
And finally, SCAR’s precoded functions are alot more useful to use…
However, you can use api calls in plugins…
Well, you usually don’t even have a choice”
