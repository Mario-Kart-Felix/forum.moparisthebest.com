Brief-

What is an HTML File?

* HTML stands for [b]H[/b]yper [b]T[/b]ext [b]M[/b]arkup [b]L[/b]anguage
* An HTML file is a text file containing small markup tags
* The markup tags tell the Web browser how to display the page
* An HTML file must have an htm or html file extension
* An HTML file can be created using a simple text editor

Why HTM and HTML Extensions?

.htm is a more basic form of HTML. There is less coding, and it can do things that the .html extension can’t. On the other hand, .htm limits the HTML codes you can use with it, meaning that some codes that may work for a .html extension will not work for a .htm extension. I prefer .html, because it enables you to add more coding. Although either extension is fine.

Note For HTML Editors-

You can easily edit HTML files using a WYSIWYG (what you see is what you get) editor like FrontPage or Dreamweaver, instead of writing your markup tags in a plain text file.

However, if you want to be a skillful Web developer, I suggest using a plain text editor such as ‘NotePad.exe’, to learn to primer your HTML skills.

HTML Tags

* HTML tags are used to mark-up HTML elements
* HTML tags are surrounded by the two characters < and >
* The surrounding characters are called angle brackets
* HTML tags normally come in pairs like <b> and </b>
* The first tag in a pair is the start tag, the second tag is the end tag
* The text between the start and end tags is the element content
* HTML tags are not case sensitive, <b> means the same as <B>

Tag Attributes-
Tags can have attributes. Attributes provide additional information to an HTML element.

The following tag defines an HTML table:

. With an added border attribute, you can tell the browser that the table should have no borders:

Attributes always come in name/value pairs like this: name=“value”.

Attributes are always specified in the start tag of an HTML element.

Attributes and attribute values are also case-insensitive. However, the World Wide Web Consortium (W3C) recommends lowercase attributes/attribute values in their HTML 4 recommendation, and XHTML demands lowercase attributes/attribute values.

Always Quote Attribute Values

Attribute values should always be enclosed in quotes. Double style quotes are the most common, but single style quotes are also allowed.

In some rare situations, like when the attribute value itself contains quotes, it is necessary to use single quotes like-
name=‘John “ShotGun” Nelson’

Basics-

When making a HTML document, ALWAYS add this markup at the very top of the HTML doc-

<html>

Adding that helps the web browser know where the HTML coding starts.

At the Bottom of the HTML document, ALWAYS add this markup at the very bottom of the HTML page-

</html>

Adding that tells the web browser where the HTML coding of the web page ends.

Markups-
Headings-
These are the HTML tags for the different sized headings-

Large Heading-

<h1>Large</h1>

Larger Heading-

<h2>Larger</h2>

More larger-

<h3>More Larger</h3>

Even More Larger-

<h4>Even More Larger...</h4>

Second To Largest-

<h5>2nd To Largest!</h5>

LARGEST-

<h6>LARGEST</h6>

Paragraphs-

Paragraphs are defined with the tag.

[code]

ParaGraph1

ParaGraph2

[/code]

HTML automatically adds an extra blank line before and after a paragraph.

Comments in HTML-

The comment tag is used to insert a comment in the HTML source code. A comment will be ignored by the browser, and will be invisible in the web page. You can use comments to explain thing to you in your code, which can help you when you edit the source code at a later date.

Note that you need an exclamation point after the opening bracket ( < ), but not before the closing bracket.
