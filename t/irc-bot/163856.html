I’m bored so I’ll write a little backbone explanation of how the bot works.
Class SettingsHandler loads all of the IRC Bot’s properties from a XML file, and then stores them in Fields. If that returns true, it continues and the bot then proceeds to open up a new Thread, ConnectionHandler, and inside the run method, it connects to the server, using the properties loaded.
Once the bot connects, the bot then calls parseInput in class InputHandler which as you may of guessed, parses the input. If a command is executed, InputHandler then calls parseCommand from CommandHandler, which parses the String containing the command, breaking it up into the host name and the actuall command.
customCommand is then called and parses it, executing whichever command is executed.
