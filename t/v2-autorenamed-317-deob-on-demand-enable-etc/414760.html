yes it would be faster to do the renaming manually. but, zkm automatically renames all the fields corresponding to the class. for example say you had a class file under the name of

“Class123” which you renamed in your IDE, to “ACoolClass”

all occurances of that class would be named as, for example “aClass123_231” or so.

whereas since i renamed with my tool before renaming with zkm and excluded the renamed classes from my zkm rename script, i had done the class automatically named “ACoolClass” where all occurances were something along the lines of “aCoolClass221”, which is a lot easier to read imo. even though you should go rename the fields anyways, it’s a lot easier to understand the code with it renamed like this (imo)
