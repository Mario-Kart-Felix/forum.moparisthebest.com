When I signed my jar file, I used the plain-old command prompt.
“If you didn’t make a Keystore”:

Run your command prompt as an administrator, then type:

keytool -genkey -alias alias-name -keystore keystore-name
BE SURE TO INCLUDE ALL OF THE SPACES AND DASHES! -Where it says "alias-name", make up some random name that you would like to call your alias. -Where it says "keystore-name", make up some random name that you would like to call your alias. ....After you have completed all of that, it will ask for a keystore password, just make one up and remember it.

Then the command prompt will ask you 7 questions. You do not ahve to answer them.
This is what the finishing project should look like

xen0oo.png925×595

^^^^^^^Everything above basically just makes your keystore^^^^^^

Now we will actually work on signing your jar file.
-So back in your command prompt, you will type:

jarsigner -keystore keystore-name -storepass keystore-password -keypass key-password jar-file alias-name
-Where it says "keystore-name",obviously put your keystore name. -where it says "keystore-password", obviously put your keystore password. -Where it says "jar-file" you will put the entire pathway to get to your jar file.... for example:
C:\Users\Obama\desktop\JarFileHere.jar
*everything is case sensitive btw.* -Where it says "alias-name",obviously put your alias name there. So far it should look something like this: [IMG]http://i39.tinypic.com/23rwqc0.png[/img]

The command prompt will then ask you for the keystore password. just enter the password and it will sign your jar file!

I know that people can just use Jarmaker to sign their jar files. . . But I was having technical difficulties with jarmaker and it wouldn’t let me make a keystore so i decided to sign my jar file the old fashion way.

This also might be hard to follow but just imaging having to learn from Oracle.http://docs.oracle.com/javase/tutorial/deployment/jar/signing.html
Well I hope this helps some people like it would have helped me.
