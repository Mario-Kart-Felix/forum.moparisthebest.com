Mopman July 30, 2016, 3:41am #2

I’m not a good coder, but there’s two points I’ll add:

. Some people just can’t read eBooks. I do use eBooks, but that’s because I’m too poor/cheap to spend money on paper copies, however I know I would learn whatever I’m trying to learn faster if I could take it away from the PC. Reading from a screen just dosen’t always work as well. If you’re finding it hard with eBooks, trying buying a decent real book and giving that a go.

. Remember that you can read all the Java programming books in the world and you still won’t have a clue how to code bots. Programming books can only take you so far, most programming-related skills must be learnt after you have got the basics of the language down, either by reading more articles/papers/books or by your own experimenting. As an example of this, I did a year of a Programming course at college and there was two definate groups within the class - those who could program before starting (they learnt nothing at all) and those who knew nothing about programming (they learnt syntax, ie what you could learn from a book, but at the end of the course still couldn’t write a simple application).

My advice would be to give the book another go-through to get the basic syntax and ideas nicely concrete in your head, create a few simple programs without looking at a single piece of example code, unless you really really must and then read up on the topics that pertain more to rs cheating.

Mopman July 30, 2016, 3:41am #8

Example code can be used to learn, however learning to program by reading examples can make people end up being copy&paste programmers who use sample code all the time and don’t develop problem solving skills at all. (Look at most VB coders). This is fine in a way and with so much sample stuff available on the net you can build a lot of programs simply from taking bits people have already written. RS Cheating or anything but simple applications though, is going to involve a lot of finding out of your own and if you’ve never needed to do this you’re gonna be stumped.

It’s like when people recommend you write a Tetris clone or something before you try to get into game programming. Read gamedev and count the amount of people who “write” their tetris clone by copying and pasting someone elses code and then wonder why they haven’t magically gained the knowledge required to write their dream MMORPG. I’m not saying using example code is bad, but it’s probably better left until you can already code, imo.
