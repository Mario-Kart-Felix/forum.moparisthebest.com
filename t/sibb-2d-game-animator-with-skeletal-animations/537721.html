So this is a 2D game I’ve been working on for quite some time. It was made entirely from scratch, no game engines were used. The only resources currently in use are the java library and netty for networking.
A lot of the aspects of it were inspired by various games, most notably Baten Kaitos, Final Fantasy, and Maplestory.

The combat/fighting system will be as follows:

While traveling through areas, there will be monsters spawning on the maps. The maps are very maplestory-esque. If a player runs into one of these monsters, a battle will ensue -- however, it will transition to a battle map for fighting where there will be several monsters to fight. Player health will be a set amount that will not regenerate inside or outside of battle. Although, there will be different character types of course -- healer, fighter, mage. Healers will be able to heal (obviously), however all of the character types will be able to use items on one another for healing. This idea comes from a yu-gi-oh turn-based RTS. The combat system is, however, [b]not[/b] turn-based. It will be free-style, similar to maplestory. This makes for the best multiplayer experiences in my opinion. Players will have a wide variety of skills at their disposal, similar to many modern RPGs. It will not simply be click and attack as that is very dull. Upon completion of the battle, the player(s) will be sent back to the map and location in the map where the battle first began, and the monster on that map will disappear. Monsters will only respawn when all of the monsters on the map have been beaten.

Anyway, I haven’t gotten incredibly far in the development yet – still not even an alpha, really. The framework is made for the most part, though, so content creation isn’t too far off.

Along with this game comes a 2d sketetal animator. Because I plan on having a lot of characters that I don’t want to individually animate or create sprite sheets for, as well as a lot of weapons, I’m creating a program to animate skeletons and attach images to the skeleton. Why did you choose Netty for networking? Why not ditch the ease-of-use and go straight for NIO? You’ll see much better performance without the complicated overhead of it.

eyeownyew August 4, 2016, 11:22am #6
imthenull:

Why did you choose Netty for networking? Why not ditch the ease-of-use and go straight for NIO? You’ll see much better performance without the complicated overhead of it.

Much better performance? The performance would be identical, the only difference would be the amount of code I’m required to write. I can rely on Netty to not have any issues whereas with NIO I’d have to spend time tweaking and perfecting the base. And, unless I were to do it perfectly, NIO would not have the same scalability as Netty; a large number of users would not necessarily be an issue, but it would not be handled as well.
