kihomura July 31, 2016, 5:58am #1

This should be really easy for all of you. This was tested with Project Czar.
Warning: Your NPCs will no longer attack back if you do this tutorial; Will find a fix.

Go into your [tt]NPCHandler class[/tt] and search for

HP <= 0

You should see this (if you’re using the correct base that this tutorial is directed at)

if (HP <= 0) { // This will cause client crashes if we don't use this :) - xero HP = 3000; }

Now, we need to make it so the NPC’s HP is no longer 3000, so we need to remove this if statement. To remove it correctly, select from the closing brace ([tt]}[/tt]) to the word “if” and press backspace.
Now, save and go into your client class.

Go to packet 131.

Now for the actual listening part.

So, we will need an if-statement to start.

Take this for example:

if (HP == 0) { sendMessage("You cannot attack this NPC!"); }

If you were to compile, you would get an error saying that it cannot find the variable ‘HP’. So, we will need to specify which class file the variable is in. In this case, it is our NPCHandler class.

So, we would need to change our if-statement to

if (server.npcHandler.npcs[npcIndex].HP == 0) {

But now, if you notice, you get no compiling errors, but your client will crash. So we will need to add a return statement into the if-statement.

return;

Now, we will need to add [tt]teleportToX[/tt] and [tt]teleportToY[/tt] so our characters don’t initalize combat with the NPC, so include this in your if-statement

teleportToX = absX; teleportToY = absY;

This teleports your character to where he’s standing, so he does not run up to the NPC and initalize combat with him/her.

Now, put this into your if-statement

server.npcHandler.npcs[npcIndex].IsUnderAttack = false;

That makes it so your character doesn’t engage in combat with the NPC, therefore he wont follow or attack you.

(Make sure you put everything above the [tt]return[/tt] statement.)

And there you go how easy was that?
