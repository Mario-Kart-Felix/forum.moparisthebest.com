Hmmm, I added if statements to your color finding, and added standards.

Try this:

[code]program broncotrolleys;

//
// Made By retributionmike
// Game Link - http://www.homestarrunner.com/broncotrolleys.html
//

{.include si.txt}
const
trisket = 19574;//color of the trisket
pnutbuda = 233;//color of the red letters on the peanut butter jar
orange = 11515580;//color of the orange
trolleystomake = 100;//how many bronco trolleys do you want to make
var
x1, x2, x3, y1, y2, y3, a : integer;

procedure treskut;
begin
if findcolor(x1, y1, trisket, 0, 0, 548, 398) then
begin
clickmouse(x1, y1, true)
wait(300);
end;
end;

procedure peenutbooter;
begin
if findcolor(x2, y2, pnutbuda, 0, 0, 548, 398) then
begin
clickmouse(x2,y2, true)
wait(300);
end;
end;

procedure oj;
begin
if findcolor(x3, y3, orange, 0, 0, 548, 398) then
begin
clickmouse(x3, y3, true)
wait(300)
end;
end;

Begin
Repeat
treskut;
peenutbooter;
oj;
a:=a+1;
Until(a=trolleystomake)
Writeln(‘bronco trolleys are made, winned k’)
End.[/code]

Yep, good job, and no problem.

Also, when using if statements they only excecute on thign after it. So, if you want it to do two things, (such as in your script where you Click the mouse and wait) You have to have if statement excecute a begin clause. Therefore, it will do everything after the begin statement, until the script hits the end; that follows it.
