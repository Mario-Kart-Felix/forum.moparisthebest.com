User32.dll Wrapper

General Programming
nade July 30, 2016, 4:26pm #1

This actually took quite a bit of time to make. The hooking of MessageBoxA was done pretty quickly except for the fact that it was crashing during the hook. I solved the problem a few minutes ago and here I am sharing it. I doubt most of the people here will find it useful, oh well.

Your basically forcing the PE file to use your version of the user32.dll instead of the original one. All the functions you don’t need to hook will be redirected to the original using the .def file in the project workspace. For functions you need to hook like “MessageBoxA”, you need to write a new function. The hardest part ( which isn’t really difficult ) is to ‘finish off’ the function to prevent errors and crashing.

http://servbit.org/MsgWrapper.zip
