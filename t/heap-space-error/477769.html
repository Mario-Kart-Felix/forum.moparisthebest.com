Christmas_tree August 4, 2016, 2:05am #2

Java heap space is the amount of allocated memory Java has to run. Default on 32 bit is 128mb.

If the heap space runs out, you’re out of memory and the program crashes. This 718 you speak of is probably poorly coded and the reason it uses all the heap space is because of a memory leak. You can use utilities to help you see which methods are using the most amount of memory and try and analyse it from there. Alternatively you could try and run the garbage collection to free some memory up. Also you could increase the amount of memory your server is allowed on the operating system.

I expect there will be some “fixes” released, but one cannot guarantee anything.
