[size=14pt]About the application[/size]
This is an application to make managing and updating clients easier for both players and developers. I made it because I’ve always found myself rather annoyed at how often server owners would update their clients and how the very few launchers I’ve seen for servers, they always tend to fail if the webhost is offline due to attacks and had such unnecessary flashiness. This is designed to be as simple as possible; meaning no fancy GUI or user interaction beyond the initial setup.

Because it uses arguments to gather all the information it needs, all that’s really needed is a .bat file for each server you play. Whether it’s provided by the server owner or created by you, it’s simple to make and an example is included the the download.
Note that the client must be a .jar file with a direct download link for the manager to work!
As of now, the program supports redirected download pages. So long as it redirects to a direct download, (No mediafire or Uppit,) it can fetch the uri and download the client.

[size=14pt]Usage[/size]
Do not just run the program! It will crash! Run it via a .bat file with two arguments.
The first argument is the server name - Example "YourScape"
The second argument is the link to the client - Example "http://yourscape.com/client.jar"
Example .bat file:

@echo off
RSPSM2 "YourScape" "http://yourscape.com/client.jar"

Eyeownyew:

What does it do? From the output it looks like it just redownloads the client

It redownloads the client each time it’s ran, therefore always keeping the client up to date. It also backs up the latest client downloaded so if the host ever goes down, it can still run the latest version you have. Not the most innovative way of doing it, I know, but after much work on a different version of this I intended to sell to servers, I decided the simplicity was better. I originally wanted it to check the current version from a text file on the server’s website but it had a lot of issues and it was a lot more difficult to make everything work properly.
I know plenty of people could do better, I was just surprised that they hadn’t yet from what I’ve seen. Plus doing it this way has the advantage of it working with almost every server, so long as it has a direct download link. It doesn’t like a few, such as perfectionx and the likes, because of their client patching system, but you don’t even need it in those cases.

ohokay:

Would this not be easier to integrate into the client itself rather than having a separate application? What about mac users?

It could, and some clients have, such as Vitality and their own game launcher, however a lot that I’ve seen have issues and so I created this, primarily for myself, so I could easily just launch whatever server I wanted to play at the moment without having to worry about the client not being up to date because they don’t have an automated updater. It can easily be integrated into the client, but not many ever seem to, so I compensated for that.
As for mac users, if anyone truly wants it for mac, I’m willing to port it using mono or possibly rewrite it in Java despite me being far more experienced in C# than any other language. I did actually have a test Java version once but I got rather annoyed with it and scrapped it. So if any Mac/Linux users need it that badly because they don’t want to write their own, I can port it, just ask.
