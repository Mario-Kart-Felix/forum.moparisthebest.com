the cache is exactly what it advertises itself as – the erm main file cache
the “ondemand” system for resources allows you to request resources by type and id and supports urgent as well as extra requests
it first checks the cache and if data is there it does a verification check by crc32 checksum and a version, both found in the versionlist archive
if it passes it just reuses the data, else it sends the request to the update server and awaits a response lol
