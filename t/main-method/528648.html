mod_taharok August 4, 2016, 10:17am #16
imthenull:

String [ ] kids [ ] = new String [ 2 ];

Nice try.

There is no difference between [tt]type[] name[/tt] and [tt]type name[][/tt] in most situations. The following is an example where it does make a difference:

int[] anIntArray1, anIntArray2, anIntArray3; // these are all type int[]
int anInt4, anIntArray5[], anInt6; // these mixed types work differently; the first and last are type int, but the middle is int[]

Another aspect is preference as well. For people more familiar or preferred to C/C++, keeping the brackets after the type makes more sense (sort of). It really depends on preference.

As for the main question itself, there is another variety that’s also equivalent. If you’re familiar with variadic parameters:

public static void main(String[] args) {
}

public static void main(String args[]) {
}

public static void main(String... args) {
}

These are equivalent because variadic parameters just converge to an array when compiled.
