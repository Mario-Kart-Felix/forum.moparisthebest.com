Protosstribe August 4, 2016, 7:21am #5

One way is to send the files to the client on every first login (and subsequently every time the map changes on the server-side). By every first login I am referring to the first login since the client program was started.

Every time the files are sent, they should be encrypted. The key to decrypt them should be different every time.

Instead of just sending the files encrypted, would be to send a unique class able to decrypt said files together with the files. This way it would be harder to decompile the class performing the decryption. If someone do decompile it, it is of no use, simply because the next class will be totally different.

Edit: It’s by 99% certainty not possible to guarantee that noone can reverse anything related to this problem. It might, however, become much harder.
