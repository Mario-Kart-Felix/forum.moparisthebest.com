Looked at the CycleEvents source - the timing method in this thread is more efficient.

On a related note, this is making timer tasks in general, but you could use this idea to make a timed event manager for your server. You could have a Timer object for the entire server which runs TimerTasks, and you could subclass to have specialized TimerTasks for doing large categories of things (e.g. managing animations).

Furthermore, you could have more than one Timer so that groups of related tasks share a timer; then you could disable a certain set of repeated events, for example, without affecting other events.
