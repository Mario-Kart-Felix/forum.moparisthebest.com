Ruby_Forums_Coder August 4, 2016, 1:54pm #4

First problem:

No need to assign a separate tag for logo, you can include that in the img tag.
Don’t ever use tags, they’ve been deprecated for years. All styling should be done through CSS.

Further deprecation is here:

<h2><p><font color="white">Hey there!</p></h2>
	<h3><br>The forums are up and running! We hope you enjoy.</font></h3>

Don’t use font tags, once again they are deprecated. Styling through CSS. Your header tags should also generally have a selector on them so you know what they’re for and can individually style as needed, not a necessity but it’s a good practice to get into.

Further issues:

You’ve used the tag after declaring it a HTML5 document, if you’re using HTML5 you should generally be paying attention to the semantic meaning behind using or .

<br>No idea still
<br>No clue at all

Don’t use line breaks in this instance. Think about what you’re trying to do. You’re trying to LIST the staff members. So use a list, not linebreaks. With web design always think about what you’re trying to do, there’s usually a tag just for it.

<div id="box1">

Don’t name things “box” “box1” etc, have better naming so others know what they’re referring to. Also, since “box1” is used more than once it should be a class selector, not an id selector.

You should also never have more than one open and one closing body tag in a document, it only needs to be initiated once. You open it once and close it twice, the middle closing tag shouldn’t be there.

On to the CSS:

	margin-bottom: 5px;
	    border-bottom-left-radius: 3em;
		    border-bottom-right-radius: 3em;
			padding-left: 10px;

Stick to using either em or px, preferably em for a fluid and responsive template.

A general tidy up of the CSS is needed, I can’t even read most of it because it’s such an eyesore. Work on tabbing and general neatness of code and then I’ll take a look at that.

Improvement from the HeavenX template, but still pretty shitty with a lot of room for improvement.

Keep at it, I suppose.
