Muroth:

use both

No.

If, for this or that reason, you’re using PHP. You’re going to of course want to save sessions. If you don’t know the prime usage of sessions, don’t even try to make something with PHP. Especially something that usually uses sessions. You can use cookies, but only use them for a, “Remember Me” feature. Never store usernames and passwords in the cookies, as they can be retrieved by hackers, if someone was to get into your computer.

Sure. A session is PHP’s way of carrying variables from page to page, and it does this using the browser’s memory, since PHP can’t actually remember variables after load. So I can tell a browser a user is logged in via a session variable, go to a different page on my site, and the browser will tell that next page that the user is logged in. We’re talking about how secure it is.
