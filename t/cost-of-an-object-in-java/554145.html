eyeownyew August 4, 2016, 5:12pm #2

You shouldn’t be creating a list every 100ms and using it once, at that point you should definitely be reusing memory
Regardless, if you’re using memory correctly (and reusing indices instead of making a new list every 100ms), memory is not nearly as important to optimize as processing efficiency. The size of classes is unnoticeable unless you use just an absolutely absurd amount of them, which requires very bad practices in the first place

lothy August 4, 2016, 5:12pm #13

Generally speaking Java’s GC is optimised for allocation of small objects. The main issue that might arise is memory fragmentation, depending on how the virtual machine is carving up system memory provided to you on the heap.

The reality is that you probably want to re-use the same list but what you’re currently doing won’t be a deal breaker.
