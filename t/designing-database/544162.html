When a rare drop comes through, just make a simple if statement. Run a query to select all data from the table where item id is the item just dropped and user id is the player. Then the if statement checks to see how many rows have resulted from this query, if it’s 0, insert a new entry. If it’s anything more than zero (which it shouldn’t be, it should be a max of 1.) just run an update query on the item id and set it to itemid+1. RSCShadow August 4, 2016, 1:22pm #8

You’d probably want to make a row for each time an item’s dropped, and store something like:

This schema is pretty versatile in what you can do with it as well. You can get totals of how many items a particular player has gotten drops of, or totals of how many drops the entire player population has gotten, sort it by date dropped etc.

Then you can use npc_id, item_id and player_id to JOIN with other tables. You can also just store the names instead of IDs for any of the above, but in your case, denormalizing the data doesn’t provide any benefits and will most likely just slow down the most common queries you’ll be doing.

If you have any questions on the queries you’d need to write, sort out the table first and ask.
