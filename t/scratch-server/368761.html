Aftermath_961 August 1, 2016, 12:58am #4

First, you need the actual server component that handles connections.

Then, you need to figure out what data sent means what when sent between client/server.

Then you just need to send the appropriate things to the client and understand what the client is trying to say and handle it.

Player saving / loading can be done by just printing to a file, or with serialization (look up Serializable and serialization in Java).

Anyway, in a rough order:

  1. Server component (handles connection)
  2. Some sort of packet handler that can just read the packets for now, doesn’t have to actually do anything with them
  3. Have your server send the initialization code to each client as it connects and add a case in the packet handler for logging in, work on displaying the player here

Practically everything else from here in terms of functionality (item on item, item on player, etc…) is just handling and sending the correct packets.

After you’ve done all that stuff, you can add in features like saving player data and combat and whatever.

Aftermath_961 August 1, 2016, 12:58am #6

For connection handling part just look up ServerSocket, Socket, and input/output streams from sockets.

Basically, ServerSocket listens for connections and accepts connections which are represented by Sockets. Sockets have input and output streams. You can send things to clients with output streams and read things with input streams.

For figuring out what packets do what, I suggest you just check out a source with more features and just grab the packet IDs and information about how to read them.

When he says networking he probably means whether you use the java.io or java.nio packages. You should probably be using java.io if you’re just starting out since it’s easier to use. Look up whatever you want on www.exampledepot.com
