Why not build in SQLite for a small database storage platform in place of text files?

SQLite is a software library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine. SQLite is the most widely deployed SQL database engine in the world. The source code for SQLite is in the public domain."

http://www.sqlite.org/whentouse.html

You figure that saving to text files is the fastest and easiest way but there are two large downsides to this - player save files are incredibly difficult to translate into human readable information and its difficult to parse into various web-integrated uses such as highscores and user login authentication systems. No SQL to configure but IMO it is more difficult reading the player text files. Make the server GUI able to modify all of the player attributes.

A light database can run without the end user having any idea but it can enable those who can utilize it to do a whole lot more than what is currently possible.

The only issue would be for noobs who cannot edit databases. The server comes with a GUI so all we would need is to build in more functionality such as password and rank changing.
