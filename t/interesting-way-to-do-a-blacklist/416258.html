slavemaster August 1, 2016, 3:13pm #9

I’m not saying my test throws an exception; I’m saying your code throws the wrong /kind/ of exception on ill-formed IPs.
That being said, your algorithm doesn’t work because you don’t understand how to implement the algorithm. Consider the case of adding 255.255.255.255 and 0.0.0.0 to the blacklist.
This activates all bits for “ones” as well as “zeroes”, meaning that therefore isListed() will return true for any valid IP passed to it.
You don’t associate any context with the state of a bit; e.g. the third octet of 192.168.0/24, while also seen in 255.255.0/24, aren’t necessarily equivalent to matching addresses.
Achieving constant-time for all operations is only possible here because you don’t actually implement a solution. Similarly, you can compress data down to 0 bits, but decompressing is going to be a hell of a lot harder.
