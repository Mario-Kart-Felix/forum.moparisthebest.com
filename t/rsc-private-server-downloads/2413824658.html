Peter: I compared them both on the same user-load, which was a 1 to a few characters. Not too many, but atleast they were equally compared. The latency between client and server-responses were greatly decreased on RSCDEF compared to RSCDaemon. If you for instance walk, RSCDEF will walk almost instantly, and if you wield etc. the same thing there.

Also, if you think about how computers really does work… Computers can have one thread running at one time per each and every processor they have. So, the more processors, the more processes (threads) can be ran. My server uses 1 thread per client, and thus, the more processors you have, the faster they will process their updates.

Another thing is that my server uses different containers for different areas, which contains all NPCs, ground-objects, wall-objects, ground-items etc. Everytime you will iterate through all of these, everytime it will go much faster, because there are less to iterate through at all times. Of course if alot of players are in one area, it will take longer for that particular area, but still less than RSCDaemon, because RSCDaemon have them all in one container (World.java).
