jediz August 4, 2016, 1:18pm #26

If you remember Azureify #194. I’m the one who worked on that project. Since I feel generous today, I thought I would comment on this thread. The map data was stored on the server back in 2003-04 thus, we don’t have the actual 194 maps. (Unless someone saved them in a vault somewhere)

I did however succeed in loading 289 maps on an actual 194 client. I loaded 289 models, and object definitions because the models/objects changed between 194-289. Then I dumped the 289 maps/landscapes from the cache and renamed the individual files into a format readable by the 194 client.

I don’t remember the individual packet ids and structures off the top of my head. But here is a rough diagram.

1. On login, a map region packet is sent from the server to the client.
2. If client is missing map data, the client sends a packet to the server requesting the map data. (Skips this step if the maps have already been downloaded by the client from the server)
2.1. Server sends map, and landscape data to the client to be cached. 
3. Player and NPC render packets sent
4. Magic (Landscapes and players visible on screen)
