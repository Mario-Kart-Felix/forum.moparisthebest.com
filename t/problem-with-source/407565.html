Where to start…

Well based on the error code presented, the main ‘ant’ has escaped your computer. What does this mean? It could be a number of things:

  1. Your anti-virus program has detected it as an insect/bacteria-like entity (like virus), and has quarantined it.
  2. Windows by default, has sprayed your system with RAID to protect against mass infection, thus killing this ‘ant’ from mass invasion of your file system.
  3. You fucked up paths and disregarded simple instructions given in the installion manual.

You may try contacting your software provider for assistance, or your local repair shop for help. The good news is, there are alternatives to executing the source code, typically by compiling it and executing it through Java’s VM. Now you might ask, well why would I do try such a thing? If you already didn’t know, the high-level language used to write the code is written in Java. Luckily, the developers have allowed us a way to compile this code into what’s called “bytecode” which can be interpreted to machine code (low-level) at runtime.

Now, you depending on what operating system you’re using, you can utilize native tool(s) to create files locally. By doing this, you may want to consult your system administrator or your common sense to accomplish this. From there, you can write a small “shell” script (also known as batch in Windows) to do what otherwise is considered tedious to write out time and time again, which in your case would be sent to Java’s VM for execution. From there, you can feel content you’ve done what most find near impossible to do.
