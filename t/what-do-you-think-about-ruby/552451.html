EcoPS_GM August 4, 2016, 3:19pm #15

Ruby’s a fine language, indeed. It’s syntax is actually quite beautiful, and has the availability to simply ignore common punctual and structural syntax. It’s a powerful, flexible language and I wouldn’t be appalled to the idea of using it, or recommending it. However, just because it has a pretty syntax doesn’t mean it’s a good language, does it?

Compare it to Python - as of 1.8, Ruby has been known to be slower than Python. However, since 1.9, that has been rendered negligible, and 2.0 has seen added performance enhancements. Ruby’s flexibility also allows it to write functions of equivalence in fewer lines of code than Python.

require 'active_support/all'

new_time = 1.month.from_now

[code=python]
from datetime import datetime

from dateutil.relativedelta import relativedelta

new_time = datetime.now() + relativedelta(months=1)[/code]

Notwithstanding - that’s extremely basic code; I’m just providing an example. Either way, Ruby’s a fine language to use.
