[center][size=36pt]How to Add a Timed Server Message[/size]
by: Davidi2[/center]

Difficulty: 1/10
Files modified: Engine.java
Time needed: 2 minutes

This is probably the easiest tutorial I will ever make, but questions are still popping up about it. It’s all in one file, and is less then 20 lines of code.

So let’s open up Engine.java, and find the variable declarations. Find any int and on the line below it add:int messageTimer = 0;

Now find your void process() { and search for your other timers. Add if(messageTimer > 0) { messageTimer--; } else { for(Player p : players) { if(p != null) { p.frames.sendMessage(p, "Your message here!"); //Change your message! } } messageTimer = 600; //Change to 10 times the number of seconds you want 600 = 60 seconds }Please note that there may be small errors. Please post them so I can fix. (Wrote in a hurry at school)

What this does is declare a new timer variable, and decrement it and check it every 1/10 of a second. If it is at 0, it goes through each player and sends the message.

For one stop relying on the “process” to do a perfect cycle at a fixed rate every time.(Which your code relies on)
And for two just check the time elapsed and then once the time is up execute.

[code]public abstract class DelayedEvent {

public long startTime;
public int timeToWait;

public DelayedEvent(int timeToWait) {
    this.startTime = System.currentTimeMillis();
    this.timeToWait = timeToWait;
}

public boolean isReady() {
    if((System.currentTimeMillis() - lastTime) >= timeToWait) {
        lastTime = System.currentTimeMillis();
        return true;
    }
    return false;
}

public abstract void executeEvent();

class SimpleImpl extends DelayedEvent {
    public SimpleImpl(int timeToWait) {
        super(timeToWait);
    }

    @Override
    public void executeEvent() {
        // Do your shit here.
    }
}

}[/code]

Example usage:

SimpleImpl event = new SimpleImpl(600); if(event.isReady()) { event.executeEvent(); }

You should probably use some sort of event manager for that, for example one that fires a task off a thread that checks them all. Or you could use a ScheduledExecutorService for an event type thing.
