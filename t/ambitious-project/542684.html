cleako August 4, 2016, 1:04pm #2

I am currently working on doing the exact same thing in terms of stripping all of the game assets and building a new game from scratch using the RSC game engine. Its not too terribly difficult to build new game assets but I hope to make things easier. My project is to create a zombie survival game, similar to DayZ using the RSC game engine. I am using Peeter’s DarkQuest source as my base which is based off the RSCA source which is based off of the RSCDaemon source. While Moparscape Classic is a well done project, it mixes in a lot of scala programming on top of the java and the added complexity is not personally my cup of tea right now. I have been fixing and improving several of the community tools and plan to create all of the necessary guides for how to use them (2D landscape editor, 3D landscape editor, sprite editor, map generator, etc).

The goal is to keep everything completely open source and available through GitHub. The programming IDE that I am using for the project is NetBeans which I have come to prefer over Eclipse. The RSC private server community on here used to be absolutely atrocious in terms of immaturity and uppity programmers but over the last few years, most have lost interest and left. Those who are here now have been great to work with from what I have experienced. I would suggest keeping what you develop running just within the classroom and not publicly playable as there remains at least one or two individuals that choose to launch daily DDoS attacks on various RSC private servers for different reasons.

MS paint works best for editing sprite PNG files. The rest of the RSC assets are proprietary and use the various RSC tools that the community has developed over the years. Most of it is XML based. The server uses MySQL for the database and the client cache is downloaded from a webserver. The webserver requirement for the cache can be removed - I just need to make some modifications to the client and tools that depend on it.

This is the project as well as various repositories for each part, including the tools that I am working with all inside GitHub: http://cleako.github.io

Would text-based instructions and video recorded demonstrations be the most useful for your students as non-native English speakers? Not sure what other methods for instruction would work well.

cleako August 4, 2016, 1:05pm #11

Arguments against aside, a multi-language supporting game should be viable with duplicate talking NPCs, such as bankers that are separated by water through two islands with one island for English speaking NPCs and the other for Arabic speaking NPCs.

My thoughts on how to do a multi-language client would be to offer two client.jar files with one in English and the other in Arabic in the cache with a launcher GUI that first does a MD5 hash check for any client.jar updates and then display both game world and language selections. The world IP setting would be saved in the client.properties file in the cache folder for which IP to connect to and then the launcher would open the appropriate client.jar based on which language was selected in the launcher. Both clients would share the cache with the exception of Items.rscd, Npcs.rscd, Spells.rscd, and Prayers.rscd. Duplicates of the above mentioned .rscd cache files would be made and one set would be translated to Arabic. The Arabic client would then point to the differing Arabic cache files and various hard coded java game messages would be translated too for the Arabic client, such as game menu text. Each client would do it’s own MD5 hash check for updated cache files at launch so players wouldn’t have to wait for cache files of a language they don’t know to download.
