pplsuqbawlz July 30, 2016, 6:57am #35

Lol The update server is protocol “js5” which is found in the client so, no, it’s not me naming this shit.

JS5 I can only assume stands for “Jagex Server 5”; None the less, it’s not particularly complicated, the client initalizes a connection with id of 15 and then a signed int “version”; The updater either sends back an out of date, unknown version, or some other error-otherwise we receive an “ok” which is 1 I believe(might be 0). After which-The client loads all of the cache files and sends there 4 byte crc in a packet using format (len, crc1, crc2,… crc13); The server checks the crc’s and sends back an “ok” or “unknown version” and so on… The client on recieve of an “ok” sends back a handshake which is “0, 1” or “0, 3” (at least in this version)" and the server sends back it’s handshake and BAM we’re finally done with update server…

Of course this is the bare minimum required to make an update servers chatter; and it’s not got all of the JS5’s protocol implemented, but it does provide for a successful “startup” of a client. I would know; I use it for testing!

Peace out and I hope this helps someone here.
