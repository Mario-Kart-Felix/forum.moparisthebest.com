Vrunk August 3, 2016, 10:36pm #13

An example improvement:

package sandbox;

import java.util.Arrays;
import java.util.HashSet;
import java.util.Set;

public class Main {

	// Considerations:
	// - If you don't mind re-compiling every time you change the list of IDs, storing it as a constant in the class is fine.
	// - If you want to be able to change it on the fly (i.e. while the server is running), you should store it on the file system and load the data into memory.
	//   This will allow you to re-load the file (and the IDs) by issuing a command to the server while it is running, changing the method's behavior at run-time.

	private static final Set<Integer> NON_PROJECTILE_SPELL_IDS = new HashSet<Integer>(
			Arrays.asList(12939, 12987, 12901, 12861, 12951, 12999, 12911, 12871, 12963, 13011,
					12919, 12881, 12975, 13023, 12929, 12891, 1190, 1191, 1192));

	// CPU usage:
	// - Because the data set is so small, it is debatable as to whether an array would be faster than a HashSet.
	//   Depending on the hash function used, computing the hash for a single integer is potentially more expensive
	//   than testing 19 ints (worst case) for equality with the int provided to the function.
	// - If you do use an array, it should be sorted so that you can use a binary search.
	// - Unless you data set is much larger, there will not be a noticeable difference in CPU usage for an array versus a HashSet.

	// Memory usage:
	// - An array or primitives (int) will have better memory usage than a HashSet due to the overhead of wrapping the primitives in an Object (Integer).
	// - Once again, unless the data set is much larger, there will not be a noticeable difference in memory usage for an array versus a HashSet.

	private static boolean isProjectile(int spell) {
		return !(NON_PROJECTILE_SPELL_IDS.contains(spell));
	}
	
	// Example
	public static void main(String[] args) {
		for (int i : Arrays.asList(1, 12861, 2, 12975, 3)) {
			System.out.println(Arrays.asList(new Object[] { "Is projectile?", i, isProjectile(i) }));
		}
	}

}

TL;DR

Don’t use switch statements for data.
