Lin August 4, 2016, 2:01pm #6

What I recommend, because this is how I went about doing it, is determining what it is you want to make, and then working towards that goal. You learn as you go that way, and don’t waste time learning things you have to work extra hard to learn how to apply. For example, if you want to make a game, look up game tutorials and examples. Mess around with completed code to see what works and what doesn’t. Don’t worry about understanding everything fully. When you expose yourself to lots of code, you eventually pick up on good and bad habits, learn what to do and not to do, and overall how to create what you want the best way that you can.

Don’t ever pick up a programming book, because what you’ll read is “Here, you can do this like this. It’s not the best, but you’re just getting into programming so it’s okay. You’re not smart enough to learn how it can be done better, and this book is not going to tell you where exactly this knowledge might apply and how broad of a topic it is. Good luck.” Even the programming classes I’ve been forced to take in college are like this, and coincidentally everyone who learned to program on their own (and didn’t take the class) was better than everyone who took the classes without previous experience.
