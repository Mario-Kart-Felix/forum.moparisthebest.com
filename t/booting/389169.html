On newer versions of Ubuntu (and therefore newer versions of Grub) this happens automatically, but it may not have done on 9.04.

Edit /boot/menu.lst and add

title Windows
rootnoverify(hd1,0)
makeactive
chainloader +1

Usually this is already in the file, but commented out.

I took a wild stab in the dark at the (hd1,0). If this doesnâ€™t work for you, you will need to find out what grub thinks the Windows boot partition is and replace it here.

Unfortunately you may have to use a bit of trial and error.

However, one thing you can do is press c at the grub menu to enter the console and run geometry (hd0) and geometry (hd1) - this will show you how many partitions are on each drive, at which point you can eliminate the ones you know about.

If you did a standard Windows install on the Windows drive, it will either contain one partition in the case of XP or below, or two (one really tiny one) in the case of Vista/7. Unless itâ€™s an OEM install, in which case it might be different.
