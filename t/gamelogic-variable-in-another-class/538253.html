    if (inventory.getSlotsOccupied() + amount > inventory.MAX_SLOTS)
        return false;
    startMining(rock, amount);
    return true;
}

public void startMining(Rock rock, int amount) {

            inventory.put(items.getItem(items.getItemID(rock.toString())), amount);

}

}

The unoriginal way I recommend to do this is to have a parent class, such as Player, that stores all of the player-specific data, like Inventory. When it comes to classes like Miner where the only code in there is to execute the same code that’s essentially going to be the same regardless of the player, you either make everything in the class static or have a static instance of the class that can be referenced in some global class (Not sure of the overhead in Java regarding either approach). This way there is no extra memory usage for each Miner. Then in your Start method, you pass Player and the code then just uses the information of the player you passed.
