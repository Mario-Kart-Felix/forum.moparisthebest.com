cruel July 30, 2016, 1:11am #17

Welp… mostly just injecting code to redirect calls to BitBlt [and such] that SCAR calls before its functions. Just injects at runtime so that no one has to patch anything.
That way, itstead of getting the actual contents of the window, it searches whatever I wish.

I had to do that after failing with the SetTargetBitmap or w/e…
It worked technically, but it relied on constant refreshing (which I had a function to set the refresh interval). For for time sensitive operations (like many randoms), this would be bad since it’d require high refresh rates thus making it overkill for the portions of the script that didn’t need it.
So just having it refresh when SCAR made the API calls makes it easier.
However, I don’t think it works perfectly… not to mention it’s useless on RS without a good mouse clicking routine.
