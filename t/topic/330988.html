This:

SELECT rscd_invitems.* FROM `rscd_players` left join (rscd_invitems) on (rscd_players.user = rscd_invitems.user) WHERE `login_ip` = '127.0.0.1'

Is MUCH faster than doing what you’re doing:

SELECT * FROM `rscd_invitems`
WHERE `owner`
IN (
        SELECT `owner` FROM `rscd_players`
        WHERE `login_ip` = '127.0.0.1'
        OR `creation_ip` = '127.0.0.1'
)
ORDER BY `id` ASC
LIMIT 0, 1000;

Proof:

mysql> SELECT * FROM rscd_invitems WHERE user IN ( SELECT user FROM rscd_players WHERE login_ip = ‘127.0.0.1’ OR creation_ip = ‘127.0.0.1’ ) ORDER BY id ASC LIMIT 0, 1000;
ERROR 1053 (08S01): Server shutdown in progress

I cancelled this query after 6 minutes because I didn’t want the tables inaccessible to find out exactly how long it would take to complete.

and my query:

They are selecting the same fields, however the join concats both tables into one then runs a normal where query, whereas the where IN takes every result from the inner select and then runs the parenting query for each one (as far as I understand it). Besides, like I said, it is selecting the exact same fields for each query.

Also, which fields have you indexed? Fairly sure (from what I have gathered from reading your IRC communications) that you’re indexing pretty much all of them.
