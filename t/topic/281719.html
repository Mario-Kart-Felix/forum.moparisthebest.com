PKERZ July 31, 2016, 8:10am #5

That error happens when the package-name does not equal to the underlying folder-hierarchy. For instance, your class called “org.rscdaemon.server.packetbuilder.client.MiscPacketBuilder” must lie in the folder-hierarchy “org/rscdaemon/server/packetbuilder/client/”… And the file in that folder must be called “MiscPacketBuilder” with an extension of “.java” (or “.class” if it is for class files). The folder-hierarchy (or folder-name) pre the above mentioned folder-hierarchy is of course dependent on how you setup Apache Ant or whatever you use to build your project…
