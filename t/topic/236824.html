It can still break. The could remove the main method, which wouldn’t be strange at all since it’s unused (at least by the users). Just like they removed Signlink from the unsigned jar.
There is a chance that one of the dummy arguments contains a SIPUSH between 503 and 600 ^^.
I do use it though, except I just look in the init method which contains it as well.

Btw. they changed standard size of the standalone client to 1024*768 in case nobody noticed yet Which is different then the standard size of the applet (as called used in init).

If you want to make a reasonably fail safe client checker, then you should search the client methods for a method containing the text “js5crc” then continue search to a checkcast with Socket.

            lc.W = new od((Socket)hc.M.f, of.X);
            kh kh1 = new kh(5);
            kh1.c(-1, 15);
            kh1.c((byte)115, 514);

Get the second invokevirtual, check both arguments for a SIPUSH. If both are SIPUSH, goto the method ( kh.c(B, I) ) and find the next ISHR. Then go back to the previous ILOAD
ILOAD, of course, gives you a variable index, use it to calculate the argument index with (iload_2 would be argument 2 as 0 is this).
Then use your variable index to determine which of the 2 SIPUSH is the right one.

But 80% of the time there is only one SIPUSH, so it remains reasonably quick.
