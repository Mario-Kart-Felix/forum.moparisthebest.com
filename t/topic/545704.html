[center][size=28pt]Guide on using Julia.[/size][/center]
[center][size=12pt]Helping you on the way.[/size][/center]

Note: Not updated to the beta version yet.

[size=16pt]Server sided:[/size]

[size=12pt]Server Properties:[/size]
[spoiler]
The server properties are the most important things that identify your server.
You can find and edit them by clicking on properties:

The properties screen will pop up and here you can make the server truly yours:

I think this is pretty clear how to use it, if it isn’t reply on this thread and I will expand this section.
One thing I want to notice is that resetting defaults will reset to the properties of Julia, like they were when
you first downloaded the server. This is in case you fudgeed up everything and want to restart from zero.
[/spoiler]

[size=12pt]Changing starter:[/size]
[spoiler]
The starter is the items a player starts with when he logs in for the first time.
You can access the editor via here:

This is how it looks like:

As you can see, a list shows you the currently set starter items.
You can remove them or add new ones by providing an item id (see links at bottom of the guide)
and an amount of the item.
[/spoiler]

[size=12pt]Managing players:[/size]
[spoiler]
You can find the player manager via here:

This is how the manager looks like:

The list will give you all the players registered, so also the one’s that are NOT online at the moment.
You can edit their rights or punish them for bad behavior.

To interact with a player’s account you have to select it from the list.
The current details will automatically be displayed and from there on you can edit the player.
Once done editing, hit update.

Reload: Pressing the reload button will refresh the players list and the online players count.
[/spoiler]

[size=12pt]Using extern commands:[/size]
[spoiler]
The commands panel allows you to execute commands as server manager.
You do not have to be logged in to use this as you operate as server manager and not as a player.
To go to the commands panel navigate here:

The panel shows you what commands are available, how to use them and even an example on using them,
so this shouldn’t be to hard.
You don’t need to put ‘::’ before the command as we’re not in game here.

Hitting the execute button will execute your command.
The last command button will refill the last command you executed in case you want to spam a command.
[/spoiler]

[size=12pt]Editing npc drops:[/size]
[spoiler]
This editor allows you to add/remove and edit drops for an npc.
Navigate to the editor via here:

When start entering an id in the ‘npc id’ field, don’t panic.
The list automatically loads the drops it knows the for the npc with that id.
Just enter the complete id of your npc and then start editing what you want.

You simply enter an item id, the amount and the drop rarity to add a drop for an npc.
If you select a drop from the list you can also edit the drop.

The clear button will clear the screen from all the input to start fresh. It wil NOT remove your drops.
The add drop button will add a new drop for the npc with the details you provided.
The remove drop button will remove the drop you have selected from the list.
[/spoiler]

[size=12pt]Editing npc definitions:[/size]
[spoiler]
Editing npc definitions is not something you do a lot, but if think your npc should be stronger or weaker
or you want to add a bunch of health, then you need to navigate to the editor via here:

Important! Don’t just add a definition, there is still a bug in this version that doesn’t check
if there’s already a definition for the npc.

First search for the npc id, enter the npc id in the field above the find button and hit find.
If there is a definition for the npc it will load all the details and you can start editing!
If there is none it will tell you and you can safely add a new definition for that npc.

To edit simply change the details to your liking and hit the edit button.


[/spoiler]

[size=12pt]Editing npc spawns:[/size]
[spoiler]
This is something you will probably pass by a lot!
This editor allows you to add new npc’s in your game and remove them.
Navigate to the editor via here:

When you click on show all spawns you will see a list with all the existing spawns.
To remove a spawn you need to write the exact data in the text fields and hit the remove spawn button.
!!!I will make this easier in the next version!!!

To add a new spawn you provide the id of the npc as well as the coordinates where
you want it to spawn. (z stands for height)
You can also indicate via the moves checkbox if you want the npc to walk around.
Once you entered all the details you hit the add spawn button.

[/spoiler]

[size=12pt]Editing item definitions:[/size]
[spoiler]
Editing item definitions is something you wont touch that much BUT
can be extremely useful sometimes.

It allows you to edit the rights for an item (e.g. make it members-only).
And sometimes you pass by an item and see it should be stacked when it’s not,
or it’s in the wrong equipment slot. Or maybe the bonuses are wrong!
Or maybe you want to make an admin item with insane bonuses!

You can edit it very easily with this editor!
Navigate to it via here:

!!!Important: Just like with npc definitions this version has a bug that doesn’t check
if the there is already a definition for the item.!!!

^So if you want to edit an item, FIRST try finding the item by entering the item id in the search field
and hit the find button.
If there is a definition for the item it will load the details and you can start editing.
If there is none you can go ahead and add a definition yourself.

Once you edited all the details to your likings, hit edit definition.
If you fudgeed some shit up or want to start fresh, hit reset to clear the screen,
this will NOT remove your definitions.

Here’s how the editing of a definitions goes in it’s work.

[/spoiler]

[size=12pt]Editing item prizes:[/size]
[spoiler]
Editing prizes of items makes the economy how you want it.
Navigate to the editor via here:

!!!note: Ignore the currency box, leave it at coins. This has to be reworked in the next version.!!!

When entering an item id, the current price for the item will be automatically shown.
To edit the price simply enter a new price and hit confirm.

Ignore the 2 ‘note’ words at the bottom, will be gone in the next version.
[/spoiler]

[size=12pt]Editing shops:[/size]
[spoiler]This is not available in the latest released version yet. Stay tuned for the next version.[/spoiler]

[size=16pt]Client sided:[/size]

[size=12pt]Setting up:[/size]
[spoiler]
The cache has to go in the c:\ directory of your computer
Once that’s done you should be able to run the client locally.
In case you edit the client properties you might have to change the cache name,
read ‘client properties’ for more details.
[/spoiler]

[size=12pt]Client properties:[/size]
[spoiler]
To set up the client to your liking,
go inside the ‘data’ folder and open the properties.ini file.
There change ‘Julia’ to the name you want for your server.

If you have a download link for your cache,
put the link after ‘download=’ where it now says
www.moparisthebest.com

In case you want to host the server, you want to change
’127.0.0.1’ behind ‘host=’ to your host ip address.

You can also provide your website by replacing
’www.moparisthebest.com’ behind ‘website=’ by your website.

The cache has to be named YOUSERVERNAME_cache
So in this case the server is named 'Julia’
so the cache is named ‘Julia_cache’
[/spoiler]

[size=12pt]Client background[/size]
[spoiler]
To change the background of the client, go inside the cache,
open the ‘Sprites’ folder and there, there is a file called
background.png
Replace that file by your background. BE SURE it’s the same format(.png)
and named the same (background)
[/spoiler]

Useful links:
This link can provide you item codes by searching for an item name: http://itemdb.biz
Some id’s wont be working for the server as they are custom edited or from another revision.
All the ones from revision 317 should work though.

This link will provide you codes for multiple stuff like item’s, npc’s, objects and more: http://rspscodes.synthasite.com/
This is only for revision 317.

In the next Julia version you will be able to search items/npc’s and other id’s via the control panel search function.
