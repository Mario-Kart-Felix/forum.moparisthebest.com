davidi2 August 4, 2016, 12:06am #15

There are reasons it FORCES you to add try blocks in some cases, and it’s because exceptions can be thrown, no matter how ‘good’ or well written the code is. Your logic is flawed – there are two main reasons to print stack traces, one being debugging like you said, to fix code. The other however is to fix problems that the code isn’t affecting, for example reader classes detecting a corrupt or mis-formatted file. How would you know what the problem was without knowing what the exception was? Even though it has nothing to do with the code itself, it’s still a possibility. Whether or not you want the end user to see these exceptions depends on the situation. You don’t always have to print stack traces, but the reason isn’t because ‘the code is written properly’.

Just for future reference.
