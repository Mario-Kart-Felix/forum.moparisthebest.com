such as, in this case:

iadd

->

ineg
isub

AND

isub

->

iadd
ineg

however this is randomly done to each arithmetic instruction so you cannot predict what the instruction list will be like, which is why you should use some form of instruction searcher (see frank_'s updater) or use a regex pattern searcher (see BCEL library docs)

for example, to find this pattern: (position x and y)

getstatic int wc.r //worldX getstatic s cg.P getfield int[] cc.V //queueX iconst_0 iaload ineg isub

and for y

getstatic s cg.P getfield int[] cc.kb //queueY iconst_0 iaload getstatic int vd.w //worldY iadd

a pattern as follows would suffice (i think :o)

String pattern = "(FieldInstruction)+ ConstantPushInstruction ArrayInstruction (FieldInstruction)? (ArithmeticInstruction)+";

then just check matcher.group(0), matcher.group(1) and matcher.group(2) and check the field types to get your fields
