it isnt entirely heuristical, im playing around with examining class hierarchy, and will eventually examine other things as well, in order to define certain classes.

the node candidate is determined by looking for classes that have 5 parents. the only 2 occurances are the superclass of client, and some other class. following up the trail of parents of this class, node is the one directly before Object:

java.lang.Object - 0 ... re - 1 ... ta - 2 ... dh - 3 nf - 4 eh - 5

now, since the client changes every update, this is just a good guess as to where to look.
in the event the heuristic is wrong, no harm is done and the node transformer can be run in a more “traditional” fashion.

anyway, for now, its just fun and games, and learning how to use clojure effectively.
