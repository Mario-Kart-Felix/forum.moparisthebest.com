have you ever seen a command like this in your client?:

						if(inputString.startsWith("::addmodel"))
							int id;
							String[] args = inputString.split(" ");
							id = Integer.parseInt(args[1]);
							addmodel(id);

Or methods like this:

	public void models() { 
		for(int ModelIndex = 0; ModelIndex < 50000; ModelIndex++) {
			byte[] abyte0 = getModel(ModelIndex);
			if(abyte0 != null && abyte0.length > 0) {
		
				decompressors[1].method234(abyte0.length, abyte0, ModelIndex);
				pushMessage("Model added successfully!", 0, "");
			}
		}
	}
	public byte[] getModel(int Index) {
		try {
			File Model = new File(signlink.findcachedir() + "/Raw/"+Index+".dat");//there we go.
			byte[] aByte = new byte[(int)Model.length()];
			FileInputStream fis = new FileInputStream(Model);
			fis.read(aByte);
			pushMessage("aByte = ["+aByte+"]!", 0, "");
			fis.close();
		
			return aByte;
		}
	public void preloadModels() { 
		File file = new File(signlink.findcachedir()+"Raw/");
		File[] fileArray = file.listFiles();
		for(int y = 0; y < fileArray.length; y++) {
			String s = fileArray[y].getName();
			byte[] buffer = ReadFile(signlink.findcachedir()+"Raw/"+s);
			Model.method460(buffer,Integer.parseInt(getFileNameWithoutExtension(s)));
		}
	}

If you have the preload models method, make sure it’ being used in your clients start up. If you don’t. You need to use (or maybe add then use) the addmodel command, to add the model to your client. The ID for the addmodel command is not the ID you use to spawn, it’s the name of the file in your raw folder. (don’t add the dat. just the numbers).
