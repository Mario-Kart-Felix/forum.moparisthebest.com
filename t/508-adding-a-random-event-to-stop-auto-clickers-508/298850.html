[center][size=17pt]Adding a Random Even to Stop Auto-Clickers![/size]
By: Davidi2[/center]

Difficulty:
2/10: Follow the tutorial and you should have no trouble.

Source Tested On:
DavidScape 508

[center][size=13pt]Intro[/size]
If you have a Bulby-Based Server (Berky-Scape, DavidScape, Tokkul-Scape), then you know that some of the skills there can be auto-trained with an auto-clicker. RuneCrafting, Woodcutting & Mining all have that problem. By creating a simple Random Event, you can stop the auto-clickers and make training on your server much more fair.

[size=13pt]Instructions[/size][/center]

1. Open up Player.java, which is located in your Players folder.

2. In Player.java, search for public void fmwalk(int x, int y)This is usually the first void in that class.

3. Above that void, add this void: public int randomSkill; public int chosenSkill; public void randomEvent() { frames.showInterface(this, 134); frames.showChatboxInterface(this, 210); ActionTimer = 10000; randomSkill = Misc.random(23); frames.setString(this, "Please click on " + Misc.getSkillName(randomSkill) +":", 210, 1); }

Explaining the code:
The first and second lines declare two new variables that will be used throughout the Random Event. The “public” part means it can be accessed by any file or thread, and the “int” part dims it as a number, instead of a list, String, or array ect.
The third line declares a public void that can be accessed by any file in your server. Once the void is initiated, the code frames.showInterface(this, 134); frames.showChatboxInterface(this, 210); opens two different interfaces. The first being out main interface, and the second being the chatbox interface. The format is

frames.showInterface(player, interfaceId); frames.showChatboxInterface(player, interfaceId);
The reason I used “this” instead of “p”, like most of the time, is because this code is inside the player class. Any time you reference that is itself, you use “this”.
[center] [/center]
After that,

ActionTimer = 10000; randomSkill = Misc.random(23); }
makes the timer makes it so the player cant do anything, such as continue mining. Then the random skill is chosen, by using the Math random function.
Finally, the code:

frames.setString(this, "Please click on " + Misc.getSkillName(randomSkill) +":", 210, 1); Changes the string for the chatboxinterface we opened (210), to the random skill set before. The format for this is:

frames.setString(player, "TEXT", InterfaceId, LineId);

Now lets get back to the tutoral.

4. Now open ActionButtons.java, located in your Packets folder.

5. Search for: switch (interfaceId) { Directly under that, add:

case 134: //This is the main interface we opened before in player.java. This is basically saying: If (interfaceId == 134) { switch(buttonId) // This makes a new switch, for all the different buttons. Each button has a different ID, thats how we tell which skill was chosen. { case 29: //If the person clicked on button 29... p.chosenSkill = 0; //Then they chose Attack. We know this because the first button, 29, is attack. break; case 30: //If the person clicked on button 30... p.chosenSkill = 2; // Then they chose Strength break; case 31: //Continues on for all skill buttons p.chosenSkill = 2; break; case 32: p.chosenSkill = 4; break; case 33: p.chosenSkill = 5; break; case 34: p.chosenSkill = 3; break; case 35: p.chosenSkill = 6; break; case 36: p.chosenSkill = 16; break; case 37: p.chosenSkill = 15; break; case 38: p.chosenSkill = 17; break; case 39: p.chosenSkill = 12; break; case 40: p.chosenSkill = 9; break; case 41: p.chosenSkill = 14; break; case 42: p.chosenSkill = 13; break; case 43: p.chosenSkill = 10; break; case 44: p.chosenSkill = 11; break; case 45: p.chosenSkill = 7; break; case 46: p.chosenSkill = 8; break; case 47: p.chosenSkill = 20; break; case 48: p.chosenSkill = 18; break; case 49: p.chosenSkill = 21; break; case 50: p.chosenSkill = 19; break; case 51: p.chosenSkill = 22; break; case 52: p.chosenSkill = 23; break; case 28: //Exit button p.frames.removeShownInterface(p); //Closes Main Interface p.frames.removeChatboxInterface(p); //Closes the Chatbox interface p.randomEvent(); //The person didnt do the event, so start it over. This goes back to player.java and does the void we set before break; case 2: //Confirm p.frames.removeShownInterface(p); //Closes Main Interface p.frames.removeChatboxInterface(p); //Closes Chatbox interface p.randomEvent2(); //Continue With the Event. This void will be explained later on. break; } break;
The Comments Explain It.

6. Now its time to add the second void in Player.java. So open it, and add this under the first randomEvent void you had: public void randomEvent2() { //Declares Public Void if (chosenSkill == randomSkill) { //Checks to see if the player chose the correct skill. frames.sendMessage(this, "Correct! You may continue."); //If so, tell them. Player = "this", then the string that will be show Engine.playerItems.addItem(this, 995, 5); //This gives them 5 coins for doing the even. The item id is 995, the player is "this", and the amount is 5. ActionTimer = 1; //Removes the ActionTimer so they can go back to skilling. } else { //If wrong button disconnected[0] = true; //Punishment. In this case, it logs them out. In the threads main process, it checks if the player is disconnected, and removes them. } }
The Comments Explain It.

7. Your all done with player.java! Now open ObjectOption1.java in your packets folder. Inside, search for case 1309: Under it, there should be something like p.wc = new Woodcutting(p); p.wc.startCutting(p.clickId); If so, continue, if not, you have wrong source, and will have to adapt the code.
So now replace p.wc = new Woodcutting(p); p.wc.startCutting(p.clickId); with: if(Misc.random(50) < 25) { p.wc = new Woodcutting(p); p.wc.startCutting(p.clickId); } else { p.randomEvent(); }
What this does is that every time you click on a tree, 50% of the time you will get a random event. I made it 50% because the trees dont fall down, and you only have to click it once per inventory. So basically you get a random even every other inventory. To make it more often, change 25 to a higher number, less often, a lower number.

8. So woodcutting is done. Now lets do mining. Now open ObjectOption1.java again in your packets folder. Inside, search for case 6671: Under it, there should be something like p.mi = new Mining(p); p.mi.startCutting(p.clickId); Now do the same thing with this. Replace that section with if(Misc.random(50) < 25) { p.mi = new Mining(p); p.mi.startCutting(p.clickId); } else { p.randomEvent(); } Again, What that does is that every time you click on a rock, 50% of the time you will get a random event. To make it more often, change 25 to a higher number, less often, a lower number. Make sure you test it first before changing the number. That way you know if you want it more or less often.

9. So woodcutting and mining are done. Now its time for runecrafting. Open ObjectOption1.java again in your packets folder. Inside, search for case 24841: Under it, there should be something like

if(p.ActionTimer == 0) { p.ActionTimer = 3; p.requestAnim(791, 0); Engine.playerItems.addItem(p, 554+Misc.random(12), Misc.random(30)); p.addSkillXP(25*p.skillLvl[20] ,20); p.frames.sendMessage(p, "You make some runes..."); } } break; Now do the same thing with this. Replace that section with if(p.ActionTimer == 0) { if(Misc.random(50) < 47) { p.ActionTimer = 3; p.requestAnim(791, 0); Engine.playerItems.addItem(p, 554+Misc.random(12), Misc.random(30)); p.addSkillXP(25*p.skillLvl[20] ,20); p.frames.sendMessage(p, "You make some runes..."); } else { p.randomEvent(); } } break; This time when you click on the altar, only 6% of the time you will get a random event. To make it more often, change 47 to a LOWER number, less often, a HIGHER number. Make sure you test it first before changing the number. That way you know if you want it more or less often.

Well that’s pretty much it. I hope you learned something from this tutorial. Have fun!
