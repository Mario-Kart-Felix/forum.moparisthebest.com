Unhandled July 31, 2016, 10:56am #1

This may be somewhat more on the software side, but it still has a lot to do with hardware.

OK, in 2007, I installed 64bit Vista on my Athlon 64 3500+ with 2gb of ram. It was absolutely horrid, everything was lagging, even opening the start menu had a noticeable delay. I figured screw it and installed 32 bit Vista. Everything became fast again, I would say about twice as fast as 64 bit (opening windows, start menu, switching between firefox and visual studio etc). I eventually went back to 32 bit XP for a while, but that’s not important.

When Windows 7 RC1 came out, I installed the 64bit edition on the same Athlon 64 3500+ with 2gb of ram. Surely it would be faster, right?! Not a chance, same problem as Vista x64, everything was sluggish. So I installed Windows 7 RC1 32 bit, and everything became fast again.

Now, the other day my new kit arrived and I put it together. Quad core Phenom, 4 gigs of ram etc. I install Windows 7 RTM x64, and holy shit it’s just as slow as my old computer. I raged and installed Windows 7 RTM 32 bit, and of course everything is fast again.

I understand how 64 bit creates larger pointers requiring more CPU cache etc, and I understand how windows has to use Windows On Windows to emulate 32 bit for a lot of apps, but surely that shouldn’t make 64 bit twice as slow as 32 bit, seemingly regardless of the specs you run it on. I want to use all 4gb of ram, but if 3.25gb of it is faster than 4gb…

WHAT THE FUCK?

ps. I have never ever had this issue with 64 bit linux.
