Zlacki August 4, 2016, 6:47am #51
The Enforcer:

Use C++ for superior code.

Maximum portability would be to use C99 with OpenBSD’s GCC, which is what I’m doing. It points out warnings most other versions of GCC wouldn’t, and if it compiles without warnings or errors on OpenBSD, it will compile everywhere else the same(barring non-portable code such as pthreads and Xorg drawing libraries.) C compilers also often produce faster code than equivalent conventional C++ would.
The client will have to use a lot of preprocessor checks for graphics rendering, I’ve made a little progress on creating the client so far, I am working on porting the code for decoding the original 204 BZ cache right now, and will likely make my own cache format based on xz/lzma2 for maximum efficiency.
No updates besides that. I have the basic client engine done and graphics engine mostly working on Xorg-based systems thus far.
