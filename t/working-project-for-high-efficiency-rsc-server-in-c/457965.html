Hello,

For the last week or so I have put together a working(in the sense that you can login and equip/dequip items, look at objects, and kind of walk) RuneScape Classic server in C. I am running into some troubles with designing the player position packet(I redid much of the client, including reconstruction of many packets), but this should be resolved soon.

I have based my client off of a non-refactored 202 deob, but it’s much more modern and OO-driven now.

This compiles on as many systems as I’ve tried on, OpenBSD(current development environment), FreeBSD, Linux, Windows, Mac OS X…

I plan on hosting it on my quad-core, 1Gbps line, 5TB/mo bandwidth VPS, once I get a beta out in a month or two(I am a NEET so I have all the time in the world to work on it).

I am here asking what the community(you guys) would be interested in. A fun server with PvP as a priority, such as that one server that shall not be mentioned, or a full replica like RSCEmulation but with more strict replication rules.

Also, please voice your opinions on whether this is a poor investment of my time. I could easily switch this framework over to a RuneScape 2 server, if you collectively feel it isn’t worthwhile this late in RSClassic’s life.

Any suggestions at all are welcome; I am willing to take this in any direction you guys see fit. I would like to maximize playercount, and that is usually done by giving the players what they want

Feel free to email me or contact me on MSN, email is at bottom of post.

Aleksandr_Volkov August 4, 2016, 12:00am #19

Well the money issue was secondary to my wanting to play Classic again, and I started this project before I had seen the private server scene already existed on this site.

Either way I intend on finishing the project, the only decision to be made is whether I host it or release it under ISC or public domain.

Once I found this site and saw source code to projects like ‘EasyRSC’, I had decided to continue my project yet(this was after a decision to stop the project), as I disagreed with many design decisions, the code had unsavory hacks all over the place and seemed very messy to me at best. It had an event loop but didn’t make great use of it, and cluttered the loop with unrelated code in many projects source codes I’ve skimmed.

The design goals for my project are to create a responsive server(with a max delay of 50ms on LAN with a sizable amount of shit going on, may be lower if I decide this is too high I need to do live tests), that has a nice website and good gameplay.

Eventually, I’d like to pick up where RuneScape Classic left off, with a reimplemented client with modern design patterns and technologies(probably not in Java, but trust me when I say it will be portable and have bins or be compilable for all mainstream systems, including BSDs), use of modern technologies in rendering, such as OpenGL, and as backup OpenGL SW renderer, possibly target DirectX if I get really bored, etc. Maybe replace much of the 2D art in the game with 3D art, which can not be done by me(the modeling) as I am no artist, have APD and aspergers, and can not use 3D modeling tools on my system.

All of this talk of client-sided efforts is secondary to my completion of a RuneScape Classic server that I deem playable, of course.

Right now I am working on replacing some of the quick-fix hacks I’ve got in place for region system and some libev workarounds. I’ll try to keep you guys updated and might setup a git on my VPS.

Problem I am having with Region system.

edit: I should mention the server is designed modularly in a big way. The RSC extension is just one module; I do intend on creating an RS2 module for it and hosting an RS2 server eventually, but that is for another time =]
