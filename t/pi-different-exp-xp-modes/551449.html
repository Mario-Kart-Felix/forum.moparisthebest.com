sk8rdude461 August 4, 2016, 3:07pm #2

Yeah. It’s called using an Enum or a single integer.

Why the hell would you use 10 booleans for game modes?

public enum GameMode { EASY, MEDIUM, HARD, EXTREME; }

Then to save it you would just have to use the “toString” method. And for loading (Since you probably have the text file saving) you will probably need to make a for loop in the player loading, and then see of the text value of the saved Enum is equal to the toString value of the item in the Enum.

Alternatively, you can use an integer:

[code=java]
private int gameMode = -1;

public String getGameMode() {
switch(gameMode) {
case 0:
return “Easy”;
}
}[/code]
And I’d hope you know how to save that.

The whole part about the “finishedTut” boolean can easily be avoided.

  1. check if the saved enum value is null.
  2. check if the integer value is -1.
    If 1 or two is true (depending on your method), then send the dialogues. If not, don’t.

That should give you a good start…
