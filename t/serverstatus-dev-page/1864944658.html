Alright, but if you really want to garuntee the safety of your members, then simply:

  1. Get each and every client approved by a staff member (most likely MSCP Mods as global moderators cant be bothered.) This might seem like a hassle, but the people requesting approval have to be patient. It CAN be done.

  2. Once approved, that client can be uploaded to a seperate server, or even your own (moparisthebest.com/moparisthebest.com) and kept there. Bandwidth shouldnt be a problem, because just about the same amount will be taken up regardless if they use yours or their own. Space on that server could be a problem, but requirements (since most of the people here wouldnt think of these) could be something like: no cache included, no other files included, just class files etc. But if space IS a problem, then that extra server can come in. There are plenty of free hosts. And, now that I think of it, you can very cheaply direct link a .jar file from Uppit and run it like that?

  3. Setup a quick system linking to those approved clients. There could be a user/pass/email/other credentials needed to get the link to that client and to post it on the server status page, etc. You could also work on a login-system for the status page where the only way to view it is by logging in using your moparisthebest.com account. Then you can link between accounts and clients easily and efficiently. Not to mention, security-wise that would be a good idea.

  4. The only way a client on that server can be modified is if a staff member uploads over that file/overwrites it. There could be a few ways around this, something like having the link change every upload and disabling the ability to upload over existing clients. Instead, just delete them. That way no harm can be done unintentially by the server owner. And I’m pretty sure the staff here can be trusted enough. Maybe MSCP mods approve, then admins upload if you really do care.

  5. To quicken up the pace of approving clients, a quick bytecode scanner can be made. Sounds foolish, but when the task is to look through an entire project like we would have to, its a great idea. I’ve started on one earlier (in attempt to learn and practice regex in Java), and it actually turned out great. It can do something like warn you of dangerous or potentially harmful code. If some is found, then the moderator could quickly check on it, etc. And if speed is an issue, we could switch to BCEL or ASM, or any other bytecode library. (I only used regex because like previously stated, it was to learn and practice it. Turned out quite fun in the end.)

Well, there’s my point of view. I really think its possible, and wouldnt be too hard to fix up.
