I’ve gotten NOOBscape v3 deployed. I was a bit disappointed - its definitely got its bugs. But it’s working pretty well. Here’s a little overview of what I’ve done with it:

[ul][li]I’ve pulled the Applet off the server and have JRuby acting as the loader. Going to use that to eventually act as a management bridge between either a CLI or a web UI and the game server.[/li]
[li]Also slowly moving all of the persistence out of the flat file properties. Will probably use Mongo for production data storage. Will do the same for entities at some point, including client entities for easy entity updating.[/li]
[li]Going to attempt to tackle performance and scalability issues too. I moved the server over to ParallelGC. That had a noticeable impact, but I’m sure there’s a dozen more things that could help.[/li]
[li]Going to make a scripting interface for the client using a custom language - similar to AutoRune. Then I want to make a headless version of the client that can run a script.[/li][/ul]

Truthfully, I’m just tinkering on nights and weekends but it’s fun!
