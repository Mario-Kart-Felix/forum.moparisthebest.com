Ruby has inspired me to create a web design tutorial. Seeing as this site is all about Private servers, why not make something on that. I’ll be writing the code out as I make this tutorial. So it will be some time before it is fully up.

NOTE: I will not be using HTML syntax highlighting in the code brackets because I use some html5 elements and these forums only support html4.

Section 1: Setting up
So this is the easy step, I think it is easier to create websites with stupidly simple folder structuring.

Step (1) Let’s start off by creating a new folder (I will be using my documents). Right click, hover “New”, Click Folder. Then Title. Mine will be named “A Website Demo”.

Step (2) Moving on, Within that folder we have just made, lets create some more. I will be creating 2 more folders, “images” (no caps), and “assets”. This is really self explanatory. All site images will be held within my “images” folder. html documents will be held in the root folder (what folder we are located in currently), and any other code/files will be located in the ‘assets’ folder. I simply do this to organize the folders for myself. It is not needed, but a form of organization is recommended.

Section 2: Writing the HTML

Step (1) Let’s create the shell of our site, shall we not? (I am using Notepad++) Open up your designated text editor, preferably one with html and css syntax highlighting, as that is what we are using.

NOTE: Using Dreamweaver is recommended, simply because it is more openly used in the field, and has a lot more tools available to you. But it is not needed.

[Side Tutorial]
If you are using Notepad++ it is best you set up syntax highlighting before saving. As there are over 30 languages it supports.
Open Notepad++, click “Language”, then hover over “H” for Html (or the letter of your desired language), wait for the menu to expand, and click html. The document should shift slightly, and your good to go. Do The same for your CSS Document later on.

[/Side Tutorial]

Now, I start off by writing a small shell of the webpage, again, this isn’t needed but it’s always good to have it handy. Here is what I enter so you can copy paste (you learn better by typing it yourself):

The code, explained (A tid bit):

- In HTML 5, they changed the DOCTYPE tag to be simple, as with a lot of other tags, see W3 - The html element represents the root of a document. (This is really obvious..) - The head element collects the document’s metadata. (Users don't see the head element in use, but instead elements inside of it.) - The title of a document is specified by the TITLE element. Like when you see "Facebook" in the tab on Google Chrome. - The body element represents the body of a document. Or in other terms, the core of your page (There are exceptions, of course). Other element, codes etc. will be explained as we add them. Each element (the words within the angle brackets (Example: )) has an opening tag, and a closing tag. So if you have then you need . Now click “Save” or “Save As…” because it’s a new document it will give you the same prompt either way. navigate to the folder you made to contain your website (See Step 1) and save this document.

[Extra Information]
If you are using Notepad++ like I am, it comes with a built in “Test” feature that can be located in the “Run” menu of the program. This only works after saving.
If you are using Dreamweaver, it comes with plenty of built in test/debug features. Such as live view, and live code. It also allows you to split the screen so you see both the code and the live view of the webpage.
[/Extra Information]
