First you need the Map Editor which can be found HERE

How to run the editor
I recomend using the “runsrc.bat” since it allows you to turn the “object: #” on and off so you can actually see the map instead of loads of yellow “object: #” everywhere.

Loading a 377 cache
This is very simple. Simply open up “RS2 MapEditor.Standalone Jagex 3D Engine.src.com.rs.io.fs” and then open “JAGFS.java” and find:

if(fileLength < 0 || fileLength > 0x7a120)

Then change that to:

if(fileLength < 0 || fileLength > 0xffffff)

Similar to the way you load a 377 cache on a 317 client.

NOTE: You do not compile this so make sure you do download Oyxgen’s edit. After re-reading the tutorial I realised I didn’t say about compiling and jaring the stJ3D. Once again, make sure you do download one of the links below and replace it with your file or the map editor WILL NOT WORK.

Also make sure you use the runjar.bat instead of runsrc.bat. Then you should have no problems.

You can also download Oxygen’s edit.

The cache
Before you edit the map or any map, I recommend delete the cache folder. And copying your client’s cache folder to the editor. Make a backup of your cache so you know, if anything does go wrong, you will always have it safe, and be able to replace it if anything does go wrong.

You CAN use a 317 or 377 cache.

Getting the coordinates of a location
First load up your client and server, then login. There are two ways to do this.

  1. Once you’ve logged in, go to the place you wish to edit. Then logout, then go to your character file and look at the X and Y position. Note them down for a bit later.
  2. Again, once you’ve logged in, go to the place you wish to edit. Now type in ::mypos or ::cords (whichever one you have on your server). Note down the coordinates for later.

Now you have the coordinates, get your calculator! Either use your own, one on a phone, or the computer one. Type the second number in first, and divide it by 64, then do the same for the first number, note them down in this order.

Eg. Varrock centre coordinates are X: 3210 Y: 3424.
Divide them by 64:
3424 / 64 = 53.5 (53)
3210 / 64 = 50.15625 (50)
Now in click load map, a little box should pop up. Type in the numbers you just got. Look at the example below if you don’t get it.

Then click OK and the area should load.

Getting the coordinates of a location V2
First you need to get the coordinates of the location you want to go. So go to location and find the X and Y coordinates. Which is X: 2606 Y: 3093. Then divide them both by 64. So it’s X: 40 Y: 48, now because the Map editor is slightly weird, you have to put the X and Y backwards. So click the load map button, and type in the box 48 then a space, then the second number, in this case 40. Now click OK and it will load the map.

Please if you need more help, PM me or post back here and i’ll be happy to help.

Creating a basic map
Let’s load a nice blank area. The coordinates are X: 2701 Y: 2981. Now do what we did above.
If you got it right, you should have loaded a square of water, nothing else. If you haven’t I think you need to re-read the previous section and get it right.

Now on the creating the basic shape of the land, so you need to find the texture you want from the palette tab, I’m going to use road for this example. So select the road texture, by going to the Palette tab, and selecting “road”. Now go to toolkit and select “Paint Overlay”. Now we can begin to map the area. So find a starting point, and make it into the shape you want.

I recommend creating the outline of the map first, then filling it in. Also, if you make a mistake or decide you want to put it back to normal, simply select the water texture from the palette tab and click on the tile that you want bank to normal.

My map so far.

Now I’m sure you don’t want your map to be flat on the floor, so go to the toolkit tab, and select the “set height” button from the toolkit tab, select where you want to edit your map and use your mouse wheel to adjust the height.

Note: I recommend you click 1 tile away from the very edge to give an effect like real RuneScape land, also, you only need about 2 wheel clicks down for the height. You also may need to adjust it a bit more to get a good effect.

This will take quite a lot of time, depending on how big you do the map your making. Once it’s done, it should look smooth a long the top. Example below.

Mapeditorhelp5.png828×587

Now we will add a small area for a little square building. Simply select a texture from the palette tab. I’m going to use the “woodenfloor” texture. And I’m going to make a small square near the edge of the land. Like below.

Mapeditorhelp6.png660×576

Now, you need to make it walkable, so you must click on the tile you want to make walkable, then on the “Tile Editor” tab, make sure “Clipping” is Un-checked.

You can also add some grass around it, and make it look nice. This is my finished map.

Mapeditorhelp7.png682×536

Now go to the File menu and click Save. Now you can close the editor.

Adding it in-game

Now you’ve finished making the map I’m sure you want to add it to your client. First go to my other thread, and do the very short tutorial. Now simply replace your cache with the map editor’s cache.

Make sure you make a backup of your cache first!

Now let’s test it, so open up your client and server, and go to the coordinates: X: 2701 Y: 2981. (you may need to use ::noclip if you appear in the water)

Note: My map looks different from the examples because I used a bad cache and it would load in my client, so I had to remake it quickly.

To finish off, you just need to add objects and decoration, and you’ll be done.
