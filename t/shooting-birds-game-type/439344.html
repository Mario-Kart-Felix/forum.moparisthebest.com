Your simplest option is to use a sprite sheet and create a different image for each frame.

BufferedImage sheet = ImageIO.read(new File("sheet.png"));
int width = 30, height = sheet.getHeight(); //30px
BufferedImage[] frames = new BufferedImage[sheet.getWidth() / width];
for(int i = 0; i < frames.length; i++) {
    frames[i] = sheet.getSubimage(i * width, 0, width, height);
}

And inside your player model you could keep track of the current animation & frame and when you draw updates do:

g2d.drawImage(player.animation().nextFrame(), x, y, null);

Sorry for the delayed reply. Haha, seems like everyone’s wrote a top-down zombie shooter :p. Yes, it only takes 6-8 sprites per animation, and you can apply 2d transformations (rotation, translation, etc.) to make your character face the correct direction.

The math for rotating your player towards the mouse goes something like this:

float x_diff = character.x - (float) mouse_x;
float y_diff = character.y - (float) mouse_y;
float theta = Math.atan2(x_diff, y_diff);
float angle = theta * (180 / Math.PI);
g2d.rotate(-angle);
