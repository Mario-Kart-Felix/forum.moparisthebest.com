Do you mean to share passwords between client and forum? And then to also get whatever associated data from the RSPS server to the forums?

Depends on the forum software, but if you are forcing them to create their accounts for the server on the forums, see how the authentication works in the forum source and port it over to your server. Or you could just write a simple script and have the server query that script and return a response. You’d have to secure that response, however, or people could just spoof a correct response.

I’ve seen servers in the past do something like this. Much like Pwnd said, you could have some sort of automated registration script which contained the variables for username and password, and then executed. But you’d have to catch exceptions that say the username is already in use, or anything else that would impede registration, send it to the server (RSPS), then generate a code to send to the client that will display to the user the issue.

Ambokile August 4, 2016, 1:21pm #5

When I’ve linked up Grand Theft Auto servers with forums in the past, I’ve had the forum set up with a database on the forum server, and then a game server set up with a database on the game server. I’d then have a simple script which I could query with something like checkpassword.php?username=Ambokile&password=pass, which would then use the forum’s encryption method to return 1 or 0 depending on whether or not they gave the correct credentials. The player’s game statistics would be on the game server but their login information would be on the forum server. This is another approach for when you need to be sparing with your external connections.
