sdouble July 30, 2016, 3:56am #6

Is this your first attempt at learning a programming language? If so, I strongly recommend starting with something else. While being an extremely easy language (one of the easiest I have ever worked with), it isn’t a good starter language. It allows sloppy coding to work fine which teaches you very little. The best thing you will walk away with is basic programming logic. The code is not case sensitive, you don’t have to declare types, and it’s not even the latest VB available. The IT department at the company I work for doesn’t even allow our programmers to write in pre-VB.net My point is this: VB is a nice language for doing incredibly helpful things, incredibly fast. But it should not be your first stepping stone because there are no other languages similar. VB.net is VERY different from VB6 and the knowledge gained from learning VB6 won’t help you very much with VB.net If you really want to know a language, you will want to go with something like java or C because the way the code is written is very similar to so many other languages. At first I didn’t care for it much, but I’ve become a huge fan of java. The code is very clean and very easy to read. And since I’ve worked A LOT with javascript (not the same as Java, but yet again the code formatting is pretty much the same) and PHP, the jump to Java was very easy to make. Sure, they might be more difficult than VB6. But the fact that when you learn the structure, you’ve learned the structure for many languages, spending a little extra time at the beginning is worth the benefit in the long run. but when you learn the style of VB coding, you’ve only learned the style of 1 and a half languages. Since VB.net has gone more towards the way of other programming languages. I started programming with whatever was on the old Apple II GS computers. With the whole

10 blah blah
20 blah blah
goto 10
(I don’t even remember it anymore =P)

and my “real” programming began with VB3, I regretted starting with VB when I started learning other languages.
