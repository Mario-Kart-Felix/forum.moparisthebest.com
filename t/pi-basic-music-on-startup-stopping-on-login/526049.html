THIS IS ALL CLIENT SIDED

Honestly I didn’t see a good place for this to go, seeing as RS2 Client Hackings description didn’t really seem to fit this, plus this is a mere snippet and I didn’t wanna post 2 bits of code in that section. Anyways for this snippet you will need to have done this:

http://www.rune-server.org/runescape-development/google/tutorials/493470-pi-simple-music-sound-system.html

Now this is the easiest thing in the world to add, but I just figured someone who wanted to have a basic music system would want this. What’s this going to add? Basically this is going to add that classic client loadup jingle, and then stops it upon login. I’m pretty sure that’s what happened on runescape anyways…

So if you don’t already have this jingle1.mid then you can download it here: https://dl.dropboxusercontent.com/u/29233368/jingle1.rar

Just make a folder in you’re cache called Music and throw it in there. Not sure if you already have this jingle or not? Go to your main cache directory and look for ‘jingle1’. If you have it, good, just slap it in that Music folder. Now in your main startup method in client.java add this bit of code (which would be: public static void main(String args[]) {

Music.Music("jingle1.mid");

Just add it directly under try {

Now in the same class find your login method private void login and somewhere under there look for loggedIn = true;

Directly under that add this:

Music.stop();

What does this do? This basically says “if players login status == true then execute the stop method from Music.java” which obviously stops all currently playing tracks. Also, if a mod feels that this fits in the Client section then feel free to move it, as afore mentioned I just wasn’t sure.
