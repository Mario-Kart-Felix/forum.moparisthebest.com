RSCShadow August 4, 2016, 3:10am #5
On the second point, use jquery to provide responsive validation client side, then server side validate with PHP as well for those with javascript turned off. Look into the jquery.validate plugin for starters.
I recommend sticking to using built in HTML5 validation and a fallback if you [b]really[/b] need it to work in IE. It's a lot less of a pain in the ass and you don't have to work with poorly-written code.

http://www.the-art-of-web.com/html/html5-form-validation/
http://html5pattern.com/

The rest is fairly straightforward, just make sure you escape strings when making any communication with the database, use sessions to store the current login (cookies are insecure), and use a secure method of encryption for passwords such as Bcrypt (md5 has been cracked, and I think SHA1 has a vulnerability or two).
You need to use cookies to store the session though. Cookies themselves aren't insecure (if your website doesn't have any exploits), storing the username and password in them is though.

SHA1 also hasn’t been cracked, it’s just not meant for password storage. It’s a general hash that computes too quickly so it isn’t ideal for password storage. Same with SHA256, SHA512, or any other similar hash. Bcrypt is slow and can’t be sped up “unfairly” using different hardware (GPUs). This site explains it in more detail: http://codahale.com/how-to-safely-store-a-password/
