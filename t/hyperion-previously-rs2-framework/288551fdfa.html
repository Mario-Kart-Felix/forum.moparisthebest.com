Graham July 31, 2016, 1:29pm #160
Palidino_:

Jesus, 50-80% CPU usage? My server uses an average of 1-5%, and I could swear even Winterlove servers use under 20%.

Can you give more information about your system, how many players you had logged in, etc?

I’m saying this, as performance in Hyperion is going to vary as updating is done in a thread pool with one thread for each CPU core.

On my PC, which has a dual core 2.2ghz CPU, 2 GB of RAM, Windows 7, Java 1.6.0_16 and running Hyperion using the server JVM, I get the following stats:

0 players - 111mb of RAM used - 0% CPU usage
1 player - 112mb of RAM used - 0% CPU usage
100 players - 118mb of RAM used - 3% CPU usage
200 players - 125mb of RAM used - 10-15% CPU usage
500 players - 147mb of RAM used - 24-30% CPU usage
1000 players - 186 mb of RAM used - 50-60% CPU usage (above 50% is good here as it shows it’s splitting load over multiple cores, else it’d only reach 50%)
2000 players - 259 mb of RAM used - 88-89% CPU usage

So it does reach above 50% CPU usage when you get 1000 players online (on my system), were you testing with that number of players?

(These tests were conducted locally with allgofree bot and one other player logged in on the client).
