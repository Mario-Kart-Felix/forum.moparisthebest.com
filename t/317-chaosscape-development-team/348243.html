Introduction:

Hi guys,

Before I go into depth about the development team and products I want to tell you a little about myself, I hope that this about me section will help make up your mind if this is a worthwhile project to take part in and if you think I’m the sort of person who would make good use of your time.
Hi, I’m EwfsPrince (my alias) I’m 18 years old from South Australia and currently finishing high school. My biggest specialty is web design and coding, which I’ve been doing ever since I was 10 years old.
I also code Java but I admit I’m not as fluent as I would like to be in the language, however there are some coding similarities between Java and PHP for example which have made learning a lot simpler and I am usually able to overcome any problems with my Java work – even though sometimes they may take a bit longer than the typical fluent coder.
I am recruiting a team of dedicated people to help assist in the development of ChaosScape and in addition also in the development of our server related products that will be released under GPL licensing to the public.

All members of the development team will be rewarded for their contributions, one way we reward everyone is by giving them free Unlimited hosting + cPanel (subject to fair use policy) which is provided by the web hosting enterprise that I run and manage.
We want our team to be successful, which is why we are spending a lot of time creating tools and privately owned websites to help create a friendly working environment. We consider ourselves a different breed of development team and we do require very mature team, we prefer to recruit people who are over the age of 16 but we are willing to make exceptions if maturity is shown in the application process.

Our goal
Our goal is to change the way we develop private servers. Not in the types of servers we code, not whether we code 317’s or 600+'s but the way we think about developing our servers. For too long as MoparScapians have been held back in our progress by leechers and over time our more confident coders have left the community because of them and the fact that the RSPS’s aren’t moving forward quickly enough.

What do I mean by moving forward?
All of our servers are underdeveloped, there isn’t a server out there that isn’t and it’s because of a number of reasons which most of you would probably be able to work out but one of them is that we don’t collaborate our work, we mainly post them on a forum and be done with it. It is proven when we share and talk about these things we learn differently to how we would just by reading, we soak up new information that we didn’t understand before.
There’s more I want to say, but I won’t waste any more of your time on this point.

So, ChaosScape is more than a server project and I’m just going to give you a brief feature list of what is going to be released:

[list]
[li]A 317 standard source[/li]
[li]A 317 MySQL server version source[/li]
[li]A fully functional RuneScape site with PHP + MySQL with intergration with the MySQL server.[/li]
[/list]

But it doesn’t stop there!
We will also be creating tutorials, reference guides specific for our server source which will in effect teach you how to develop with our source code which will make it an ideal base for your next project.
All of the code is open, none of it is anti-leech.

Won’t this project source just get leeched?

All of our work will be continuously worked on, this means that leechers will have NO chance to compete with us. In my theory leeched servers only occur when a development team stops working on their source, and then posts it and 2 months later (once everyone forgotten about it) it gets posted under a different name. This source is more like a package and once it’s been released it’s structure and identity will not be mistaken for anything else and the average user will be able to tell how much the source has been modified. EASY!

We have most of the site and both 317 server sources almost built, they just need a few more patches than I will release the first distribution to the public. Once it has been released the development team will be able to develop on it and make changes which will be released in the next version and so forth.
Some of you might be wondering why I was discussing about collaborating work, at the moment it won’t make entirely sense to you but in the release of the first version it will become clear.
