Anyway, regarding what has been said by him so far:

  1. Using this/super is obviously optional. I always use the this keyword in constructors, mutator methods (setters), and overridden versions of equals(Object). Unnecessary, but ultimately just a matter of style. I use super in constructors too. As long as it remains consistent, it‚Äôs fine.
  2. Given that his code is quite readable, commenting on style adds no value to the discussion.

As far as usable feedback goes on my part, I offer this:
You should have designed your bot in such a way that it was extensible, because adding extra functionality in its current state is going to be a nightmare.

What you really need is an input handler interface of some sort, and then you can implement a concrete input handler for each type of expected input. You would then map a String (such as PRIVMSG) to an InputHandler, and when that type of message arrives you would delegate the handling of the message to the appropriate InputHandler.

You could then reuse such an interface to handle commands too, by having the PRIVMSG InputHandler have its own collection of InputHandler classes, each designed to handle a single command.

Check the Java 6 API for java.util.Map and java.util.HashMap.

As for the interface, you could use:
public interface InputHandler {
void handle(String[] inputArgs);
}

Obviously inputArgs would be the string of text that the bot received from the server, split up into parts using an appropriate regex. By passing it to the handler in that form, you avoid extra operations on the input before delegating it to the handler.
