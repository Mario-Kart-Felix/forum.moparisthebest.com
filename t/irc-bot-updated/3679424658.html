Aftermath_961 August 1, 2016, 12:57am #35

From a design point of view, you’d be better off parsing output from the server into the commands (JOIN, PRIVMSG, MODE) and then forwarding them to onJoin/onMessage methods. Then you can extend your Bot class with a more specialized version of those methods while leaving the IRC logic intact. I suggest you look at the PircBot source, it’s very well-designed.

Also, it looks like your Connection class is unused, so I didn’t really read through it.

Your Bot appears to handle input and output in a single thread, which is a bad idea. You should have a separate thread for input and output (or use the new IO [java.nio] to do both in a single thread, but do not use the old blocking IO in a single thread for both reading and writing).

Your code appears to have several points where output is written but not flushed. This will result in commands effectively being queued until something comes from the server that triggers a flush() call in your code. You should probably call flush() wherever you’re done writing.

Also, if you’re making a bot, you’d be better off just using PircBot as it’s already been done before (and well), but if this is just for practice, that works too. Good luck.
