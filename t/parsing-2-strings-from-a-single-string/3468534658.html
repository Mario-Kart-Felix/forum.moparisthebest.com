If you don’t like things doing work for you, you probably shouldn’t be using javac. It takes the source code, checks it, and then automatically converts it into bytecode to be ran in the java virtual machine. You could instead write your java programs directly in bytecode and skip the whole javac/compiler thing. It makes just about as much sense as not using a tool to make your life easier, at least a decent text editor with tab support and syntax highlighting. It’s like trying to nail something with a brick instead of a hammer, you will get the job done but it will take much longer and you won’t get as good of an end result.

And again, you would get much better experience from implementing an official, documented protocol such as IRC.
