xEnt July 30, 2016, 11:56pm #4

I’m not quite sure what you’re saying, but player.getActionSender().sendStats(); just updates the client’s of the current stats/Exp. I think what you are after is to check if the Exp you added/decreased effects the Level change. I don’t think the client does this check, the server just sends it the stat numbers.

in the Package com.rscd.util it has a Formulae class. In there is a method called experienceToLevel, you give it you’re exp and it returns the level. (using incExp this should auto set the level)

since you’re using setExp you should do something like this.

int newDefExp = getExp(1) - 5000;
int newHpExp = getExp(3) - (5000 / 3);
setExp(1, newDefExp);
setStat(1, Formulae.experienceToLevel(newDefExp)); // This calculates the Exp to level, and set’s the stat level to the new exp.
setExp (3, newHpExp);
setStat(3, Formulae.experienceToLevel(newHpExp)); // Same thing, but for HP.
getActionSender().sendStats();
checkEquipment(); // Just in case you’re levels have changed that were required to wear something, it will check and remove equipment.
