lothy July 31, 2016, 10:13am #8

I’ve actually wondered how castle wars works for a while. Are the battleground and waiting room sections actually within a single region, which means that people are declared ‘in-game’ by being within this region?
It may well have been related to clipping, but this all depends on how the server maintains the group of players that are currently within the game.

For example, if they maintained the players by adding them to a collection of some sort (after teleporting them into the battleground part when the game starts) then the two guys holding the flags would have been subject to the same game mechanics that everyone else is subject to (ie, they’d be teleported back into the lobby after the game finishes, and if they had the flag equipped then it would be removed from them), despite being outside of the game area.

If however the game didn’t finish as written above (ie, everyone within the region was teleported to the lobby and so on) then leaving the region would exempt you from this ‘game over’ process, thus allowing them to keep the flags.

There’s really nothing wrong with the second solution because you can usually expect that clipping won’t be an issue, meaning the players could never leave without the game ending or the use of the portal in the respawn room.
