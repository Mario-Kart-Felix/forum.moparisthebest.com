moparisthebest August 1, 2016, 7:50am #15

I have used the sponsored server money to purchase a server for MoparScape.org, which will be released in the next few days after I finish putting some final touches on MoparScape. Afterwards, I’d like to finish Cherokee into a full-fledged runescape bot, to do this, I need an updater, and I don’t particularly want to write one myself, so I’m asking here if anyone would be willing to write one for the community? I could pay you out of the sponsored server money to do this. It needs to output mappings to a text file (instead of modifying bytecode, this allows us to distribute a legal text file, as opposed to copyrighted code) , preferably similar to the format already supported by Cherokee. This updater needs enough fields to enable calculating the positions of world tiles, interfaces, and objects, npcs, and items in the world, possibly injecting callbacks to events in the future. Would anyone be willing to do this? I’m thinking the updater itself should be GPL’d (or similar) and publicly available, with example trivial transforms/patterns, with the complicated ones kept a secret so Jagex can’t intentionally break them.

Here is what I am thinking bot-wise:

Right away:

[ol][li]Move to Github[/li]
[li]The input package is mature and still works well, I don’t think we have to do anything/much of anything to it.[/li]
[li]I plan to restructure it quite a bit, roll some of the applet-loading and bytecode-editing code into the main Cherokee and out of the rs package.[/li]
[li]Incorporate some changes into the AppletStub applet-loading part, including sending the correct User-Agent, and rolling in the SecurityManager I wrote for MoparScape 4 with the permissions it expects for an unsigned client (option for signed/unsigned I guess)[/li]
[li]Write the AccessorMethods class specific for RS2, to provide helper methods to the scripts, so it can start being used right away for runescape as a full-fledged, undetectable bot.[/li][/ol]

At this point the bot would work (with the same level of ease as Aryan, just not detectable), and we can continue to improve it from a ‘good’ bot into an excellent bot.

[ol][li]It already supports aryan-style scripting, but it should be easy to abstract it a little more to support different types, such as event based scripting, and even scripting with other languages (python, ruby (for Mopman), javascript, lua, basically whatever)[/li]
[li]Support for getting access to applets in different ways, such as with a java agent, and maybe even attaching to the VM after it is already running, if feasible.[/li]
[li]World Walker.[/li]
[li]Remote control/headless support.[/li]
[li]Plugin for Simba to use functionality provided by Cherokee. (modification of SMART)[/li]
[li]Proxy support.[/li]
[li]And of course anything else that you can imagine.[/li][/ol]

That was kind of “an unorganized stream-of-consciousness” post as well, but I figured I’d let you guys know my plans, and get input as to whether or not anyone here would be interested in writing the updater and/or contributing to the bot itself.
