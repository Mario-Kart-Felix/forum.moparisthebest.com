CodeForFame August 4, 2016, 1:45am #13

So, I’ve been tinkering with this… Logging every packet that comes in (except ping, ‘trap’, dummy, and bare packets) seems to be around 100 packets per minute per player. With 200 players on for 24 hours, that’s only 28.8 million packets per day. Just storing each packet as plain-text in the following format:
hh:mm:ss:SSSS playerName opcode payload

That comes out to around 71 bytes per entry. Though, it greatly depends on what each user is doing. That comes out to about 1.9GB of logs generated per day for a constant 200 players. Assuming a 50% compression rate (which is pretty reasonable considering the number of packets that end up being identical) you’ll end up with around 1GB of logs per day. A 500GB harddrive is pretty standard on servers, so you should be able to hold 500 days of logs at a time.

Performance wise, it seems just fine, even with log files that are 4GB+ (though, I was using a SSD). I was able to add like 100m entries, 1 at a time (without closing/opening the file), in under 30 seconds. So, if you just have it write to disk every minute it won’t be noticeable, especially when it’s on a different thread.

lothy August 4, 2016, 1:45am #14

You log actions, not packets. So you wouldn’t record the payload; instead, you would record the outcome of an action.

Additionally, this goes in a database. So say you log an item drop:
8 bytes - unix timestamp.
4 bytes - player id (foreign key referencing player table).
2 bytes - item id (foreign key referencing item table, but this id could/would also correspond to the actual item id in the game).
4 bytes - the amount dropped.
In short, the memory footprint is considerably lower than your prediction.

[code=Scala]package org.moparscape.msc.gs.log

import org.moparscape.msc.gs.connection.RSCPacket

abstract class Logger {
def log(P: RSCPacket)
}

package org.moparscape.msc.gs.log

import akka.actor.Actor
import org.moparscape.msc.gs.connection.RSCPacket

case class Action(p: RSCPacket)

class LoggingService(logger: Logger = new BatchedLogger) extends Actor {
def receive = {
case packet: RSCPacket => logger.log(packet)
}
}

package org.moparscape.msc.gs.service

import akka.actor.Actor

class IOService extends Actor {

private type F = () => Unit

override def receive = {
case f: F => f()
}
}

package org.moparscape.msc.gs.log

import java.io.File
import java.io.FileWriter
import java.text.SimpleDateFormat
import java.util.Calendar
import java.util.concurrent.TimeUnit
import org.moparscape.msc.gs.Instance
import org.moparscape.msc.gs.connection.RSCPacket
import org.moparscape.msc.gs.model.Player
import akka.dispatch.ExecutionContexts
import scala.concurrent.duration.Duration

class BatchedLogger extends Logger {

private var packets = ListRSCPacket
private val ignoredOpcodes = List(-1, 0, 3, 5)

Instance.loggingSystem.scheduler.schedule(Duration(1, TimeUnit.MINUTES), Duration(1, TimeUnit.MINUTES)) {
Instance.IOService ! write _
}(ExecutionContexts.global)

private def write {
val today = Calendar.getInstance.getTime
val data = packets.map(e => (
if (e.isBare) “not logged in” else e.getSession.getAttachment.asInstanceOf[Player].getUsername,
if (e.isBare) -1 else e.getID,
e.getData.mkString(","),
new SimpleDateFormat(“hh:mm:ss:SSSS”).format(e.getCreated))
).reverse
val s = data.foldLeft("")((a, b) => if (ignoredOpcodes.contains(b._2)) a else a + b._4 + " " + b._1 + " " + b._2 + " " + b._3 + “\n”)
val f = new File("." + File.separator + “logs” + File.separator + new SimpleDateFormat(“MM.dd.yy”).format(today) + File.separator + “event.log”)
if (!f.exists) {
new File(f.getParent).mkdirs
f.createNewFile
}
val out = new FileWriter(f, true)
try out.write(s) finally out.close
}

override def log(p: RSCPacket) {
packets ::= p
}

}[/code]
