[SIZE=6]STEP 2: SETTING UP ENVIRONMENTAL VARIABLES[/SIZE]

Alright now that we have downloaded what we need for running a server lets get our/your server ready for playing

Setting up Environmental Variables:

  1. Now once you have downloaded Jave JDK SE you are going to need to set up environmental variables so that your computer knows where to find and use Java.

  2. Let’s do this by clicking START:

  3. Once your in START click on COMPUTER:

  4. and go into your C:DRIVE or Local Disk:

  5. In that folder you may have to scroll down but go into PROGRAM FILES:

  6. Once in that folder go into JAVA:

  7. In the folder Java you will see Jdk 1.6.0_26 or whatever version you downloaded:

  8. go into that and go into “bin” and right click on “appletviewer” the last option “Properties” select:

  9. Now a new window will pop up the third information bar will be Location: C:\Program Files\Java\jdk1.6.0_26\bin

  10. COPY C:\Program Files\Java\jdk1.6.0_26\bin

  11. Now go back into START:

  12. RIGHT CLICK COMPUTER and select the option Properties:

  13. On the LEFT select the option “ADVANCED SYSTEM SETTINGS”

  14. A new window will pop up and on the bottom right select the option “ENVIRONMENTAL VARIABLES”

  15. There will be two things User Variables for “Your name” and System Variables we will NOT BE TOUCHING SYSTEM VARIABLES:

  16. In the User Variables Select New and name it PATH

  17. the Variable Value will be this - %JAVA_HOME%\bin

  18. Click Okay and select New in User Variables again:

  19. Name this one JAVA_HOME it is also essential for the name to be exactly like that:

  20. The Variable Value will be this C:\Program Files\Java\jdk1.6.0_26\bin\

NOTE: THE 26 IS THE CURRENT JAVA RELEASE IF YOUR VERSION IS DIFFERENT PLEASE CHANGE THE 26 TO YOUR VERSION!

  1. Select Okay and Okay to the next 2-3 windows and it will be saved

  2. So go ahead and access your VPS by going into START and search “Remote Desktop Connection”

  3. RUN the program and type in your username and password for your VPS:

  4. Now that we are in our VPS we are ready to get going!

Here is where we get our server running on our VPS:

  1. Wherever you have saved your Source for your server go to that now and RIGHT CLICK it:

    [SIZE=6]
    STEP 5: GETTING YOUR SERVER UP AND RUNNING    [/SIZE]

  6. In the Source on your VPS RIGHT CLICK Compiler and change the JDK version to your version 26 in the case you followed this guide.

  7. Now that we have changed that LEFT CLICK Compiler and run it, once it has finished Compiling you will click any button to continue.

NOTE: YOU MUST COMPILE AFTER ANY CHANGES YOU MADE TO THE SERVER!

[SIZE=6]STEP 6: Hooking your Client up with your VPS![/SIZE]

  1. Run the Server on your VPS and go to your Client on your own desktop.

  2. Open the client and go into GUI.JAVA and change the IP ADDRESS to your VPS.

  3. Once you have changed it save and go into CLIENT.JAVA and search for “127.0.0.1” and change that to your VPS as well.

  4. Change the Compiler in the Client to your version and Compile!

  5. Run the Client and if you can log in your server is now up and running!

Now that you guys have your server up and running you can have some fun editing and adding to your server and Now I will teach you guys how to do that now!

  1. Now go into SERVER / WORLD

NOTE: WHEN I PUT A / THAT MEANS THAT WHATEVER COMES AFTER THE / IS INSIDE WHAT IS BEFORE THE /

[SIZE=5]EXTENDING ON OBJECTS MAKING THEM HAVE ACTIONS![/SIZE]

  1. Remember the object ID and go into SRC / SERVER / MODEL / PLAYERS / Actionhandler.java

NOTE: THE / IS WHATEVER IS AFTER THE / IS WHAT IS IN WHATEVER IS BEFORE THE /

  1. Search for movePlayer and you should see something like this

case 70: c.getPA().movePlayer(2872, 5269, 2); c.sendMessage("You have entered Armadyl, To leave click the Pillar."); c.sendMessage("Note: Ruby bolts (e) and Diamond bolts (e) are recommended!"); break;

the “case 70” is the object and the c.getPA().movePlayer is the command on moving the player to whatever is in the ()'s the “2872” being the X COORD the “5269” being the Y COORD and the “2” being the height. Ignore the c.sendMessage this is an example and I used the God Wars dungeon.

NOTE: IF YOU WANT TO MAKE THE OBJECT TELEPORT THE PERSON INSTEAD OF JUST MOVING CHANGE IT TO SOMETHING LIKE THIS:

case 5959: c.getPA().startTeleport2(2539, 4712, 0); break;

and it is the same thing as editing or adding the movePlayer except it is startTeleport and will have the animation of teleporting.

NOTE: IN SOME CASES THAT WILL CAUSE AN ERROR! IF IT DOES CHANGE IT TO .startTeleport(X, Y, H “modern”); AND IT SHOULD WORK FINE!

  1. Do ::mypos where you want the npc and make a new spawn for example i will use the npc 494 a banker:

  1. Compile and do ::update 0 and close and re-run your server!

  2. If you have done it correctly your npc should be right where you put it!

[SIZE=5]MAKING A NPC A SHOP![/SIZE]

  1. Now that you have an npc spawned go into DATA and go into shops.cfg You will see something like this:

NOTE THIS IS NOT MY FULL SHOP BUT FOR AN EXAMPLE I CUT IT DOWN A BIT!

  1. Make a new shop and make the shop = any # but one that has not been taken already, for this case I will use 100.

  1. “New Shop” is the name of your shop and make sure to give it a name!

  2. the 2 and 2 is what the shop is so this 2 is buying only a 1 would be selling and buying.

  3. next are the items 10458 is the item ID for example and 10 is the amount in which is in.

NOTE: YOU MUST HAVE A TAB AFTER THE shop = 100 or whatever number your shop is!

  1. To find the item ID’s you can go into item.cfg!

  2. Once you have found all the ID’s you want in the shop go into SRC / SERVER / MODEL / PLAYERS / Actionhandler.java

  3. Once in actionhandler.java search for (10) and you should see something like this

case 919: if (c.isDonator >= 2) { c.getShops().openShop(10); } else { c.sendMessage("You must be at least a Super Donator to view this stop!"); return; }; break;

this is a donator shop the case is the npc ID and if your making a shop that has no need for rights you will see something like this :

case 919: c.getShops().openShop(10); break;

The “case 919” being the npc ID you were wanting to use and the c.getShops() being the getter or Accessor .openShop() being the command to open the shop and the number inside those () is the shop number.

So to open our shop 100 you would do something like this:

case 919: c.getShops().openShop(100); break;

NOTE: YOU MUST PUT BREAK AFTER OR IT WILL CAUSE MANY ERRORS , THIS IS BECAUSE BREAK; IS MEANING THAT THIS CASE IS DONE EXECUTING.

  1. Compiler do ::update 0 and close and re-run server

  2. If do correctly when talking to npc it will open the shop and have the items you put in!

[SIZE=5]EXTENDING ON SHOPS[/SIZE]

NOTE: IF YOU WANT TO ADD RIGHTS OR RESTRICTIONS ON SHOPS YOU WILL NEED TO ADD AN IF STATEMENT LIKE SO:

case 919: if (c.isDonator >= 2) { c.getShops().openShop(10); } else { c.sendMessage("You must be at least a Super Donator to view this stop!"); return; }; break;

This is a donator shop like shown above it is stating that if isDonator is greater or equal to 2 then open shop, else if it is not send the message blah blah, this is useful for making a donor shop at home or an object such as a portal that you want to have restrictions on, it will cause curiosity in the players which makes them want to have donor
