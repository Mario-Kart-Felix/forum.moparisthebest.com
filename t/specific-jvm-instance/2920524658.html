xeez July 31, 2016, 10:19am #35

frank_, one can benefit from both. The attach API requires one to pass the process id as a parameter in order to retrieve its respective JVM. So basically, one can list all processes (via whatever method one wants) and look for processes that are named “java.exe” (or “java”, depending on which OS you are running), get their process id’s, iterate through them one by one in which, through each iteration, one can get the respective virtual machine, attach an agent that implements the [tt]java.lang.instrument.ClassFileTransformer[/tt] class, revise the bytecode for specific patterns, and done. You can also simply implement the [tt]java.lang.instrument.Instrumentation[/tt] class, and look for the class’ “code source”, as oksuper_ suggested.
