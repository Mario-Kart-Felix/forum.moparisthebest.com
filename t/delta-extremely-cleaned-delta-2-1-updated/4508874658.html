ExtremeX-Scape August 3, 2016, 7:44pm #24

I’m just talking methods that have a specific call to the client / player classes.

You don’t have to make literally everything static. (i.e: variables, getters. etc.) You can still access the class by an instance. (to get getters, public variables, etc)

private static Class instance;

public static Class getInstance() {
    if (instance == null) {
        instance = new Class();
    }
    return instance;
}

or a method similar to the one you have. Were just creating the class (but not referencing the class by everyone)

[code=java]
private static final Class class = new Class();

public static Class getClass() {
return class;
}[/code]

Note: The accessor for the second doesn’t have to be static.
