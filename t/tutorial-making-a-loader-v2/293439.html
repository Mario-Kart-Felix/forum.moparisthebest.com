Now this may all seem kind of weird looking at first, but don‚Äôt worry, i‚Äôll explain. Please feel free to refer to the Javadocs for more in

depth discussion on any of the objects found here.

appletResize, getCodeBase, getDocumentbase, getParameter and getAppletContext are all methods inherited from AppletStub. To

keep descriptions brief, getParameter(String name) returns a parameter for the applet (discussed later) from the HashMap we created.

getCodeBase and getDocumentBase are simply for the Applet itself, view the javadocs for more info. getAppletContext and

appletResize are useless to us.

Parameters are peices of data used by the applet to manipulate itself. These include height, width, world id etc. We store them in a

HashMap which means we can access the parameter via a String we map it to.

We also, as you can see, leave the constructor for our class empty, for now.

Seeing as they don‚Äôt really have a choice, Jagex leaves the parameters for their applet in the Javascript code. It can also be seen in

HTML if you tell the server you don‚Äôt have javascript. Take a look: http://world37.runescape.com/plugin.js?param=o0,a2,m0,s0

Now we have the Applet all prepaired to be launched, now we just have to actually load it. For this we will be using a ClassLoader, or

more specifically, a URLClassLoader.

As you probably know, runescape can be loaded as either Signed or Unsigned. In which case it uses loader.jar or runescape.jar

respectively. For the purpose of this tutorial, we will load an Unsigned client, using runescape.jar and the ‚Äúclient‚Äù class. However, if

you are looking to load a Signed client, point your ClassLoader to ‚Äúhttp://world37.runescape.com/loader.jar‚Äù and the main class is

‚Äúloader.class‚Äù.

Now that we have that out of the way, let‚Äôs create our ClassLoader.

ClassLoader clientClassLoader = new URLClassLoader( new URL[] { new URL( "http://world37.runescape.com/runescape.jar") });

Fairly straight forward, it creates a new ClassLoader (using the URLClassLoader object) and assigns it the runescape.jar classes. Next,

we create a new instance of the main class, and cast it to an Applet object.

Class<?> clientClass = clientClassLoader.loadClass("client"); Applet loader = (Applet) clientClass.newInstance();

Just use the ClassLoader to load the main class (client.class) and then create a new instance of it and cast to Applet.

The field ‚Äúloader‚Äù in now our Applet object which will be the game. Therefore, we want to make sure that Applet uses the stub we

have been working on, so we just call:

loader.setStub(this);

Now let‚Äôs do some editing of our frame. Since our class extends JFrame, it is initself our GUI, so there is no need ot create a new

JFrame object.

setTitle("Runescape Loader ~ The Bank"); JPanel theGame = new JPanel(new BorderLayout()); theGame.setPreferredSize(new Dimension(768, 560)); setDefaultCloseOperation(EXIT_ON_CLOSE); setSize(768, 560);

This is pretty self explanitory. We set the title of the frame, create a new JPanel object (it will be used as a container for our Applet

later), set the sizes of both the panel and frame, and set the default close operation to exit the whole application (close that JVM

instance).

Now we add the actual Applet, and start it up.

loader.init(); loader.start(); theGame.add(loader); getContentPane().add(theGame, BorderLayout.CENTER);

Now our loader is done!

Here are some common errors associated with loaders, and some soultions.

OutOfMemory

This error is thrown when the JVM is out of memory to run the rs client. The client uses a fair amount of memory, so don‚Äôt run too

many ‚Äúbig‚Äù applications in the background. Try assigning more memory to the JVM via -Xmx. This error occurs at runtime.

Could Not Reserve Heap

This error is thrown when the JVM can‚Äôt reserve enough memory to run the application. Close any big programs you are running

and check what processes you have going. This occurs before runtime.

Other JFrames freeze

If you are making a bot and when you start the loader all the other frames in your bot freeze, it means you have this issue. As already

stated, the rs client uses a lot of memory. The only way to get around this is to run your loader in a separate thread.

If you have any more errors, feel free to post!

Also, if I have made any mistakes, please let me know.
