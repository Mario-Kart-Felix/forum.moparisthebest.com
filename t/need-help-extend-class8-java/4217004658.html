zyle1992 August 2, 2016, 6:51am #30
tyb97:

^A switch can improve efficiency, because in an else-if setup, it has to check all the choices until one checks for true, in a switch, it goes directly to the correct choice.

It can improve performance, ONLY in large groups (5+). Anything less, and the number of bytecode instructions necessary to get the switch block to work is greater than those required in a simple if-else.

Also, as I stated before, if-else blocks and switch blocks work the same. switch blocks don’t “go directly to the correct choice”. It tests each one, just like a switch statement.

To the JVM

if (anInt == 0) {
    doSomething0();
}
else if (anInt == 1) {
    doSomething1();
}
else if (anInt == 2) {
    doSomething2();
}
else {
    leave();
}

is exactly the same as

switch (anInt) {
    case 0:
        doSomething0();
        break;

    case 1:
        doSomething1();
        break;

    case 2:
        doSomething2();
        break;

    default:
        leave();
        break;
}
