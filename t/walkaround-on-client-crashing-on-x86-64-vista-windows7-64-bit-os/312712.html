This is a malformed walkaround for the people who are having problems with this.

Most run batch files are simular to this:

Quote:
@echo off
Start java -Xmx500m -cp .;Theme.jar Gui
exit

The walkaround is basicly simple, just replace Start Java with the directory to your java executable.

@echo off
cls
"C:\Program Files (x86)\Java\jre6\bin\java.exe" -Xmx500m -cp .;Theme.jar Gui
exit
pause

mod_taharok July 31, 2016, 12:21pm #19

If you are referring to the clientside walking algorithm crashing clients in the 64 bit VM, then all of what you said is wrong. The reason the client crashes is because there is some sort of error in the hotspot compilation of the walking algorithm (or related code). I don’t know the VM enough to tell you why this error is happening. Send the error logs to Sun so they can fix it.

Anyways, a solution is to run the client in interpreted mode using the runtime flag [tt]-Xint[/tt] . I must warn you that this will make the client extraordinarily slow. Hotspot compiles popular (or often executed) bytecode into native machine code to speedup the processing of Java applications. Disabling it makes you taste what Java was like prior to hotspot (a real piece of shite, for the most part).
