The netty 4.x is very similar to the 3.5, a lot of things have been renamed so you might have little trouble finding the classes / methods you need.

Usually a server only needs one encoder, decoder and login decoder. Login decoder is the default decoder for when clients open connection because the first data they send is the login data, after you have decoded and handled the login data properly you need to replace logindecoder with the game packet decoder.

On topic: I prefer Netty because it has always been more efficient on my servers and I’ve became really familiar with it so creating networking for any application is really easy for me.
