Getting RS2 Data Files by RichyT

Well, I was bored last night, so I had a play with the .dat files.

These files contain everything (I believe), with a hex editor, you can edit NPC names, what they look like, their levels. You can edit the graphics (if you figure out how it works)

Hell, you could re-skin the whole thing <3

Firstly, you need to dump all the files. Start by making a DUMP folder relative to where you run your bot from.

Make a new file, call it SaveByte.java, and stick this in it:

[code]
import java.io.*;

public class SaveByte
{
public static void save(String s, byte[] aByte)
{
try
{
FileOutputStream fos = new FileOutputStream(System.getProperty(“user.dir”) + “\dump\” + s);
fos.write(aByte);
fos.close();
//System.out.println(s + " saved!");
}
catch (Exception e) { System.out.println("error saving " + s); }
}

public static byte[] open(String s, byte[] aByteO)
{
	try
	{
		byte[] abyte = new byte[aByteO.length];
		FileInputStream fos = new FileInputStream(System.getProperty("user.dir") + "\\dump\\" + s);
		//System.out.println("opening file: " + s);
		fos.read(abyte);
		fos.close();
		return abyte;
	}
	catch (Exception e) { System.out.println("error loading " + s); return null; }
}

}[/code]

Comment out the debugging if you want, but it slows things down a little.

Next, you need to dump the files!

Haul your ass on over to Class18.java

Find the method, method197(String s, byte abyte0[])

Basically, this method is returning the contents of each file (s), as a byte array.

If you scroll down, you come accross this little thing.

return abyte0;

Now obviously, that is returning the byte of the file.

So, for the first run, your going to have to first, declare:

String lastS;

At the top of Class18, I’ll explain later.

Then, stick this back in the method.

if(!s.equalsIgnoreCase("index.dat")) SaveByte.save(s, abyte0); else SaveByte.save(s + "-" + lastS, abyte0);

Put that, just above

return abyte0;

Now, run!

Look in your DUMP folder, and blam, everythings there.

Now, originally, there are not all those index files. However, RS loads a new one each time when unpacking a new file, so they have to be renamed (they are all called index.dat) As far as I can guess, they map the files or something, but honestly I’ve no idea!

Now, for the second run. Comment out all that stuff we added previously, and go back to method197.

Put this code where it should be:

[code]
if(!s.equalsIgnoreCase(“index.dat”))
return SaveByte.open(s, abyte0);
else
return abyte0;

//return abyte0;[/code]

That means, if the file is not called index.dat, (since we dont know how/what its for), it will load it from DUMP instead of from jagex’s server!

Run it, scream hax, because if you did it right, it worked.

Now, the fun part.

Get yourself a hex editor, a quick google comes up with XVI32 (freeware)

Open up some files, and play around!

The .txt files contain the bad words (change them if you fancy), and other things contain the big lists. (OBJ.dat, NPC.dat)

With the hex editor, if you play around a bit, and notice patterns, you’ll find yourself editing names/levels/sprites of npc’s.

Thats about it. Anything else I havn’t been bothered to try

If you fancy being quite hot, then you could make a new java program, that allows you to view (and edit) the other .dat files -they contain the graphics (buttons/menu’s etc) to your fancy

Hope that helped, you are now uber-hax0r
