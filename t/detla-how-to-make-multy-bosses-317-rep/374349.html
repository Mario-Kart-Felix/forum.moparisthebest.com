rova1 August 1, 2016, 1:42am #8

@deathscapepkz: you’re code would be incorrect because he just said they are different locations, and if you look carefully at the second one it would be impossible.

OK. For each of those co-ordinates, you must define the region in the area to be multi-combat.

For example, teleport to your first location: 2508-4720

Run to the south-west corner and type ::mypos
Run to the north-east corner and type ::mypos

You now have your lowest x,y and highest x,y which you add to the code I gave earlier. Remember if your area is wider in the middle you need to compensate for this, because this will make a perfect 4x4 square area into a multi-zone only. So in that case you would need to no-clip into the deadzone.
