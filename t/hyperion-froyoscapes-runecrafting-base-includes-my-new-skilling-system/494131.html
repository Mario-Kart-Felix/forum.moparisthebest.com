[size=18pt]Step 1:[/size]
Setting the base up.

Make a new package. I suggest it to be org.hyperion.rs2.model.skills or org.hyperion.rs2.content.skills. Inside that package, make another package. Name it either org.hyperion.rs2.model.skills.impl or org.hyperion.rs2.content.skills.impl. We’ve got the packages set up.

NOTE: This uses package org.hyperion.rs2.model.skills and package org.hyperion.rs2.model.skills.impl. If you use the other one, you’ll need to convert the packaging in the class.

[size=18pt]Step 2:[/size]
Implementing my skilling system.

In org.hyperion.rs2.model.skills or org.hyperion.rs2.content.skills, make a new java class and name it Skill.java. In it, add the following:

package org.hyperion.rs2.model.skills;

import org.hyperion.rs2.model.Player;

/**
 * @author Arham 4
 * 
 *         An interface that manages the skills in an organized manner.
 */
public interface Skill {

	/**
	 * Is the player skilling?
	 * 
	 * @return SkillConstants.isSkilling[SkillConstants.SKILLNAME];
	 */
	public boolean isSkilling();

	/**
	 * Executes the skill.
	 */
	public void execute(Player player, int objectId);

}

This is an interface. Why do we have this? So the classes look more tidy. Think of this as a pre-formatted layout of a class that implements this.

Now, after doing so, make another java class file. Name it SkillConstants. Inside it, you add:

package org.hyperion.rs2.model.skills;

/**
 * @author Arham 4
 * 
 *         The variables for skills.
 */
public final class SkillConstants {

	/**
	 * Is the player skilling?
	 */
	public final static boolean[] isSkilling = new boolean[25];

	/**
	 * Resets the <code>isSkilling</code> boolean.
	 */
	public static void resetActiveness() {
		for (int skill = 0; skill < isSkilling.length; skill++)
			isSkilling[skill] = false;
	}

}

This is just setting the skills active or not, so it’s easier to manage.

Now, make a new java class file (Last one in this step, I swear!) and name it SkillData. Inside it, add:

package org.hyperion.rs2.model.skills;

/**
 * @author Arham 4
 * 
 *         Manages all the data classes.
 */
public interface SkillData {

}

I only made this because the org.hyperion.rs2.model.skills.impl (or org.hyperion.rs2.content.skills.impl) is impl, AKA implement, and skill an interface is implement, I just followed the rule of the package by implementing this interface to all the data classes (Yes, I don’t like making all the data and stuff to do in the skill in the same class).

Critism incoming:

Skill interface:

- Why is there a boolean for isSkilling? If the skill instance is called they're obviously skilling.

- The point of a skill system is to make writing skills easier and neater, also not rewriting code, i.e from harvesting skills, where it's basically, animation - harvest - repeat.

- The interface basically does nothing, but make having execute and isSkilling necessary methods.

- For this kind of thing abstraction would be best.

SkillConstants:

- why is isSkilling boolean array static? Static means there's only ONE global state for the array, meaning if one of the values in the array is turned to true for 1 player, it will be true for every player in the server.

- Seems rather pointless really.

Clawz_fury August 4, 2016, 5:36am #14

Why are you using an interface for this… lol? And why do you have two interfaces? The design makes no sense. Make an abstract class.

The only abstract methods you need are pretty much one to reset the skill, and one for the exp rate for the skill… You can also make an execute method although that design is flawed and is evident in your system. How? It doesn’t support all skills:

For example

public void execute(Player player, int objectId);

Not all skills use an object (herblore, fletching, fishing, crafting, thieving, slayer, etc.) , and some skills may require more parameters than just an integer.

Change the execute method to a reset method, to reset the skill. You don’t need an execute method, you can simply make a static method with an instance of the player as a parameter or you can instance the class in the Player class; and call some sort of “start” method from there. You’ll notice once you start working on skills like fishing and cooking, that your ‘base’ is pretty much useless, or extremely hard to work with.

Seems like you haven’t really grasped the concept of inheritance yet, but good job arham! Keep working hard
