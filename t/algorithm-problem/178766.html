( You may need to know what Big O Notation is )

We can analyze space usage in the same way we analyze time usage. An O(1) bound would mean that the amount of memory required was always the same, regardless of n. (Or, at least, always the same once n hits some point.) An O(n) bound would mean that the amount of memory required grew linearly with n (perhaps an array of size n was used). An O(n^2) bound might imply a matrix was involved, etc..

Suppose we have an (unsorted) array of size n containing all numbers from 1 to n + 1 except one
number. Give an algorithm that finds the missing number in O(n) time and O(1) extra space. (e.g.
O(1) (constant) space once you discount the space required to store the input array.)

If no one comes up with a solution for awhile, I’ll give a big hint that will help you crack it. But I am sure someone will post the solution before then.

Varek gets the cookie!

The thing that made me realize the answer is that in order for the algorithm to complete in O(n) time you can only look at each value once, also you need to store something about each variable in something fixed length. Now you know that you need to combine all variables to a single one and use math to find the answer!

Now remember from math class that the sum of 1 to n numbers is (n(n+1))/2 (in this particular problem, the added wrinkle is that n is actually n+1, since this is all numbers from 1 to n+1) and that gives us the answer

missingNumber = ((n+1)((n+1)+1))/2 - sumOfNumbersInArray;

simplified:

missingNumber = ((n+1)(n+2))/2 - sumOfNumbersInArray;

Which is what varek came up with (remember x/2 == .5*x)

Hopefully someone learned something from that ;D

Less efficient but without using mathemathical formulas :

program prog2;

{$APPTYPE CONSOLE}

uses
  SysUtils;

const
  n = 7;
  arr : array [1..n] of Integer = (1, 2, 5, 3, 7, 4, 8);

var i : Integer;
    count : Integer;

begin

 count := n+1;
 for i := 1 to n do begin
   Inc(count, i);
   Dec(count, arr[i]);
 end;

 writeln('Missing number: ', count);
 readln;

end.
