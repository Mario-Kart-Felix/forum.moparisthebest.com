Let me tell you most of what I know about OOP, using analogies

An object is like a hairless rat with blank eyes no tail and no limbs, yet it does have options to add those ‘missing’ features

When you make an instance of an object, you give it those features. So if the master class was called Rat.java, there would be a constructor in it like

public Rat (boolean hair, int hairColor, boolean feet, int[] feetLength, boolean tail, int tailLength, int eyeColor) {
this.hair = hair;
this.hairColor = hairColor;
this.feet = feet;
this.feetLength = feetLength;
this.tail = tail;
this.tailLength = tailLength;
this.eyeColor = eyeColor;
}

And when you say ‘new Rat(true, 2, … etc’ you make a new rat that has those features that you input into the constructor.

Now when you guys say OOP. I’m thinking you mean something like this, where most everything is an object:

public Rat (boolean hair, HairColor hairColor, boolean feet, int[] feetLength, boolean tail, TailLength tailLength, EyeColor eyeColor) {
this.hair = hair;
this.hairColor = hairColor;
this.feet = feet;
this.feetLength = feetLength;
this.tail = tail;
this.tailLength = tailLength;
this.eyeColor = eyeColor;
}
