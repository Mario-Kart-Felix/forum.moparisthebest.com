Information
I’ve seen a few posts about people who would like to use the orginal RSC login screen where it fades to the 3 different areas around lumbridge. Just for fun, I found an older RSCD client that used this, and made it work in the newer clients.

I am not going to include a button colour and location tutorial.

Tutorial
First of all you will need open the mudclient class (mudclient.java) and find:

    private final void drawLoginScreen() {
        hasReceivedWelcomeBoxDetails = false;
        gameGraphics.f1Toggle = false;
        gameGraphics.method211();
        if (loginScreenNumber == 0 || loginScreenNumber == 1 || loginScreenNumber == 2 || loginScreenNumber == 3) {
            gameGraphics.drawPicture(0, 0, SPRITE_LOGO_START);
        }
        if (loginScreenNumber == 0)
            menuWelcome.drawMenu();
        if (loginScreenNumber == 1)
            menuNewUser.drawMenu();
        if (loginScreenNumber == 2)
            menuLogin.drawMenu();
        gameGraphics.drawPicture(0, windowHeight, SPRITE_MEDIA_START + 22);
        gameGraphics.drawImage(aGraphics936, 0, 0);
    }

Change that to:

    private final void drawLoginScreen() {
        hasReceivedWelcomeBoxDetails = false;
        gameGraphics.f1Toggle = false;
        gameGraphics.method211();
        if (loginScreenNumber == 0 || loginScreenNumber == 1 || loginScreenNumber == 2 || loginScreenNumber == 3) {
            int i = (loginTimer * 2) % 3072;
            if (i < 1024) {
                gameGraphics.drawPicture(0, 10, SPRITE_TEXTURE_START);
                if (i > 768)
                    gameGraphics.method232(0, 10, SPRITE_TEXTURE_START + 1, i - 768);
            } else if (i < 2048) {
                gameGraphics.drawPicture(0, 10, SPRITE_TEXTURE_START + 1);
                if (i > 1792)
                    gameGraphics.method232(0, 10, SPRITE_MEDIA_START + 10, i - 1792);
            } else {
                gameGraphics.drawPicture(0, 10, SPRITE_MEDIA_START + 10);
                if (i > 2816)
                    gameGraphics.method232(0, 10, SPRITE_TEXTURE_START, i - 2816);
            }
        }
        if (loginScreenNumber == 0)
            menuWelcome.drawMenu();
        if (loginScreenNumber == 1)
            menuNewUser.drawMenu();
        if (loginScreenNumber == 2)
            menuLogin.drawMenu();
        gameGraphics.drawPicture(0, windowHeight, SPRITE_MEDIA_START + 22);
        gameGraphics.drawImage(aGraphics936, 0, 0);
    }

    private final void resetPrivateMessageStrings() {

        gameGraphics.storeSpriteVert(SPRITE_MEDIA_START + 10, 0, 0, 512, 200);

        drawGameMenu();

Jython July 31, 2016, 6:38pm #5

that is pretty poorly refactored. for one the first 2 ‘zoom’ vars are far clip planes for polygons belonging to sprites (mobs) and models (trees, walls). the second two are related to the fog of war fall-off rate and terminal z plane. setCamera() indeed sets the camera state to be targetting a specific location with adjustable distance. GameCamera itself however does not only handle the camera but scene management in general. fadePixels() is a quick method of darkening a color, and method221() applies a blur convolution. method232() plots a transparent sprite (hence the transition being gradual). EngineHandle as i’ve said before is simply a manager of the game terrain; the ground and objects. method401() loads all terrain data about a specific region of the map (byte0, byte1 are region x/y) from maps#.jag and land#.jag, while method428() populates all internal model lists for example those of walls and rooves.
