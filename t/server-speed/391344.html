-Xmx sets the max heap size for the application.

What does that mean? The higher the better?[/quote]

No. What it does is pre-allocate the specified amount of RAM at startup, and that RAM belongs to the java process until it exits. So for example if you do “-Xmx1024” it allocates about 1 GB to the java process. It affects the speed of the application very little. I mean, there might be some small run-time optimizations (pointers relative to the start of the segment, instead of the current active memory)
