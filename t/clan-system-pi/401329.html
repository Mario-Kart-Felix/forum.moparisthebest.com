This is some what ‘cool’, and you might want to think about adding it to your Server. Yes, I wrote this guide myself and credits for code.
Basically there’s a ‘set’ owner for a certain clan that someone joins, so it’s not always ‘first person joins is owner’.

For instance;

case 0: this.name = "So the name of the clan"; this.owner = "Owner's name in here"; break;

And instead of actually declaring this.owner = under every case, you could have this public Clan(int channel) { this.owner = "Help"; switch(channel) { case 0: this.name = "So the name of the clan"; break; so all the cases would have this person for the owner of that clan.

Right, enough explaining , lets get to it!

Create a new class file src\server\model\players and call it Clan.java, then put this in it;

[code]package server.model.players;

import server.Config;

public class Clan {

public Clan(Client c, String name) {
	this.owner = c.properName;
	this.name = name;
}

public Clan(int channel) {
	this.owner = "Help";//instead of having this.owner = declared under every case, you could have this here so all the cases would have this person for the owner.
	switch(channel) {
		case 0:
		this.name = "So the name of the clan";
		this.owner = "The owner of the clan";
		break;
		
	}
}
	
public int[] members = new int [Config.MAXIMUM_CLAN_MEMBERS];
public String name;
public String owner;

} //written by james-revolutionx. You can keep on making as many cases as you want.[/code]

The code has also been explained within.
This also allows the clan still to be active whilst the player is offline.

Then go to src/server, look for a file called config.java.

Then declare these two methods;

public static final int MAXIMUM_CLAN_MEMBERS = 100; public static final int MAXIMUM_CLANS = MAX_PLAYERS;

This is so there’s a ‘maximum’ amount of clan members.

Your done!

Any errors please post them, I will keep updating this regularly (to make the system even better with more features).
