Jython July 31, 2016, 10:57am #14

Ryanz, no point; he uses HTTPS (HTTP over SSL). There is a script that takes in several hashes of different files or something and a set of auth codes. The auth codes are in the format PRODUCT_IDxAUTH_KEY. The script parses the product ID, checks if the auth key is a valid key, and if so, it prints out various information on the products. This info is used to build the first interface, which contains list of products you have authed. It then connects to another script which you pass to it a variable named SSID. This number is generated by your IP address. It looks for all products authed to that SSID, and for each one, it passes it on to the packer code. The packer code builds a ZIP file containing all product jars; it is sent from the script and downloaded by neXus, and then the product jars are saved to your temporary directory. In these jars are the classes of the products you bought.
