Sepranem July 29, 2016, 9:45pm #6

you can’t change the detail setting, and to make new monsters you go into autospawn.cfg (if you can’t open, just use textpad or wordpad, ANY text editor should work) then make a new line and put in:

spawn = 3496	2355	4958	0	2360	4963	2350	4953	1	Dessourt

now to do some editing. change the number after spawn= to the NPC ID of the monster you want (a full list is included in moparscape31\all_IDs), then change the next number (2355 in this case) to the X-coord of where it should be spawned, and the next (4958 in this case) to the Y-coord. the next number (0 in this case) is the height, unless you want it to be a floor above, or below, then DON’T mess with this (just for your information, -1 is underground, 0 is normal ground level, and 1 is a floor above). the next four numbers are the ranges of motion that you will allow your NPC to have. the first is the X coord of its first range, the next is the Y, then another set of ranges. (like this: RangeX1,RangeY1,RangeX2,RangeY2). the last number is the walkType, just a way for the npc to walk (just set it to 1). now the last thing you need to do is give a description of the NPC; just put in the name for example.

then save autospawn, compile, then runserver, and enjoy the new monsters!

Note-
if the npc’s freeze, then make sure that the HP lvl in NPC.cfg that corresponds to the monster is correct (you might have to add the monster, just copy another line and change what needs to be). if this still doesn’t work, then the npc is bugged, such as the kq.

hope it helped ya!

~Sepranem~
