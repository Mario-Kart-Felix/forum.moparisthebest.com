Fox2007 August 4, 2016, 11:33am #2

You followed my tut for the Dynamic Dialogues, there’s absolutely NO NEED to put

handledDialogues.put(“ModPanel”, (Class) Class.forName(ModPanel.class.getCanonicalName()));

that in the dialoguehandler class whatsoever. All you have to do is add the dialogue class into

src.com.rs.game.player.dialogue.impl

the dynamic dialogues automatically reads the classes from that directory and adds them into the map.

remove this from DialogueHandler:
handledDialogues.put(“ModPanel”, (Class) Class.forName(ModPanel.class.getCanonicalName()));

Next, take all your dialogues in the folder: com.rs.game.player.dialogue
and move them into the package: src.com.rs.game.player.dialogue.impl

When moving DO NOT include Dialogue and Dialogue handler, leave those in com.rs.game.player.dialogues

if you’ve already done that and your dialogues are all in src.com.rs.game.player.dialogue.impl
Just add the ModPanel class into that, and restart server. If done right it should work afterward.

Hope dis helped :L

(btw i suggest using eclipse for moving a mass amount of classes like that…auto-updates all the imports and packaging for you… :L)
