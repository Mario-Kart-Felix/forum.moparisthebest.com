Mendorphins August 1, 2016, 1:23pm #2

Basically for the shops, if you open up the .cfg file (look for it), you can then edit the numbers.

Example: shop = 1 General Store 1 1 4151 100 590 100
^

shop = 1: This is the ID. When you click an General shop owner, you will set it to call shop 1 (follow suit from already added shops, usually in ShopManager.java)

General Store: The name that is displayed in the shop interface

1 1: The first number one is the buy declaration. 1 means you can buy anything. The second one is sell, you can sell anything to the shop. If it was a number 2, you could only buy and sell what the shop already has in stock (specialist shops such as fishing or magic shops)

4151 100: 4151 is the item ID for the Abbyssal whip, 100 is the item amount. So the shop will display 100 whips.

590 100: The same as above display 100 Tinderboxes, as 590 is the Tinderbox item ID.

Experiment until you get the hang of it is the best way.

For NPC locations, check autospawn.cfg. npc = 1 3233 4084 0 means spawn a man on the coords 3233, 4084 height level 0.

When editing a .cfg file, always use the tab, if you have any stray spaces anywhere (except shop names) then you will get runtime errors!

For the crown, edit/replace the image in either the sprites folder or cache folder (need a cache editor). Keep the same name otherwise you’ll get errors.
