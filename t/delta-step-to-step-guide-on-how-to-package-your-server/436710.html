This tutorial is for all people who still use Delta (like me).
We all know it’s a complete mess. One big step in getting some organization is packaging your server.
This is what I’m going to teach you with this Step to Step Guide.

But also people who want to learn how to package, either to package more or give their PI/others a personal touch,

I included a mass load of pictures to make it very easy.[/b][/center]

Now let’s get started.
[b]To package your server you must understand,

  • how to package your files obviously
  • how to handle imports
  • how to setup your compiler(if you still prefer notepad above an IDE) and run[/b]

This all we are going to learn now.

Let’s locate to the source you want to package and open it.

Make 2 new folder in your directory. One called “bin” and one called “src”.
Your java files will come in the src folder and what you need to run your server,
(class files, ini files, cfg files,…) will be coming in your bin folder.

Now delete all your .class files since they will be useless in the current directory.

Now move all your .java files into the “src” folder

Make a new folder inside the “src” folder called “server”
You can use any name you want, and put your files as deep as your sexuality wishes.
But let’s just follow my steps for now to not get troubles later in the tutorial.

Move all your java files into that folder now

Now we have our files where we want em we will start packaging them.
You must declare the package a file is in on top of every single file in the directory.
So on top off every .java file put the package declaration “package server;”

Now we of course also can make subpackages, not everything should stay in the same package, then it’s just a regular delta.
We will move the clickingMost class into a new package.
Make a new folder called “actions”

Now move your clickingMost class into that folder and change the package declaration to server.actions;
You can make as many subpackages as you wish, but you must always declare it in the files in that package!

Now our server is packaged. Barely but I’m not packaging your server for you, I’m teaching you the basics on how to do it.

Now go back to your main directory and move all data folders and your server.ini into the bin folder.
Since our class files will be compiled to there, and the server will run via the files in the bin.

Optional if preferring to not use an IDE as compiler.
Now we must edit the compiler to be able to find the files again.
Edit your compiler like this:

Everytime you add a new subpackage you must add it in the compiler.
But ONLY if the folder contains actually .java files, else your compiler will simply not work.
So everytime you add a new package you space at the end and add a new line like this:

Now we must edit the run, since the location of your files is changed.
The server class is the main class to run the server.
If you open your current compiler you will see in the compiler it says something like:

@echo off java -Xmx526M server pause

Now since our server class moved to the server.server package we must declare that in our compiler to it would become

@echo off java -Xmx526M server.server pause

But since we went messing around with folders, and the regular compiler expects the files to be in
the same directory as the compiler itself we must locate it right just like the compiler with adding cd bin

Now we will be using eclipse to fix the errors and imports which packaging caused.
You can do this by hand but have fun hurting your fingers then.
Most info will come with the pictures now.

Run Eclipse and open a new java project

Open the folders and you’ll see classes with a red cross. These contain errors.

When you go on an error with your cursor eclipse will mostly give you some suggestions
to fix the error. Though don’t go blind on that, you still need your common sense. Eclipse is a program
it can be wrong too… a lot.

If you go on it with your cursus you’ll see it can’t find the client class,
you can follow eclipse’s suggestion and simply click “import client” in the suggestions box.

Note on above images:
so when eclipse asks you to change the visibility, it will open a new class where the variable is declared.
It will already automatically change it to public but you still must save the new opened file so the changed become … saved.

That’s about it,
If you followed this tutorial correctly and have an IQ above zero you should now be able to package your servers and
package it more.
