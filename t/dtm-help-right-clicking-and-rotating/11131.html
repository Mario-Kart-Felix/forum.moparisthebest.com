Is it possible to make the screen rotate until it finds a bitmap/DTM? If so how…
It would be easier, and less detectable if you used something like RotateBitmap(bitmap: Integer; angle: Extended):Integer;
then put it in a repeat loop, with an increasing angle for each loop. Then have it search for the Bitmap, like this:
[scar]program New;
{.include SRL\SRL.scar}
Procedure RotateTheBitmapUntilFound(Increase: Extended);
var Bitmap, Finish: Integer;
Angle: Extended;
begin
Bitmap := BitmapFromString(19, 10, ‘z78DA737343000324E046474’ +
‘04F1B91EDC2652F6638101332B8D4106323F126106F0EA92A29B7’ +
‘91F850A2C446DA89E312C19F1E28111FD87C47FF3C0E0100D4D21’ +
‘E84’);
Angle:=0;
while (Angle < 360) do
begin
RotateBitmap(Bitmap, Angle); //Rotates the Bitmap about 5 degrees each time.
if FindBitmap(Bitmap,x,y) then
begin
Finish:=Round(Angle);
writeln('Found it '+IntToStr(Finish));
break;
end; //Breaks out of the loop.
Angle:=Angle+Increase;
end;
end;

begin
RotateTheBitmapUntilFound(2.5);
end.[/scar]
P.S. I am not to familiar with Rotated Bitmaps, so that may be incorrect on the Angle.

In order to Find a Rotated DTM, all you have to do is use the function FindDTMRotated(DTM:LongInt;var x,y:LongInt;x1,y1,x2,y2:LongInt; StartAngle, EndAngle, AngleStep:Extended;var Angle:Extended):Boolean;

Whats the function for right clicking?
In SRL, just use Mouse(x,y,randx,randY,false); X, and y are the coordinates. Randx, and Randy, are the random clicking added to the coordinates. False, makes a right click.

How would i be able to make it wait until the last inventory slot is full?

In SRL, do something like this:
[scar]repeat
FindNormalRandoms;
wait(10);
until(InvFull);[/scar]
