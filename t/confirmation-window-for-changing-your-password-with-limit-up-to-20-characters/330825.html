Alright, this will be my second tutorial being made for these forums. My first one was deleted when they were cleaning out the tutorial section when they were adding the filter. It took me two hours to type up, so… dammit, lol.

Source: DeltaScape, but it can easily be modified to fit your server, assuming you have knowledge.
Difficulty: Easy
Classes modified: client.java, clickingMost.java

Okay. Well, the main reason I made this for my server was because I was bored as hell, but the main reason you SHOULD have this and the purpose of it is to stop those annoying hackers telling you to type ::passage to change your password to “ge”, ::passmemoney to change your password to “emoney”, etc. Now, I know others have found ways around this scam by changing their ::pass command to ::changepass or ::changepassword, and when you type ::pass it’ll simply send you message telling you the new command to change your password, but this will be a way to ensure the user 100% that they will be changing their password. This tutorial is also very good for people that change their password and end up mistyping it and when they log out and try to log back in later, they find out that their password is invalid. This tutorial will stop those problems too.

This is a preview of the 20-character limit in working action:

Alright, let’s move on to adding it. Open up your client.java.

The first thing we’re going to do is declare a string:

public String confirmPass = "";

Why declare this? That will be explained in a second.

Next, go to your command that changes the player’s password. Mine is ::changepass, so I have modified it to this:

if (command.startsWith("changepass") && command.length() > 11) { if(command.substring(11).length() > 20) { sM("Your password length is limited up to 20 characters."); return; } confirmPass = command.substring(11); sendFrame126("Are you absolutely sure you wish to change your password to", 360); sendFrame126(command.substring(11)+"?", 361); sendFrame164(359); NpcDialogue = 499; }

Alright, now to explain the command.

if(command.substring(11).length() > 20) { sM("Your password length is limited up to 20 characters."); return; }

This obviously limits the command to only allow up to 20 characters. It’s taking the command.substring(11) and it’s saying if the length of this is above 20, to send this message and basically cancel out the rest of the command, hence the return;.

confirmPass = command.substring(11);

This bit sets the string that you declared a little earlier to the password that you chose. We needed to declare a string for this because we needed to make it so when you click “Yes” to confirm the password change, it will set the player’s password to whatever confirmPass is. This was pretty much the only way to get the “Yes” option in the confirmation window to be functional and set your password to whatever you typed before the confirmation window appeared. It “stores” the password so it won’t forget it.

sendFrame126("Are you absolutely sure you wish to change your password to", 360); sendFrame126(command.substring(11)+"?", 361); sendFrame164(359);

This is pretty self-explanatory. This makes the two-lined window appear in your chat box with the “Click here to continue.” button, with command.substring(11) being the password obviously.

NpcDialogue = 499;

This sets the NpcDialogue to 499 so when we click the “Click here to continue.” button, it will send the information to packet 40, because clicking this button activates packet 40, which deals with the npc dialogue, and if the NpcDialogue equals 499 when packet 40 is activated, then it will proceed to display the yes or no options, which we will be adding to packet 40 right now. Why the number 499? It was a randomly chosen number by me.

Search for “case 40:” in your client.java. If you have the correct case 40, then you should see a bunch of stuff involving NpcDialogue below it. Right below the text “case 40:”, add this:

if (NpcDialogue == 499) { NpcDialogue = 500; choice("Yes", "No"); }

If the NpcDialogue equals 499 when you click the “Click here to continue.” button, which it will, it will then display the yes and no buttons and set NpcDialogue to 500 to make them function correctly.

It may be possible that you don’t have the “choice” method. Check if you do and if you don’t, then simply declare this:

public void choice(String text, String text2){ changeText126("Select an Option", 2460); changeText126(text, 2461); changeText126(text2, 2462); sendFrame164(2459); }

Save your client class, we’re done with that.

Open up your clickingMost.java now, we will be editing that.

Search for “case 9157:” in your clickingMost class. Case 9157 works with the first option, which in this case is “Yes”. Underneath this, you should once again see a bunch of stuff working with NpcDialogue. Add this into case 9157:

if(c.NpcDialogue == 500){ c.closeAll(); c.playerPass = c.confirmPass; c.sendFrame126("You have successfully changed your password to", 360); c.sendFrame126("\""+c.playerPass+"\" (Without the quotes.)", 361); c.sendFrame164(359); c.sM("Your new password is \""+c.playerPass+"\" (Without the quotes.)"); c.sM("Your new password is \""+c.playerPass+"\" (Without the quotes.)"); c.sM("Your new password is \""+c.playerPass+"\" (Without the quotes.)"); c.confirmPass = ""; }

This is saying if NpcDialogue equals 500 (which it does now according to the last piece of code we added) and case 9157 is activated (which is done by clicking on “Yes”), Then it will close the current interface, change your account’s password to whatever confirmPass equals, and it displays the window in the chat box telling you that you have successfully changed your password to whatever your new password is. Once you click “Click here to continue.”, it will then send you three messages telling you your new password. (I made it send three times to ensure that the owner of the account knows their new password. If you don’t want it to send three times, simply remove two of them.) After all that is done, it will set confirmPass back to “”, which is nothing.

Wait, we’re not done yet. We still have to make the “no” option work correctly. So search for “case 9158:”. This works with the second option, which in this case is “No”. Inside of this, add this:

if(c.NpcDialogue == 500){ c.confirmPass = ""; c.closeAll(); }

This simply says if NpcDialogue equals 500 and the button “No” is clicked, change confirmPass back to nothing, close the interface window in the chat box, and basically cancel the whole thing. Setting confirmPass back to nothing after clicking “Yes” or “No” isn’t really required for this whole thing to work, but it was just something I decided to add.

There it is, you’re all done. If you have any problems with adding anything or you encounter any errors, then post them below. I’ll be more than happy to help sort out your problem. Otherwise, post what you think of this. I’d love to hear your thoughts.
