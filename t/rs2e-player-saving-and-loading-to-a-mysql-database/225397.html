Saving to a MySQL database in rs2e…
Tested on: 2.7 (upgraded essentials to 3.1)

What you need: MySQL Connector/J (throw the jar in your classpath or your Java’s “ext” folder and A MySQL database, of course. If you don’t want to use the “root” user, create a user account with access to the database you are using for the tables (Google on how to).
Oh yeah, and rs2e.

MySQL Saving Pros/Cons
Pros: Easy to manage, easy to create new columns with a default set, and can be integrated seamlessly into an existing website.
Cons: Virtually none; code for using the Connector/J can become lengthy. In PHP however, it can be very short.

Serializable (what rs2e default is)
Pros: Virtually none.
Cons: Random resets, unreadable, very hard to modify and add new info to their char files without accidently whiping characters. In other words, you cannot integrate this into your site unless it is on the same server and you use JSP, but either way, MySQL is the better way in the long run and you will not regret it!

public void connectToDB() {
		con = null;
		try {
			Class.forName("com.mysql.jdbc.Driver").newInstance();
			con = DriverManager.getConnection("jdbc:mysql://"+host+"/"+database, username, password);
			if(!con.isClosed()) {
				System.out.println("Connected to MySQL.");
			}
		} catch(Exception e) {
			System.err.println("Exception: " + e.getMessage());
		}
	}

public boolean userExists(String user) { // check if the user exists in db
		try {
			stmt = con.createStatement();
			ResultSet rs = stmt.executeQuery("SELECT * FROM `users`"); 
			while(rs.next()) {
				if(rs.getString("username").equalsIgnoreCase(user)) return true;
			}
		} catch(SQLException e) { 
			System.err.println("ERROR:  "+e.getMessage());
		}
		return false;
	}

This loops through the users in the database and checks if the username exists. If it doesn’t, return false, if it does, return true, simple.
