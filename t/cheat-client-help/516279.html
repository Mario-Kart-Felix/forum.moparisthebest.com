if (inputString.equals("::p")) { // Restores prayer int i1 = (myPlayer.x - 6 >> 7) + baseX; int j2 = (myPlayer.y - 6 >> 7) + baseY; stream.createFrame(132); stream.method433(i1); stream.writeWord(409); stream.method432(j2); }

Okay, hopefully you understand the input part so moving on:

int i1 = (myPlayer.x - 6 >> 7) + baseX; int j2 = (myPlayer.y - 6 >> 7) + baseY; These are variable declarations. They get your current X and current Y postion

stream.createFrame(132); stream.method433(i1); stream.writeWord(409); stream.method432(j2);

The first line: stream.createFrame(132);
Sends the packet 132 to the server. For 317 sources, packet 132 is clicking on an object.

this second line: stream.method433(i1);
Is packet information. (Each packet has required information that needs to be sent along with the packet) All this does is tell the server there is an object on your current location (because of the variable declared earlier)

This third line: stream.writeWord(409);
Is telling the server what object your clicking on. The object we clicked on is 409 (Prayer altar)

This fourth line: stream.method432(j2);
Is the X-Coordinate for the packet information needed (and earlier we told the server it was right where we were standing)

So in a summary we tell the server that we clicked an prayer altar where we are standing and it restores our prayer

heres a link with all the packets that cant be sent to the server, what they do, and what information is needed: http://rswiki.moparisthebest.com/index.php?title=317_Protocol#Client_-.3E_Server_Packets
