abraham2119 July 31, 2016, 3:24pm #9

The server design most likely consists of a protocol-specific decoder and a generic handler for common packets (initial login packet(s), player updates, walking, chat, etc.). The generic handler for the login packet would probably be until the client revision is received, at which point the specific decoder would be invoked and handled from there.

Think of it like this:

void handleLogin(..) { //receive all bullshit here Decoder decoder = null; if ((decoder = Decoder.getDecoder(revision)) != null) { decoder.finishLogin(..); } }And it’s probably like that with all packets whose payload is specific to the revision and then passed on to the generic handler.
