	public void dia (final int diaId) {
		c.NpcDialogue = diaId;
		c.NpcDialogueSend = false;

	 * When clicking a certain NPC, sending the starting dialogue.

	/*
	 * Continuing a dialogue when the player clicks, "Click here to continue.".
	 */
	public void continueDialogue() {
		
		/*
		 * Storing dialogueId's in arrays.
		 * - minOne: Dialogues that should decrease one ID.
		 * - plusOne: Dialogues that should increase one ID.
		 * - finishDias: Dialogues that should be finished.
		 */
		final int[] minOne = {};
		final int[] plusOne = {};
		final int[] finishedDias = {};
		
		/*
		 * Executing the right action for the above arrays.
		 */
		for (int diaMin : minOne) {
			if (c.NpcDialogue == diaMin) {
				c.NpcDialogue --;
				c.NpcDialogueSend = false;
				return;
			}
		}
		for (int diaPlus : plusOne) {
			if (c.NpcDialogue == diaPlus) {
				c.NpcDialogue ++;
				c.NpcDialogueSend = false;
				return;
			}
		}
		for (int fin : finishedDias) {
			if (c.NpcDialogue == fin) {
				c.NpcDialogueSend = false;
				c.closeAll();
				return;
			}
		}
		
		/*
		 * Dialogues that meet other requirements than above.
		 */
		switch (c.NpcDialogue) {
		case 9999: //example
			c.NpcDialogue = 9999;
			choice("Yes", "No");
			break;
		}
	}
	
	/*
	 * Creating dialogues to be send.

	public void UpdateNPCChat() {
		c.getCS().sendFrame("", 976);
		switch (c.NpcDialogue) {

		case 9999: //example 
			talk("Test", 945);

		}

Here are the keys for making your dialogue:

public void startDialogue (int npcId) {

This is an important void. This allows the server to detect the npc (from saying case NPCID:) and makes the server resort to the next void we will mention next. You add one by doing:

		case 2705: //guard
                        dia(1);
			break;

[ul][li]The number in front of case is judged by the NPC’s ID. So for Man you’d do case 1. For npc ID’s, go to http://rspscodes.synthasite.com/npc-codes.php.[/li]
[li]In between the ()s, is the number we will use next method.[/li][/ul]

Now this is where we actually ADD the talking and everything.

public void UpdateNPCChat() {

Here is the format here:

		case 1:

There are many replacements for talk.

talk - Allows NPC to talk 1 line. - talk(“Hi”, NPCID);
talk2 - Allows NPC to talk 2 lines.
talk3 - Allows NPC to talk 3 lines.
talk4 - Allows NPC to talk 4 lines.
ptalk - Allows the player to talk 1 line. - ptalk(“Hi”);
ptalk2 - Allows the player to talk 2 lines.
infodia - Sends a 4 lined statement with no continue option. - infodia(“Noob”, “nOob”, “noOb”, “nooB”, “NOOB”); - NOOB is the title.
itemMessage - Sends a statement with an item in the side. I do not know how to use this.
choice - Sends a dialogue with 2 options. - choice(“Yes”, “No”);
options2 - Another dialogue with 2 options - options2(“Yes”, “No”);
options3 - Sends a dialogue with 3 options. - options3(“Yes”, “Maybe”, “No”);
options4 - Sends a dialogue with 4 options.
options5 - Sends a dialogue with 5 options.
fivechoices - Another dialogue with 5 options. - fivechoices(“Yes”, “Maybe”, “Kinda”, “Probably”, “No”);

So from case 1 to 5 I want the dialogue to continue and at case 6 the chat ends. Soooo, this is what I put:

		final int[] minOne = {};
		final int[] plusOne = {1, 2, 3, 4, 5};
		final int[] finishedDias = {6};

So basically, here is the translation:

minOne - Go back one dialogue.
plusOne - Forward the dialogue.
finishedDias - Finishes the dialogue.
