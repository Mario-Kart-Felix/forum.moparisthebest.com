Idk about you guys but I am tired thread after thread of the halfassed skill menus. I made my own, modeled after runescapes 2 odd years ago, but seeing all the shit released I decided I would rework all of mine. Originally it was over 17k lines of code(didnt loop or shorten in any way). I’ve worked on and off for the past week getting it down to under 2500 lines. I made it to be user friendly. You only have to 4 lines of code to client.java and some in actionbuttons.java. The rest is contained in the class.

All the sendFrame174s and sendFrame126s for each line are put in 1 method, not onl y to make it easy to read, but to make it easy to edit/add on to

menuLine(String level, String lines,  int ids, int lineCounter);[/code]
the String level is used to place the required level of the item, the line is used for the description of the item, and the ids is used for the item going on the interface beside it.
It makes it simpler than 3 different things for 1 line. The line counter is just to identify what line you are on. IE first line of interface is lineCounter 0 and each line after that is +1 to it.

[code]	private void optionTab(String title, String currentTab, String op1,
			String op2, String op3, String op4, String op5, String op6,
			String op7, String op8, String op9, String op10, String op11,
			String op12, String op13)[/code]
This one looks huge, but its really not, most of the Strings here will be "" seeing as most of the interfaces dont use all 13 buttons on the sidebar. 
The first one is the title(Name of the skill, like "Attack"). The Second one identifies the current clicked tab. While op1-13 identify the menu buttons in the order from top to bottom.

Now we get to the complexes, like the attackComplex(int screen).
Each complex method is named using the full skill name +Complex. What these do are house all the skill menus. I found it far more efficent than using 90 different methods to use 1 for each skill to make for easy lag free tab switching. The int screen, is used to tell the method which menu the player calls. Here is an example of one of the menus

For those of you who havent really messed around with frame174, here is my method for the item on interface

public void writeInterfaceItem(int id[]) { synchronized(c) { c.outStream.createFrameVarSizeWord(53); c.outStream.writeWord(8847); // 8847 c.outStream.writeWord(id.length); for (int i = 0; i < id.length; i++) { c.outStream.writeByte(1); if(id[i] > 0) { c.outStream.writeWordBigEndianA(id[i]+1); } else { c.outStream.writeWordBigEndianA(0); } } c.outStream.endFrameVarSizeWord(); c.flushOutStream(); } }
You can add it to the actual skillinterfaces class just make sure to change c.getPA().writeInterfaceItem(item); to writeInterfaceItem(item)
