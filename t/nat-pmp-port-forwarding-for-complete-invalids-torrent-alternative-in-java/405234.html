Hello, it seems quite a large portion of the user base here takes advantage of NAT-PMP within µTorrent. Instead of using a torrent application to accomplished this (and assuming you’re too fucking stupid to configure your router/don’t have access to it), why not use a Java implementation right in your server?

I’m a little surprised as how none of you have stumbled upon jNAT-PMPlib; considering it’s extremely easy to use.

Here’s an example:

import com.hoodcomputing.natpmp.*;

/**
 * Example class to explain the library.
 * @author T4_
 */
public class Server {

    /** The port to 'forward' and the lifespan to remain open*/
    private static final PORT = 43594, LIFE = 120;

    /**
     * Attempts to use NAT-PMP on your router given the PORT and LIFE.
     */
    public static void applyNATPMP() {
        NatPmpDevice pmpDevice = null;

        try {
            pmpDevice = new NatPmpDevice(true);
            ExternalAddressRequestMessage extAddr = new ExternalAddressRequestMessage(null);
            pmpDevice.enqueueMessage(extAddr);

            pmpDevice.waitUntilQueueEmpty();

            MapRequestMessage map = new MapRequestMessage(true, PORT, 0, LIFE, null);
            pmpDevice.enqueueMessage(map);
            pmpDevice.waitUntilQueueEmpty();

            int extPort = map.getExternalPort();
            System.out.println("NAT-PMP success on: " + extPort);

        } catch (Exception e) {
                e.printStackTrace();
        }
    }
}

There’s a javadoc included in the release, so you should have no trouble figuring out how to use the library (ESPECIALLY since the test class above is all you need to do).
