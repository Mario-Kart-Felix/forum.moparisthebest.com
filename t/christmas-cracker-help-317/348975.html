This probably isn’t the best way to fix your problem, but anyways…

In your client folder, open class8.java and find “public static final Class8 method198” and add this:

if(i == 339) //The item ID of an item you don't need, in this case I took cooked cod { class8.aStringArray189 = new String[5]; class8.aStringArray189[1] = "Open";//New option class8.anInt174 = 2637; //Model ID (I think this is model id for christmas cracker) class8.anInt181 = 595;//Model Zoom class8.anInt190 = 5;//Model Rotation class8.anInt198 = 4;//model rotate side ways increase to move right in circle class8.anInt204 = 138;//Unknown class8.anInt199 = -1;//noteable class8.anInt169 = 0;// model offset increase to move to the right class8.anInt194 = 0;//model offset increase to move up class8.anInt175 = 56;//Unknown class8.anInt197 = 116;//Unknown class8.aBoolean176 = false;//Stackable class8.aString170 = "Christmas Cracker";//Name of the new item class8.aByteArray178 = "I wonder whats inside... I hope it's crack.".getBytes();//examin info }
This would (if I didn’t write something wrong) mask cooked Cod with Christmas Cracker.
Now I wasnt able to find where my server handles when you click on a peice of food, but if you search around on yours you might find it and you can add something like “if itemID == 339 then do your code snippet”.

Hope this helped… and if it didn’t, you at least got a bump

EDIT: Don’t use a cooked Cod, use a strange box, since it already has the proper “Open” option
Just do as I did above and change it’s name and model and then in potions.java add this:

case 3062:
c.deleteItem(3062, c.getItemSlot(3062), 1);
c.addItem(Item.randomPHat(), 1);
break;

Hope I helped
