For those of you who still want to use this source, but with another client. I’ve determined why it wasn’t working with any other client. I recently downloaded another source, and I wanted to set it up with the port 43595, seeing that was port forwarded, and shit was already running on it like that. I changed the port within the server to 43595, and then in all of my 718 clients (in loader.java) (those that I was trying to get running with Rune-Evo’s source). When I tried to connect with those clients to the running server, it did not connect.

This made me think, well how come… I ended up switching back to port 43594 in the server, but I kept the port 43595 in the clients loader.java. Guess what happened next. With the server running on port 43594, and the client SUPPOSED to be loading on port 43595, the client allowed me to connect and log into the server, which tells me the reason the other clients were not working, was because all other clients were still connecting to port 43594, and not the server port of 43595.

In other words, you have two options to run another client w/ this server:

A. find the other variable (probably in another java file), that over rides the port in loader.java, keeping the client connected to 43594 and change it to 43595

B. Change the server port to 43594, and use any client u want :).

If your still having issues getting your client to properly connect to the server…

I use JDK1.7.0_71 with the JRE7 that gets installed with it. I make sure the server and client ports match!

JDK1.7.0_71 Download: [spoiler]http://www.mediafire.com/download/ndixw184i92hwhx/jdk-7u71-windows-x64.exe[/spoiler]

In the clients loader.java, w/o needing to edit your server ports…

public static final int SERVER_PORT = 43595;
public static final int REVISION = 718;
public static final int SUB_REVISION = 1; <-super important setting on the client, if it's not 1 this will break the client ever connecting!

Using the default installation path for the JDK1.7.0_71 and the JRE7… Never had a problem with the client crashing ever. Always make sure the pathing is directly to what it needs to connect to! Also make sure the server compiler and run batch files have the matching paths exactly like the client. I found this removes the bulk of issues in connectivity and also makes sure it’s using the correct form of java on both ends.

Client compiler path
"C:\Program Files\Java\jdk1.7.0_71\bin\javac"

Client run.bat path
"C:\Program Files\Java\jre7\bin\java"

After all that is done your client and server should connect w/o a problem. Only need to change the ip in the client for a friend to connect outside of the server network!
