Having this event running the whole time a player is logged in is dumb…

You should start the event when a player equips an item (and you check it’s ava’s accumulator) and you should end it when they don’t have it on.

[code] private static final String ACCUMULATOR_MESSAGE = “Your ava’s accumulator has binded two steel arrows to your inventory.”;

public static void startAccumulatorEffect(Client c) {
	CycleEventHandler.getSingleton().addEvent(c, new CycleEvent() {
		@Override
		public void execute(CycleEventContainer container) {
			if(c.wearId == 10499) {
				c.getItems().addItem(886, 2);
				c.sendMessage(ACCUMULATOR_MESSAGE);
			} else {
				stop();
			}
		}
		@Override
		public void stop() {
		}
	}, 500);
}[/code]
