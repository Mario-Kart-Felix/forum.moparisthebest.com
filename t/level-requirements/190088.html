LeScape July 30, 2016, 8:45pm #8

Well let’s see, it does matter.
For example, rune chainbody. You could do…

if (ItemName.startsWith("rune")) {
return 40;
}

But that would also include rune arrows, we don’t want that. So you could add an endsWith.

if (ItemName.startsWith("rune") && !(ItemName.endsWith("arrow"))) {
return 40;
}

That’s just one context. Also, startsWith and endsWith are case sensative, so you would have to change the above to:

if (ItemName.startsWith("Rune") && !(ItemName.endsWith("arrow"))) {
return 40;
}

You could also just add IgnoreCase to the end of the operator, try experimenting. If all else fails:

if (ItemName.equals("Rune chainbody")) {
return 40;
}

Although, one block of code-per-item would be kinda shitty to do.
