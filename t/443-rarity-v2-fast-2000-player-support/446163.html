Quick Big Update: This project is being converted to be a 443 server. For the latest updates, check the latest replies in the thread. It’s no longer going to be all about optimization and a fast update loop, because there are many things I can do that I don’t want to, like event-based “mask updating” that will reduce the loop time to below 10ms with 2000 players.

Welcome to the development thread of yet another server of mine. You probably remember Rarity, a server I claimed was fast and well-done, but the truth is, it had many underlying problems. Actually, Lex did, since that was what Rarity was based off. So, I decided to start a server from scratch and completely change the way the core is written (Thanks to Super for the help).

Simply put, the server uses asynchronous networking and tasks - if you don’t know what tasks are, read up on them. This allows for as little processing as possible, and the server runs great on a multi-core machine.

Another big thing this server has on Rarity is bots now properly work - they won’t get disconnected over time, and the AGF bot can actually connect to it. However, it has a couple of problems. I disabled packet encryption because of a very weird bug with packet receiving. The first packet would be okay, then the next batch would be all messed up, then they would be completely fine afterwards. I’m not sure about sending them, but I disabled both encryptions anyway. When a large number of bots are connected, say 500, only about 495 would actually be connected when it finished. I have no idea what’s going on here, but I’m not really concerned about it.

So with about 500 players or so, the performance is still very decent on my computer. All packet processing is still perfectly instant, whereas in Rarity it would be nearly impossible to process packets within a reasonable amount of time. I am happy with the performance. Here’s a screenshot (the 129ms is for the 294 players and the rest is for the 101).

So what is there for content? Well, pretty much nothing, but now that I have the core out of the way, I plan on copying everything else Rarity had and cleaning up what I’ve got and adding combat. My goal for this project is to create a fast and functional C# server ready for content to be added. Hopefully I’ll reach it!

~Lin
