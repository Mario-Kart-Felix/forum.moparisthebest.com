Recently i got some motivation to get back into rs development, and this time i decided it shall be private servers.

If you don’t know who i am, well, i was the guy who wrote the Runescape bot RSBot.

As every single server i’ve seen this far is a piece of shit (winterlove-based awful structure) me and a close friend have been working on a properly structured server (performance MUCH better than winterlove-servers) with a built-in easy-to-use script engine powered by JavaScript.

JavaScript enables us to dynamically load scripts directly on runtime, which means you don’t have to recompile or even restart the server in order to test new things!

The server is currently in a pretty early stage (login works, basic item system, walking around, framework for packet-handling done), but we will get it done pretty quickly as we work alot on it. Our goal is to replicate Runescape be as much as possible like it was back in revision #317.

Our focus is on stability, modularity and performance. This piece of work will most likely be open-source when we feel we are ready to do this. Documentation for the script-engine will be available as well.

We will be having our own server hosted on a dedicated box (estimated to support around 2000 concurrent players, just like the real runescape servers), and of course it will be 100% free to play.

A script engine is now is implemented, and a rough example script is here:

function clickTree(sender, id) { if(id == 1276) { sender.getActionSender().sendMessage("You clicked a tree, have some gold :)"); sender.getInventory().addItem(995, 1500); } }

This is probably not how the final scripting system will be, but one thing is for sure, when this is released you won’t ever have to recompile your server again as the core will call everything from script-files. (Yes, we can dynamically reload scripts during run-time by typing ::reload in the client).
