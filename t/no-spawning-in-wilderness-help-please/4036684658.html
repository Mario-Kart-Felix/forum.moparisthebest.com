Yes you could do

try {
    // stuff
}
catch (NumberFormatException nfe) {
    // more stuff
}
catch (IOException ioe) {
    // more stuff
}
catch (Exception ex) {
    // more stuff
}

But perhaps you have more than one command that can throw the exception. Let’s say, teleporting and spawning an item. Both take numbers as arguments, but if the command is entered incorrectly, it should spit out different responses. For example

try {
    // 999 commands
}
catch (NumberFormatException nfe) {
    player.sendMessage("Invalid syntax: ::tele x y[ z]");
}

If the player enters the pickup command and enters the number wrong, it will still tell them that the syntax is “::tele x y[ z]”. Where as something like this

if (command.equalsIgnoreCase("tele")) {
    if (args.length < 3 || args.length > 4) {
        return;
    }
    try {
        final int x = Integer.parseInt(args[1]);
        final int y = Integer.parseInt(args[2]);
        final int z = (args.length == 4) ? Integer.parseInt(args[3]) : 0;
        player.setLocation(x, y, z);
    }
    catch (final NumberFormatException ex) {
        player.sendMessage("Invalid syntax; use: ::tele x y[ z]");
    }
}
else if (command.equalsIgnoreCase("item")) {
    if (args.length != 3) {
        return;
    }
    try {
        final int id = Integer.parseInt(args[1]);
        final int amount = Integer.parseInt(args[2]);
        final Item item = new Item(id, amount);
        player.getInventory().add(item)
    }
    catch (final NumberFormatException ex) {
        player.sendMessage("Invalid syntax; use: ::item id amount");
    }

tells the player exactly what was wrong with the command, and is unique to each command.
