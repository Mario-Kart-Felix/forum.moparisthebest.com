My code:

[code=Cpp]#include

for (int i = 1; i <= 100; i++) {

} else if(i%3 != 0 && i%5 == 0) {
cout << “Buzz” << endl;
} else if(i%3 == 0 && i%5 == 0) {
cout << “FizzBuzz” << endl;
} else {
cout << i << endl;
}
}
return 0;
}[/code]

I was wondering if anyone could tell me any points to improve.

Taharok:

You can also optimize the conditionals by checking for multiples of 15 first, then for multiples of 3, then for multiples of 5, rather than doing two remainder calculations for each conditional. You can basically halve the number of modulus checks this way.

Solving this problem using modulus operations is totally impractical in the first place. However, before you put an “optimization” into action you should analyze the resulting algorithm - it might not be an optimization after all. You’re afraid that two modulus operations on each iteration is too much, so you tried to optimize it with the following.

for (int i = 1; i <= 100; ++i)
{
	if (i % 15 == 0)
	{

	}
	else if (i % 3 == 0)
	{

	}
	else if (i % 5 == 0)
	{

	}
}

n = 100
mul15 = 100 / 15 = 6. modulus operations: 6 * 1 = 6.
mul3 = 100 / 3 - mul15 = 27. modulus operations: 27 * 2 = 54.
mul5 = 100 / 5 - mul15 = 14. modulus operations: 14 * 3 = 42.
other = 100 - mul15 - mul3 - mul5 = 53. modulus operations: 53 * 3 = 159.
total modulus operations: 6 + 54 + 42 + 159 = 261.
261 > 200.
