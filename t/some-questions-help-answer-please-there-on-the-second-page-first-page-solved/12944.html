frank July 29, 2016, 9:48pm #20

You dont really need to know much about the mothod signature, its just that if you want to call a specific method, you have to have the same number of parameters (usually only of concern in overloading a method).

Paramters are the variables (or arguments) the the method takes.

methodName(int i)

when you call the method you need an argument of type int (because of its signature). when you call it you will use something like int aValue = methodName(3);
so, “i” gets set to 3 (within the scope of the method)
