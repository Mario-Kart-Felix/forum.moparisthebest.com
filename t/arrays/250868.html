for (int i = 0; i<names.length; i++) {
split[i][] = names[i].split(" ");
}

Also, making the second dimension of the array the same length as the list of the names seems inefficient.
You could just make it 3-4 sizes max?

A [][] is an array of an array. This the small but crucial difference between the c/c++ version and java’s.

  • c/c++/others use an array and then some math to convert the two indexes into a single index of the array it self.
  • java have an array of objects. These objects are arrays but these single dimension arrays are not of a fixed size.
    This means that the second dimension’s size can vary depending on the index in the first dimension.
