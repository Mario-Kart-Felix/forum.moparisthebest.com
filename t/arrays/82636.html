moparisthebest July 30, 2016, 7:52am #8

ArrayLists and LinkedLists each are better than the other in specific instances.

ArrayLists are faster when:

-you need random access (like list[5] list[9], because it iterates all the way up from 0 to the number you want each and every time you call it)
-you need to iterate backwards (for the same reason as above, it would iterate from 0 up to the last item, then 0 to the last number-1 each time you want an item)

LinkedLists are faster when:

-you mostly add on the end (just creates a new node and adds it, no resizing of arrays)
-adding on the front (same as above)

They should have about the same speed when:

-iterating forward
-getting the size() (of course)

(note arraylist allocates an array bigger than the amount of items it has in it, and when you add enough to make it resize, it doubles the capacity of the array, so it doesn‚Äôt really resize things that often at all.)

I probably left a few things out, basically if you aren‚Äôt sure one great thing about java is you can just use the List interface in your code, then go back and change one line later to switch between arraylist, linkedlist or any other kind of list.
