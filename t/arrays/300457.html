moparisthebest July 31, 2016, 10:47am #13

Just did some testing, weird:

public class Test{ public static void main(String[] args){ args = {"bob", "tom"}; for(String s : args) System.out.println(s); } }
Will not compile.

public class Test{ public static void main(String[] args){ args = new String[]{"bob", "tom"}; for(String s : args) System.out.println(s); } }
Will compile.

public class Test{ public static void main(String[] args){ String[] strr = new String[]{"bob", "tom"}; for(String s : strr) System.out.println(s); } }
Will compile.

public class Test{ public static void main(String[] args){ String[] strr = {"bob", "tom"}; for(String s : strr) System.out.println(s); } }
Will also compile.

So lawl is right. Still it looks like it would be best practice to use the ‘new String[]{…}’ since it works every time.

abraham2119 July 31, 2016, 10:47am #14

you cant do ‘args = {…}’ because this type of initialization for an array is called an “array initializer” (iirc), which will only work in the initialization that comes along with the declaration of the array

so ‘String[] crap = {…}’ will work but ‘String[] crap; crap = { … }’ will not
