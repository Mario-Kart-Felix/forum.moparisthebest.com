In my case, the converters that I worked to create went from RSCD_v25 to Demonik.

I’ll be a little more specific: RSCD_v4, also known as LEFTYS_RSCD_NOFORUMS (what Reines later released on Reinet, which is a more updated version of RSCD_v25) uses an almost identical database structure as Demonik. My converter scripts took RSCD_v25 and converted all the players into RSCD_v4 players. In doing so, it made each player an individual account and gave the “shared” bank under each forum “owner” to only one of the players under each owner.

You really don’t want to go in the opposite direction, the amount of work to do it is not worth the time. If you like RSCD_v25, then by all means, go the RSCD_v4 route instead. You won’t even need PunBB! (IPB forum allows external authentication if you really wanted to…)

darscape July 31, 2016, 1:58pm #24

The other problem is registration, the “name” column is generated as a series of numbers based on the number in the “owner” column and the “username” column. The server reads the “name” column when players log in and it acts as a primary key for all other tables, such as experience, curstats, bank, friends, ignores, invitems, ect. What is nice about RSCD_v4 is that it ignores the “name” column and instead reads the plaintext “username” column and uses that to connect the dots.
