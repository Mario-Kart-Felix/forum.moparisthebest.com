unlimitedorb February 16, 2019, 6:02pm #15
White Castle:

Uhh, couldn’t you do that with any bytecode library? I thought that’s why their purpose was.

Some libraries have unique aspects that can be better suited for certain tasks than others. ASM has the ability to be faster than BCEL if used correctly, but it’s a bit less flexible in some situations. It just depends on what you want to do and how you want to achieve it. Using the ASM object API rather than the core event API certainly provides you with an advantage as far as ease of use, but the nature of how class information is stored can be a real performance hit as all of is stored in memory right off the bat. Just because he provided a sample of a certain use of ASM doesn’t mean that that’s all the library is capable of doing. If anything, you should look at it as a showcase to see the syntactical differences between other bytecode libraries because you might find that it’s easier to use and a better way to do what you are already doing.
