[quote=“RandQm, post:1, topic:549194”][ul][li]Many multi threaded events for speeding up stuff and reducing cycle load.[/li][/ul][/quote]Be very careful with this. Multithreading is the easiest way to get seemingly random errors and desyncronization issues, and unless it’s properly done in a way that ensures all players see the same thing, you’re wasting your time. Proper uses of multithreading (might) include path finding, updating players in a region that can be completely cut off from another region (this is complicated but if you can pull it off, congratulations), networking, file I/O, and background player tasks, such as growing crops every once in a while. Good luck.
