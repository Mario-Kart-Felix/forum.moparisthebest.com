TestDummy July 30, 2016, 8:04am #11

Hm, I did something like this a few days ago, but I didn’t use frames.
I made a template HTML page (with some PHP tossed in) that had everything that repeated on the older pages (navigation, copyrights, etc.)
and made it so the content of every other page was included into the template page.

I put this in the body of my template page.

[code=php]<?php
$id = strtolower($_GET[‘id’]);

$page = array(
“keyword_1” => “filename”
“keyword_2” => “filename”
// etc…
);

if (array_key_exists($id, $page)) {
print file_get_contents($page[$id]);
}
else {
print file_get_contents(“error.html”);
}
?>[/code]

All the other pages that this called on only consisted of the body content that actually changed. Everything else was stripped away.

This thread isn’t the most appropriate place to ask, but is there anything inherently wrong with my design? I mean, it works when implemented, but I mean from a security standpoint.

Edit: You’d have to edit all 120 pages to make this work right initially, but afterwards, you wouldn’t need to worry about sidebar changes.

Heh, I knew I’d give a bad description. Perhaps my idea is bad.

Anyways, with my idea, if you were to implement it would be to

  • Make a template page that is like all of your others, includes the sidebar and everything you want but none of its own content
  • This template page would have the PHP inside of the tags, or wherever your content changes between pages.
  • The PHP grabs a HTML file you point it to, which should only have the body text that you want to include and nothing else, using the keywords in the array.

Can any one else explain it better?

To paraphrase, the PHP would go on one page and one page only. The pages are switched by changing the keyword. If “anime” was a keyword to your page of, um, anime, you’d have it like ‘filename.php?id=anime’. You’re loading up the same page each time, the change in keyword grabs a new file.
