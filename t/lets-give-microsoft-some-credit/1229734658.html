Windows isn’t unstable. It just has a ton of crap that IS unstable linked into the kernel. When that crap crashes, windows crashes with it. It’s because of how windows developement worked: Each new version that had new functionality that required changes to the internal workings of the system broke programs that were using said workings instead of the API because doing so was faster. So, appcompat was born. It detects these programs by name and then hacks everything around it so that it works the same way it did on the previous release of windows that this program works on. And a whole lot of those hacks are for various device drivers, which of course live in kernel space. When one of those drivers changes and the hacks no longer work correctly, boom.
