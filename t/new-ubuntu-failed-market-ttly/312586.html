Free software is supposed to improve with each new release. Take OS X, which gets faster. Cleaner. Better. Sure, they have a much smaller hardware base to work on, but it can be done. Ubuntu with the potential for thousands of developers surely can do a better job? Or at least, surely it could at least move forward??

Since when was OS X free software? And a ‘smaller hardware base’ is a bit of a stretch, try less than 10 extremely similar hardware platforms, all developed by the company developing the software. Would it support all of these different USB network adapters or graphics cards? No. It’s a horrible comparison.

In fact, the entire term “early adopter” refers to the fact that most experienced Ubuntu users upgrading to the latest version will always wait at least a month before doing so, in order to ensure most major bugs are fixed. Is this seriously acceptable? Is this what you expect from a Linux system? Surely this is some kind of morbid, ironic joke.

This is the whole purpose behind Linux and free software as a whole. The user always has a choice to stick with the old and stable, or the cutting-edge and potentially unstable. My servers are running Ubuntu 8.04 LTS and Debian Stable, I wouldn’t dream of installing the latest version of any Linux distro days after it was released, because I don’t like headaches, so I don’t ask for them.
