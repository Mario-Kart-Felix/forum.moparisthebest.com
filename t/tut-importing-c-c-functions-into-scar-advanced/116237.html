For some time it has been possible to write plugins in delphi which export functions and can be loaded by SCAR. This allows you to seriously expand the possibilities of SCAR by utilising the power of non-interpreted processing which is very useful for intensive works like image-processing. However, not everyone likes to code in Delphi, some prefer other languages like C++. Therefor I have done some research to come up with the following. In this tutorial I will explain how to export C++ functions to delphi and how to load them trough a SCAR plugin.

First of all, we need a C++ compiler. For this example I will use Borland C++ Builder 6 which is somewhat the C++ equivalent of Borland Delphi 7. It should be possible to use other compilers as well though. We are going to simply make 1 file in this tutorial with our function, to keep it easy we‚Äôre going to use a .c file. However, to make the file compile we will need to have a project file as well, unless you are using for example a commandline compiler.

First we open Borland C++ Builder 6. We choose File => New => Other‚Ä¶ . Here we click (to keep it easy), ‚ÄúLibrary‚Äù. The following should appear in the compiler:

#pragma hdrstop
#define Library

// To add a file to the library use the Project menu ‚ÄòAdd to Project‚Äô.[/code]

Now we simply save this in a subfolder of our SCAR plugin folder, to keep it structurised we name the folder after the SCAR plugin we will write later ‚ÄúCppTestPlugin‚Äù. I named the projectfile ‚ÄúCpp_Lib‚Äù, though it doesn‚Äôt matter how you name it. Now that we have our project to compile we will make a new file where our actual code will be located. We choose File => New => Other‚Ä¶ and then we select ‚ÄúC File‚Äù. Now we get an empty file, we‚Äôll save this in the same folder as the project, to keep it easy I named the file ‚ÄúMain.c‚Äù. Now we go into the Project menu and click ‚ÄúAdd to project‚Ä¶‚Äù. Then we go to our folder and select the Main.c file.

Now we have our basic C++ files set up, now we‚Äôll write a simple C++ function to export to SCAR.

The simple function I wrote increments an entered number by 10 and the returns it:

int Inc10(int Numb) { return Numb + 10; }

This we add to out Main.c file.

Now that we have our function to export we still need it to be exported, for that we add the following line above the function we just added:

That is it for the C++ part, the only thing left to do is build the results of what we‚Äôve made. For that click Build in the Project menu. Now there should appear a Main.obj file in your folder with the C++ files. That‚Äôs what we need. This file we will link in our Delphi plugin.

Now that we have written the C++ part we can write the Delphi part of the plugin. First to start off we open Delphi. There we choose File => New => Other‚Ä¶ . Then we click DLL Wizard. Just delete everything between ‚Äúlibrary Project2;‚Äù and ‚Äú{$R *.res}‚Äù so we have a clean environment to work with:

First of all we‚Äôll save the library, just save it in the Plugins folder, I named it ‚ÄúCppTestPlugin.dpr‚Äù. Now we‚Äôll add the includes that the library needs to function:

Add that under the library line. Now we have:

FastShareMem,
SysUtils;

{$R *.res}

begin
end.[/code]

Now we will link the obj file we made earlier with C++ to the delphi library, to do so we use the compiler directive ‚Äú$L‚Äù. The L stands for Link. behind that we add the path relative to the delphi project file, this we add under ‚Äú{$R *.res}‚Äù (Which is the line that includes the dll‚Äôs resources for example the version number and such):

Now we get the following:

FastShareMem,
SysUtils;

{$R *.res}

end.[/code]

Now that the obj file has been linked to our library we can import the actual C++ function we wrote. After quickly translating the function line we get: ‚Äúfunction Inc10(Numb: Integer): Integer;‚Äù however, because it‚Äôs imported from C++ we need to add _ before the function name: ‚Äúfunction _Inc10(Numb: Integer): Integer;‚Äù. Now we need to add 2 more extra keywords to make it work, cdecl and external to indicate that it‚Äôs an externally loaded function from C++. So this results in the following:

[code=pascal]library Project2;

FastShareMem,
SysUtils;

{$R *.res}
{$L CppTestPlugin\Main.obj}

function _Inc10(Numb: Integer): Integer; cdecl; external;

begin
end.[/code]

Now we have our imported function, however, we can‚Äôt directly export it, so we‚Äôll just make a small Entry function:

function Inc10(Numb: Integer): Integer; stdcall; begin Result := _Inc10(Numb); end;

This allows us to export it by adding the stdcall keyword.

To finish off we add the default stuff needed for a SCAR plugin to export a function. The FunctionCount function:

function GetFunctionCount(): Integer; stdcall; export; begin Result := 1; end;

The FunctionInfo function:

function GetFunctionInfo(x: Integer; var ProcAddr: Pointer; var ProcDef: PChar): Integer; stdcall; begin case x of 0: begin ProcAddr := @Inc10; StrPCopy(ProcDef, 'function Inc10(Numb: Integer): Integer;'); end; else x := -1; end; Result := x; end;

And then we export those:

exports GetFunctionCount; exports GetFunctionInfo;

Now our plugin is finished:

[code=pascal]library CppTestPlugin;

uses
FastShareMem,
SysUtils;

{$R *.res}
{$L CppTestPlugin\Main.obj}

function _Inc10(Numb: Integer): Integer; cdecl; external;

function Inc10(Numb: Integer): Integer; stdcall;
begin
Result := _Inc10(Numb);
end;

function GetFunctionCount(): Integer; stdcall; export;
begin
Result := 1;
end;

function GetFunctionInfo(x: Integer; var ProcAddr: Pointer; var ProcDef: PChar): Integer; stdcall;
begin
case x of
0:
begin
ProcAddr := @Inc10;
StrPCopy(ProcDef, ‚Äòfunction Inc10(Numb: Integer): Integer;‚Äô);
end;
else
x := -1;
end;
Result := x;
end;

exports GetFunctionCount;
exports GetFunctionInfo;

end.[/code]

What remains now is to build our plugin and test it, so press Build in the Project menu. Then open SCAR. You can easilly test it with the following:

Number = 5;

begin
WriteLn(IntToStr(Inc10(Number)));
end.[/code]

You will know it works if SCAR prints out a number that‚Äôs 10 more than your constant number.
