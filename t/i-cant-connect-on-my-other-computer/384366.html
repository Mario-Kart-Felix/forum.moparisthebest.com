Wintercanon August 1, 2016, 7:51am #6

Your socket listener is listening on localhost (127.0.0.1/0.0.0.0). You have to modify your server to listen on the actual IP address of your computer or whatever the host they are connecting to is. I suggest setting the listener host at runtime using an argument, that way you do not have to recompile when running it locally or publically.

Wintercanon August 1, 2016, 7:51am #10

If you’re running a server, it should be done on a server system. It is fine to test things out on your development machine and let a few people try tests, but your personal computer will be ‘chewing’ resources on the graphical user interface and other various graphical tasks. The performance will also vary depending on what you’re doing on it at a referenced time. A machine dedicated to running the server will prevent security exploits to your personal machine as well as dedicate the resources required.
