semaphore July 29, 2016, 8:59pm #15

I don’t know exactly how to answer your question. I’ll answer both questions you could have possibly meant to ask. The ‘class object’ is available as soon as the system or custom classloader loads the class in question and is available by simply accessing the class static variable. ie. a.class. The other possible meaning was instance of that class which would pose a different problem. Yes, you would need access to that instance object in order to reflect on it only if it is a nonstatic member. This means that for nonstatic members, it would be a bit tough.

However, if the member was static, piece of cake; "null"s the parameter.
