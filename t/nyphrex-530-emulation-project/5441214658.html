I guess I should talk about what I’m currently doing seeing as people are actively reading the thread. Currently I am designing a JSON-RPC2.0 platform to utilize for various services as we see needed. It’s a fairly simple protocol (very flexible), and honestly I didn’t feel like breaking Rails out over and over again. I could have probably used Sinatra, but eh I wanted to have the option to use different transfer protocols.

First goal of using this platform is to create an interface where Jarrod and I can execute GrandExchange offers. Why? Because why the hell not, the GrandExchange is an integral part of the game and what better to show off our skills by creating a website platform to merchant items from your character. If you’re good at designing algorithms to adjust suggested prices for items based upon transactions and offer information then I will grant you access to the API so you can show your stuff. There is no security layer as of yet, I’m not entirely sure we need one.

Since this is more of a ‘we want to release this to the public project’ I sadly can’t release code. But I love input on ideas, criticism (except from hoe mods that name start with the D).

The server utilizes Redis to cache non-expired JSON documents, is built in Java, and utilizes Lua for scripting (URI -> domain, command -> code stub).

On a side note I’m kinda dissapointed with the speed of Jedis for single commands in an synchronous setting. There is a pipelining feature but it really complicates caching, I was getting about 800 commands/500 ms? I might just implement an in memory caching system but I wanted to have easy access to runtime-analytics. CBF to append a core library function. Until I swamp the server, I mean I don’t think theres an issue with the current caching in place.

Documentation about JSON-RPC over HTTP: http://www.jsonrpc.org/historical/json-rpc-over-http.html
