This isn’t even close to what this design accomplishes. When transmitting frames they first get passed through their ‘normalizer’ codec which normalizes the streams, deobfuscating them and sorting them by their length.

Taking your example:

int id; int data; switch(option) { ONE: id = getByteA; data = getMEInt; break; TWO: id = getInt; data = getNegByte; break; // etc... } return new RandomThingMessage(id, data);

Code like this will never been seen, say you have those two frames, normalized they would both have the same structure, when handling them there wouldn’t be a need to read different obfuscated streams for each option in this case, they would be the same. This reduces a lot of the verbosity that tags along with obfuscated frames.
