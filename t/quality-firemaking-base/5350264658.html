thomas, I really don’t think he gets what you are trying to say…

strucker, when working with incoming packets (dropping an item, wearing an item, using an item on an item, item on an object, walking, etc) YOU NEVER TRUST THE DATA BEING SENT

int objectId = in.readByte(...); int itemUsedOnObject = in.readShort(...); int slot = in.readInt(...);

you see the data above? DO NOT TRUST ANY OF IT

and do you know why? because people can use things like cheat engine and wpe to inject false packets. so say you didn’t have any sort of check and we had this code.

[code=java]// … code I showed above here

if(itemUsedOnObject == 4151 && objectId == 1) {
// double the players money or some cool shit
}[/code]

now if someone injects a fake packet… they can double their money. why? because there was no check to see if the player even had the item!!

adding this check is referred to as packet validation (I think), and it’s as simple as just checking if the player has the item before doing the action…
