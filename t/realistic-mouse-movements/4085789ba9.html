[quote=“slavemaster, post:3, topic:408578”]How did his original algorithm work? What about WindMouse? The physics-based one used in Kbot and Powerbot?

What we know:

[ul][li]Jagex can mark your account as “suspicious” (flagged in Aryan days) – /usually/ triggered by reports, according to the Player Moderator forum leak[/li]
[li]Jagex can activate client-side mouse tracking send it to the server; logged data includes local mouse position and time… probably activated for “suspicious” accounts[/li][/ul]

Given enough tracked mouse data, Jagex can accurately represent the progression of your mouse movement and model it mathematically. This model can likely be mapped to known “profiles”, if you will, of spline-based algorithms like Benland’s, using some statistical classification. Fool-proof accurate? Not quite, but potentially valuable information. Does Jagex do this? Maybe.

I think byte brought an article detailing something similar in the past… some sort of classifier (naive Bayesian maybe?) backed by biometric training samples, however I think it’s impractical to track every player to construct “normal” profiles for each. Plus, I doubt it would be too accurate either, considering automated tracked mouse input that went by unnoticed would taint the sanctity of the normal profile, and given enough of such pollution, the profile would fail to accurately distinguish the authentic player movement from typical movement.

Mouse movement algorithms that learn are a lightly-touched topic on the board. Enforcing each individual bot user to keep their own profile of mouse data, fed to a machine learning algorithm backing a classifier (back-propagation on a multi-layer perceptron ftw? ) could potentially add a bit of “uniquity” (it’s in Webster’s Unabridged in 1913, it’s a real word! I swear :P) to the bot and give it a more organic, /human/ feel. Anybody try such an idea out? I think Robotzindisguise uses this method…

Many algorithms used in the scene today use a set tolerance and a bit of psuedo-randomness to help make the movement seem less “controlled”. One problem I’ve seen however is that humans aren’t /quite/ like this; in-fact, a model using a 2D probability distribution with appropriate decay rate, with highest “intensity” around the ideal destination point, seems more accurate here.

And on the topic of variable speed… I can go on and on. I think I’ll quit speaking for a little while. :o[/quote]
