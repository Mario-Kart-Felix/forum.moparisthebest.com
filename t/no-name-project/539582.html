don’t use Graham’s task scheduler, just make your own so you can make it as flexible as you want and completely adaptable to your server

also iirc graham’s task scheduler is completely single threaded which means that it can’t handle more than one task simultaneously which can cause something like

-submit task-
Thread-1: carried out task
-submit task-
Thread-1: carried out task
-submit big task-
Thread-1: carrying out task
-submit task-

if that thread is still carrying out the task, then the last task we submitted will have to wait until the thread is done because we only have one thread handling tasks

this causes a delay depending on the size of the task that had to be carried out (usually small) which means: inaccurate timing

you COULD just use graham’s task scheduler and make it run on a scheduled thread pool instead (you would only have to change one line)

but don’t take anything I say too seriously because I’m not an expert on concurrency and that whole explanation could be wrong
