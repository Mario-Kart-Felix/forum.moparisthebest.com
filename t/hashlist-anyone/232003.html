Since my time of learning how to use the java.util.List Collection utility, I have mainly used ArrayList. ArrayList is one of the fastest List collections and is supposed to represent a mutable generic object array. Well, I recently looked through the code of ArrayList, and I didn’t like what I saw. I thought they did a very horrible way of handling and finding objects inside their code, so I set out to design my own List implementation whose processing speeds could compete with that of ArrayList’s.

I started this list a few days ago, and finished only recently (yesterday or two days ago). I have tested it up against ArrayList, and it has surpassed the speeds tremendously. I could not believe my eyes when I saw the test results, so I went out and checked my code over and over again. Usually when something like this happens unexpectedly, there is usually a failing bug in the background. I tested my list so thoroughly that I found no bugs whatsoever.

Now, getting past the introduction. This list, as its name and the title of this topic imply, is built upon a hash framework. The list gains the hash’d value of an object being entered into the array, and then stored it perfectly mapped against its respective object. This way, when searching the list for an object, it searches against the hash value of the object instead of the object itself. This was the original idea of the list, and the idea which was successfully implemented inside this new list construction.

I built a small test class (source code can be seen at the bottom of this post) that tested my hash list against an array list. The results were spectacular. I tested my list implementation against array list with 100 objects, 1000 objects, 10,000 objects, and 100,000 objects. The types of objects I tested were both doubles and Strings. Strings are a lot slower for my list, because their hashcode() method is much more expensive. Anyway, the following are the results for each list, for each number of objects:

The following is the code I used to test the performance of HashList against ArrayList:

import java.util.*;

public class Test {

	private static int len = 100;

	public static final void main(String[] args) {
		ArrayList<String> arrayListStr = new ArrayList<String>();
		ArrayList<Double> arrayListDbl = new ArrayList<Double>();

		HashList<String> hashListStr = new HashList<String>();
		HashList<Double> hashListDbl = new HashList<Double>();

		long l1 = timeListStringFill(arrayListStr);
		long l2 = timeListStringGet(arrayListStr);
		long l3 = timeListStringIndexFind(arrayListStr);
		long l4 = timeListClear(arrayListStr);
		System.out.println("Processing ArrayList with "+len+" Strings: filled "+
				"in "+l1+"ms, iterated in "+l2+"ms, indexed find in "+l3+"ms, "+
				"cleared list in "+l4+"ms");
		
		arrayListStr = null;
		System.gc();

		long l5 = timeListDoubleFill(arrayListDbl);
		long l6 = timeListDoubleGet(arrayListDbl);
		long l7 = timeListDoubleIndexFind(arrayListDbl);
		long l8 = timeListClear(arrayListDbl);
		System.out.println("Processing ArrayList with "+len+" Doubles: filled "+
				"in "+l5+"ms, iterated in "+l6+"ms, indexed find in "+l7+"ms, "+
				"cleared list in "+l8+"ms");
		
		arrayListDbl = null;
		System.gc();
		
		long h1 = timeListStringFill(hashListStr);
		long h2 = timeListStringGet(hashListStr);
		long h3 = timeListStringIndexFind(hashListStr);
		long h4 = timeListClear(hashListStr);
		System.out.println("Processing HashList with "+len+" Strings: filled "+
				"in "+h1+"ms, iterated in "+h2+"ms, indexed find in "+h3+"ms, "+
				"cleared list in "+h4+"ms");
		
		hashListStr = null;
		System.gc();
		
		long h5 = timeListDoubleFill(hashListDbl);
		long h6 = timeListDoubleGet(hashListDbl);
		long h7 = timeListDoubleIndexFind(hashListDbl);
		long h8 = timeListClear(hashListDbl);
		System.out.println("Processing HashList with "+len+" Doubles: filled "+
				"in "+h5+"ms, iterated in "+h6+"ms, indexed find in "+h7+"ms, "+
				"cleared list in "+h8+"ms");
	}

	private static final long timeListStringFill(List<String> list) {
		long l = System.currentTimeMillis();
		for (int i = 0; i < len; i++)
			list.add(i + "");
		return System.currentTimeMillis() - l;
	}

	private static final long timeListDoubleFill(List<Double> list) {
		long l = System.currentTimeMillis();
		for (int i = 0; i < len; i++)
			list.add((double)i);
		return System.currentTimeMillis() - l;
	}

	private static final long timeListClear(List<?> list) {
		long l = System.currentTimeMillis();
		list.clear();
		return System.currentTimeMillis() - l;
	}

	private static final long timeListStringIndexFind(List<String> list) {
		long l = System.currentTimeMillis();
		for (int i = 0; i < len; i++)
			list.indexOf(i + "");
		return System.currentTimeMillis() - l;
	}

	private static final long timeListDoubleIndexFind(List<Double> list) {
		long l = System.currentTimeMillis();
		for (int i = 0; i < len; i++)
			list.indexOf((double)i);
		return System.currentTimeMillis() - l;
	}

	private static final long timeListStringGet(List<String> list) {
		long l = System.currentTimeMillis();
		for (String s : list);
		return System.currentTimeMillis() - l;
	}

	private static final long timeListDoubleGet(List<Double> list) {
		long l = System.currentTimeMillis();
		for (double d : list);
		return System.currentTimeMillis() - l;
	}
}
