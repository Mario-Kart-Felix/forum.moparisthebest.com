You can’t simply modify the RSA key since the public key, stored within the client, was made in accordance with the private key, stored within the server. If you change it, the server will not be able to decrypt the login packet and will disconnect you and most likely ban you. In fact, the proxy bot doesn’t even have to involve itself with the RSA encryption since the point of using a proxy bot is letting the client handle things like the login and simply injecting packets at will. The only thing that is probably needed to be modified within the client is replacing the dynamically and randomly generated cipher seeds with static ones that the proxy bot is acquainted with. Alternatively, you can even skip this step by changing the location of the bootstrap classpath via something like the ‘-Xbootclasspath’ argument, in which you can modify the ‘java.lang.Math.random’ function to pass the value returned from the call to the proxy bot.
