Basically, upon login, the client encrypts the last packet in the login handshake which contains a lot of important data, including player credentials and most importantly, the cipher seeds. These values are used to seed the two separate instances of the ISAAC RNG, one for incoming data and the other for outgoing data. Without these, it is not possible to either properly decode each packet or properly encode them. The problem lies within the fact that this specific login packet is encrypted using an RSA encryption, and cannot be decrypted without the ‘private-key’, known only by the server. The only crucial information you need from this packet are the seeds, so the few options you have are to either:

abraham2119:

The only thing that is probably needed to be modified within the client is replacing the dynamically and randomly generated cipher seeds with static ones that the proxy bot is acquainted with.

or

abraham2119:

Alternatively, you can even skip this step by changing the location of the bootstrap classpath via something like the ‘-Xbootclasspath’ argument, in which you can modify the ‘java.lang.Math.random’ function to pass the value returned from the call to the proxy bot.

or

Moparisthebest:

edit the client to change the RSA keys and get the ones from the client to use in your bot.

The only option that doesn’t involve modifying the actual client is the second one, which is the one I recommend you stick with if you really do not want to modify it.

Regardless, you seem to lack basic knowledge of the client and of its protocol, which means that developing a packet-based bot seems to not be an easy task for you to accomplish. If you are really so keen in developing a bot, just learn to use a bytecode-modification library and stick with the conventional bytecode-based bots.
