[tt]int AttackingOn[/tt] is a primitive, which basically means it is “just a number”. It has no methods. It is a space in memory of type [tt]int[/tt] with contents [tt]X[/tt].

[tt]Integer AttackingOn = new Integer()[/tt] is an object, which may or may not have the substring method you called.

The solution is to have a method [tt]public int getStarterType() { return starterType; //wrapper method to get starterType integer from player data }[/tt] in the player object/class.

Why a wrapper method? Best OO practices (protect the integer from being changed… allow the integer to remain [tt]private[/tt]… allow easy modifications in the future… etc). You could call [tt]starterType[/tt] directly (if it’s [tt]public[/tt]) by [tt]player.starterType[/tt] if you wanted…

[tt]if(player.getStarterType() == …) { …[/tt]

AttackingOn is an integer, I assume it’s an index. You’ll have to get the player at that index in the array where the clients are stored. And relating to wrapping the int in an Integer, no amount of wrapping will give your integer a team ID.

It’ll be something like server.playerHandler.players[AttackingOn].starter to get the other player’s starter information. That probably won’t work, change server.playerHandler.players to the array that keeps your players on your server.[br][br][size=1]Posted on: July 23, 2009, 10:10:03 pm[/size][hr]But anyway, this’ll be what the code you need will look like:

if (starter == PlayerHandler.players[AttackingOn].starter) { // stop attacking however you do it on your server }

This is easier to use and more extensible than checking for every possible starter ID if it equals the opponent’s. This checks instead to see if you have the same ID.
