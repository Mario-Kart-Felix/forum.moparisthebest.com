UTT (Universal tick time)

sinisoul August 4, 2016, 3:22pm #1

I was thinking of how possibly Jagex may store time relative to say a universal time. I know for a fact that their earlier versions of RS did not store a unix timestamp of when your farming patches may have been completed. So, I came up with this possible solution:

Take time X, this time is constant. Say July 8th, 2015 0:00:00 +0, subtract the amount of time from that point and divide by the time between ticks and you get a format which is the amount of elapsed ticks from that time that you could store in a 32 bit value for up a span of about 30 years of elapsed time. Which of course is much longer than the expected lifetime of how long the game would be available to people before having to change this format.

Would be useful for farming, couple other features. Thoughts?

shappeh August 4, 2016, 3:22pm #4

Yeah I can’t think of a better option that would allow you to store it server-side without losing time if the server went down.

You could have it store the time the server was previously started as the UTT instead of something like July 8th, 2015 0:00:00 +0. Then when the server starts again (for whatever reason it stopped), it checks the current time versus the previous start time and updates the time remaining when a related event triggers it.

The server sends the remaining time, measured in X, to the client before an event has completed. So 0 means that the event has completed. You could even have negative values for expired time.

Some benefits of this:
If the server crashes it can still calculate the time elapsed.
The time allowed is no longer limited so long as the server restarts every thirty years like you said.
Support for expired time amounts.
