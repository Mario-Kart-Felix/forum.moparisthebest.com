I started a chess game project roughly two weeks ago. My purpose was to be able to say “I created a game”, since all my previous attempts have been epic failures. The reason I chose chess was because I became recently attached to the game ever since Jagex released funorb and I could play against other people. So, I started this Java Chess Game project. It started well, but steadily grew more complicated and confusing. The two people I relied on to help me with bugs couldn’t read the code after the first week, leaving only me to fend for myself. I got through all the bugs that came at me except for a few painting issues and the final check and checkmate algorithms.

After two weeks of tedious hard-work on this game, the repeated failure of the check and checkmate algorithms took away all my inspiration to continue. So, I am releasing my chess game so that those who are willing to spend the time to understand the code, may learn something. I am sure I will be criticized for a few things with the code, but please consider that this entire game was hardcoded from the beginning.

The final version of chess is attached to this post.

Note: I believe only two methods were documented. I am not much of a documenter, therefore I didn’t document much. I only documented the methods that took the most thinking so that I could map them out in an organized way before coding them.
