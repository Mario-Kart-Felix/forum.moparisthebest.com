[anchor=clickme]I. Why this is better[/anchor]

[ul][li]Makes adding commands much easier[/li]
[li]A lot faster, because this utilizes a hashmap rather than a bunch of if statements[/li]
[li]Has support for non-existent commands (will send you an message: “The command doesn’t exist.”)[/li]
[li]Has support for commands which you are not a high enough rank to use (will send you an message: “You do not have a high enough rank to use this command.”)[/li]
[li]Supports commands with arguments and commands that don’t require arguments, and nothing has to be changed[/li][/ul]

Was tired of seeing a bunch of if-statements for 5000 commands that will probably won’t be used, and was tired of instancing local Strings when a simple array could have been used.

[anchor=clickme2]II. Adding the functions[/anchor]

We need to add the functions for the command. To put it in simpler (but incorrect) terms, we essentially need to have a method (void) as a parameter. But in Java this isn’t possible for obvious reasons. Therefore, we can use an even better alternative: an Interface.

[code=java]package server.rs2.players.commands;

import server.rs2.players.Player;

/**

  • An interface for every command

  • @author lare96
    */
    public interface Command {

    /**

    • The command execution
      */
      public void execute(Player player, String[] cmd);

    /**

    • You have to have at least this rank to use the command
      */
      public int rights();
      }[/code]

This interface basically handles all of the commands functions, as you can see (the rights, the command itself, and the substrings handled in an array).

[anchor=clickme4]IV. Changing the commands packet[/anchor]
Go into your Commands.java or whatever you have and make it look something like this

[code=java]package server.rs2.players.packets;

import server.net.incoming.PacketType;
import server.rs2.players.Player;
import server.rs2.players.commands.LoadCommand;

/**

  • Commands
    */
    public class Commands implements PacketType {

    @Override
    public void processPacket(Player c, int packetType, int packetSize) {

     /**
  * The whole string
  */
 String playerCommand = c.getInStream().readString();
 
 /**
  * The command substrings, handled in an array
  */
 String[] cmd = playerCommand.split(" ");

 /**
  * Checks if the map contains the command you typed
  */
 if (LoadCommand.commands.containsKey(cmd[0])) {
     
     /**
      * If it does, it checks if you are a high enough rank to use the command
      */
     if (LoadCommand.commands.get(cmd[0]).rights() <= c.playerRights) {
         
         /**
          * If you are, it executes the command
          */
         LoadCommand.commands.get(cmd[0]).execute(c, cmd);
         
         /**
          * If you aren't high enough, it sends you this
          */
     } else {
         c.sendMessage("You do not have a high enough rank to use this command!");
     }
     /**
      * If the command doesn't exist, it sends you this
      */
 } else {
     c.sendMessage("The command ::" + cmd[0] + " doesn't exist!");
 }

    }
    }
    [/code]

What that does, is it basically checks if the command was loaded into the map, and if it was it proceeds to check if you are a high enough rank. If you are, it executes the command.

[anchor=clickme5]V. Using our new system[/anchor]

You can either add a command by creating a new class, or you can do it in the LoadCommand file. I’ll be doing both ways. No way has any performance benefits over the other, its the same thing.

So lets add two commands, ::message (playerName) (message) and ::test. For the first command, lets use the class. I don’t use this method just because it takes longer and I’m kind of lazy. But, for the sake of this tutorial lets do it. Create a new class and name it whatever. Make it implement the Command interface. It should look like this:

[code=java]public class MessageCommand implements Command {

@Override
public void execute(Player player, String[] cmd) {

}

@Override
public int rights() {
    return 0;
}

}[/code]

Now we’re gonna make the command send the message to the selected player, and we’re going to make it so only owners can use it:

[code=java]package server.rs2.players.commands.impl;

import server.rs2.players.Player;
import server.rs2.players.PlayerHandler;
import server.rs2.players.commands.Command;

public class MessageCommand implements Command {

@Override
public void execute(Player player, String[] cmd) {

    /**
     * Gets the first substring! since the command is ::message (playerName)
     * (message), the substring is 1!
     */
    String playerName = cmd[1];

    /**
     * Gets the second substring! since the command is ::message
     * (playerName) (message), the substring is 2!
     */
    String message = cmd[2];

    /**
     * Loop through all players...
     */
    for (Player p : PlayerHandler.players) {

        /**
         * Disregard blank slots
         */
        if (p == null) {
            continue;
        }

        /**
         * Get the player and send them the message
         */
        if (p.playerName.replaceAll(" ", "_").equalsIgnoreCase(playerName)) {
            p.sendMessage("Msg from " + player.playerName + ": " + message);
        }
    }
}

/**
 * We do return 3 because 3 is the owner rank on most servers!
 */
@Override
public int rights() {
    return 3;
}

}[/code]

Now to load the command on startup, go into LoadCommand.java and add this anywhere in load().

Instead of MessageCommand, put whatever you named the class. And thats it! you can do ::message name_here hi! and if a player named “name here” is online they will get the message “hi”.

Now for the way that I do it, open up LoadCommand.java and somewhere under load() add:

        }
    });[/code]
