Enums should be used to represent fixed data types (ie. Planets, colors, etc). Enums are constant structures and should not be changed after creation. Therefore, all pertinent data should be supplied in the constructor, and all data you want visible should have an accessor method. (Note that it is not necessary, but is accepted as the coding standard)

As Davidi2 will admit, its quite easy to abuse them. Take for example a class that deals with orientation. One could say that there are only 4 orientations, and could use an enum like so

public enum Orientation {
    NORTH,
    SOUTH,
    EAST,
    WEST;
}
