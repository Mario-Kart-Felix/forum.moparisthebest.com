Specifications;

[code]Cin Virtual Machine

  • 16 bit words
  • Standard signedness will be used for signed types. (Most significant bit shall be used to indicate the state)
  • 1 million words of RAM.
  • Internal registers; R (Residu/Excess), O (Overflow), U (Underflow), PC (Program Counter), SP (Stack Pointer), E (Error)
  • Public acces registers; X, Y, Z
  • Hardware related registers will be added in a later version.
  • Alot of the original Assembly registers are left out. Cin will try to function without them, thus allowing
    more flexible ways around common problems.

Instructions are defined by 1 single 16-bit word.
This word is split up in either 3 or 2 words depending on the opcode.
The opcode is the highest 5 bits of the word.
OOOOOOxxxxxxxxxx
The remaining x bits are either 1 word as a whole or split up in 2 words
OOOOOOaaaaabbbbb

INSTRUCTION SET FOR 3 WORDS; O, a and b
0 SET -> set a to b
1 ADD -> set a to a+b
2 SUB -> set a to a-b
3 MUL -> set a to a*b
4 DIV -> set a to a/b, set R to the remainder of the division
LOGICAL OPERATIONS
5 IFE -> run next operation only if a==b
6 IFN -> run next operation only if a!=b
7 IFG -> run next operation only if a>b
8 IFL -> run next operation only if a<b
9 IGE -> run next operation only if a>=b
10 ILE -> run next operation only if a<=b
BITWISE OPERATIONS
11 AND -> Bitwise ‘and’.
12 BOR -> Bitwise ‘or’.
13 XOR -> Bitwise inverted or.
14 BSR -> Shift a to the right by b amount of bits. Any bits thrown off the word are stored in R.
15 BSL -> Shift a to the left by b amount of bits. Any bits thrown off the word are stored in R.

INSTRUCTION SET FOR 2 WORDS; O and a
0 JMP -> Jump to a, push PC to the stack
1 POP -> Reads the value on top of the stack
2 PSH -> Pushes a to the stack.
3 REM -> Removes the value at [SP] on the stack. Usefull for removing data on the stack that relates
to a closed program.

SYNTAX
SET [0] |1| -> sets the memory cell 0 to the value 1.
SET PC |1| -> sets the PC register to 1.
SET PC [1] -> sets PC to the value stored in the memory cell 1.
SET [PC] |1| -> exactly the same thing, there’s no need to define the value of a register.

[i] -> value at memory cell i;
|i| -> the literal value i;
i -> will thrown an error and halt the system.
You can't define a value simply by it's identifier. This is only possible for registers.

For now, only the SET instruction can asign a literal value to a memory cell.
In order to use the other operations you asign 2 values to 2 different memory cells and then
use those 2 memory cells in the operation you want.
For example;
SET [0] |1|
SET [1] |3|
ADD [1] [0]
[1] is now 4.

FLOW CONTROL
The generic labels are used throughout the programs.
Use JMP to jump to that context.
Using JMP also pushes the current PC to the stack.
This offers for an easy return to the original context;
SET PC POP
or
JMP POP
The thing with that last example is that it will push the PC to the stack once more.
Hence why when returning from a label to the original context, we will use the first example.
A label redirects to the following operation. This allows labels to be represented as numbers
and as such, they can be added/removed from the stack.
[/code]

Err, I think you might want to get a book or something on basic hardware to help you around.

Anyway, when you calculate something in assembly languages, a number of things will happen:

[ul][li]The result of the addition will be stored in register A[/li]
[li]The flags register will be set according, the [abbr=Zero flag]Z flag[/abbr] will be set if the result of the calculation is 0, the [abbr=Carry Flag]C flag[/abbr] will be set if the result of the calculation is bigger than what the hardware supports, the [abbr=Sign flag]S flag[/abbr] will be set if the [abbr=Most Significant Bit]MSB[/abbr] is equal to 1[/li][/ul]

In most computer architectures there are jump instructions for each of the conditions that can change in the flags register, this is done for overflow detection, determining whether a calculation is negative etc etc
