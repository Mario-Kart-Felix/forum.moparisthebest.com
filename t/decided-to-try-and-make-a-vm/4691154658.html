Bowser jr:

Cres: I think you have misunderstood the use of a stack.

I know what it is, I just can’t figure out a way to implement it.
A stack is basically a heap of memory that consists of data that is being used by functions in active programs.
When you ‘push’ a value onto the stack, the SP increases or decreases (depending on how you build your stack) by 1. The SP will constantly point to the latest value ‘pushed’ onto the stack.
When you read from a stack or ‘pop’ a value from the stack, you retrieve the value the SP is pointing and then increase/decrease the SP by 1 (opposite of the previous).

I just can’t make up if the stack is an actual part of the memory or a seperate block of memory. I also don’t know if there’s just 1 stack or if there’s a stack for each individual program/function.
If a stack is an actual part of the memory; does it start at the bottom or at the top?
That’s my main concern but for the rest, I’m pretty sure of what I’m doing xD

Also, about the registers, I won’t follow Assembly standards. As I stated in the OP; the E register will handle any errors. Errors can be anything that could interrupt the whole system such as an overflow, underflow, unknown instruction, inavailable index, etc…
The language’s behaviour is still open for debate; an instruction asigning a value to E internally could either be ignored or cause the whole program to halt and prompt out what went or could have went wrong.

In general lines, my wannabe language will try to copy Assembly’s behaviour allbeit in a slightly different way.

About registers and overflows and other types of errors;
The instruction will be simulated aforehand and only in a number of situation will E be set to a value.

  • Overflow - a bigger value thn what the system supports.
  • Underflow - Since I’ll be using unsigned values, any instruction that asign a value under 0 will set E to this errorcode.
    When the simulation is done; E will be set (or not set at all if the instruction is valid) to the proper errorcode.
    After the simulation, the instruction will be executed. If E is set to any value besides 0 (0 will stand for no errors at all) then the instruction will continue.
    If E is set to any other value, the instruction will either be ignored or the whole program will halt. (this depends on the nature of the instruction).
