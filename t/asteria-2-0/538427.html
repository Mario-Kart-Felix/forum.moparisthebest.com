[B][U]Introduction[/U][/B]
Asteria 2.0 is a runescape emulator built off of my previous release Asteria. Asteria as a whole has been in development for almost a year now, and I’ve learned so much since starting the project!

[B][U]What’s changed?[/U][/B]
Well, to put it simply: everything. All of the problems that the original Asteria faced have been solved with much research, time, and dedication! Almost everything has been rewritten, and I took the time out to write up the major differences between this version and the last.

[U]Design & Complexity[/U]
The old overall design didn’t really have much thought put into it, so I rewrote a bunch of things and actually planned what I was going to do this time. As a result this version looks completely different from the first one, and is a lot more complex.

[U]Content[/U]
This release contains full combat, and due to many requests I stopped development on skills and removed them because people wanted this to have a ‘blank project insanity’ feel in terms of content.

[U]Networking[/U]
The biggest mistake I made with Asteria 2.0 was removing the cycle based reactor design that came with Runesource. The networking turned into a complete mess, and actually worsened the performance of Asteria. I brought back the original reactor design with a twist: accept events are handled on another thread, its completely safe, and faster.

[U]Task/Events/Workers[/U]
Yeah, no more thread pool… simply because carrying out general game logic like that in parallel is not safe at all. A completely new cycle based task system has been designed to take its place.

[U]‘Idle’ States[/U]
If any threads but the main game thread haven’t been recieving any activity for a certain amount of time they’ll automatically terminate themselves and wait for new work without eating up resources. This allows more resources to be dedicated to the threads that do currently have work to carry out.

[B][U]I found a bug with Asteria I can’t fix! Can you help me?[/U][/B]
If you can help everyone else out using Asteria. So how can you help everyone out and get help with your issue? If you find any errors or bugs present in the release currently on github then [URL=https://github.com/lare96/asteria-2.0/issues/new]click here[/URL] and make an issue and I’ll fix it as soon as possible, then you can then redownload it or manually add the fix by yourself.

[B][U]Will there be a 3.0?[/U][/B]
In theory yes, because I will continue to update the git when necessary (if you guys report bugs or I find bugs later on) but in all honesty I’m most likely not going to do an ‘official’ 3.0 release because I want to take this, jack it up with content, and actually host it. Anyone is welcome to take this and continue the development as long as you give the appropriate credits.
