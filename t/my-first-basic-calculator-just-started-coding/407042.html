[quote=“dain bramage, post:3, topic:407042”]Goodjob, I guess. I always use this, though (rather than a scanner):

public static String getUserInput() throws Exception { BufferedReader buf = new BufferedReader(new InputStreamReader(System.in)); return buf.readLine(); }[/quote]

But that method, though perfectly valid, is terrible because every time that method is called (which probably will be pretty often) it will create new BufferReader and InputStreamReader instances. They only should be instaniated once!

@op: I understand this is probably one of your first programs, but you should really try a more function orientated approach instead of a procedural one (object orientatated would be a bit over the top for this program). Continue to learn about methods and it’ll make more sense.
