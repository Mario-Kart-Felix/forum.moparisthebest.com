Anyways back to the simple bug fix.

Go into Buttons.java

Find case 153:

It should look like this:

case 153: if (c.playerEnergy > 0) { c.isRunning2 = true; } break;

The problem is playerEnergy does not exhist.
So go ahead and either delete the “if (c.playerEnergy > 0) {”
or comment it out to look like this if you want to add playerEnergy yourself…

case 153: //if (c.playerEnergy > 0) { c.isRunning2 = true; //} break;

All in all just a slight mistake in the coding can drastically make someones experience bad by having to re click run on after they click to run somewhere.

Enjoy.
Hope this helped a few people as I just spotted it and decided to put a bandaid on it.
