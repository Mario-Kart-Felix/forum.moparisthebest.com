So as some of you may know I’m writing my own Runescape private server emulator. I’m not sure yet for what revision client it is going to be, though I’m leaning towards 377. I’m thinking 377 because it is more pure than 317 as the 317 client uses a 377 cache and has RSA disabled. Some of my goals for this project are: to write a stable and event oriented framework. To integrate scripting (preferably Ruby) into the game and to have a fully functional and event driven login and update server.

In my last project I posted all the updates in the thread, though I’m not going to do that this time as I have set up a public subversion repository. Originally the way I structured my framework wasn’t working and I was having problems with Netty. I recently started rewriting it so expect it to be quite bare for now. The code is inspired by some of the works of blakeman8192 and Graham. So credit to them where due.

What I wanted to use this development thread for is getting ideas from other participating members of the community and to get assistance with problems I am having at a certain reference in time. All updates can be viewed in the revision log at Assembla. Do NOT expect me to keep this completely up to date every single day as I have found other interests in my life. Though please, I would love to hear people’s thoughts, comments and constructive criticism on this.
