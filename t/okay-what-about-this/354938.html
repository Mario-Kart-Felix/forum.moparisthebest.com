Jython July 31, 2016, 6:38pm #4

the Surface (GameImage) class stores all data about all loaded sprites (pictures) in the game. there are two methods to define a sprite from an external source: one which decodes a TGA file and one which uses jagex’s own proprietary format. unfortunately you may be disappointed to know that most people use neither of those and simply modify code in what i called pregame_display():

public void pregame_display() { logged_in = false; screen.skip_lines = false; screen.clear(); if (ui_state == 0 || ui_state == 2) { int i = (ticks * 2) % 3072; if (i < 1024) { screen.sprite_plot(0, 10, 2500); if (i > 768) screen.trans_sprite_plot(0, 10, 2501, i - 768); } else if (i < 2048) { screen.sprite_plot(0, 10, 2501); if (i > 1792) screen.trans_sprite_plot(0, 10, ui_sprite_off + 10, i - 1792); } else { screen.sprite_plot(0, 10, ui_sprite_off + 10); if (i > 2816) screen.trans_sprite_plot(0, 10, 2500, i - 2816); } } if (ui_state == 0) initial_ui.display(); if (ui_state == 1) register_ui.display(); if (ui_state == 2) login_ui.display(); if (ui_state == 3) lost_ui.display(); screen.sprite_plot(0, wnd_h, ui_sprite_off + 22); screen.copy(graphics, 0, 0); }

the first bits of code handle animating the pre-rendered views of the world. you’d simply comment them out and do something like drawing a loaded java2d Image via graphics.drawImage(). note you’d probably want to rid yourself of that last sprite_plot() as it draws that rather ugly blue bar at the bottom.
