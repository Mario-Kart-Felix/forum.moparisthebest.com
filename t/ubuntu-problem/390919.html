zyle1992 August 1, 2016, 11:12am #2
  1. Chances are, it’s looking for the cache in the C drive. Since Ubuntu doesn’t set letters to drives. Easiest way to do it is put the cache in ~(home), and then in the signlink class, under findcachedir() replace it with this:
public String findcachedir() {
    return String.format("%s/%s/", System.getProperty("user.name"), ".your_cache_name");
}
  1. Subsequently, since it can’t find the cache, and at start up, it gets the CRC values from the cache, it refuses to connect to any server, because it has no cache to load from.

I thought I did…

Anyway here’s what you do.

  1. Go into your client’s source files
  2. Go into the “sign” directory.
  3. Go to the method “findcachedir”
  4. Replace all that code with the code I previously posted.

As long as the cache is valid, and the client can find it, you should be able to connect. Some other things that don’t let it connect, is because of the CRC check when the client starts up, or if it can’t find the cache, it will request the cache from the server through port 43595. If the server isn’t set up for it, you can’t connect either.
