jimmy5410 July 31, 2016, 10:58am #7

[quote=“Howso57, post:6, topic:302017”]Ok so I want to stick with runescape.jar

What would I need to put into the provideSignlink method?[/quote]

Alright. You’ll need to start off by replacing a few of the classes in the client with the ones from the loader.jar. In the 562 client, these classes are

an ud or va mf
You replace these classes, because that’s what the loader.jar does (look in the run() method of the loader class). You can then start your client normally, but because you’ve replaced these classes, you must invoke the provideSignlink() method. This can be done by adding the following code right after you create a new instance of the client class (and right before you set the applet’s stub).

Class signlinkClass = this.classLoader.loadClass(SIGNLINK); Constructor<?> signlinkConstructor = signlinkClass.getConstructors()[0]; Object signlinkInitialized = signlinkConstructor.newInstance(APPLET, 32, "runescape",29); client.getMethod("providesignlink", new Class[] { signlinkClass }).invoke(null, new Object[] { signlinkInitialized });
Where SIGNLINK is the name of the signlink class (in the 562 client, ud), and APPLET is your new instance of the client class cast as an applet. The above code comes from the run() method of the loader class.

If you didn’t screw this up, then you should have a working signed client.
