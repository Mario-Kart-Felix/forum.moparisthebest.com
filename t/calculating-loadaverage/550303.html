Lin August 4, 2016, 2:56pm #2

I don’t claim to be correct, but I’m pretty sure multiple threads on a single core/processor, for example, don’t actually run concurrently. The processor just switches which thread is being processed but can’t process two at the same. Since the actual workload the processor experiences (provided you ignore any expenses required by switching threads) doesn’t change if you had 1 thread vs 2+ threads, you probably ignore them.
