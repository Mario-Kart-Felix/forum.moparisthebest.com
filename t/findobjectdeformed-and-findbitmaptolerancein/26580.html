FindBitMapDeformedTolerancedIn is a function that examins the shape of an object aot its color. It is in fact SCAR’s only shape checking algorhythm. If you feed it tolerance and contourtolerance it will set a initial tolerance value. I dont think it loops. The looping you will need to do manual, I think. At least thats what I have done with FindObjectDeformed. Now, this function is only usefull for static objects, like doors, symbols, trees, banks and other static stuff. It is simply too slow for fast moving objects.
