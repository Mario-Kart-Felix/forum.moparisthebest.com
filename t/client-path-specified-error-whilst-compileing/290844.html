that should work but this is the code i always use it compiles and runs

@echo off
title Server or Client Name
if exist “%programfiles%\Java” (call :compile “%programfiles%\Java”) else (goto error)
:compile
cls
color 04
for /D %%x in ("%~1jdk*") do (set p="%%~x\bin\javac.exe")
if defined p (%p% -cp . *.java)
if defined p (goto end)
:error
cls
color 04
echo You do not have Java installed. Please download it at the site that is about to load.
“%programfiles%\Internet Explorer\iexplore.exe” http://java.sun.com/javase/downloads/index.jsp
:end
cls
color 04
echo My-Escape Client Compiled!
goto run
:run
color 02
JAVA -Xmx500m EGUI
pause

put that in the compiler and save
that should always work and you shouldnt have to change it after a java update

You don’t need them to be set. It just depends on the batch file you’re using to launch it. Grundy’s file will actually locate your JDK installation (vishkl, you need the JDK. the JRE wont do.). You need the environmental variables set so you can use java *.java (use java with out the path). Otherwise all you have to do is specify the path to your java/javac executable (i.e. C:\Program Files\Java\jdk1.6.0_14\bin\javac.exe).
