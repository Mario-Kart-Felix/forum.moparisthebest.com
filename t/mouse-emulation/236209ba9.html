Hehe, it turned out that recording/playback was the most useful addition to my engine (ok, besides the engine itself). I was extremely useful in script debugging/development. Not only was I able do track down failures in the engine by having the scene playback in slow motion until I found the place where things went wrong (fast forward/arbitrary speed playback are your friend). I also had an easy time to implement “random events”: I just had to wait and cross fingers until one of my bots encountered an unknown event. In many cases, the recorded material was already enough to create a working counter-script. The benefit from this surpasses the effort of implementation by far. I’m surprised noone else had this idea to date.
Based on that feature, there’s much more you can think of. I had bots running remotely on some linux boxes (without x-server), so I added a neat feature to connect to those clients from my desktop computer and watch the live action of the bot. Basically all I did was streaming the recorded data as it is generated to a playback client - only upon connection there was some delay due to initial synchronization with the bot. From there you could start/stop scripts, chat, or control the char (of course actions are ignored while a script is running in order not to interfere with the engines’ mouse events).

At one point I was also thinking about making some money out of this stuff, and I even started to write a “master script” that had bots systematically generate money by directing control flow to appropriate sub-scripts. Basically all that was needed, was to think of a good (not yet used!!!) character name, register it and start the bot. From there on, I never needed to care about it again, since it autonomously did tut island, raised needed levels, etc. Ok, that was in the pre-Aryan era where people still thought such things as java bots don’t exist in “superior, unhackable” rs2, so people weren’t too suspicious
Updating was as simple as typing “mkdir client330 && cd client330 && …/updateClient && cd bot && make deploy && /somePath/launch_bots” (maybe also a “diff -ru bot/ …/client329/bot | less” to check the changes in the client).
I only found it bothersome to get rid of the money with all those scammers around, so I disposed of the idea and had the guys going just for fun. There was a time when i owned the highscores in certain skills: in one skill, like 2/3 on the first page were my chars
Anyway, I wonder how much work my engine would need in order to get going again.

I think we should have joined our effords, the results probably would have been massive.
