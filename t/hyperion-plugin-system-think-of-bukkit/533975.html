also im done with posting since it seems to be mostly a big mess in here…[/quote]let me spell it out for you

let’s say you created a Hyperion plugin system. you would have a whole API that calls the different Hyperion methods through your plugin system… for example

package plugin.hyperion; class PlayerMethodProvider { //sendMessage and shit like that based on Hyperion methods }
If you were to use a different source, NONE of those would work, and you’d essentially have to rewrite most of the plugin system again because Java is not that dynamic. When you look at a plugin system for something else, like minecraft, they don’t have to change their methods very often because the game methods don’t change drastically at all, they just have to find the hooks in the server/client and tie those back into the already existing methods…

So technically its feasible to have a plugin system that works with multiple sources, but every source would need its own adaptation of the method providers. IE - the plugin system the OP is writing would NOT work for any other source than Hyperion out of the box.
