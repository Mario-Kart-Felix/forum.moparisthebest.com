It’s not really Java, it’s the setup of a RuneScape Private Server.

This is how it works. I’m sure you’ll understand if you really do have a background knowledge in basic programming.

The client, for example MoparScape client, is a special tool while playing a server. Each button you press, each time you click, a packet is created and sent to the server. If you click on an NPC, a certain packet is sent to the server and identified. If you click on the floor and start to walk, it tells the server through a packet. Without a client, you could not play the server.

The server is what handles everything the client tells it. All of the packets are handled in the server. The client class, which is the cramped up class, is an extension of the class Player, which handles everything to do with the player that logged into the server. If you wanted to, you could move everything from client.java into Player.java. That’s why it’s an extension. It even says “extends Player” at the top of client.java.

The client class handles a lot of things. Quests, dialogues, information and text on side bars, chatting, etc. Why? Because in most servers it has the parseIncomingPackets() method. This method has a simple switch statement which handles every single packet that you tell it to. Packets are identifed by numbers. For example, packet 103 is the command packet. Anything you type into the client chatbox that has a prefix of a double-colon is sent to the server through packet 103. The server then parses that packet and String and performs the command, or action, you want it to. The parseIncomingPackets() is a large method, and instead of creating seperate classes to handle these packets, the early users of private servers decided to cramp it all in the client class. Unforunately, nobody has decided to change this. So we all got used to it. I recommend you use handlers, though. It’ll help a lot with organization, neatness, and sometimes lag.

Each Java file makes up the source code for your server. The Java compiler, javac.exe, works like the C++ compiler, or any other language compiler. It compiles your source code, making smaller files out of it, just like compression tools like WinZip and WinRar. The Java executor/runner tool, java.exe, uses these class files to run a program. In this case, the server. Class files are not editable, but if you would like to re-create the source code out of these files, you may use a Java decompiler like JAD (free 3rd party program). The re-created source code will not be 100% accurate, but over time, the decompiling process has developed enough for it to be a very close re-construction of source code.

To make changes in a private server, simply make any changes you want in the source code (.JAVA files), and then compile it with the JDK compiler (javac.exe). Then run the class file (the main one should be server.java) that starts up the server with java.exe in your JDK/bin directory, and ta-dah. You got yourself a running server.

Hope I helped.
