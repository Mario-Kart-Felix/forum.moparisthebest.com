ddosd August 1, 2016, 8:41am #1

Since ‘Java Text’ isn’t open source, I decided to make my own base for java based text games. I haven’t used java in a very long time, so this helped me remember a few things. I made sure that I commented everything too, in the attempt to teach a few things. Anyways, make sure you open it with wordpad, not notepad, otherwise it will not display correctly. It’s also noteworthy to know that the program doesn’t close if you enter an invalid command.

Contents
Contracting - Lets the user do contracts. Selects a random item and gives a value of to 12. The users gets payed the amount the items are worth, plus a little extra.
Event Management - Allows events to be run on a new thread, over and over.
Investments - Uses Events. If you have money invested, every minute you will gain or loose money.
Skills - Firemaking, Woodcutting, Mining, Smithing.
Autosave - If activated, every minute-and-a-half, your game is automatically saved.
Shop - Complete shopping.
Security - A new implementation that activates whenever a user types certain tracked phrases at around the same time. After 3 reports, the user is given a random phrase to enter. If failed, a blackmark is added to the user. 4 black-marks an the users account is reset.

I’ve just been trained that the only comments that are required, are Javadoc type comments. I’ve been taught to split code up into usable chunks(methods, etc). Upon reviewing my comment I realise that it was improperly worded.

Comments aren’t required, but are useful for explaining parts of code that may be hard to follow. Line by line comments, are erroneous. But properly documenting methods(via Javadoc or something similar), splitting up code into different methods, and appropriate method and field names keeps the amount of comments down to a minimum.

Here are some examples:

int a = 0;// total sum
int b = 2;// step size
for(int i = 0; i < 20; i+= b) {
    a += i * b;// increase sum
}

This type of code, although documented well, should be rewritten instead of having comments explaining what’s going on.

int sum = 0;
int stepSize = 2;
for(int i = 0; i < 20; i += stepSize) {
    sum += i * stepSize;
}

That code is easier to follow, and doesn’t require comments, because the field names make it self documenting.
