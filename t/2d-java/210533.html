system July 30, 2016, 11:10pm #8

the main(String[]) method is the entry-point for the JVM. It needs to be there (atleast somewhere), so the JVM can start the application.

Why it is static?
Well, this is mainly because you havn’t created an instance yet, and therefore it has to be static. Instance-members (variables, methods etc) can only be used if you already created an instance, and call on them through this instance. All instance-members are not related to any other instance-member of the same type in any other instances of the same class (if that made any sense?).

Static, however, normally means the member will change to the latest changed member (if applicable). If you call on a static-member, there are only one ‘instance’ of that member. If you modify a static-member from one class in another class, and call on it in yet another class, this static-member will be what you changed it to in the other class. It seems complicated how I write it, but I don’t know how else to write it.

I hope you understand, though.
