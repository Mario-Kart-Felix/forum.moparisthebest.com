BobDole81 July 29, 2016, 10:59pm #5

The error is because you’re declaring a class of type Socket, not type SocketHack (or whatever its called).

You’re going by the idea that you can legally do this in java, using java.util.List for example:

List<?> myList = new ArrayList<?>();
// ...
ArrayList<?> myArrayList = (ArrayList<?>)myList;

You can do that because technically myList is of type ArrayList, even though its stored in a superclass type. Unfortunately, you can’t create a class of type ArrayList<?> or something and cast it to some example-nonexisting subclass. This makes it tough for you, i know, since you need to change the initialization of the Socket object to that of a SocketHack…but I can’t think of an alternative really than to change the type during the declaration of that object.
