Introduction
Many people still have unorganized winterLove-based servers with them. So let’s package them into different directories according (preferably) the category they should belong to.

Packaging
How do we do it? Let’s look at an example below.

[code=Java]package packaging;

public class PackageExample {

public static void main(String[] args) {
	System.out.println("I'm packaged.");
}

}[/code]

Once you have created that class, save it under a folder named [tt]packaging[/tt] and nothing else or you will be returned with a [tt]NoClassDefFoundError[/tt] error if you try to run by using [tt]java PackageExample[/tt] instead of [tt]java packaging.PackageExample[/tt] (the one which you should use).

Additionally, you could have a package in another (also known as sub-packaging), as typed out below.

[code=java]package packaging.random;

public class RixxxRocks {

public static void derp() {
	System.out.println("I'm a derp.");
}

}[/code]

Using a package
Now it’s time to learn how to use a package! You followed the tutorial, right…? There are actually 2 ways to use a package but I shall teach you only one - importing. When you use the [tt]import[/tt] keyword, you can call classes only with the public modifier within the package. Let’s try using the import method! Important: You cannot import sub-packages by just importing a package!

[code=java]package packaging;

import packaging.random.RixxxRocks; //import just the RixxxRocks class
import packaging.random.*; //import all the public classes in the random library

public class PackageExample {

    public static void main(String[] args) {
            System.out.println("I'm packaged.");
	RixxxRocks.derp();
    }

}[/code]

The output would read:

I'm packaged. I'm a derp.

More
Read up more on packaging here or here. Always remember to name classes with a capital letter for its first letter of each word! A package should always be in lower cases.

To test whether you have really learned anything, package that server (and of course import the used classes) into this:

2vtqn0z.jpg760×507

Compile it with this new batch file (alternatively, you can use ANT but this batch file saves the hassle):

@echo off title Compiling... javac.exe -d bin/src src/com/server/Server.java src/com/server/io/*.java src/com/server/misc/*.java src/com/server/model/item/*.java src/com/server/model/npc/*.java src/com/server/model/player/*.java src/com/server/model/player/io/*.java src/com/server/model/player/save/*.java pause echo Done cls Build.bat

and run it with this:

@echo off title Cleaned WinterLove cd bin/src java Server pause

If you can compile and run without any errors, congratulations, you have passed this test! Now you can proceed to package your own server - but remember, always make a backup before doing so! I cannot emphasize how important it is to make a backup.
