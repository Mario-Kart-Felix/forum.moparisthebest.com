Wintercanon August 1, 2016, 8:00am #11

Well there are different components to a server. The networking only handles byte communication between the client and server. You can use standard Java IO for that, then there is NIO and APIs built upon that like MINA, Netty and Grizzly. You need to write a component to handle the input and output.

Action handling is usually to do with all actions that occur within the server such as player/npc processing, player/npc updating, anything that is recurring, queued or delayed. Events are like actions but are happenings that fire off in the server. Events are such thing as logging in, updating or receiving a packet.

Scripting is obviously the use of an interpreted language for fast-paced development to remove the need to compile. Updating refers to sending different bytes to the client to denote such things as character appearance, walking and multiplayer rendering.

There are many more components to a server, with unlimited configurations of each. But these are just the core components.
