stCky August 4, 2016, 1:59pm #25

At my course, we were first taught HTML/CSS and basic Javascript. We then glanced over procedural PHP basics. After we understood the basics of syntax and functions we moved onto Object-Oriented PHP and MYSQL for database interaction. Then after those we were taught the basics of MVC and frameworks.

This is just a basic run-down of the path we took at my course, I was lucky enough to get a job before graduation and spent ~2 years banging out PHP jobs, e-commerce sites, blogs etc for small companys.

This year I started a new role as a node.js developer and can honestly say that node is the future. I toyed with RoR and from the get-go felt it was way too heavy. I found node to be a slight learning curve coming from PHP, with packages and server-side javascript and also having zero experience with command-line interfaces, but after a few months I’m more than competent and it’s really changed my perspective coming from PHP.

That isn’t so relevant now though, if you’re just starting to learn programming I’d suggest you look at object-oriented PHP and MYSQL for database interaction. The syntax is similar to JavaScript, when I was learning I felt it was familiar after learning js. You can take it a step further and start looking into MVC frameworks such as CodeIgniter or Laravel - these are a touchy subject though and web devs often argue about frameworks as much as we’ve seen people bitching over RoR in this thread.

Pick something that piques your interest and you can’t go wrong

z August 4, 2016, 2:01pm #31

I’ve been writing a web application using Django for the last few months for a software engineering class. I feel that I’ve been confined to restrictions based on the flexibility of Django itself, so unless you know how to rethink your ideas to fit Django’s framework, you might find difficulty with that. Also, in order to fully leverage Django’s power, you have to have a strong knowledge of web development (i.e. MVC, templates, database models, …) itself because of the power it contains. Though, there is a lot of ‘hand waving’ involved with some parts, so you don’t get as much control. Oh, and it also comes bundled with its own web server, so you don’t have to worry about that.

The last time I used Flask, I found it delightful and easy to follow, but I didn’t extensively look into its usage enough to grasp its full potential. There’s some fascinating things I’ve read about it though, so that’s a route to take.

Doing any sort of LAMP (Linux, Apache, MySQL, PHP) work will give you full control of your web application. You’ll have to configure things from the ground up though. While it might be easy to get those four components running, the web application will need to be built yourself, unless you decide to use a framework (I have no experience using PHP frameworks).

I haven’t looked into RoR at all though, but the last time I tried it (a few years ago), it was a massive elephant of files to associate with a simple Hello World script, which turned me off.

So, to summarize, Django gives you a lot of power and less flexibility, LAMP gives you a lot of control and flexibility with deficiency in scaling, and Flask/RoR are uncertain to me. Seems like RoR is good though, from other replies I’ve read here.
