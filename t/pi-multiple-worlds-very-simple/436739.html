Hey guys, people seem to be asking for this a fair bit these days so I just decided to post it. I’ll be teaching you how to do it the quick, simple but badly explained and silly way in comparison to how you should do it.
Skip to “2” if your using different bases which playersave differently.
Character files in both bases need to save the same, what I mean by this is that the layout of how information is saved must be the same so lets say you have a Kill/death ratio both your bases for saving your kdr will have to be saved in the same format i.e. kdr = 3.4 if the format it is saved in is wrong then one of the worlds overwrites any data saved ruining the save file for the other world.

Make a folder in your C: drive called characters(click computer on your start up menu and you should see it) .
Now you want to make a common saving location for the character files and accessing location. This means that rather than saving the character files in the characters folder(Source\Data\characters) it will save it in the place you choose which will be somewhere both servers can access it.
You’ll want to open up your playersave.java file(Source\src\server\model\players) as this is the file which saves your players files pretty much. Search for ./Data/characters/"+playerName+".txt, this is the path for where the Rsps saves your players files(and what to save it as). ), then change it to C:/characters/"+playerName+".txt and compile. Now all your player files should be start saving and being read in C:/characters on your hard drive.

2.
Now, you want to be able to actually host two servers at once. All you need to do is for one server and client find the port(usually 43594) and change it to a different port i.e. 5555. Compile and then portforward.
You now have two working worlds which can be hosted of the same computer and load the same character files.

The tutorial is bad I know, feel free to reword, modify etc.
