Introduction
So you want to convert an if-statement eh? You want to reduce that 5 lined-if statement to 1 line huh? Well, you’re in luck, here I am to tell you a quick guide how. In this guide, I will…

[ul][li]Tell you how to convert a if-else statement and if-elseif-else statement.[/li]
[li]Tell you a cool, near little trick that even 12 year olds can know how to do (Coming from the 12 year old himself)![/li][/ul]

Conversion
First, let me tell you WHAT you are doing.

Basically, you are going to make one if statement reduced to one line. Seems magically? Not really. Here is a simple example:

	if (playerCommand.startsWith("noclip")) {
		if (c.playerRights >=3) {
			c.sendMessage("Noclip is now on!");
		} else {
			c.sendMessage("You can't use this command");
		}
	}

Will become this:

        if (playerCommand.startsWith("noclip"))
                c.sendMessage(c.playerRights >= 3 ? "Noclip is now on!" : "You can't use this command");

Haha, when I first heard of the if-ternary statement, it was the most weirdest thing I’d ever seen. If this is your first time, be sure to tell me if the ternary statement looks creepy or not!

Explaining what I did
Lets make that code I provided you the key code of this tutorial. Sound alright?

So, there are 2 ways to really think of this.

Number 1: Split the statement in two, from the “:”.
Number 2: Translate it out.

Do note, these ways to think of it are only how you think of it when you start out. Later on, you’ll be knowing how to read them without doing these stuff.

Using way #1

                c.sendMessage(c.playerRights >= 3 ? "Noclip is now on!" : "You can't use this command");

Split this from the “:”

                c.sendMessage(c.playerRights >= 3 ? "Noclip is now on!"
                                                                        : "You can't use this command");

The first part if the first if statement, the second part is the else.

                c.sendMessage(c.playerRights >= 3 ? "Noclip is now on!"

Becomes

		if (c.playerRights >=3) {
			c.sendMessage("Noclip is now on!");
		}

And

                                                                        : "You can't use this command");

Becomes

		  else {
			c.sendMessage("You can't use this command");
		}

Using way #2
Think of when you ask a question. The if-ternary can be translated like this: “Is this like this? If so, do this. If not, do this.” The “?” in the statement is basically the start of the sentence, “Is this like this?” Once you reach “:”, you can see the whole sentence. “Is this like this? If so, do this.” After that “:”, the second sentence starts, “If not, do this.”

It seems complicated to you, right? It really isn’t.

Lets try it out.

                              c.playerRights >= 3 ?

"Is the player rights greater than or equal to 3?

                              c.playerRights >= 3 ? "Noclip is now on!" :

“Is the player rights greater than or equal to 3? If so, say, ‘Noclip is now on!’”

                                                                          "You can't use this command");

“If not, say, ‘You can’t use this command’”

Nested ternary operator
The nested ternary operator is quite ugly in my opinion. It just seems so crammed up in one place. However, if you make it look good, it does seems VERY logical. Haha.

	if (playerCommand.startsWith("noclip")){
		if (c.playerRights == 3) {
			c.sendMessage("Noclip is now on for you oh owner!");
 		} else if (c.playerRights == 2) {
			c.sendMessage("Noclip is now on for you oh administartor!");
		} else if (c.playerRights == 1) {
			c.sendMessage("Noclip is now on for you oh moderator!");
		} else {
			c.sendMessage("You can't use this command");
		}
	}

Eh, I just want to handle that last part of the sentence (“oh RANK_NAME”) from a nested ternary operator.

	if (playerCommand.startsWith("noclip")) {
		String rank = c.playerRights == 3 ? "owner" : c.playerRights == 2 ? "administrator" : c.playerRights == 1 ? "moderator" : "";
		if (c.playerRights >= 1) {
			c.sendMessage("Noclip is now on for you oh " + rank + ".");
		} else {
			c.sendMessage("You can't use this command");
		}
	}

It looks pretty hard huh? Lets reformat that String.

		String rank = c.playerRights == 3 ? "owner" : c.playerRights == 2 ? "administrator" : c.playerRights == 1 ? "moderator" : "";

Becomes

		String rank 
			= c.playerRights == 3 ? "owner" 
			: c.playerRights == 2 ? "administrator" 
			: c.playerRights == 1 ? "moderator" 
			: "";

Doesn’t it look so much more nice now?

Conversion
Conversion is the same as the if-ternary statement, just with a bunch of more “Is this like this? If so, do this.”

Using way #1

		String rank 
			= c.playerRights == 3 ? "owner" 
			: c.playerRights == 2 ? "administrator" 
			: c.playerRights == 1 ? "moderator" 
			: "";

Split this from every “:”.

			  c.playerRights == 3 ? "owner" 
			: c.playerRights == 2 ? "administrator" 
			: c.playerRights == 1 ? "moderator" 
			: "";

For every part, its a new if statement.

		if (c.playerRights == 3) {
			rank = "owner";
		}
		  else if (c.playerRights == 2) {
			rank = "administrator";
		}
		  else if (c.playerRights == 1) {
			rank = "moderator";
		}
		  else {
			rank = "";
		}

Simple, right?

Using way #2
Again, just a buncha more questions.

                String rank
                        = c.playerRights == 3 ? "owner"
                        : c.playerRights == 2 ? "administrator"
                        : c.playerRights == 1 ? "moderator"
                        : "";

Answer:
“Is the playerRights equal to 3? If so, make the rank String ‘owner’. Is the playerRights equal to 2? If so, make the rank String ‘administrator’. Is the playerRights equal to 1? If so, make the rank String ‘moderator’. If not, be blank.”

Using way #3
You’re now going to go like, “YOU DIDN’T TEACH US A WAY 3!?” Yeah yeah, now I am. This way 3 only works for nested ternary operations.

  1. You organize the code to make it equal-looking.
                String rank = c.playerRights == 3 ? "owner" : c.playerRights == 2 ? "administrator" : c.playerRights == 1 ? "moderator" : "";

Becomes

                String rank
                        = c.playerRights == 3 ? "owner"
                        : c.playerRights == 2 ? "administrator"
                        : c.playerRights == 1 ? "moderator"
                        : "";
  1. Every thing is a question.
    “c.playerRights equal to 3? Then make it ‘owner’.
    c.playerRights equal to 2? Then make it ‘administrator’.
    c.playerRights equal to 1? Then make it ‘moderator’.
    Else just make it blank.”

Who this method is for…
For people who are visual. Its all stacked up like a questions format (There are even question marks there!).
