Superhan:

i’m not sure what an ordinal is…

In Delphi, a type is an ordinal type if its values have a Predecessor (through the Pred function), and a Successor (through the Succ function). Integers, Sets, Chars, etc, are ordinals, whereas Reals, Strings, etc are not.

Note that being ordinal is not directly related to being ordered. A real number, for example, is not an ordinal, because the distance between a real value and its successor would be infinitesimal.
