[size=12pt][u]More Information

The idea behind the name “Fear Factory”, is that I first have to find a large building (or teleport between a few), so essentially a Factory, ran by an evil scientist who experiments and keeps a hoard of creatures all around the building (factory), so a Fear Factory. It will be the main source of monsters to kill, and you need a certain kill count for each floor and I may even incorporate the overall kill count for something else, like a ranking system in game, but I haven’t quite figured that part out yet… the drops will be good too.

Although the person I downloaded the source from does not deserve any credit, I still know it’s not yet a source by me.

I can assure you there is no more malicious code amongst the Java files (I checked them all, line by line). It has been deleted and I now plan to turn this whole pile of shit around in to great game-play.

A lot of you may not agree with this but game-play is at the top of my priorities, before performance. Mainly because I have more ideas and knowledge about content than the inner-workings and framework. I will get there though, step-by-step.

I want my server to have a lot of information, whether ingame or on some forums. I won’t be using commands for playerRights < 3. Only I will have access to commands. The worst thing you can experience when logging on to a new server is lack of information. Every server is different, so how are you supposed to know where most things are? So I will be making sure that everything is explained clearly and easily available for the task in question.

I have an OCD that every NPC dialogue must be perfect, line for line, no gaps or straying words, that every sendMessage must use proper spelling and grammar. I strive for a professional, concise and simplistic look. If an area is interactive or in use, then everything around it should be interactive and in use. What’s the point in having a lever that does nothing?

So this is a source that I’ll be tidying up, adding to, editing bits, removing parts and pretty much perfecting to my standard.

I know everything you have to say about what I just wrote, stuff like:

“You will never have a perfect server.”
“If you can’t code, don’t bother.”
“Performance is key, noob.”

and that’s fine, you can say these sort of things - they do not phase me. I’ll be doing what I’m doing and step-by-step making a game that is my interprutation of fun.
