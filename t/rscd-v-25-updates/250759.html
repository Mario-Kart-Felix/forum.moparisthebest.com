system July 31, 2016, 4:04am #9

Actually, every user should not have to iterate through every other user to inform these about the events that they have themselves created (it would take too long). They should be posted to a “global” queue for a particular area (or the like). Every user should also receive these events from the same “global” queue and process them accordingly. This is the best approach I can figure out…

If you iterate through some collection of users and informs these of the events, the update-process will stop doing anything else until this have been done. The more users to update, the longer it will take to perform. If you instead post this event to a global queue, every user can peek the last few events that the user in question have not yet received, and update in accurate order (from the earliest to the latest events). This also does not necessarily have to be done in one update-cycle, which allows for other updates to be processed in-between, and therefore does not stop the update-cycle for any particular user at any given time.

Since this only happens in a very high populated area (lets say 150 people are in edgeville all within a map radius of each other.

1 thread is controlling this, to update everything that happens to others is a lot of CPU cycles for the queue, 1 person disconnecting (leaving the queue) may effect the queue, 1 person causing the queue to screw up by leaving the queue whilst the thread is still iterating through it’s jobs of updating everyone may cause another to fit the position in the queue, therefore get packets he/she are not supposed to. Then lets say the packet ID is to show the bank screen, the correct payload is not supplied (bank items etc) therefore client freezes/disconnects.

150x people in 1 area.
100 people every second would sent out some sort of entity update packet, might be movement, changing armor/wep, talking, killing npcs, opening gates etc.

when 1 person does something, it has to update 150x to everyone else.

100 people doing something x 150 updates to everyone around you = 15000 packets that thread is sending out per second (and this would be it’s minimal in a highly crowded area)

So it’s understandable that a few people don’t get what they are supposed to.
