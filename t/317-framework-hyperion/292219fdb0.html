Miss Silabsoft:

I had a interesting issue was working on the item dropping method and ground items found that I could not get items to show in the correct location. I outputted the localX and localY from the server and they did not match up with the client.

The location class gets the region by dividing by 8. However, client side, the region is only updated when the map region packet is sent, which is every 4 regions, so the localX and localY on the client will be larger.

I fixed this in rs2hd (my old server) by storing the location where the player entered the region.

http://svn.grahamedgecombe.com/svn/rs2hd/trunk/src/com/rs2hd/model/Location.java

There are two methods: getLocalX/Y(Location location) that get the local coordinates from a specific location. So, if you use a method like that and pass the last known region (iirc Hyperion already has getLastKnownRegion), you should get the same results.

Hey, just started taking a look at the changes you’ve made since the last time I looked.
I suggest this revised method in place of the current getCombatLevel() method, simply because it’s the actual formula that Jagex uses.

    public int getCombatLevel() {
        int atk = getLevelForExperience(0);
        int def = getLevelForExperience(1);
        int str = getLevelForExperience(2);
        int hp = getLevelForExperience(3);
        int prayer = getLevelForExperience(5);
        int range = getLevelForExperience(4);
        int magic = getLevelForExperience(6);

        int meleeCombat = (((13 * (atk + str)) / 10) + def + hp + (prayer / 2)) / 4;
        int rangeCombat = (((13 * ((3 * range) / 2)) / 10) + def + hp + (prayer / 2)) / 4;
        int mageCombat = (((13 * ((3 * magic) / 2)) / 10) + def + hp + (prayer / 2)) / 4;

        return Math.max(meleeCombat, Math.max(rangeCombat, mageCombat));
    }
