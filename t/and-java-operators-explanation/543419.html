Kattoor August 4, 2016, 1:11pm #3

They’re called ternary operators. You can say they’d ‘replace’ an if / else statement. Example:

String doesStringContainHi(String s) {
    if (s.contains("String contains hi")) {
        return true;
    } else {
        return "String doesn't contain hi";
    }
}

… could be replace by…

String doesStringContainHi(String s) {
    return (s.contains("hi") ? "String contains hi" : "String doesn't contain hi");
}

So: condition ? what if true : what if false

davidi2 August 4, 2016, 1:11pm #11

To expand, the amount you can nest cleanly depends on where you’re using it.

An example of a pretty clean nested ternary statement is one I’ve seen in compareTo methods that have a simple one variable comparator (when you cant cast to integer, because if you can you can ignore the ternary all together):

public int compareTo(E other) { double difference = value - other.value; return (difference == 0 ? 0 : (difference > 0 ? 1 : -1)); }
