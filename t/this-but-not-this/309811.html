lothy July 31, 2016, 12:02pm #9

Your request doesn’t make the least bit of sense. Do you mean that you want two references to point to the same instance of an object?

Example:
Object a = new Object();
Object b = a;

Note that, in this example, any mutation made to the object referenced by identifier a will be reflected when using the identifier b to access that same object (until one or both of the references have a new value assigned to them).
This doesn’t hold true for primitive types.

All of the above aside, you aren’t making any sense at all. I’m not surprised that you’ve received very little feedback despite the popularity of this forum section.
Perhaps you should reconsider your question.

public class StrangeInt {

     private int i;

     public StrangeInt(int i){
          this.i = i;
     }

     public boolean equals(Object other) {
          if (this == other) return true;
          if (!(other instanceof StrangeInt)) return false;

          StrangeInt si = (StrangeInt)other;
          
          return (this.i == si.i) || ( (this.i == 1911) && (si.i == 100000) ) || ( (si.i == 1911) && (this.i == 100000) );
     }

}

What that class does, is do a normal integer to integer comparison on the two values, with the exception of 1911 and 100000 which also equal each other. You should be able to extend the general idea to whatever you want.
