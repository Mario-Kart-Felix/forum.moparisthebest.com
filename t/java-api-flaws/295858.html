General Programming
xeez July 31, 2016, 10:09am #1

Some people and I were recently discussing this issue on another thread (http://www.moparisthebest.com/smf/index.php/topic,389094.0.html)

As we all know, Java is a language that after compiled, can be interpreted on diverse systems. There is, however, specific code that can limit the use of an application to specific systems.

For example, a simple application that creates a “File” object, in which the fed string parameter’s directory-separator is “”, will not run on certain linux distros.

Even though the API provides enough resources to make any code cross-platform, (in this case, one can use [tt]java.io.File.separator[/tt]), should it not simply be abstract from all these little details, as Unhandled stated in the original thread? Are these small things “flaws” in the API?

State your opinions and expand upon them.

Here are my opinions quoted from the original thread:

[quote=“xeez`, post:17, topic:294173”]I assume it should, but they can’t be as stringent with file-interaction, which basically depends on the string passed as directory, on the same level on every single system.

To make this clearer, an example: a file on UNIX which contains the character ‘’ (backslash), does not imply that it is referring to a directory, rather than just part of the file name, whereas on a Windows system it does. Like this, there are many examples.

Many things in Java are abstract for the language’s programmer concerning cross-platform issues. Such as memory handling, external connections, etc.

Sun tries to make Java be as flexible as it can be, while maintaining platform-independency; but at the end of the day, it is in the hand’s of the programmer.[/quote]

[quote=“xeez`, post:19, topic:294173”][quote author=Mopman link=topic=389094.msg2917105#msg2917105 date=1250531795]

[/quote]Do you have the same opinion concerning system calls? Do you think java should restrict programmers to use system commands that are compatible with all platforms? Obviously not. Because this issue is strictly platform-dependent. Same with file directories.
