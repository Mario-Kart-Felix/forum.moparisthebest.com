moparisthebest July 31, 2016, 4:19pm #12

As to modifying a file via the terminal, I love to use ‘nano’, obviously if it is a file you need root permissions to modify you type ‘sudo nano’, so to open a specific file, something like ‘sudo nano /etc/X11/xorg.conf’ etc etc. All the shortcuts you need to save, close etc are at the bottom of the screen.

As to:
https://bugs.launchpad.net/ubuntu/+source/bluez/+bug/550288/comments/10
It looks as if the driver in the newer linux kernel changed the name of a module or filename from ‘hiddev*’ to ‘hidraw*’, and ubuntu (or maybe debian) didn’t update their udev rules to reflect this change. That is what you did and now it works.
