This tutorial shows you how to create a server starting from scratch! (Yes this is extremely easy, but this tutorial is aimed for people who don’t have as much knowledge about networking and what not!)

Note:

  • I will not show you Login

  • This is only partially network setup

  • You will need some java knowledge if you want to develop a server from scratch

  • Something’s may have not been explained; so please feel free to ask me about certain things, functions and any question you may have!

Introduction:
This tutorial will show you how to use NIO networking (library’s should already come with java) and setting up a basic client/source connection. You will however need to figure out how to get the client loaded up because some of the higher version servers usually have CRC, update server etc enabled so you will need to work it out.
I will only use 1 class file because the fewer the files the more easier it is to follow what is all does and it’s great for noobs

package com.rs2;

import java.io.IOException;
import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.SelectionKey;
import java.nio.channels.Selector;
import java.nio.channels.ServerSocketChannel;
import java.nio.channels.SocketChannel;
import java.util.Iterator;

public class Server implements Runnable {
	private Thread thread;
	private Selector selector;
	private ServerSocketChannel channel;
	
	public static void main(String args[]) {
		try {
			new Server().bind(43594).start();
		} catch (Exception e) {
			e.printStackTrace();
		}
 	}
	
	public void start() {
		thread.start();
	}

	private Server() {
		try {
			this.channel = ServerSocketChannel.open();
			this.selector = Selector.open();
			this.thread = new Thread(this);
			channel.configureBlocking(false);
			channel.socket().setReuseAddress(true);
			channel.register(selector, SelectionKey.OP_ACCEPT);
		} catch (Exception e) {
			e.printStackTrace();
		}
	}

	@Override
	public void run() {
		while (true) {
			try {
				selector.select();
				Iterator<SelectionKey> keys = selector.selectedKeys()
						.iterator();
				while (keys.hasNext()) {
					SelectionKey key = keys.next();
					if (key.isValid()) {
						if (key.isAcceptable()) {
							accept(key);
						} else if (key.isReadable()) {
							read(key);
						}
					} else {
						key.cancel();
						channel.close();
					}
				}
			} catch (Exception e) {
				e.printStackTrace();
			}
		}
	}
	
	private void accept(SelectionKey key) throws IOException {
		ServerSocketChannel ssc = (ServerSocketChannel) key.channel();
		SocketChannel sc = ssc.accept();
		if(sc != null) {
			System.out.println("Connection received from: " + sc.socket().getLocalAddress().getHostAddress());
			sc.configureBlocking(false);
			sc.register(selector, SelectionKey.OP_READ);
		}
	}
	
	private void read(SelectionKey key) throws IOException {
		ByteBuffer buffer = ByteBuffer.allocate(2048);
		SocketChannel channel = (SocketChannel) key.channel();
		int status = channel.read(buffer);
		if(status == -1)
			return;
		//key.attachment().process(buffer); // TODO: process player
	}
	
	public Server bind(int port) throws IOException {
		channel.bind(new InetSocketAddress(port));
		return this;
	}
}

Your server should start up successfully when you try to run it. When you try connect a client to it the server should print out the IP of the player trying to connect.

Define outperforms? If you have a separate thread for each connected client dedicated to simply processing incoming data, of course it will be better, but the overhead from each thread and having to switch between them will quickly start to take its toll.

Sure our hardware is getting better and better each day, but we still haven’t reached a point where we can effectively switch between large numbers of threads within short periods of time. In our case, up to a theoretical 2,000 connected clients per world, and all processing needs to be done well within the 600ms cycle.

Netty will perform better than Mina, hands down. (Remember, it was designed to!) But… there are still some things in Mina that I really like. Particularly, the session attribute map. Netty 4 made it a little better by including an attribute map with each channel, but they way you access them is still not as easy as in Mina.
