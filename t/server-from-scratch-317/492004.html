[size=16pt]What is this?[/size]
This is a new server that I made from scrach. It has NIO networking and contains of a JAGGRAB server, Ondemand server and game server. I do not recommend this server to use because it’s not complete and there is no way to write content yet (you can’t even walk!). The entire server exists of only three threads.

[ul][li]JAGGRAB server. The JAGGRAB server runs on its own. It binds on port 43595. The JAGGRAB server is responsible for updating the client’s archives cache.[/li]
[li]The Ondemand and game server. These two servers run on one thread, because they’re both bound on port 43594.[/li]
[li]The world updater thread. This thread is responsible for updating players. This thread updates the players every 600 milliseconds.[/li][/ul]

[size=16pt]Features[/size]

[ul][li]JAGGRAB[/li]
[li]Ondemand[/li]
[li]Packet encoding/decoding[/li]
[li]Login[/li]
[li]Player rendering[/li]
[li]GFXs[/li]
[li]Animations[/li][/ul]

Known bugs.

[size=16pt]About the Ondemand server[/size]
The Ondemand server is a non-blocking NIO server. It uses the same port and thread as the game server. It doesn’t make use of additional worker threads or any thread pools, so performance is not optimized. The server sends the #317 cache to the client.

[size=16pt]About the game server[/size]
The game server shares the same thread as the Ondemand server. It supports basic player updating and packet handling, such as interfaces, character design and command handling.
