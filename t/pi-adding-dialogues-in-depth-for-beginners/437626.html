I know there are plenty of these guides, but I couldn’t follow any when I first started, so I’ll take the point with this guide and explain almost every line of code.

I hope you enjoy it.

So first of all:
What will this tutorial do for me?
Well, hopefully this will help beginners to understand dialogues in a Project Insanity source, this doesn’t mean you copy and paste my codes, this means you learn from it and adapt.

How hard will this be?
This is easy, it’s just getting your head around the methods to begin with, I’d rate it about 3/10.

What can I use this for?
You can use this to get NPC’s you’ve spawned to “talk” with players. This is good for starting quests, asking if people want teleported somewhere, giving warnings and I used it to create a tutorial.

What files will I need to edit?
So you’ll need to edit the following files:
Data\cfg\spawn-config.cfg - this is to spawn the NPC.
src\server\model\players\DialogueHandler.java - this is to actually create the dialogue.
src\server\model\players\ActionHandler.java - this is to tell the dialogue when to open.
src\server\model\players\ClickingButtons.java - this handles the clickable options.

[u][i]Creating a dialogue.
So first, we want to actually create what the npc will be saying.
To do this we need to open up DialogueHandler.java, you can see the path to it under the “What files will I need to edit?” heading at the top.

Please note: I will be using the case numbers of 33->45 because they are unused. I’m using my own server tutorial as a guide here, but you can adapt it to anything.

[center]This is what we will be creating:
[/center]

So lets look at my first part of code:

		case 33:
			sendNpcChat4("Hello!", "I am the SurvivalPKz guide.", "I can teach you all about SurvivalPKz.", 
			"Would you like to know more?", c.talkingNpc, "RuneScape Guide");
			c.nextChat = 34;
		break;

This is basically telling the npc to say that to anyone who clicks on him.

Lets break it down:

sendNpcChat4

This tells the source how many lines of writing to send, I only need 4 so I put a 4 at the end. But you can do anything from 2 lines up to 4 lines using this method, just replace the number at the end. But, for 1 line, it must be left as “sendNpcChat” otherwise you will get errors.

"Hello!", "I am the SurvivalPKz guide.", "I can teach you all about SurvivalPKz.", 
			"Would you like to know more?",

This is the actual Dialogue that is sent to the NPC for the user to see. These corrospond to how many “sendNpcChat” lines you used, if you put in a 3 at the end you’d have to use something along the lines of this:

"yourtext", "yourtext", "yourtext",

These lines are parsed by two speech marks - “yourtext” and then a comma - “yourtext”, just remember to this for all of them or you’ll get errors.

c.talkingNpc, "RuneScape Guide");

This puts your NPC’s head to the left when he is speaking, this makes it look more professional. Mines says “RuneScape Guide” because that’s what I’m using, you should change it to whatever NPC is speaking for you. Note how this is the actual NPC’s name, and not his/her npcid, remember that this is how it works.

If you wanted it to just display a notice, leave it as:

sendNpcChat3("hello", "are you following", "this epic guide?");

The same number of lines method takes action here, so keep that in mind also.

			c.nextChat = 34;

This tells the dialogue what chat to go to next, this works on case numbers, So if you were on case 33, like I was, and your next dialogue that you wanted displayed was 34, then your c.nextChat would equal 34. Simple, right?

break;

This tells the source that the dialogue has finished now, if you don’t add this in it’ll never work, this must be there!

Okay, so that’s your first dialogue created and explained! Now to make it work!

[u][i]Making The NPC Speak
So now that you’ve created some dialogue, you want to make it work, otherwise it’s just useless code. So open up your ActionHandler.java, the path to this is explained at the top of the guide.

Once you have that open, look for the following:

 public void firstClickNpc

This is basically telling your source that when someone clicks this NPC, it opens the dialogue.

Underneath that, add the following code:

        case ***: 
            c.getDH().sendDialogues(33, npcType);
            break;

The 3 asterisks or “***” beside your case should be your npc id, for me it is 945, so mines looks like this:

        case 945: 
            c.getDH().sendDialogues(33, npcType);
            break;

This just tells the source to open case 34 in the DialogueHandler.java as soon as someone clicks the npc. The break does the same as the last time, it makes the source recognise it’s the end, so remember this or it won’t work.

Please note: some project insanity sources use “i” instead of “npcType”, if this is the case, then just replace where I have npcType with a single “i”

You’re now done with this part of the tutorial! If you compile and go in-game and talk to your spawned NPC, the chat should now pop up, but it won’t be much use without the ClickingButtons.java side which is coming up next.

[u][i]Adding Clickable Options:
So this makes the dialogue interactive, and lets people choose their next option, so take a look at my code for this:

		case 34:
			sendOption2("Yes I would like to know more.", "No I would not like to know more.");
			c.dialogueAction = 94;
		break;

So lets break it down:

case 34:

This is just simply the case of the dialogue in DialogueHandler.java

sendOption2

This is similar to the sendNpcChat#, this just tells the source how many options to send, in this case, it’s sending 2 so I put the number 2 in. This can go all the way up to 5, unlike dialogue which can only go to 4.

("Yes I would like to know more.", "No I would not like to know more.");

This is simply just the options which are there to choose, change them to whatever options you wish to have.

c.dialogueAction = 94;

This part is extremely important in making the clickable option work. Just because I used the number 94 doesn’t mean that you have to, the number can be changed to anything not being currently used in DialogueHandler.java

Now, the next part is the hard part.

What you want to do is, get in-game and speak to your npc and he’ll display your options, for example, mines displays this:
[center]
[/center]
Now that your options are showing, click the first option and take a look at your run.bat file for the source, it should say something like:

yourusername - actionbutton ####

The “#” stands for the number. That it’ll say to you.

Now you have that number go to ClickingButtons.java and search for “case ####” the #'s once again stand for your number, once you’ve searched that it’ll show the case, if the case is already being used and it’s nothing to do with something you’ve added in or you see that’s important, delete it. Then paste the following code in:

if(c.dialogueAction == 94) {
					c.getDH().sendDialogues(35, 945);
				}

This tells it that when this is clicked it should open dialogue 35 using the npc of 945.

Here’s it explained some more:

if(c.dialogueAction == 94) {

Make sure this is the same dialogue action you had in DialogueHandler.java, for me it was 94.

					c.getDH().sendDialogues(35, 945);

This tells the source what dialogue to open up and who’s saying it. In my case, I wanted it to load up dialogue 35 with the NPC 945, which is the RuneScape Guide.
