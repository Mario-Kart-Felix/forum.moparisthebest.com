sinisoul August 3, 2016, 11:58pm #163

Well, as promised, I think its about time that I talk about some features of the new base I’m working on called Nova. If you’re not interested in the process of designing a base, stop reading now, it’ll be pointless for you to continue.

Initial Research

It’s been about three weeks since I started getting into in depth research of the client which included documenting as many packets, and menu actions as I could. The 443 protocol page is almost as populated as the 317 protocol page. Eventually, it will be more documented than 317. I hope that this will attribute to a possible shift in research towards 443. You can find all the research that I have currently documented on the wiki: http://rswiki.moparisthebest.com/index.php?title=443_Protocol

This page is not set up like the others, because I documented menu actions and also global client states. The reason why I did this was so I could design a solid event system.

Events

Most of the currently written portions of Nova are events and listeners. The content events such as EntityOptionEvent, ItemOptionEvent, SocialListUpdateEvent, and ExamineEvent are the core events for all of the repetitive client actions that don’t include walking, chat, and a few other cases. There is a listener for each of the events, and currently the option events are coordinated through an OptionEventHandler and OptionEventController.

These events will each have a decoder class used to create the event and will be easily configurable. Since events of the same type, but a different option id, have spoofed variable streams it will be simple to instantiate a decoder to solve this problem. The API is meant to be flexible enough so that new events can be added.

package org.evelus.nova.content;

import org.evelus.nova.content.events.EntityOptionEvent;
import org.evelus.nova.content.events.ItemOptionEvent;

/**
 * Evelus Development
 * Created by Hadyn Richard
 * 
 * Controller used for coordinating option events with their appropriate handler.
 */
public final class OptionEventController {
    
    /**
     * Constructs a new {@link OptionEventController};
     */
    public OptionEventController() {}
    
    /**
     * The option event handler for players.
     */
    private OptionEventHandler playerOptionEventHandler;
    
    /**
     * The option event handler for npcs.
     */
    private OptionEventHandler npcOptionEventHandler;
    
    /**
     * The option event handler for ground items.
     */
    private OptionEventHandler groundItemOptionEventHandler;
    
    /**
     * The option event handler for still objects.
     */
    private OptionEventHandler stillObjectOptionEventHandler;
    
    /**
     * The option event handler for items.
     */
    private OptionEventHandler itemOptionEventHandler;
    
    /**
     * Sets the player option event handler.
     * 
     * @param playerOptionEventHandler  The option event handler.
     */
    public void setPlayerOptionEventHandler(OptionEventHandler playerOptionEventHandler) {
        this.playerOptionEventHandler = playerOptionEventHandler;
    }
    
    /**
     * Handles a player option event.
     * 
     * @param event     The event.
     * @param optionId  The option id.
     */
    public void handlePlayerOptionEvent(EntityOptionEvent event, int optionId) {
        if(playerOptionEventHandler != null) {
            playerOptionEventHandler.onCall(event.getId(), optionId, event);
        }
    }
    
    /**
     * Sets the npc option event handler.
     * 
     * @param npcOptionEventHandler The option event handler.
     */
    public void setNpcOptionEventHandler(OptionEventHandler npcOptionEventHandler) {
        this.npcOptionEventHandler = npcOptionEventHandler;
    }
    
    /**
     * Handles a npc option event.
     * 
     * @param event     The event.
     * @param optionId  The option id.
     */
    public void handleNpcOptionEvent(EntityOptionEvent event, int optionId) {
        if(npcOptionEventHandler != null) {
            npcOptionEventHandler.onCall(event.getId(), optionId, event);
        }
    }
    
    /**
     * Sets the ground item option event handler.
     * 
     * @param groundItemOptionEventHandler  The option event handler.
     */
    public void setGroundItemOptionEventHandler(OptionEventHandler groundItemOptionEventHandler) {
        this.groundItemOptionEventHandler = groundItemOptionEventHandler;
    }
    
    /**
     * Handles a ground item option event.
     * 
     * @param event     The event.
     * @param optionId  The option id.
     */
    public void handleGroundItemOptionEvent(EntityOptionEvent event, int optionId) {
        if(groundItemOptionEventHandler != null) {
            groundItemOptionEventHandler.onCall(event.getId(), optionId, event);
        }
    }
    
    /**
     * Sets the still object option event handler.
     * 
     * @param stillObjectOptionEventHandler The option event handler.
     */
    public void setStillObjectOptionEventHandler(OptionEventHandler stillObjectOptionEventHandler) {
        this.stillObjectOptionEventHandler = stillObjectOptionEventHandler;
    }
    
        
    /**
     * Handles a still object option event.
     * 
     * @param event     The event.
     * @param optionId  The option id.
     */
    public void handleStillObjectOptionEvent(EntityOptionEvent event, int optionId) {
        if(stillObjectOptionEventHandler != null) {
            stillObjectOptionEventHandler.onCall(event.getId(), optionId, event);
        }
    }
    
    /**
     * Sets the item option event handler.
     * 
     * @param itemOptionEventHandler    The option event handler.
     */
    public void setItemOptionEventHandler(OptionEventHandler itemOptionEventHandler) {
        this.itemOptionEventHandler = itemOptionEventHandler;

     * Handles an item option event.
     * 
     * @param event     The event.

package org.evelus.nova.content.events.decoders;

import org.evelus.nova.content.events.EntityOptionEvent;
import org.evelus.nova.content.events.builders.EntityOptionEventBuilder;
import org.evelus.nova.event.EventDecoder;
import org.evelus.nova.event.params.WordParameter;
import org.evelus.nova.io.Buffer;
import org.evelus.nova.net.IncomingFrame;

/**
 * Evelus Development
 * Created by Hadyn Richard
 */
public final class MobOptionEventDecoder implements EventDecoder<EntityOptionEvent> {
    
    /**
     * Constructs a new {@link EntityOptionEventDecoder};
     * 
     * @param idParameter   The id parameter.
     */
    public MobOptionEventDecoder(WordParameter idParameter) {
        this.idParameter = idParameter;
    }
    
    /**
     * The id parameter for this event.
     */
    private WordParameter idParameter;

    @Override
    public EntityOptionEvent decode(IncomingFrame frame) {
        EntityOptionEventBuilder builder = new EntityOptionEventBuilder();
        builder.setId(idParameter.decodeValue(new Buffer(frame.getPayload())));
        return builder.build();
    }
}

[SIZE=24pt]Default Configurations[/SIZE]

Runescape clients differentiate variously revision to revision. Part of the process of developing a server, mining frame ids, streams, and context in order to correctly emulate a Runescape server. This has been a growing issue in the back of my mind. There are some consistencies between revisions, but they are never exploited. Development time would be drastically cut in half for developing a new platform.

I have started Flare to create a standard on how the networking client code will have to be refactored. The standard will effect:

[ul][li]Incoming Frames[/li]
[li]Outgoing Frames[/li][/ul]

All frames will be given an id, name, a description for its purpose, use, and payload structure. Clients that completely follow the standard will be listed for download. If you are interested in assisting; fork the project and send me some pull requests with your updates or changes.
