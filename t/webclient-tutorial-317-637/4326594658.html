The title is misleading. Firstly, saying ‘support’ makes it sound like the webclient supports and loads multiple revisions, but it doesn’t. Something like ‘Webclient tutorial for multiple revisions’ would be better.

“508 clients are already webclient compatible”: Did you test this? If so, with what clients? Only the BerkyScape client, or the standard type 508 ip changer? I’ve never had a 508 client be ‘webclient ready’, per say, I had to add the cache downloading just like with all the other revisions.

You’re putting the client edits AFTER the jar is created and signed? People will need to recompile and then start the process over again, you should fix this. Put the client edits BEFORE the jaring/signing part. Use the iurl and anchor tags to link to the jaring and signing part, so people can skip to that after completing the client edits. You can see an example of those in my tutorial lists.

In which case, just use identical HTML with those few things changed, rather than the different styles you have now.
