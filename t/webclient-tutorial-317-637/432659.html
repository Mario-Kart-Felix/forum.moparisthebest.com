[center]Properly .JAR’ing the Client[/center]

1.
You’re going to want to open up the JARMaker. Set the Output File to the location of where you want the client to be .jar’d to.

2.
Next to, or under the Output File selection, you should see something like application.jar or demo.jar. Change that to client.jar.

3.
Set the Input File selection to where your .class files within your client are.

317 - Delta The .class files are usually right with the .java files.
317 - PI The .class files are usually right with the .java files or could be in a file called bin or data.
508 - BerkyScape The .class files are located in the files folder along with the .java files.
562 - ??? Reserved
637 - TorvaPkz v4 Located with the .java files.

4.
The next thing you are going to want to do is to click Manifest Template. You should then see something like this.

317 - PI client

You will then want to click Main-Class and type in what your base may be. (Another way to see what your main class is, is to right click your run.bat and would see something like

@echo off title Edge4Pk - Backup run java -Xmx500m -cp .;Theme.jar client pause
The client is the main class so that is what you will need to set your “Main-Class” as.

After you set your Main-Class, click Set Manifest and then click Create JAR!

The .JAR for webclients should be signed, so here are the steps to properly signing your client.

[center]Properly Signing the .JAR’ed Client[/center]

1.
You should see some tabs that look like this,

Select the JAR Signer tab.

2.
The first thing you are going to want to do is set the Signed JAR File option to: client.jar

3.
Your Jarsigner Location should be set by default, if it isn’t click the button with the … on it and find where your Java files are saved to. For Windos OS, it is usually “C:/Program Files/Java/jdk(Your JDK version here)/bin/jarsigner.exe”

4.
Now click Generate Keystore. The Keytool Location should be set, if not refer to Step 3 and instead of /jarsigner.exe, it will be /keytool.exe Instead of saving it as keystore

4.
As to where your Output Location was for the JAR’d client, set the Output Keystore Location to the same spot.

5.
Keystore Alias, Public Key and Private Key should be written down or well remembered. Set it to anything AS LONG AS YOU REMEMBER.

6.
You can just delete everything from Common Name through Province and select Generate Keystore

7.
Now you should select the …button. Click that and where ever your Out Keystore Selection was saved to, select the keystore file and then click OK.

8.
Now put in the information you wrote down at Step 5 for Keystore Alias, Public Key and Private Key. Then click Sign JAR!

private static void unZip() {

    try {
        InputStream in = new BufferedInputStream(new FileInputStream(fileToExtract));
        ZipInputStream zin = new ZipInputStream(in);
        ZipEntry e;

        while ((e = zin.getNextEntry()) != null) {

            if (e.isDirectory()) {
                (new File(getCacheDir() + e.getName())).mkdir();
            } else {

                if (e.getName().equals(fileToExtract)) {
                    unzip(zin, fileToExtract);
                    break;
                }
                unzip(zin, getCacheDir() + e.getName());
            }
        }
        zin.close();

        File cache = new File(fileToExtract);
        gui.dispose();
        cache.delete();

    } catch (Exception e) {
        e.printStackTrace();
    }
}

private static void unzip(ZipInputStream zin, String s) throws IOException {

    FileOutputStream out = new FileOutputStream(s);
    byte[] b = new byte[BUFFER];
    int len = 0;

    while ((len = zin.read(b)) != -1) {
        out.write(b, 0, len);
    }
    out.close();
}

For a webclient, you don’t need to modify the manifest; you’re just going to confuse people with that part. The main class is declared in the HTML applet tag.

Instead of using Jar Maker, just make some simple batch scripts for this. Have them put the script wherever the .class files are, and just use the simple command line arguments to do the entire process (besides client editing) in one step.

The title is misleading. Firstly, saying ‘support’ makes it sound like the webclient supports and loads multiple revisions, but it doesn’t. Something like ‘Webclient tutorial for multiple revisions’ would be better.

Right now it just looks like a clusterfuck of different tutorials mashed together.

That’s about it for now at a first glance. You’ve actually inspired me to make something better, a program that will just do everything for you if you can simply point it to your clients main class.
