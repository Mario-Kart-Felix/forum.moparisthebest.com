moparisthebest August 1, 2016, 12:07pm #16

I didn’t want to get into this debate, but your only other alternatives for a browser based game are:

  1. Your own browser plugin. (think Quake Live), not a very good solution.
  2. Flash. Alright for trivial games, I personally dislike it and I don’t think anyone thinks it’s a good option for something relying heavily on networking.
  3. HTML5 + Javascript. Probably a good alternative a few years from now when a decent amount of users have supported browsers. Right now very few users have HTML5 support.

So like I said, for certain projects, certain languages have the most advantages or disadvantages. For a cross-platform, easily deployed and/or browser based game, that needs some networking support, Java is your only option right now.
