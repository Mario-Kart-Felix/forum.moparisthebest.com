Hi. I have played some servers that have “two worlds” that are acting like 2 different servers. Because that’s what it is, is 2 servers running at once. I have found a way to have 2 servers running at once loading the character files from both servers because the “two world” server have 2 sets of character files.

Here’s how you do it. Note that this is probably more work then it’s worth.

Step 1

Run your world 1 server (your regular one)

Step 2

Find your server.java file and open it. Search for “listen”. The port should be set to 43594 or 5555. Change it to 43595. Make sure you set one of the listening ports to 43595 and the other one to 5555 or 43595. Compile your server then run it. Now you have 2 servers running from the same set of files.

UPDATING

Updating will now be a pain. When you do a update you have to set the port back to 43594 then compile then redo the steps above.

OTHER GLITCHES

Now you open up your server to duping, not losing items when you die and a character file may not save correctly. Players could now login to both servers at the same time. If they do this one of their profiles won’t save correctly.

Last Part

To do this you will either have to have a client that is able to switch ports by clicking change world or change port tool or something or you will have to make a world 1 and world 2 client. World 2 running the port as 43595.

If I missed something or something does not work please tell me and I will fix it.
