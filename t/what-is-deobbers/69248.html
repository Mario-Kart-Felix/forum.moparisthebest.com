Mopman July 30, 2016, 6:17am #2

Obfuscation, in the Java sense of the word, means to mess around with a class after is has been compiled so that it cannot be decompiled (or decompiling it results in invalid code). Jagex do this with the Runescape client, so that we can’t mess around with it. However, a number of people have written deobfuscators to undo this classfuckery.

However, deobfuscating the client is no longer as useful as it once was, because running one without heavy modification to certain traps (which nobody has attempted yet? as far as I know…) will result in a ban. They are still a handy tool for learning more about how everything works.

yakman July 30, 2016, 6:18am #7

note: the word ‘deobber’ and ‘deobfuscator’ mean the same thing

you know methods in java, normally they have nice meaningful names like sendPacket() or drawHealthBar()

but jagex change the names so they make no sence, they end up like d764fg45() and ggfh35234(), this is called obfuscation

what we do here is to remove this obfuscation, and make the names of things have more sence, now they are called method34() and method645()

heres an example
Before Obfuscation

public void drawHealthBar() { }

After Obfuscation

public void d45645() { }

After Deobfuscation

public void method453() { }

please note: in the rs2 client its not as simple as drawHealthbar
