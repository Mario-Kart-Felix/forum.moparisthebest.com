Example of my site:

With above form in the php code, you need to click “Next Step” to Continue every time.
It registers if you have filled the fields with anything, if you havent you can’t proceed.

When the form is completely filled, and you reach $step = 6;
The form will say “Submit Report” on the submit button.
On click it will lead to: php/report_player_send.php
Which looks like follows.

$objCon = new mysqli('DBhost', 'DBusername', 'DBpassword', 'DBname');

if ($objCon->connect_error) {
    die('Could not connect (' . $objCon->connect_errno . ') ' . $objCon->connect_error);
}

$type = $_POST['type'];
$cat = $_POST['cat'];
$player = $_POST['playername'];
$report = $_POST['report'];
$proof = $_POST['proof'];
$ip = $_SERVER['REMOTE_ADDR'];

$sql = "INSERT INTO report_player (type, category, player, report, proofs, logged_ip) VALUES ('$type', '$cat', '$player', '$report', '$proof', '$ip')";
$objCon->query($sql) or die;
header('location:../index.php?page=prrec')//link to your thank you page

When the info’s have been through php/report_player_send.php (within a splitsecond) it will send the infos to your database plus send the user to a thank you page that confirms the info’s have been sent.

Vulnerable code is bad, props for being one of the guys who go for MySQLi but the whole point of using the improved version is more or less for the prepared statements feature. This secures the code from SQL injection which ultimately lets any user manipulate your database. When you get round to making the administration panel that allows you to view and handle the reports be sure to secure the code from XSS attacks as well, I’d go more in depth but that could take up a whole post. You’d be wise to look into it.

You should be doing something more along the lines of:
[php]
$stmt = $objCon->prepare(“INSERT INTO report_player (type, category, player, report, proofs, logged_ip) VALUES (?,?,?,?,?,?)”);
//prepares the statement for database execution

$stmt->bind_param("iissss", $type, $category, $player, $report, $proofs, $logged_ip);
	//binds the parameters of the statement, first two being integers and the final four being strings.

$stmt->execute();
	//executes the query, safely and secured.

[/php]
