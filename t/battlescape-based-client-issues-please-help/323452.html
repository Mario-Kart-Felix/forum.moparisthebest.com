Yeah, he’s right.
Most clients can’t run without a client.java, unless it’s named something else. Client.java is probably one of the main files needed.
To find if there’s any hidden files, this is what I do.

  1. Right click on your folder, and select the “Properties” button, it might be “Options” depending on your computer.
  2. Click on the “Hidden” folder, to trigger it, then press “OK”.
    THIS WILL HIDE ALL THE FILES IN THAT FOLDER. THAT IS NORMAL, KEEP READING.
  3. After you pressed the “Ok” button, right click the folder again, go to “Properties” again, Un-trigger the “Hidden” option, by clicking on the box next to it, and this should now make all folders viewable. Click “Ok” once you have done that, and you’re finished. Go check if you have these files to change:
    Clickingmost.java - Changing the co-ords of where the teleport takes you.
    Texthandler.java - Changing the name of a teleport
    Client.java - To change where the teleport takes you, IF YOU DON’T HAVE CLICKINGMOST.JAVA.

Changing the Co-ords of a teleport:
First, make sure you are an Administrator, or Mod on your server so you can use the “::mypos” command. This command tells you your co-ords. For an example, 1432, 6534.

Step 1. What we do is go to the area that the teleport takes you to, the EXACT spot would be great. Type the “::mypos” command, so it tells you the co-ords. We need these to be able to change the teleport. It would be best if you wrote these down.

Step 2. Go to the area where you want, for the teleport to take you, type the “::mypos” command, and read the co-ords. It should come up with something quite simular to this:
PosyX: 3000 PosyY: 4000.
The two numbers are the co-ords. We need them.

Now when you have the co-ords, right click and choose the “Edit” button on clickingmost.java. This is how we edit the file.
Press Control + F (To find things) and enter the first FOUR DIGITS in the original co-ords of the teleport. Type in the box that popped up the co-ords of the original teleport (we found them in Step. 1). In this case, it would be “1432”. You most likely won’t get any results, so if you do, we will remove the LAST number of the co-ord in the search box, to get some better results. This usually happens because a teleport takes you to a different tile each time, usually 3-4 tiles apart. ANYWAY, in this case it would be “143”. You most likely will get a result this time. Keep finding the numbers, until you find TWO sets of numbers that look quite close to the original Co-ords. Say it’s “1436” and “6538”.
Change these to where you want the teleport to take you to (found in Step. 2).

NOTE: IF YOU ARE STILL NOT FINDING ANYTHING, TRY FINDING THE FIRST 3 NUMBERS IF THE CO-ORD, IN THE SECOND NUMBER! IN THIS CASE IT WOULD BE: 653.
If you find results, do the step ABOVE this step.

That’s pretty much it, I think… Save, Compile, and you’re finished.

LESSON 2:
Changing the Teleports name:

Step 1. Find the name of the teleport you would like to be changing. You can simply do this by logging in, going to the magic book, and finding it there.

Step 2. Open the Texthander.java file (Right click, and chose EDIT to open it). Press Control + F (this opens the search/find box) and type in the name of the original teleport you would like to change. You will most likely find a result straight away.

Step 3. Replace that name with what you would like to rename the teleport with.
NOTE: The line under that name will be what the text says under the name. Such as “The town of Varrock”.

Step 4. Changing the colour of the teleport name.
Here are just SOME of the many colours you can do. What you need to do, is place them infront of the text.
@red@
@xxx@
@whi@
@gre@
@yel@
@ora@ - I think
Example - @red@ Varrock
Example 2, for UNDER the name teleport. - @whi@ The home town of ________

That’s pretty much it. Save, Compile, and that’s all you need to do.
If you get compiling errors, see what you did wrong.
Hopefully I helped.
