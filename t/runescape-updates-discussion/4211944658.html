That charset table has 64 values… it’s an encoding scheme using radix-64, like Base64, except with the charset being rearranged i.e. [+, /, 0-9, A-Z, a-z] versus Base64’s [A-Z, a-z, 0-9, +, /]. If you calculate the number of bits stored in a 32-character-long encoded string, you have 32 * log 64 = 192 bits. Another seeming random 192-bit long quantity I’ve encountered with the client is what is saved in random.dat. Maybe there is a correlation.
