You need to add some internal state somewhere that indicates which stage of loading the client is at, and how much of that stage is complete (e.g., load_stage=ANIMATIONS, stage_progress=8/200 animations). Maybe this exists somewhere else already?

Then your progress view can call this periodically to work out the current progress, using a map that assigns each load stage to a percentage of the overall progress (run your own experiments to work out what these percentages should be). The rate of increase probably wont be very smooth so Iâ€™d use an interpolator to make it nice!
