The enum was like so:

BOB(score, id, "name") [/quote]Ah, yes. Enums are pretty much always used to store constant values. The two main ways to think of them are either as symbols (when its literally just a name, like in other languages) or as constant instances of a class you define (in cases where you add variables and methods

for example a comparison would be like

[code=java]
enum Instrument {
TROMBONE(“53d”),
CLARINET(“42b”),
FLUTE(“32f”);

private final String wut;

private Instrument(String wut) {
this.wut = wut;
}

public String upDaButt() {
return wut;
}
}[/code]

is similar to:

[code=java]

public class Instrument {
public static final Instrument TROMBONE = new Instrument(“53d”);
public static final Instrument CLARINET = new Instrument(“42b”);
public static final Instrument FLUTE = new Instrument(“32f”);

private final String wut;

public Instrument(String wut) {
this.wut = wut;
}

public String getDaWut() {
return wut;
}
}[/code]
The enum is cleaner in this case, and the way it stores the instances is better.

There are instances where sometimes the values wont be always final and you use them and maybe modify them once, but for the most part don’t consider them to be something you want to mutate.
