The Idea of a Secondary Cache

Games
Runescape
Lin July 31, 2016, 6:00pm #1

Well, I’ve been looking into the client figuring out all sorts of crazy things, like placing those damn map functions around (And the fact that it’s semi-automated is… kinda shocking) and instead of making a tool to edit all the existing data, we can make our own cache that overrides existing stuff. This is pretty much hardcoding, but everything we do would be in one file (It wouldn’t be a zip, or some other type of known archive, but it would be my own).

I’m honestly sick the current way of hardcoding stuff and the fact that we can’t edit everything (Well… I’m not sure how far Tom’s Cache Suite goes but I don’t think it loads on-demand data) with ease. If we used the secondary cache, even if it meant using the same format for things such as maps, we would still be able to edit everything as long as it was programmed in.

Now, if we only allowed the overriding of existing content, we wouldn’t be able to edit things already in-game. But, that’s where either using a cache extractor comes in hand, or dumping the data as it’s loaded during the client’s run time. I’m not sure how far the dumping tools out there go, but if they don’t do everything we need, I’d gladly dump things such as maps, or the “landscape file” (so I’ve heard).

So I’d love to work on something like this, which includes tools, but it depends on how much it’d be used. What do you guys think? Give me your opinions.

~Lin
