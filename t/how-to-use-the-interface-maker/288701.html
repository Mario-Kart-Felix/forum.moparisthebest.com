Getting Started
To get started, you must first download the latest version of the Interface Maker. Choose one of the links below.
Uppit (Recommended)
Mediafire

Once you’ve downloaded it, go ahead and and open it (You must have WinRAR). Please note that you need the .NET Framework to run the application. If you get an error while trying to open it, or don’t have WinRAR installed, look at the end of this topic.

The Layout
The layout is quite simple. On the left, you have buttons. These are what you will click to add something. The frame you see in the middle is your interface. It’s very basic right now, but that’s for you to change. On the top is the menu bar. From here, you can do things like clear the controls, save or open an interface, generate code, and more.

Controls and How Things Work
Most likely, you’re going to want to edit that blank interface. This is done by clicking one of the buttons on the left. Label adds text, and Image (Or button), and then the Last button duplicates the last thing you messed with. This is useful when making a frame that needs several controls.

To move a control you’ve added, simply click and hold it, and move the mouse. Let go of the click to place it. If the images exceeds the boundaries of the interface, it will be reset to the closest place it can be to your wanted location. You might notice labels with more text tend to reset back much sooner than they should. This a known bug and can be fixed with some math tweaking.

If you want to remove a control you don’t like, simply right click it and it will be deleted. To clear them call, click on the “Clear” button from the File menu. Keep note that once you delete a control, you cannot get it back.

You might be thinking that it’s impossible to line up controls perfectly, but you are wrong. If you hold down the “Ctrl” key while moving a control, it snaps it depending on its size and only moves in pixel counts of its size. Since this is not always the answer, you are given the power to use the W, A, S, and D keys to move the control pixel by pixel. If you hold down the Ctrl key while pressing one of them, it moves it depending on its size like holding down Ctrl and moving the mouse.

Adding Labels
To add a label, click on the Label button and then on your interface. Now double click on the “Label” and it will bring up a box. This box consists of a few parts.

The first part is to change its text. Change it to anything you want; it’s up to you. The next part is disabled, so don’t worry about that. The font is a drop-down box that lets you change the font the label has been given. Currently, each number is unknown, so it’s up to you to experiment! (HINT: 2 is the small font.)

When you’re finished, click on the “Save” button and you’re done with that label!

Adding Images and Buttons
Click on the Image button, and then click on your interface. You’ll see a commonly-used close button, an X in a box. Reposition it and double click on it. A box will appear, similar to the Label’s. The image index is the most important part. Scroll through it and you’ll see the image change. There’s currently only 11 to choose from. Please note that ‘10’ is the mainframe and covers up the whole interface. Your controls do not get deleted when using it. The next option is a check box, which you select if you want the image to be a button (Something you’re able to click). If you checked it, you’ll notice some more options came up. You really don’t have to worry about any of this in the Interface Maker version, except for the “Close Button” check box. Check this if the button is a close button.

When you’re finished, click on the “Save” button just like the Label’s, and you’re done.

Custom Sprites
Custom sprites have been made simple to add, because they would be added often. The Interface Maker doesn’t provide many sprites, so it’s up to you to add the rest.

Start by dragging the image you want into the Sprites folder that came with the Interface Maker. Please note it must be PNG format, and the Interface Maker uses the pure White color for transparency (This will be changed later on). However, in the client, the transparency color actually has to be transparent. I’d recommend just using a transparent one.

After you’ve added the image to the Sprites folder, get the filename (Without the .PNG). For example, if I added an image called “sword button.png”, it would be “sword button” (Without the quotation marks). Next, open up CustomSprites.ini, which also came with the Interface Maker. You are to now add the next number, an equals sign, and the file name without the .PNG. So, it will look like this.

#=filenameBy default, the # should be 11, because it left off at 10. Be sure you do not skip any.

So, if I had no custom sprites and used the same “sword button”, it would look like this.

11=sword button

That’s all there is to it!

Saving/Loading Interfaces
Currently, you can only load interfaces created with the Interface Maker. To do this, click on “Save Interface…” on the File menu to save your work, or “Load Interface…” to load an existing one. It loads interfaces with the extension of “INT”, so I’d recommend saving it with that (Saved with that extension by default). As you can see, Interface Maker came with a very basic interface. Feel free to work off of it.

Generating Code
When finished with your interface, click on the “Generate Code” from the File menu. The Sprites needed are very important, and you need to follow this step.

Every sprite needs a " 0" at the end, before the png. This might sound complicated, but it’s easy. Basically, just make the filename look like what it tells you.

It always requires you to use the mainframe.png, so you must put the mainframe.png in your client’s interface sprites folder. However, you would rename it to mainframe 0.png. This will hopefully be changed in a later version.

Next, just follow the instructions given with the generator form.

Making Buttons Work
This is rather easy, but this section is only for servers based off of DeltaScape. This means you will have clickingMost.java. Open it and find this line of code.

client c = (client) server.playerHandler.players[a];Below that line, add this.

System.out.println("Clicked button: " + c.actionButtonId + "");This makes it so whenever you click a button, it sends you the ID of it. Compile and restart your server. Open your interface and click the button. In your console (The black box) should print out “Clicked button: #####”. Remember that number.

Go back to clickingMost.java and add a new case, with the value of the number that was printed into the console when you clicked the button. This code must come after a “break;” somewhere.

case #####: //Replace ##### with the number that was in the console. //Stuff break;That’s all there is to it.

Errors, and How To Fix Them
.NET Framework Missing
If you need the .NET Framework, download it at This location, install it, and you’re done.

Getting WinRAR
Download and install WinRAR Here. Select the one that’s right for you.

The Client Doesn’t Load - An Error Occurred While Loading RuneScape
This error simply means you have an invalid interface ID. Be sure it’s below 20000.

The Client Crashes when Interface Opened
You’re trying to overwrite an existing interface. Change the ID when generating the code, and replace what you added with the new code. Keep trying until it’s normal.

The Interface Appears Messed Up
Again, you’re overwriting an existing interface. Follow the step above to solve it.

A Sprite Doesn’t Load
The only thing that could be causing it as far as I know is you didn’t get the proper file name. Remember it has to match what the generator said you needed.

This is the end of the guide. Anything I’ve missed, please let me know.
