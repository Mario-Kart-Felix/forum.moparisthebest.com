1.0 – Background:

xHTML is a newer form of HTML. Everything in xHTML is set to be more proper coding. Before starting with xHTML, you should probably be at least a little familiar with HTML.

// ---------------------------------------------- \

2.0 – Differences:

HTML uses tables, where xHTML uses “divs”. HTML leaves tags opened, while xHTML requires you to close all tags. xHTML uses docotypes.

// ---------------------------------------------- \

3.0 – Tags:

A tag is the thing inside of < and >. Often times with tags, there are two of them. An opening one and a closing. For example, the code to make something bold is <strong> and then when you want the bolding to end, you do </strong> The / closes the tag. So, it should look like this:

Not bold text here. <strong> Bold Text Here. </strong> Not bold text here.

So, in the “differences” section I mentioned that all xHTML tags are closed… In HTML, the image tag is <img src="url" alt="description" border="0"> Notice how you don’t see the closing /… Well in xHTML, the closer is there. It should look like this… <img src="url" alt="description" border="0" /> The same goes for most other tags, such as <br> in HTML would be <br /> in xHTML.

xHTML types and Docotypes:

A docotype works with the <html> tag to figure out which type of code you are using. There are 3 types of xHTML… There is Strict, Loose, and Transitional. Strict means that in order to have valid code, you need every little bit of code to be xHTML. Transitional means that you still have a little bit of HTML, but that’s okay. Loose means that you are just starting with xHTML, so the majority of your code is still HTML. Generally, I try to use transitional xHTML on every website that I make.

In order to do so, you need a docotype and an HTML tag. Below is what I always use:

[/code] The two "en" bits at the end there is the language. In order to use that code above, just replace the top part of your code with it. You code should go from something like: [code] Site Title Content [/code] to [code] Site Title Content [/code]

4.0 – CSS:

CSS stands for “Cascading Style Sheets.” It basically tells your website what colors and images to use.

To use CSS with your website, there are many ways to do so. My favorite is to make a new directory called “css” and then make a new css file in it called “css.css”. Then, go into my index.html/htm/php and put the following code:

<link rel="stylesheet" type="text/css" media="screen, print, projection" href="css/css.css" />

before my </head> tag. That code says that you want to take a stylesheet that is in text format, that can print and project colors onto your screen from the directory “css/css.css”.

Inside the file css.css, you should now add a bit of the following…

body { background: #000000 url('http://') no-repeat; color: #FFFFFF; } a { color: #CCCCCC; } a.hover { color: #999999; } p { padding: 3px; }
What that there will do is make your background black (or if you change the part after “url”, it will use that image), will make all font in your site white except for links, which will be a tad darker, and when you hover over them, will go even darker still. Then, if you have a <p> tag anywhere, that will make it have a padding of 3 pixels around it.

Now… in CSS, you may also see something like below:

#header { color: #000000; } .headerclass { color: #000000; }

The “#” and " . " in front of the “header” and “headerclass” say if that part of the CSS is and “ID” or a “Class”… the period makes it a class and the number sign makes it an ID. To use them, read on to the next section.

// ---------------------------------------------- \

5.0 – Divs and Tables

So… In xHTML as I mentioned before, tables are virtually obsolete. The only thing that tables are used for in xHTML is to put something an a set format of many columns, as that is one thing that divs cannot do.

Divs and tables are similar in the way that they are both used to organize your website and can both utilize the CSS in your site. To give a table the property that is defined in your CSS, inside the <td> tag, place something along the lines of id="header" What that does, is that it tells your site to use the id “header” from your CSS, which we defined earlier in this tutorial. It is the one that looks like #header { in your CSS. Your <td> tag sould now look like <td id="header"> with the closing tag still being </td>

Divs are basically used the same way, except that where tables already have many variables defined by themselves just for being tables, divs start out as just well… nothing… If you don’t define something in your CSS, it won’t happen in your Div. To add a div to your website that uses the “header” id that we defined above, use the code shown below.

<div id="header"> Text Here. Blah blah. </div>

One cool feature of divs that I use a lot is the float feature, so I just thought that I would mention it. In your code, if you are trying to get two divs next to each other, instead of using a table, just add the below code to your CSS.

float: right; for what you want to be on the right, and use float: left; for what you want to be on the left side. That will make it so they are each right next to each other.
