Hello!

I have been into this RSC private server-scene for quite some time by now, and I am still very interested in development of such. But, the thing is that time really does not allow me to do so, as I have many other projects going on at the same time. This in turn have made me decide that I probably should let this go. Though, I will still be visiting this site on and off, and perhaps even do smaller projects and/or work relating to the content on this site - MoparScape.org.

Since I really never got as far as I would want with my latest RSC private server project - FreeScape 2, using the RSCDEF ‘server-framework’, (which took very much time to get where I stand today), and I don’t want this project to be in vain, I thought I should let people apply for continuation of this project. By this I don’t mean I will release it to everyone that applies… I will only release the source etc. to people that I deem see fit for the particular purpose of continuation, and that I think that I can trust.

Why do I do it this way, and not just release it like everyone else? The answer to this question is fairly simple; I have seen alot of releases which have failed (I have done a few of them myself), due to the lack of continued development. Everyone just uses the latest and best source (not saying this is the best, but it is the latest - that is “released”), and they normally don’t add new features to it either (the only code-change is the name of the server and client (if even that)).

Further down I will list what should be in this application of yours - a form of template. But before you read this, I would also like to point out that I will omit any documentations that I currently have in this project. This due to most of which havn’t been implemented yet, or fully thought through. Some of them are unique compared to other servers that I know, others not…

The Application Template-Form:

Name:
Age:
Experience:
Purpose:

Name - This because it is good to know the persons name to whom you’re talking.
Age - Usually, the older you are, the more reliable and mature… Note: ‘Usually’!
Experience - This is only for experience within programming, not how to bake a cake etc. The more, the better.
Purpose - You have to have a purpose, else there is no need… Say, for instance, you only want to see the code - That is not a valid reason for continuation and to obtain the source.

When you have filled this Application, please either PM it to me, send me it in an E-mail (my E-mail can be viewed in my profile), or just write it in this topic. Do not create a new topic about it.

Last, but not least, some of you might wounder what RSCDEF stands for, and as I have omitted that README file from the released version, I will write it here:

RSCDEF = RuneScape Classic Development & Emulation Framework
