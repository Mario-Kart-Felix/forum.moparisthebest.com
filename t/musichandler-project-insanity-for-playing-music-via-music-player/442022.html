What this does is when you click a song on the music player, it will play a song: dependant on the switch-statement case.

Note: there are various and [size=12pt]better[/size] ways of doing this, I know, don’t lecture me about it.

What you will have to do:

[ul][li]Add more cases (depending on the button clicked).[/li]
[li]Find the correct songs for the button.[/li][/ul]

For this to work you’ll need a working music system (preferably cups’s).

Create a new class and call it MusicHandler - this is where all the music will be handled.

[code]package server.model.content;

import server.model.players.Client;

/**

  • Handles music.
  • @author IntBauk

*/

public class MusicHandler {

/**
 * Handle all our music in a switch-statement here.
 * @param c
 * @param button
 */
public static void handleMusic(Client c, int button) {	
	
	switch(button) {

		/**
		 * 7th realm: Varrock song played for debugging.
		 */
		case 46165:
			c.outStream.createFrame(74);
			c.outStream.writeWordBigEndian(125);
		break;

		/**
		 * Adventure: King Black Dragon song played for debugging.
		 */
		case 16191:
			c.outStream.createFrame(74);
			c.outStream.writeWordBigEndian(23);
		break;
			
		default:
			c.sendMessage("No record of song found.");			
	}
}

}[/code]

Then go to ClickingButtons and underneath the processPacket method, before the switch-statement add:

After that, you’ll then have to import it: import server.model.content.MusicHandler; (depending on the location of the MusicHandler class is).

This is just another way, instead of jamming every case in ClickingButtons.
