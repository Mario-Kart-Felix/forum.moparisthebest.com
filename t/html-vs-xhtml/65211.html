Mopman July 30, 2016, 5:50am #6

The idea behind XHTML and CSS/XSL is that you fully seperate design and content. The XHTML file contains all of your text and divisions and the CSS (or other stylesheet format, but most likely CSS) contains all of the layout and formatting. This was something that we were all supposed to be doing in HTML 4 anyway, but XHTML concretes it a bit more. It’s also a lot stricter on things like end tags and nesting. If you’re just learning, start doing it the proper way, XHTML + Stylesheet and never ever use tables for layouts.

BobDole81 July 30, 2016, 5:55am #10

Makes no difference really…you don’t need to ever use xhtml really…you’ll be set with plain html. But conformers to web standard like myself would recommend you learn XHTML first and just stick to it. It is basically the same thing as HTML, but with several additional restrictions. However, the restrictions will help design your html structure in such a way that it should appear pretty similar globally. Obviously different browsers have different parsers and will display certain things differently. But XHTML reduces the number of ways to do many things, and in turn helps to display things more consistently.

Obviously, it doesn’t work perfectly, but XHTMl is just a nicer standard of HTML (which doesn’t have much standardization to begin with).
