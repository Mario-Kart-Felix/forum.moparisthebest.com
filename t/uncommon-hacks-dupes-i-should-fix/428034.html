All you need is, as stated before, validate input on the server side, an to discourage any future changes.

What I mean by this is, as suggested by MITB, have the player change their inventory to have 1000 partyhats. When the player tries to force the items into the server (with a bank deposit, dropping the item, etc), the server notices the discrepancy, and rejects the packet. However, you can also resend the player’s inventory data so that all of the changes that were made client side, are reverted.

Now, with programs like CheatEngine, one can freeze the space in memory, so that even if the server resends the true inventory data, CheatEngine will reject the changes, so that it still shows as 1000 partyhats. This still will not make any impact on the server, due to the validation.
