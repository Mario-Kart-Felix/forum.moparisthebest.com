I’ve written up a registration process (which I use for my website), that is simple, yet efficient.

What it does, and what it checks for - it may do more…

[ul][li]Checks if the user name is taken or not in the database[/li]
[li]Checks if all fields were entered[/li]
[li]Proper submission and connection checks[/li]
[li]Checks for length of the strings[/li]
[li]Includes email, password, date and user name[/li]
[li]Checks for matching verification for passwords and user names[/li]
[li]Gives the password a sha1 salt hash[/li][/ul]

I made a table and used these fields:

    ID - Primary - Auto Increment
    Username - Varchar - 25(L)
    Password - Varchar - 25(L)
    Email - Varchar - 50(L)
    Date - Date

Two glaring problems come right out at me.

First, strip_tags() doesn’t provide protection like mysql_real_escape_string() does, you should ALWAYS use that function before passing any variables to mysql.

Second:
// Add some encryption to our passwords
$password = md5($password);
$v_password = md5($v_password);
md5 is a hash, not encryption, they are two totally different things. Also, a simple md5 hash for a password is a horrible idea and easily bruteforced in seconds with rainbow tables, you should at least use sha1 and ALWAYS ALWAYS ALWAYS use a salt.
