 public String value();

lothy August 4, 2016, 2:10pm #37

I think a map is the only reasonable way to do it. There’s no other way that you can decorate a class with additional fields without either adding them in code or extending the class.

If you use a hashmap with a reasonable load factor (default 0.75 is probably fine) then it should be very close to O(1) lookup time. Still far slower than accessing a field in an object by a getter method, but that’s the tradeoff I guess.

The other approach is a composition approach. Essentially the player object references other objects (e.g.: PluginOne, PluginTwo, etc) which themselves encapsulate all fields relevant to the plugin. There’s an extra layer of indirection to get to the fields but it at least drastically reduces clutter within the Player class and mitigates the potential for it to become a god class.
