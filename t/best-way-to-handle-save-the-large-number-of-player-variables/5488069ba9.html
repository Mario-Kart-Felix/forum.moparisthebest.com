Jagex probably does things server-sided the same way they do things client-sided. Load a byte for the data type, read data following. Stop reading when you reach type 255 or whatever. You can build off of this to have the smallest possible file sizes. Loading inventory? Type 25, maybe. Then each item has a type. 00 is a normal item - ID, assumed not stackable, so no extra 4 bytes needed for quantity. 01 might be stackable items, so ID then quantity. In your example, like with dealing with coal amounts, don’t store them until players actually deal with the coal carts. This way, a fresh player has a bare-bones save file. It’s extremely small and quick to load, although unfortunately you’d be forced to have ugly loading code. Small price to pay, though, as efficient systems aren’t as easy to write and deal with code-wise.

For quests, use bitmaps. No need to use more than a bit for flags, so quests should be minimal. In fact, for most quests all you would need is a single byte representing the quest step. Farm plots don’t have to have data once the area is back to its normal state - even so, each patch would only need a byte - that is, if you use the bits efficiently. Maybe use 2-3 bits for the plant state, another bit for if it’s diseased, another for dead, etc. You’ll probably need an extra byte or two for dealing with anything timer-related, though.

Anyway, just some thoughts. I only skimmed the thread so I apologize if I mentioned anything that was already covered.
