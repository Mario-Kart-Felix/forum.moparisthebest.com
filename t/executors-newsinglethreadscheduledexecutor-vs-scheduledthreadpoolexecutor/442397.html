notice

and how the Executor will use a thread pool containing 6 threads to execute scheduled tasks. if this executor is only going to ever execute 1 task (Server.run()) then it’s utterly pointless having 5 threads on standby waiting to execute tasks that won’t come

whereas

nominally says that it executes all scheduled tasks on a single thread, and since i assume you’ll only be ever using the executor to execute Server.run() then you’re better off using this.
