That’s one way. Another way is to distinguish between ‘messages’ and ‘packets’. One packet might contain n messages.
The receiving end then ACKs the messages received using a bitfield (so each message is ACK’d 33 times).

Why do this? Because messages that don’t need to arrive don’t have to be re-sent if they aren’t ACK’d; you just fire and forget (e.g.: teleport bubbles, skilling bubbles, etc).
