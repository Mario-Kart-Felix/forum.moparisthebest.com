If you’re going to make a tutorial on importing, you should include static imports, because I find them really useful, especially if you use classes purely for constants.

Example: Config class in PI

And I like to use them in minigames. Here’s an example of a minigame constants class I’m working on right now.

EDIT: ignore conventions. Ik the names should be all capital but I hate typing that, I do it at the end. Also make sure they are public and not private.

[code]package server.model.content.minigames.hunger;

public class Constants {

/**
 * Random items that are found on the ground, or in reward boxes
 */
public static final int[] randomItems = {
	4151, 1149, 1147, 1187, 1201, 1215, 1249, 1305, 1319, 1377, 1373, 1434,
	3204, 2491, 2497, 2503, 4587, 3140, 4087, 4091, 4093, 
};

/**
 * Rare items only found in rewardBoxes (E.G crates)
 */
public static final int[] rareItems = {
	11694, 11696, 11698, 11700, 11718, 11720, 11722, 11724, 11726, 11728, 11732,
	4718, 4716, 4720, 4722, 4724, 4726, 4728, 4730, 4732, 4736, 4738, 6585,
	4745, 4747, 4749, 4751, 4753, 4755, 4757, 4759, 4708, 4712, 4714, 565, 560
};

/**
 * Array of all the rewardBox possible objectIds
 */
public static final int[] rewardBoxes = {
	3685, 3686, 3687
}; 

/**
 * Maximum size of the minigame
 */
public static final int SIZE = 8;

}[/code]
