Mopman August 4, 2016, 5:12pm #5

They encrypt the login packets with RSA for the reason you say - to protect the user data in transmission. RSA is slow, though, so for the opcodes they seed an ISAAC instance, send the seed in the RSA packet as ‘protected’ data, and then use that to obscure the opcodes.

It’s to prevent bots tampering with the traffic without having access to the ISAAC seed - ie. they force you to be ‘in client’ where they can try to detect you. You can’t do a proxy bot like Autorune running on a remote machine without something in the client grabbing stuff and potentially leaving fingerprints so to speak.

No. ISAAC is used to encrypt frame opcodes because it makes it more difficult for bots to listen to or tamper with frames exchanged between the client and server. The seed used by the cipher is created at runtime, so if you want to decrypt a frame opcode, you have to dynamically obtain the seed currently in use. If ISAAC was not used, the frame opcodes would be the same for every client (although the opcodes would change every revision, as Jagex does and has always scrambled them every client build), and so you could trivially identify frames simply by listening to traffic.

RSA is used to encrypt sensitive parts of the login data (username, password, ISAAC seed, more in newer clients) because with a decent key size it is effectively unbreakable; however, as Mopman mentioned, it is fairly slow, so you don’t want to be doing this for every frame exchanged between the client and server.
