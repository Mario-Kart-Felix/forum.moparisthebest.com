mod_taharok August 2, 2016, 2:36pm #5

You should research state systems. Although I don’t really condone providing any sort of server help, in terms of a game which works over a chat system (such as a command on a private server or an IRC message), you need to have some sort of state machine which remembers the progress of a game for each player playing. Each command will test the state machine to see if its compatible with that stage, then update it and provide the user with information. You also need to remember to cleanup after yourself, as these types of games can easily be designed to never cleanup unfinished or finished games, etcetera.
