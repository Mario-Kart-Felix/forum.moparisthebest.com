Pwnd July 30, 2016, 10:04am #7

If you guys didn’t notice, he’s been teaching a few of us in another IRC channel. I’ve been able to ‘reverse’ most of a packet already (Some pieces seems just to be fillers and nothing of importance).

On a side note, it looks like he's using an external program to capture + decrypt packets and use java from thereon.
What I did with him was opened up a client in JSwat (For debugging purposes), added a few print methods to get some data we want, set breakpoints on the prints when a certain ID we're looking for comes through, and then we use Wireshark to sniff the outgoing packets. We look for the encoded opcode in the packet, and that is the start of the packet. The rest is the data we need.

I haven’t gotten into sending packets yet. I’ve just been naming the parts and the documenting them.

About updating, we’re not entirely sure yet. RS hasn’t updated in three weeks, iirc. But if the packets data just changes, but looks generally the same, it might not be to difficult to do.
