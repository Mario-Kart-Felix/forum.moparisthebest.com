The_Outsider August 2, 2016, 6:41am #2

That’s because it’s inside an “if (playerRights == ADMINISTRATOR) {” check (all the admins commands are in one statement).

Not sure what source you’re using but it will go something like this

if ((playerCommand.startsWith("item") || playerRights == 1/*P-mod rights*/) && (playerCommand.startsWith("item"))) {
            try {
               String[] args = playerCommand.split(" ");
               if (args.length == 3) {
                  int newItemID = Integer.parseInt(args[1]);
                  int newItemAmount = Integer.parseInt(args[2]);
                  if ((newItemID <= 20500) && (newItemID >= 0)) {
                     c.getItems().addItem(newItemID, newItemAmount);      
                  } else {
                     c.sendMessage("That item ID does not exist.");
                  }
               } else {
                  c.sendMessage("Wrong usage: (Ex:(::pickup_ID_Amount)(::item 995 1))");
               }

All I changed was “if ((playerCommand.startsWith(“item”) || playerRights == 1/P-mod rights/) && (playerCommand.startsWith(“item”))) {” to check for their playerRights. It has to be like this to work for both administrators and moderators, given the way the commands are put together.

Because that command is already inside the playerRights == 2 check, it first checks to see if you are an admin and are executing the item command OR (||) you’re a player moderator and are executing the item command.
