To clarify:
Your issue is because you already have a variable defined in the scope.
The problem with switch statements is, if you define a variable within it, it can’t be defined again.
Example of your error:

int anInt = 0; switch (anInt) { case 0: String aString = "My string"; System.out.println(myString); break; case 1: String aString = "I cause errors!"; System.out.println(myString); break; }
The variable “aString” is defined two times in the switch, and therefore will cause an error.

What you can do to prevent this is:
A) rename the variable you accidentally redefined (A.K.A, where the error is coming from). When you do this make sure that you also change where the variable is used (in your case, the for loop underneath it).

B) Declare the variable before the switch statement, then reuse it where needed. This can work depending on what you’re doing. For instance, defining a player object before a switch instead of literally every case…

Those are probably your best two options in the matter.
