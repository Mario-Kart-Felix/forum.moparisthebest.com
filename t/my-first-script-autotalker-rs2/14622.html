Weibs July 29, 2016, 10:02pm #4

Ok, a good typing function, taken from SRL, would be SendText2(Text: String): Boolean;
This function will type the keys with random waits inbetween, and will sometimes even make mistakes. If the function does make a mistake, it will return true, but you don’t really need to do anything with that.

Heres an example of how to have the script choose randomly one of the messages and type using SendText2;

Say you have 4 messages.

[scar]case random(4) of
0 : SendText2(msg1);
1 : SendText2(msg2);
2 : SendText2(msg3);
3 : SendText2(msg4);
end;[/scar]

That would choose a random message out of the selection. The only things you’d need to change would be the number within random(). Put it as the number of messages you have. Also, if you have 5 messages, you’d need to add another line to the case…of statement. Like this…
[scar] 4 : SendText2(msg5);[/scar] I think you get the pattern. If you have any other questions, just reply back.

One more thing, to use that case statement, you could put it in a procedure like this…
[scar]procedure RandomMessage;
begin
case random(4) of
0 : SendText2(msg1);
1 : SendText2(msg2);
2 : SendText2(msg3);
3 : SendText2(msg4);
end;
end;[/scar]
And then you could call it in the main loop or where ever you’re calling the typing procedure, but you’d have to add a wait after it so that it doesn’t type one message right after the other.

Hope this helps,

-Weibs.
