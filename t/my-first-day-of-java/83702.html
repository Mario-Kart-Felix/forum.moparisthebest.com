That’s really a matter of opinion. I believe that’s fine if you have it all on a single line like this:

if(condition) doThis();

That is fine, because it is very easy to see that only one thing will be carried out should that condition return true.

However, something like this isn’t so good:

if(condition) doThis();
If the statement is set out like that, it should have braces. Especially if you develop code that is going to be added to by multiple people.
What happens if someone wants to add something to be carried out if that condition returns true? Something like this could take place:

if(conditon) doThis(); doStuff();
The developer could easily think that doStuff will only be called if condition returns true, but there are no braces there, meaning it will get called in all situations. That’s where programs can become bug prone if they’re evolving over time.

Oh, and with regards to nesting if statements, it doesn’t really matter. Again, it’s a matter of opinion.
This:

if(condition) { if(condition) { doThis(); } }

is no different to this:

if(condition && condition) { doThis(); }
However, nesting if statements instead of putting all conditions into a single if statement can allow more flexibility depending on what you’re doing.

I’d prefer something like this when working with string stuff:

if(str.length() > 2) { if(str.contains("@")) { System.out.println("String could be an email address"); } else { System.out.println("String is not an email address"); } } else { System.out.println("This string is too short"); }

instead of this:

if(str.length() > 2 && str.contains("@")) { System.out.println("String could be an email address"); } else if(!str.contains("@")) { System.out.println("String is not an email address"); } else { System.out.println("This string is too short"); }
