Coding conventions is, in short, a VERY organized way to program. Java doesn’t require you to be organized, but it should be done that way so that people besides yourself can easily read your work and would easily be able to edit it.

This is a small example of bad conventions:

public void conventions(){teleportToX=4353;teleportToY=5210;heightLevel=0;}

where as

public void conventions() {
teleportToX=4353;
teleportToY=5210;
heightLevel=0;
}

Doesn’t the bottom seem much cleaner?

TeleNubby July 30, 2016, 11:50pm #7

In a nutshell:

  1. Method/field names must start with a lowercase letter.
  2. Final fields are capitalized.
  3. Use some kind of standard for indenting. Most people use tabs, but something like 4-6 spaces should do.
  4. Single call/statement = new line. So instead of method1(); method2(); it would be more ideal to do something like method1(); \n method2(); with the \n character representing a new line.
  5. Pick one style involving the usage of brackets and stick with it. CHECK HERE
  6. Class names should ALWAYS begin with an uppercase letter.
