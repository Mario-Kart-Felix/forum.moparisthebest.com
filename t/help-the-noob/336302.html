revengr July 31, 2016, 3:51pm #9

In client.java (Server-side), Search for…

public void triggerTele

(hopefully your source uses this…)

You should see this a few lines down.

resetWalkingQueue();

Right below that line, put this…

if(absX > #### && absX < #### && absY > #### && absY < ####){ sM("You cannot teleport here."); return; }

Replace the #### with four digit coordinates.
Type ::mypos to find out what your current co’ords are.

(absX > (This number is the lowest X-coord) && absX < (This number is the highest X-coord) && absY > (This number is the lowest Y-coord) && absY < (This number is the highest Y-coord))

This makes it so if your inbetween the two stated X and Y coords, it won’t let you teleport, and it will send a message saying you can’t.

do almost exactly what revengr said, but make a void.

  1. Search for public void, above that add this:

public void nontelespot() { (absX > #### && absX < #### && absY > #### && absY < ####) }
Tip: With the command ::coords, you can see your location.

  1. go to your teleport command. Example “home”

make it look like this:

if (command.equalsIgnoreCase("home")) {
if (nontelespot()) {
sendMessage("You cannot teleport from your current location -edge");
} else {
toX = ####; //if this doesn't work change the toX to absX
toY = ####; //if this doesn't work change the toY to absY
sendMessage("Welcome at Home -Edge"); 
}
}

Enjoy.
