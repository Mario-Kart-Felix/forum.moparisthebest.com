Runescape Reflection Explorer v1.0

Basically, we can use this handy dandy tool to reflect all the fields in the client at runtime. So, we can watch everything unfold as the client runs. By doing this, we can easily find out what values in the client do what, or find new fields which were never thought of before.

Features:

[ul][li]Field filtering by value, name or type[/li]
[li]Ability to compare fields with previous load, to find differences[/li]
[li]Ability to rename fields[/li]
[li]Can remove fields from the output, to remove “spam fields”[/li]
[li]Easy to use interface keeps your work productive and easy[/li]
[li]Can grab every index of single, and multidimensional primitive type arrays[/li][/ul]

Known issues:

[ul][li]Currently, it can’t reflect non static fields. The problem is, I realize I must create a new instance, but what am I to do when the class does not have a no-arg constructor to it?[/li]
[li]Some fields duplicate themselves in the output. There is only a few to my knowledge, and I have no idea why. It is more of a cosmetic issue than a program issue. I am working on it however.[/li][/ul]

Screenshot:

Other info:

This is, to my knowledge, the second reflection explorer ever made. The first one was made by Ruler, and, to my knowledge, could only be used on the “client” class. It is also nexus dependent.

This explorer features a built in client loader. Now, I realize I will be criticized for this, so let me explain it. The loader sucks. Using this loader could very well get you banned. Treat this as a warning, PLEASE. I recommend using a throw away account with any cheating tool. On top of this, I am using a URL loader, but the program needs the runescape jar in the current directory. It is stupid, I know. Basically, the program parses the class names from the jar, then uses them for loading classes via the web. I will be changing it (probably tomorrow) to load from the jar.

Finding many fields with this tool is easy. For example, with the latest jar, the Login Index can easily be found by loading all the fields at the login screen and searching for the value of “10”. From our results, we determine that the Login Index is ua.b.

I had a look at the code, and while it is good for looking at static variables in the runescape client, I decided to go ahead and write a generic one that will work for any class. This is command line only, but feel free to write a fancy GUI for it. I also haven’t implemented array support yet as it seems like something to be done at the users request in a GUI instead of printing lots of arrays out.

package com.cherokee.utils;

import java.lang.reflect.Field;
import java.lang.reflect.Modifier;

public class RuntimeViewer {

    public RuntimeViewer(Object o) {
        getFields(o);
    }

    private void getFields(Object o) {
        getFields(o.getClass().getName(), o.getClass(), o, "");
    }

    private void getFields(String name, Class klass, Object o, String tabs) {
        Field[] all = klass.getDeclaredFields();
        if(all.length == 0)
               System.out.println(tabs +"NO FIELDS FOUND!");
        for (Field f : all) {
            boolean isStatic = Modifier.isStatic(f.getModifiers());
            //if o is static, we can only look at static members of this class
            if(o == null && !isStatic)
                continue;
            System.out.print(tabs + (isStatic ? "static "+ klass.getName() : name)  + "." + f.getName()+" Type: "+f.getType().getName());
            Object val = getObject(f, o);
            // assures the value is primitive or a class provided by sun before displaying the value
            if (f.getType().isPrimitive() || (f.getType().getPackage() != null && f.getType().getPackage().getImplementationVendor() != null)){
                System.out.println(" Value: " + ((val == null) ? "null" : val.toString()));
            // if its another type, recurse on it to display it's values
            } else{
                System.out.println(" - ");
                  getFields(f.getName(), f.getType(), val, tabs+"\t");
            }
        }
    }

    private Object getObject(Field f, Object parent){
        boolean accessible = f.isAccessible();
        if (!accessible)
            f.setAccessible(true);

        Object val;
        try {
            val = f.get(parent);
        } catch (Exception e) {
            //should never get here
            e.printStackTrace();
            val = null;
        }

        if (!accessible)
            f.setAccessible(false);

        return val;
    }
}
