Telenim July 31, 2016, 8:43am #4

Well, I find this in 90% of the sources if you click an item or an object that currently has no case to it, you can look in your run CMD and find it says something like this

[Telenim]: Clicking Action Button: 4140 (Varrock Teleport) - Action accepted
But if it has a case to that ID, then you will just have to search for an empty case or find what the current action is such as, you wanted to find an object (portal) to teleport you to Pest Control, but that ID was already being used, and you know that it sends a message
sendMessage("Welcome to pest control");
you could simply search for that in your Client Class, and it would show up with the whole case, and you will be able to edit that.
