Protosstribe August 4, 2016, 6:40am #6

I wish I was further than I am. But that’s life.

Currently I can pretty much log in (and log out), walk and watch bots walk around. I had following, but had to temporarily remove it in order to make it fit in with the architecture in a better way. But, I think I will soon have a primitive version of combat implemented. The Domain Model for it is pretty much finished. Other than in-game features my server supports versions 201, 202 and 203 (204), at the very same time.

One large problem I had earlier (which also took quite a lot of my time) was that I could only display 255 players at once (if you remember my last topic?). But I realized that this is a restriction in (or a design decision made for) the client itself, rather than a bug. It is probably so that computers with slow connections won’t get overwhelmed with data (or something along those lines).

As my goal with this server is more than just an empty source, and I got limited time to work on it, it will take some time. But I do have a website for it. There’s not a lot on it so far, but you could check it out once in a while if you wish. I might have added something new to it by then. You never know. The website is http://www.macroing.org/Projects/RSC4J .
