I’m making a new base from scratch (I don’t care if you think it’s stupid, it’s none of your buisiness)… I’m tired of rscd/demonik, and I just don’t like it (not debating about this, or elaborating)…

Key features:
Regions - Players in regions will update together, and only loop through them, instead of everyone in the world

Caching - Going to be storing more data in ram, for things like bans, jails, mutes, ect

Quest System - Self explanatory…

Multiple Worlds - I know it’s already in rscd, but I’m adding it and having it so it shows friends online in diff worlds, ect

“Script” sysem - For things like additional features, like npcs, object, ect able to update without server restart

Auto Updating - Going to be using checksums to check files (should also help fight bots)

Comments:
Going to make it so you can easily customize it… Also, removing xml files. Putting settings into ini files. Npcs, items, ect into MySQL.

Going to make it bug free the first time around (hopefully)

Using Netty for the networking bit.

ATM I’m calling it RSCBase, might change later.

I am releasing this once I feel it’s ready.

Progress:
World Management - Done. You can add handlers by simply putting the files in the packethandler folder.
Server - Everything done but content
Engine - Done
Connections - Done
Tasks - Done
World - Done
Regions - Done
Basic Content - Foundation Laid
Quest System - Pretty much done, still need to add sine things
AutoUpdating - Done
Database Management - Done
Database Connector - Done

Will add more to the above once I get the base of both done.

Total time spent: Over 5 hour 38 minutes
Note that includes planning, and any time on it, even if it doesn’t has my undivided attention (aka me talking to ppl on vent/msn/irc)
