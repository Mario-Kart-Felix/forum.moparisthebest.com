[font=open sans][size=16px]What is the best way to replace the player’s body with an animal NPC?[/size][/font]
[font=open sans][size=16px]combatSprite, combatModel, walkModel, Sprites1, and Sprites2 from the npcdef would need to be set for the player.[/size][/font]

Not looking for actual code, just needing ideas of how it would be best implemented. My thoughts are to set the player to cease displaying body sprites in the client and replace them with animal body sprites temporarily on demand using a command during the proof of concept phase. Equipped items will not be permitted until far later so skip that. The walkModel, combatSprite, combatModel, Sprites1, and Sprites2 objects from the npcdef need to be integrated into the player. Later “updateAppearanceWindow()” can be used. It was first accomplished in 2006 on a RS2 317 private server by a friend that I knew replacing the player’s 3D model with an NPC’s 3D model.

Idea 1: Find where the client points to the player walking. Have an if statement asking if the player is human or animal. If animal, determine animal type with another set of else if statements. Copy walkModel sprite sequence variation references for each else if. Repeat for combatSprite and combatModel. For Sprites1 and Sprites2 (optional), have the server’s clientupdater send “updatePlayerApperances§;” after setting them for the player after ceasing to use the previously loaded body sprite information from the database. If the player wants to revert to human form, it would need to reattach the previously loaded body sprite information from the database.

Any better ways of going about it?

cleako:

It was first accomplished in 2006 on a RS2 317 private server by a friend that I knew replacing the player’s 3D model with an NPC’s 3D model.

RS2 has always had support for it, all you had to do is to send transform mask along with the npc type index which is all part of the player update packet.

NPC and Player drawing are almost identical in client, it is rather easy to do.
Modify the player update packet so it also sends the npc index to transform the player to(-1 if none) and store that in the Mob class in client. Next, check where players are drawn and then add if statement check for the transform id if its higher than -1 then instead of using the players sprites, load the sprites from NPCDefinitions.

Good luck.

EDIT: I just checked the RS2 client and that is exactly how it is done in RS2…

RSCShadow August 4, 2016, 1:58pm #19

When I made my Node server, it was really trivial to make the player appear to be an NPC. I did it by accident at first. Pretty sure you can just dump the animation cache for name -> ID mapping and just send any NPC animation ID instead of armour ID. You can even use an unmodified client source and a lot of sprites look perfectly acceptable with the default width/height of the player.

        while (player.updates.playerAppearances.length) {
            updatee = player.updates.playerAppearances.shift();

            packet.addShort(updatee.index);
            packet.addByte(updateTypes.APPEARANCE);

            // The animation index is incremented by one every time the player's
            // appearance is updated.
            packet.addShort(updatee.animationIndex);

            packet.addLong(updatee.encodedUsername);

            // The length of the following animation ID sequence (which, for
            // players, should always be twelve). The animation ID sequence
            // covers animation identifiers for the player's head, body and legs
            // and any accompanying wielded items.
            packet.addByte(updatee.sprites.length);

            for (i = 0; i < updatee.sprites.length; i += 1) {
                packet.addByte(updatee.sprites[i]);
            }

            // The hair, top, trouser and skin colours, respectively.
            packet.addByte(updatee.appearance.hairColour);
            packet.addByte(updatee.appearance.topColour);
            packet.addByte(updatee.appearance.legColour);
            packet.addByte(updatee.appearance.skinColour);

            // The player's combat level.
            packet.addByte(3);

            // Whether or not the player is skulled.
            packet.addByte(0);
        }
