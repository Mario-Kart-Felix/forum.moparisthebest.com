Ah, the idiosyncrasies of different IRC daemons. This is basically something you have to put up with. It will make your buffer parsing code very messy, but tough shit, basically. There are other things like this, for example servers are SUPPOSED to terminate messages with CRLF, some just use LF. Just have to work around it basically.

http://www.irchelp.org/irchelp/rfc/chapter4.html#c4_2_5

NAMES message. You’ll want to make sure you’ve joined first, if you NAMES before you join you’ll not get people who are set +i.
Remember to keep your local copy up to date with joins parts and quits.
