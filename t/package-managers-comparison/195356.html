[quote=“Moparisthebest, post:13, topic:195356”]Ok, I have fixed the few errors, made the comparisons page validate correctly, and changed the banner image.

Onto the ABS/makepkg thing, is it the equivalent of dpkg’s source packages? or is it the equivalent to checkinstall? which when executed instead of the ‘make install’ command builds a deb or rpm package?[/quote]

More similar to the latter. Basically, with makepkg you create a “PKGBUILD” file which describes how the source is to be compiled and the information on the package, such as name, dependencies, what it provides (ie which dependencies it fulfills). Then, you run makepkg on this PKGBUILD to create a binary archlinux package which can be installed via pacman. It’s the package creation process, it’s just really really simple so that you can use it to create packages whenever you need to install something from source.

On top of that, Arch provides the AUR, which is a repo much like the pacman repos but for source packages (PKGBUILDs) - fully community provided.

If you don’t want to faff around with makepkg, you can get a pacman front-end called “yaourt” which offers a one-command install from the AUR/source.
