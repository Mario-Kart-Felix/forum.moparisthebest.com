Supah_Flyizzle August 4, 2016, 1:30am #77

alright, i’m definitely not reading every post in this thread. i’ll just throw some words onto the internet and hope they resonate. please note that i’m not perfect and my ideas could probably be improved, so if you want to improve upon my idea go ahead.

mopar, as a rs development forum, suffers from too much fragmentation and inconsistency. i think the “rswiki” on mitb is absolute shit (besides the cache format, coordinate system information, ondemand, jaggrab documentation, etc.). not only that but nobody knows about it existing, which is probably because it’s not good enough to really advertise, which in turn makes it stagnant. it would benefit the site if we had a forum where people can post and have their information actually be seen for things like the protocol, formats, whatever. basically the informative threads section on rune-server but executed in a way that separates facts, concepts/theories, etc. using thread tags or something. the reason the wiki just doesn’t work is because wikis are generally just hard to maintain and write content for, especially if the userbase is used to using bbcode. learning how to use the new markup in a creative and efficient way is just too much work. if people want to have their information added or if they have a correction to the information in the main post of the thread, they can post a response and either the op or mod can make the appropriate changes. i imagine forum threads to be more accessible and enjoyable to read than wiki articles. it’s all part of the experience that way, and that’s what would actually ADD to the value of mopar. having all these separate sites and shit only takes away from the main site and fragments the experience and only makes it hard to find information that could potentially make mopar members more intelligent by allowing them (by giving them easy access) to actually learn more than tutorials and downloads so that they can write their own shit and gain a better understanding of the client and server design.

object/item/npc databases is a good idea but we need to do it better than the other sites, which is entirely do-able. hell, i’d love to be a part of making that a reality. i guarantee that if i were to help create the databases i could improve on the basic ideas they’ve already implemented on other sites. i’d rather contribute to mopar than make my own site. ruby, if you want to do something, i’d like to help you.

it seems like there’s a couple changes going on and that’s actually a good thing, it shows that the site isn’t being totally neglected.

also i think mopar’s server section layout sucks. changing it would probably be beneficial. i can’t see how keeping it the same would really benefit the quality of the forum in the long-run, which is why i think it should change. anything, as long as it changes. doesn’t have to be even remotely close to my idea but as long as it’s an improvement.

runescape development server tutorials featured server downloads bases remakes archive (dead links, etc.) etc. help (general, server) development teams (if inactive for 2 weeks or so, move the thread to archive) archive server/client discussion (theories, concepts, information, etc.) runescape client development tutorials featured downloads featured archive (dead links, etc.) help models requests runescape classic development client tutorials downloads discussion help server tutorials downloads help

just something to benefit the section and make it easier to navigate and categorize information instead of having a clusterlove of old, dead servers and downloads with dead links. we also would need a rating system so that people can actually give ratings to servers. maybe we can have a system where we can rate specific aspects of servers and categorize them. when i knew nothing about servers and wanted to get into working on them i was just like “well what server is the best? how do i know without spending hours and hours of time that i don’t have trying to figure out what server is the best? what’s the difference between servers?” etc.

as for an “rsps sponsor” love no, that makes the whole site biased and partial to one server. that’s just a bad idea. funding isn’t the problem.

that’s all i got, sorry for the poor grammar and lack of capitalization. just kind of got a stream of consciousness thing happening in this post.
