Well, I thought for AGES about how the webclient JAR is saved in the temporary internet files, and I discovered, if I found a way to empty the temporary internet files every time the page is loaded, it would delete the jar from the users temp internet files, and it could be loaded again, No errors as of yet.

To do this, make sure your play page is .php, and make sure the webserver you are using supports php.
You can test this on localhost if you have xampp or php installed.

Here is the code, put it at the very top of your page

<? header("Pragma: no-cache");
header("Cache: no-cache"); ?>

This clears all files from the cache, now I fixed it so it works every time you refresh the page!
