zstars:

If it was easy, one would think there would already be detailed public documentation about the current runescape protocol and a working program to connect to the server without using a modified client. But apparently, there is not.

The basics are almost common knowledge among the people who work with the client. Once you start looking at the code, the protocol is fairly apparent. It’s a fair bet that anyone in our little group who would want to make a packet bot already knows a good deal and could figure out the rest, so there hasn’t really been a need for documentation. I’ll admit that this doesn’t account for people like the OP, but nothing defines this community quite so much as lack of spontaneous motivation - and let’s just say that a dedicated, skilled newcomer is a pleasant surprise.

Since a lot of us are holdovers from Classic or at least early RS2, we’ve seen packet bots before and we’ve watched Jagex build security designed to prevent them. It’s not terribly complicated to start a connection. It is complicated to build a game state and respond to it correctly without using the client. It would take even more work to do so undetectably. Then you have the updates to contend with if you hope to use your system longer than a week or two at the outside. That’s why we don’t bother. But if you’ve got a different motivation - like the OP - and you’re not just looking for an efficient cheating system at minimum effort, then a packet bot would be cool. I’d like to see one started, and even more to see one finished.
