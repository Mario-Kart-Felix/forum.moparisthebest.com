IBOT made by ruler. It includes many contributions from leading community members. Their will be a “about” page to list contributions. It started out after much reading of how java events are managed. It began as UBI (Universal Bot Interface) which allowed any bot to send mouse clicks, mouse movements, keys to a applet. It was first ported to SRL (a include for SCAR) by GenoDemon. Next it was ported to a client made by Cruel who using windows api got the applet to be off screen and aquire a bitmap of it thru the windows api.

It can trick clients into thinking they have “focus”, ect. It can also block events. This allows multiple applets to run on the same OS and be tricked into thinking they have focus and they are recieving mouse events.

Color Recogntion
-Everything scar can do
-Advance methods that can return arrays of all matches
-Direct access to the pixel arrays
-OCR near complete
-Extendable

Threads
-Bot GUI
-Mouse Controller
-Random thread
-Script thread

Advantages
-faster than scar
-Long Lasting
-low up keep
-less dectable
-Very powerful and well thoughtout gui
-Pixel explorer to speed script development
-less updating if any at all

Disadvantages
-not as efficent as a client edit in theory
-scripts take more intelligence to produce
-Few currently in existance
