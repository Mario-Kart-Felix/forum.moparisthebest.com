With the recent release of Old Runescape the developers have been posting a ton of photos of their work. I stumbled across the thread by Leanbow and also the archive of Andrew Gowers notes from 2001. In his notes he outlines a very primitive scripting system which wouldn’t be very hard to implement to be honest.

Notes:
One server cycle is 0.6 seconds, so 100 cycles = 1 minute
Probabilities range from 0=impossible to 256=certain
[hr]
Misc commands
[table]
[tr]
[td]thinkbubble();[/td]
[td]Should only be called if the player has just triggered a ‘use’ script. Displays a thinkbubble above the players head which contains the object they are currently using. For instance this is used to display the axe above your head when you chop a tree down.[/td]
[/tr]
[tr]
[td]ifmale();[/td]
[td]Test the player’s gender, useful for getting gender specific language correct in the scripts.[/td]
[/tr]
[tr]
[td]nodefault();[/td]
[td]Stops the game executing the default action when this script is complete. For example can be used to stop an object from being taken, or a spell being cast.[/td]
[/tr]
[tr]
[td]openshop(shop);[/td]
[td]Displays the specified shop dialog, and allows the player to buy and sell items. Doing so frees the npc, so no npc specific commands can be used after this point.[/td]
[/tr]
[tr]
[td]displaybalance();[/td]
[td]Displays the players current bank balance[/td]
[/tr]
[/table]
[hr]
Control flow based
[table]
[tr]
[td]Delay(delay);[/td]
[td]Delays both the script and the player for the specified number ofserver cycles[/td]
[/tr]
[tr]
[td]Pause(mindelay, maxdelay);[/td]
[td]Pauses the script for a random amount of time inbetween the specifiedminimum and maximum. The player is free to do other stuff meanwhile, useful for making trees regrow etc… Be very careful when using delay orpause, as the player may have logged out meanwhile. Normally this doesn’t matter, but don’t rely on things after it to do anything permament whichaffects the player (such as setting a player variable) as it may be lost.[/td]
[/tr]
[tr]
[td]modpause(mindelay, maxdelay);[/td]
[td]Similar to the pause command, except the total delay is modified bythe number of players in the game. If more than 60 people are playing, the delay is calculated as: delay=(delay60)/player-count;[/td]
[/tr]
[tr]
[td]ifrandom(probability);[/td]
[td]Randomly sets the condition flag to true or false. A probability of0=impossible, and 256=certain[/td]
[/tr]
[tr]
[td]jump(labelname);[/td]
[td]Jumps to the script block with the trigger ‘Label,labelname’ The otherscript will not return to this one[/td]
[/tr]
[tr]
[td]fork(labelname);[/td]
[td]Starts the script block with trigger ‘Label,labelname’ running at thesame time as this one! Be careful with this command If the target block has no pause, wait, say, or npcsay commands then will behave like agosub command. (due to non-preemptive multitasking)[/td]
[/tr]
[tr]
[td]end();[/td]
[td]Terminates processing of the current script block[/td]
[/tr]
[/table]
[hr]
Chat based
[table]
[tr]
[td]mes(string);[/td]
[td]Displays the specified message at the bottom of the playerscreen[/td]
[/tr]
[tr]
[td]say(string);[/td]
[td]Makes the players character say the specified string, all nearbypeople will also see this. The command will then automatically delay the script depending on the length of the string.[/td]
[/tr]
[tr]
[td]multi2(string, label, string, label);[/td]
[td]Displays a multiple choice menu with 2 items, depending which theplayers chooses the relevant block of script will be called. Note: The player may decide not to choose either item in which case the scriptterminates[/td]
[/tr]
[tr]
[td]multi3(string, label, string, label, string, label);[/td]
[td]Displays a multiple choice menu with 3 items[/td]
[/tr]
[tr]
[td]multi4(string, label, string, label, string, label, string, label);[/td]
[td]Displays a multiple choice menu with 4 items[/td]
[/tr]
[/table]
[hr]
Movement based
[table]
[tr]
[td]changelevel(int level);[/td]
[td]Moves the player to the specified level, without changing the x and yposition. Level-codes: 0=ground floor, 1=1st floor, 2=2nd floor, 3=basement[/td]
[/tr]
[tr]
[td]changelevelup();[/td]
[td]Moves the player up one level if possible.[/td]
[/tr]
[tr]
[td]changeleveldown();[/td]
[td]Moves the player down one level if possible.[/td]
[/tr]
[/table]
[hr]
Stat based
[table]
[tr]
[td]ifstatrandom(stat, base-probability, top-probability);[/td]
[td]Like the random command but stat modified, if the stat is 0 thebase-probability is used, if the stat is 100 the top-probability is used. If the stat is inbetween then the 2 values are interpolated.[/td]
[/tr]
[tr]
[td]advancestat(stat, base, exp);[/td]
[td]Permanently increases the specified stat, and sets the new levels asthe current normal. The equation used is: stat+=base+expstat;[/td]
[/tr]
[tr]
[td]addstat(stat, constant, percent);[/td]
[td]Temporarily adds constant+(currentpercent)/100 to the 2nd playersspecified stat. Where current is the current level of the stat[/td]
[/tr]
[tr]
[td]substat(stat, constant, percent);[/td]
[td]Like addstat, except it subtracts.[/td]
[/tr]
[tr]
[td]healstat(stat, constant, percent);[/td]
[td]Like addstat, but won’t take the players stat above the normallevel[/td]
[/tr]
[tr]
[td]ifstatup(stat);[/td]
[td]Sets the condition to true if the players stat is currently above thenormal level, false otherwise[/td]
[/tr]
[tr]
[td]ifstatdown(stat);[/td]
[td]Sets the condition to true if the players stat is currently below thenormal level, false otherwise[/td]
[/tr]
[tr]
[td]ifstatabove(stat,value);[/td]
[td]Sets the condition to true if the players stat is above (but not equalto) the specified value, false otherwise[/td]
[/tr]
[tr]
[td]ifstatatleast(stat, variable, value);[/td]
[td]Sets the condition to true if the players stat is more than or equalto getvar(variable)+value, false otherwise[/td]
[/tr]
[tr]
[td]giveqp(value);[/td]
[td]Gives the player the specified number of quest points[/td]
[/tr]
[tr]
[td]ifqp(value);[/td]
[td]Sets the condition to true if the players quest points are more thanor equal to the specified value, false otherwise[/td]
[/tr]
[/table]
[hr]
Variable based
[table]
[tr]
[td]ifvar(variable, value);[/td]
[td]Sets the condition flag to true if the variable is equal to thespecified value. Sets false otherwise[/td]
[/tr]
[tr]
[td]ifvarmore(variable, value);[/td]
[td]Sets the condition flag to true if the variable is greater than (butnot equal to) the specifed value. Sets false otherwise[/td]
[/tr]
[tr][td][/td][/tr]
[tr]
[td]ifvarless(variable, value);[/td]
[td]Sets the condition flag to true if the variable is less than (but notequal to) the specifed value. Sets false otherwise[/td]
[/tr]
[tr][td][/td][/tr]
[tr]
[td]setvar(variable, value);[/td]
[td]Sets the variable to the specified value. A variable can only hold thevalue 0-127. Always try to zero variables when they are no longer needed to save memory.[/td]
[/tr]
[tr]
[td]addvar(variable, value);[/td]
[td]Adds the specified amount to the variables value. A variable can onlyhold the value 0-127. [/td]
[/tr]
[tr]
[td]subvar(variable, value);[/td]
[td]Subtracts the specified amount to the variables value. A variable canonly hold the value 0-127. [/td]
[/tr]
[tr]
[td]randomvar(value);[/td]
[td]Sets the variable ‘random’ to a random integer between 0 andvalue-1[/td]
[/tr]
[tr]
[td]addbigvar(variable, value);[/td]
[td]Add the specified value to the variable. A big-variable can holdvalues between 0-268435455[/td]
[/tr]
[tr]
[td]subbigvar(variable, value);[/td]
[td]Subtracts the specified value from the variable. A big-variable canhold values between 0-268435455[/td]
[/tr]
[tr]
[td]ifbigvarmore(variable, value);[/td]
[td]Same as ifvarmore except for big-variables[/td]
[/tr]
[/table]

Coordinate based
These commands all require a map coordinate to operate. The coordinate used can be set using the setcoord command below. If the setcoord command is not used the system will attempt to use the coordinate of the item the player is currently interacting with.
[table]
[tr]
[td]setcoord(coordinate);[/td]
[td]Sets the active coordinate to the specified absolute position. Thecoordinate can only be worked out with the aid of the runescape map editor.[/td]
[/tr]
[tr]
[td]playercoord();[/td]
[td]Sets the active coordinate to the players current position.[/td]
[/tr]
[tr]
[td]addobject(object, count, time);[/td]
[td]Adds the specified object at the active coordinate. If the object isstackable - count indicates how many to place in the stack. Time indicates how long in server cycles before the object despawns.[/td]
[/tr]
[tr]
[td]addnpc(npc);[/td]
[td]Spawns the specified npc at the active coordinate.[/td]
[/tr]
[tr]
[td]addloc(location);[/td]
[td]Adds the specified location at the active coordinate.[/td]
[/tr]
[tr]
[td]ifblocked();[/td]
[td]Sets the condition flag to true if the square indicated by theactivate coordinate is currently blocked, sets false otherwise[/td]
[/tr]
[tr]
[td]teleport();[/td]
[td]Teleports the player to the active coordinate.[/td]
[/tr]
[tr][td][/td][/tr]
[tr]
[td]showeffect(type);[/td]
[td]Displays a special effect animation of the specified type at theactive coordinate.[/td]
[/tr]
[/table]
