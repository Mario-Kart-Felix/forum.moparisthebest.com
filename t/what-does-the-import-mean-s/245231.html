[quote=“Jordiescape, post:2, topic:245231”]It imports classes from the main Java files that come from Java, here is a helpful overview that will become a friend =D.
Overview of some classes[/quote]The import keyword is not limited to classes in the prepackaged API by Sun; as it can be used to import your own packages within your own program or library.

The import keyword corresponds with the package keyword. This is how it works (I’ll show you an example and then explain):

package com.telenubby.rs2;

class ConnectionThrottler
{
       public ConnectionThrottler()
       {
              //Ignore - as this is only an example.
       }
}

The package keywords places a class into the specified package. Packages are based on the folder/setup your class is placed in. For example, if my class was located in the file …/org/runescape/cheating then I would use the package keyword like so:

package org.runescape.cheating;

Generally, the first sub-name in the package name would be something very general, such as: org, com, net, gov, info. ‘Com’ would represent a commercial program, and ‘net’ would represent something related to networking. The second sub-name usually involves something to do with the creator(s) of the program, and the third usually involes something to do with the program itself (e.g. general field or related topic).

The import keyword is used like so:

import java.io.*:

What this does is import ALL classes (because of the character ‘*’) in the java.io package (which would be in the folder …/java/io/). The Java compiler automatically includes a path to Sun’s prepackaged API in the classpath, which means its there for you to use and you dont have to worry about setting it up. This also limits you from creating classes in these predefined packages.

Another example would be:

import com.telenubby.rs2.ConnectionThrottler;

Instead of importing all classes for your use in the package provided, you are only importing the class named ConnectionThrottler. This way, you can import indivual classes rather than entire packages. I perfer this over the first option I mentioned, as it helps you learn what you are using.

As Jordiescape said, a list of the packages Sun has prepackaged for you can be found HERE or on your local harddrive.

Good luck.
