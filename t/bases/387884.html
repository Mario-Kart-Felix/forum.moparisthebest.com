They all have their pros and cons.

PI: Very customizable, packaged, but not popular on the stability side, not helping the private server community move forward as quick as it could.

Hyperion: A stable and quite advanced source. Can be confusing if you don’t know what you’re doing, but respected quite a lot due to it being a step forward.

Winterlove: The base of probably 98% of every server out there. The first private server ever written, filled with bugs and not very advanced in terms of how servers are now. You couldn’t even eat food without a weapon in most of them.

Delta: No idea. Never used. Quite frowned upon as they have gained a bad reputation, stability wise I’m guessing, but chosen still to this day by newcomers, as they are quite easy to understand.

Shards: Again, another stable and respected source. Apparently easy to understand, and quite blank so good for newcomers and veterans alike.

Can’t think of anymore.

[quote=“Rash, post:2, topic:387884”][quote author=xknightpk link=topic=483678.msg3543596#msg3543596 date=1301021575]

The way they are packaged, stability, etc.[/quote]

Pretty much this. It really just depends on the programmer’s style. Some people want to have everything in as few classes as possible(like winterLove servers), and some like to make things as Object Oriented as possible(like Hyperion).

Stability is also another thing. winterLove servers use standard, blocking IO which is acceptable for small network applications, but for scalability, Non-blocking IO is recommended. In fact in Java 7, the standard IO will be build upon NIO. You’ll still be able to have blocking networking by doing socketChannel.configureBlocking(true);

Performance is generally the deciding factor. For example, traditional winterLove servers create a new thread for every client. This is a problem because the server will become more laggy, and more unstable as more clients are connected. With Hyperion for example, there are only a few threads, and “events” handle everything else. That way, instead of updating every player, every tick of the server, it only updates players that require an update, and so on and so forth.
