[code]package com.io9Labs.simpleshit;

import java.util.Scanner;

/**

  • SimpleDivisibilityChecker, as the title suggests, is a simple

  • divisibility checker written in Java =P
    /
    public class SimpleDivisibilityChecker {
    /    *

    • The main() method which is called by the JVM when the application starts

    *@param args The arguments passed by the JVM from the command line
    */
    public static void main(String[] args) {
    checkDivisibility();
    }

    /**

    • Checks divisibility lol. =D
      */
      public static void checkDivisibility() {
      getValues();
      if((checkWhat % checkBy) > 0)
      System.out.println(“Sorry, but " + checkWhat + " is not fully divisible by " + checkBy + " D’=”);
      else
      System.out.println(“Yes! " + checkWhat + " is divisible by " + checkBy + " with a remainder of 0! =D”);
      }

    /**

    • lels gets value using a java.util.Scanner object
      */
      public static void getValues() {

      Scanner getter = new Scanner(System.in);

      System.out.print("Please enter what number to check for divisibility (Whole numbers only please :]): ");
      checkWhat = getter.nextInt();

      System.out.print("\n\n\n");
      System.out.print(“Check if number is divisible by: “);
      checkBy = getter.nextInt();
      System.out.print(”\n”);
      }

    static int checkWhat;
    static int checkBy;
    }[/code]

moparisthebest July 30, 2016, 4:04am #2

The only comment I will make on your code is:

static int checkWhat;
static int checkBy;

That by default sets those variables to package access, which is very very rarely used in Java. In the case of the program you wrote they should be:

private static int checkWhat;
private static int checkBy;

It does not matter in your current code, but in the future when you start making bigger and bigger programs you will want to properly encapsulate your variables.

Otherwise the code doesn’t look too bad for a start, perhaps check that the input is in fact valid and I don’t enter a letter or something.
