RSCShadow August 4, 2016, 8:56am #3

I’d recommend looking into SMACSS and OOCSS (you can find a lot of videos on YouTube discussing this) if you want to organize your CSS. You have selectors that just select all the nav and footer elements and apply styles to them, as if there could only be one per page.

* { padding: 0; margin: 0; color: #7a8888; transition: all 0.5s ease; font-family: 'Prototype', Arial, sans-serif; }This is also not a very good idea. If you need to reset element styles to look more consistent, I’d recommend a reset library or Normalize.css. You definitely shouldn’t be setting transitions like that either, it’s unnecessary, could cause issues if you didn’t want transitions in some areas and is most likely pretty inefficient.

I do think the site looks good but you should definitely look into fixing these issues.

EDIT: You may find a grid framework really helpful. I really like Yahoo’s PureCSS grids.
