I would like to address this once quickly. There is one thing you have to learn in computer science, the people who write compilers for a living are far smarter than you have ever thought about being. It is a bad (and wrong) assumption that you are smarter than the compiler, you are not, this code being a case in point.

It is NOT faster to ‘go backwards because you only access players.length once’, because the compiler (again, way smarter than you) caches what players.length returns (it knows it is static and doesn’t change, so it only calls it once) and therefore calls it only once.

If you really want to optimize your code and save 5ms over 100,000 iterations, when you use n++ like that it is slower than ++n because n++ has to copy the value to a new register, use the old register for the return value, increment the new one, and then update any pointers. Where as ++n just increments the value in place. However, this is a case in which the compiler will probably optimize this for you again.
