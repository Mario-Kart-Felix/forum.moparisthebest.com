My server has recently been going offline a lot, and the biggest issue is that it isn’t easily resolved, since the command line for the server shows no signs of anything happening. There are no errors being thrown, it just stops updating.

I looked into this and found out this is called a Deadlock, which is caused when two synchronized threads are waiting for the other to finish. Since they are waiting on eachother, they lock up and the server stops responding.

You can use jconsole to detect deadlocks in a java application and figure out what threads are causing it, but I can’t get it to connect to the server.
Does anyone know how to get it to connect to an RSPS, or know any other ways to detect a deadlock?
