Mopman July 31, 2016, 1:07am #10

The $PATH variable is handled through /etc/profile, so all you have to do is open a new terminal (or run source /etc/profile" if you don’t want to reopen your terminal at the current time) - not log in and log back out. If brand new terminals aren’t getting the proper $PATH, you have set your terminal emulator up incorrectly (certain terms don’t act as login shells by default, cough gnome terminal).

The /opt thing is used for programs which do not adhere to the standard filesystem hierarchy and wouldn’t fit with the rest of the system if they were put there. KDE was once in /opt, although if I remember correctly the issues Arch had with the hierarchy were fixed and it is now in the main filesystem.

The symlink issue is again, a philosophy thing. Personally that sounds like a horrible hack to me, but I can see where some people might see otherwise. Same with the dependency thing, I’d be pissed if pacman -S kde pulled in the xorg metapackage Different strokes for different folks I guess.
