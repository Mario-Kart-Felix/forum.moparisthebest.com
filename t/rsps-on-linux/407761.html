zyle1992 August 1, 2016, 1:42pm #6

As mitb said, most clients do something like this:

public String findcachedir() {
    return "C:/.file_store_32";
}

What happens if you don’t have a C: drive? Or perhaps you do, but don’t have permission to write to it? Then it can’t do anything. However, I posted a “fix” to this a while back in the snippets section that makes it universal

public String findcachedir() {
    return String.format("%s/%s/", System.getProperty("user.home"), "your cache name here");
}

I’m not sure how mitb did it in the MoparScape clients, but I’m sure it’s something similar.

ExtremeX-Scape August 1, 2016, 1:43pm #10 
	public static String findcachedir() {
		final boolean exists = (new File(System.getProperty("user.home") + "/Cache/")).exists();
		if (exists) {
			return System.getProperty("user.home") + "/Cache/";
		} else {
			final File f = new File(System.getProperty("user.home") + "/Cache/");
			f.mkdir();
			return System.getProperty("user.home") + "/Cache/";
		}
	}
