Wintercanon August 1, 2016, 2:54pm #13

Also with a database you can easily access the data remotely, such as through a website, for account management and such. But there is some overheard to connecting to the database and querying it. It becomes more efficient when as Taharok said you have a lot of entries. The problem with reading and writing to files is the disk will block if it’s busy. A serious fault in your code can sometimes overwork the hard drive or waste space by writing unneeded data to files.

In my opinion your best bet is to serialise the data to a binary file. Simple convert the username, password and position to a byte array and write it to the file. Just make sure when you read the file, you read it in the same order as you wrote it. Writing to text files isn’t a good idea. Text files can get corrupt and you can have encoding issues. They will work but it is generally faster and safer to create your own data format.
