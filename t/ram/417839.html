It’ll make it faster, yes but it wouldn’t make THAT much of a difference. If you really want a quicker and more snappier PC you’ll probably need to upgrade both the RAM and the CPU (Unless you already have a beast CPU :D)

You have to pay attention to the RAM timings (them numbers that look like 7-7-7-21). This is how fast the RAM can process data, basically. (They refer to how many CPU cycles (ticks) it takes for the RAM to ready the data). Also, you’ll need to look at the maximum speed of the RAM (e.g: 1600mhz); you’re not going to get this speed just by putting them into your computer (They’d most probably run at 1333mhz). That speed is the maximum it’s able to run at which means you’ll need to overclock your RAM to get it to run that fast and the more RAM the harder it is to overclock. Also, the more you overclock the more unstable the system is; to make it more stable you’ll have to increase the RAM timings (therefore making it slower).

So, the point is, getting fuck loads of RAM isn’t always the best thing. I’d say anywhere from 4-6GB is enough. Sorry I’m not very good at explaining things.

Graham August 1, 2016, 3:35pm #17
zyle1992:

Anything over 4 is useless on a 32-bit system.

That isn’t quite true: some 32-bit processors support something called physical address extension (PAE) which allows you to address 64 gigabytes of physical memory, however, each individual program can still only use up to 4 gigabytes of virtual memory.

Nevertheless, PAE is a bit of a hack and getting a 64-bit system is much better in many aspects (not just from the increased amount of address space - instructions are encoded more efficiently, there are more registers, compilers can assume you have features like SSE and thus output faster code by default, etc.)
