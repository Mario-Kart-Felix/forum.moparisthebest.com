RandQm:

Save the time when the donator has claimed his donator status in their character file and upon login check if 30 days has passed since that.

This.

You can use Java’s Calendar to handle all of it. It has a before() and a after() function that can determine whether or not if a date is before/after a time…

http://docs.oracle.com/javase/7/docs/api/java/util/Calendar.html

Easiest method to accomplish this would be to use the SimpleDateFormat to get the Month, Day, Year, Hour, Minute, and Second from the Calendar, and save it +30 days (There’s a function to add days too!) from when they redeemed it.
Then each time they login you could have it check if the date is past their donation time. However, if the player doesn’t logout in a few days, that would mean they didn’t lose their powers. So you’ll have to account for that.

Also. PLEASE do not call it donating if they’re paying monthly for “Extra features”. That is not donating, that is a subscription.
