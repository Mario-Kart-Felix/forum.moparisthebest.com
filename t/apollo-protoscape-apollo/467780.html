This is my Apollo derivative of my project ProtoScape. I removed the cache’s because that would make the server files massive. I had a lot of plans for this and to make it something but I could never find enough motivation to finish it and nobody else really wanted to help either.

What needs to be done:
I was almost done implementing Hyperion’s finished region system (not the one the standard Hyperion comes with, used google i think is what it was called. Graham really built upon that). People kept telling me to also just use google’s combat since it was the best so I scrapped mine and fixed theirs up. Melee is almost perfect and projectiles is messed up. I was going to have special attacks be plugins with a listener in the combat and the same with bosses. Atm there’s an anti-leach (more like I was halfway done implementing something so nothing works atm). If you go in the xml file that contains all of the events and chain handlers and whatnot you just have to add in the event chain handlers and whatnot for the new item interactions. I implemented almost all of Chris Fletcher’s work as I really loved pretty much everything he came out with. His item verification system to prevent accidentally forgetting about item checks (inspired by an idea by Graham if i’m not forsaken) is incomplete. All it needs is the events updated though. Tons of skills are added and most are bug free. Action class needs to be expanded for support for combat following and whatnot to include a proper queue and more constructors for different action types. 377 is highly under-developed. Most of the 317 protocol packets are done, but some may be incorrect as I didn’t check all of them (mostly object animation ones because they aren’t actually animations but rotation packet iirc).

What I wanted to eventually do:
I wanted to change the protocol of the server and create a program kind of like a deobfuscator or renamer that would send a jstring before every packet describing what the packet was and the payload so the server could then build the corresponding event from that information. That way every single client revision would work once you got login and lobby working. I also then wanted to open up my site to have a script repository where people could write scripts and upload them to the repository for free or for a cost. Then members could download/buy the scripts at their will and it would auto-add to their ProtoScape. If RuneScape changed the way things were done in a new cache revision then the main team (me and a few programmers) would either make modifications to the core and release a patch (have an update server or something before the person runs their server to make sure it is up to date and notify them elsewise) so that way you would have a server that could run every revision of runescape from 1-current providing you have the cache and client.

FAQ:

Why quit? I feel like there’s better, more important things in life to do than RSPS. I wanted to revolutionize the scene of RSPS, but I don’t think I understood how much work was going to be required and how it may only boost my ego and let children be able to make money easier.

Why release all of my hard work? Because I hope someone takes this and improves upon it and rewrites the shitty parts of it and makes it into what I once dreamed of creating.

Why do you suck ass at coding? To each their own, but in my self defense, this was the first thing I started programming. As I got better at programming I realized some ways I could of done things easier, but I always believe time spent reinventing the wheel is time wasted, so I never redid anything. I was going to redo everything once I had everything finished, but then I guess that’s just a viscous cycle that never ends of self improvement.

Why is none of the content in plugins? First off I hate ruby. Secondly I hate ruby. And thirdly I believe combat is better in the core where people can’t love it up but can extend it with the use of plugins. So you things that are dynamic with combat (new special attacks/weapons, new bosses) should be dealt through plugins, whereas static content such as formulas, checks, and whatnot should be inside.
