[size=18pt]Network Topologies - Basic revision guide[/size]

Network Topoligies are - in a nutshell - how computers can connect and interact. There are several ways computers can interact:

[size=14pt]Bus network[/size]

A B C

A bus network is a network where all computers are connected via one cable called a BUS. All nodes on the system share the same backbone to connect to the network.

If node B wishes to connect with another node on the system, the line is first checked to see if a signal is being sent, as the bus line can only accept one connection at a time. In the result of the line being clear, a signal is sent from the node. The node knows to send the data to a specific machine id, so it will reject any other node on the system that does not match that id. Usually on a basic Bus network, fewer than 14 devices are connected. Any more than this, the network can suffer from performance issues.

Packet collision

Simply put, if two packets are simultaneously sent down the Bus network cable, they collide and corrupt. The workstation however can recognise this and usually sends a random resend time.

Advantages

Bus networks are very cheap to set up as they only have a few cables connected to the backbone and the backbone itself
Bus networks are easy to set up and relatively easy to maintain
Bus networks are ideal for a temporary cheap network situation.

Disadvantages

If there is a problem with the backbone, the entire system will fail as all nodes are connected via this cable.
If one node fails, the whole network will fail.
Low security, as packets are not encrypted and are easy to intercept
The more nodes, the slower the network
Administration is difficult as there is no “master” of the network.
One virus on one node can affect all nodes

[size=14pt]Ring network[/size]

A ring network is very similar to a Bus network, except all ends are connected together.

Packets are sent around the ring network to be picked up by the appropriate node. All data is sent around in the same direction, so packet collisions are rare. Packets from multiple nodes can be sent via the ring due to this.

Advantages

Data transfer speeds are faster as you are only sending in one direction
Data transmission is simpler as all packets travel in one direction only
Due to the architecture of the Ring network, packet collisions are very rare

Disadvantages

Data packets are passed through every node on the network which can slow down the network
The entire network is dependent on the ring cable, so if the cable is broken the network fails
If one of the nodes on the network fail, the network will cease
All nodes must be on for the network to be active
Finding problems on the ring can be difficult

[size=14pt]Star network[/size]

Star networks are now the most common type of network in existence. Simplistically, all connections are made to one hub or switch. The central hub acts as a connection point for all nodes within the network.

This type of network is more secure than the others as data is sent straight to the central hubs, and is not transferred through every node on the system meaning that packet interception is not possible. All security for the network can be handled within the central hub.

This type of network is more expensive than the others, and can often require new hardware and sometimes new software.

Advantages:

No interception of packets
Compared to the other two, performs better
Easy to connect new nodes
If one node is down, it doesn’t affect the network

Disadvantages:

More expensive
Too much dependency on central device has its own drawbacks. If it fails whole network goes down
Performance and as well number of nodes which can be added in such topology is depended on capacity of central device.

Glossary: Node - A device connected to a network
