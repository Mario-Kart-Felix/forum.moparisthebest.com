abraham2119 July 31, 2016, 6:43pm #7

Don’t listen to Davidi2, he’s a clueless moron.

The thing with your current design is your use of OOP is kind of flawed. What you should have is the main class reading the next two numbers and then performing the required arithmetic operation, instead of simply pasting the same exact code in each one of the operation classes. Regardless, the operation handling can be setup in a map holding a character as the key (the operation character) and the selected handler as the value.

I wrote up an example that should be easy for you to understand of what the application should be:

import java.io.Console;
import java.util.HashMap;

class Main {

	public static void main(String[] args) {
		Console console = System.console();
		ArithmeticOperation operation = null;
		while(true) {
			String s = console.readLine("Select the operation you want to perform: \n" + 
				"'a' for addition, 's' for subtraction, 'm' for multiplication and 'd' for division: ");
			if (s.length() > 1 || (operation = operations.get(Character.toLowerCase(s.charAt(0)))) == null) {
				System.out.println("Not a valid selection.\n");
				continue;
			}

			int firstNum = 0, secondNum = 0;
			try {
				firstNum = Integer.parseInt(console.readLine("Enter the first number: "));
				secondNum = Integer.parseInt(console.readLine("Enter the second number: "));
			} catch(NumberFormatException e) {
				System.out.println("Not a valid number.\n");
				continue;
			}
			System.out.println("Result: " + operation.solve(firstNum, secondNum) + "\n\n");
		}
	}

	private static HashMap<Character, ArithmeticOperation> operations = new HashMap<Character, ArithmeticOperation>();

	static {
		operations.put('a', new Addition());
		operations.put('s', new Subtraction());
		operations.put('m', new Multiplication());
		operations.put('d', new Division());
	}
}

interface ArithmeticOperation {
	public int solve(int firstNum, int secondNum);
}

class Addition implements ArithmeticOperation {
	public int solve(int firstNum, int secondNum) {
		return firstNum + secondNum;
	}
}

class Subtraction implements ArithmeticOperation {
	public int solve(int firstNum, int secondNum) {
		return firstNum - secondNum;
	}
}

class Multiplication implements ArithmeticOperation {
	public int solve(int firstNum, int secondNum) {
		return firstNum * secondNum;
	}
}

class Division implements ArithmeticOperation {
	public int solve(int firstNum, int secondNum) {
		return firstNum / secondNum;
	}
}

Keep in mind however, usually you shouldn’t use OOP for such a simplistic handling routine, instead, you’d use a switch statement, but I did it this way to keep some of the design of your original program (in essence) and to teach you a bit about abstractness.
