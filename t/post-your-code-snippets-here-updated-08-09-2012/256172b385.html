In this post, I’ll be showing you how to add “hidden” characters. The purpose is to allow admins and mods to investigate certain players for rule breaking (without them noticing).

The process is really quite simple.

[ul][li]Add a “hidden” variable to the Player[/li]
[li]Check if the player is hidden in the update packet. If they are, ignore/remove them[/li]
[li]Add a command for mods/admins to make themselves hidden[/li][/ul]

[size=18pt]1. Adding the variable[/size]
This part is easy. Go into your Player class, and add the following.

private boolean hidden;

And because we all want to follow simple object-oriented techniques, we’re going to add a getter/setter pair.

public boolean isHidden() {
    return hidden;
}

public void setHidden(final boolean hidden) {
    this.hidden = hidden;
}

[size=18pt]2. Modifying the update process[/size]
This part is a little bit more difficult. I’m not going to get into specifics (PI vs Delta vs Czar, etc) because the process is the same regardless; the only things that change are the names.

Go into your update process, and add the following code where the other players are added to the current update list.

if (otherPlayer.isHidden()) {
    if (currentPlayer.getPlayerList().contains(otherPlayer)) {
        currentPlayer.getPlayerList().remove(otherPlayer);
    }
    continue;
}

as well as where the movement updates for the other players is added.

[size=18pt]3. Adding the command[/size]
At this point, you should all know how to add commands to your servers. This command is very simple.

if (command.equals("hidden")) {// NOTE: feel free to add whatever requirements you'd like.
    final boolean hidden = Boolean.parseBoolean(args[0]);// Where args is the array of strings after the command name.
    player.setHidden(hidden);
}

That’s it. Now whenever a player uses the hidden command, it will be marked as hidden, and will disappear to all players that can currently see it, and not show up to future players.

@Atomic: Well for one thing, you’re checking to see if the name.toLowerCase() equals one of the strings in the unspawnable list; however, your list contains strings that aren’t exclusively lowercase.

  "Ahrim", "Dharok", "Guthan", "Karil", "Torag", "Verac",

You’re also declaring the strings twice, one for the “proper”, and one for lowercase, taking up twice the amount of memory.

Second, Strings inherently take up more memory than ints. Ultimately, it’s up to the VM to determine how much memory classes take up (if I recall), but it’s usually 8 bytes + the sum of all primitive data types contained in the class, + the sum of all non-primitive, non-null datatypes. So for example, let’s say we had the following class:

public class Node {
    private long id;
    private Node next;
    private Node previous.

    // methods, constructor, etc
}

The Node class has 8 bytes initially (object index, and name) + 8 bytes for the long, and if next and previous aren’t null, 16+ bytes until a null is reached.
