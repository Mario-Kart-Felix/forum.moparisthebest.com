Sorry, but using printf is wrong. The function printf is a legacy function from C, which should always be avoided, unless there is an extremely strong reason for a particular case that suggests otherwise.

Printf is a variadic function, which tend to be completely type unsafe (And printf is, indeed, type unsafe) besides other problems. There is hardly ever a reason to use a variadic function in C++ at all.

Output through streams (Done through overloaded << operators), on the other hand, are the way to go, and are type-safe.

The reason why you are getting errors while using these is, most probably, and somewhat ironically, the best reason for you to use them.

If you choose the wrong type with printf, it is most likely to corrupt the stack, to lead to UB, or even to cause an exploitable vulnerability. If you choose the wrong type with streams (Which is a lot less likely, seeing how the type is automatically deduced), the program won’t compile, which means you will be able to correct the error in seconds.
