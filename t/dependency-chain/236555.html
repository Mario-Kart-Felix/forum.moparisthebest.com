There were no good implementations of a dependency calculator/chain that I could find (in fact, I couldn’t find any at all), so I wrote my own.

Of course, this could be optimized a bit (pretty sure…), but it’s fairly simple. Quite useful if you are writing an updater and need to manage the order in which your transforms (or “hooks”) need to execute. A lot can be improved upon.

DependencyChain.java:

[code]package cp.dependency;

import java.util.*;

public class DependencyChain
{
private LinkedList ordered;
private List unordered;

public DependencyChain()
{
    ordered = new LinkedList<T>();
    unordered = new ArrayList<T>();
}

public DependencyChain(Collection<? extends T> initial)
{
    this();
    unordered.addAll(initial);
}

public void addDependent(T d)
{
    if (!unordered.contains(d))
        unordered.add(d);
}

public void removeDependent(Dependent d)
{
    unordered.remove(d);
}

public LinkedList<T> getOrderedChain()
{
    return ordered;
}

public List<T> getDependent()
{
    return unordered;
}

public boolean calculate()
{
    return calculate(500);
}

public boolean calculate(int max_cycles)
{
    ordered.clear();
    List<T> temp = new ArrayList<T>(unordered);

    for (int i = 0; i < temp.size(); i++)
    {
        T d = temp.get(i);
        if (d.getDependencies() == null)
        {
            temp.remove(i);
            ordered.addLast(d);
        }
    }

    int c = 0;
    while (!temp.isEmpty() && c++ <= max_cycles)
    {
        for (int i = 0; i < temp.size(); i++)
        {
            T d = temp.get(i);
            boolean met = true;
            for (Class<?> clazz : d.getDependencies())
            {
                if (!linkedListContainsClass(ordered, clazz))
                {
                    met = false;
                    break;
                }
            }

            if (met)
            {
                temp.remove(i);
                ordered.addLast(d);
            }
        }
    }

    return c <= max_cycles;
}

private boolean linkedListContainsClass(LinkedList<? extends T> list, Class<?> clazz)
{
    for (T d : list)
    {
        if (d.getClass().isAssignableFrom(clazz))
            return true;
    }
    return false;
}

}

[/code]

return new Class<?>[] {A.class, D.class};

{
public Class<?>[] getDependencies()
{
return new Class<?>[] {A.class, B.class};
}
}

public static void main(String[] args)
{
    DependencyChain<Dependent> dep = new DependencyChain<Dependent>(Arrays.asList(new D(), new B(), new A(), new C()));
    dep.calculate();

    System.out.println(Arrays.toString(dep.getOrderedChain().toArray(new Dependent[4])));
}

Nicely made, but it might not be always as usable. If your updater hooks the same field multiple time (to be sure it’s right), you would have a slight problem with it.
If you don’t have that, then it’s relatively good, although using annotations looks cleaner. (frank_'s way)

Otherwise you have to use something like this: (or if you want to safe a few nanoseconds)

public void hookAll(boolean doLoaderHooks)
	{
		Transformer[] hooks = this.hooks.toArray(new Transformer[0]);

		for(int tCount = 0; tCount < hooks.length; tCount++)
		{
			int pCount = 0;

			for(int hookIndex = 0; hookIndex < hooks.length; hookIndex++)
			{
				try{
					Transformer hook = hooks[hookIndex];
					if(hook == null) continue;
					if(doLoaderHooks != hook.isLoaderHook()) hooks[hookIndex] = null;
					else if(!hook.canHook()) pCount++;
					else
					{
						hook.startHooking();
						hooks[hookIndex] = null;
					}
				}catch(Exception e){ e.printStackTrace(); }
			}

			if(pCount == 0) break; //Everything hooked!
		}
	}
