imthenull August 4, 2016, 1:28pm #3
lare69:

design has nothing to do with performance

… yes it does

The fallacy that different design patterns have no direct impact on application performance is really common sense. If you take a moment to think about it, a program with a complex design pattern will more than likely have to invoke a multitude amount of subroutines, as to where a more simple design has all the data more closely present. Note that this is NOT the only way a more advanced pattern can effect a program. Often a more complex design pattern is chosen for the sake of future expansion, in which case a minor performance loss is offered in return.

A small example of this would be the infamous FizzBuzz problem, take a look at these two solutions…
https://github.com/EnterpriseQualityCoding/FizzBuzzEnterpriseEdition/
http://pastebin.com/qpCuSWVu

In the end it all depends on what you want to accomplish, sometimes cramming everything one a single file is the most logical thing to do and other times it would be to expand it upon hundreds of files. In this particular situation the latter would be most desirable.

Would one do you think would have better performance when the execute command holds over 2000 lines or various code from the Java API? I mean, they all do the same thing, so why should the performance be different?

1: while a boolean running is true, a method execute is called only if the difference between the started time and the system’s current time is greater than or equal to 1000 milliseconds.

long startLogMillis = System.currentTimeMillis();
while (running) {
	if (System.currentTimeMillis() - startLogMillis >= 1000) {
		execute();
	}
}

2: while a boolean running is true, a method execute is called and the current thread is paused for 1000 milliseconds.

while (running) {
	execute();
	Thread.sleep(1000);
}

3: A separated thread is scheduled at a fixed rate which will call for the method execute every 1000 milliseconds.

(ScheduledExecutorService).scheduleAtFixedRate(new Runnable() {
	@Override
	public void run() {
		execute();
	}
}, 1000, 1000, TimeUnit.MILLISECONDS);

Your performance will always depend on your design.
