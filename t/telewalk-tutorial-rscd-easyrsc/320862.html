I’m assuming this tutorial will sum up RSCDv25 and EasyRSC, but if not, I’m using EasyRSC so it will work with that.

The purpose of this tutorial is to give admins the power to “telewalk” (To teleport on click instead of walk). It should be simple to follow if not post a question :).

The first file we’ll be editing is “Player.java”. Find the line containing “private boolean requiresOfferUpdate = false;” and add this in underneath it:

	private boolean willTeleport = false;

	public void setTeleport(boolean b)
	{
		willTeleport = b;
	}

The next file we’ll edit will be “CommandHandler.java”. Scroll all the way to the bottom and just above the last two “}”'s and add in:

	if (cmd.equals("telewalk"))
	{
		if (player.willTeleport())
		{
			player.setTeleport(false);
			player.getActionSender().sendMessage("Unset");
		}
		else
		{
			player.setTeleport(true);
			player.getActionSender().sendMessage("Set");
		}
	}

And finally we’ll edit “WalkRequest.java”. Now find:

		else if(player.isBusy()) {
			return;
		}

And right under that put:

		else if(player.willTeleport())
		{
			int startX = p.readShort();
			int startY = p.readShort();
			int x = startX;
			int y = startY;
			int numWaypoints = p.remaining() / 2;
			for (int i = 0; i < numWaypoints; i++) {
				x = startX + p.readByte();
				y = startY + p.readByte();	
			}
			if(world.withinWorld(x, y)) {
				player.teleport(x, y, true);
			}
			else {
				player.getActionSender().sendMessage("Invalid coordinates!");
			}
			return;
		}

And you’re done! Just don’t forget to compile and any player that is an admin (rank 4) can now telewalk by typing “::telewalk”.
