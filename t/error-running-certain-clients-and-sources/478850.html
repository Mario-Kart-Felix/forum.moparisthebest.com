The client is trying to go to the x86 program files, which is only on 64 bit computers, and things are only stored there if you get a 32 bit download.
So you can remove that bit and you should be good to go. (it’s highlighted in red)

Also, assuming the variables are set right. For compilers you should be able to just set it to “javac” no quotes, instead of a classpath like you have. To run it, say the client, you can set it to “java” no quotes. So it’d be like so:

from this:

C:\Program Files\Java\jre6\bin\java.exe

to this:

java

and this:

C:\Program Files\Java\jdk1.6.0_25\bin\javac.exe

to this:

javac
