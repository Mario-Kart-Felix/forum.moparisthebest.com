pkedpker August 1, 2016, 1:09am #6

Thats why I recommand virtual hosting from a reliable trusted host like HostGator of GoDaddy as long as you keep paying the 20/30$ a month you your virtual operating system stays on 24\7.

I’ve been running a java server on a hostgator vps for 50 days now straight without ever restarting the process (although I could install it as a service) it’s a bit trivial on a redhat linux.

Only catch i’ve noticed on hostgator is that if you exceed the resources given to you… I am using a Level 1 plan and the maximum number of TCP connections opened at one time is set to 800 meaning if you exceed it (which i did many times, on popular days) the vps restarts itself and your application your hosting can experience downtime until you start it up or service kicks in.

I’ve used godaddy as well and I hate their customer support over the phone no matter how in detail you explain what you need or what your problem is they will read shit off a book and give you the run around. I’ve called them 10 times that day and always got random answers nothing that helped. Yet when you are buying a package from godaddy and you tell them does this support hosting java applications? they will tell you, that you must upgrade to a non-shared plan for a extra 2$ a month, no biggy i thought and what did I get? I got some useless crap called tomcat which restarts every 24 hours and couldn’t even run my java server without porting it to servlets some crap like that. Unlike HostGator which is nice for running anything on your vps, it’s internet speeds are capped at 500 kb/s well for level 2 package (you won’t notice this as a website host.

But I did a wget test on a 100 MB file. With GoDaddy I got 22mb/second and with HostGator the mere 500 kb/s.

What i’m trying to say there is no magic host that allows everything you want all hosts got advantages and disadvantages.

Best host of course would be a dedicated server which is out of my budget any probably anyones.
