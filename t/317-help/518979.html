The client holds the visual side of the game. What I mean is it holds all the items for you to actually see and play the game. It sends data over to the server and the server will relay data to tell have the client show what you did.
So like walking. You see your player and map. You then click a place on the map. That gets sent to the server from the client. The server verifies that you can walk there, and tells the client to animate the player and go ahead and walk to the position.

Now, you’re going to download a client AND source in a package, hopefully it comes with the cache. You want to get updated JRE and JDK. If you have a 64 bit Computer and OS, you want to get both the x86 and x64 downloads for JDK and JRE, if you need links, I can get you them.

From there you’re going to want to open the server and clients compiler.bat and make sure they have this in there:
“C:\Program Files\Java\jdk1.7.0_21\bin\javac.exe”
and there will be junk after that.
The version of JDK is important as it will give a “The system cannot find path…” error otherwise.

From there you’re going to want to edit the IP in the client. Find client.java (CLIENT SIDED) and search either “127.0.0.1” or “server =” usually most 317’s have server =.
You want to set it to 127.0.0.1 as that’s local host.
Then run the compiler for the client.
Now Run the server and make sure it doesn’t have an IP you need to change (Project Insanity’s don’t have on, most delta’s do).
After the server is running, run the client.
Login with any user/pass and you’re golden.
