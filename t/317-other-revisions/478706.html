This thread is mainly for the liberty of newer users.

Let’s face it, you’ve come here because you’ve played runescape at some point in your life and you have the urge to create a private server so that you and your friends can chill on it. That’s everybody’s first idea. But if any of you are like me, it doesn’t stop there. You become intrigued. You realise there’s quite a lot to this “programming” stuff even though the game code looks like nonsense to you.

I’m here to tell you to begin programming on a 317, even if you move to a newer revision later on. Here are some of my reasons:

  1. 317 servers were the first servers ever produced, and therefore are better documented. Without question. If you take ten minutes of your time to look at the RS Wiki, you will note that the 317 protocol has the most information.

  2. Generally, there is more support for a 317. If you have a problem with a 317, most users of this forum will be able to help you and support you in some way. Sure, some users can help you with 700+, but your support is relatively very limited

  3. 317 is easier to pick up. Seriously? If you’re a new member, go download Phonescape or another classic server. Begin to look through it. Note the structure and how the game code is placed together in such a brutal yet effective way. I started on Phonescape back in '08 and to this day I remember being taken to the East Varrock bank and thinking “omg how did I spawn here?!?!?!?!?!”. Inquisitively lead to me finding out about a crappy method called ResetPlayerVars which ran once when you first logged in

  4. 317’s have more content. There’s a lot of poor code out there, but much of it works. What many people fail to realise is it’s where the bad code is used. For example, the other week I installed a fishing system on some kids AllstarScape. The code wasn’t the best. But why does it need to be? If you really want to be picky about server code, you should start with removing the thread per client.

  5. 317 clients have more edits than other revisions clients out there. If you want to edit your client in a certain way, there are more tutorials and client hacks for 317 clients than the other revisions

  6. 317 servers do not have to send map data. Many may be thinking this is an idiotic remark as we can dump map data now. But even in the latest revisions, we have reports of map data getting corrupted and often not showing whole sections of the world map. 317’s do not have this problem and never will.

  7. If you wish to move on to a later revision, you can forward your experience of identifying packets to help you understand the later revisions better. If you know that the click-to-attack method in a 317 in 74 (I think off the top of my head!) you can use this knowledge of identifying packets and bring it forward to assist you in your later revisions

Inb4 tl;dr start with 317’s. More info on them.
