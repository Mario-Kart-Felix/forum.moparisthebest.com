Lin August 3, 2016, 2:20am #8

It really depends. If I’m going to use something dynamic, I never use regular arrays (obviously). Recently I’ve been using hash sets (or hash lists in Java) and dictionaries for memory-saving and performance reasons. For example, my server’s world system is made up of hash sets and dictionaries (essentially the same thing) because each tile will have a unique location, but not all of the tiles will be used at once. Rather than use a list to add the tiles to and cycle through all of them (even in regions), I find it best to use hash sets and use the coordinates for indexing. That way, it saves on memory and is friendly on performance.

For other ordinary things, I mostly use lists if it’s for something small and they won’t impact performance, like a shop or properties editor just because of the RemoveAt and Contains methods.
