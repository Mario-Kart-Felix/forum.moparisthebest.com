On the topic of data storage, i view it as a personal preference. You basically categorize them in so many different ways that I find doing so to be insane. Two of these categories would be serialized vs. non serialized (TSV, CSV, binary, etc).

On the topic of serialization I either use googles protocol buffers, or I just say fuck it and use object outputstream. The limits of using objectoutputstream (limited to java, fugly, high overhead) are not large enough to persuade me to use protocol buffers all the time.

However, I think you were more referring to preferred collection.
On this topic, I side with sini.
I will either use a basic Stack implementation (if there is no problem with it being FILO), a map (since it has O(1) lookup time it is very nice for data storage), or just arrays. Lately I have been using HashMaps even less. I used to use them all the time; however, when I was going through my code a few months ago I noticed that I was using them in cases when the fast lookup time is not that beneficial compared to what is spent in allocating it.
