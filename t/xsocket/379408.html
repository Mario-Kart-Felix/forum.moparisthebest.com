Dinoboy August 1, 2016, 7:06am #2

MINA, netty, grizzly, xSocket, etc are all the same.

they are just wrappers for the NIO api to try to make it easier for people without great networking skills to use (although I personally believe using the java NIO api is easiest since it gives me much more flexibility then using any of the wrapper APIs).

do these wrapper APIs increase performance? no

your question is if xSocket is better or worse? it’s the same, it’s all NIO underneath. the API is of course different, and if you find it easier then just using plain NIO, netty, or whatever you want go for it.

i personally recommend just reading some tutorials on NIO

i don’t know what the big deal is about netty, xsocket, mina and whatever people are using on these servers today. it’s as if they except a performance increase or something :\

Dinoboy August 1, 2016, 7:06am #3

well of course performance won’t be the same since each API has it’s own different implementation. MINA queues up all data to be sent before writing anything, netty sends things on the fly when the queue is empty (as in it hasn’t queued up data yet, so when network activity is low it will perform better than MINA, but when network activity is high, it will do the same as MINA seen it will need to queue up data) which gives it a slightly faster speed. so if you’re talking about, let’s say, a multithreaded MMORPG the difference will not be big.

but then you also need to consider the fact that both these APIs are a step down from using the java NIO API (although again, the difference is not big enough that you should worry about it, and it also depends on what type of server you are trying to implement, you won’t get a performance decrease, obviously, if your server uses the same/similar implementation as the aforementioned APIs)
