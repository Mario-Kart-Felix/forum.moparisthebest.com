public int sOCPacketClicks;

 * A flag to see if the process of reset the packet 132 clicks has started
 */

[/code]

Then this in second click object packet

and add this class:

[code=java]package server.util;

import server.event.Event;
import server.event.EventContainer;
import server.event.EventManager;
import server.model.players.Client;

/**
*

  • @author Strucker

*/
public class AntiAutoClicker {

/**
 * The maximum click per minute a player should do, because stalls are
 * re-stashed every 3 seconds, so 60/3 = 20, and that is not likely to happen
 * because i doubt any player is that good.
 */
public static final int MAXIMUM_CLICKS_PER_MINUTE = 20;

/**
 * The wait time until the clicks are refreshed are 60k milliseconds
 */
public static final int RESET_TIME = 60000;

/**
 * 
 * @param player
 * 		the player that will have its clicks added
 */
public static void addClicks(Client player) {
	
	/**
	 * Every time the player clicks the second option of an object
	 * that variable is increased
	 */
	player.sOCPacketClicks ++;
	
	player.sendMessage(""+player.sOCPacketClicks+"");
	/**
	 * If the player has the same amount or more than the maximum amount of clicks
	 * per minute continue
	 */
	if(player.sOCPacketClicks > MAXIMUM_CLICKS_PER_MINUTE) {
		
		/*
		 * The player is walked away from where he is to stop his power skilling
		 * or whatever he is spamming.
		 */
		movePlayer(player);
	}
	
	/**
	 * If the process of refreshin the player clicks hasnt started continue
	 */
	if(!player.isRefreshingClicks) {
		
		/**
		 * Sets the flag that tells wether the clicks are being refreshed or not to true
		 */
		player.isRefreshingClicks = true;
		
		/**
		 * Starts the process of refreshing the clicks
		 */
		refreshClicks(player);
	}
}

/**
 * if the player has gotten to the maximum number of clicks avaible
 * the player will walk back three steps
 * 
 * @param player
 * 		the player that will be automatically walked three steps back
 */
public static void movePlayer(final Client player) {
	
	/**
	 * Moves the player back
	 */
	player.getPA().walkTo(-3, 0);
}

/**
 * After one of the first click the clicks will be reseted
 * 
 * @param player
 * 		the player that will have his clicks reseted
 */
public static void refreshClicks(final Client player) {
	EventManager.getSingleton().addEvent(new Event() {
		public void execute(EventContainer c) {
			player.sOCPacketClicks = 0;//Resets the clicks
			c.stop();
		}
		}, RESET_TIME);
}

}
[/code]

So how a normal stall works is that when you steal from it the items disappear from it and the stall is restashed in 3 seconds, so it would mean 20 clicks in 1 minute to get all the items from the stall.

What this does is that if you click it more than 20 times in one minute it will walk you back.
So it combats AutoClicker and massclicking

*Note
sOCPacketClicks stands for secondObjectClickPacket
