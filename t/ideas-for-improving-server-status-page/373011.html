These are my current ideas to improve the page:

[ol][li]Server Status Control Panel, This would show you things like:
[list type=decimal]
[li]Last time the server was checked[/li]
[li]Any errors given while checking to see if it was online, like DNS didn’t resolve, connection timed out, connection refused, etc etc[/li]
[li]Any future enhancements, like page impressions on your ad, time people played your server with moparscape, etc[/li]
[/list][/li]

[li]New online checking program, with a longer timeout than the current one, which should report online more consistently even with network traffic.
This could also:
[list type=decimal]
[li]Send you an email (which you could forward to a text message) when your server is offline, with the error returned[/li]
[li]Check servers that came up offline once more after all others have been checked, this would give them another chance of not being ‘offline’ with no decrease in uptime. Still not sure about this one, comment.[/li]
[/list][/li]
[li]Dynamic DNS for ss.moparisthebest.com, this would allow you as a server owner, instead of using a no-ip, you could have .ss.moparisthebest.com point to your server, and update it with the same type of client that you use for no-ip. Would you use that?[/li]
[li]Much more integration with your site, could include:
[list type=decimal]
[li]Registration callback page. You put a link to a page on your server on the ‘edit server’ page, this page implements an API so that users can register for your server from a page on moparisthebest.com, my server tells your server about it and your server tells my server whether it is accepted or denied with a reason.[/li]
[li]Vote callback page. You put a link to a page on your server on the ‘edit server’ page, this page implements an API so that my server can tell it what users voted on your server, along with info you need so that you can track which users voted on your website. You can offer rewards for votes, or any number of other things with this.[/li]
[li]Perhaps I can implement an OpenID system, so that people can register on your website simply with their moparscape forums user/pass, or maybe even just log in with this, so they have the same login across all servers[/li]
[li]MoparScape ad revenue share. Allow your users to play your server using moparscape, and you get multiple advantages:
[list type=decimal]
[li]You put your google adsense code in your ‘edit server’ page, and something like 50% of the time (negotiable), when people play your server using moparscape, I will display ads with your code (that YOU get paid for) instead of my ads.[/li]
[li]Users are comfortable playing moparscape, it is a familiar interface, and they know their computer is safe from malware and viruses if they do so[/li]
[li]Your server can automatically be played by ALL systems, people running Linux, Mac OSX, or Windows, with the same client[/li]
[li]MoparScape supports custom clients as well (you can write a modified client to run inside moparscape, much easier than writing your own web client), and soon HD clients that are also cross-platform[/li]
[li]You don’t have to worry about legal problems associated with hosting the client[/li]
[/list][/li]
[/list][/li][/ol]

And if your server supports full integration (registration callback if registration is required, and allows people to play on the moparscape client) then you will be rewarded with a better position on the server status page, an official ‘moparscape approved’ status, and possibly more. In the future, I will probably require ALL sponsored servers to support full integration, and may even require ALL servers to support it, but that’s quite a ways down the line.

Please comment on all of these as to whether you would use them, or support them, or change them somehow. Or add in your own feature requests. When you are referencing one of the ideas, please use a format like 3.1.4 (this would be referring to the line about MoparScape supporting custom clients)

Thanks for helping to improve this!
