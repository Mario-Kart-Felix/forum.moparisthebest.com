Moparisthebest:

Any ideas on how I would implement that though?

Award an ‘uptime’ point each time that your probe discovers that the server is online. Award a ‘downtime’ point if the server is down when you probe it.

Then calculate a score, perhaps using uptime_points / downtime_points.
So a server that has 500 uptime points and 7 downtime points would score 71 (71.42 rounded down), and a server that has 500 uptime points and 150 downtime points would score 3.

The scoring formula doesn’t really matter, so long as you apply it consistently across the board. You could take other metrics into consideration, such as server longevity… but the point system effectively does this in my opinion.
