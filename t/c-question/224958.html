zstars July 31, 2016, 12:49am #4

Maybe you’re not supposed to add the printf at all. The function getchar() is supposed to only return EOF if an error occurs or if the end of the stream is reached. An error is unlikely to occur, and you are reading from the console, so there is no real End of File (EOF). Is is indeed a crappy example if I am right, but, I think that the loop is likely to loop forever.

If you want a different behaviour, post what you want to achieve and we can see. But, that code simply waits for EOF, and, if you don’t put the printf into the loop, you will only get the number of characters after the EOF is found.
