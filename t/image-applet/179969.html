yakman July 30, 2016, 7:37pm #7

theres two way i can think of around this

one way is to make your own class java.awt.Canvas thats the same as the real one, and override getGraphics(), then use the non-standard option “-Xbootclasspath/p” to add your class before the real java library classes

or you could do some client-editing, there is a class in the runescape client which is a subclass of java.awt.Canvas, find it and use a bytecode library to change its superclass to one of your classes which extends java.awt.Canvas, then override getGraphics() as above

once you have getGraphics(), you can “return image.createGraphics();” instead of the real ones, or you can mess about it a bit so it paints on both, or some other way, experiment!
