  • Introduction

I decided I would write a simple tutorial on how to run the RSCAngel source code in a LAN/Offline environment.

** Requirements

This guide assumes that you know how to compile java source code and setup MySQL/xamp, if you do not meet this requirement, please view other tutorials on this board to bring you up to this standard, it should only take you a max of 30 minutes.

Please ensure your jdk is installed properly, your JAVA_HOME path is correct and you have apache ant installed (or the ability to compile using javac directly)

*** Downloading

First download the source code from here: http://rscangel.org/dev/rscangel_server_source.06.08.2010.tar.gz

You will also need the client source code: http://rscangel.org/dev/client_src_svn.03.08.2010.tar.gz

Winrar will be able to extract .tar.gz files, I suggest using this.

***** Compiling the client

Extract the client archieve and then open the file: src\org\rscangel\client\util\Config.java

You will need to change:

public static String SERVER_IP = "f2p.rscangel.org", CONF_DIR = System.getProperty("user.home") + File.separator + ".rsca", MEDIA_DIR = System.getProperty("user.home") + File.separator + ".rsca"; public static int SERVER_PORT = 43595, MOVIE_FPS = 5;

So that the PORT matches your world.xml port and the IP address is localhost (for the scope of this document).

compile using either the compile.sh script:

export JAVA_HOME=/path/to/jdk ant compile

or make a compile.bat script:

set JAVA_HOME=C:\program files\jdk ant compile

You shouldn’t have any problems compiling the source

Once you have the rscd.jar file, open it in winzip and add in the lib folder and org/xmpull

You will also need the data directory: http://rscangel.org/rsca.zip

****** Database

Included with the server source is a schema for the database RSCAngel used, you will need to import this sql file.

Via command line: mysql -u myuser -pmypass mydatabase < database_structure.sql

If you’re using phpmyadmin you will need to just import the file.

You can go ahead and delete all of the rscd_ tables, there are lots of redundant tables in there which were used for the website, but I’m not going to go through and clean it up for you. The tables which the loginserver and gameserver use (Pets added a MySQL connector into the gameserver because he didn’t want to interface with the LS (sigh)) are the rsca2_ tables.

******* Configuring

You will need to configure the loginserver and gameserver configuration files before attempting to run the server.

The loginserver uses the ls.conf file:

<?xml version="1.0" encoding="UTF-8" standalone="no"?> RSCD-LS Config localhost mydatabase myuser passhere localhost 34526 localhost 8186
  • lsport and queryport can be changed to any available ports, providing you make sure to repeat the changes in the world.xml file. It is best to ALWAYS have the lsip as localhost regardless of if you’re running a public server or not. The clients do not need to interface with the loginserver directly, allowing them to do so will open up huge bugs and exploits to your server.

The gameserver uses the world.xml file:

<?xml version="1.0" encoding="UTF-8" standalone="no" ?> - RSCD Config 39 localhost 43594 RSCAngel USA 800 3 localhost 34526 rsca passhere localhost
  • Servernum should be higher than 2 as the loginserver is coded to only reset the online flag in the rsca2_players when loginserver 3 connects. This was a nasty hack at the beginning of the server which was never resolved.

******** Running

The client:

You can run the client with the following command “java -cp client.jar org.rscangel.client.mudclient”

The server:

Now that you have the server compiled and the database imported, you should be able to just go ahead and run the server. I normally use the java command when running the server, as ant creates two processes, itself and java.

For Linux users, modify the included run_server.sh script:

cd /home/rsca/F2P_RSCD/ cp runserver.log logsbackup/runserver.log.`date +%F.%s` cp logs/mod.log logsbackup/mod.log.`date +%F` nohup /home/java/jdk1.6.0_14/bin/java -Xms128m -Xmx2048m -classpath /home/rsca/F2P_RSCD/lib/mina.jar:/home/rsca/F2P_RSCD/lib/pircbot.jar:/home/rsca/F2P_RSCD/lib/bsh.jar:/home/rsca/F2P_RSCD/lib/xpp3.jar:/home/rsca/F2P_RSCD/lib/slf4j.jar:/home/rsca/F2P_RSCD/lib/xstream.jar:/home/rsca/F2P_RSCD/lib/mysql-connector.jar:/home/rsca/F2P_RSCD/lib/hex-string.jar:/home/rsca/F2P_RSCD/conf/server/quests:/home/rsca/F2P_RSCD/rscd.jar rsca.gs.Server world.xml p2p yes > runserver.log&

This will backup the log files into a logsbackup folder (you will need to create this folder). Also make sure to change all of the paths from rsca to your user and directories. The options on the command line are, ‘mainclass serverconfigfile p2p/f2p launchIRCBOT(yes/no)’. The IRC bot’s classes are in ‘src/rsca/irc’. Not sure how complete the p2p/f2p switching is, we never really used it.

For windows users:

set java_path=C:\program files\java\jre6\bin\java
set RSCD=C:\RSCD
%java_path -Xms128m -Xmx2048m -classpath %RSCD\lib\mina.jar;%RSCD\lib\pircbot.jar;%RSCD\lib\bsh.jar;%RSCD\lib\xpp3.jar;%RSCD\lib\slf4j.jar;%RSCD\lib\xstream.jar;%RSCD\lib\mysql-connector.jar;%RSCD\lib\hex-string.jar;%RSCD\conf\server\quests;%RSCD\rscd.jar rsca.gs.Server world.xml p2p yes

or

ant runserver

Though I’m not sure if the buildfile has the correct parameters

And for the login server, again Linux users can use the included run_ls.sh script and modify it accordingly. Windows users can use:

set java_path=C:\program files\java\jre6\bin\java
set RSCD=C:\RSCD
%java_path -Xms56m -Xmx128m -classpath %RSCD\lib\mina.jar;%RSCD\lib\xpp3.jar;%RSCD\lib\slf4j.jar;%RSCD\lib\xstream.jar;%RSCD\lib\mysql-connector.jar;%RSCD\lib\hex-string.jar;%RSCD\rscd.jar rsca.ls.Server ls.conf

or

ant runls

(maybe).

Linux users, remember to chmod +x *.sh before attempting to run the scripts.

Make sure MySQL is running.

Now run the loginserver first, followed by the gameserver if the loginserver runs without problems.
