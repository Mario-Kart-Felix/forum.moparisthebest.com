An example of a library is for example, you need physics in your game, you could create your own, wich would take in A LOT of time and you would need to have great knowlegde of all sorts of advanced mathematics or you could use a library, basicly pre-made physics by someone else.
People can’t tell if you used a library or made your own, unless your own physics are really crapped out physics.

Using a library is the best solution, for the best results for the people that play the game.
However, it is not so good for the developer because of the fact that by using other libraries your not thinking, you don’t gain expierience, I’d say only use a library when your really need it.

For my flash game, I used a pixel perfect collision detection library by someone else, it works great, the rest, everything, physics, everything, I made myself, did not use one bit of code provided on another site.

I used that library because I was not able and am still not able to make pixel perfect collision detection on my own. If I was, I would probably not use the library.

So my advise is, only use other libraries when it takes to much work to make your own/you have limited knowledge to make your own/you know a 100% you can make your own and you have made your own before but need a quick library for a quick project.

m1lkman:

Did Andrew Gower use a library to create Runescape or did he create his own library.

He built his own software rasterizer and used Java 2d to actually push the buffer to the screen. He also used the Java audio api to play software synthesized and sampled sound. Basically, he took reinventing the wheel to the extreme, but in his defense, he wrote RuneScape before the JSR specification for an opengl binding was written. I think he started game dev before hotspot even existed in the VM. I mean, he was an early pioneer of Java game development.
