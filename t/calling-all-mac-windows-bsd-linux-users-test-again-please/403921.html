[code]java -jar ResourceGrabber.jar
Usage: ResourceGrabber [[TORRENT|MAGNETURL|URL] SAVE_PATH EXTRACT CRC]
TORRENT is a path to a .torrent file
MAGNETURL is a magnet link
URL is a url to a torrent file, or any file over HTTP
SAVE_PATH is the absolute path to save the torrent and .resume file to
EXTRACT is true if you wish to extract the downloaded file
CRC if not 0 will only download the file if the CRC does not match, it will checksum every file in the save_path recursively.

You can specify as many downloads on the command line as you wish.[/code]

An example that will download the free and legal Ubuntu 11.04 iso via bittorrent is this:

Don’t forget the quotes due to the funky characters in there.

Also try multiple downloads at the same time, you can specify as many as you want!

You can download anything supported by Java’s URLConnection class (as far as I know, HTTP and HTTPS, also possibly FTP), and any torrent via file, magnet url, and HTTP url to the torrent itself. It will pop up a nice little GUI showing you the progress, and also tell you some key points via command line. This will download my platform-specific java_client to your computer, but will only run it if the CRC matches.

It will NOT download or extract any files with the following extensions, for safety:
“.exe”, “.bat”, “.cmd”, “.com”, “.sh”, “.bash”

Please post logs if you have any problems, and as always post hints/suggestions. All of this code will be released free and open source under the GNU/GPL, probably at github. It is written modulary enough where you can write any front-end you want, this one just happens to be Swing.

If any of you are actually worried, or just curious, I attached the full java source, as well as the C++ source code to the binary, and my build scripts. When I officially release it, it will probably be at github:
