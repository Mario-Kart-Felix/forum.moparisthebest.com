So. If you look at your classpath, you’d notice it’s just a filepath to where the jdk bin is.
To change it to jdk8 u 25, you’d change the 1.6.0_11 to 1.8.0_25 (You can also see this in your folder where you installed the jdk).
Same goes for path…

“The right java” would vary based on what you’re doing. Typically, the latest release will be fine. However, Java 1.8 has changed some security settings, making it almost impossible for self-signed applets to be used in web browsers. A.K.A, your webclient you’d use. most people just opt to compile the client on 1.7, which still allows self-signed applets (Without using the whitelist).
You can have Java 1.8 and compile to 1.7 by using the -target option.
See this link for more.

As for the compilers “original” code. that’s just regular compiler code.
They used the directory for the java compiler because they probably didn’t have environmental variables set. So, since you do, you can replace the quotes and what’s in them with “javac”. This will still target the java compiler. “java” will run a java application and “javaw” is the exact same, but there will be no associated command window.
