[center][size=22pt]What is a Hash Table?[/size]
[hr][/center]
In this tutorial we will be focusing on the HashMap class, which is a hash table implementation of the Map interface. Well that’s great but what is a hash table? Generally speaking a hash table is a data structure that maps keys to values using a hash function.

[ul][li]A key is a unique identifier used to reference the value in the map.[/li]
[li]The value is whatever data is stored in the map.[/li][/ul]

Before we jump into examples note that HashMap is a generic class. I’ll let Java Documentation explain that.

A generic class is defined with the following format:

class name<T1, T2, …, Tn> { /* … */ }
The type parameter section, delimited by angle brackets (<>), follows the class name. It specifies the type parameters (also called type variables) T1, T2, …, and Tn

Lets start with a simple example:

    // Assume java.util.HashMap is imported
    HashMap myMap = new HashMap();

You’ve just created a new instance of HashMap named myMap.
Before we continue with this explain lets examine the methods that will be used in this tutorial.

[ul][li]put(K key, V value)[list]
[li]Associates the specified value with the specified key in this map.[/li]
[/list][/li][/ul]

[ul][li]get(Object key)[list]
[li]Returns the value to which the specified key is mapped. If there is no mapping for aforementioned key it returns null.[/li]
[/list][/li][/ul]

In a nutshell, the put method populates your Map and the get method returns the value of whatever is held where the specified key is mapped.

[ul][li]clear()[list]
[li]Removes all mappings from the HashMap[/li]
[/list][/li][/ul]

[ul][li]containsKey(K key)[list]
[li]True if key exists, otherwise false.[/li]
[/list][/li][/ul]

[ul][li]keySet()[list]
[li]Returns a Set view of the keys in the HashMap.[/li]
[/list][/li][/ul]

Lets add on to our example:

    // Assume java.util.HashMap is imported
    HashMap myMap = new HashMap();
    myMap.put(1, "moparisthebest");

Right so now our HashMap is no longer empty, the String “moparisthebest” has been added to the HashMap with an Integer key of 1.

Moving on:

    myMap.get(1);

We stored “moparisthebest” using the key 1, so calling the get method and entering 1 as the argument we can safely assume that it would return “moparisthebest”.
[hr][hr]

[center][size=18pt]Generic Type Invocation[/size]
[/center]

While the above example works just fine and dandy its not correct. Remember that HashMap is a generic class, it needs to be parameterized which is done through [b]generic type invocation[b].

Using the previous example:

    // Assume java.util.HashMap is imported
    HashMap<Integer, String> myMap = new HashMap<Integer, String>();
    myMap.put(1, "moparisthebest");

Now we have very clearly identified that the key will be of type Integer and the value will be of type String. Before we parameterized the HashMap the following could have been added without compiler errors

myMap.put("hi", "derp");

As you can see the key is a String. Parameterized, the key can ONLY be an Integer.

[center][size=22pt]Hash Tables and Their Usefulness in RuneScape Servers[/size]
[hr][/center]
There’s countless things you could use HashMaps for in a RuneScape Server lets look at a few examples.

First and foremost, rather than having all instances of players in an array you could put them in a HashMap

    HashMap<String, Player> players = new HashMap<String, Player>(2000); // 2000 player limit

    // ... Someone logs in
    Player p = new Player();
    p.playerName = readName();
    players.put(p.playerName, p);

    // ... Someone logs out
    String playerName = "deathschaos9"; // person logging out
    players.remove(playerName);

In instances like this you have to ask yourself “Whats the best variable for the key and what should the value hold?”. In this instance I chose playerName as the key because it is always unique and as for the value it only makes sense (in this case) to use the instance of the player as the value.

What else can we do?

1. Quickly teleport a player to you. Assume: Key = playerName, Value = instance of Player

    HashMap<String, Player> playerMap = new HashMap<String, Player>();
    // ^ Contains all online players, you want to teleport me (deathschaos9)
    if (playerMap.containsKey("deathschaos9")) {
    	playerMap.get("deathschaos9").setPos(myX, myY);
    }

2. How many players are online? Assume: Key = playerName, Value = instance of Player

    HashMap<String, Player> playerMap = new HashMap<String, Player>();
    int playersOnline = playerMap.size();
    System.out.println("Players: " + playersOnline);

A really neat trick that I don’t see utilized enough (this applies to arraylists as well) is knowing that the remove(key) method actually returns the value that is deleted.

ArrayList Example (minesweeper context):

instead of //ArrayList<MineSquare> available; is an ArrayList of the squares that are valid to have a mine on them (depending on the rules, this usually just means all squares EXCEPT the first one the player has clicked) //MineSquare has a method setMine() that sets the square to a mine :) for(int i = 0; i < numMines; i++) { int randomIndex = Math.random()*available.size(); available.get(randomIndex).setMine(); available.remove(randomIndex); }we can do for(int i = 0; i < numMines; i++) { available.remove(Math.random()*available.size()).setMine(); //selects a random available square. removes it from the arraylist (so that it wont be chosen again), and because remove() returns the value removed we can set it in the same line }
Again this is just an example, it applies to maps and not just arraylists

Some more things not mentioned in this:

If you are ever using a hashmap with integer keys, immediately re-evaluate you need for a map. Could you be using a normal array or arraylist?

A great analogy is like the legend on a roadmap or graph. In the case of a roadmap, you’d have a HashMap<Symbol, Meaning>. In a graph, you’d have HashMap<Color, Series>.
