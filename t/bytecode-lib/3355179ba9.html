They also realized it was pretty stupid for Borland to try to compete with Microsoft with Delphi.NET, which was a half-assed cross-breed between Delphi and .NET, they tried to make it as much like the native Delphi as they could, which was a huge mistake and lead to the fact that rly no one used it. Embarcadero has since Delphi 2009 dropped the product and moved on to Delphi Prism, which is a .NET implementation of pascal, but not compatible with Delphi like Borland tried with Delphi.NET. Also Prism integrates in Visual studio rather than to be integrated into the Delphi IDE like Delphi.NET was. Which I think is pretty great, since the Delphi IDE was never intended for a .NET language… Prism is powered by the RemObjects Oxygene compiler technology, which is pretty great, it cares somewhat less about being conform with normal pascal syntax to ensure that it doesn’t hold back on language features. Though my license includes it, I don’t use Prism myself, the nice thing is that it does allow you to use Mono, so you can do cross-platform development with it, if you like Pascal/Delphi and .NET, it’s nice to check out. http://www.embarcadero.com/products/delphi-prism
