SilentCJ August 4, 2016, 1:35am #18

My only criticism would be the use of center tags, ew. Not just an opinion either, they’re deprecated now, just a heads up

Always remember, CSS for presentation, HTML for semantics.

Oh, and if the script is going to be used more than once it should be stored in an external file in a function, then just call the function when it’s needed.

Edit2: on looking at your CSS, couple of suggestions, where you declare your margins and padding, you do each side separately, for cleanliness of code you should define it in one property. e.g. margin: 10px auto 20px auto;
The order is top, right, bottom, and left respectively. See http://www.w3schools.com/cssref/pr_margin.asp for reference, and this is the same for margins as well.
With the border-radius CSS3 property, it’s not necessary to define the prefixes for each browser anymore, it’s become a standard as just border-radius. However, the prefixes for box-shadow and most other CSS3 properties are still needed.

Also look into HTML resets for CSS.
