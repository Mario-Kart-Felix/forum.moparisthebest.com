Avoid using mysql_* functions. Unnecessary direct control, especially for beginners, is just a security liability. Use PDO.
I’ll give you some code from one oh my sites.

mypdo.class.php

[code]<?php
define(“DB_HOST”, “localhost”);
define(“DB_NAME”, “my_database”);
define(“DB_USER”, “root”);
define(“DB_PASS”, “”);

class MyPDO extends PDO {

public function __construct() {
	$dns = 'mysql:host=' . DB_HOST . ';dbname=' . DB_NAME;
	return parent::__construct($dns, DB_USER, DB_PASS);
}

public function getPrepared($sql, array $bindings) {
	$result = $this->prepare($sql);
	$result->execute($bindings);
	return $result;		
}

public function getPreparedArray($rownum, $sql, array $bindings) {
	if (count($bindings) == 0) return false;
	$result = $this->getPrepared($sql, $bindings);
	if ($rownum === 0)
		return $result->fetch(PDO::FETCH_ASSOC);
	$arr = $result->fetchAll(PDO::FETCH_ASSOC);
	return ($rownum) ? $arr[$rownum] : $arr;
}

}[/code]

To use the above object, initialize it in your script with:

include "mypdo.class.php"; $db = new MyPDO();

Now you can use $db wherever you want to interact with the database. There are a bunch of PDO methods you can find on the documentation along with the two methods (getPrepared and getPreparedArray) which I added for bit more ease.

For instance, you can still do direct queries like so:

define("IP", $_SERVER['REMOTE_ADDR']); $userinfo = $db->query("SELECT * FROM users WHERE ip = '".IP."' LIMIT 1")->fetch(); define("USERID", $userinfo['id']);

Or you can get the above $userinfo[‘id’] just by itself assigned to $id:

You can use getPrepared() like so:

$name = "Paul"; $sex = True; $result = $db->getPrepared("SELECT id FROM users WHERE name = :name AND sex = :sex LIMIT 1", Array(':name'=>$name, ':sex'=>$sex)); $image_id = $result->fetchColumn();

A prepared statement will place the variables correctly into the SQL string without you having to concatenate yourself, which would allow for SQL injection vulnerability. I made getPrepared() so that you could use an array like the above for the variable key pairing.

Lastly, getPreparedArray() will return a particular row. This will SELECT all rows and put row 4 into $userinfo:

$userinfo = $db->getPreparedArray(4, "SELECT * FROM users WHERE item_id = :item", array(':item'=>$item));

Anyhow back to topic, if you want to insert rows, you could add something like this method to MyPDO in the above mypdo.class.php:

// User input should not affect $table directly public function dbInsert($table, $values) { if (count($values[0]) > 1) $fieldnames = array_keys($values[0]); else $fieldnames = array_keys($values); $sql = "INSERT INTO $table"; $fields = '(' . implode(',', $fieldnames) .')'; $bound = '(:' . implode(',:', $fieldnames) .')'; $sql .= $fields.' VALUES '.$bound; //print_r($values); $stmt = $this->prepare($sql); if (count($values[0]) > 1) { foreach($values as $vals) $stmt->execute($vals); } else $stmt->execute($values); }

Then use it like so:

$userinfo = $db->query("SELECT * FROM users WHERE ip = '".IP."' LIMIT 1")->fetch(); if ($userinfo){ $id = $userinfo['id']; } else { $db->dbInsert('users', array('ip'=>IP)); $id = $db->lastInsertId(); } define("USERID", $id);

That will look for the user with that IP, and create them if they don’t already exist and then set the new id to USERID.
Anyhow, no reason to use my extended class… but you’ll probably want to customize MyPDO for your own uses. Just remember that PDO’s prepare() is a good thing if you don’t want people to fuck with your database.

EDIT: notice I didn’t use prepared statement with IP constant when putting it in the query… it is assumed to be safe from user input, that is why.
