Hello all! This is something I made in 15 minutes because it was pissing me off having to add objects in client.java, and then compiling.

Description: To make it so you can load objects from a configuration file like NPCs.

Difficulty: 1

Tested on: DeltaScape (Should work for all)

Classes Modified: server.java, client.java, objectLoader.java

Step 1: Getting the objectLoader class
First off, download the file attached and stick it in your server folder with the rest of your classes. You can compile from here if you want.

What this class does is declares 5 arrays indicating each loaded object’s properties. They’re pretty much all self explanatory.

The next part starts off a method to actually load the data from the objects.cfg file, or whatever you told it to load. It loops through each line of the file and stops if it finds [EOF] or that there’s nothing left.

The rest of the code splits up the data into 5 parts by tabs. They assigns the current object properties. The objectCount is what tells the server which object it’s trying to load, and how many objects that have been loaded. The variable you previously saw, line, indicated the current line it was reading. “l” Was the data of the current line.

To the moderators: Sorry I couldn’t really explain the code better. There’s not really much to explain.

Step 2: Making it read
Okay. Now you’re almost done. Open up your server.java. Now declare a public static “objectLoader” called [i]ObjectLoader[i]. You don’t have to set its value as we’ll set it later. If you’re not sure if you did it right, it looks like this.

public static objectLoader ObjectLoader;

Next scroll down to your “main” method. It looks like this.

public static void main(java.lang.String args[])Below that, we’re going to set the ObjectLoader’s value and make it load the file.

Note: If you’re using DeltaScape, it goes below the [i]throws NullPointerException {[i].

ObjectLoader = new objectLoader(); ObjectLoader.loadFile("./config/objects.cfg");So what did that do? It assigned ObjectLoader as a new objectLoader, the class we just made. It then tells it to load the objects file. In this case, it’ll be loading “objects.cfg” from the config folder.

Step 3: Making the objects
This is the final step, and the most important. Without this, everything would be useless.

Open up your client.java. Search for [i]case 121:[/b], which is what’s called when you load a map. We’re going to make it create the loaded objects from the file.

Note: If you’ve already got objects added in the newObjects or deleteObjects method, or anywhere else, you do not have to delete them. They will still be there.

Now below the case 121:, add this.

for(int oc = 0; oc < server.ObjectLoader.objectCount; oc++) { int oID = server.ObjectLoader.objectID[oc]; int oX = server.ObjectLoader.objectX[oc]; int oY = server.ObjectLoader.objectY[oc]; int oO = server.ObjectLoader.objectO[oc]; int oF = server.ObjectLoader.objectF[oc]; makeGlobalObject(oX, oY, oID, oF, oO); }Now, it may seem complicated. However, it could all be broken down into two lines, but would look messy.

It starts off by creating a for loop that will loop through the object creating code for however many objects there were loading.

The next parts assign new variables according to the loaded object’s properties. It knows which object to create because of oc, the current object number.
oID = The loaded object’s ID property
oX = The loaded object’s X property
oY = The loaded object’s Y property
oO = The loaded object’s Type property (Eg. 10)
oF = The loaded object’s Face property

And finally, the last line of code creates the object based on what it loaded. You should know how that works.

Congratulations! You completed the Loading Objects from a CFG File quest! You earned: 1 Quest point and the ability to load objects from a CFG file.

Note: You MAY change the name of the file to load. That’s what I assumed you all would use though.
Reminder: Don’t forget to create the objects.cfg file in your config folder!

The format of the objects:

object=ID	X	Y	Face	Type

Example:

object=869	3268	3408	0	10

Enjoy all!

~Lin
