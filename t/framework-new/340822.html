It’s nice to see you’re using try-catch, but try looking into also adding in finally. p.s. you don’t want to try-catch more than you need to.

Here’s an example of try-catch-finally:

Socket s = ...;
OutputStream os = null;

try {
    os = s.getOutputStream();

    // Use the output stream to your hearts content
} catch(IOException e) {
    e.printStackTrace();
} finally {
    if(os != null) {
        try {
            os.close();
        } catch(IOException e) {
            e.printStackTrace();
        }
    }
}

The point of using finally in that scenario is, what if you make the OutputStream, and you get an exception before you get to close it? Then the program jumps down to handling the exception and continues on without closing it. The finally block ALWAYS gets executed, whether you catch an exception or not, so it’s good to use for cleaning up.

If you really are looking into making a framework, be sure to document all your functions and fields, so that way the person using your framework will be able to see what everything does. Here’s a few links to stuff to look into:[

The Sun Java Tutorials
Documenting your applications
Concurrency Tutorial
