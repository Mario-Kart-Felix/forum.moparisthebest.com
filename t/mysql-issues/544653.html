Download the tool from the link I gave you. It also has a port checker built into it. It’s a lot better than canyouseeme.org 's.
If the checker cannot see you, possible reasons are:
Firewall blocking it
Anti-malware/anti-virus blocking it
your network ip changed (not the same as your broadcast one) thus the portforward not working anymore
you didn’t activate the port forwarding correctly
you aren’t checking the port is active when the server is running.
another application is running on the port causing it to be “blocked” (Though the server would error saying it couldn’t bind to 43594)
and so much more.

Like I said.
MySQL runs on port 3306. RSPS runs on 43594 or 5555. MySQL doesn’t block port 43594 unless you for some reason reconfigure it to not be on port 3306 and to be on one of the RSPS’s. And if you did, the RSPS would error in 1 of 2 ways: 1) The server cannot bind to 43594/5555 because MySQL is already bound OR 2) MySQL cannot be bound to 43594/3306 because the RSPS is on it.
