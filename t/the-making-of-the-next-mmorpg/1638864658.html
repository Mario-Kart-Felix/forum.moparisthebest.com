As stated earlier that C++ and C has quite some identical syntaxes with Java, not only these two does. Another programming language that looks even more like Java would be C#. C++ and C is made back-compatible, but when C# was created not too long ago (in the early 2000, and I don’t remember what exact year), they (Microsoft) took the advantages that Java, C and C++ had and made this language fully OO (Object Oriented).

C# does use something that Java doesn’t have, however. This feature is called properties. It can be made in Java with normal set and get methods, but doesn’t completely work like it. You could say it looks like you’re using methods and public modifiers both at once.

The below is not really on topic, nor in the correct section of the forums, but anyway (as an example).
Here is an example of such, using a class (Object) called AClass:

[code]private string aVariable;

public string MethodName
{
set
{
aVariable = value;
}

 get
 {
      return aVariable;
 }

}[/code]

How to use it:

[code]/** Using the get property. */
string aString = AClass.MethodName;

/** Using the set property. */
AClass.MethodName = aString;[/code]
