semaphore July 31, 2016, 3:15pm #6

I don’t think you understand the question you’re asking. On the one hand you’re asking if it’s possible to read class info from a class that has not yet been loaded. Then you say the thing that tipped you off to it was someone essentially accessing the value of a static field ( Field.get(…) takes as a param the instance upon which to act - null means the field is static )

So to answer your question - no there is no way to do anything with a class with reflection before that class is loaded.

If you want to access class info before it is loaded just use a java-based agent implementing premain.

That approach works for dynamically changing the classes each time the app is run without making permanent changes to the libs. On the flipside - such processing will be expensive and take a while (potentially each run).

For a more permanent fix patching the app libs will lead to a single cost up front but every subsequent invoke of the app will be relatively quick.

There are a few java-based tools that I hacked using the agent method. A few others I just patch the public key/sigcheck. Each of those use the methods I mentioned respectively.
