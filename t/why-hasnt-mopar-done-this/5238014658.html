[li]All basic functionality complete. (in other words, you don’t need quests completed, but all basic skills[sub]1[/sub] should function, all doors, ladders, trapdoors and such should work correctly etc)[/li]
[li]The functionality should be independent of the client version[sub]2[/sub], in other words you need to have a packet encoder/decoder that is specific to a client version, that decodes packets to a version independent format that is then passed onto the game logic script(s). (if you need clarification, let me know)[/li]
[li](optional[sub]4[/sub], but would be a great bonus, and easier for you) Scripting support for the game logic. Examples of good languages you could use would be python, ruby or javascript. [quote=“Moparisthebest, post:1, topic:261879”]Clarifications:
[1] By ‘basic skills’ I would say at minimum magic, attack, defense, strength, range, mining, woodcutting, firemaking, fishing, cooking. Possibly a few more simple ones I am forgetting. I am not talking about farming, slayer etc. Although the more game logic implemented the more money you would be payed for your server.
[2] The client version you use doesn’t matter to me at all, just keep in mind that the only thing in your server that should be specific to any version is an encoder and decoder.
