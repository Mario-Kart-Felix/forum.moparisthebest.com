Hey guys, I’ve finally finished the first version of that 377 server base (written from scratch) that I promised all of you after you voted (80 to 11) that you would be willing to use a 377 server if a good one was provided for you. This is my baby, and I’ve worked pretty hard on it for all of you, ;).

At the moment, the project is very bare in terms of content. Basically, as version 1, all it is is really a good base for intermediate to advanced server developers. As more versions are developed, it will have more content and will be more playable out of the box.

Features

[ul][]Completely free and open-source, courtesy of the GNU General Public License v3.
[]Implements the professional, highly-scalable Apache MINA networking library.
[]Login packet with RSA encryption (what real RuneScape uses) enabled with a private 512 bit key-pair for security unprecedented by any other RuneScape emulator.
[]Highly efficient server networking and design, while following the K.I.S.S. (Keep It Simple, Stupid) principle wherever possible.
[]Never before seen approach at packet event handling: event listener chaining, inspired by a thread in the MoparScape forums veteran section.
[]Thread pooling.[/ul]

Details
Along with the server, I have written (from a deob) a partially-renamed client for you guys. Thanks to Galkon for helping me refactor the client. It’s still in the process of being made, but it’s playable. The client, most importantly, loads and uses the public RSA key file of the rs377d project. I’ve also included an RSA key generator, so you can generate your own keys (read the README for a tutorial of how to apply the new key files to your server/client). The client comes with an IP/port switcher, and a (joke of a) protocol monitoring system, which still needs to be completed. Inside the server project is a README html file containing a ton of information about the project and all that neat stuff. You should read it, because I wrote it for you.

FAQ
What the hell is a daemon?
A daemon is a sub-process which runs in the background and provides a service of some sort. The name for the project “rs2d” is actually an acronym for “RuneScape 2 Daemon”.

What base is this server written from?
This server is written from scratch, no base was used to build this server off of. Don’t ask this question. Seriously.

Whats a SVN?
SVN is an acronym for Subversion, which is a “revision control system”. It’s basically a way of allowing multiple people to update one central server source at the same time, because only the parts that were changed (individual parts in each source file) are uploaded to the SVN. If you want to use SVN, just use TortoiseSVN, which integrates with the right-click in Windows to make SVN easy to use.

How can I compile/run this server?
As this server project is taken seriously, it is developed in a professional way. I use an IDE (Integrated Development Environment) to develop this server, as IDEs vastly improve how productive programmers are. I use Eclipse IDE, which compiles, error-checks, and can run the server for me very easily. If you’re still using Notepad (or Notepad++) and batch files to compile/run your server, take the step up to an IDE - it may be difficult to get used to, but overall it greatly improves your programming experience.
