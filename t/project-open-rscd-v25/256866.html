Hello everyone!

Since the overall public development of the RSCD_v25 server is just about dead, I have come up with an idea that could benefit everyone here. For those familiar with the MANGOS world of warcraft private server - that is an opensource server that has been worked on and improved for the greater public good to the point where it is nearly perfect at emulating a WoW realm. Let us work together and improve the RSCD_v25 daemon and associated website so that everyone may benefit from the development.

It is not the “proprietary code” that keeps players playing your private servers over others. The reason your player base comes is the server avaliblity and overall quality of the service the administrators and moderators provide. Instead of complaining about leechers, leech proofing source code, and fighting, lets put our heads together and continue improving this code. Jagex has screwed us out of Runescape Classic and it is our job to provide this for ourselves and our friends.

I have created a sourceforge project for the Open RSCD_v25 development. I am interested in having everyone that would like to contribute, do so. The goal is to have it remain as close to the original version of Runescape as possible.

SourceForge

Open_RSCDaemon v4

Download Open_RSCDaemon v4 for free. This is for the development of the RSCD_v4 server daemon. The code is GPL licensed.

SVN access has been set up and once more people are interested, I will start creating developer accounts for SVN access.

Right now I am stripping all identifiable code from the DARScape Classic source which contains all the security improvements, bug fixes, and code additions along with the updated and secured punBB forum. I will also be uploading an emptied database too. Once that is done, the source will be uploaded to sourceforge.

I ask anyone who has added code improvements to either the punBB forum, the RSCD punBB mod, or the RSCD_v25 server to request to join this project so that we may merge the code and continue to develop this awesome server.

While using an unmodified source would be okay, the time saved by using an already updated and fixed source which has been proven to withstand 2+ weeks of uptime without crashing seems like a better idea. Another thing is that the security fixes to the punBB forum in which I have made are already implemented in the code I am running. Another issue is that there has been fixes and updates done by Uberpker and Nick (forgot his alias) that I do not remember how to add back in. Nick is MIA right now but Uberpker is still around.
