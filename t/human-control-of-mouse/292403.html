Human control of mouse

koolmelee July 31, 2016, 9:44am #1

I am interested in creating a mouseMove function that moves the mouse to a given x, y coordinates.

I am planning on just making it linear first, then adding some randomness to it.

How exactly does a computer “see” human mouse input? Is a full path of adjacent cursor positions seen by the computer or are positions skipped when the user moves the mouse rather quickly?

In addition to the above, how often would an online game poll for the mouse position, and if it were detecting botting, would it expect to see the mouse hitting adjacent pixels always or could it skip some?

Mopman July 31, 2016, 9:44am #2
koolmelee:

or are positions skipped when the user moves the mouse rather quickly?

This. The game client doesn’t need to poll for the position itself, as Java’s input system is event-driven, however as far as I can guess there would be a delay between events significant enough to cause dropped positions (someone can confirm this I’m sure)

However, the best way to find out exactly what the client would see is to gather some data - write an app to capture mouse movement events and test it yourself - you’re going to need the data anyway to implement your algorithm, surely, unless you plan on using something that is already available.

koolmelee July 31, 2016, 9:45am #3

Did a test that draws points on panel each mousemotion event in java.

Seems a user easily skips several pixels with normal use. Going fast, one can skip 20 pixels or so.

What would need to be implemented(my opinion) to make it look more human:
-Use several different functions with slight random path deviation.
-Accelerations(starting and stopping)
-Randomly overshoot target.

I searched this site, and read a few articles. I really like the bezier curve method. Great random paths. One could add some randomness/noise to it as well as acceleration.
