IP1 July 31, 2016, 3:11am #4

Some routers need portforwarding, some do not. Most do, however, like you; my old house did not require Port forwarding.

Make sure to do the following:

~ Register a Domain
~ Check the Domain in No-Ip DUC Client
~ Allow firewalls to access that port
~ Portforward the port via router (Click here for help)
~ Make sure the client has the correct IP address/Port
~ Make sure the server is running on the correct Port you put in client/portforwarded to on router.

IP1 July 31, 2016, 3:11am #6

Hmm… Have you made sure the router you have isn’t on the list? And is it a router or Modem from AT&T?

Also, log on to your router, usually this and it usually has a Portforwaring/Port Triggering tab.

Press the Portforward and select custom service, and then enter the Description (name), the port (usually 43594 or 5555, whichever port you are running your server on), and finally as the IP address have it 192.168.1.4

And make sure you allow it to access the ports in your Windows Firewall.
