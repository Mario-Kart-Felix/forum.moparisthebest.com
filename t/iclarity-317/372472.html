A new server base from scratch.[/center]

I am starting a new server project, however I am writing it from scratch. This is the first time that I am writing a RuneScape private server from scratch, so this is a challenge and a learning experience for me. My biggest obstacle will be player updating - I do not know the 317 protocol well enough to implement it myself, so, I will most likely be getting help along the way (I will, of course, list the people who are kind enough to give help). This is a small project; not my main one, and I have other IRL commitments so I won’t be updating it rapidly.

Features

[ul][li]Open-source, under the GPL (Version 3) license.[/li]
[li]Loads server configuration upon start-up.[/li]
[li]Takes advantage of the Netty networking API, for server to client and client to server communication.[/li]
[li]Fully completed login decoding.[/li]
[li]Fully completed packet decoding, though packet handling is still to be written.[/li]
[li]A ScheduledExecutorService to execute a fixed schedule at a rate of 600ms (The server tick) to manage tasks in the server i.e. events, player processing.[/li]
[li]Blakeman8192’s Event Executor, tweaked by me to run on server ticks (As that is the rate that events are meant to run at).[/li][/ul]

Revision updates

[b]Revision #4:[/b] A few small updates. [b]Revision #3:[/b] Player registering added into the finishing point of login decoding. Packet decoding complete. Player name cleansing added (Removing unwanted characters and capitalizing the first letter of the username). Staff name loading from the configuration xml file. [b]Revision #2:[/b] Added the server to the svn (Already had most of the current content in it complete). [b]Revision #1:[/b] Creating svn, with the blank trunk folder.
