CoonHunter August 3, 2016, 2:35am #15

[quote=“xx matty xx, post:14, topic:441667”]Doubt it - it seems to be a bit of a pointless method because the JVM should garbage collect automatically, and even a call to gc() doesn’t guarantee that the garbage is collected immediately.

My guess is that it’s not a memory leak but a cache that’s not getting cleared out, like something that caches map data.[/quote]
My chatroom’s server (currently being damn near completely-rewritten for the 4th time) would use about 5mb of memory for about 1/4 second when someone logs in. Problem was that the gc() for some reason was never running, and after a few hours of testing logging in and out (along with at one point 20 people online) it went from 3mb idle, to over 200mb of memory used. After I added a force’d gc() call every so often it now it stays under 4mb memory used even with 20-30 people online. I’m not sure if its just the jre7 version I am using, but force calling it solved all memory problems I had.
