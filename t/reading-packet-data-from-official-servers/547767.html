T41 August 4, 2016, 1:58pm #12

The opcodes change with each ISAAC nextInt() return value. The server is able to decrypt the opcode because it also has an ISAAC cipher with the same seed. The arguments to the opcode (the song id or whatever it is) never changes within the same revision, but may be subject to change across revisions (historically it hasn’t happened often afaik).
