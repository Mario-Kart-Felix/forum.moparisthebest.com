I’ve tried multiple packets on PI, but of course Sanity was nice enough to patch it. It send the slots of the items, then the server checks if the player has those items in the players list/array.

		int usedWithSlot = c.getInStream().readUnsignedWord();
		int itemUsedSlot = c.getInStream().readUnsignedWordA();
		int useWith = c.playerItems[usedWithSlot] - 1;
		int itemUsed = c.playerItems[itemUsedSlot] - 1;
		UseItem.ItemonItem(c, itemUsed, useWith);

You still could send items that are not there, but that will just throw an exception server-sided; won’t do much harm.

I’ve tampered with multiple packet sending client-sided, but only got so far as to throw an exception server-sided using the move item packet.
