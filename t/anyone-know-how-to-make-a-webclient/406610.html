you will need to sign the jar.
the reason it is happening is because it will be using System.getProperty(“user.home”) to find their home directory, so that it can open the appdata folder.

java needs permission to do this, as well as modify files when it is running as an applet, so you will need to sign the client to be able to allw it to do so, and the user will need to click run on the box that pops up with the certificate.
