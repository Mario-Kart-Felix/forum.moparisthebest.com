TeleNubby July 31, 2016, 1:40am #18

Since the answer is somewhat scattered between posts, and due to my lack of entertainment at the time of this post, I’ll go ahead and restate everything in a ‘cleaner’ format. Oh, and yes, I know you already solved this, but others in the future may want to know as well.

The bug in you’re code can be pinpointed to right here:

	            while (s.hasNext()) {
	            	if (s.equals(lol))
	                	System.out.println("its equal");
	            	
	                System.out.println(s.next());
	            }

Or even more so:

	            	if (s.equals(lol))

Variable s is a Scanner object, while variable lol is a String object. Two objects, when compared, will never equal each other. In Java, an object’s toString() method return’s the object in its own String representation (so the equals() method is comparing two Strings).

Let’s go even deeper. Your Scanner object would look somewhat like this (taken straight out of the API’s source code):

    public String toString() {
        StringBuilder sb = new StringBuilder();
	sb.append("java.util.Scanner");
	sb.append("[delimiters=" + delimPattern + "]");
	sb.append("[position=" + position + "]");
	sb.append("[match valid=" + matchValid + "]");
	sb.append("[need input=" + needInput + "]");
	sb.append("[source closed=" + sourceClosed + "]");
        sb.append("[skipped=" + skipped + "]");
        sb.append("[group separator=" + groupSeparator + "]");
        sb.append("[decimal separator=" + decimalSeparator + "]");
        sb.append("[positive prefix=" + positivePrefix + "]");
	sb.append("[negative prefix=" + negativePrefix + "]");
	sb.append("[positive suffix=" + positiveSuffix + "]");
	sb.append("[negative suffix=" + negativeSuffix + "]");
	sb.append("[NaN string=" + nanString + "]");
	sb.append("[infinity string=" + infinityString + "]");
	return sb.toString();
    }

When calling the toString() method on the String object, it returns itself. You can already see that the entire String representation differs from Scanner to String, as each class/instance/object has its own set of fields/paramters and own unique name/type.

So to conclude that part of this little speech, different objects, when compared, are always different. It doesn’t matter if you made your own String class with the value “hello world” and compared it to Sun’s String class with the value “hello world”, because even then the class’s name and/or type is unique and different from one another (yours would be something like test.type.String and theres would be java.lang.String).

To fix this, you’ll have to use a String representation of the value being inputted into the console, instead of the actual Scanner object itself. Here’s a tidied up and fixed version of your code:

              //Read on a continous level until the Scanner object does not have any input
              while (s.hasNext()) 
              {
                     //Read the String inputted by the user
                     String theInput = s.next();
                     if (theInput.equals(lol))
                     {
                            //Display message indicating the two Strings are equal to one another.   
                            System.out.println("its equal");
                     }
                     System.out.println(s.next());
              }

You can see a difference here. Instead of comparing the Scanner object itself, we compare the String inputted into the console by the user to the String you are checking for (which in this case is “0:4”).

That’s all.
