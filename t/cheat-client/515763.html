A cheat client sends client -> server packets telling the server that a certain action has happened (even though it hasn’t) for example clicking on an object. You can send a packet tellin the server you clicked on the certain object and since most servers don’t check to see of the objec is there it goes straight to the functions for clickkng said object.

No you do not need to make a whole new client. You can use any client that has been released.

To get a cheat client to work on servers you have to set your client to the server ip. And it’s all in java except for client interfaces and cache editing, but you shouldn’t have to worry about it.
