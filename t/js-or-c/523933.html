RSCShadow August 4, 2016, 9:09am #6

Depends on your attitudes and target audience. There’s nothing wrong with JavaScript, it’s still object-orientated, it just doesn’t use the “classical” model (though the “class” keyword is in ES6 and will be used as shorthand for writing function constructors). If that bothers you, feel free to use TypeScript, a compile-to-JS language designed by Anders Hejlsberg (the lead architect of C#) which does provide you with types and classical classes.

The canvas API is extremely easy and the browser is certainly capable of a 2D RPG. And keep in mind if it’s written in JS, it’ll be able to run on almost any platform without much work to add support. The user also wouldn’t need to download any “suspicious”-looking executables and you wouldn’t need to worry about providing different downloads depending on the device. I’d be curious to know why you wouldn’t want to use JS/TS.

EDIT:
On top of that, you don’t need to worry (at least for desktops) if the user already has a C# runtime installed. You may be able to package your application with it, but I imagine that’ll make it a lot more bulky.
