Turned into a development topic for my game.

Well, I’ve been working on a game for a bit, and I’m proud with how it’s coming. The game is text-based, but has buttons and other common Windows controls you’d find in other programs (Such as text & combo boxes). It’s being written in C# .NET using the .NET Framework 2.0 for compatibility. Here is where you can catch the news for it.

The LiaC server has been completely rewritten, though only the core was really changed. This included stability, speed (Now 3 threads total verses the previous 5), and code neatness.

The server also has a better multiplayer system, and is much easier for me (Before, I for some reason used 2 variables; one for the socket indexes and one for the players… Both were terrible and would have overwritten previous player connections.) This one, instead of continuing from the last filled player slot, searches for an open one and uses that.

The IO system is still the same; easy to write and read. I’ve also added an item database. Basically, it’s like the item configuration in the RuneScape cache, except mine’s not a binary file and is loaded from the server :). The item configuration only sets the price, the use options, if it has the ability to be equipped, the description, and the name. Other things like equipment stats will be loaded from a separate file.

It also has a packet timer, which when hits 3, logs the other user out. This goes for the client; when it hits 3, the client’s main window closes and goes to the login screen.

So now the client development. It’s coming along good. I’ve added the inventory little group, and the “Use Item” window. I’ve also added packet sending, and item requesting.

Right now, LiaC’s inventory system is being added. Testing with a friend showed me that it was my computer that only got the problem, where it worked fine for him. I guess that also means it’s fully functional of having multiple players.

Now there’s where the LiaC game is at. Still in the very beta stages, getting the base engine down. Soon it’ll be over and the game will have started to be worked on.

~Lin
