I’m pretty sure Sun Java ditched the hotspot compiler years ago, and now their compiler is based on JIT, so you already use a Just In Time compiler most likely.

And yes, it has the ability, at least in theory, to make code that is actually faster that C or C++ because it can optimize on the fly, and pre-compiled languages can only optimize based on what in knows during compile time. I’ve never actually ran benchmarks on this, but I know they are plenty close enough now performance-wise to not matter at all in a desktop application.
