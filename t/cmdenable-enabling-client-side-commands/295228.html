Earlier a member named fath8m was asking me how to activate client-side commands. I then went through and explained to him how to do it. Guiding him through it, I realized it could be quite a burden to do manually. So I created this tool, cmdenable, with newbs in mind. It’s source is NOT pretty, and I’m positive this is potentially detectable, seeing as it hardcodes applet parameters, like the ‘session’ variable. It loads the client unsigned, which can affect performance. ANDDDDD a repeat, the code is meant to be shit, it’s so newbs can go through it easily.

The source and binary are attached.

To get help:

java -jar cmdenable.jar help

It works by finding the login method, which is the method that pushes the string “loggedin”, then the command handling method, which is inside the class with “::pcachesize” in the constant pool. The command handling method is searched for the code that compares the localPrivilege field to 2 (>=). It then searches the login method for where localPrivilege is set (to the value read from the server login response packet), and injects

iconst_2 putstatic <localPrivilege> I
right afterward, to give you the privilege to use the commands.

So newbies, have fun and hopefully learn a thing or two.

But remember, don’t ever program like this.
