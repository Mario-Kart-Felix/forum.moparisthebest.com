I’ve learnt how to improve the client’s cached sprite system. By making it rgba. It is now encoded with different parameters.

Usually, It’s just set to calculate pixels from width to height with it’s int value to set the rgb color for each pixel. This is normal but the quality is lower but it’s a lot less memory ram consuming.

So I added two other values, a boolean to define if the sprite needs alpha(transparency) support as some sprites don’t really need alpha support and that would be just a waist of memory. The other value being the final alpha int making it rgbAlpha. It after sets the pixel with an alpha value and when you’re rendering, you’re inquiring the data. I’ve used the rendering method released on R-S by Pb900.

Pretty much basic but it’s cool to mess with stuff
