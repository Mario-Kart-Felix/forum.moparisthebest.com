When referring to the whole collection of components, it is called an Interface. Hence InterfaceEditor, ::interface, etc. However, in the client, there is no class that specifically identifies itself as strictly a collection of components and maybe a background layer or whatever you want to call the container, they are all the same class. The name Widget is more ambiguous and can refer to a container (interface) or component. We call them the same thing, it’s just the way Jagex designed the client it doesn’t support refactoring that way.

At first glance it’s obvious it’s for prefixes and suffixes, however I wanted to explore a bit more to find out how exactly to set these server sided.

One of the variables was rather easy, [tt]anInt3958[/tt] is set during the player appearance update packet as part of the byte flag that also contains gender, etc. It’s an index in a string array for the prefix and suffix (noting that they are both the same). Did a bit more digging, found it in the script executor (pls ignore failed mass rename):

if (5422 == opcode) { int i72 = intStack[--k]; l -= 2; JagexString JagexString_55 = stringStack[1 + l]; JagexString JagexString_26 = stringStack[l]; if (-1 > ~JagexString_26.length()) { if (null == StaticMembers.aJagexStringArray3802) StaticMembers.aJagexStringArray3802 = new JagexString[Node_Sub13_Sub18.anIntArray3218[Class158.anInt2014]]; StaticMembers.aJagexStringArray3802[i72] = JagexString_26; } if (~JagexString_55.length() < -1) { if (DummyOutputStream.aJagexStringArray45 == null) DummyOutputStream.aJagexStringArray45 = new JagexString[Node_Sub13_Sub18.anIntArray3218[Class158.anInt2014]]; DummyOutputStream.aJagexStringArray45[i72] = JagexString_55; } continue; }
This is the only place in the client the values inside the prefix/suffix arrays are actually set. They are also initialized here, and the size of the arrays is one.
[tt]Node_Sub13_Sub18.anIntArray3218 = {1, 4};[/tt]
[tt]Class157.anInt2014[/tt] = game (0 = rs, 1 = stellar dawn I believe, so the size is 1)

So it only supports 1 prefix/suffix pair, meaning if the value sent during the appearance block is not 0 you’ll get an index oob exception. Doesn’t seem very useful?
