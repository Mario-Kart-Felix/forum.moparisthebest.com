[quote=“Snow_, post:295, topic:523919”]TIL about Java 8’s awesome forEach statements.

For example, following code:

ArrayList<Bullet> bullets = new ArrayList<>(); ArrayList<Player> players = getPlayers(); for (Player player : players) { for (Bullet bullet : player.getBullets()) bullets.add(bullet); } for (Player player : players) { for (Bullet bullet : bullets) { if (bullet.hitPlayer(player)) playerHit(player, bullet); } }
could be written like this:

players.forEach((player) -> player.getBullets().forEach(bullets::add) ); players.forEach((player) -> bullets.forEach((bullet) -> { if (bullet.hitPlayer(player)) playerHit(player, bullet); })); }

Another example:

for (Player player : getPlayers()) player.update();
to:

You’d want to be careful with using forEach, it can be restrictive. Most cases just stick with the for loop.

Ambokile:

Example pls

I can’t fix you a literal example, but you can’t do things like:

int i = 0;
players.forEach((player) ->
    ...
    i++;
);

You’d get a compiler error. I don’t think you can do any atomic operations.

There is no performance gain and forEach might execute in parallel. You also can’t handle exceptions.

But pros: People may argue that you get Limited flow-control but in reality that is exactly how a loop should be. No need for a premature break.
