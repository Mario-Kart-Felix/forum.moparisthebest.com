This is theoretically possible, but practically it won’t survive. Scaling human movements which don’t follow any predictable pattern or formula, or law, will work in time, but a computer can catch up if these scaled movements are read from a database. The bottom line is unless you know what goes on on their end you’re in the dark. Eventually they’ll map out these movements script-for-script if necessary, and create an exception for them as well.

Depending on the field of application, publicly available bots like RSBot are at risk of Jagex just picking apart whatever system it uses right from the code itself. However, in a large scale this could work…

Gathering movements, of say users of the program itself on the fly, scaling them like you say right then and there, and using them in working scripts, or the scripts of others users by communicating through a server, could cover Jagex being able to detect hard coded systems.[/quote]

You aren’t going to do something stupid like hard code 200 mouse methods to choose from and release it in a public bot. I am talking about recording thousands upon thousands of mouse movements, and on runtime, the program could pick a certain number of random ones for itself, then from them choose the ones to modify and use. Jagex couldn’t possibly detect a system such as this that relies only on randomness and a continually growing database of actual human mouse movements.
