moparisthebest July 31, 2016, 10:36am #40
Unhandled:

Wouldn’t it be 765^503? (Every possible move from one pixel to another)

Theoretically, but not in practice.

After all, a move from 1,1 to 473,283 is the same as a move from 473,283 to 1,1 (reversed) right? So that cuts it down by half right away. Also a move to 1,1 and 1,2 for the purposes of a human mouse click is identical, so that cuts it down another half. We could also slightly adjust mouse movements to hit targets near the actual end target. All sorts of these kinds of optimizations and assumptions can be done to cut down the number of mouse moves we store in memory, not to mention that since all of these would be read only, all of the bots running on one system can access all of them from the same location. (ie, one instance of these mouse movements, multiple bots)
