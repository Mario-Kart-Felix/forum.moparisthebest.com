sk8rdude461 August 4, 2016, 1:36pm #2

the method process in client.java (public void process() { ) runs every 600ms, or 1 server tick. Trading should not even touch that method. Trading should be done in a (from what I’m told, I’m too lazy to rewrite mine) container. How it’s currently handled is terrible, I can guarantee it.

If you want to add a timer, or a method to check how long it has been since the player joined. You should use an Event.
CycleEvents (it’s been released, google around for it) act based on server ticks, meaning if you set it to 5, it would run every 3000 milliseconds (5 x 6 = 30).
PI comes with an EventContainer/Handler that runs off milliseconds. So setting it to 5 would make it run every 5 milliseconds, and we don’t want that probably. I honestly wouldn’t use the EventHandler for anything more than a timer.

Yes that is what I’m saying.
The timer doesn’t need to be executed every 600 milliseconds.

I’d use events to make it execute every second, or even every minute.

[code=java]
public void tradeTimer() {
EventManager.getSingleton().addEvent(new Event() {
@Override
public void execute(EventContainer timer) {

		}
	}, 1000);
}[/code]

The above code is an event that’s executed every 1000 ms (or 1 second).
All you have to do is add some code to make it work like a proper timer.
