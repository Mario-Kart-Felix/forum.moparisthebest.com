Close Razor, Your definately getting there, You’ve got most of it right, However the If part is wrong.
If works like,
If (Condition) then
If the condition is true then it will do the next 1 command after the then.
If you put false in there then the condition is false and it not do the command afterwards
If you put true like you had then it will always do the next command.
Also, If’s and thens and basically all loops except repeat only do the next command.
That’s why to group single commands into one you use begin and end; like I did up there, If you look by the indenting youll know where.

It’s the same as putting it into a procedure called PressF1 and just having PressF1; after the then.
Other wise it will only push F1 down in the if statement
and wait 20 ms then simulate releasing the f1 key anyway because of the grouping error with begin and end;

If you still have questions then reply

Note this is presuming you wanted to only press it down in the if statement

Oh and to check the colour at 136 144 do if (GetColour(136, 144) == 0) then.
