I was talking to frank_ and I was thinking, what if the community wrote a deobfuscator together? Ok, that’s a little far-fetched. More like, why don’t select few accepted people from the community who can program well help to help create a deobfuscator that can tackle JAGeX’s various annoying obfuscation, like arithmetic complicating, shift complicating, dummy arguments, dummy variables, method counters. Things we CAN’T reverse are things like name obfuscation or ‘class spreading’, where classes are split up into several static methods, like what happened to the World/SceneGraph class. The main reason is because deobfuscators aren’t particularly hard to make, they are just very boring because you have to implement all these different transformers. Hopefully having multiple people will speed up the process and make it less of a bore. Anyway, who thinks this is a good idea?

Edit: Forgot to mention, also because writing an AST parser is going to be tough
