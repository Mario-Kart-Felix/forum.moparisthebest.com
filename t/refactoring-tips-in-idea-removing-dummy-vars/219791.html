Just be sure to remove it straight from Class3 (un-refactored class name which contains the huge static initializer (mostly dummies)) as well. It’s initialized there in a format such as:

ClassName.varName = 0; (as Lothy described).

So when you see a dummy inside of a method rather than directly in it’s parameters, perform the following:

  1. Find the field’s declaration (in Eclipse click on the variable and hit F3).
  2. Remove the declaration, and you should end up with a total of two errors.
  3. Identify each error and remove the line that causes it.
    3a) The first would be something along the lines of aVar++; (cannot find symbol)
    3b) The second would be something like ClassName.aVar = 0; (cannot find symbol)

And a quick (almost exactly the same) guide on performing the same task described by Miss Silabsoft in Eclipse:

  1. Find method and click on it’s name/body.
  2. Right-click, point to refactor, and then click Change Method Signature.
  3. Find the dummy variable and and click on it in the list given to you on screen.
  4. Click the Remove button and hit OK (continue through the rest of the procedure).

Eclipse shortcuts for refactoring/removing dummies (keyboard shortcuts are not the only way to use the features) :
ALT+SHIFT+R = Rename the current selection.
ALT+SHIFT+C = Change the currently selected method’s signature.
CTRL+SHIFT+G = Find all references to the current selection.
F3 = Find the declaration of the current selection (where the selection is declared).
