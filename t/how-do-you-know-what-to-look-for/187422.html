Unhandled July 30, 2016, 8:27pm #4
solarwind:

2. How did they make the renamed deobs in the first place? How did THEY know what to look for?

You can’t obfuscate your calls to the Java libraries, in File f = new File(), you can’t obfuscate the “File” part, just the name. So if you saw a line of code like this

File aFile999 = new File(aString999 + "something.dat");

You might be able to deduce a few things. One being that it was part of loading something from the cache, because dat files are stored in the cache, and that aFile999 must be an instance of File, representing something in the cache. Another might be that aString999 could be the absolute path to the cache folder on the disk, because the File constructor accepts an argument of a location of a file.
If you run the client in a debugger, with a breakpoint set on that line of code, you would be able to see the contents of aString999, which, assuming your deduction was correct, should be something like “C:\WINDOWS.jagex_cache_32” or “/tmp/”.
Of course this is rather basic, and it gets more complicated as the obfuscation gets better, but you get the general idea.
Sorry if what I explained was a little too obvious :$.

Comment out stuff and see what changes is a very basic one, can be useful, but kind of crackish cause you dont learn that much.

Put loads of System.out.println() for variables and see how it changes as the client runs.
Use a java debugger to see how variables change, this is a fancy System.out.println(), and you can do things like suspend threads

the Eclipse IDE has this really great feature they call Hot Code Replace, its basicly using the API call redefineClass(), the way it works is, you have your program running, and if you edit the source code, Eclipse will compile the and replace it in the already running JVM as though nothing happened, its a really great feature for general java developing, and works very well for rs2 client hacking.

of course Intellij IDEA probably has this feature too, but it has to rely on the javac compiler, Eclipse has its own compiler, which means its can be very selective about what it does, it will only compile the few lines you’ve just edited, while poor javac and IDEA have to recompile everything from scratch.
this means Eclipse’s compiler does its work before you even move your mouse, and IDEA has a short pause.

this quick compiling really allows Eclipse to take advantage of that API call redefineClass()

oh btw, with IDEA you can choose between javac, Eclipse’s compiler and an IBM one iirc, but with all of them you have to compile everything from the beginning.

another thing that Eclipse has which IDEA doesnt is Open Sourcing, with Eclipse, if you have any problem using the debugger or anything, you can easily got to its IRC Channel, where there are many people happy to help you, there is also a community-maintained wiki, and a massive forum.

[quote=“joey0101, post:19, topic:187422”]OK, thats all fine and good but… the deober is broken.[/quote]Fixing a Regex Renamer isn’t a super difficult task. I have one which is works to a certain degree which I will be more than willing to upload for you guys.

As for finding what you want, I respect your idea of not ripping other people’s code. Although I believe in Open-Sourcing, I do not believe in code ripping.

First, you will have to understand how the client works. Now, just like if you were trapped on an island, you have to realize that it could take some time depending on how good of a learner you are.

Now, what I am about to suggest goes against what you are trying to do, however it will help you in a the long run.

Find an updater output posted on these forums and download the runescape JAR for that output.

Now, for each field which you want to investigate, go to the class involved and do a series of System.out.println’s. Figure out how the fields and classes work together. As you work, start renaming your own methods, and start to heavily comment what does what.

At this point, you have all you need to start making your own patterns for the fields which you were investigating.

I would suggest trying to find a renamed Deob but it can sometimes be a hard task. Therefore, I suggest you work with a fresh unchanged client. Not only will it get you started right away but it will also show you how the obfuscated client works and give you an idea how Jagex’s obfuscation works. Which is also very important.
