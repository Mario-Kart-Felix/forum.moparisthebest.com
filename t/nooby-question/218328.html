The programming style of c vs java is barely comparable in alot of areas. Theres being able to program in a language, then being able to program well in that language. Especially since c is in not oo, the code you will be producing will probably make the majority of java developers balk.

However, since you have said you are used to c, then skip straight to effective java (its basicallys a pitfalls/good practice guide, I dont know any developer that has read it and said it was a waste of time). Since you come from a strong c background, your design pattern knowledge may be limited, so that part is still good advice.

And regards to how many rsc sources there are, if you dont like what you are using - roll your own. When I share code I hope that people learn from it, rather than hack at it and not understand what is going on. I know it may be daunting to start from nothing and get somewhere, and you have to in some cases answer some tough design questions that may or may not bite you in the ass down the line depending on your knowledge of the problem domain… but its where the true fun lies.
