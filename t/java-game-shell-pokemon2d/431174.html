This is the version of Pokemon2D on my Arsenal thread before I worked on the walking a bit. I’m still going to continue that project, but this being a big stress period during college, I’m forced to spend less time towards this, and more time towards my education. So, seeing as this covers some novice things needed for people to start building their game. I thought I would release so that others can learn from it, and even improve it. I’m not a professional programmer, as expecting to get code equal to one would be rather silly. Things can be improved and many things are in the middle of being changed out. Ex: Spriteloader was being removed which just grabbing images with the default tool kit, but I got caught up with other things and was distracted from finishing.

Please before running just go into Fetch.java and change this:

public boolean isDownload() {
		return isDownload;
	}

To:

public boolean isDownload() {
		return false;
	}

[list] The trick to making the buttons appear to change as they are clicked, is to change the image opacity to like 60 or 70 percent and then setting the button’s value for opaque to false.[/list]

[list] If you are interested in future development involved with Pokemon, look into the PokemonRipper class in org.pokemon.util. It dumps many sprites needed in development with pokemon. Changing the STRING_INDEX value, and adding more addresses to siteUrls array, will allow for even more sprites. Would also recommend changing the folder it saves to “cache/download/test” each time you dump as well, or you will just be overwriting files with the same name and not getting anywhere. [/list]
