[/quote]By not having 500 half-assed, badly implemented plugins, but instead having nearly all extra content officially updated by Jetbrains to comform to high usage standards.[/quote]Any plugins you don’t like can be disabled. Personally, I used IDEA for about an hour until I came to the conclusion that it was very similar to JCreator (which I used for like 6 months) and dropped it. As opposed to IDEA, eclipse is 100% open source (unlike the ‘ultimate edition’ for idea) and contains integrated features that just completely eliminate IDEA from the competition. The method/variable auto-completion feature, the on-the-fly javadoc parsing (and display of it when using the auto-completion feature), and the advantage that it uses its own compiler and therefore provides much more flexibility than if it were to use javac, for example. Also, the compilation-upon-file-save feature is very useful in my opinion, along with the red (upon error) / yellow (upon warning) underlining in the file. I know other IDEs, like Netbeans, which have very similar features, but I’m quite sure that IDEA doesn’t have all these (if any at all). In addition to that, eclipse is very light-weight in comparison and provides many advantages if you program other languages as well, since with a simple plugin, full support for that language would be integrated. I, for one, use eclipse for developing C/C++ for the reasons above AND because unlike any other C IDEs I know of, eclipse doesn’t force you to use a certain (and possibly outdated) compiler and instead lets you choose the one of your preference.
