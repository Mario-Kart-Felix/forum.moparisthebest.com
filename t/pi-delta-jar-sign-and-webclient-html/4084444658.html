Center elements are not used within head or html elements. The reason being is the head element is a container for the other head elements such as: , , , …Etc… Head elements are containers that instruct the browser where to locate styling sheets, external JavaScript files, and more.

The title tag will not align to the center because it’s defines the title name of the web document, it doesn’t have formatting functionality.

A correct HTML document would look like this.

[code]

This is a correct document [/code]

You can use the element after the body element but not within the head elements, it won’t work. I’ve also noticed you’ve used a division and a table cell with a id so it can be styled with CSS. You haven’t defined a table or table rows, only one cell. The browser will not know how to interpret and display that element. I’m not sure why you used a table cell with a style id or a division unless you’ve included CSS which I cannot see, unless of course you meant for the division element to define sections.

Here is a much cleaner looking document, I’ve changed the indentation so it all matches. Removed unnecessary elements and styling ids, and added a internal CSS to separate content from style. There is also a html comment in case you want to know what the division is.

<html>
<head>
<title>Server name here</title>
<style type="text/css">
body {
background: #000;
}
</style>
</head>

<body>
  <div id="gameframe">
    <applet name="APLET_NAME" archive="Public Link Here" code="client.class" height="503" width="765">
       <param name="java_arguments" value="-Xmx1024m">
    </applet>
  </div> <!-- end of division "gameframe" container for the Java applet -->            
</body>
</html>
