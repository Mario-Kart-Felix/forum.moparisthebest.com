Attaching a snippet of code where the exception is occurring would make it easier to identify however,

ArrayIndexOutOfBoundsException occurs when the index you reference in your array does not exist. The most common reasons this will occur is
A. The array itself is null (it has not been initialized )
B. You generate a dynamic index but do not test to make sure it falls within the bounds.

To ensure this does not happen in your code. If you pass an array into a method, then the first thing the method should do, is test that the object is not null. If you are dynamically deriving the index, you should always test that the value of the index is less than the array length.
