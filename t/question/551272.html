You’re pretty close.

This is a server to client packet error.
A t1 packet error means the packet doesn’t exist in the client.
A t2 packet error means the client had a problem reading the packet. This one is a bit more “tricky” as it displays a bit more information.

[quote=“peterbjornx”]T2 Message format:
T2 - , , - , , - [/quote]

The solution would be to go into the client and find the packet (Assuming for a T2 error) and make sure the data you’re sending is the same it’s expecting.
If the client expects 3 bytes, then you should be sending 3 bytes from the server. If it expects a String, then you should be sending a String. Etc.
