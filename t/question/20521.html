Weibs July 29, 2016, 11:14pm #2

Well, first off, after your if statement checking to see if you have died, you need a begin:

if(wearedead)then begin writeln('We Have died..') writeln('Logging out and terminating script') Logout; ProgressReport; TerminateScript; end;

When using If statements, if you don’t put a begin after it, it will only execute the first line afterwards, but if you add a begin, then it will execute everything between the begin and end;. What you had before would’ve wrote the line ‘We Have died’ only if wearedead returned true, and then no matter what, it would’ve wrote the line 'Logging out and terminating script, and then terminated the script.

As for the continuous chopping, I think it may have something to do with your chop procedure, I’m guessing the script is getting stuck in a loop there. If you want you can pm me that procedure and I’ll take a look at it.

Weibs July 29, 2016, 11:33pm #15

I’m guessing color is the symbol color that you want the user to either manually pick and enter it as a const, and if they don’t want to pick it, it automatically finds the color and replaces the const with it. If thats the case, you won’t be able to change the const, so it wouldn’t work, but, you could do a little ConstToVar converter at the beginning of the script, like this:

[SCAR]const
//The user defines the symbol color here:
TreeSymbolColor = 0;

var
//The variable that the TreeColor const will be put in is defined here:
TheTreeSymbolColor: Integer;

//----------------------//

procedure ConstToVar;
begin
TheTreeSymbolColor := TreeSymbolColor;
end;

//And then you could add the color picking procedure like this:

procedure GetTreeSymbolColor;
begin
if TheTreeSymbolColor = 0 then
begin
if (FindSymbol(x, y, ‘rare tree’)) then
begin
TheTreeSymbolColor := GetColor(x, y);
end;
end;
end;[/scar]

I hope that was what you were looking for.
