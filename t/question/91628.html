zstars July 30, 2016, 8:54am #9

Back in RSC a massive exploit was discovered, which allowed people to basically “create” items. Of course, it was fixed soon, and lots of accounts banned, but still a lot of cloned phats remained. This, however, was because the server didn’t use to check if a certain packet it received was legitimate. A similar exploit, was the one that allowed people to attack banker NPCs.
These exploits, however, are just based on server bugs (failure to check the legitimity of the content of certain packets), and they are very hard to find, especially without a proper packet sniffer / sender tool, and in fact it is well possible that Jagex managed to get rid of all of these exploitable bugs.

Note that a character uploading exploit has close to none chances of existing because the only character uploading is done from the Server to the Database, there is no server packet check that could fail. Besides, even if Jagex was on drugs, and they had overlooked an exploit (which here is kinda impossible anyway) we couldn’t discover it (without actually attacking their servers, which you better don’t do…), as it all takes place behind the scenes. A character uploading exploit is supposed to have existed in Blizzard Entertainment’s Diablo 2, but it’s just a rumour, and it would have been caused by a different reason.
