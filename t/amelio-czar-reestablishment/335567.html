[center][size=24pt]Amelio: Reestablishing Czar[/size][/center]
[center][size=20pt]Current Version: 1.1[/size][/center]

[size=12pt]Goal:[/size]
I’m going to say this once, expecting you to have logically read the first line in this thread: I love czar, and I don’t care how unstable and obsolete you think it is. The whole goal of this project is for me to learn the basic core and generics of Runescape Private servers. No matter if it’s Czar, Devolution, or something completely different from wL such as Hyperion, they all follow the same format and design. I’m hoping that by sifting through Czar (Allstar) for a long enough time, I’ll become accustom to the methods used and learn more efficient ways to process them.

Sub-Goal:
I understand there’s a large portion of the RSPS scene that still uses Czar like I do. I’m hoping that by making the source more stable, I’ll provide a more convenient window for that user-base in order to suit them with the source they prefer.
[hr]

[size=12pt]Progress:[/size]

Size:
So far I’ve gotten the whole source down to ~2.5Mb. That’s incredibly small compared to Allstar’s 10Mb - 12Mb (forgot the original size; if anyone can post a pic of the original size that would be great).

[center][/center]

I’ve also gotten the client class to 652Kb. Once again, that’s less than half the size of Allstar’s original client class size (around 1,400Kb – once again a pic with original size would be great). I’ve also removed 5 useless folders: [font=times new roman]connectedfrom, flagged, flaggedauto, logs, and tools[/font]. I either distributed the information to other folders or just completely eliminated their use.

[center]

v3gh0y.png367×509
[/center]

Optimization:
It’s been a while since I’ve been in the RSPS scene, so just getting back sort of handicaps my capabilities. I haven’t done too much other than reduce the number of item classes by merging them into two rather than four. I plan on getting it down to one. There’s an incredible number of methods that can be optimized by simply using an array set up. I’ve also created a new class for commands (though I don’t suggest having many commands at all). Other than that, the only thing I’ve done is merged most of the “moreinfo” save-data into the “character” save-data so that eventually the server will run off of just the character folder.
[hr]

[size=12pt]Future and Resources:[/size]

Future:
As I’ve said, I plan on getting this project to the greatest potential Czar has. For next step I plan on rebuilding some skills using arrays rather than the redundant if-statements that Allstar and Czar are so famous for. I also plan on merging all the character save information into one directory rather than having both “moreinfo” and “character”.

Download:
If you want to download the source and perhaps help contribute to the project the link to the thread containing the current release will be provided below. I don’t plan on having a “team” for the project, but a few “hands-on” suggestions could help.
