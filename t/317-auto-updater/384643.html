Have it connect to a website and have the site check for a version number, (this is upon login), and send a variable to the client. Setup one of the login return variables to return. “A new update is available.” if a certain number is returned from the site (new update true or false). And don’t forget to have the client open up the url that is linked to your client cache and files.

What you would have to do: Setup a url from which the file or update can be run. Make a simple program that will replace the old files with new ones. Call it an updater. Everything else is pretty simple.

OR you can do the easy thing have your client be a JNLP. And Just setup a URL to execute it. You can just constantly update the JNLP and have the cache download like a webclient would. The client can check the cache edition pretty easily.

METHOD 2 is much easier.
