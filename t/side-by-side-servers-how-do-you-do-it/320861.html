darscape July 31, 2016, 1:41pm #9

Welcome to java programs on linux. For some reason, Windows handles java in RAM far better and does not need to use anywhere as much.

Your choices are: play around with the <jvmarg value="-Xmx512m" /> until it has just enough to fit both servers in RAM without having them crash once having run out of allocated memory, have Windows Server 2003 or
2007 installed and operate through remote desktop with the ability to use your RAM in the most optimal way, or you can add additional gigs of RAM. Once your listening server is running, you just have to worry about the world servers connecting to it locally. Use different ports for each world server or if you have several static IP addresses on your server, bind each world server to a seperate IP and keep using the default port.
