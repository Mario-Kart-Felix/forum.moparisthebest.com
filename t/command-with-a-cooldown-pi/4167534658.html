make sure to change all of the ints back to long. System.currentTimeMillis() returns a long.

characterfile.write(Integer.toString(p.cooldown), 0, Integer.toString(p.cooldown).length());

change that to

characterfile.write(Long.toString(p.cooldown), 0, Long.toString(p.cooldown).length());

and change

p.cooldown = Integer.parseInt(token2);

to

p.cooldown = Long.parseInt(token2);

Should fix you up.

Oh and the command’s checks are wrong, should be like this:

if(playerCommand.startsWith("vote")) { if (c.cooldown < System.currentTimeMillis() && c.vote >= 0){ c.cooldown = System.currentTimeMillis(); c.vote ++; c.getPA().sendFrame126("www.oloadx.smfnew.com/index.php?topic=2.msg3#new", 12000); } else{ c.sendMessage ("You have already voted for today, please vote again tomorow."); }}

the above is assuming c.vote has to be greater than 0 for them to be able to use the command, otherwise use

if(playerCommand.startsWith("vote")) { if (c.cooldown < System.currentTimeMillis()){ c.cooldown = System.currentTimeMillis(); c.vote ++; c.getPA().sendFrame126("www.oloadx.smfnew.com/index.php?topic=2.msg3#new", 12000); } else{ c.sendMessage ("You have already voted for today, please vote again tomorow."); }}

for just the cooldown check.
