The Java docs can me the most powerfull asset to a good java programmer, but it can also be a very confusing thing for a new one.

First off the latest docs can be found at:

http://java.sun.com/javase/6/docs/api/

http://img149.imageshack.us/my.php?image=screenshotdu4.png
This is the main page, and the start of our small but educational journey.

We now choose our package.Different packages allow us to do different things.Got example java.io allows us to do input and output whilst java.maths allows us to do maths equations more easier.We are going to choose java.lang which is there by default.Now we are prsented with numerous things all of which are usefull i nthere own regard.Basicly we choose what type of thing we wan’t to do and click the interface or class that has what you wan’t do do with it included.So say if we wanted to do System operations like telling the time, we choose System in the class summary.http://img246.imageshack.us/my.php?image=screenshot1vs3.png <-- (scroll down a slight bit).

In this area of the documents we get a description, method summary and field summary as ell as a few other tools like inherited methods.The Field goes after the class when using it.

System.out.println("Hello");

System being the class, out being the field and println being the method.Scroll down to method detail and find a random one.In some cases the field name isn’t needed, like for setProperty().

The console() detailed method tells us what it returns, when it was first implemented , paremeters and finally what it throws.These are all usefull for a java programmer.
