(This is all off my previous knowledge from when I first started programming a delta over 5 years ago)
For thieving stalls, you’re going to want to add code into your first click object packet. Delta uses client.java to handle all of it’s packets. So you would search the packet id (You can find most information on packets and the 317 protocol here) 132.
If your delta has the packet id’s in a switch statement, then you’re looking for “case 132”. (If you don’t know what a switch block is, Read Me)
You should have the object id, x, and y ints in there. Underneath them, if there isn’t already one, add a switch block for your object id. Then inside that switch block add a case for your stalls. Take note of the stalls: Rotation, X, Y, Z (height), and what it has on it (cakes, fish, etc).

This is where it gets tricky
You’re going to want to make a new class. (What is a class?)
I’d title it something like Thieving or ThievingHandler. (Proper conventions suggest you title a class with an uppercase letter for each word)
In that class you’re going to want to store the data for all your stalls. You can do this in a multiple of ways: Arrays, enum(s), HashMaps, etc.
From there, I’d make a method for the following:
Check if the object clicked is a thieving stall.
Replacing the stall with an empty one, then back (assuming you don’t already have an object method that does this) in X time.
Giving the user the items and exp, unless they’re caught.
What happens when/if they’re caught.

After that, you need to add it to your first click object packet.
If you made the methods and such static, you can call it like so:

case 132://  First click object packet
     //  The ints objId, objX and objY are received via stream from the client. Imagine they're here instead of my comment.
     Thieving.yourThievingMethod(int objId, int objX, int objY);
     //  The rest of the case here
     break;

I told you to make the switch statement for the stall ids, but you didn’t need to because I realized you can just have a method check if it’s a stall id or not. Sorry.

However, that should be a good overview on what you should do to make thieving. I will help with the others after you’re done with thieving. Take it one step at a time.
