Is it a packet sent from the client to the server? I thought it was one sent to the client?

Anyways, to get this effect without modifying the client, and requiring all of your players to download it again (if you don’t have an auto updater), you can do it on the server side.

In your Player class, you can add a field

public class Player {

    // ...

    public long getTimeLastPacketReceived() {
        return timeLastPacketReceived;
    }

    public long updateTimeLastPacketReceived() {
        timeLastPacketReceived = System.currentTimeMillis();
    }

    private long timeLastIdlePacketReceived;

And then in your packet handler

public class IdlePacketHandler {

    public IdlePacketHandler(final long timeout) {
        this.timeout = timeout;
    }

    public void handle(final Player player, final Packet packet) {
        final long timeDifference = System.currentTimeMillis() - player.getTimeLastIdlePacketReceived();
        if (timeDifference >= timeout) {
            player.getActionSender().sendLogout(false);
            player.updateTimeLastIdlePacketReceived();
        }
    }

    private long timeout;
}

Then you can specify the timeout as

new IdlePacketHandler(10000); // number of milliseconds, this is 10 seconds

I agree. But as I said, if he doesn’t have a “good” way of handling code updates made to the client, then (if he has a sizable player base) players will likely get the impression that the client/server is unstable and turn away from it. If he doesn’t care, then by all means change the value in the client. In other games I have played, and other applications I’ve developed, disconnects like this are better handled on the server side, because then only one aspect is changed.

The server can control the idle timeout from 1 minute to 10 minutes, to 30 seconds, and no user has to do anything on their side; which is really what making an RSPS is usually about. It’s not the best exercise in learning how to program (and certainly not how to program well in most cases), or learn about how networking actually works, and it’s usually not for your own personal use (few people actually host a server whose maximum expected population is 3). It’s about getting as many players as possible in your game, which translates to getting as many dollars flowing into your bank account as possible (via “donations” or “premium features” and ad revenue).
