Description - This is a tutorial explaining the usage of switch statements in RuneScape Private Servers.

Difficulty - This is a pretty easy concept, so I’ll say 1/10.

Tested Server - This will work on any server, undoubtedly.

Alright, let’s get started.

First off, you’re probably wondering, ‘What is a switch statement?’. A switch statement is a statement that loops through the cases and executes a case if called.

Here are some rules to remember. The switch keyword is always followed a parenthesized expression that relates to an integer.

Example:

switch(playerRights) {

‘playerRights’ is an integer expression, because the possible values of the variable can only be numeric.

The switch expression is always followed by a series of cases. In this example, each case specifies a player right.

Example:

[code=java]switch(playerRights) {
case 0: // playerRights 0 (Normal player)
// Send a message meant for normal players
break;

case 1: // playerRights 1 (Moderator)
// Send a message meant for moderators
break;

case 2: // playerRights 2 (Administrator)
// Send a message meant for administrators
break;

case 3: // playerRights 3 (High administrator)
// Send a message meant for high administrators
break;

case 4: // playerRights 4 (Hidden administrator)
// Send a message meant for hidden administrators
break;

case 20: // playerRights 20 (Root administrator)
// Send a message meant for root administrators
break;
}[/code]

As you can see above, each integer after the case keyword indicates a certain player right. Also, there must be a break; after each case. The break ensures that the execution of one case does not flow into another.

Now, onto a few examples of how and where a switch statement can be used.

Take this command, for example.

if(command.equalsIgnoreCase("staffitem")) { if(playerRights == 0) { sendMessage("You are not a staff member."); } else if(playerRights == 1) { addItem(1038, 1); } else if(playerRights == 2) { addItem(1040, 1); } else if(playerRights == 3) { addItem(1050, 1); } else { sendMessage("You are a hidden administrator. You do not receive any items."); }

We can use a switch statement instead.

[code=java]if(command.equalsIgnoreCase(“staffitem”)) {
switch(playerRights) {
case 0:
sendMessage(“You are not a staff member.”);
break;

case 1:
    addItem(1038, 1);
    break;

case 2:
    addItem(1040, 1);
    break;

case 3:
    addItem(1050, 1);
    break;

default:
    sendMessage("You are a hidden administrator. You do not receive any items.");
    break;
}

}[/code]

As you can see, the switch statement makes the command much easier to comprehend. The default clause is executed when none of the case values apply to the player. A hidden administrator’s player right is 4, and there is no case for it; therefore, the default clause is executed. The default clause serves the same function an else statement.

Here is another example: Item on object.

if(ItemID == 526 && UsedOnObjectID == 6552) { addSkillXP(100, 4); deleteItem(526, 1); } if(ItemID == 528 && UsedOnObjectID == 6552) { addSkillXP(150, 4); deleteItem(528, 1); } if(ItemID == 530 && UsedOnObjectID == 6552) { addSkillXP(200, 4); deleteItem(530, 1); } if(ItemID == 532 && UsedOnObjectID == 6552) { addSkillXP(250, 4); deleteItem(532, 1); } if(ItemID == 534 && UsedOnObjectID == 6552) { addSkillXP(300, 4); deleteItem(534, 1); } if(ItemID == 4834 && UsedOnObjectID == 6552) { addSkillXP(500, 4); deleteItem(4834, 1); }

Instead of using so many if-statements, we can use a switch statement.

[code=java]if(UsedOnObjectID == 6552) {
switch(ItemID) {
case 526:
addSkillXP(100, 4);
deleteItem(526, 1);
break;

case 528:
addSkillXP(150, 4);
deleteItem(528, 1);
break;

case 530:
addSkillXP(200, 4);
deleteItem(530, 1);
break;

case 532:
addSkillXP(250, 4);
deleteItem(532, 1);
break;

case 534:
addSkillXP(300, 4);
deleteItem(534, 1);
break;

case 4834:
addSkillXP(500, 4);
deleteItem(4834, 1);
break;

default:
sendMessage("You must use a type of bone on this altar to recieve experience!");
break;
}

}[/code]

As you can see again, the switch statement makes the portion of code much easier to comprehend. The default clause is executed when the player uses an item that is not specified as one of the cases on the object.

Some things to keep in mind -

  1. Always remember to end each case with a break; unless you use a range of cases.

  2. A switch statement requires integers, and does not accept a String type. For example, you cannot use a switch statement in this case:

[code=java]if(command.equalIgnoreCase(“Names”)) {
switch(playerName) {
case Jc22493:
sendMessage(“Jason”);
break;

case The Soul:
    sendMessage("Conor");
break;

case Chassy13:
    sendMessage("Danny");
break;

case xx qwertyuiop xx:
    sendMessage("EvanEvanEvanEvanEvanEvanEvan");
break;

case X Blitz X:
    sendMessage("Noob <3");
break;
}

}[/code]

You will receive errors while compiling, because the value after the case keyword is not an integer.

  1. You cannot enter a range for the integer. For example, you cannot do this:

if(UsedOnObjectID == 1000) { switch(ItemID) { case 1-5: addSkillXP(500, 0); break; } }

Instead, do this:

if(UsedOnObjectID == 1000) { switch(ItemID) { case 1: case 2: case 3: case 4: case 5: addSkillXP(500, 0); break; } }

  1. Do not have two of the same cases under one switch statement, or you will receive a ‘duplicate case label’ error.

This ends the tutorial, and thanks for reading! I hope you have learned something new today, and will use this programming device in the future. Please reply with any questions or concerns.
