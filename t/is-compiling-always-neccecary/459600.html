Fridder August 4, 2016, 12:05am #8

I’ll try to make sense of this for you.
Your computer reads the output code (called the bytecode) that is generated from the compiler. The compiler takes the ‘.java’ (class) files and turns them into ‘.class’ files. You can read the ‘.java’ files, but your computer cannot. Your computer can read the ‘.class’ files, but you cannot.

Every time you modify a ‘.java’ file, you need to re-run the compiler, which basically overwrites the old ‘.class’ files with the new ones that were generated from the compiler.

In short, you need to build (compile) your application in order to execute it.

The compiler transforms the source code into ‘tokens’. In our case, ByteCode is the ‘token’ which is the form of instructions that the JVM will execute.

You as a programmer do not need to understand Java ByteCode at all.

Read more here: http://en.wikipedia.org/wiki/Java_bytecode and here: http://stackoverflow.com/questions/1419269/what-happens-when-i-compile

m.dog311 August 4, 2016, 12:08am #20

The Java Virtual Machine (JVM) loads & interprets the bytecode of the class files at runtime, and translates it into machine code so that it can be executed. The java files are nothing more than a “user friendly” set of files that we can actually work with. The compiler builds the java files into executable class files. The reason you don’t have to compile when you edit things like your player saves, is that your player saves aren’t java. There is code in your server that loads and processes them, and they are loaded as they’re needed (not just at runtime).
