zyle1992 August 4, 2016, 2:16am #5
3lv3n snip3r:

Make it loop.

Depending on how you do that, you could end up in a deadlock. For example

while (player.isAFK()) {
    player.performAnimation(SLEEP);
}

(note: you should have a field for being afk, because it is definitely a change in state)

I’m not sure if the client supports “set until not” functionality, but if not, you’ll have to mark the player as performing the animation every tick.

zyle1992 August 4, 2016, 2:16am #7
Clawz fury:

how would using a while loop result in a deadlock? I thought you’d have to be using multiple threads to get a deadlock??

That is the most common form of a deadlock. But in this sort of environment, you run into this

1. Check if player is afk
2. Perform animation
3. Go to 1

Since that’s all the thread would be doing, it would block any other program execution.
