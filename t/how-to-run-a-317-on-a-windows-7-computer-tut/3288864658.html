There is an easier way to run a 317 on a windows 7 computer… without downloading some damn “converter”…

Change your environment variables, especially if your running on a 64 bit machine and if you are on 32 bit Windows 7 there should not be any problems. If your do not want to set your environment variables you can even just do this…

Edit the “run”
Lets focus on a code that may look something like this

-Xmx300m -cp . Class51 30 0 highmem members 32

If you were on 64 bit and ran a 317 client like this it would immediately shutdown on you. Were missing our 64 bit path.
“C:\Program Files (x86)\Java\jre6\bin\java.exe”
Now we can not just paste this over the code we have, some people try this and it does not work or they try to copy+paste the code of the client they configured to work for 64 bit in another client. But not all clients have Xmx300m. Not all clients are the same (most are, but the original ones aren’t).

In conclusion our full code would be this (THIS IS JUST AN EXAMPLE READ ABOVE, THIS MAY NOT WORK FOR YOUR CLIENT IF YOUR RUNNING ON A 64 BIT OS!!!)
“C:\Program Files (x86)\Java\jre6\bin\java.exe” -Xmx300m -cp . Class51 30 0 highmem members 32
