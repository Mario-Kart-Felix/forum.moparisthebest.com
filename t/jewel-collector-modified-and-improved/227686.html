Jewel Collector is a game developed by myself, graphically created by Archer, and had sound done by Adam Cooper. It is a simple arcade game that was started a few days ago, and all in all, took 3 days to complete.

You can download it here:
http://jc.telenubby.info/JewelCollector.jar

You can play it as an applet here:
http://jc.telenubby.info/index.html

You can view the documentation on it here:
http://jc.telenubby.info/doc

The source, binary files, resources, and documentation are also included inside of the Jar file. The source code is not licensed, so you may edit and redistribute it freely. However, we ask that you still accredit us for creating the original version, but you are not forced to do so.

I have fixed all of the bugs and added all of the updates you commented on for version 1, so if you have anymore, let me know and I will see what I can do. This game is not perfect, but it is pretty good and is quite fun.

[quote=“supamagier, post:14, topic:227686”]Yea… indeed.

How come the music only breaks in the applet?[/quote]

I am using the default applet music system when playing music. When running it locally, the applet wrapper class (JCFrame) implements the media classes for the applet and uses the media system from [tt]javax.sound.sampled[/tt]. Apparently the default media player in the applet system that the browsers uses doesn’t behave the same way the javax.sound.sampled does. Doesn’t really matter though.
