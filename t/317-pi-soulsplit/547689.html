If you want it to work properly, I would consider using a cycle event. That makes it so the player has to complete the entire healing process before being able to do it again. It will also (if done properly) prevent players from being able to spam the button, causing them to mass heal.

With what you’ve provided, really nothing can be done.
And you really shouldn’t be worried about “nubs will try to rip it”. I doubt there’s any code that you have that’s “special” or even worth taking that isn’t already released or can be figured out.
I mean, it’s soulsplit. There’s only so many ways to accomplish it.

Here’s a cycle event:

CycleEventHandler.getSingleton().addEvent(this, new CycleEvent() { int timer = 20; @Override public void execute(CycleEventContainer container) { if(timer > 0) timer--; else container.stop(); System.out.println("Timer: " + timer); } @Override public void stop() { // Runs when the cycle event STOPS. hence it's name. System.out.println("Event stopped."); } }, 1);

Every server tick (600 ms) that will count down from 20 to 0. At 0 or lower, it will stop.
You can change how many ticks it takes for the event to run by changing the “1” at the end of the code.
The only downside is that it doesn’t run every second. So if you wanted a second by second update, this may not be the best solution.
But luckily, the entire server is designed to run on the 600 ms tick. So you shouldn’t have that issue ;).
