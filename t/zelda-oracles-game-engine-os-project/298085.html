Well… In case you haven’t noticed, I love Zelda (Lin… k) and I’ve been really wanting another GB© adventure, but it never happened. I once made a decent engine in Game Maker, but the only problem was… It was made in Game Maker…

Anyhow, I’m trying to create a simple-to-work-with GBC Zelda Engine to work with (Link’s Awakening (DX), Oracle of Seasons/Ages) so people can easily create their own games with it. It’s being done in C# and since I’m no master with DirectX or anything else, it uses the GDI+.

My last game didn’t go so well, as it was single-threaded and I did some things a horrible way… But I’ve learned from my mistakes and trying to keep this one decent with performance. So far, it’s not going bad (After about 2 days maybe?)…

Progress:
-Tile engine done.
-Allows tile animations. The animations are loaded from AT.dat and use the AT.bmp files for the tiles.

To-do:
-Allow the drawing engine to handle views (Like when scrolling from room to room)
-Everything else… Sprites, combat, full header, music, sounds, combat, collisions (yuck)

I admit there’s things I could’ve done better, and stuff I could’ve removed, but like I said, if it works, and well, then no need to change it. For now, I’m just posting so you can take a look at it… Flame me… And maybe see how you can do some things… Just don’t leech
