The client has flagged this because one or more of your data files (models / sounds) did not run through the unpacker correctly. To determine which file it is, you can add comments in your client’s body.

Example:

protected final byte[] load(String fileName) {
	if(!(new java.io.File(fileName).exists())) {
		java.lang.StringBuffer buffer = new java.lang.StringBuffer();
		buffer.append("Error loading file: ");
		buffer.append(fileName);
		buffer.append(" ensure valid path and file name.");
		System.out.println(buffer.toString());
		buffer.setLength(0);
		/*
		StringBuffer is MUCH less expensive to the processor than the simple += concat operations.
		*/
	}
}

As for a solution to your problem, I’m not very good at web-based application, but have you tried dropping these files in the same directory as your codebase?
