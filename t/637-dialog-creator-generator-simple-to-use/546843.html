This looks like too much work for what you’re trying to accomplish.
I do like the idea. But how you’re going about it, not so much.

You should make a system (yes, an entire system) that would save all the dialogues to an external file (like JSON), maybe even have a folder for it…
Lets say you’re going to have a “dialogues” folder. Now in that folder, would be some odd amount of JSON files. Each file would be for one NPC or one NPC group (because sometimes you have multiple NPCs with the same name but diff IDs). And in each JSON file, would be all the dialogues for the npc(s). You could make an object for all the dialogues in a scene. Example:
(Remember this is an example, not working code rofl)

{ "npcs": [ 0, 1, 2 ], "dialogues": [ { "systemid": 0, "dialogueid": 0, "expression": "HAPPY" "phrases": [ "Talking", "about", "the", "quest", "here" ], "nextdialogue": 1 }, { "systemid": 0, "dialogueid": 1, "expression": "ANGRY" "phrases": [ "More", "about", "the", "quest", "here" ], "nextdialogue": 2 }, ] }
Of course I haven’t added all possible data in the example. Because it’s a damn example.

But here’s an explanation of the data you’re reading (most if not all is obvious):
[font=Verdana] “systemid”: The ID of the dialogues in the server. This would be used to dictate what dialogue to pull up. Ex: getNextDialogue(SystemId), and then it’d return the next dialogue that had the same systemid, but the next dialogueid…[/font][font=Verdana] “dialogueid”: The ID of the dialogue in that particular set of dialogues. “expression”: The expression your the entity would be displaying in the dialogue window. “phrases”: Obviously the data that is in dialogue window. You’ll need some form of content control here ( like no more than 5 lines of text and only X amount of characters per line). “nextdialogue”: The next ID in that particular set of dialogues.
