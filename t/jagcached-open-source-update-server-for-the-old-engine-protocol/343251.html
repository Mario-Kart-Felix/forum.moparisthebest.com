Jagcached: open-source update server (for the old-engine protocol)

General Discussion
Graham July 31, 2016, 4:58pm #1

jagcached is an update server (HTTP, JAGGRAB and ondemand) for old engine clients (officially it stands for nothing, unofficially, you might think of it as the Jagex cache daemon). I’ve tested it with the 317 and 377 clients and caches and it works fine - delivering the entire cache to them using the original protocol.

It’s released under the GNU GPL and version 1.0 is available for download here:

http://github.com/downloads/apollo-rsps/jagcached/jagcached-1.0.zip

Most of the work inside it will be integrated in Apollo, but I decided to release this independently as the only other option right now is Hyperion and picking apart the update/game code isn’t easy.

The best thing for you to do is to either integrate it with your own game server or proxy requests for ondemand updates either to jagcached or proxy game server sessions through jagcached to your server.

Or just use it is a reference/toy, which is probably the only serious use it’ll get around here since most people write their own custom cache downloaders for their webclients (this is infinitely better and needs no modification from the original deob!).

It’s hosted on github - the project page is here:

GitHub

apollo-rsps/jagcached

jagcached - An open-source file server, designed for use in a massively-multiplayer online game environment.

You can also clone the git repository using this URL (if you know what you are doing):

git://github.com/apollo-rsps/jagcached.git

It can be built with Apache Ant: http://ant.apache.org/ and uses the Netty API.

There is also a jaggrab server and ondemand server for 317 built into MoparScape 4 and released under the GNU/GPL as well, it’s always been in the source jar in the org.moparscape.userver package. My servers convert the cache request to an HTTP request and then feed that over to the client, this allows the cache to be hosted at multiple HTTP mirrors that can be hosted on cheap shared hosting, and in many cases for free. I think that is a better way to handle it from the client side so the game server doesn’t need to waste bandwidth serving up the cache, and so there is no single point of failure to host the cache making it much harder to take down.

Anyway, open source code is always good and welcome, and I am interested to see how you handled it. So thanks for releasing.

The client requests parts of the cache from an update server, the update server fulfills that request by sending the requested file back.

Also as clarification on my past post, this update server and the one in MoparScape have two different purposes. This is to serve as an update server for multiple clients to be centrally hosted. Mine is to simply serve one client at a time and translate requests to serve files from a standard HTTP server.
