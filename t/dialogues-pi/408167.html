Well, i made some dialogues for my server, and I learned it all pretty quickly. So I decided to make a tutorial for others to understand.

First, sendStatement.

case 2500: sendStatement("Beware! This portal leads to rune rocks in the wilderness!"); c.dialogueAction = 234; c.nextChat = 2501; break;

There’s an example, now let’s break it down.

sendStatement("Beware! This portal leads to rune rocks in the wilderness!");

The sendStatement is 1 of the many methods that can be used, which basically sends the statement wanted, and sends “Click here to continue.”

c.dialogueAction = 234;
You can use that for adding teleporting when they click the “click here to continue”(Will be explained later)

c.nextChat = 2501;
is used for, handling the next chat like the next case under this is “2501.”

Second, sendOption

case 2501: sendOption2("Go through the portal.", "No way! Thats too dangerous!"); c.dialogueAction = 234; c.nextChat = 0; break;
This is used for sending options(hence the name “sendOption”) and adding actions to these.

sendOption2("Go through the portal.", "No way! Thats too dangerous!");

Now, the sendOption can also be used as sendOption3, it all matters how many options you want. If you want to add 3 options it would look like this.

sendOption3("Go through the portal.", "No way! Thats too dangerous!", "Another option here");
and so forth.

Now for NpcChat methods.

case 3: sendNpcChat4("Hello!", "My name is Duradel and I am a master of the slayer skill.", "I can assign you a slayer task suitable to your combat level.", "Would you like a slayer task?", c.talkingNpc, "Duradel"); c.nextChat = 4; break;

Now the NpcChat is used for quests because they are RS like. Remember to have the c.talkNpc and “NPC name”

Now for the dialogueactions.
DialogueActions are used for an action like teleporting a player, or giving a certain item to the player, or many other things. Now, let’s make an dialogue for ::empty.
Add this into Commands.java

if(playerCommand.startsWith("empty")) { c.getDH().sendDialogues(50, 0); }

If you want an npc to handle the dialogues, just add it under firstClickNpc as so.

case NPDID: c.getDH().sendDialogues(##, 0); // replace ## with the case Id you used in dialoguehandler.java break;

Now to finish ::empty.

Go into DialogueHandler.java and add this.

case 50: // the case IDs can be used randomly but cannot be used twice. It's needed for Actionhandler.java sendOption2("Are you sure you want to empty your inventory?", "Nope!"); c.dialogueAction = 6668; c.nextChat = 51; break;
Let’s make the empty work now.
Clickingbuttons.java

is used for all the first options whilst using dialogueActions.

add

} else if (c.dialogueAction == 6668) { c.getItems().removeAllItems(); }
The dialogueAction == ## is c.dialogueAction = 6668; in dialoguehandler.java. You can also make it move them somewhere else for other dialogues, just change the c.getItems().removeAllItems();.

Somebody bumped this… so let’s finish it.

Okay now i’ll teach you how to make your player chat.

case 201: sendPlayerChat2("I'm searching for a adventurous quest.","Do you have an quest for me?"); c.nextChat = 202; break;
The first method being called is like the “sendOption” method okay?
So, the # depends on how many quotes you are using, and lines you want to send.
sendPlayerChat#(“I’m searching for a adventurous quest.”,“Do you have an quest for me?”);

c.nextChat = 202;

This obviosuly sends to the next case Id so it would send to this…

case 203: //DOSTUFF break;

ALWAYS remember to use break; or else the case Id will never stop, and it won’t end the action.
