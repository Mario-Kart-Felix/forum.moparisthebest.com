Quite simple actually.

What it looks like to me, is a new model got put in place of the dh’s wield model.

Here’s what I suggest:
Find out what model number the DH uses for the wield models (there might be a male and female, so you have to check both). I made a snippet on dumping the data in the private server client hacking, I’m not sure of the link though.

Look in your cache for a “raw” folder. If it exists, search for the ids you got earlier.
If you get results that are .dat’s or .gz’s, try removing them from the folder… Note: I didn’t say delete.
Now try loading up your client again and see what the DH looks like.

The .dat’s (or .gz) you just removed had a purpose, probably. So you need to figure out what.

Try looking in the ItemDef file (may be named Class8, ItemDef_2, or similar) for the same IDs.
This time if you find it, you need to change the values to something else… A number that isn’t being used would be best.
Now, at the same time. Rename the .dat’s (or .gz) to the same values.

Save the item def and put the files back into the cache.
Compile and run, and it should be good to go.
