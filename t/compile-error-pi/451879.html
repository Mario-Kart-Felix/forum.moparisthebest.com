Well it’s fixed, but you don’t know why it happens so, meh.

The reason why you were getting the error is most likely because you had a case outside of a closed switch statement which will cause the orphaned case error to happen. I’ll write out a simple example for you to look at and analyze.

switch(condition) { case 1: //code break; case 2: //code break; } default: //just a default case //code break; }

To fix it just

switch(condition) { case 1: //code break; case 2: //code break; default: //just a default case //code break; } }

Then it should work :).
