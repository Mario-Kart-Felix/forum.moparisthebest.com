When a timer, let’s say the actionTimer, goes down by one each time if it’s ever higher then 0 (An example of it)

if (actionTimer > 0) { actionTimer -= 1; }

When it goes down, how long does it take to go down 1? Like if it goes down 1, is that one second? So if it’s set to lets say 60, would you need to wait 60 seconds before you can do the action again?

xx_qwertyuiop_xx July 31, 2016, 8:00am #2

Assuming the process() method is called once every 500 milliseconds (which it isn’t, unfortunately I have no idea what accelerates it) it will take 30 seconds for a timer set at 60 to reach 0.

If you’re going to create timers this way, do it in the process() method in Player.java.
It’s always called every 500 ms without any increase or decrease in speed.
