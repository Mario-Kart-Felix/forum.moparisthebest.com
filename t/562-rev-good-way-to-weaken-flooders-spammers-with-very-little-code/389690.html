[color=blue]NOTE This doesn’t stop them from logging in, but they wont be in your home location, or near a large group of players, so its almost pointless for them to do this.

This is very simple and also a good base for you to do other things in your 562 revision server.

In Actionsender.java search for starter. You should see an if statement. Under the if statement put this.

player.setLocation(3200, 3200, 0); player.isTeleBlocked = true;

You can modify these coords to have the player be set somewhere else.

In my servers I put an NPC that teleports them to the “Home” Location.

You can do this by opening up Npcpackethandler.java.

Once you have this open, under private void OPTION1 put you if statement like so…

if(npc.getId() == #NPC ID#) { p.turnTemporarilyTo(npc); npc.turnTemporarilyTo(p); p.getActionSender().sendChatboxInterface(241); p.getActionSender().animateInterface(9850, 241, 2); p.getActionSender().sendNPCOnInterface(#NPC ID# ,241, 2); p.getActionSender().sendString("*NPC NAME HERE*", 241, 3); p.getActionSender().sendString("Hello and welcome to the server!"+p.getPlayerDetails().getDisplayName()+"!", 241, 4); p.tele(#COORDS HERE OF HOME#); }

Things left for you to do: Spawn the npc you want in the correct location, and find a npc that talks.

What this does: When a player logs in for the first time he is in a different location (good idea to make it closed in) and he isn’t near the major population of the server so he can’t flood advertise. Very easy and you can do more with this if you’d like.
