Unhandled July 31, 2016, 5:46am #2

A lot of what you are doing is just a bad idea, regardless of the language.
You should only paint in, well, OnPaint()
You should never call Thread.Sleep() in a UI thread, EVER.
Do you realize that every time the user moves their mouse the program will freeze for 50 ms? That may not seem like much, but that event is fired many, many times throughout moving the mouse from point A to point B.

What you want to do is maintain a list of blue and a list of green circles that will be drawn in OnPaint(), then simply clear the list of green circles when you want them to disappear and call Invalidate().

I’m pulling this all from C# experience, but seeing as you are using VB.net it shouldn’t matter.

The best way of doing this is spawning a thread and passing it a delegate method, which will be called after the new thread has slept for a specified time. So instead of blocking the UI thread, the sleep is done asynchronously.
Luckily, .NET has done this for us in the form of the System.Windows.Forms.Timer class

What do you mean timer’s can only measure 18 times per second? You set the interval in milliseconds, and there are 1000 milliseconds in a second of course. You will never be able to get down to 1000 times a second, The lowest I’ve had it reliably go is about 60 times a second. It’s just a limitation of how the OS’s scheduler works.

I was doing some benchmarks recently, and I set the timer interval to 17ms, or about 60 times a second, which it seems to be able to maintain reasonably well.
