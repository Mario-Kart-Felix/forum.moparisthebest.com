The jar can be unpacked because it’s appended to the file, so opening the executable as a ZIP file works because there’s a valid footer.
Here, I decompiled the classes using jad:

[code]// Decompiled by Jad v1.5.8g. Copyright 2001 Pavel Kouznetsov.
// Jad home page: http://www.kpdus.com/jad.html
// Decompiler options: packimports(3) noctor nonlb space
// Source File Name: Hello.java

package test;

import java.io.PrintStream;

public class Hello {

public static void main(String args[]) {
    System.out.println("Hello, world");
}

}[/code]

[code]// Decompiled by Jad v1.5.8g. Copyright 2001 Pavel Kouznetsov.
// Jad home page: http://www.kpdus.com/jad.html
// Decompiler options: packimports(3) noctor nonlb space
// Source File Name: HelloWorld.java

package test;

import java.awt.*;
import javax.swing.JFrame;
import javax.swing.JTextField;

public class HelloWorld extends JFrame {

public HelloWorld() {
    JTextField jtextfield = new JTextField("Hello, world");
    getContentPane().add(jtextfield);
}

public static void main(String args[]) {
    HelloWorld helloworld = new HelloWorld();
    Toolkit toolkit = Toolkit.getDefaultToolkit();
    int i = toolkit.getScreenSize().width;
    int j = toolkit.getScreenSize().height;
    helloworld.setSize(100, 100);
    helloworld.setLocation((i - 100) / 2, (j - 100) / 2);
    helloworld.setVisible(true);
    helloworld.setDefaultCloseOperation(3);
}

}[/code]

Jar2Exe using the “hide” mechanism instead places the archive in a type 10 RCDATA resource, which is somehow processed prior to being fed through the ZipArchive library for unpacking.
The encryption option merely adds an additional layer to this hide mechanism by employing a homebrew stream cipher named XOR256 to the data. XOR256 used to be documented here, but as you can see the page is suspended, likely because the code went proprietary. You can find it here, however:
https://scenarioframework.svn.codeplex.com/svn/src/Scenario%20Libraries/cppbOSCrypto/Crypto/bOSXOR256Stream.h
https://scenarioframework.svn.codeplex.com/svn/src/Scenario%20Libraries/cppbOSCrypto/Crypto/bOSXOR256Stream.cpp

That all being said, all of these techniques can be readily defeated using a JVM debugging agent.
