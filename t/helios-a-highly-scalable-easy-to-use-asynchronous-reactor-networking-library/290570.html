Helios - A highly scalable, easy to use, asynchronous reactor networking library

General Discussion
system July 31, 2016, 9:28am #1

Helios is a highly scalable networking API I developed out of boredom for people to use.

The library is very easy to use, it contains an example class (source is included in the jar) of how to implement and use it.
Helios is for people that want to take advantage of the asynchronous Reactor networking design pattern, without going through the troubles of implementing it themselves. You can use other networking libraries, such as Apache MINA, but in my opinion all of these libraries are incredibly bulky and you wont use more than 10% of what they have to offer.

Helios does only what it needs to do - provide a stable, fast, and lightweight implementation of non-blocking IO, following the Reactor design pattern.
I’m releasing it for you guys to use with your server frameworks/other miscellaneous networking applications/whatever you want to use it for.

Download: http://uppit.com/v/7NY0Q39T
