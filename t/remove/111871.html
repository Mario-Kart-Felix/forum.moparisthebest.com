Eh_Pretty_Easy July 30, 2016, 11:04am #2

Let’s see…public void NewObjects() { search for that then you there should be AddGlobalObject( or makeGlobalObject( under it. I’ll show you an example. AddGlobalObject(2999, 3383, 2469, 0, 10);//portal When you spawn a object, you put the void name infront of everything else, depending on which void you added (AddGlobal or makeGlobal, doesn’t really matter) after that comes the coordinates. The ‘2999’ is the X coordinate, and the ‘3383’ is the Y coordinate and after them is the object ID for a portal in this case 2469. That 0 means the way the object faces, don’t mind about it and the 10 is the height level if I remember correctly. So when you want to add a new object, you just add this inside the public void NewObjects: AddGlobalObject(XXXX, YYYY, ID, 0, 10);//whatever and fill it with the coords and ID you want. I hope I managed to explain something.

Yes, 0 means the direction it faces. But you may want to know that. Like if you were going to place an altar somewhere, and you wanted to turn it into the direction you desire. Here’s the facing direction numbers you put in place of the 0 every time you add a global object.
0= WEST, -1 = NORTH, -2 = EAST, -3 = SOUTH
