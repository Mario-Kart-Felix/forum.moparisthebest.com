[center]Intro[/center]
Ok, we’ve all seen those registration pages that tell you whether or not your password is secure. To be honest, they don’t know crap. They just check if you have numbers, letters, and symbols. Most ‘secure passwords’ is just the same password somebody normally uses, maybe with a number or symbol added to it.

[center]Using Passwords On Multiple Sites[/center]
Yes, I said normally use. There isn’t ANY REASON AT ALL to use a different user and password for different sites, as long as the owner(s) don’t go on the other site. For example: Its ok to use your moparscape password on myspace, because you probably wouldnt find moparisthebest on myspace. . . Its NOT OK to use your private server password as your runescape password. Not only is it probably the most related example i could think of at the moment, but 80+% of servers actually try to hack you. Even if its the same username, if they figure out your runescape password, you’re done.

[center]Strong Password[/center]
A good password is one like this:

YV8wqD7ryP4
I won't be able to remember that
There is easy ways to remember this, well, not remember, but never lose it.

This code: YV8wqD7ryP4

You can check by looking on the day it was added, and watching the video(:)). If the video seems like it is offensive, find another. If the video has been on for 6+ months, there is no need to find another video.

[quote=“ThatOneServer, post:1, topic:135310”][center]Using Passwords On Multiple Sites[/center]
There isn’t ANY REASON AT ALL to use a different user and password for different sites, as long as the owner(s) don’t go on the other site.[/quote]

Erm, wtf?

Think about it for a second. The chance of a website having a vulnerability that allows random people access to passwords is what? Let’s call it 0.5%, just to pull a random number out of our ass. Now, say you use that password on 200 sites. That makes 100%!

The more website you use a single password on, the more you are increasing your chance that the password will be discovered and the more problems you are creating for when it eventually does. It has nothing to do with the owner of the websites.
