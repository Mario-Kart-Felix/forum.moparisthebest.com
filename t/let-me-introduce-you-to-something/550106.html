It really irritates me the level of negligence people have when designing/coding website templates. Unlike many other languages, things may work in one place and not in another in web design. Let me reiterate; your website will display differently depending on:

Doctype
Screen resolution
Device
CSS code

So, the purpose of this thread is the following: Let me introduce you to some sh*t that you should have learned on your first day.

Doctype:

The minimal doctype required is not . If you don’t have a doctyp your browser will go into quirks mode and things will display weird.

508 Compliance & Accessibility:

Make sure your website can be used by the disabled. Some people may be def and using instead of will not get your point across. Read up more here

The Difference Between A Class and An ID:

Look at the phrases: Class and ID. Think about them, what is a class? What is an ID? CORRECT! Classes can be used multiple times one the same page, whereas an ID should only be used once per page.

Opac Images:

Stop wasting your time. Images are a thing of the past. This goes for double borders, etc.

rgba(0,0,0,0.3);

Will display opac black. Ding Ding.

I think his explanation was a bit simpler than yours, ableit both made sense xD

Id = identifier (unique, think #country-list, #nav-menu)

An ID would be a unique name for an element that you only have 1 of, each element has it’s own name and doesn’t share it with any other element. #london isn’t #sanfrancisco, #london is #london.

Class = classification (groups, elements that need to be referred to as a group)

Classes would be name for a group, for example the elements with ID’s #london and #sanfrancisco both can all have the class .city so in our css we can set a rule for all cities, instead of writing each ID out.
