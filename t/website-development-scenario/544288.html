Alright, we’re going to keep this clean and nice, not another huge debate on PHP vs. Ruby, we have many of those. Vote in the poll above based on the following scenario.

You are building a website for someone who already has a working design, but needs the server-sided code done. They need a log-in system so that their visitors have access to custom pages and resources that are password protected. These resources having different tier ranks depending on what level the member needs to access. For instance, a recent business report is for Administrators only, and they don’t want the normal members to use it. This website has about 300 page visits per day, with about 120 members. Only 5-8 members are Administrators.

If you want to throw in why you chose your answer, feel free to do so, but keep it clean. Throwing in why you didn’t choose a different language is a +.

Why I chose PHP: PHP is not scale-able, for a site with ~120 members, it would work just fine for this scenario. This scenario can easily be achieved with a simple log in system, and just use MySQL to determine ranks. The resources that need a logged in user to be able to access them can easily be checked with isset();

Pwnd August 4, 2016, 1:20pm #4

Rails because… Same with Justin, Devise gives you your authentication system. In Rails, you don’t have to see the database at all, personally. You just create the models, model validations, and a few other things specific to what you’re getting at. It has a built in templating system, and I love using HAML. SEO friendly links are available right off the bat. No need for mod-rewrite, and all of the URLs are simply defined in your routes.rb file. Extending the project would be insanely easy, especially if you started the project off using a testing framework.

Infinite reasons why I would choose Rails.

I did not chose PHP because I would have to write everything from the ground up, if I did not have bits and pieces from previous projects already available. I’d have to put together everything I’d need for a MVC environment. I’d have to write the templating system, create and manage all of the tables that would be necessary, write out the authentication system, and have to deal with messy view pages (because inline PHP is fudgein’ gross). And testing PHP code is insanely difficult because a development environment isn’t really provided for you.

[quote=“The Web Guy, post:7, topic:544288”]Can someone show me how you’d embed HTML inside Ruby?[/quote]So, I would include the HAML gem in my project, because HAML is awesome.

But, you’d have a template file, that’d have the general layout of all of your pages. (nav, footer, empty body) This would be stored in your views folder (/app/views/layouts). This particular file I would use ERB for because it is not necessary to do templating pages in HAML, in fact it’d be more annoying.

File: application.html.erb

[code=ruby]

My Title <%= stylesheet_link_tag "application", :media => "all" %> <%= javascript_include_tag "application" %> <%= csrf_meta_tags %> <%= render 'layouts/nav' %> 
<div class="container">
  <% flash.each do |key, value| %>
    <div class="alert alert-<%= key %>"><%= value %></div>
  <% end %>
  <%= yield %>
</div>

<div class="container">
    <hr>
    <%= render 'layouts/footer' %>
</div>
[/code]

You can see some lines that say “<%= render ‘layouts/footer’ %>”. I have taken some things out as partials and put them into different files.
The “nav” partial looks like this (this uses ERB).

File: _nav.html.haml

%nav{ class: 'navbar navbar-inverse navbar-fixed-top', role: 'navigation'}
  %div{ class: 'container' }
    %div{ class: 'navbar-header' }
      %a{ class: 'navbar-brand', href: root_path } Derp Title

    %ul{ class: 'nav navbar-nav' }
      %li
        %a{ href: about_path} About
      %li
        %a{ href: '#' } Stuff
      %li
        %a{ href: contact_path } Contact
      %li
        %a{ href: login_path } Login

HAML is whitespace sensitive. There is a lot less typing to do and it is very clean. You may notice stuff like “contact_path” and “login_path”. These are automatically filled in based on how my routes.rb file is setup. You don’t ever have to go back to change your URLs unless you change the name of the path itself.

Devise: https://github.com/plataformatec/devise
