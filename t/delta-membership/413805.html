Well since it wouldn’t matter since they can’t see it anyways.

I would do it like this:

Instead of subtracting the amount of days left every single day, just do it when they login. To avoid subtracting twice on the same day… store the last date and on login subtract the last day from the current date. Now you have how many days you need to subtract for. If they have 0 days left, they don’t have membership status, if it’s 1 or more then they have that many days left.

You could make a table in mysql (or file, mysql’s easier though) that has the rows,

(int 10) id, primary, auto-increment (varchar 32) username (bigint 12) timeout
And basically add the username to the table with the time (System.currentTimeMillis()) that they're membership will EXPIRE on (you can make an easy php script to do this for you). If they buy again and they're already in the database, simply add 31 days to the timeout.

Also, when you are checking for if someone has members,
run a query to delete all that have timed out first (“DELETE FROM character_members WHERE timeout > time()”),
and then check (SELECT * FROM character_members WHERE username = ‘USERNAME’).

This would be an actual decent way, opposed to all the other crap ways you could do it. Good luck!
