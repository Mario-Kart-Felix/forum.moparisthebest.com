mod_taharok August 1, 2016, 7:41am #9

You should use equals(), not ==. The operator checks whether the references are the same, but equals() checks the values. When you do something like “Mystring,” that is a constant in the constant pool of the class you are using it in. This means “Mystring” in a different class is an entirely different object. So the == would return false if you checked it to the string constant in a class that didn’t use the same constant when adding it to the list. String has a special method called intern() to store and manipulate constant strings across your entire application, in order to share constant values. Nevertheless, it’s likely more beneficial to simply use equals() instead of the equality operator.

Granted, the above functionality may actually be desired in certain contexts. I am assuming it is unwanted in your application, in which case I am addressing bugs that you may likely face in the continued development of the application.
