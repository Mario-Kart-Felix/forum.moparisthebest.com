Proxy Server (With Other Features)

General Programming
TeleNubby July 30, 2016, 11:33pm #1

Well, I spent my free time on this small little project. Turned out to be quite nice, and I like it. Anyways, here’s the scoop.

This is a proxy server (obviously written in Java), that I wrote up to learn and get used to the java.nio packaged provided in the Java API.

But the fun thing about it is that I added a few additional/bonus features.

[ul][li]Host name filter (parses and blocks names using regex)[/li]
[li]Display any page you want (I used this to display a ‘host name blocked’ page and ‘host name not found’ page)[/li]
[li]HTML source encoding (optional) (my favorite feature - it is also enabled when you download the JAR but by default it is disabled)[/li][/ul]

And for a list of known bugs to slow down any complaints:

[ul][li]When using the encode HTML feature, the response to the client may be slow (but it’s fast enough to keep you from dieing and worth it if at work/school).[/li]
[li]When using the encode HTML feature, and if the header contains a ‘Transfer-Encoding: chunked’ field, the ‘chunked’ data headers in the HTML source code may be left there when the page is displayed.[/li][/ul]

The only real big downsides to this are that the encode HTML feature can be a little slow at times (seems to work very well on MSCP/MITB for some reason), and that this server is currently on a ‘thread-per-client’ set up. Yes, I know it’s bad. But I have no real intentions to create a huge server out of this, and using a thread-per-client set up to learn and use NIO was good enough for me at the time.

The encoding works using JavaScript. If the client has JS disabled, the page will not display and a string will be displayed instead. It works by taking advantage of the unescape() method that comes with the version of JS. It will ‘escape’ each character in the HTML source to it’s hexadecimal counterpart and append a ‘%’ in front of it. Using this can render any content filters that work by keyword, especially at school or at work, useless.

And of course, there are credits:

[ul][li]Zachera - Led me to a great website explaining how to encode your HTML source code. (For anybody curious about that, the link is: http://scriptasylum.com/tutorials/encdec/encode-decode.html - I also forgot to mention I ONLY added the escaping part.)[/li]
[li]Taharok - Helped me out on a few parts here and there and brought in the idea of encoding.[/li][/ul]

I’ve uploaded the JAR archive (source only - you can compile it) as an attachment here. There is a total of only four classes, named Buffer, Filter, Server, and Session.

Enjoy and PLEASE leave appropriate comments/constructive criticism/other helpful tips for the future when using NIO or for any general programming purpose. The mentioning of any bugs would be great, too!

P.S. I named the archive JProxy as I couldn’t think of a better one at the time.

P.S.S. I know MINA exists. No need for criticism on that.

First of all, let me congratulate you. The code looks clear and stylish
Then, here’s a few things to fix (not performance-wise…)

Selector.selectedKeys() is type Set, so you can use those generics properly:
(you could also use the for-each loop instead of manual iteration)

Iterator<?> sockIt = sockSelect.selectedKeys().iterator(); while(sockIt.hasNext()) { SelectionKey sockKey = (SelectionKey)sockIt.next();
->

Iterator<SelectionKey> sockIt = sockSelect.selectedKeys().iterator(); while(sockIt.hasNext()) { SelectionKey sockKey = sockIt.next();

You can set an argument final too, so you don’t need to declare a final temporary variable in order to use it in the inner class:

private void startProxySession(SocketChannel sockChan) { final Server thisServ = this; final SocketChannel tempSock = sockChan; new Thread() { public void run() { try { Session theUser = new Session(thisServ, ourFilter, tempSock);
->

private void startProxySession(final SocketChannel sockChan) { final Server thisServ = this; new Thread() { public void run() { try { Session theUser = new Session(thisServ, ourFilter, sockChan);

You know System.arraycopy! Why not use it?

private byte[] combine(byte[] origSet, byte[] appendSet) { int currentIdx = 0; byte[] newBytes = new byte[origSet.length + appendSet.length]; for(byte b : origSet) newBytes[currentIdx++] = b; for(byte bb : appendSet) newBytes[currentIdx++] = bb; return newBytes; }
->

private byte[] combine(byte[] a, byte[] b) { byte[] newArray = new byte[a.length + b.length]; System.arraycopy(a, 0, newArray, 0, a.length); System.arraycopy(b, 0, newArray, a.length, b.length); return newArray; }

Duplicate method, Buffer.getBytes(ByteBuffer) and Session.getBytes(ByteBuffer).

Also, you should definitely register the SocketChannels for reading and writing in the Selector, drop the one-thread-per-client model and design it well
