[center][size=24pt]Creating a Server Control Panel[/size][/center]

A server control panel will enable you to perform a wide variety of functions out-of-game.

For more information regarding the packages used, check out the JavaDocs: http://download.oracle.com/javase/1.4.2/docs/api/

[center][size=18pt]Basics[/size][/center]

The panel will be created using Swing. A Swing application creates a GUI (Graphic User Interface) that can have a number of different components (buttons, text boxes, radio buttons, etc).

The first step to creating a Swing application is to create a new class. Then, import the javax.swing, java.awt, and java.awt.event packages (the others are for Event Listeners, which we will cover later):

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ControlPanel
{

//Code goes here.

}

Every class that defines a Swing component has several methods to determine its properties such as display text, size, etc. The easiest way to manage it all is to do it all in the constructor.

The next step is to create a JFrame class, which creates a frame to hold all of the components that are added to the control panel. I make the class extend JFrame so that whenever a new instance of the class is created, a JFrame is created automatically; but you can create a new JFrame in the constructor if you want.

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ControlPanel extends JFrame
{

    public ControlPanel()
    {
       this.setTitle("Server Control Panel");
       this.setSize(300, 300);
       this.setVisible(true);

    }

}

In the constructor, I call the JFrames setSize method to set the size of the frame to 300 by 300 pixels, setTitle to set the title to Server Control Panel, and setVisible to make the frame visible (it wouldn’t do any good to have a frame no one could see :P).

Before you can add components to the JFrame, you need to add them to a JPanel. Simply create a new JPanel and call its add method and pass the name of the component you want to add onto the panel. Then add the panel to your JFrame by calling its add method. Here it is in our class:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ControlPanel extends JFrame
{    
    public ControlPanel()
    {
        //Creates a new JPanel to add components to
        JPanel panel = new JPanel();

       //Sets the title, size, and visibility of the frame. Then adds the panel to the frame.
       this.setTitle("Server Control Panel");
       this.setSize(300, 300);
       this.setVisible(true);
       this.add(panel);

    }

}

Now we are ready to start adding components. The three components that we will focus on are JButton (a button), JLabel (a single line of text), and JTextField (a box the user can type in, for getting input). We create a new instance of each of those classes and add them to the panel. The constructors for the JLabel and JButton each take a String as a parameter that determines the text displayed by the component, so pass them the text you want on the label/button. The class now looks like this:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ControlPanel extends JFrame
{
    private JButton ban, kick, mute;
    private JPanel panel;
    private JLabel value;
    private JTextArea player;
    
    public ControlPanel()
    {
        //The components added to the control panel
        panel = new JPanel();
        ban = new JButton("Ban");
        kick = new JButton("Kick");
        save = new JButton("Save All");
        value = new JLabel("Player Name");
        player = new JTextField(10);

        //Adds the components to the panel
        panel.add(value);
        panel.add(player);
        panel.add(ban);
        panel.add(kick);
        panel.add(save);
        

       //Sets the title, size, and visibility of the frame. Then adds the panel to the frame.
       this.setTitle("Server Control Panel");
       this.setSize(300, 300);
       this.setVisible(true);
       this.add(panel);

    }

}

I added three buttons (kick, mute, and ban) for kicking, muting, and banning players, a text field for getting the username of the recipient of the punishment, and a label to describe the input.

[center][size=18pt]Event Listeners[/size][/center]

An event is when anything occurs on a Swing application (from writing a text field to clicking a button). There are several interfaces representing different events in the java.awt and java.awt.event packages. When one of those interfaces is implemented by a Swing application, it listens for the event it represents to occur. When it does, a certain method gets called to execute an action in response to it.

In other words, to make a Swing application do things, you need to add an event listener. To do this, just implement the interface of the type of event you want to handle. We will be implementing the ActionListener interface, which listens for any action to occur and then calls the actionPerformed method automatically to handle it. The class will look something like this:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;

public class ControlPanel extends JFrame implements ActionListener
{
    private JButton ban, kick, mute;
    private JPanel panel;
    private JLabel value;
    private JTextArea player; 
     
    public ControlPanel()
    {
        //The components added to the control panel
        panel = new JPanel();
        ban = new JButton("Ban");
        kick = new JButton("Kick");
        value = new JLabel("Mute");
        player = new JTextField(10);

        //Adds the components to the panel
        panel.add(value);
        panel.add(player);
        panel.add(ban);
        panel.add(kick);
        panel.add(mute);
        

       //Sets the title, size, and visibility of the frame. Then adds the panel to the frame.
       this.setTitle("Server Control Panel");
       this.setSize(300, 300);
       this.setVisible(true);
       this.add(panel);

    }

    //Listens for an event to occur

    public void actionPerformed(ActionEvent e)
    {
        //Code goes here
    }

}

When actionPerformed is called, we need to identify the source of the action before we run code to respond to it (so you don’t ban everyone instead of kicking them :P). This is done by calling the getSource method of the ActionEvent that is passed to actionPerformed, and then writing code for each different component. For example:

public void actionPerformed(ActionEvent e)
{
        if(e.getSource() == ban)
        {
            PlayerHandler.kickNick = player.getText();
            c.appendToBanned(player.getText());
        }
}

Then call the components addActionListener method and pass it the JFrame to assign the Action Listener to the component.

When the ban button is pressed, the player name returned by the text field’s getText method is kicked off the server and banned.

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;


public class ControlPanel extends JFrame implements ActionListener
{

    private JButton ban, kick, mute;
    private JPanel panel;
    private JLabel value;
    private JTextField player;
    public Client c;

    public ControlPanel()
    {
        //The components added to the control panel
        panel = new JPanel();
        ban = new JButton("Ban");
        kick = new JButton("Kick");
        mute = new JButton("Mute");
        value = new JLabel("Player");
        player = new JTextField(10);

        //Adds the Action Listener to the buttons
        ban.addActionListener(this);
        kick.addActionListener(this);
        mute.addActionListener(this);

        //Adds the components to the panel
        panel.add(value);
        panel.add(player);
        panel.add(ban);
        panel.add(kick);
        panel.add(mute);

       //Sets the title, size, and visibility of the frame. Then adds the panel to the frame.
       this.setTitle("Server Control Panel");
       this.setSize(300, 300);
       this.setVisible(true);
       this.add(panel);

    }

    //Listens for an event to occur

    public void actionPerformed(ActionEvent e)
    {
        if(e.getSource() == ban)
        {
            PlayerHandler.kickNick = player.getText();
            c.appendToBanned(player.getText());
        }
        
        if(e.getSource() == mute)
        {
            c.appendToMute(player.getText());
        }

        if(e.getSource() == kick)
        {
           PlayerHandler.kickNick = player.getText();
        }

    }
}

Note:The exact code to be called may vary from server to server. To adapt it, check the process your server goes through to ban, mute, or kick a person and modify actionPerformed so it will ban/kick/mute the person.

[center][size=18pt]Color and [font=impact]Font[/font][/size][/center]

To customize the control panel, you can change the color and font of some of the Swing components.

To change the font, first create a new instance of the Font class. The Font class’s constructor has 3 paramaters: the name of the font, the style identifier (bold, italic, or both), and the font size. Then call the setFont method of the Swing component you want to add the font to and pass it the instance of the Font you create. To make a component inherit the font from its parent, pass null to the setFont method. Times New Roman is an easy font to read, so I’m going to use it as the font for the control panel; you can change it later if you want. Here is the code:

import javax.swing.*;
import java.awt.*;
import java.awt.event.*;


public class ControlPanel extends JFrame implements ActionListener
{

    private JButton ban, kick, mute;
    private JPanel panel;
    private JLabel value;
    private JTextField player;
    public Client c;

    public ControlPanel()
    {
        //The components added to the control panel
        panel = new JPanel();
        ban = new JButton("Ban");
        kick = new JButton("Kick");
        mute = new JButton("Mute");
        value = new JLabel("Player");
        player = new JTextField(10);

        //Changes the font to Times New Roman
        Font f = new Font("Times New Roman", Font.PLAIN, 14);

        //Adds the Action Listener to the buttons
        ban.addActionListener(this);
        kick.addActionListener(this);
        mute.addActionListener(this);

        //Adds the components to the panel
        panel.add(value);
        panel.add(player);
        panel.add(ban);
        panel.add(kick);
        panel.add(mute);

       //Sets the font of the components
        value.setFont(null);
        player.setFont(null);
        ban.setFont(null);
        kick.setFont(null);
        mute.setFont(null);

       //Sets the title, size, font, and visibility of the frame. Then adds the panel to the frame.
       this.setTitle("Server Control Panel");
       this.setFont(f);
       this.setSize(300, 300);
       this.setVisible(true);
       this.add(panel);

    }

    //Listens for an event to occur

    public void actionPerformed(ActionEvent e)
    {
        if(e.getSource() == ban)
        {
            PlayerHandler.kickNick = player.getText();
            c.appendToBanned(player.getText());
        }
        
        if(e.getSource() == mute)
        {
            c.appendToMute(player.getText());
        }

        if(e.getSource() == kick)
        {
           PlayerHandler.kickNick = player.getText();
        }

    }
}

To create different colors, you need to create a new instance of the Color class. Color’s constructor takes three ints as parameters, each of which controls the level of red, green and blue (respectively) present in the color. Or, you could use one of the color constants provided by the class (the choices are black, blue, cyan, dark gray, gray, green, light gray, magenta, orange, pink, and yellow). Then call the Swing component’s setBackground or setForeground method and pass it the Color instance you created. Like the font class, passing null to either method causes the component to inherit the parent’s color.

I’m going to style the panel with red and blue, since those are my two favorite colors. Feel free to change the colors to suit your own taste. The class looks like this:

[center][size=18pt]Images[/size][/center]

To add an image to a Swing application, you create an ImageIcon class and then pass it to the constructor of the component. Many Swing components can display images, including JLabel, JButton, and JCheckBox. The ImageIcon class’s constructor accepts either a String representing the file name/path of the image, or a URL representing the location of the image on the internet.

I am not going to add an image to the control panel; but the code looks something like this:

ImageIcon pic = new ImageIcon("banhammer.jpg");
JButton ban = new JButton(pic);
