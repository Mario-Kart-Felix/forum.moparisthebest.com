Creator

Screenshot:

Purpose: This program will read all of the files in your characters folder, take the skill data of each character, and insert that data into a database.

Why? Unlike phail in-game highscores, this will allow people to search the highscores while not in-game (After the client is made, which shouldn’t take long, people will be able to search the highscores through it). Hell, you could transfer the data over to a website database and create a webpage for searching the highscores.

What you need already:

  • MySQL (http://www.mysql.com/)
  • A database named ‘Highscores’
  • A folder named ‘characters’, where your character files are stored

Instructions:

  1. Download the .zip posted
  2. Extract the files
  3. Put the two files (MySQL_Connector.jar, which lets the program connect to your database, and CreateHS.jar, which is obviously the main program) in the folder above your characters file.
  4. Run CreateHS.jar
  5. Wait for the character files to load, and then fill in these fields:
  • Host: The IP address of the database to insert the highscores information (If it’s your computer, it’s ‘localhost’ ;-))
  • Port: The port your database runs on (Default should be 3306)
  • User Name: The user name used to log into your database
  • Password: The password used to log into your database
  1. Click “Update Highscores”, and wait for the bar at the bottom to ‘go blue’ (Just wait – this may take a few seconds, or longer, depending on how many character files there are)
  2. Enjoy your in-database highscores (In the hschars table, highscores database)

Client

NOTE: Client must be configured before use. Configuration makes it so that you can distribute the .jar (optional, but recommended) created to players on your server.

Screenshot:

Purpose: To search a database made by my Highscores Creator. The client CAN connect to remote computers’ databases, meaning, you can hand out the JAR you make to players on your server to search your highscores.

Why? This makes it so that you can search the highscores of a server while the actual server is not even running; as long as MySQL is up, which on most computers can be made as a service to run in the background, people can search it. It also has almost no load on a computer, as MySQL is VERY lightweight.

What you need already:

  • Nothing but a database to search

Instructions (to create the JAR):

  1. Extract the files from Highscores_CLIENT.rar/.zip
  2. Open HSSetup.java
  3. Change the values of the variables to what you want (Explanations/definitions of the variables are in comments beside each variable)
  4. Run CreateJAR.bat

[b]Downloads are attached. (There is a download in both the .rar and .zip formats)

  • Highscores_CREATOR.rar/Highscores_CREATOR.zip - Highscores Creator (CREATES the Highscores database)

  • Highscores_CLIENT.rar/Highscores_CLIENT.zip - Highscores Client (Connects to a Highscores database (Must be configured first) and searches it) (Not yet released)[/b]
