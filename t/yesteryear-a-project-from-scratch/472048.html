[center][size=14pt]Intro[/size]
Yesteryear is a new server with revision #317 written completely from scratch. It is developed by me, and my goal is to make a basic server with login, player and NPC updating, walking, and maybe later on some basic content like trading and following.
I will only work on this when I have time. And since I am kinda busy with school this will be about one hour a day. I am trying to keep you up to date by providing the latest version of the server.

[size=14pt]Networking[/size]
Yesteryear is written on the java.nio API. It makes use of the Reactor pattern. Which means, a single selection thread that distributes nonblocking network events to worker threads in the event handler. Although I think it still has flaws in the design (for example synchronization), it works pretty well.

[size=14pt]Goals[/size]
As I said in the intro, I will not implement much of content itself but especially updating and basic stuff like trading and banking.

Goals:

[ul][li]Implementation of the Reactor pattern DONE[/li]
[li]Player login DONE[/li]
[li]Packet handling (ingoing and outgoing)DONE[/li]
[li]Player updating[/li]
[li]NPC updating[/li]
[li]Walking[/li]
[li]Perhaps more…[/li][/ul]

Does that mean 1 thread for the entire server? Isn't that bad?
Yesteryear August 4, 2016, 1:38am #7

[quote=“Black Not, post:6, topic:472048”]Could you go into more detail regarding

a single selection thread that distributes nonblocking network events to worker threads in the event handler

Does that mean 1 thread for the entire server? Isn’t that bad?[/quote]
There’s one thread that selects selection keys, these selection keys hold event data (OP_ACCEPT, OP_READ and OP_WRITE). These events are then distributed over a few worker threads. Worker threads are normal threads that are waiting to handle events (work). These worker threads actually read and write to the socket.
