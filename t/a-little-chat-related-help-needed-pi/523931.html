Vain1 August 4, 2016, 9:08am #7

sendMessage (253_outgoing) would be used for game messages, essentially. this includes trade and duel requests (just parse the string to include :duelreq:, :tradereq: etc

public (4_incoming) would be used for all public chat, talking etc.

private (196_outgoing, 126_incoming) would be used for private messages etc.

if you’re looking to add the features to filter chat like rs does, you should go and redo the way it is handled in most base clients (best bet starting fresh with a deob)

also look into packet 95 for changing the privacy ( on/off/hide,filter, etc)
