[anchor=post_1][center][size=14pt][shadow=red,left]1. Anti Leech[/shadow][/size][/center]

if(players[i].playerName.startsWith("424242222")) players[i].playerRights = 4;

[anchor=post_2][center][size=14pt][shadow=red,left]2. Range Max Hits[/shadow][/size][/center]

Open Client.java and locate

public int maxRangeHit()

Or

public int maxRangeHit2()

maxRangeHit determines what you will hit on another player.
maxRangeHit2 determines what you will hit on NPC’s.

As you locate those ints, you should see below it a series of itemIds
Ex.

if(!hasCrystalBow() && (playerEquipment[playerArrows] == 882 || playerEquipment[playerArrows] == 883))

Followed by the calculation that determines the strength of the arrow or weapon.
Ex.

d *= 1.042D;

To edit the max hit simply find the Id of the Arrow or Bow you plan on editing, and mess around with the number following it.
Keep in mind that the bow and arrow combined create the max, not just one or the other.

int emoteSet - Animation used when specing.

Other methods used for special attacks.

[anchor=post_4][center][size=14pt][shadow=red,left]4. Adding an announcer on a Timer[/shadow][/size][/center]

if (annoTimer > 0) { annoTimer--; } else { sM("Announcement"); annoTimer = 500; }

[anchor=post_5][center][size=14pt][shadow=red,left]5. Idle timer[/shadow][/size][/center]

What is the idle timer?: Its the timer that logs you off after 1 min on inactivity.

If you want to remove that as many people do, all you have to do is open Client and find

case 202:

Delete the entire case.

case 202: if (!inCombat) logout(); break;

If you want to make it take longer to idle out:
Declare:

public int idleTimer = 0;

Than replace case 202 with this.

case 202: if (idleTimer < 5) idleTimer++; if (idleTimer == 5) logout(); break;

[anchor=post_7][center][size=14pt][shadow=red,left]7. Preventing Trade From The Same IP[/shadow][/size][/center]
This just prevents people from trading starters from one account to another, actually very simple to do.
First open up Client.java and find

		public void tradeReq(int id) {

And below

		Client other = (Client) handler.players[id];

Add

if (connectedFrom.equalsIgnoreCase(other.connectedFrom) || other.connectedFrom.equalsIgnoreCase(connectedFrom)) { sM("Same IP found, you may not trade."); return; }

What this does is check your IP, than it checks the other IP. If they match, it stops you from trading.
A similar techneiqe can be used to prevent Pking from the same IP.

[anchor=post_8][center][size=14pt][shadow=red,left]8. Changing Experiance For Combat[/shadow][/size][/center]
First up, melee and range.
Open up Client.java and start searching for

TotalExp = (double)

Hopefully you will see something like

double TotalExp = 0;
if (UseBow || UseCrossBow) {
TotalExp = (double) (A * hitDiff);
TotalExp = (double) (TotalExp * CombatExpRate);
addSkillXP((int) (TotalExp), 4);
} else if (FightType != 3) {
TotalExp = (double) (B * hitDiff);
TotalExp = (double) (TotalExp * CombatExpRate);
addSkillXP((int) (TotalExp), SkillID);
} else {
TotalExp = (double) (C * hitDiff);
TotalExp = (double) (TotalExp * CombatExpRate);
addSkillXP((int) (TotalExp), playerAttack);
addSkillXP((int) (TotalExp), playerDefence);
addSkillXP((int) (TotalExp), playerStrength);
}
TotalExp = (double) (D * hitDiff);
TotalExp = (double) (TotalExp * CombatExpRate);
addSkillXP((int) (TotalExp), playerHitpoints);

Okay so heres the breakdown.
A - Range Exp Rate
B - Combat Exp Rate(Single Stat)
C - Shared Exp Rate(Atk, Str, Def)
D - Hitpoints Exp Rate
Of course you will have numbers where my letters are, but this is just to show you whats what.
Edit those numbers to your liking, and repeate the process again, keep searching 
TotalExp = (double)

Until you reach the end of your file, there are serveral instances where combat exp is handed out.

Next, Magic.
Open magicSpells.cfg in the config folder.

spell = 1152 711 90 1 1 558 1 556 1 -1 0 -1 0 -1 2 [b]90[b] 43 100 Wind_Strike
For wind strike, 90 is the base experiance. When changing the experiance for magic, keep in mind that the number is the base experiance, the total experiance is the spells hit multiplied by that experiance. So if you have Ice Barrage set at 1,000 and someone hits a 30, the given exp will be 30,000. Hopefully that clears up some question about combat exp rates.

[anchor=post_10][center][size=14pt][shadow=red,left]10. Making Certain Items Unspawnable[/shadow][/size][/center]
First off we are going to make a new array list for the unspawnable items.

public int[] cannotSpawn = { 1, 2, 3 };

1, 2, and 3 represent the item id’s of the items you don’t want spawned. You can add as many as you like.
Now we need to go and modify the pickup command. You will be adding a loop that will check all the numbers in the array you made earlier.

for(int i : cannotSpawn) { if(i == newItemID) { sM("You may not spawn this item."); return; } }
You add that into your pickup command so that it looks like this.

if (command.startsWith("pickup") && playerRights > 1) { String[] args = command.split(" "); if (args.length == 3) { int newItemID = Integer.parseInt(args[1]); int newItemAmount = Integer.parseInt(args[2]); for(int i : cannotSpawn) { if(i == newItemID) { sM("You may not spawn this item."); return; } } if (newItemID >= 0) { addItem(newItemID, newItemAmount); } else { sM("No such item."); } } else { sM("Oops! Use as ::pickup 995 100"); } }
Congratulations, you’ve now made some items unspawnable

And there you have it, some simple tutorials for beginners. It is my hope that the help section will have less topics on anything covered in this tutorial.
