Note:

++ Increment operator; increments a value by 1
– Decrement operator; decrements a value by 1

Now, the difference between pre- and post-fix are in the evaluation of the expressions as part of larger ones. When used as standalone expression statements, they have the same net effect; when used as part of a more complex expression, it matters.
The increment/decrement operators can be applied before (prefix) or after (postfix) the operand. The code result++; and ++result; will both end in result being incremented by one. The only difference is that the prefix version (++result) evaluates to the incremented value, whereas the postfix version (result++) evaluates to the original value. If you are just performing a simple increment/decrement, it doesn't really matter which version you choose. But if you use this operator in part of a larger expression, the one that you choose may make a significant difference.

So, that being said, (a == b) and !(a != b) are equivalent; similarly, (a != b) and !(a == b) are as well. In the case you're comparing objects, these operators are used for [i]identity equality[/i]. For example, ("abc ".trim() == "abc") will evaluate to false, since the literal string object "abc" and the result of "abc ".trim() do [b]not[/b] share object identity, while ("abc ".trim().equals("abc")) will evaluate to true because they're still [b]meaningfully equivalent[/b].
