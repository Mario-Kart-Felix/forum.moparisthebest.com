_s11 August 1, 2016, 1:01pm #3

yes, there’s a difference between x86-64 and x86 assembly… x86-64 has more registers that you can use (and they’re 64bits long compared to x86’s 32bit ones (same applies to memory addresses))

also the Linux kernel isn’t a bootloader, it’s a kernel as the name says. you’re going to have to learn about executable formats and file systems (FAT teehee) so you just get the kernel loaded so needless to say, you’ll be fucking around with a lot of interrupts and memory. also, you don’t have to write the whole bootloader in assembly although you will need assembly knowledge for a lot of things.

oh, and look at http://wiki.osdev.org/ as it provides some damn good information on kernels and bootloaders. as for learning assembly, look at the Art of Assembly books and Programming From the Ground Up
