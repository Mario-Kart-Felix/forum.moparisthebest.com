Onix August 3, 2016, 3:05am #2

Well, to learn what each class files does, you will need to open the file with a notepad, or even the source with an IDE, such as Eclipse or Netbeans. Then you will need to read the code line by line and use common sense to figure out what each class is doing. Another big help for understanding is the naming conventions of the classes, such as the ‘player’ classes, which handle player related stuff.

To figure out where to put something, also relies on common sense. If you want to make clicking an object do something, you will need to know where that packet is handled, this can be done by reading through the files from the first part of this post. Some servers even have individual classes for this, such as ‘ObjectHandler’.
