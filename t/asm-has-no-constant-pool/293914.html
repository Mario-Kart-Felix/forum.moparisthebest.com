BAsh July 31, 2016, 9:55am #3

ASM doesn’t use a constant pool. Rather than using indexes into the constant pool, ASM copies the data out of it.

It means that when you change the name of the class all you have to do is change a String, rather than find out which class definition the index is pointing to and then find the index of the name of that class definition and change that. Additionally all you are doing is changing a class name rather having the side affect of changing whatever else references that entry.

It adds a layer of abstraction which has a trade off in speed, but it is much easier and faster to program and debug.
