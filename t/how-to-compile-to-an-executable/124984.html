This is a tutorial on how to compile your code into a executable file, (.exe). I’ve noticed a fair few people have been asking me how to, so I thought I would make a tutorial on how to; this is a pictured tutorial with everything pretty much explain. I will provide a sample code if you want to try this tutorial. Now, to get started, we will need a type of compiler; and that compiler is called a “Native Compiler”. There programs of these compilers differentiate, and a lot of these programs need to be paid for, so I will supply a program. The program I will supply is called NativeJ. It has a relevant name don’t you think? Below is the download link.

I don’t think I need to run on about how to install it. Now we need to run the program, so click your installed program, and you should come to a main page and a new project wizard. It will look similar to the following picture.

As you can see, it is asking for us to specify a project name. If you click browse, it will asking where you would like to save your project. I like to make a folder on my desktop with the name of the project and do all my project work in there. It makes it a lot better then having the files all over the place. So, go ahead and specify your project directory. Then, click next. Simple right? After you have clicked next, you should see something like the picture below.

Now, you can probably gather, it is asking you to specify the name of your executable file which will be created. Note: Make sure you leave the “.exe” part at the end. Of course, with out the quotation marks. So, say I wanted my file which is created called “Program”, in that field, I would supply an answer of “Program.exe”. If you’re happy with your executable file name, click next. Remember, you can make last minute changes. After clicking next, you should see a picture like the following.

If you have realized, or is you haven’t, it is asking us to supply the wizard with the icon that will be used on our executable file which is created. If you don’t supply a icon, it creates the default icon. I like the default icon, it looks like the picture below.

I don’t mind the look of it. But, if you wish to change to your perference, go ahead. Now if you’re happy with your icon to be used, press next. And you should see the next frame, looking something like:

As you can probably gather, it is asking us to supply our main-class of our application. You should know what a main-class is, but, if you don’t, it is the text after your class line. Example:
public class Main {…
In this case, Main would be our main-class. Get it now? So if you enter in your main-class into the wizard and press next, you should come across a new screen. It should look similar to the following picture.

Now, if can read or see, you’ll notice that the wizard is asking whether our program is a Console Application or a Graphical Application. If you are unsure on what yours is, it is basically.
Console Application – A application that works with your console. It usually is the applications that use System.out.println and does not use a Graphical User Interface(GUI). A simple console application looks similar to the following:

public class HelloWorld { public static void main(String[] args) { System.out.println(“Hello World”); } }
Notice how it prints the line inside a console program?

Graphical Application – An application that uses a Graphical User Interface (GUI). It usually uses the swing or the AWT package. A simple Graphical Application looks similar to the following:

[code]import javax.swing.*;

public class HelloWorldSwing {

private static void createAndShowGUI() {
    JFrame frame = new JFrame("HelloWorldSwing");
    frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

    JLabel label = new JLabel("Hello World");
    frame.getContentPane().add(label);

    frame.pack();
    frame.setVisible(true);
}

public static void main(String[] args) {
    javax.swing.SwingUtilities.invokeLater(new Runnable() {
        public void run() {
            createAndShowGUI();
        }
    });
}

}[/code]
You may notice it imports the swing package. The other optional package is AWT.
So, if you have decided what yours is, click next, and you should see a frame similar to the picture found below.

What that is, is it’s asking us to supply the wizard a jar file. So, you’ll need to jar your program, and add it too the list, by clicking add + in the wizard. So once you’ve supplied your jar file to the wizard, click next and you should see a frame like the following picture:

What that is saying, is that you have completed the wizard, and you are now ready to compile your code natively. So, click finish, and you should come to the main screen. It will look the following picture.

This is the place where you are able to make your last finishing touches, or edit a few things. Why they have the whole wizard I’ll never know. But, if you’re happy how your settings are, click Generate Executable, it can be located up near the top. Or, in easy terms, look at the picture below.

When you click Generate Executable, a new window will open up. It will look something like this if it was successful.
