  1. I know that you need to add globalobjects into the client.java at least that’s what I did. awile back… I had a command that allowed you to spawn them ingame but those were only there until everyone logged out or the server restarted, permanant objects must be added in the client.java and such i.e.
public void NewObjects() {//X Y ID DIR 10 (Dir, 0=West -1=North -2=East -3=South)(Narbmark: OBJECTS) AddGlobalObject(2891, 9698, 2038, -1, 10);//dummy

XY Coords, ID = Object ID, Direction it’s facing… and yeah…

2.That’s the autospawn, shopconfig… also you have to give them a unique shop ID in the client.java
In the client.java you would set up the SHOP ID for the new NPC’s shop like…

else if (NPCID == 1872) { // Sonji testing supplies PutNPCCoords = true; WanneShop = 66;

66 being the Shops ID… then in the autospawn you’d set up the NPC of your choice where he’s located as such…

//------NpcID---XCor----Ycor--Height--RangeX1-RangeY1-RangeX2-RangeY2-WalkType-Description spawn = 553 2894 9691 0 0 0 0 0 2 Sonji testing Supplies

The ID of the NP, his X and Y coords, height, the ranges denote where his walking distance is… and such…

then for the shop you would open up the shop.cfg and set up what items he sells, his pricing i.e. sells for more buys for less… kinda thing… as such…

shop = 66	@whi@Sonji Testing supplies				2	2	6529	1000000	6306	1000000

This is where that unique ID comes in again that we set up in the Client.java… I’m not telling you how to do it… I guess i’m just covering basic points… o.O

  1. Making them talk is involved in the npchandler.java, where you can denote what several NPCs do individually… such as…

} if (npcs[i].npcType == 553) { if (misc.random2(50) >= 1) { npcs[i].updateRequired = true; npcs[i].textUpdateRequired = true; npcs[i].textUpdate = "Sonji, Come get some supplies!"; }
