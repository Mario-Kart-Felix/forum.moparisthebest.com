[quote=“light232, post:10, topic:295650”]Reactor.getSingleton().switchOp(socketChannel, SelectionKey.OP_READ);
in the handleRead void isnt necessary[/quote]Yes it is, look at the constructor. It switches the interested IO operation (to 0) so the demultiplexing thread stops flagging the same key over and over in the time it takes to initialize a new IO handler and thread. It must be switched back to read after done so we can continue waiting for data

[quote=“light232, post:8, topic:295650”]public static Reactor getSingleton() { try { return singleton == null ? singleton = new Reactor() : singleton; } catch (Exception ex) { logger.fatal("Could not initialize Reactor!", ex); } return null; }

private static final Reactor singleton = new Reactor();

Initializing a Reactor may throw an IOException, and I wanted to make sure that it is caught properly. Never overlook proper exception handling. Also, I like to initialize singleton objects on the fly instead of everything at startup. I’m not sure why but I don’t like coupling declarations and initializations.
