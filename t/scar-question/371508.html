Maplestory has an anti-cheat (GameGuard as far as I know) which hooks SendInput/SendMessage/PostMessage and other mouse/keyboard related API calls.

I don’t remember where I read this, but if you are intellectually inclined and some-what computer literate you can do an interesting by-pass by calling the function while skipping the first 5 bytes (which are modified by the GameGuard detour)

DWORD myAddress = GetProcAddress(GetModuleHandle("User32.dll"),"SendMessageA") + 0x05;

And

__declspec(naked) LRESULT cdelc mySendMessage (HWND hWnd, UINT Msg, WPARAM wParam, LPARAM lParam)
{
	__asm 
	{ 
	...	jmp dword ptr ds:[myAddress]
	}

(You can also push these and not use function header)

All I’m trying to say is that GameGuard looks at the hWnd parameter to see whether or not that call will succeed. SCAR won’t get you anywhere and this is most likely over your head.
