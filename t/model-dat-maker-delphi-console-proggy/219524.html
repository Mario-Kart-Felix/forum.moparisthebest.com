I’ll post some more helpful dribble below, the proggy is really for those that understand the file format and might be able to use the code in their own programs.

Begin dribble;

dat maker notes.
model vertices MUST be integer values.
The model space extents are approx +/- 32,000
The material colors can be calculated from bentskis rsmv.
The program attempts to traverse the model triangle by triangle, in the order the triangles are written to file. SEE compression notes.

metasequoia specific notes.
In metasequoia set the grid to 1, click move button, move vertex to new point and back again to remove any non integer values.
Might be wise to keep large models in ortho mode.

compression notes. (Google 3D compression)
You can use metasequoia to compress the model.
Compression is simply about creating a single 'net '(or as few as possible).
Much like peeling an orange, you can peel it in a single strip or remove lots of smaller pieces.
You can trick metasequoia into writing triangles to file in the order you want them to be, triangles are usually written in order of creation.
select the starting triangle, cut and paste, then the next triangle cut and paste…continue until finished.
your triangles will be put into seperate objects and the first triangle will be at the top of the list.
You should now have an empty object and lots of objects with 1 triangle.
Select the original object and ‘merge all’ (under the misc button)
The triangles will now be written in the way you ordered them.
Remove extra vertex points with the join closed vertices command and save.
Takes about 10 minutes to do, but may take hours trying to figure out which is the best compression path.
Compression is entirely optional.

End dribble;
