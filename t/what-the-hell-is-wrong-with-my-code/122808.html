Using the asterisk wildcard tells the compiler to find all the imports in that package that your program uses. If you decompile a java program after compiling it, you will see it does not compile it with the asterisk. It makes all the imports for each class that you need importing. So if you use, I would personally say, three or more imports from the same libary, an asterisk helps make the imports neater.

moparisthebest July 30, 2016, 12:36pm #16

The compiler optimizes the imports at compile-time anyway, and therefore never actually imports more than you need. Therefore speed is not affected at runtime or compile-time. However proper coding practice indicates that you should explicitly import only what you need so that later programmers can easily see what you used or didn’t use.
