I’ve noticed WAY too many people having problems finding their JDK so they can compile whatever source they need to compile. My first recommendation is getting an IDE; you do NOT have to compile with an IDE, when you run the program it will compile. You can find more information about Eclispe with Google.

If that just doesn’t fly, then you need to edit your compiler/path/classpath and replace that crappy jdk version that doesn’t work with the one that does work.

For Windows XP 32 (x86) Bit users:
• go to Start (bottom left)>My Computer>Local Disk (C:/)>Program Files (x86)>Java

For Windows XP 64 Bit users:
• go to Start (bottom left)>My Computer>Local Disk (C:/)>Program Files>Java

For Windows Vista/7 users:
• go to start (bottom right)>search “Java”>go to the folder “Java”>find the jdk folder

Now look at this picture, the “circled” folder is what you will replace in your .bat program.
So if you have “jdk1.6.0_05” set as your classpath/path/in your compiler replace it with EXACTLY the one that is shown in your folder. (For example, only I would replace jdk1.6.0_05 with jdk1.7.0_05, because jdk1.7.0_05 is the version I have, not you.)

So, what did that do? Well, you found out your jdk version (whatever the folder was called) and now go back to your compiler/path/classpath and change it accordingly.

Any questions post here, I will not answer non-related jdk issues, and I will not do it for you.
