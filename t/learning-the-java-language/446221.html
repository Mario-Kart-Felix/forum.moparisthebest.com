Lin August 3, 2016, 3:27am #6

The way I learned was almost entirely through experience. I started writing C++ when I was 11 with my cousin using a really old Visual C++ IDE, and then I got into RSPS. I noticed Java was a lot like C++ in the way it ended “lines” with semicolons, used parenthesis, used curly brackets, had very similar type names, etc… I didn’t understand half of the stuff I was doing, but I’d still add content from tutorials and looking at what the server already had, like how adding a shop usually required 3 lines and a “wanneShop” check. Over time, I learned why things happened and was eventually able to write my own code. I never read a single tutorial on learning a language.

If I were to give you one recommendation it’d be just to think of a project that you’re interested in and won’t drop it with no reason. Just go for it and whenever you get stuck just refer to relative documentation and google for support, then read it and fully understand what you’re implementing.
