If you log in or enter the region after an object is changed/added, you’ll not be able to see it in that state.
So I decided to wrote a system to handle that.
If you login or change region or whatever, the object will be visible for all players in its current state.
I also want to point out that; firemaking, woodcutting, mining ect… is all using my object system,
it is not using any private methods for the object state changes, timers, ect…
