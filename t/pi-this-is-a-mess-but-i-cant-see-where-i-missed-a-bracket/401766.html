Yeah, the code can be reduced to

private static final String[] SKILL_NAMES = {
    "Attack", "Defence", "Strength", "Hitpoints", "Ranged", "Prayer", "Magic", "Cooking", "Woodcutting", "Fletching", "Fishing",
    "Firemaking", "Crafting", "Smithing", "Mining", "Herblore", "Agility", "Thieving", "Slayer", "Farming", "Runecrafting"
};

private static final String ACHIEVEMENT_STRING = "%s reached level %d in %s!";

public void announceAchievement(final Skill skill) throws IllegalArgumentException {
    if (skill == null) {
        throw new IllegalArgumentException("skill cannot be null");
    }
    for (final Player p : PlayerHandler.players) {
        if (p != null) {
            final Client c = (Client) p;
            c.sendMessage(String.format(ACHIEVEMENT_STRING, c.getName(), 99, SKILL_NAMES[skill.getId()]));
        }
    }
}

Although, I’m not sure if there’s a performance increase in choosing String.format() instead of concatenation in this case.
