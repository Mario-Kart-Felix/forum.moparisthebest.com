Admit it; the RSPS scene has become a bit dry now-a-days.* We are becoming more of an emulation if anything. So the purpose of this? I shall keep this thread simple and to the point, two simple points:

[ul][]An introduction to new ideas for RSPS.
[]An idea for implementation of these new ideas.[/ul]

Sui Generis plans to fully utilize Git and the features BitBucket offers through Wiki documentation for ideas held within the source code, issue tracking to know what bugs are present, as well as propose new ideas to the server, and the basic commits which shall allow a user to find the specifications for adding a certain content by clicking the commit ID.

[I]Sui Generis is not a server that is going to be hosted.[/I]
