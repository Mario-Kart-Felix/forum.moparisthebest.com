public String getItemName(int ItemID) { for (int i = 0; i < server.itemHandler.MaxListedItems; i++) { if (server.itemHandler.ItemList[i] != null) { if (server.itemHandler.ItemList[i].itemId == ItemID) { return server.itemHandler.ItemList[i].itemName; } } } return "!!! ITEM NOT FULLY ADDED !!! - ID:" + ItemID; }

Next search for this:

public boolean wear(int wearID, int slot) {

Now, lets say you want to add a level 40 defence requirement for all rune items. Instead of repeatedly doing something like this:

if (wearID == 1079 && playerLevel[0] < 40) { CAM().sendMessage("You need an defense level of 40 to wear rune"); return false; }

You could doing something much simpler:

if (getItemName(wearID).startsWith("Rune") && itemType(wearID) != playerWeapon && playerLevel[1] < 40) { CAM().sendMessage("You need a defence level of 40 to wear rune items."); return false; } else if (getItemName(wearID).startsWith("Rune") && itemType(wearID) == playerWeapon && playerLevel[0] < 40) { CAM().sendMessage("You need an attack level of 40 to wield rune items."); return false; }

With this all rune armor requires 40 defense, and all rune weapons require 40 attack.


getItemName(wearID).startsWith(“Rune”) checks all items that start with the word “Rune”

itemType(wearID) Checks for the item’s type.

itemType(wearID) != playerWeapon And itemType(wearID) == playerWeapon are both needed because weapons and armour have different requirements for use.

That’s it. I hope this helped
