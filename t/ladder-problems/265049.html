Ok, in your case there are multiple ladders, so you should use objectX. objectY instead of absX, absY, because in the absX/Y statement, the ladder will only function if you are standing in the exact spot you entered for the Abs’s, so instead use an objectX/Y and from the looks of it your editing things in the “clickingmost.java” wrongo, thats for things like emotes etc, i suggest you do it in the client.java, in the client.java find [if(objectID ==)] (without []) then under the closing } hit enter, then on a new line type:

if(objectID == 1757 && objectX == OBJECT’SXCOORD && objectY == OBJECTYCOORDS){
toX = 2399;
toY = 3100;
heightLevel = HEIGHT;
}

On some servers the object’s id and coords are in the server DOS window, but anyway, this statement says, if the object is the ladder (1757) and the ladder is here (object coords) then it takes you to the given coords.

Also the teleportToX, where did that com from, if you want to TELEPORT it would be triggerTele = (coords) But that would make it look like your teleporting, not climbing a ladder.

The reason i prefer objectX/Y is because with that you can click the ladder from any area close to the ladder and it will take you up, not only if your standing right beneath the ladder which is pretty hindering.

Also if the ladders are the same object ID at both of its parts (top and bottom) then add this to the if statement, && heightLevel == CURRENTHEIGHTLEVEL; so it would look like this:

if(objectID == 1757 && objectX == OBJECT’sXCOORDS && objectY == OBJECT’SYCOORDS && heightLevel == CURRENTHEIGHTLEVEL){
toX = 2399;
toY = 3100;
heightLevel = HEIGHT;
}

And finally, the heightLevel, the heightLevel tells your client if your going to go up or down, the basic ground level is 0, if your going up a level make it 1, for example, in castlewars, the heightLevel for the platform where the Standards/banners are for each team, is heightLevel = 3;

And you’ll have to add 2 if statements for every ladder (top and bottom).
