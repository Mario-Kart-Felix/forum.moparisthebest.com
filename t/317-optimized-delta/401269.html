I took Jonny’s source, DeltaClean, and edited and cleaned it to make it run smoother and make it easier to program.

What I did:

[ul][]Handled most packets
[]Seperated commands from the client class
[]Reduced the client class by approximately 92kb (from the original DeltaClean)
[]Loaded custom objects through array
[]Player deaths are handled by events
[]Fixed the combat delays
[]Skill/action checking not through process
[]Fixed the cycle time to 600ms
[]Packaged the server
[]Made objects and such be handled through a switch statement (to prevent future users to use if-and-else statments[/ul]

Old features:

- Everything DeltaScape has - Grahams EventManager - Full Indentation across all classes - Removed over 1000 dead code fragments - Follows 100% Convention across classes (Except for the basic packaging + method names(way to many to refactor)) - Has multiple notorious Delta-Bugs Fixed - Runs ALOT smoother then regular DeltaScape - Has tips and guides on how to edit basic things for beginners - Works with most clients (Galkons refactored etc) - Will hold atleast 30% more players then an average DeltaScape - Alot more - Download and look. - Re-made Trading - X log fixes - Common Dupes fixed

Example of better combat:

To remove the debug messages:

  1. Go to your Player class,

  2. Find ‘public boolean debug = true;’

  3. Set it to false
