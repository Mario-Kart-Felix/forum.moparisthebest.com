sk8rdude461 August 4, 2016, 2:12pm #11
Coldmedicine:

We just use a VPS…

Doesn’t mean shit?
Github is for version/content control. your VPS is used to host your server/website/whatever.
You can use github to push updates to the VPS. And you wouldn’t even have to send the entire server over to the VPS over and over. All you do is sync the VPS to the github and boom, anything you changed is updated. Anything you didn’t stays the same (Why update something that didn’t have any change?).
The way you’re doing it now is probably either A) Zipping the source and then downloading it on the VPS (Via dropbox, mediafire, whatever) or B) Working on the VPS itself (You’re an idiot if you do this. Never do this). And both have huge disadvantages. As I previously stated, why constantly download the exact same, unchanged, files when you only need the updated ones. How about if one of your updated files replaces an old one and causes a huge exploit. But wait, you don’t know what file it’s in since you have no way to track your changes. Or, if you’re actually writing code on the VPS, what if you save your changes, cause an error, and then forget where the error is (or don’t know how to solve it). Your server will be S.O.L until you fix it, and you can bet players will hate that.

TL;DR:
Your method is wrong, use GIT or similar.
