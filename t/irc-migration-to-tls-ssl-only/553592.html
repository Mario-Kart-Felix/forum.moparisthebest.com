IRC migration to TLS (SSL) only

General Discussion
T41 August 4, 2016, 4:33pm #1

Last night a new policy was put in place on the IRC server that enforces only secure connections on port 6697. The plaintext port (6667) now displays an error communicating this policy and immediately disconnects the user. At this point in time we are using self-signed certificates so please disable certificate verification in your client (usually called SSL verify or accept invalid SSL). I believe moparisthebest is going to be providing me with legit CA signed ones in the future. Below are snippets for configuring these settings in the most popular IRC clients:. Be aware that the webirc chat is using TLS so that will always work.

irssi:

/server add -ssl -network moparisthebest -port 6697 irc.moparisthebest.com

weechat:

/set moparisthebest.addresses "irc.moparisthebest.com/6697"
/set moparisthebest.ssl on
/set moparisthebest.ssl_verify off

quassel:

mIRC:
Setup the OpenSSL library if you already haven’t, then /server irc.moparisthebest.com:+6697

znc:
Under the edit network interface in a box titled, “Server of this IRC network”:

irc.moparisthebest.com +6697
