yakman July 30, 2016, 9:34pm #15

you also might want to learn about java threading and concurrency, that bug about still thinking there is a bug is probably caused by a race condition

bugs that only happen sometimes, when the wind is blowing the right way, or the phase of the moon is right, are typical of thread-related bugs

like you use the field “running” to control the other thread, what happens if its false? the thread loops uselessly and probably lags the game
use Object.wait() and Object.notify() for this

also

	//{{DECLARE_CONTROLS
	java.awt.Button pauzeKnop = new java.awt.Button();
	java.awt.Button stopKnop = new java.awt.Button();
	java.awt.Button controlsKnop = new java.awt.Button();
	java.awt.Button instructionsKnop = new java.awt.Button();
	java.awt.Button rulesKnop = new java.awt.Button();
	java.awt.Button mainMenuKnop = new java.awt.Button();
	java.awt.Button playKnop = new java.awt.Button();
	java.awt.Button instructionsKnop2 = new java.awt.Button();

and you do “import java.awt.*;” at the top?
if you have ambiguous classes, just do like “import java.awt.Button;”

and this?

showTime = Integer.parseInt(timeField.getText())+":00";

what if your user inputs something which isnt parsable as an integer? it would crash that whole thread you started
your program needs to be robust, meaning it can handle any input gracefully, you need to expect everything,

try this

int inputtedTime = 0;
try {
    inputtedTime = Integer.parseInt(timeField.getText());
}
catch(NumberFormatException ex) {
    //do something here, probably tell the user he's screwed up
}
showTime = inputtedTime + ":00";

how about putting your class Player, Bomb, Item and Explosion each in their own separate files
