Part 2: Let the “fun” Begin

What you’re going to do now is unzip/unrar our server source, as well as the client you downloaded, unless you’re playing the Silabsoft webclient.

You will use the Java compiler javac to compile your Java programs and the Java interpreter java to run them. To check if Apple’s implementation of Java 2 Standard Edition (Java SE 6) is already installed:

* Run Software Update.

* Run Applications/Utilities/Java/Java Preferences and verify that the Java Applications version is set to Java SE 6.

Command-line interface:

You will type commands in an application called the Terminal.

* Open a terminal window. You can find this under Go -> Applications -> Utilities. Drag the Terminal to your dock since you will be using it frequently.

* You should now have a Terminal window somewhere on the screen. It will have a prompt that looks something like:

machine:~ user$

* To check that you are running the right version of Java, type the commands in boldface below. You should see something similar to the information printed below. The important part is that it says 1.6 or 1.5 (and not 1.4).

machine:~ user$ java -version java version "1.6.0_15" Java(TM) SE Runtime Environment (build 1.6.0_15-b03-226) Java HotSpot(TM) 64-Bit Server VM (build 14.1-b02-92, mixed mode)

  Then type

machine:~ user$ javac -version javac 1.6.0_15

* Since you will be using the Terminal frequently, you may want to customize the default window settings (e.g., Monaco 13pt font with antialiasing).


Compile the program:

You will use the javac command to convert your Java program into a form more amenable for execution on a computer.

* From the Terminal, navigate to the directory containing your .java files, say ~user/508source/bin, by typing the cd command below.

machine:~ user$ cd 508source/bin machine:~/508source/bin wayne$

* Assuming the files are all in the current working directory, type the [b]javac [/b]command below to compile it.

machine:~508source/bin user$ javac *.java machine:~508source/bin user$

  If everything went well, you should see no error messages.


Run the Server:

You will use the java command to run your server.

* From the Terminal, type the java command below.

machine:~/508source/bin user$ java *.class Starting up 508Server on port 43594...

  If all goes well, you should see the output in the Terminal stating that the server is running. Feel free to alter the code above, I'm working from my brain on this part because my Mac is updating around 50 programs.  :eek:


Part 3: Troubleshooting

Input and Output

If your program gets stuck in an infinite loop, type Ctrl-c to break out.

If you are entering input from the keyboard, you can signify to your program that there is no more data by typing Ctrl-d for EOF (end of file). You should type this character on its own line.

Good Luck!
