zstars July 31, 2016, 9:28am #7

Java is definitely easier than C++. C# is closer to Java than to C++, and it is easier than C++ but possibly slightly harder to learn than Java (as it has more features).

I wouldn’t focus on difficulty to choose a language to learn though, but rather about what language you preffer, what do you want to achieve, what strong points does it have, and what do you want to use it for.

Also, the three languages are relatively similar (as they all are mostly object oriented), so any knowledge you acquire through any of them is likely to be helpful when using another, at least indirectly. Especifically, knowing C++ you should be able to start using Java or C# (to some extent) in a few hours, and the same applies to Java -> C# or C# -> Java.

Personally, based on the type of app, I’d recommend:

-Short gui-less script: Python / Lua
-Web applet: Java
-GUI app: C#
-Any performance critical app: C++
-Embedding: Lua / Python
-Device Driver: C / C++.
-Low level platform-specific application: C++
-Platform-independant app (not only cross-platform): Java / C#
