[center]It is NOT my fault if you fuck up your client; BACK IT UP.[/center]

Things needed:
Client that uses Gui for JFrame, or for the IP.
Notepad
client.java
Gui.java (nasty thing)
An IQ higher than 1
And copy, paste abilitys

Step 1:

Open up your client folder and make a new file called Ipset save it as a “.java” file.

Step 2:

Adding the code to the Ipset.java…

Add this code and I will explain it on the way.

First add this:

import javax.swing.*; import java.awt.*; import java.lang.String; import java.awt.event.*; import java.io.BufferedReader; import java.io.File; import java.io.FileOutputStream; import java.io.InputStreamReader; import java.io.IOException; import java.net.InetAddress; import java.net.URL; import java.util.ArrayList;
Just imports everything needed, although some of this probably isn’t needed.

Make a bit of room under that and add this:

public class Ipset extends client {

This starts our main class, this is where the coding starts.

Now, lets add this:

public static final String SERVERIP = "#.#.#.#"; // Ip to connect to.

This is the ip string, this is where you’d put a servers ip.

And close it with a:

}

File save, on to step 3.

Step 3:

Open up client.java and replace all (ctrl + h), “Gui.server” without the quotes. Replace it with this:

Ipset.SERVERIP

File save and compile, and you’re done! Simply in your run where it says,

.:Theme.jar Gui

Replace the “Gui” with Ipset. File save, and run it.
