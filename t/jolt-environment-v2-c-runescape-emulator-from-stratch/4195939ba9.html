Whats the pros and cons of running everything through scripts instead of the servers native code?

TheAJ August 2, 2016, 2:53pm #49
The Wanderer:

Whats the pros and cons of running everything through scripts instead of the servers native code?

It allows server owners to easily and quickly make changes to their liking, without the hassle of having to recompile the server. The scripts are cached, but can be reloaded while the server is on, so developing is much easier, and doesn’t require recompiling, and restarting for every change you make.

I honestly can’t think of any cons for script systems. Especially ours, it’s flawless.

Infact, every single interface that you see on our pics, etc, are done through scripts. No interfaces are hard coded in the core.

Even the gameframe initialization, is done through scripts!

[code]using System;

using RuneScape.GameServer;

namespace Content.Interfaces

#region Fields

    {
        this.owner = owner;
        this.frameInterface = inter;
    }

    /// <summary>

    /// </summary>
    public void OnOpen()
    {
        // chat options box

        this.chatFrame = new Interface(this.owner, (short)InterfaceIDS.ChatboxFrame, this.frameInterface, this.owner.GameClient.IsScreenFixed() ? (short)192 : (short)71, 1, "");
