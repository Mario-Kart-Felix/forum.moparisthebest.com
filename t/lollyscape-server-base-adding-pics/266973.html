I’ve decided to make a server base, as I’m disgusted at the quality of code in current private servers.
I have created the base from scratch; not one part of it is taken from another source.

This server will include a User Interface, sporting an announcement feature and a player manager.
Also, player saving now takes advantage Java’s serialization technology.
And there are not millions of variable declerations hogging up the Client class; all values and keys are now unified under 2 HashTables (one for Strings, and one for Integers, and I am adding one for Booleans).

Everything follows Java naming conventions e.g. hiMoparGuys() for methods, PENIS_SIZE for constants.
