Well… Seeing as your model is not going to be worn. making ints will be easy!

My client may have different variables than yours, but here’s a mockup (note I did not actually try it in game).

case 1234:// The case for your item itemDef.modelid = 0000;// the id of the model, be careful when assigning it an id, sometimes you can overrite existing models with your new ones. (Ever seen those servers that have death capes as objects? yea? this is why.) itemDef.modelRotationY = 0;// The rotation of the model image over the y axis itemDef.modelRotationX = 0;// The rotation of the model image over the x axis itemDef.modelZoom = 1500;// How zoomed the object is. I believe higher = further. itemDef.modelOffset1 = 0;// offset 1 I believe is the offset for up and down itemDef.modelOffset2 = 0;// offset 2 I believe is the offset for left and right itemDef.stackable = false;// Setting it not stackable client sided (just in case). This will remove the little '1' next to the model if it has it. Remember server sided as well.... itemDef.name = "Donator tickets";// Self explanatory... itemDef.description = "Oh look, some donator tickets.";// Self explanatory... Though you need to add the description AND name server sided too. itemDef.groundActions = new String[] { null, null, "Take", null, null };// Self explanatory... itemDef.itemActions = new String[] { null, "Wear", null, null, "Drop" };// Self explanatory... break;
