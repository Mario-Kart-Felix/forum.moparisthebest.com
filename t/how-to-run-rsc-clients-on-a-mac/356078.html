Well there are a number of ways to get the RSCE client to successfully run on your Mac with OS X Leopard (it may work on other versions of OS X but you may need to update java AND have an Intel Processor.)

Using Script Editor

Search in SpotLight for “Script Editor”

f_v2ghuxqhm_ae7f92b.png441×533

Okay now that you’ve gotten it opened up type this code into Script Editor:

tell application "Terminal" activate do script with command "cd ./desktop/RSCE*" do script with command "sh run.sh" in front window end tell
Now open up “run.bat” with TextEdit. You should see:

java -classpath rsce.jar;lib/xpp3.jar;lib/xstream.jar;lib/hex-string.jar;lib/jmf.jar org.rscdaemon.client.mudclient

Now delete everything from that file and paste this back into it:

java -classpath rsce.jar:lib/xpp3.jar:lib/xstream.jar:lib/hex-string.jar:lib/jmf.jar org.rscdaemon.client.mudclient

Now save it, then rename it “run.sh”

Open up Terminal, you can Spotlight search it, and look at the default directory.

Please, make sure that your RSCEmulation’s client folder is on your desktop, otherwise, this will not work.

Now you’re finished! all you have to do it press Run on Script Editor or save it as an application by going to File and then “save as” be sure to change the format from “script” to “application”

f_v2ghuxqkm_e0233d0.png441×533

Now you have a nicely finished RSCEmulation client running script made especially for your Mac.

Terminal Method

Spotlight search Terminal, and open it.

Now, locate where your RSCEmulation Client is at, and then use the CD command to go to that directory, like this:

cd desktop/RSCE* (the * is used as a wild card, so you dont have to type out RSCEmulation)
After that, paste this into the terminal:

java -classpath rsce.jar:lib/xpp3.jar:lib/xstream.jar:lib/hex-string.jar:lib/jmf.jar org.rscdaemon.client.mudclient

or if youve changed the run.bat into run.sh (from the applescript method above) you could use

sh run.sh

instead.
