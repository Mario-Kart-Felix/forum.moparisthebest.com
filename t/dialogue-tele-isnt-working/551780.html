sk8rdude461 August 4, 2016, 3:10pm #6

yes i know you figured it out, this post still has some resources for you

I’ve always kinda thought it’s stupid how those people never tell you the case ids for dialogues.
News flash people: They don’t change in any 317 client unless you modified the code that sends the clicking buttons, or you modified the actual dialogues in the Client.

case 9157 is the first click option for the 2 line dialogue
case 9158 is the second click option for the 2 line dialogue

Your code is structured a bit strange in 9157.

I loaded the code up into a Java Beautifier because I was too lazy to format it myself, and upon doing so, it was apparent that you were missing SOME brackets.

Here are some tips:

  1. Avoid nesting if statements when possible. In some cases you just have to, but still avoid it.
  2. When doing things like this:

if(c.dialogueAction == 102) { if((c.playerLevel[14] < 90)) { c.sendMessage("You need 90 Mining to Enter the Mining Guild"); } else { if((c.playerLevel[14] > 90)) { c.getPA().movePlayer(3046, 9755, 0); c.sendMessage("You enter the Glorious Mining Guild.."); }
You want to make the code as minimal as possible. You can accomplish this by only checking if the player does not have the requirements.
Ex:

if (c.dialogueAction == 102) { if (c.playerLevel[14] < 90) {// Checking if level is under 90 c.sendMessage("You need 90 Mining to Enter the Mining Guild");// if it is, send a game message break;// breaks the switch statement, so it does not continue after this point. Therefore not teleporting the player anywhere. } c.getPA().movePlayer(3046, 9755, 0);// Made it past the if statement, they have 90+ mining, move them. c.sendMessage("You enter the Glorious Mining Guild..");// send game message }
3) Tabbing is important. I figured out how and where you were missing your brackets based on the tabbing that was seen in the beautifier I linked above. It’s not there for no reason.
4) You may want to removeAllWindows before those sendMessages are sent (Tip: Put it before where it checks the player level so you don’t have to do it twice), otherwise the dialogues will still be shown and they won’t see the message until they walk.
