Objective:
Our goal is to create a server that provides our players and community with an equal and fun experience. When I say “balance,” I mean that we want to provide the skiller side of the community with fair XP rates and rate of supplies being gathered/used. For the combat side of the community, we plan on adding a completely balanced combat triangle so it can be fair to all members.

For those of you who are not familiar with the combat triangle, here is a diagram to explain:

Along with balance, we plan on packing the server full of content. We plan on utilizing our mapdata (508 revision) by writing minigames for all of the minigames that were in existence during the 508 revision. We eventually plan on upgrading to 600+ maps to fill the server with even more content.

We use a git provider for maintaining necessary version control and managing our source code.
If you are unfamiliar with git please read about it here: http://en.wikipedia.org/wiki/Git_(software)

What we’ve accomplished:

[ul][li]We have enabled RSA using 2048 bit keys.[/li]
[li]A proper revision system to help determine third party clients (using protocol, similar to how Jagex initially did it)[/li]
[li]Every single C2S packet has been reviewed and verified. There (should) not be any more spoofs due to unverified data being read from the client.[/li]
[li]The common stream class that was ripped from the client has been rewritten with a much more safe, secure and LEGAL version.[/li]
[li]The ISAAC algorithm was replaced with a much more secure, safe and LEGAL version[/li]
[li]Tasks and actions that correspond to a game character have been added. Also a distanced action has been implemented to ensure that no action will be executed unless you are the proper distance away from the target.[/li]
[li]HD texturing system with support for toggling between HD and SD, underlay and overlay floors are also toggle-able.[/li]
[li]Most of the old shop system was rewritten, now features more safe and smooth shopping and easy to use Currency system[/li]
[li]The old dialogue system has been trashed and rewritten with a more object-oriented approach in mind as well as ease of use. The new dialogue system supports all dialects from mob interaction to destroyable items.[/li]
[li]Various other fixes to tons of systems around the server including but not limited to: world items, world objects, combat, trading, dueling, processing and more![/li]
[li]A thread-safe derivative of Grahams task scheduler has been implemented.[/li][/ul]

We do plan on running beta once we feel comfortable with the status of the server and client as there is still work to be done!
