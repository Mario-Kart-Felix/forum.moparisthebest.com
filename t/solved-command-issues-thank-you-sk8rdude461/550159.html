I’ve also found your issue. You see here:

if (playerCommand.startsWith("pkring") && c.pkPoints > 200) { c.pkPoints -= 200; c.getItems().addItem(6583, 1); c.sendMessage("@red@You have spended 200 Pk Points for a Pk Ring!"); c.sendMessage("@blu@Now you have " + c.pkPoints + " Pk Points!"); } else { c.sendMessage("You must not have a high enough KS to have one of the rewards"); }

It’s checking that the command equals BOTH “pkring” and that the player has enough points.
If EITHER of these are not met, the message will print out.

Now, the easiest solution would be to A: Remove the else statement, or B: add an if statement within the command to check the player has the pk points AND remove the else statement.

Personally, I’d go with option B.

Which would look a bit like:

if (playerCommand.startsWith("pkring")) { if(c.pkPoints < 200) { c.sendMessage("You must not have a high enough KS to have one of the rewards"); return; } c.pkPoints -= 200; c.getItems().addItem(6583, 1); c.sendMessage("@red@You have spended 200 Pk Points for a Pk Ring!"); c.sendMessage("@blu@Now you have " + c.pkPoints + " Pk Points!"); }

But please note:
Nested if statements are bad, when you learn more about the Java language itself, you should look into rewriting the command system to have better logic. The only reason I say to do this now is to fix your issue.

sk8rdude461 August 4, 2016, 2:54pm #15

The NPCDrops.TSM file is an utter pain in the ass.
Seriously.

It was in my server for a while until I eventually changed to a JSON-based drop system.

If I remember correctly, this is the format of the file.

NPCID:RARITY (TAB) ItemID:Amount (TAB) ItemID:Amount
ItemID:Amount (TAB) ItemID:Amount

How it works:
The npc id is the id of the npc, obviously.
the rarity is how hard it is to get the drops on the First line
The Item:amount on the first line are the RARE Drops.
The item:amount on the second line are the Common drops.
You add a single tab between each, but not between the number, colon, and second number.
