davidi2 August 4, 2016, 10:02am #4

If you were running it with an IDE before, it was probably adding the MINA jar to the classpath in its own way through the projects properties. This means you need to add the jar to the classpath in your batch file as well using the command line arguments. This is assuming MINA is packaged as a jar, to be honest I’ve never even downloaded it. If it’s not and it’s just files you want to make sure that the class files are where the project thinks they are (sometimes a bin folder).

You add the jar to the classpath for one program with
-cp path/to/jar.jar
As an argument after ‘java’ in the batch file. If you are sure you have done that correctly, can you paste your run.bat? Hard to see the issue without looking at it.

If you did do the batch properly, it probably does actually mean it cant see the file. Either you forgot to copy it over, the class is missing from the jar, etc.
