xeez July 31, 2016, 9:55am #2

You can use Java’s Robot class to take a picture of the screen via [tt]java.awt.Robot.createScreenCapture(Rectangle)[/tt]. This method will return a BufferedImage. Once you have the two pictures (the screen, and your image) loaded as a ‘BufferedImage’ instance, you can compare the images pixel-by-pixel through [tt]java.awt.image.BufferedImage.getRGB(int, int)[/tt].

Have fun

xeez July 31, 2016, 9:55am #4

You can load the image with the method: [tt]javax.imageio.ImageIO.read(File)[/tt]. Then, as I said, you compare the screen’s pixels with the image’s. This, of course, will provide a tremendously slow application; although I’m not sure if speed is what you are looking for.

EDIT: Since it looks like you want the program to constantly be comparing the screen’s image with your image, and want to develop something similar to “runescape color bots”, I suggest you get the window handle of the window you suspect that will draw the same image as yours (such as the runscape client with runescape color bots), and capture an image of that window. Of course, I’m not so sure that java provides such flexibility with window-handles in general. If this is the case, I’d suggest you use JNI for a (relatively) fast application.

xeez July 31, 2016, 9:55am #6

The method [tt]javax.imageio.ImageIO.read(File)[/tt] will return a “BufferedImage” instance, just like the method you are using to capture an image of the screen. So on both images, you can call [tt]getRGB(int, int)[/tt].

Yes, that is correct. Here are links to some documentations regarding the classes you are to use:
