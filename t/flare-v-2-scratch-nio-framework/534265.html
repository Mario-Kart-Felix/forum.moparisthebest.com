 Since I'm no longer working on this I've decided to post the latest build. This build includes item definitions, an inventory container and Player Saving. The base still stands as before, very flexible and able to hold over 2000 bots. I'm going to be working with Ryley on a NIO server called Brisk so I don't really need this anymore, maybe someone can make use of it. Since the idea was always to keep this server lib free some of the file parsing is a bit hard-coded. The majority of this framework was written by myself with help from Ryley on annotation and I used parts of Lorenzo's saving from Pandemonium. The entire framework is backed by only two Threads executed with the ScheduledExecutorService. The interior design is the same as before with only slight modifications here and there to accommodate new additions.

[center][/center]

The inventory container currently supports the addition and removal of stack-able and non-stack-able items. Everything is all set-up to start on the equipment container. Id recommend re-writing parts of the EquipmentParser since its garbage right now.

[center][/center]

This was an older stability test run before the first version of Flare was posted. How-ever the server is still able to handle this magnitude of players.
