It depends on the values you want to put. If you want to store boolean values, you could use 1 int to represent 32 distinct booleans. if you want to store 2-bit values, you could represent 16 of those with an int. But, in order to get any real meaning out of them, you would have to do something like

public boolean isEnabled(final int mask, final int value) {
    return value & mask;
}

And

private static final int FIRST_FIELD_MASK = 0b00000000000000000000000000000001;// or 0x01
private static final int SECOND_FIELD_MASK = 0b00000000000000000000000000000010;// or 0x02

public void checkIfValueContainsFirstField(final int value) {
    return isEnabled(FIRST_FIELD_MASK, value);
}

But, by doing this, you’re adding a lot of extra code, and creating a maintenance nightmare. You shouldn’t be prematurely optimizing. Make it work, make it work well, and make it work better. I mean, you should never really have to look into optimizing down to a bit level anyways, as the compiler will probably do it better than you ever can anyway.

public static byte[] compress(byte[] input) {
	byte[] compressed = new byte[1];
	for (int i = 0; i < input.length; i++) {
		compressed[0] |= input[i]; // encoding every bit into one (not converted atm)
	}
	byte[] lengthToBytes = ByteBuffer.allocate(4).putInt(input.length).array();
	compressed = Arrays.copyOf(compressed, lengthToBytes.length + compressed.length);
	for (int i = 0; i < lengthToBytes.length; i++)
		compressed[1 + i] = lengthToBytes[i]; // the first byte is the entire stack of bytes
	return compressed;
}

Now how would I decompress those values if I only know the encoded value and the length of the bytes?

Maybe something like this?

public static byte[] decompress(byte[] input) {
	int length = ByteBuffer.wrap(Arrays.copyOfRange(input, 1, input.length)).getInt();
	byte[] decompressed = new byte[length]; //encoded value
	for (byte b = Byte.MIN_VALUE; b < Byte.MAX_VALUE; b++) {
		if ((input[0] & (1 << b)) != 0) {
			decompressed[0] = (byte) (input[0] & (1 << b));
			break;
		}
	}
	return decompressed;
}
