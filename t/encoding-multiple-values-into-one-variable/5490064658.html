Well, it’s not so much that it isn’t great. In some situations it makes total sense. If you have a ton of boolean values for some reason, and you are network constrained or on an embedded system or some other time where memory is an issue, it may indeed make sense to store them as a bitmask - you can then fit 8 of them in one byte, as you mentioned.

This isn’t compression, though. It works because the default type for bool (in C, I’m talking, though obviously the same principles apply in Java with different types/sizes) is 8 bits wide, and you only need to store 1 bit of information. So, by storing it in a more memory-optimal way, you can fit 8 bits in what would previously be one ‘bool’.

You haven’t compressed anything though, just stored it more optimally. If you need to store x bits of information, you either need x bits of space, minimum, or, you need to use some sort of compression algorithm that allows you to reduce the space needed (which will be specific to the data you’re storing - there is no generic compression algorithm guaranteed to reduce the size of any file, it’s mathematically impossible).

And you certainly can’t store infinite bits in one byte. That is just obviously physically impossible.
