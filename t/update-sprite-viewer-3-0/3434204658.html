Zilent July 31, 2016, 5:07pm #24

The sprites are made of a 25 byte header followed by the pixel data represented by an integer array. The culmination of sprites are then put into a zip archive for deployment. In order to properly register them with the RSCD client you’ll need to either explicitly extend the load ranges or load them individually from within the code in the media loading method.
