This is a simple fincolor method I use in Jmacro. It’s very fast, searches the whole screen in a few ms without using much memory. I’m releasing this because I’m ditching this method and working on a much more advanced one.

It works like:

findcolor(Red,Green,Blue,Red Tolerance,Green Tolerance,Blue Tolerance,Case found on screen);

Default for tolerance must be 1 and case must be 0…

So here it is…please leave credits if you use this (doubt most of you leechers will but whatever)…

package com.vibesoft.bots;
import java.awt.*;
import java.awt.event.*;
import java.lang.Object;
import java.awt.Robot; 
import java.awt.image.*;

public class findcolor{
public static boolean findcolor(int realred,int realgreen,int realblue, int redtol, int greentol, int bluetol, int times)
{
boolean found = false;
int loop = 0;
try
        {
        Robot r = new Robot();
        Dimension size = Toolkit.getDefaultToolkit().getScreenSize();
        BufferedImage img = r.createScreenCapture(new Rectangle(size));
		for (int y = 0; y < size.height; y++)
        {
            for (int x = 0; x < size.width; x++)
            {
                Color c = new Color(img.getRGB(x,y));
				int red = c.getRed();
				int green = c.getGreen();
				int blue = c.getBlue();
				if (realred > red - redtol & realred < red + redtol & realgreen > green - greentol & realgreen < green + greentol & realblue > blue - bluetol & realblue < blue + bluetol) {
				r.mouseMove(x,y);
				loop = loop + 1;
				found = true;
				if (loop > times) {
				return found;
				}
				}
            }
        }
        }
        catch (AWTException e)
        {
            e.printStackTrace();
        }
		return found;}
		}
//Call this method with findcolor(R,G,B,R tolerance + or -,B tolerance,G tolerance,times to move); and it will move your mouse when it finds the pixel; 0 for "times" means the first case

Enjoy.

Here is something I just whipped up, try it out:

public static Point findColor(Color clr, int prgrmX, int prgrmY, BufferedImage src) {
	boolean isEqual = true;
	Point point = null;
	ColorModel model = new DirectColorModel(24, 0xff0000, 0x00ff00, 0x0000ff, 0x0);
	int color = model.getRGB(model.createCompatibleWritableRaster(1, 1).getElements(0, 0, null));
	for (int h=0; h<src.getHeight(); h++)
		for (int w=0; w<src.getWidth(); w++) {
			isEqual = color == src.getRGB(h, w) && isEqual;
			if (isEqual) {
				point = new Point(h, w);
				break;
			}
		}
	return point;
}

That will compare a single pixel of the desired color with all the pixels of the image until it finds one of its own color. It will then return the location of that single pixel it found. If it cannot find any pixels of that color, it will return null.

Also, you should declare the ColorModel as a field so that it does not have to be repeatedly re instantiated.
