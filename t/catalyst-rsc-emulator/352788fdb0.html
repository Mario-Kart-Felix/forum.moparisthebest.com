[code]
Jagex Library
Here is a list of all the features which our technology already supports:

2D-Graphics
Scrolling background, designable by a map editor. Ideal for platform games, shoot-em-ups etc…
Pixel perfect collision detection.
Replacement java sprite routines, draws lots of small sprites quickly!
Translucent sprites! Great for reflections, explosions and other special effects.
Shadow masks. Give your sprites shadows for more realism.
Resize sprites. Useful for games which are really 2d but still use perspective.
Graduated or translucent rectangles - (Make great menu boxes.)
Blur and greyscale functions.
Screen transition effects - fade or scroll between screens.
Rotate, flip, or skew sprites.
Ripple effect, ripples any sprite horizontally for a watery effect.

3D-Graphics
Uses our super-optimsed engine for fast 3d graphics.
Create huge virtual worlds, potentially thousands of polygons big.
Resize, rotate, or skew 3d objects in real time!
Perspective correct texture mapping.
Gourad shading, helps make objects look more rounded.
Fogging - looks good and hides distant objects for more speed!
Dynamic light sourcing! Change the direction of the light in real-time for cool effects.
Animating textures, e.g A 3D TV could have an animating screen.
3-Dimensional collision detection.
Interlace option - only draws alternate lines for more speed on slower machines.

Music and sound
Play module files as background music! (can replay .mod .s3m or .xm format)
Play up to 64 sounds or instruments simultaneously!
24-bit mixing and sample-interpolation for optimal sound quailty.
Change the volume or speed of the music. Can also fade in and out.
Speed-up or slow-down soundeffects to change pitch.

Multiplayer games
Uses a 2nd java application which runs on your server and controls the game.
Optimised networking routines ensure smooth gameplay.
Chat facility, allows players to talk to each other.
Waiting room where players can meet and challenge each other to games.
Automatic save games - keep track of players scores.
Scoring system & league table - shows the top players.
Multiple rooms allows hundred of games to run simultaneously.
Link multiple games together into 1 program, so player can jump between them easily.
Produce logs and statistics of everything that happens!
Language filter helps prevent unwanted conversation.

Loading/Saving
Compress data for quicker load times. Uses custom compression engine for comptability with all web-browsers.
Script processor - put all the paremeters of the game in one text file so they can be easily altered to change the game, (e.g number of lives, player speed etc…)
Loading bar - shows the loading progress when the applet is run.

Other
Automatic frame insert system - keeps game running at a constant speed across a range of computers, and uses the extra available power for smoother graphics.
Dual-build - The game will run equally well as an applet on a webpage, or as a standalone java application!
Pop-up window - The game can appear in a new resizable window, rather than being embedded in the webpage.
Hiscore table - Uses a perl cgi to automatically save the best players scores. [/code]
