Best Way to make an Auto-Updater for a Program or Game

General Programming
Lin July 31, 2016, 8:10am #1

Well, there’s something I’m working on. It’s an online multiplayer game, and is coming along good (Has logging in and account creation). However, some point I plan on adding an auto-updater. I’ve done this before, but it previously looked at a file online and read the version. If the version wasn’t didn’t match the one in the program, an update was required. On download, it would download the new EXE… (This was a while back when I wasn’t that good at programming.)

Realizing that wouldn’t work for a professional game (Not saying mine is, because I know it’s not. I’m just saying that because I’m comparing it with other games), I wanted to make it replace the current EXE file. However, I only have one idea in mind that’ll make it so you wouldn’t have multiple EXE files…

Do the version checking thing like previously said, and if it needs to update, close the EXE file (the main game application) and launch another program that actually downloads the update. First it would store the file name of the game application, then kill each instant of the game program and re-download the new one, rename it, and launch it. That way, nothing would have to be done manually.

The problem I see with this is the whole application will have to be re-downloaded and can be bad if the file size gets big (Which it will, unless I decide to load images from an external source and tell it to download new ones if any).

If you can think of another way, please tell me. I was thinking of a plug-in system, but honestly, I’ve never had any experience with one and would like to stick with something like my idea.

Thanks,

~Lin
