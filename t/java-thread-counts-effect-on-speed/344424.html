Just so you know Anthony, ‘System.currentTimeMillis’ should be used as a measure of retrieving the current absolute time, while ‘System.nanoTime’ is more fit to be used as a method of retrieving the elapsed time since a previous call to this function.

It has been reported that on certain windows systems ‘System.currentTimeMillis’ carries a granularity of up to 50 milliseconds, but the documentations state that most systems measure time at a 10 millisecond interval. It’s probably not specifically applicable in this case, but for the future, you should use ‘System.nanoTime’ for getting the time elapsed between events as it will provide much more precision.
