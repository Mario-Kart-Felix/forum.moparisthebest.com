In the Javadocs of the class it states the following:

This class is used to generate native system input events for the purposes of test automation, self-running demos, and other applications where control of the mouse and keyboard is needed. The primary purpose of Robot is to facilitate automated testing of Java platform implementations.
mod_taharok August 4, 2016, 9:07am #3

It’s useful as a really quick way to automate the mouse, keyboard, or to grab screenshots of the screen. All of these functions are clunky and slow, not to mention inconvenient for something like a bot. Directly manipulate the operating system’s message queue or explicitly downloading the contents of a game’s primary frame buffer or the main screen frame buffer used by the OS can be done using much faster functions and avoid a lot of the pitfalls of robot, at the cost of more knowledge, development time, and usage of JNI or JNA.

It’s a very useful class to have sitting around.
