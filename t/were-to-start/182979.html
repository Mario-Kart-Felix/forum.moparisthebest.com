Unhandled July 30, 2016, 7:55pm #2

BCEL is the Byte Code Engineering Library. It allows you to modify compiled Java code (bytecode).
It is needed so that we can figure out which class is which, then modify the class so we can control it. Because Jagex obfuscates the client, the names of the classes change randomly, so we need to look for patterns in the bytecode, with BCEL.
Hooks are essentially code that finds a pattern in the bytecode. You could write a hook to find the ‘client’ class (which is easy, because the client class in named ‘client’ and it contains the ‘main’ method). You could also write a hook to find any other class, by finding a consistent pattern in the bytecode, then looking for it every time the client updates.

as previously stated, bcel is a library for modifying java bytecode. the purpose of a bytecode library is the give structure to the classfile data so that you may programatically modify a classfile. asm is another bytecode library that some other people prefer.

modifying the client by bytecode is needed because it is no longer efficient to attempt to decompile and match on the source of classes, though it may still work if you were to write a decent deobfuscator.
