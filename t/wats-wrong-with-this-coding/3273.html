BobDole81 July 29, 2016, 8:17pm #2

int is a primitive type. Instead, use this:

Integer.parseInt(str[0]);

That will return a primitive int value. If you want to use you’re way, valueOf will return a Integer wrapper class type, after which you need to get the primitive int type value (unless you use jdk 1.5+ which has autoboxing and auto-unboxing)

BobDole81 July 29, 2016, 8:17pm #3

^^

Also, the int parameter in valueOf is the radix parameter, in case you want to read the integer in anything other than base 10. parseInt also has a second parameter for that as well.
