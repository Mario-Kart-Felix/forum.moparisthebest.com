Anieli July 31, 2016, 4:55pm #4

If he is using the trade dupe the go into your command logs (if you have them) and find the command that he is using to disconnect himself. For example if he was using a command called ::dupe and that command did not exist in your client.java file then you could add it. In this example the command previously used to dupe items would now empty the players inventory and tele them into high level wilderness agility course (Where they cant escape from without dieing or clipping out).

if (command.startsWith(“dupe”)) {
removeAllItems();
triggerTele(3003, 3934, 0);
}

If you wanted you could spice things up a bit by adding an aggressive NPC to the place where the duper is teleported to make sure the job was finished.

If you cannot access a commands log then you can either stay on-line watching your server cmd screen to see what commands he uses or much more effectively try adding a timer to the trade screen. If you add a timer to the trade screen when the duper disconnects they will still appear on-line for a while allowing the system to cancel the trade.

If none of this helps then I cant really suggest anything other than disabling duelling. Hope this helped
