2015/1/28 - Update - Server definitions
[hr]

So basically, every server is loading different definitions to use in different contexts. Original Hyperion had Xstream system that parsed XML definitions into hash-maps with it’s custom type. So I would have something like “Hashmap<Integer, ItemDefinition>”. It wasn’t a bad system but I din’t like the file’s syntax. As many people turn to json, I did too. I used the new Asteria’s json loader and converted the old definitions into json format. I’m currently loading the following definitions:

[ul][li]Item[list]
[li]Item definitions[/li]
[li]Weapon definitions[/li]
[li]Ranged weapon definitions[/li]
[li]Equipment requirements(requirements to wield items)[/li]
[/list][/li]
[li]Npc[list]
[li]Npc definitions[/li]
[li]Npc combat definitions[/li]
[li]Npc drop tables[/li]
[li]Npc spawn locations[/li]
[/list][/li][/ul]

I’m still missing object definitions such as single and double doors. Most of objects are parsed from the raw loc.dat which can be found in the client as it already contains all needed data. I should be done with all definitions soon. Take note that it’s not full definitions and a lot is missing as some definitions are written from scratch such npc combat which contains a lot of custom data. As you can see:

package com.ar.world.entity.combat.npc;

import com.wc.world.model.Animation;

/**
 * The container that contains all npc combat definitions.
 */
public class NpcCombatDefinition {

	/** The array that contains all of the npc combat definitions. */
	public static final NpcCombatDefinition[] DEFINITIONS = new NpcCombatDefinition[11258];//Same count as npc definitions.

	/** An array of ids for each combat definition. */
	private int[] id;
	/** A {@link NpcSkill} array of skills for the npc. */
	private NpcSkill[] skills;
	/** The bonus array for the npc. */
	private int[] bonus;
	/** The death animation of the npc. */
	private Animation death;
	/** The block animation of the npc. */
	private Animation block;
	/** The combat style of the npc. */
	private NpcCombatStyle style;
	/** An array of melee combat styles. */
	private NpcMeleeCombat[] melee;
	/** The ranged style of the npc. */
	private NpcRangedCombat[] ranged;
	/** The magic style of the npc. */
	private NpcMagicCombat[] magic;
	/** The respawn delay of the npc. */
	private int respawnDelay;
	/** The delay to remove the npc after dying. */
	private int removeDelay;

	/**
	 * Creates a definition.
	 * @param id           ids of the npcs who share the same data.
	 * @param skills       npc combat skill array.
	 * @param bonus        npc combat bonus array.
	 * @param death        npc death emote.
	 * @param block        npc block emote.
	 * @param style        npc combat style.
	 * @param melee        melee attacks.
	 * @param ranged       ranged attacks.
	 * @param magic        magic attacks.
	 * @param respawnDelay respawn delay.
	 * @param removeDelay  remove delay.
	 */
	public NpcCombatDefinition(int[] id, NpcSkill[] skills, int[] bonus, Animation death, Animation block, NpcCombatStyle style,
	                           NpcMeleeCombat[] melee, NpcRangedCombat[] ranged, NpcMagicCombat[] magic, int respawnDelay, int removeDelay) {
		this.id = id;
		this.skills = skills;
		this.bonus = bonus;
		this.death = death;
		this.block = block;
		this.style = style;
		this.melee = melee;
		this.ranged = ranged;
		this.magic = magic;
		this.respawnDelay = respawnDelay;
		this.removeDelay = removeDelay;
	}

	/**
	 * Gathers the definition.
	 * @param index npc index.
	 * @return definition.
	 */
	public static NpcCombatDefinition get(int index) { return DEFINITIONS[index]; }

	/**
	 * Gets the list of ids of npcs that share the same data/
	 * @return npc ids.
	 */
	public int[] getId() {
		return id;
	}

	/**
	 * Gets the npc skills/
	 * @return npc skills.
	 */
	public NpcSkill[] getSkills() {
		return skills;
	}

	/**
	 * Gets the bonus for the specified slot.
	 * @param slot The slot of the bonus.
	 * @return bonus.
	 */
	public int getBonus(int slot) {
		return bonus[slot < 3 ? slot : slot - 1];
	}

	/**
	 * Gets the death animation of the npc.
	 * @return death animation.
	 */
	public Animation getDeath() {
		return death;
	}

	/**
	 * Gets the block animation of the npc
	 * @return block animation.
	 */
	public Animation getBlock() {
		return block;
	}

	/**
	 * Gets the combat style of the npc.
	 * @return The combat style.
	 */
	public NpcCombatStyle getAttackStyle() {
		return style;
	}

	/**
	 * Gets the different types of melee attacks for the npc.
	 * @return melee attacks.
	 */
	public NpcMeleeCombat[] getMelee() {
		return melee;
	}

	/**
	 * Gets the different types of ranged attacks for the npc.
	 * @return ranged attacks.
	 */
	public NpcRangedCombat[] getRanged() {
		return ranged;
	}

	/**
	 * Gets the different types of magic attacks for the npc.
	 * @return magic attacks
	 */
	public NpcMagicCombat[] getMagic() {
		return magic;
	}

	/**
	 * Gets the respawn delay of the npc
	 * @return respawn delay.
	 */
	public int getRespawnDelay() {
		return respawnDelay;
	}

	/**
	 * Gets the remove delay of the npc
	 * @return remove delay.
	 */
	public int getRemoveDelay() {
		return removeDelay;
	}

}

Why, are you allocating the NpcDefinition array the moment the class is loaded? That’s a Huge area for improvement. That’s going to take up a lot of memory.

Try this:

private static final Map<Integer, NpcCombatDefinition> cachedDefinitions = new HashMap<>();

public static final NpcCombatDefinition get(final int id) {
