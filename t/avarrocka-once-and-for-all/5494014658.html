Try this:

private static final Map<Integer, NpcCombatDefinition> cachedDefinitions = new HashMap<>();

// ...
public static final NpcCombatDefinition get(final int id) {
    if (id < 0) {
        throw new IllegalArgumentException("id must not be null");
    }
    return cachedDefinitions.get(id);
}

[/quote]
The way he’s doing it now is definitely faster than using a map. Although you’re guaranteed O(1) retrieval performance when using Integer as a key for a HashMap, array index retrieval is still much much faster.[/quote]

Yes, but you’re not allocating memory to fit 12345 NpcCombatDefinitions (which looks like a pretty heavy class) right off the bat, if you only load 100 in the first day of running. And it would probably only be <10ms slower for lookup for every time after the first.
