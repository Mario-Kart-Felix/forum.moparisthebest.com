Let’s start off with the basics, learning about the Java SQL function and how it works.

What the package contains;

The java.sql package contains API for the following:

[ul]Making a connection with a database via the DriverManager facility

     [list][li]DriverManager class -- makes a connection with a driver[/li]
     [li]SQLPermission class -- provides permission when code running within a Security Manager, such as an applet, attempts to set up a logging stream through the DriverManager[/li]
     [li]Driver interface -- provides the API for registering and connecting drivers based on JDBC technology ("JDBC drivers"); generally used only by the DriverManager class[/li]
     [li]DriverPropertyInfo class -- provides properties for a JDBC driver; not used by the general user [/li][/list]

Sending SQL statements to a database

      [list][li]Statement -- used to send basic SQL statements[/li]
      [li]PreparedStatement -- used to send prepared statements or basic SQL statements (derived from Statement)[/li]
      [li]CallableStatement -- used to call database stored procedures (derived from PreparedStatement)[/li]
      [li]Connection interface -- provides methods for creating statements and managing connections and their properties[/li]
      [li]Savepoint -- provides savepoints in a transaction [/li][/list]

Retrieving and updating the results of a query

      [list][li]ResultSet interface [/li][/list]

Standard mappings for SQL types to classes and interfaces in the Java programming language

      [list][li]Array interface -- mapping for SQL ARRAY[/li]
      [li]Blob interface -- mapping for SQL BLOB[/li]
      [li]Clob interface -- mapping for SQL CLOB[/li]
      [li]Date class -- mapping for SQL DATE[/li]
      [li]Ref interface -- mapping for SQL REF[/li]
      [li]Struct interface -- mapping for SQL STRUCT[/li]
      [li]Time class -- mapping for SQL TIME[/li]
      [li]Timestamp class -- mapping for SQL TIMESTAMP[/li]
      [li]Types class -- provides constants for SQL types [/li][/list]

Custom mapping an SQL user-defined type (UDT) to a class in the Java programming language

      [list][li]SQLData interface -- specifies the mapping of a UDT to an instance of this class[/li]
      [li]SQLInput interface -- provides methods for reading UDT attributes from a stream[/li]
      [li]SQLOutput interface -- provides methods for writing UDT attributes back to a stream [/li][/list]

Metadata

      [list][li]DatabaseMetaData interface -- provides information about the database[/li]
      [li]ResultSetMetaData interface -- provides information about the columns of a ResultSet object[/li]
      [li]ParameterMetaData interface -- provides information about the parameters to PreparedStatement commands[/li][/list]

Exceptions

      [list][li]SQLException -- thrown by most methods when there is a problem accessing data and by some methods for other reasons[/li]
      [li]SQLWarning -- thrown to indicate a warning[/li]
      [li]DataTruncation -- thrown to indicate that data may have been truncated[/li]
      [li]BatchUpdateException -- thrown to indicate that not all commands in a batch update executed successfully[/li]

[/list]

SQL package Description;

Provides the API for accessing and processing data stored in a data source (usually a relational database) using the JavaTM programming language. This API includes a framework whereby different drivers can be installed dynamically to access different data sources. Although the JDBCTM API is mainly geared to passing SQL statements to a database, it provides for reading and writing data from any data source with a tabular format. The reader/writer facility, available through the javax.sql.RowSet group of interfaces, can be customized to use and update data from a spread sheet, flat file, or any other tabular data source.

Private Servers

Private servers vary from SQL and text file saving, SQL can be updated without logging out and logging back in, this is why some big private servers use the SQL function on their private servers, note, there are other reasons why SQL is used in private servers.
