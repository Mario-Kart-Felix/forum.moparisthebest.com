@OP, I’ve rewritten your game for you (in a more game-like pattern/convention) and implemented an example of OOP instance handling.

package org.moparscape.help.tyb97;

import java.util.Scanner;

public class TextGame extends Thread {

    private enum Stage {
        START_GAME,
        CHOOSE_PATH,
        RESPOND_BOOL
    }

    private Stage stage = Stage.START_GAME;

    private boolean running = false;

    private Scanner scanner = new Scanner(System.in);

    private String lastResponse;

    @Override
    public void start() {
        running = true;
        super.start();
    }

    public void run() {
        while (running) {
            update();
        }
    }

    public void update() {
        switch (stage) {
            case START_GAME:
                System.out.println("In this game you will encounter many choices you have to make.");
                System.out.println("You have to make one right now.");
                System.out.println("Would you like to study something in college, or start a career?");

                stage = Stage.CHOOSE_PATH;
                break;
            case CHOOSE_PATH:
                System.out.println("\nPlease show your choice by typing, 'college' or 'career'.");

                lastResponse = scanner.nextLine();
                if (new ChoiceHandler(lastResponse).validate()) {
                    System.out.println("You picked " + lastResponse + ". Is that what you want?");
                    stage = Stage.RESPOND_BOOL;
                }
                break;
            case RESPOND_BOOL:
                System.out.println("\nYes or No?");

                lastResponse = scanner.nextLine();
                if (new ChoiceHandler(lastResponse).answer()) {
                    running = false;
                } else {
                    stage = Stage.CHOOSE_PATH;
                }
                break;
        }
    }

    public static void main(String[] args) {
        new TextGame().start();
    }

}
package org.moparscape.help.tyb97;

public class ChoiceHandler {

    private static String[] trueStrings = { "yes", "y", "yeah", "yup", "mhm", "duh" };

    private final String response;

    public ChoiceHandler(String response) {
        this.response = sanitize(response);
    }

    public boolean validate() {
        return (response.equals("college") || response.equals("career"));
    }

    public boolean answer() {
        for (int i = 0; i < trueStrings.length; i++) {
            if (response.equals(trueStrings[i])) {
                return true;
            }
        }
        return false;
    }

    public String sanitize(String s) {
        return s.trim().toLowerCase();
    }

}

Hope my code makes sense to you. I tested it and the game flows like I assume you would want it to.

whackatre:

What are the benefits of using getter and setter methods?

The practice of making all class members private and creating accessors (getters) and mutators (setters) is called encapsulation (in java). This is so that when someone wants to use your program they don’t accidentally change fields they shouldn’t change and also helps code understandability.
