Excerpt from the readme:

Accessor Method Injector v0.9 by The Bank

To run this application, simple use the Run.bat (Windows) or Run.sh (Linux/Mac) file. The application should auto-start.

If complications arise, ensure you have the latest JDK installed.

The main class is “injector.MethodInjectorGUI”

This tool is used to inject a custom accessor method into any class you want. This is useful when it comes to an obfuscated class file, as they can rarely be decompiled and recompiled easily. This way, you can easily inject an accessor method into a class without having to write your own injection code, or find a working decompiler tool.

So, what is an accessor method? An accessor method is sometimes called a “get-method”, which is the opposite of a “set-method”. Accessor methods are constructed like this:

public int getBikeSpeed()
{
return bikeSpeed;
}

We use accessor methods for a number of reasons, but if you are using this tool it is probably because you want to return a value which you shouldn’t be accessing under normal circumstances.

How to use this tool:

There are two important folders to this program. The first is the “input” folder, the second is the “output” folder.

The input folder is where you put the class(es) that you want to inject the method into. The output folder is where the final class is placed, after the method is injected. It is worth noteing that the class in the input folder is not changed, in case you choose to go back to an origional version.

Once the application is started, a table will be populated with all the classes in the input folder, you must select the one that you wish to inject a method into.

Under the table, you find three text areas. Method Name, Field Name, and Field Type.

Method Name: The name of the method you want to inject, name it what you want.

Field Name: The name of the field you want to inject, name it what you want.

Field Type: The type of the field you want to inject, this MUST be the real field type, or your accessor method will not work. If you want to use a custom Type that is not built into the JVM (for example, the class “Car” in some project), then you have to include the Class that corrisponds to that Type in the ROOT of this tool (the same directory this README is in).

Press the “Inject” button and your class will have the method injected to it.

Questions/comments etc are appreciated.

I realized I haven’t contributed much lately so, meh.

BAsh July 31, 2016, 3:42am #5

DynamicTypeStorage needs a rethink.

-First you compile the class. I recommend you use the internal method of running the compiler. You can also generate bytecode directly. This would be a preferred option.
-} while (!f.exists()); can block indefinitely.
-Class.forName(“TypeStorage”); will always return the same class in the same classloader. The first time it will create and load the class. Subsequent calls will return the old class that doesn’t reflect the file on the disk. Use a new URLClassLoader each time.
-The method would make more sense as a static method returning the correct Type.
-What you are basically trying to do is “Type.getType(”+ fieldType + “.class);”. Use Type.getType(Class.forName(fieldType)); instead and skip all the messing around with new classes and classloaders.

Its been a long time since I’ve used bcel but I suspect that Type is a string representation of the type. It probably does super(clazz.getName()); I recommend you just miss out the middle man and use the string based constructor.
