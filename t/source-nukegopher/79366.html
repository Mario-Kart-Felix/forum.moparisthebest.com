  1. Syntax of data files

All Gopher page files should be in data folder. They should end with “txt” extension

3.1 Starting pages

To start pages, you must type #START_FILE SITE_NAME. SITE_NAME is selector, which is wanted

“root” is selector for index page

3.2 Adding links & text

Text is easy to add, just write it into file.

#RAW_TEXT: Start raw mode (print text in file straight out.)

All link commands format is same, except #LINK_REMOTE

Format is: link command (tab) shown text (tab) selector

Valid link commands are:

#LINK_DIR: Link to directory text
#LINK_RTEXT: Link to raw text file
#LINK_BINARY: Link to binary file
#LINK_SEARCH: Link to search (no features using it yet)
#LINK_HTML: Link to HTML file

#LINK_REMOTE: #LINK_REMOTE (tab) shown text (tab) selector (tab) server (must be in port 70)

3.3 Adding file links

Easiest way to point into files is to point links into Server:fileList/#path to file#

Or you can just easily define selectors for them
#DATA_FILE (tab) selector (tab) file path

3.4 Defining custom descriptions for Server:fileList folders

Add this in config.cfg: SMsg-folders_name:Message
where folders_name is name of folder. If folder path was ff/ff then it would start with: SMsg-ff/ff

Also, add SDMessage messages for messages in all folders.
