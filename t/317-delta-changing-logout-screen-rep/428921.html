zyle1992 August 2, 2016, 3:14pm #16

Guys, let’s clear something up.

When you’re referring to interface data, most of it is packed in the cache. So when you tell someone that it’s in RSInterface, it’s unlikely that it actually is. If all of the interfaces were hardcoded into the client, it would be much bulkier, and much slower.

There is a reason that Packet 126 exists. It dynamically sets interface text. Otherwise, if the server wanted to make any changes (let’s say, changing the amount of energy a player has), then there would have to be a separate definition for every different possible interface configuration.

All of the text that you see in the game world (bank name, logout text, etc) can be set by the server, requiring no client-side modifications at all.

I’ve seen this plague running rampant through many clients. People hardcoding data like the bank name into the client, bloating it noticeably (in terms of footprint and performace).

There is absolutely NO reason to hardcode any data from existing interfaces into the client.
