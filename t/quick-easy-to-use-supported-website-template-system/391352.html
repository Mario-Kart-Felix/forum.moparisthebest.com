Alright, I’m in the process of writing a better alternative to make my life easier. It’s pretty simple, so looking at it will give you the best idea, but basically, it’s a template system written in PHP that requires no configuration and automatically loads the requested file from a specified folder, automatically outputs the footer and allows you to create any number of helper functions to use in your specific files.

The real beauty and magic of it are a few simple but time saving features:

[ol][li]Just fill out a menu array, and it will not only make a menu with any number of sub menus, but also (optionally) create blank file stubs for you and link them.[/li]
[li]Can be ran from the command line, so, with a simple shell script, it can output for you nice, tidy, pure HTML files that work on ANY host.[/li]
[li]Can run from a standard PHP setup, and makes links such as http://host.com/fname.php?page=home[/li]
[li]Can run with webserver rewrites set up, and makes cleaner links such as http://host.com/home.html (your clients/visitors will never know anything is even scripted!)[/li][/ol]

For example, the re-write I’m using for lighttpd right now is:

url.rewrite-once = ( "^/$" => "/index.php?rewrite", "^/([a-zA-Z_]+)\.html$" => "/index.php?rewrite&page=$1" )
This is what the rewrite rules look like for apache .htaccess files:

RewriteEngine on RewriteRule ^$ /index.php?rewrite [L] RewriteRule ^([a-zA-Z_]+).html$ /index.php?rewrite&page=$1 [L]

I’ve attached the engine+sample website with largish menu and only 2 pages made, so you can see how the stub generation works.

Would any of you use this or something similar? And if so, what other features/options would you like to see?
