iwasinmoparseas101cl July 29, 2016, 8:38pm #14

basically, the GUIs usually: extend JFrame and implement ActionListener.

so:

import java.awt.*;
import java.awt.event.*; //for ActionListener
import javax.swing.*;
public class Stuff extends JFrame implements ActionListener{
      private JPanel jp;
      Stuff(){
           super("Stuff"); //sets the title of the app
           setSize(200,300); //sets the size
           setDefaultCloseOperation(EXIT_ON_CLOSE); //closes program if the upper right x is clicked.
      }
}

Basically after that, JFrame has the following methods (more in depth shown on Sun’s website). setSize(length,width), super(String title), setBackgroundColor(Color c), setForegroundColor(Color c). Those would go in your Stuff() constructor.
You can also create Buttons and JLabels and JPanels (to put the buttons and labels in). So here’s a quick thing:

jp = new JPanel(new FlowLayout()); //FlowLayout is a LayoutManager to basically just put all components on the
                                                          //same line.  GridLayout(rows,cols) is a grid, and BorderLayout() divides the GUI
                                                          // up into North, South, East, West, and Center panels.
JButton button1 = new JButton("OK");       //Creates a new Button
button1.addActionListener(this);              //if button1 is clicked, the ActionListener implementation will happen
JLabel clickOK = new JLabel("Click OK");     //Creates a label (basic text on the GUI)

//now add the created things to the JPanel

jp.add(clickOK);
jp.add(button1);

//then up in your Stuff() constructor after the setDefaultCloseOperation you'd have:

add(jp);  //adds the button and label to the App.

hopefully this is fairly easy to understand for you…i’ll actually write up a tutorial for you later.
