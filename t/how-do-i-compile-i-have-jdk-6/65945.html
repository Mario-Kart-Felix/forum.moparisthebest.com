Well since there is still somebody on this thread who needs help, even though it has been grave-digged, I’ll help you out, lazy loc.

Each version of the JDK (which stands for the Java Development Kit), has a compiler that will compile all Java source code. It is named javac.exe, and is located in pathToJDK/bin/. So, you’ll have to find the path to your JDK, which is usually in C:/Program Files/Java/jdk.version, and then use that. Its a simple command-line tool. Lets use it the simple way.

Create a new batch file named compile.bat
Now, inside it, simply call the JavaCompiler.
To do that, insert the following line with the quotes:
“C:/Program Files/Java/jdk1.6.0_03/bin/javac.exe”
The EXE extension is optional, and you’ll have to change the path to your correct one. By default most computers use C as their main drive, and you’ll have to change your version of JDK in that path yourself.

Now, the Java Compiler has a few of its own features. Like most command line tools, it supports parameters, or arguments if you want to call it that. These are things after the tool is called. They usually have a hyphen (-) before it, or they are just paths to other files the tool needs to use. To compile your source code in a ‘basic-style’, all you will have to do is support which files you want to compile. So after the path to javac.exe, simply have a “*.java” there which tells the compiler to compile all files that end with the JAVA extension. The * character acts as a wildcard.

All in all, you should end up with a simple text document that was ‘renamed’ and converted into a Windows BATCH (.bat) file. There is also something called BATCH Programming, which is also pretty neat to look at. Its not much, but it works. Now your batch file should look like this:

"C:/Program Files/Java/jdk1.6.0_03/bin/javac.exe" *.java

Thats all you really need if you’re going for simplicity. A “pause” command at the end is helpfull, too.
