Another thing you could do, since it’s a dynamic world object (meaning it gets sent to the client via the server), add some code so that the chest is only shown to players who haven’t opened it yet.

public class Player {

    // ...

    public boolean hasOpenedBeginnerChest() {
        return openedBeginnerChest;
    }

    private boolean openedBeginnerChest;
}
if (!player.hasOpenedBeginnerChest()) {
    player.getClientContext().createObject(chest, location);
}

Or, if the default behavior is for the chest to always be there, until you open it (which I’m assuming is what you’re looking for), you can just add a visibility filter, based on whether the player has opened it or not.
