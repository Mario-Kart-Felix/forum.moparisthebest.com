sjustin1996 August 1, 2016, 1:21am #3

Possible problems:

  1. Your no-ip address is wrong (it connects to the wrong IP). In this case, you may have edited the IP input section, which should be left alone.
  2. Your client has the wrong IP in it, or you are typing the wrong IP should this be an IP-Changer client.
  3. Your IPv4 address changed, open cmd.exe and type ipconfig to make sure that you still have the right IP portforwarded.
  4. Your IPv4 address was just plain never put in correctly to portforward.
  5. You have no connection (you couldn’t post here if that was the case, so that’s not a problem here).
  6. You changed the wrong IP in your client - Sometimes there will be more than one file that the IP is stored in to connect to. EDIT: Assuming the client isn’t made by you.
