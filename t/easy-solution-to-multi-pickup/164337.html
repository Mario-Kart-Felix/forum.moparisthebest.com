Or, you could just change how the packets are sent/received between the client and the server. I haven’t looked into it, but the way you are thinking of a solution is much too complicated then it has to be.

If you think about it my way, you can see that when a somebody uses the client and clicks on an item on the ground, it sends a packet to the server to pick it up. Unforunately, winterLove, meant for her server to be somewhat of an emulator. Not the real thing. So not all of the packets have been mastered or discovered fully.

Find the packet that indicates pickup item, which after searching I found it was packet 236, and you can see it’s all right there. After searching through an old server of mine, I found:

			case 236: //pickup item
				int itemY = inStream.readSignedWordBigEndian();
				int itemID = inStream.readUnsignedWord();
				int itemX = inStream.readSignedWordBigEndian();
					if(ItemHandler.itemExists(itemID, itemX, itemY)) 
					if(absX == itemX && absY == itemY) 
                                             {
						if(itemID == 1042 || itemID == 1044 || itemID == 1046 || itemID == 1048 && actionTimer == 0)
						{
							actionTimer += 10;
							server.itemHandler.removeItem(itemID, absX, absY, 1);
							server.itemHandler.addItem(itemID, absX + misc.random(1), absY + misc.random(1), 1, playerId, true);
						} else if(itemID != 1042)
                                                {
						int itemAmount = ItemHandler.itemAmount(itemID, itemX, itemY);
						pickUpItem(itemID, itemAmount);
						ItemHandler.removeItem(itemID, itemX, itemY, itemAmount);
						removeGroundItem(itemX, itemY, itemID);
                                                				resetItems(3214);
                                                }
                                }
				break;

May not look like much, but it is. Just focus on the ItemHandler.removeItem(itemID, itemX, itemY, itemAmount); method and you should be good. It’s most likely the itemAmount that is the problem, and if not, then something in that method that involves the amount of that item being deleted off the ground is messed up. And if all else fails, then the problem will involve the removeGroundItem(itemX, itemY, itemID); method.

There’s a few things to consider and look at. Hope I helped.
