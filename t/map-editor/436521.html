This is a project I’ve been working on for a couple days. It loads maps from the cache and displays their terrain with my own form of rendering, almost perfectly. There is no texture mapping as of now; instead the program just gets the average color of the texture and displays that. This is just a sneak-peak, assuming I’ll add more. I want to add texture mapping, object loading, good blending (I added blending but removed it since it looks like garbage with the lighting), and smoother lighting… And maybe editing. Not sure yet.

EDIT: Got basic terraforming. Not sure if I’ll continue this. If I do it probably won’t pack map files to the cache - instead just generate and possibly compress them.

Anyway, I made this with C# and XNA. I ported it over to OpenGL because of z-fighting and depth issues, only to find out AFTER porting everything how to fix them. The performance is significantly better too. Here’s a couple screenshots.



Features
-Able to display any map at level 0 height (it was programmed to support any level but I’m lazy)
-Basic lighting
-A basic blending (disabled)
-Loads everything from the cache (not really anything significant but I thought I’d point it out)
-Unpacks and saves the title screen music for your enjoyment (/cache/0.midi)

Known Issues
-Lava appears magenta (transparent - not supported in the viewer. All transparent tiles will appear magenta). The overlay ID tiles have a texture of -1, meaning there is no texture. I have absolutely no idea what’s causing this.
-The bottom and right edges of the maps have the wrong height - this is just to display the overlay tiles since only one map region is loaded, the edge tiles don’t connect to the next height map. It’s not a big deal.
-When tilting the camera, some higher overlays might slightly appear over the underlay when they shouldn’t. This is because of the depth bias to get rid of z-fighting between the overlay and underlay tiles. It’s as good as it’s gonna get.

Any insight on the above issues would be helpful. And before I forget, I owe a huge thanks to the people who worked on project rename317. Without their hard work, I probably wouldn’t have been able to figure anything out. Also big thanks to Endian for his cache dumper - I was able to learn how the cache actually works and to unpack files that rely on other index files.

A lot of the loading code is copied from the rename317 client due to it being… sorta pointless to recode it from scratch, so again, big thank you to the owners of that. I also would like to thank Peterbjornx for his map editor, for it helped me when comparing my map results to his editor’s.

Finally, I would like to leave this here for your pleasure. Not my image.

And post-finally, here’s the download to the viewer. Arrow keys to move regions, mouse wheel to zoom, P to pause/unpause the program. You need the XNA 3.1 runtime and .NET 2.0+. Also be sure you have your cache data in a called “cache” in the same directory as the program (program/cache/main_file_cache.dat, etc).

It would have just taken more time to rewrite the code, and since I am kinda lazy…
