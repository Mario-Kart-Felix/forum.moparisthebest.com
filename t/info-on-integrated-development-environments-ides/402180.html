So, I just installed Ubuntu on my PC due to it’s ram failure and it’s ability to only hold about 380mb of memory (rofl). So, since I needed an IDE, I thought I might as well put them to a bit of a test. Judging that only two IDEs are really used around here (NetBeans and Eclipse), I only tested them two.

[hr]

[size=14pt]NetBeans IDE 7.0[/size]
Size: 68.4mb, approx. 200mb when installed.
Features: Base IDE, NetBeans SDK, Java SE

This thing loves to eat ram. Even on this very low ram’ed computer, it still seems to stay alive well, even when editing with it’s visual editor. While editing using this, it spec’ed the following:

CPU: 15% when idle, 45% upod editing
Memory: 317.2MB (85%)
Swap: 211.8MB (33.1%)

[hr]

[size=14pt]Eclipse Helios[/size]
Size: 121.4mb, unknown size when installed.
Features: Base IDE, Little Bits of Goodies.

Seems to use quite a bit of CPU, but this seems to use a hell of a lot less RAM upon short periods of use, but this thing is a leaky bitch. Over time, it increases a huge amount. I’m not really taking that into account though. Seems to load pieces very quick.

CPU: 61% when idle, about the same when editing
Memory: 263.6MB (71%)
Swap: 199MB (31%)

[hr]

In the end, neither wins (LOL), but basically, I recommend NetBeans if you’re PC is relatively fast, but it still can be ran on some of those old assed computers. Eclipse seems to work best when you’re on older computers, but I don’t see why it would be bad on new ones too. Hopefully this is in the write section :|. If not…
