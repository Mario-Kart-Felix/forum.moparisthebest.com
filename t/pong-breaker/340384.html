So this is a little game I had to create for my Data Structures course. It’s a little 2d game that’s a mix between Pong and Brick Breaker. Obviously since it was created for Data Structures, it does in fact include a Data Structure: the Linked List. It uses a doubly linked list; however, we only used the singly linked features ha. The algorithm for the ball is not complete. It works fine when the balls hit the sides of the block, but not the top or bottom.

I’m only including the executable .jar file for this. If you feel so compelled to decompile the class files, fine by me. If anyone knows how to fix our problem with redirecting the ball once it hits the top and bottom of the blocks, feel free to let me know what can be changed. To calculate the side that the ball hits, we used the arctangent of the ball, in relation with the block, to find the angle in degrees that the ball was travelling. If it’s travelling between 45 and -45 degrees, that ball has hit the sides of a block, else it’s hit the top or bottom.

The blocks are loaded into a linked list because the only operations that the blocks require are insertion and deletion. All of which take O(1), constant time, to complete. Thus, we chose the linked list for storing our objects.

Anyways, the .jar is attached. Enjoy, and let me know if anything should be fixed for better performance. Please don’t complain about the repainting Also, I apologize if the ball speed is too fast or too slow. It depends on the computer’s processing speed. I’m sure there are ways around this problem, making it more dynamic, but given the time frame to complete the project, this was not a big focus of ours. If you want to change the ball speed, it’s easy enough to figure it out.
