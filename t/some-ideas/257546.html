Here’s some ideas I was planning to use in a game I was planning to make (mainly features wise)

Project wise, have goals to aim for, otherwise nothing will get done. (Note that nothing is likely to get done anyways, but if we have same actual goals, we won’t waste time doing absolutely nothing)

When fighting NPCs, if you die the NPC will retain some of your items (rest are dropped onto the floor.) The next time this NPC is killed, it will drop the items it picked up, and whoever killed it gets your items.

Have a “test/debug” mode, where stat, inventory, position changing commands are available, and characters aren’t saved. Also, only allowed IPs can access the server at these times.

Have an actual economy - I can’t handle playing a game with a shit economy

Make training faster then real RS, but not ridiculously fast, ie, you can’t get flat 99’s in < 1 day…

No commands etc, available to regular players, and only limited commands available to mods/admins, and only in “test/debug mode”.

Have a reusable inventory object that can be used for shops, banks and the players inventory.

Shops only have a limited amount of money, so cannot buy endless amounts of items (keeping the economy from going over the top), and make more money by merchanting the items they buy/sell. (If they can’t sell the items after a certain amount of time, they can store them or something and get them on special request maybe?)

NPCs have an actual attack, strength, defense etc, level which varies from NPC to NPC.

Scriptable, so that once the core is done, it needn’t be touched.

Also, ground and inventory items use a reference Item object which gets modified.

The HP level is not necessarily the HP value, ie, 99 HP could mean you have 15000 or so HP. Also, hits become much higher to accommodate for this, making the game much more exciting/fast paced etc.

When I remember everything else, I’ll add it to this post.
