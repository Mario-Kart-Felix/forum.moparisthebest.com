sk8rdude461 August 4, 2016, 9:58am #10

Okay if you have cycle events, add something like this (I’m ignoring imports):

Somewhere, declare emote as an int (this all depends on what you’re doing)…

	public int emote = 0;
		CycleEventHandler.getSingleton().addEvent(c, new CycleEvent() {
			@Override
			public void execute(CycleEventContainer container) {
				if(emote == 0){
					c.startAnimation(YOUREMOTEIDHERE!!!);
				}
				if(emote == 1){
					c.startAnimation(SECONDEMOTEID!!!);
				}
				if(emote == 2){
					c.startAnimation(ANDSOON!!!);
				}
				//Now you want to end it too..
				if(emote == 9){
					container.close();
				}
				emote++;
			}
			@Override
			public void stop() {
				emote = 0;
				c.startAnimation(65535);//This is basically a reset anim.
			}
		}, 3);//3 = 3 server ticks = 600 milliseconds x 3 = 1800 milliseconds.

That’s basically it…

Edit: before Java bitches at me, CycleEvents are not easy for everyone to understand, so I made it mostly for him(or her).
The rest is up to you OP.
