COMMENTING
It’s very important, to always comment the code. Sure, some people may think it’s useless. No one is forcing you either to do it, but it is good. You can see what a certain line of a code does, or what a procedure do, so if you may forget it one day, or if you have a script with around 500 lines or less or more, large scripts. It may be hard to control it all, therefore, commenting comes handy. Here is a small example of commenting:

 // Click the banker, and then move the mouse to chat.
procedure click;
begin
ClickMouse(252, 252, True)
wait(3000)
MoveMouse(526, 262);
end;

You can even comment the lines if you want.

Comments dosen’t get compiled.[br][br][hr]

CLEAN CODE
It’s very also important, to have a clean code, with spaces, using the scripting standards. Just plain nice code. It makes it easier for yourself to code, understand and maintain your code much better, and the one that uses your script won’t stop using it just because it looks ugly. The design matters alot for some people, the code have to be nice, clean, and using the scripting standards.

Example at bad code:

procedure blah;
begin
clickmouse(525,252,true)
wait(2000)
if (fish='1')then 
Writeln('bad script');
end;

Example at good code:

procedure Blah;
begin
  Clickmouse(525, 252, True);
  Wait(2000);
  if (fish='1')then 
    Writeln('good script');
end;

USING THE COLOR PICKER
In this chapter, I will teach you how you can use the color picker, It’s a very useful tool. It’s all about using it the right way. Using the color picker, you can get coordinates and color of something…

The Debug box aka Large white box:

You click on the color picker, then let’s say you click on it something red colored. When you have done that, in SCAR, in the bottom. In the degub box (Look at pic above) something like this will show up:

 Color Picked: 8083520 at (14, 509)

The 8083520 is the color of something you have picked, using the color picker and the (14, 509) is the coordinates of that “area” you picked the color at. You can use them both. Here is a small example:

USING THE COLOR 

 // This code will, search for the color 8083520 in the coordinates/area 100, 255, 255 100. And then  click on the "color" which is ClickMouse(x, y, True)
procedure colorfinder;
begin
If (FindColor(x, y, 8083520, 100, 255, 255, 100))then
ClickMouse(x, y, True)
end;

USING THE COORDINATES 

 // This code will click with the mouse on the coordinates 14, 509, picked with the color picker. Notice how we removed those to parantes? It used to be (14, 509) we removed the two ( ) and we used the 14, 509 only 
procedure clickcoord;
begin
ClickMouse(14, 509, True);
end;

VARIABLES
There is diffrent types of variable for use, the most used are string and integer, there are other ones but they are mostly rare.

Variables type:
Integer ==>Stores numbers.
String ==>Stores sequence of letters - Any length.
Longint ==>Same as Integer
Char ==>Stores a single character
Boolean ==>Represents a logical true or false.
Byte ==>Stores data made up of 8 bits.

USING THE VARIABLES
Explaination about how to use the variables.

var x, y:integer;  // x, y is numbers, aka the coordinates, therefore they are a integer, although, they are a character as "x, y" but they contain numbers. 

var hi:string;  // hi, (the string) contains a character, aka something like "Abcd" or anything else. 

var blah:char;  // blah, stores just one char, like only "A" or "b" or "C" nothing more than one. 

var ok:boolean;  // ok, is a statement, which can later be used to do like If(Findcolor blahblahblah then blah:=True

Example of: Integer variables

program New;
var hi:integer;
begin
hi:=2;
Writeln('The variable (hi) which is a integer is number: '+Inttostr(hi));
hi:=5;
Writeln('The variable (hi) which is a integer just changed to number: '+Inttostr(hi));
hi:=hi+1;
Writeln('We made the variable hi + 1, so if it was just was 5 it will be 6 now, so the number now is: '+Inttostr(hi));
end.

Example of: String variables 

program New;
var hi:string;
begin
hi:='bah';
WriteLn(' The string (hi) contains the word: '+hi);
hi:='ah';
WriteLn(' The string, is now changed to...: '+hi);
WriteLn(' It changed to "ah" because we have a code at line 6 which does that ');
end.

Example of: Char variables 

program New;
var hi:char;
begin
hi:='a';
WriteLn('This is a small example of the variable char..the variable char contains: '+hi);
WriteLn('As you can see, it writes only one char, if you do like "ab" instead of "a" it will give a error');
end.

Example of: Boolean variables 

program New;
var x, y:integer; hi:boolean;
begin
If(FindColor(x, y, 00000, 522, 626, 726, 266))then hi:=true else hi:=false;
If(hi=false)then writeLn('The color could not be found..');
end.

Bitmaps are very useful, It’s like findcolor, but instead, it searches for a certain picture(bitmap). The picture always have to be BMP to be able to use it. To use the BMP, you have to make it to a string. Read the instructions…

HOW TO MAKE THE BMP TO STRING

[ul][]Open SCAR
[]On the menu, click on Scripts
[]Now click on “Picture to string”
[]You now got some kind of “form/dialog” open, click on: “Open” find the BMP/Picture and then click OK.
[]On the bottom of the program (SCAR) there will be something with alot of numbers.
[]Voila, you have now made it to a string. You can later copy it and use it, which will be explained in the next instructions.[/ul]

BEGIN … REPEAT
Now, we are going to talk about those parts you usally see in this script, and that you have seen in the bitmap chapter. This is needed, in all scripts. Unless, you don’t want to repeat a certain thing. Which almost all scripts do. This is not something that you need to put in your script, It’s only something to put if you want to “loop” through all your scripts, a simple example first will be put here first:

begin  // Begin all this code 
repeat // Repeat ...(blahp and blahw) 
blahp;
blahw;
until(false) // Forever, which is UNTIL(false) - It's false, therefore it never ends. Until you do it manually. 
end. // End it.

Now, we will put a full working script included that repeat so you can see how it works:

program New;

procedure blahp;
begin
Writeln(' Now we are repeating the procedure "blahp", everytime it repeats this procedure, I will write this text you see now');
Writeln(' Byebye, now it will go to the other procedure and do what it says');
Wait(5000) // We have to wait here, otherwise it will flood very much.
end;

procedure blahw;
begin
Writeln(' So you just came from blahp ? Alright, welcome to blahw! Its repeating blahw at the moment :)');
Wait(5000) // We have to wait here, otherwise it will flood very much.
end;

begin
repeat
blahp;
blahw;
until(false)
end.

You will get the hang of it in no time, you can also change until(said = 50) or anything similiar, which will mean. Whenever “said” is “50” it will stop the script; here is a working script to show that:

program New;

var said:integer;  // This has to be here because, we have said:=said+1; otherwise It wouldn't work.

procedure blahp;
begin
Writeln(' Now we are repeating the procedure "blahp", everytime it repeats this procedure, I will write this text you see now');
Writeln(' Byebye, now it will go to the other procedure and do what it says');
said:=said+1;   // Everytime it says this, It will increase "said" with 1! So it have to say this 50 times until it stop.
Writeln(' Said is at the moment at '+IntToStr(said)+'');
Wait(5000) // We have to wait here, otherwise it will flood very much.
end;

procedure blahw;
begin
Writeln(' So you just came from blahp ? Alright, welcome to blahw! Its repeating blahw at the moment :)');
Wait(5000) // We have to wait here, otherwise it will flood very much.
end;

begin
repeat
blahp;
blahw;
until(said = 5);
end.

1. What is a include?
Includes are “scripts” that makes your scripting easier, instead of sitting and create certain fuctions, or let’s say something that logs out your script, those who have created a include, have already created that code in their script(include) that you can use.

2. How do I use a include?
They usually write all their fuctions in their manual or somewhere else that you can read and see how to use. If not, you can always PM or post in the authors thread and ask him questions you feel for.

3. Where do I get all the includes?
SCAR have got a fuction that can download all the includes; Follow the instructions below.

[ul][]Open SCAR
[]Click on File, in the menu.
[]Choose Download Includes
[]Wait for it to download…and when it’s finished. You got all the includes.[/ul]

4. Help, I’m missing a include, I tried to download all includes through SCAR too!
You can always try to search the forum, for the name of that include or try looking at other forums that have any kind of SCAR section, if you still fail to find it. You can make a post in the “SCAR Help” or “Questions” forum and ask others where you can find this include.

ERRORS/PROBLEMS
Below, there is a small FAQ with errors and how to solve them, It’s even a general FAQ about certain stuff. If you want anything in here, you can always PM me.

SCRIPT ERRORS
Q: I have a script that uses uberwait, I can’t run the script because of that!
A: Change all uberwait, to wait instead.

Q: I get: --(error)-- SCAR was unable to find the client window. Whenever I try to run a script!
A: You forgot to drag over the crosshair to the client window, aka the game or whatever you are trying to use SCAR on.

Q: Line X: [Error] (X:X): Semicolon (’;’) expected in script
A: You have to put a semicolon, at the line it says.

Q: [Error] (X:X): Close round expected
A: You forgot to put another ) at the that line. If the code is something as: If (FindBitMap(blah, x, y)then You will get that error, because you have to put another ) at the end, like this: If (FindBitMap(blah, x, y))

Q: [Error] (X:X): period (’.’) expected
A: It means that it need a period (’.’) at the line it says.

Q: Line X: [Error] (X:X): ‘BEGIN’ expected in script
A: You have forgot to put a begin somewhere, mostly in the procedure.

Q: Line x: [Error] (X:X): Unknown identifier ? in script
A: You have not decleared “?” as a variable or const, or whatever it has to be decleared as.

Q: Include file XXXXXXXXX does not exist.
A: You need to download a certain include; Read the include chapter on how to get the includes.

Q: [Error] (X:X): Unexpected end of file
A: You need to put a end. at the end of the script.

The “X:X” is the number, letter or whatever , which is replaced with the X because we can’t know which line there is a problem.

Random waits
It’s important to make random waits, for some games, If you don’t, it’s way too detectable, Let’s say you want to make it wait 1 second wait(1000) this can be done, safer using random waits, example of that would be:

Wait(800+Random(300)+100)

If you want to make it like 2000, then you could try:

Wait(800+Random(500)+700)

Procedures
Procedures are needed, well, most of the time they are, It’s something that contains a script, well, small part of script, as WriteLn(‘blah’); which will just write blah in SCAR. A example of a procedure is right here;

program blah;

          
procedure okok;
begin
WriteLn('hello');
end;

begin
repeat
okok;
until(false)
end.

As you can see, it does the procedure “okok” all the time, and procedure “okok” contains “WriteLn(‘hello’);” therefore, it does that all the time, and it writes hello all the time!

Constants / Const
Consts, are kinda like settings, you can use, they are different, they can’t be changed, but variables can. Once you put it’s value, it can’t be changed, below I will explain a example of it and what and how it it works.

program new;

// Constants / const
const
cool='1'; //If you are not cool, change the '1' to a '0'

procedure hi;
begin
if(cool='1')then
WriteLn('this guy is cool man!');
begin
if(cool='0')then
WriteLn('this guy is not cool man..');
end;
end;

begin
repeat
hi;
wait(5000)
until(false)
end.

This code, checks, if cool is ‘1’ then it says that this guy is cool, else if it’s not ‘1’ and it’s ‘0’ then it will say “this guy isnt cool”. Constants can come real handy.

Cases
Cases can be used as what the script should do perhaps if a variable b is 1 or 2.

How it works:
Put the script in SCAR and run it, read the comments and you shall find out how it works.

program New;
var
  b: integer;
begin
  b:=random(2)+1; //random(2) will return a 0 or 1, so + 1 makes it 1 or 2
  case b of
    1: writeln('boat');  //what to do if b = 1
    2: writeln('birthday');  //what to do if b = 2
  end;
end. // <3 sdouble for this example. :)
