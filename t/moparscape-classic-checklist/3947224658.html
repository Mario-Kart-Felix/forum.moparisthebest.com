moparisthebest August 1, 2016, 12:16pm #40

It works fine, if you read the errors (make sure debug is checked) they are permission errors, since the client is ran inside a SecurityManager, and they are mostly caused because you don’t read the cache from the cache directory that is set, for example in mudclient.java line 108:

mc.setLogo(Toolkit.getDefaultToolkit().getImage(“data” + File.separator + “Loading.rscd”));

replace “data” + File.separator with the cache directory.

Once you replace all file access calls to only look in the cache directory, then it will run just fine. You only have permission to read/write inside that directory.

edit:
I also don’t like all of jagex’s code being inside the org.moparscape package, I’d rather you just move them up out of there into a generic ‘rsc’ package or similar.
