[CENTER]

Today I’m releasing a project that I have been working on a little while. This is a really nice base to use for people who are just getting started in the RSPS community. It’s a lot better than most Project Insanity bases. I never progressed much with the base, so I am releasing it. I may continue it in the future.

It’s based on http://www.moparisthebest.com/smf/index.php/topic,646084.0.html.

Client Download: http://www.rune-server.org/runescape-development/rs2-client/downloads/474034-updated-317-client-loading-474-data.html

Source Download: Download Delta.Source.zip @ UppIT[/CENTER]

Original features:

  • Netty networking & no thread-per-client
  • Combat(melee, ranged & magic) system from scratch (Not fully done, only a few special attack weapons added, and alot of spells are not added yet)
  • Barrows minigame
  • Basic anti-cheat fix (people that change their interfaces with cheat engine to ancients, bank or a shop interface are automaticly logged and they will not be able to do anything with it anymore)
  • Tons of useless code in process is removed or turned to tasks
  • Book Paging & being able to get the fully paged books back at Larrissa
  • Pickpocketing
  • Firemaking
  • Slayer
  • Crafting (pottery, gem cutting and amulet stringing)
  • Mining (Only the usual ores (tin/copper - runite)
  • Water fillables
  • Item requirements are done with txt (298 requirements added)
  • Item slots are checked by name (should fully work for all items)
  • Clipped following etc.
  • Music in almost all F2P regions
  • Single and double doors work
  • Ship traveling (currently only from port sarim to karamja and back)
  • New dialogue system
  • Barrows degrading (only make items degrade on death left)
    And probably some other things I forgot

Features I added:

  • Re-packaged a bit
  • Experience lamps (Enzo42/bobby4life’s snippet)
  • Fixed stability issues
  • Finished Firemaking
  • Cow Milking
  • Fixed some buttons and interfaces for the 474 data
  • Basic yell system (restrictions/timers/colors)

Things you will need to fix:

  • This source was using the Phanes client, so you will need to change the mining rock ID’s to the ones for the 474 data
  • You may need to adjust a few things for the 474 data
  • Enable music client sided
