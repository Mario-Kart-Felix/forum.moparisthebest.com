Ok this is caused by the buffer getting thrown off. Some packetsize or whatnot is screwed up, so the offset of the stream gets off. When the server reads the next packet type, its really part of the payload of the packet before it. It applies the decryption and gets a negative value, then tries to get the packetSize with packetSizes[packetType].
Now since packetType is negative, arrayIndexOutofBounds is thrown. Fix whatever is causing the offset to be messed up.
