lawl July 31, 2016, 1:40am #14

its probably because (int32 << 32) doesnt work, your compiler should have warned you that you are shifting more than the size of the type

theres several reasons why this code wouldnt work:
unsigned long Stream::readQWord() {
long l = (long)readDWord() & 0xffffffff;
long l1 = (long)readDWord() & 0xffffffff;
return (l << 32) + l1;
}

firstly, unsigned long is 32 bits (on a 32bit platform), not 64 bits, secondly when l is shifted by 32 it just becomes 0 (assumption, dont take my word for that)

this should work (assuming __int64 is 64 bits):
unsigned __int64 Stream::readQWord() {
unsinged int a = readDWord();
unsinged int b = readDWord();
return ( ((__int64) l) << 32) | l1;
}

using | (or) just makes it faster because its less instructions for the cpu and the same result
