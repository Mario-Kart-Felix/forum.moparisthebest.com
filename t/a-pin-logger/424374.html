n3ss3s August 2, 2016, 2:21pm #15

I made one and sold it for a little while, then released it for free. Basically what it does (did, graphics update broke it, cbf updating, not interested) is installs a global mousehook, then when a click occurs (and RuneScape is the active window), it checks if the pin screen is open, if so it located the positions of the digit buttons by the position of the corner of the pin screen (the pin window is the same size/proportions for all screen sizes), then grabs all the orange on the buttons and sorts them into digits by their densities (for doubles like 2, 3 and 6, 9 I checked the amount of points in the bottom row of the point array).

Stub source: http://pastebin.com/QQ5a8bCy (Yes, I’m x0riax)

Feel free to hate on me making it in C# and probable shitness to be found. Main reasons for C# are rather convinient WinAPI access and making a builder using CodeDOM is quite easy…

And yeah, this was definitely the “incorrect” and more painful way to do it, but I don’t know shit about java agents etc that I would’ve needed to be able to reflect the values…

EDIT: I didn’t make it with education in mind,

so just ask if you’re interested in knowing what’s going on at some lines…
