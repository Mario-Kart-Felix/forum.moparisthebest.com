[center]JaGOS - Java Gameserver Operating System[/center]

Background / Information: JaGOS’ aim is to be a database-driven server, the database being mySQL. “Back in the day”, servers were often backended by such mySQL, but this has been diminishing due to “lag” problems. The benefit of mySQL, however, is easily manageable data in a stable environment. You don’t need to toil through a page and a half of gibberish in a data file, when you can open up one of any number of mySQL editors, search for the row you need and be able to edit it quickly and painlessly!

The problem often lied not in the minute lag the backend produced, but in having to install mySQL to run it. JaGOS plans to integrate the mySQL server with the database files readily installed into it. Start it up and away you go, right out of the box!

But why mySQL? The answer is quite simple: efficiency. What logical sense is there in a million lines at the bottom of a file generating text when a function can be used to steamline the process? Any good programmer will tell you: there is no sense in it. All it takes - all it really needs to take - is one function to load a speech window. Another to load a character. Another to load an NPC. Instead of defining, stating, scripting, and writing everything in one file, you can write a few functions to grab and process to it from a database. A quickly shortened and more organized script is the result.

Open Source. We will be asking users of the software to actively contribute in bug reporting and testing while they use the software. Testing is simple - just use the program, and in your using it you will be testing it! Documentation will (eventually) be provided so everyone may edit at whim and customize the server base to be truely theirs.

What is in a name (JaGOS)? The approach behind JaGOS is a new, fresh look at the server world. It avoids any copyright infringements (Java Gameserver Operating System - it’s designed as a Java game server, but it just happens to work with Runescape, so so be it!). It will be built with stability and performance in mind.

The goal? Simply put - to build a fully-functioning, easily modified, official-like server. And learn along the way.

What I am looking for:

[ul][li]Java Programmers (5(ish) people): Java programmers will be necessary for the actual building of the server. Primary focus will be core functions and processes, not spawning or creating NPCs / text or any other additionals like those. IMPORTANT: by “Java programmers”, I mean Java programmers, people who can readily code in the language. People who look at a tutorial in the Tutorials section, flip around code and combine it with four other tuts, change the wording and release isn’t what I am looking for.[/li]
[li]Web / MySQL Developers (2 people): These folks will be in charge of developing the Website included with the software. XHTML / CSSknowledge is a MUST, PHP is a definite plus, and graphics are another big boost.[/li][/ul]
