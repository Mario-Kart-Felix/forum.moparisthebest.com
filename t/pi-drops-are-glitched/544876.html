lesterknome August 4, 2016, 1:26pm #6

So I investigated a little bit, in PI drops are handled as such:

createGroundItem(Client c, int itemId, int itemX, int itemY, int itemAmount, int playerId)

where item amount is loaded as the second parameter in NPCDrops

From here I would check NPCDrops.TSM, find your NPC id, and find the id for the bones being dropped. To drop only one, replace:

boneId:XXXX

with:

boneId:1

Now the server may handle bone drops from a different source since bones are dropped every time instead of as chance. I did find:

public int boneDrop(int type)

in NPCHandler, but could not find anywhere it was called.

Good luck, and let me know if it works out.
