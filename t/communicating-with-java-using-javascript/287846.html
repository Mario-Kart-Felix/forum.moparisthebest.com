The usual means of communicating between a web page and an applet is by means of the applet's parameters. Although these form part of the document, they do not seem to be accessible to JS and so additional public methods for the applet must be provided to set variables which require to be changed by JS programs.

Okay, there are many ways to do this, for example reloading an applet with JavaScript:

document.applet_html_name.init()

The init() function is called only once and is named so because init is an abbreviation of the word initialize or initialisation, which also means to start or begin something. I would use the above example like this:

I never actually thought the ‘init()’ function in an applet could be called directly from JavaScript but there you go, they are not all that different.

I know this isn’t really a tutorial, and it’s not supposed to be, the purpose of this thread is to teach people about communication between Java applets and JavaScript.
