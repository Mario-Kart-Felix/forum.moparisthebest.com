I thought this may be a cool idea to try out, with six years of professional web design on hand, I thought I would attempt to do this myself, considering the fact that I’ve played Runescape quite a bit to re-create it in my browser. Before any confusion, this is NOT a private server, this is a clone / re-create of runescape that is in your browser. Most of us know the game, “Mafia Wars” that was primarily popular on Facebook about a year or two ago, right? That is a browser based MMORPG, some people also call these types of games, “Text Based” games. Before reading further please note the two things. To start, this is not coded in Java, i’m using Html, CSS, Php, MySQL (Main coding languages for web development) and possible some javascript, which is somewhat java coding but different in many ways. And also, note that this isn’t a private server, and this is going to take a while to complete. After its done and I open it in beta mode to about 50 users, i’ll then get all of the bugs out that I can find, and then make the website easy to install for the average Joe, and hopefully release some tutorials on how to edit it. Once again, this is Text / Browser based, no java coding is used for this.

Now, Moving on… This thought randomly popped up a couple days ago and I thought it would be pretty cool to try out and see how it works out. I’ve already started coding a few things, please keep in mind that i’m doing everything from scratch, the log-in box looks exactly like runescape except it’s in your browser, like this text that you’re reading, you can click and drag your mouse over it and highlight the text. There will obviously be things that are different, it’s physically impossible to re-create Runescape 2 without using any Java coding, i’ll just throw that out there. The things you do in Runescape, you’ll be able to do on here, in a mafia wars type theme, lets go through an example, front to back, lets say you want to train your woodcutting level…

You’d click the spellbook on the right side of your screen, the minimap and inventory area acts like a minimap, and will follow you to every web page that you go to. Then click the teleport for skilling, you’ll be presented with a short loading screen, and a teleporting “.gif” short animation will show, then you’ll go the the webpage called ‘skilling.php’. From there, you’ll see a screen with options for different skills, you’d find the tree and click it which will redirect you to the ‘woodcutting.php’ file of the website. On the woodcutting.php file you now need to do some type of work or something to actually get the logs, the logs are then put into your inventory, say the logs have an item id of 600, the .php file will store the log in your MySQL table for the user, under “character_inventory_1” and it will put 600 in the Item_ID column and 1 in the Amount column. Then this will connect with the inventory.php file and tell it to display all of the information in the character_inventory_1 slot and character_inventory_2 slot in the 2nd slot of the inventory, I want to have it set up so that getting a full inventory of logs will give you X Amount of xp per log, and it takes about 4 seconds per log.

Explaining what i’m working on is somewhat hard to do, you’d have to see it to actually know what i’m talking about, i’ll post a test one in a couple days so that people can create accounts, log-in and possibly some other things. I’ve been working on it for about three hours already, I have almost an exact clone of the log-in screen, the text is somewhat noticeably different, and i have a redirect after log-in page, you’ll log in, and then be presented with a loading screen (what is the runescape loading screen when logging in, and it actually works), I have all of the mouse-overs working aswell, so when you hover your mouse over the log-in button, it will glow like it would in runescape. It’s already set up in full screen and can easily support full screen, however, you can drag your web browser to make it different sizes and play at whatever size you’d like.

This isn’t going to be all graphic fancy and such, considering the fact that at first i’m recreating it, and making it playable before I worry about HD pictures, and also the fact that this isn’t coded in java and everything is text/browser based. If you don’t know what a browser based game is, you’re really not going to get much of a concept of what i’m talking about, and chances are you wont until I release a test to the community and have people register and log-in. As for now, I have two pictures, what you see in the pictures has taken me about four hours in total, and 35-45 lines of html code, 150 lines of CSS code, and about 250 lines of php code (for the log-in system), and 12 images have been used so far. Now I need to work on the page you’re presented after you’ve logged in, which will be a two column website base, with the sidebar in the right column (where the minimap, health, inventory, and other things are), and the game in the left column, where you’ll be able to train the skills, do the quests, and where your bank will show up and what not.

Today i’m going to be focusing mainly on that third page after you’ve logged in, primarily the side bar because that’s probably one of the hardest thing to recreate, because there are about 200 active objects and links all in the sidebar, for different tabs, viewing skills, and using all of the teleports and the mage book, re-creating the teleport system will probably be one of the hardest things. You’ll be re-directed to a page when you click a teleport and it will change the info in the MySQL database to say, “Character_Location” 1-50, if you teleport to varrock it may put Character_Location as 2, then the browser reads it and it will re-direct you to the correct page.
