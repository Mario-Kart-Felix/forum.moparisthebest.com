About Parameters

Some of you probably don’t know what a parameter is. Some of you may not even know that your using it but perhaps you don’t. Don’t worry about not knowing, this tutorial will show you what it is and how to use it. You don’t need a high intelligence level to comprehend this as I’ve made it easy for novice programmers to understand it. If you still have some questions at the end, feel free to post and I’ll get back to you as soon as possible.

Let’s begin with defining which method contains a parameter and which doesn’t. First of all, take a look at these two simple methods.

[PHP]
public void teleport(int X, int Y) {
teleportToX = X;
teleportToY = Y;
}[/PHP]

[PHP]
public void teleport() {
teleportToX = 125;
teleportToY = 125;
}[/PHP]

Let’s say that the top method is method A and the bottom one is method B. Method A contains parameters while Method B doesn’t. So, let’s focus on method A for now. You may be wondering, how does it contain a parameter?

public void teleport([b]int X, int Y[/b]) {

Inside the parentheses, contains two parameters. They are X and Y. Both of them are declared as of type int however this is just an example. You can declare these parameters type as Int, Double, Char, String, Byte and a few more. Anyways, when you call upon a method that doesn’t have any parameters, then you don’t fill anything in the parameter field. For example:

teleport();

That would work if we are calling upon method B. However, if we call upon the Method A which does contain parameter fields, then it wouldn’t work. Since there is two parameter fields, we must call upon the method with two parameter fields. If there is 3, we must call upon three and thus 4 and 4 etc.

teleport(1, 2);

That won’t cause any errors. Let’s take a look at the method itself once more.

[PHP]

When we call upon it, the first parameter field we fill in will represent X. The second will represent Y. Please note however, this is an example and other methods may vary. Anyhow, if we’ve declared X as an integer, we must call upon the parameter as an integer. If it’s of type String, we call upon it as String. But you may be wondering how does it teleport using the parameter fields we’ve put in?

[QUOTE]
public void teleport(int X, int Y) {
teleportToX = X;
teleportToY = Y;
}[/QUOTE]

Notice the bold text? We get teleported judging what X equals to. X equals to what ever the first parameter field is. See the flow going on? Why do we do this? It saves room and makes things easier although this is just an example, there are many other methods that use parameters. But let’s take a look at another example.

public void Example(String S) { if (S.equalsIgnoreCase("Poop")) { println("Poop!"); } else { println("No Poop."); } }

When you call upon this method, you won’t be filling the parameter fields with numbers. There is only one parameter field and it’s type is of String. It’s name is S.

Example("Poop");

For example, if we call upon the method like this, then S will equal Poop. In the method, you’ll see this;

 if (S.equalsIgnoreCase("Poop")) {
                println("Poop!");
        } else {
                println("No Poop.");
        }

When we called upon the method, if the first parameter field is Poop (which it is from the example above), then it’ll print the line on the console, Poop!. However, if we didn’t put Poop for the parameter field, then it’ll print the line, No Poop. This can be used for many other types such as double and int.

Hope you get the basics and fundamentals of parameters! If you have any questions, post away.
