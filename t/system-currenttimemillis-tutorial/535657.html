Okay I’m going to attempt to explain to people who want to use System.currentTimeMillis() for timing how they work and why you should use them.
Another reason is because I’m retarded and I have to reteach to myself constantly

First a basic example of how to use these timers

if (CURRENT_TIME - LAST_TIME > DESIRED_DELAY) {
	LAST_TIME = CURRENT_TIME;
	System.out.println("Congratulations! You have just printed this entire line!");
} else if (CURRENT_TIME - LAST_TIME < DESIRED_DELAY) {
	System.out.println("Unfortunately, you have just printed THIS entire line..");
}

CURRENT_TIME: System.currentTimeMillis() which is the current time
LAST_TIME: last time the action occured;
DESIRED_DELAY: your desired delay which can be any positive number(for this tutorial)

What this does exactly is checks to see whether or not the CURRENT_TIME minus LAST_TIME is greater than or less than the DESIRED_DELAY

for example lets give these variables a value
CURRENT_TIME = 10
LAST_TIME = 0
DESIRED_DELAY = 5

If we take a took back at our if statement and plug in this data what would the output be?

if (10 - 0 > 5) {
	LAST_TIME = CURRENT_TIME;
	System.out.println("Congratulations! You have just printed this entire line!");
} else if (CURRENT_TIME - LAST_TIME < DESIRED_DELAY) {
	System.out.println("Unfortunately, you have just printed THIS entire line..");
}

Of course you knew that! Because ten minus zero is greater than five, the ouput would be…

LAST_TIME = CURRENT_TIME;
 System.out.println("Congratulations! You have just printed this entire line!");

Now this is where the true beauty happens. Because ten minus zero is greater than five, LAST_TIME is now equal to CURRENT_TIME.

Therefore, if we were to plug in data right now we would see something similar to this.

if(10 - 10 < 5) //CURRENT_TIME - LAST_TIME < DESIRED_DELAY
System.out.println("Unfortunately, you have just printed THIS entire line..");

However, since CURRENT_TIME is always being increased and LAST_TIME stays the same eventually CURRENT_TIME minus LAST_TIME will be greater than five again.

10 - 10 = 0 < 5
11 - 10 = 1 < 5
12 - 10 = 2 < 5
13 - 10 = 3 < 5
14 - 10 = 4 < 4
15 - 10 = 5 = 5
...then
15 - 15 = 0 < 5
16 - 15 = 1 < 5

Reason to use this? Well, theres probably a better way to handle timers, but don’t ask me. However, I’m positive this is better than sticking this crap inside your process methods.

if(WC_TIMER > 0)
WC_TIMER--;

The reason being that the server doesn’t check the state of this timer the entire time the server is on every 600ms. Instead, it could check it while a new event/thread is running, or only once when a command is issued.
