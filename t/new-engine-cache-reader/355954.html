Hello all.

I havent released anything in awhile since i have been working on my Typhoon rewrite. I have current wrote a Cache Reader that supports the 400 > 604 and 605+ cache support. The naming isn’t perfect and if any of you find any issues with the naming, please provide the relevent information and i will fix it. I tested this and it does work, i ripped the ItemDef class from dragonkk 613 release to test it, but removed it in this release because it was ugly :awesome:.

Credits:

Tom - Decompression Methods
PeeHPee - a few tweeks to the code.
Method - He informed me that the 605+ FIT uses the Whirlpool hash.
XLogic - I got sent his cache reader but it was very ugly so i rewrote it. If it wasn’t for him, then this wouldnt be released.

Link: UppIT - Free File hosting - cache.rar

Some of the code if a bit yukky and does need a better way of doing it, but it works.
Help:

If you want it to support the new fit, then simply do this in the CacheIndex class.

this.cacheFIT = new FileInformationTable(id, index255, Format.NEW);

and to support the old header simply do.

this.cacheFIT = new FileInformationTable(id, index255, Format.OLD);
