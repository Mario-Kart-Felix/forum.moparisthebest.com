[code=java]
private static String removeDuplicates(String string) {
Set visitedCharacters = new LinkedHashSet<>();

	for (char character : string.toCharArray()) {
		if (visitedCharacters.contains(character)) {
			continue;
		}

		visitedCharacters.add(character);
	}

	StringBuilder bldr = new StringBuilder();
	visitedCharacters.forEach(bldr::append);

	return bldr.toString();
}[/code]

Regex solution:

private static String removeDuplicates(String string) { return string.replaceAll("(.)(?=(.*))(?<=(?=\\1.*?\\1\\2$).+)", ""); }

Pattern and Matcher are actually the classes you normally use when you’re dealing with regular expressions in Java.

Basically, Pattern is the class that represents a regular expression, and has a few utilities (such as case insensitivity, which was actually talked about in the thread) to help do more complex regex operations.

For example, if I replaced that line 2 with:

Then:
[tt]Jenniferj[/tt]
would become:
[tt]Jenifr[/tt]

because it is no longer case sensitive, so the last j is still removed.

So basically, just think of “Pattern” as an instance of a regular expression.

The Matcher on the other hand is somewhat like an iterator over all of the possible regex matches in a given string (char sequence). You use these classes when you want to find regex matches, not just replace/split, etc, because it gives you more tools to deal with them.

http://docs.oracle.com/javase/7/docs/api/java/util/regex/Pattern.html

A pattern is a compiled representation of a regular expression; I’ll try and break up David’s post for you:

(.) - matches any except line breaks (equal to [^\r\n])
(.*) - matches 0 or more of the preceding token
\1 - references the first capturing group

The matcher is used to find a match, respecting the compiled regular expression, and while it can you are able to do 3 kind of match operations; Matcher#matches, Matcher#lookingAt and Matcher#find

If the matcher can find a match within the specified text (Jennifer) replace the first subsequence of the input (Jennifer) with the given replacement; the given replacement is ‘$1$2’

$# captures a group and inserts the result of the capture into the specified group

Then iterate and repeat until there are no more matches

davidi2 August 4, 2016, 2:23pm #30

In addition, perhaps one of the most important parts of the pattern are the parenthesis. Any parts of a regular expression inside parenthesis are called capture groups, and when you find matches you can refer to those groupings how AtomicInt_ said. So the (.) is the first letter group, (.*) is the second group, which is basically the text BETWEEN the repeated letters and, \1 is a third part of the regex which isn’t grouped, which is basically just a reference to the first group (the first letter). We don’t group that one because we are just going to discard it. The replace basically says “ok, stick group 1, the first letter, and group 2, the letters in the middle, back into the text, but discard the repeated letter”.

You can get an idea of what is happening with this code:

[code=java] String text = “Jennifejr”;

    // Creates a regular expression pattern instance that is case insensitive.
    Pattern p = Pattern.compile("(.)(.*)\\1", Pattern.CASE_INSENSITIVE);
    
    // We are going to loop until we are done replacing.
    while (true) {
        
        // Create a matcher based on the current text using the compiled pattern. 
        Matcher m = p.matcher(text);
        
        // Can the matcher find any instances of the regular expression in the text?
        if (m.find()) {
            
            // Replace the text, removing the second copy of the character.
            text = m.replaceFirst("$1$2");
            
            System.out.println(text);
            continue;
        }
        
        // No more matches found, end the loop.
        break;
    }
    
    // All done!
    System.out.println(text);[/code]
