Game Engine or Personal Framework

General Programming
Tutorial August 4, 2016, 9:48am #1

Basically, I’ve been really interested in game development lately, so I figured I would try to get everyone’s input on which route you would take. As you might know, there are tons of pre-built game engines out their, such as Unity, and you can do truly amazing things with this without having a ton of programming knowledge. Would you rather use an engine like Unity, or build your own game framework with the support of a graphical engine that works with your desired language?

system August 4, 2016, 9:48am #2

Personally I’d start with some like Valve’s Source Engine and make a HL2 mod, or something along those lines. You have all the models and sounds that you’ll need to create a functioning game, unlike with your own game framework where you’ll have to gather all the models and stuff. Others will probably shoot this down and say that you should start from the bottom and all that, but whatever. Just my opinion.

Yesteryear August 4, 2016, 9:48am #4

I don’t like to use external libraries, but in the case of game development it’s different. You can’t just write a good game framework, especially not a 3d framework.

Protosstribe August 4, 2016, 9:48am #5

If it’s a particular game you’re trying to create, you’re probably better off using some existing librar(y|ies) or framework(s). Unless you have created a library or framework for it yourself already, of course.

For me, however, it’s the framework that’s the most fun part to create. At least for now. I am creating my own engine. Including the 3D software renderer.

Even if you can hack together a working 3D software renderer quite fast, it takes a lot of time to get it right. You will want to be able to reuse all of its parts in other projects.

mod_taharok August 4, 2016, 9:49am #6

I’d agree more with object here. Unless you want to go through years of self-teaching, you can’t just write a game engine from the bottom up without SOME sort of reference (whether that be a book, other engines, engine devs, etc.) A game engine is among the most complex software in the programming industry, so it’s certainly no small feat to accomplish. Unless you already have extensive programming knowledge or are willing to go through the crazy amount of effort it takes to do it on your own, start smaller.
