@protip: Use an IDE(Eclipse/Netbeans)

There is no, I repeat NO reason to run Java under wine.

In Ubuntu, type this command in the command prompt:

This will install the JDK and JRE.

sudo apt-get install openjdk-6-jdk

I would suggest writing your tasks using ant(becoming the standard). So this is what you would do in ant:

<project name="project name here" default="build">
    <!-- Declare some useful variables -->
    <property name="src.dir" value="src" />
    <property name="build.dir" value="build" />
    <property name="jar.file" value="${build.dir}/${ant.project.name}.jar" />

    <!-- Remove old compiled files -->
    <target name="clean">
        <delete dir="${build.dir}" />
        <mkdir dir="${build.dir}" />
    </target>

    <!-- Compile source -->
    <target name="build">
        <!-- if you don't set includeantruntime to false, it will generally complain-->
        <!-- also, you can remove the "listfiles=true" if you don't want to see all the files that were compiled -->
        <javac srcdir="${src.dir}" destdir="${build.dir}" includeantruntime="false" listfiles="true" />
    </target>
</project>

That’s just a little example on compiling your files. For other tasks(compiling, including different classes/libraries, etc) check out the ant documentation.

Now to run your server/whatever else via your Terminal/Console, all you have to do is this:

cd path/to/your/server/files
java package/name/to/Server

Note: You may have to include the -cp argument if you have other libraries(netty/mine, xstream, etc).
