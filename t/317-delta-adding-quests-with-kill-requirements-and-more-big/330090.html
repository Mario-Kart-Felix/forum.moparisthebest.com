In this tutorial I’m going to explain the following subjects:

1 - Creating the variables for quest-progress saving, and saving them to the character’s save file
2 - NpcDialogues. Stringing together stories, and handeling choices
3 - Counting specific NPC kills
4 - Quest rewards

Difficulty: 4/10.

  • This tutorial contains quite a bit of copy-pasting, but also requires some creativity from the reader.

Files used: Client.java, NpcHandler.java, ClickingMost.java, Player.java

Base used: Delta cleaned, but I’m guessing it should also work on different bases.

Alright, let’s begin.

Creating the variables for quest progress saving

An important aspect of questmaking, is being able to save the progress the player has made. Nobody want’s to start a quest all over again after they have logged out, or have gotten disconnected.

To start off, you will need to declare these two variables in Player.java:

public int quest1; // This variable will contain what queststage the player is currently at public int rats; // This is optional. This variable will contain what amount of rats the player has killed

NOTE:
Now in my quest, the player had to kill a certain amount of rats. Ofcourse, you can use a different name for this variable if you are not going to use rats in your quest, but I’m going to use this variable for the rest of the tutorial.

Ok, so you have declared the two variables. Great, now what?
Well, the variables you have added, are currently not getting saved when the player logs out. Let’s change this.

Actually saving the quest progress

Open up Client.java, and search for this:

else if (token.equals("character-rights"))

You should see something like this:

else if (token.equals("character-rights")) { playerRights = Integer.parseInt(token2); }

After the }, add this:

else if (token.equals("character-quest1")) { quest1 = Integer.parseInt(token2); } else if (token.equals("character-rats")) { rats = Integer.parseInt(token2); }

This will load the values of character-quest1 and rats, from the player’s save file, into the variables quest1 and rats.
Great, so you can now load variables from the character’s save file. Now it’s time for the saving itself.

Still, in Client.java, search for:

characterfile.write(Integer.toString(playerRights)

You should find this:

characterfile.write("character-rights = ", 0, 19);
characterfile.write(Integer.toString(playerRights), 0, Integer.toString(playerRights).length());
characterfile.newLine();

After the characterfile.newLine();, add this code:

characterfile.write("character-quest1 = ", 0, 19); characterfile.write(Integer.toString(quest1), 0, Integer.toString(quest1).length()); characterfile.newLine(); characterfile.write("character-rats = ", 0, 17); characterfile.write(Integer.toString(rats), 0, Integer.toString(rats).length()); characterfile.newLine();

Now, you are probably wondering (or not), what the 19 and 17 in this code mean.
This number represents the amount of characters that has to written in the character’s save file.
If you count the amount of characters (spaces are also characters), between the "'s of

"character-quest1 = "

you will see there are 19 characters. In characters-rats = are 17 characters. Hence the 19 and 17.

This is very important! If you do not set these numbers right, the string will not get written in the character’s save file. So make sure you add the right number!

Phew! That was pretty hard, wasn’t it? Well you can relax for now, now it’s time for something a bit easier.

NpcDialogues

Now, before we can continue, you must choose the NPC you want to use as the quest starter.
We will need his NpcID to display his face in the dialogues, so make sure you have one in mind.
For this tutorial, I have picked Sir Vyvin, his NpcID is 605.

To be able to talk to an NPC, we must first create some dialogues.

In Client.java, search for the folowing method:

public void UpdateNPCChat() {

This is the method that handles all the NpcDialogues. It handles all these different dialogues using cases.
Each case has a different number, this way you can choose between different dialogues, or letting dialogues continue into another dialogue, but we will get to that part later.

First, let’s create our first dialogue.
Now this is where your creativity comes in. This is the time to start making up a story for your quest.
To give you an idea for a quest, I’m going to use some of my own dialogues.

In the method UpdateNPCChat(), add this after switch (NpcDialogue) {

break;  // End of the case, this shows the game there is no more code to process in this case.

           // Click here to continue on the dialogue of case 4.

Counting Specific NPC Kills

Ok great, Sir Vyvin gave us the order to kill 5 Rats. Now we need to know how to count how many rats we have killed.
Remember that variable Rats? It’s time to put it to use.

Open up NpcHandler.java, and search for

if ((temp != null) && !temp.disconnected) {

Anything below this code handles what happens when a player has killed an NPC. This is just what we need.

Below this piece of code, add this:

if ((npcs[i].npcType == 47) && (temp.quest1 == 1) && (temp.rats < 5)) { temp.rats += 1; }

Now let’s break this down.

temp = the client of the player
(npcs[i].npcType == 47) // This is the ID of the killed NPC. I used the NPC with ID 47, this is the small lvl 1 rat.
&& (temp.quest1 == 1)   // This means if the killed npc = id 47 and (&&) quest1 is equal to 1
&& (temp.rats < 5))       //  And the variable rats is smaller then 5 Then

temp.rats += 1;            //  Increase the value of the variable rats by 1

Congratulations! You now know how to count specific NPC kills.

Now, in Client.java, find the method

public boolean process() {

And in it, add this code:

if(quest1 == 1 && rats >= 5){ quest1 = 2; sM("I have killed 5 rats, I should return to Sir Vyvin."); }

This code checks if the player is currently working on the quest (quest1 == 1), and if he has killed enough rats (rats >=5).
If the player meets these requirements, quest1 gets set to 2, which enables the player to talk to Sir Vyvin again and recieve his reward. He also gets to see a message so he knows he has killed enough rats.
