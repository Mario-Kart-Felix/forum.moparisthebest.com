This is just an idea I had that (I believe) would be a nice addition to the game.

What I’m thinking is online storage of the different (single player) worlds. This would of course be a feature only available for subscribers. All the world data would stored in Notch’s basement or something similar.

When you select the Single Player mode, the client requests a list of all the worlds, bound to the current account. At which point the server (used for world storage) would send the world data to the client. Now, the user could choose to import the world data, so that it is saved directly to the local machine (for offline playing) and then next time the client connects, it would send the changes to the world storage server (if the user wanted to).

Now, there may be a slight issue with lag. My proposition to fix this would be when the player selects the world they would like to play, the server sends only basic map data (player position and map seed). At which point the client would generate the world normally. Then, the client would request all changes made to the “default” world, reducing the amount of bandwidth usage.

Does anyone else think this is a good idea? If you need further explaining on some of the finer details i would be more than happy to elaborate.
