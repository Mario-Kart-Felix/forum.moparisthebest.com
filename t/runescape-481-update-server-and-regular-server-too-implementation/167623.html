Hi, I’m making a (version 481) update server [edit: now a full-fledged runescape server] implementation, which you may or may not know, sends the current cache to a connecting client. Given the stucture of the recent clients (> 377) (almost entirely static, nearly impossible to follow), doing this is pretty difficult. I would like to open discussion and contribution to this, if anyone is interested.

Involved files are here: http://www.id3y.com/rs/

Questions, comments, and most of all constructive discussion welcome.

Edit, 7 Jan 08:

The update server portion is now fully operational (besides needing a few tweaks, as the client sometimes freezes during the loading sequence; more specifically during “loading interfaces”) and I have moved onto implementing the login session, and am now working on implementing packets. If anyone is interested, I could use alot of help on this, if only discussion. I’ve updated the files at http://www.id3y.com/rs/, so check there. You need to unzip the cache and put [tt].jagex_cache_32[/tt] into whatever folder you’re running the server from.

Most of the client packet transmission operations are based in [tt]Class11.gameCycle[/tt], and the packet parsing is done in [tt]Class53.parsePacket[/tt]. For now, the Runescape server is still on version 481, so if you want to connect to it for debugging purposes, just root around in client for [tt]Class47_Sub11_Sub14.aString2674 = true ? “127.0.0.1” : “se3.runescape.com”;[/tt], and change that value to false. You need to be sure that the RSA modulus ([tt]Class73.n[/tt]) is adjusted according to what server you’re connecting to:
The official RS server modulus: [tt]7162900525229798032761816791230527296329313291232324290237849263501208207972894053929065636522363163621000728841182238772712427862772219676577293600221789[/tt]
