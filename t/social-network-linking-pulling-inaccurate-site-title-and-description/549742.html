Social network linking pulling inaccurate site title and description

General Programming
silabsoft August 4, 2016, 2:20pm #1

This one has been bugging me for awhile. I have a google+ account for my website and occasionally post changes and such on it. Recently I added a new page to my site. It’s not a static page but one that is dynamic and I wanted to announce it on my plus account.

The page I want displayed is: http://cryptospout.com/faucet.php?coin=LTC

However when i post the link in google+ It comes up as:

which would be on this page: http://cryptospout.com/faucet.php?coin=DOGE

The script is pulling the correct meta and title details so I don’t see why it would be grabbing the data from the other page. The only thing I can assume is that because i’ve linked faucet.php before google+ is just lazy and does not requery the data. I would guess that would mean my solution would be to then do a mod rewrite to make a more SEO friendly page url but maybe I am missing something?

[hr]

Problem Solved: Because I was using properties to define a service in the body the social networking sites were using those over the standard meta data at the head of the page. By providing the header with opengraph properties all social media sites now appear to be pulling the correct information.
