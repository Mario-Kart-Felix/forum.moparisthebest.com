i have had trouble with this too. i asked around but didnt find much help. but what i found out is that you have to look very carefully at your client.java lol…so search your client.java for “globalobjects” you will find soemthing that resembles this

[Quote]
public void AddGlobalObj(int objectX, int objectY, int NewObjectID, int Face, int ObjectType) { [/quote]

yours may not be exactly like this but it should have pretty much the same things…object id, y pos, x pos, ect… now you have to follow what your client tells you to do. if it is like mine you should put your object in in this order…

AddGlobalObject(3273(x-pos), 3431(y-pos), 6475(object id), -1(the way you want your object to face), 10(your objects type));//ice chunk(discribes object...this is not needed but can be usefull for furture use)

(this is the codes for what position your object will face) FACE: 0= WEST | -1 = NORTH | -2 = EAST | -3 = SOUTH
((this is the codes for which type of object you have)it will almost always be object type 10) ObjectType: 0-3 wall objects, 4-8 wall decoration, 9: diag. walls, 10-11 world objects, 12-21: roofs, 22: floor decoration

now again the part within the parenthese will NOT vary but the AddGlobalObject may. the only other type of AddGlobalObject ive seen was MakeGlobalObject. if you client already has addglobalobject in it then use that if it has makeglobalobject in it use that. NOTE it will not work if you do not have the right words/order…hope i helped

o and you should put all of your gobal items under this

[quote] public void NewObjects() { [quote] in your client.java…ps make sure you complile after adding objects
