Clawz_fury August 4, 2016, 2:22pm #4

  1. The EventManager shouldn’t even be used anymore. The way most people utilize it makes it not thread safe, and Jagex themselves have stated that all tasks are ran in 600ms intervals making the EventManager (generally) inaccurate. It’s also a bit redundant because the Java API already provides ScheduledExecutorService which can do everything the EventManger can, with a ton more functionality. Use Graham’s new cycle based task system instead, you can find it here.

  2. Yeah, as Vain_ stated that’s a pretty bad idea. One major gain in using tasks is that you’re only processing for (most) things when needed. Everything should have it’s own individual task that is only ran when needed.
