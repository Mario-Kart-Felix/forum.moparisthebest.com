lothy July 31, 2016, 10:39am #47

Generally speaking, a good reliable UDP implementation will very closely imitate the positive features of TCP while giving you that extra control that TCP doesn’t offer.
When using UDP you have several channels that you write to. You may have two channels, reliable and unreliable. You may have extra channels that offer further guarantees, such as ordered arrival.

The whole point in using UDP over TCP though is that, in the long run, it will scale better and you won’t find yourself hitting the wall.

so yes, BUFFER SIZE IS IMPORTANT WITH UDP

also, some packets may not come in order, which means that you should develop some sort of ‘tag header’ which is included in each packet in order for the receiver to sort out the packets properly

the reason most games use udp (regarding mopmans previous posts) is because games have time-specific actions that NEED to be done within a certain time in order for all the clients to be synced. so if its not going to arrive unnoticeable moments after the action was performed, then theyd rather it not come at all and be updated with the next packet to arrive
