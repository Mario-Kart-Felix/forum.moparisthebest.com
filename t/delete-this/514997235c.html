@ReaperXScape
Utilizing the npcdrops.cfg instead of the item2.java, I know that most Allstars using item2.java is adding a bunch of code to both item2.java and npchandler.java each time you want mobs to drop items where the Devolution way is simple and you can have multiple drops without adding a lot of codes into both classes.

in npcdrops.cfg you can do this

----------ID----Item ID-Amount-Chance out of 100 npcdrop = 1 995 10 50 npcdrop = 1 526 1 100

This makes the NPC man drop 10 gp at 50% chance and 100% chance of bones.

While with Item2 and Npchandler
Code ripped out from a Allstar’s Item2

public static int Man[] = { 995,526}; 
public static int randomMan() {
    return Man[(int) (Math.random() * Man.length)];
}</blockquote>

Then in Npchandler

if (npcs[NPCID].npcType ==1) { ItemHandler.addItem(Item2.randomMan(), npcs[NPCID].absX, npcs[NPCID].absY, 1, GetNpcKiller(NPCID), false); }
You'll end up having only 1 drop either 1 gp or 1 bone, which isn't a very efficient drop system.
