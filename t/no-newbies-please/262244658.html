[quote=“semaphore, post:18, topic:26224”]I have to admit that I downloaded the source file out of sheer, morbid curiousity. That sure is a great deal of work just to solve the ‘strange box’.

Why not just hook the ‘main context[sup]1[/sup]’ array and grab index 312. The value in that array is -1 until a strange box interface is open. That int is the entire encoding of the strange box itself (yes, faces and all). Of course, you will have to figure out the encoding yourself. Once you figure it out, imagine the thrill of being able to say ‘I did it myself with only limited help.’ Keep in mind that this approach requires you to grab a single runtime hook and access it. Read only. No writes needed. For a hybrid system, this is almost a necessity as this concept will give you the ability to solve a vast majority of the random events deemed ‘unsolvable’ in scar or scar-esque systems.

Of course, for those of you that insist on doing things the hard way, feel free to continue using a hammer to try and screw a bolt into a slab of concrete, as this post is less for you than those that develop internal or even hybrid systems.

[sup]1[/sup] main context array is the name that I give it. It seems to hold various tidbits of data that the game would need quick access to… ie. run/walk state, attack type, etc. It reminded me of a webserver context and so it was named.[/quote]
