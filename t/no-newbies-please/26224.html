Well, the basic approach that he took was thus:

  1. Find the largest single color area, assume that is the shape
  2. Find the edge of that area, that is the outline of that shape
  3. Count the number of vertecies on the shape, you now know what the shape is (I know a way to do this, but it dosen’t look like its what pups did)
  4. Assume that the only other color area within the shape is the number
  5. Find the border of that colored area (the number)
  6. I’m a bit sketchy on how to continue from here and will have to think on it more… But, basicly you determin the number from the outline that you have.

    public void findMatched(int x, int y, boolean bools[][],
            short colors[][][], RGB color) {
        if (!bools[x][y]) {
            if (colors[x][y][0] == color.r && colors[x][y][1] == color.g
                    && colors[x][y][2] == color.b) {
                bools[x][y] = true;
                findMatched(x + 1, y, bools, colors, color);
                findMatched(x - 1, y, bools, colors, color);
                findMatched(x, y + 1, bools, colors, color);
                findMatched(x, y - 1, bools, colors, color);
            }
        }
    }
