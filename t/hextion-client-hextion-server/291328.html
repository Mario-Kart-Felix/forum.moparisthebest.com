You may call me crazy, or you might not believe me, but here’s the opportunity to test my abilities on my new client project I am holding. My new project consist of a client and server, both of which are blank and use good Java conventions unlike some clients I can mention.

Features

Models
I have added the models without hard coding them, this is my own custom client that is packaged, so far the client is as basic as it gets, there is a little Jframe menu that I have added and a world selection menu. The models are loaded from a child cache that consists all the sprites, interfaces and models I add into there, it is then loaded within a millisecond on start up and does not take a long time to load the cache fully.
The main cache I am using is moparscapes client cache, as it is the only client without no bugs/glitches.

Interfaces
I have added custom interfaces into the cache, some are supplied by moparscape and some are supplied by other forums, the interfaces also get loaded from the sub-cache but do not work 100% yet, they do not override the current interfaces on the main cache and therefore do not cause any glitches.
The “Sprites” also get loaded from the sub cache, (Easy modification)Some modification needs to be done, the game frame that is added doesn’t work properly yet, the mapback is out of place.
I have not tested the models yet, I am still trying to find a 508 + model file, if someone could provide me with one I will start to add the pictures of the models.
