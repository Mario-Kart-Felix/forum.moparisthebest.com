mod_taharok July 30, 2016, 5:21pm #4

If you are using sprites, you should have some sort of array for each the columns and each sprite in those columns. When you are painting, use an offset and loop through the arrays until only the screen is covered with the certain section of map. Then, when the character moves, increase or decrease the offset so that the map paints the next or previous columns/rows.

If you are using one large image, then if it is a BufferedImage, you can use BufferedImage.getSubimage() to get the current offset place of where the character is standing. However, I do not recommend that you use one large image, as it will consume more memory and cpu usage as you continually repaint the game.
