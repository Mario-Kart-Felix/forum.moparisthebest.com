You make more than me and pay far less than I do for my education. Everyone has choices.

Regardless, I disagree with nearly everyone on this thread (like usual). Neither C nor C++ are shit. Having thoroughly learned C and having to write a game engine in it, I can say it is extensively difficult to use for large applications. This doesn’t make it shit. It is an extremely good language for low-level operations. That is both its purpose and greatest strength. It is not even close to shit.

C++ and Java. How are they NOT similar? They do have significant differences, yes, but they are both object-oriented. In fact, much of what Java uses was inspired by C (guess where C++ came from) and C++. C++ happens to be a much more powerful language than Java in most situations, but that doesn’t make it superior. C++ is a much harder language to use than Java, so in most examples, I would recommend Java over C++, assuming those are the two choices. The main reason for this is a combination of compatibility and development costs. Developing in Java is very cheap and easy, versus C++ which is much more expensive and difficult. Why overcomplicate things for no reason? Java is better in most situations, and therefore is my recommendation unless you really need the speed and low level functionality, plus the power that C++ offers.

Seriously, this is not even a debate.
