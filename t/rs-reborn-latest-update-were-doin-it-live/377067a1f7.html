ISAAC should be in both the client and server. It’s a cipher used to encrypt the packet opcode to prevent network sniffing. The ISAAC in private servers is insecure because usually the client session key is sent within the RSA encryption block, but it can be intercepted in private servers. Btw it stands for Indirection, Shift, Accumulate, Add, and Count.
