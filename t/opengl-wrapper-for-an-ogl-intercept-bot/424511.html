hey discarded,
To start get http://code.google.com/p/glintercept/
its only 1 MB
then go to program files where you installed it mines like
“C:\Program Files\GLIntercept_1_0_2”
Copy paste the OpenGL32.dll and gliConfig.ini to
“C:\Program Files\Java\jre6\bin”
in my case that’s where I pasted it.

You might like to use gliConfig_FullDebug.ini so rename that to gliConfig.ini instead. I don’t know the difference but yeah.

It generates a huge call tree of all openGL calls that runescape makes using that data, which is in C++ format as well, don’t move in the game. Anaylze it, bit by bit until you find what you like… to hook.

You’ll notice while the game is even loading it will generate 3 folders Images, Shaders and DisplayLists.

Most importantly you’ll get a file called gliInterceptLog.txt, which will grow in size around 2 megabytes every second which has all the openGL calls.

Images folder has alot of runescape extracted images… like wilderness/prayer skulls, water animations, fonts, admin, player mod skulls,

Shaders folder has txt scripts… which looks like some kind of assembly code…
DisplayLists folder has a bunch of files… each file has around 5-10 lines of openGL calls… but all of they seem to have glBegin() and end at glEnd() they are all saved based on glNewList(####, GL_COMPILE) DisplayList_####_####.txt

I havent figured out why my NUMPAD_0 1,2,3,4,5 etc… doesn’t get triggered in game even though I copied my modified opengl32.dll is loaded into java.
