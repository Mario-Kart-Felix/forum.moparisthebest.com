zstars July 30, 2016, 10:24pm #13

Parsing is not necessarily an easy task, especially, if the language happens to be context-dependant. IDEs that support somewhat advanced “intellisense” usually first obtain an Abstract Syntax Tree of the code, and use it for their code information, code completion, and other features. For some languages, the compiler itself can generate such an AST (or similar), or there are tools available that do so (Usually, to XML format). If that’s not the case, and the language is somewhat hard to parse, you might want to use one of the many free tools / libraries available, such as ANTLR, LEX, YACC or boost::spirit.
(Note: For C++, and for parsing a not too complex language, easiest one to use is probably boost::spirit).
(Note2: Depending on the language and on the features you require, it might not be necessary to parse the whole code, or not necessary to generate an AST at all).
