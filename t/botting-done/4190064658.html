Eh, I got out of it that it’ll be “impossible” for reflection bots to exist. Simply means to me that, at least from the Java side, there won’t be an easy way to access the state of the client. I mean, they could have switched the majority of the client state to native code, thereby eradicating the ability to use the Java reflection API. I believe they are also indefinitely removing unsigned mode, which makes sense because you can’t load custom libraries in the unsigned applet environment (as would be needed if the core of the client was native). As for the enormous amount of time needed to reprogram the majority of the client to using native code, it’s not like they’ve been working on this for a mere weeks. It seems like this update has taken them months. What else would take them months, besides porting their client to a new language?

Seems mostly plausible to me. =/
