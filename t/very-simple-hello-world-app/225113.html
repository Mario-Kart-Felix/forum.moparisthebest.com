Today we will be creating an applet similar to the one that can be downloaded at: http://runesurge.info/tutorials/java/applets/danny/hello%20world%20app.rar

Create a new class with this in it:

[code]/*

  • helloWorld.java
  • Version 1.0.0.0
  • 11-Aug-2008
  • Created by Danny
    */

import java.applet.;
import java.awt.;

public class helloWorld extends Applet
{
public void paint (Graphics g)
{
g.drawString(“Hello World!”, 50, 25);
}
}
[/code]Thanks jensie for the amazing wonderful conventions post.

Your imports are at the topic of the class, these packages are used to give functionality to your applet.

public class helloWorld extends AppletNormally, you would just type something similar to this:

public class helloWorldBut in our case, we want to expand the packages we are using, so, we add this:

extends AppletMore about that can be found really anywhere…

Now we are getting into the method…

public void paint (Graphics g)These are called methods. These are what your applet is really made of. I guess I really can’t explain methods to you now, just google it, like Storm said. But the:

(Graphics g)Enables the method to show the text on your applet.

Now, for the real fun. Whats inside the method.

g.drawString("Hello World!",50,25);This tells the method (Paint), what to do. Methods are like your brain. They send info through your body to make it do a specific function. Like the g.drawString would be used to, in your brain’s point of view, to show text in your head or something like that. After the quotes, you see two numbers. These are the coordinates in which the text “Hello World!” Will be displayed. I can’t really explain coordinates right now, since to be honest, I really wouldn’t know.

Now i forgot, the brackets. The openers and closers of methods. An example of opening a method:

public void goKillYourSelf() {As you can see, there is a bracket [{] after the method. This is the opener.

An example of a closer:

g.drawString("Go Kill Yourself!", 50, 25);}The bracket [}] at the end tells the method that it is finished with it’s job. If you accidentally forget to add a bracket, the result can be nerve-racking.

But, the class itself has an opener and closer as well, don’t forget that!

So, all in all, our goKillYourSelf class turns out like this:

[code]
/*
*Class goKillYourSelf
*Created by Danny
*/
public class goKillYourSelf {

public void goKillYourSelf() {
g.drawString(“Go Kill Yourself!”, 50, 25);
}
}
[/code]Now, to run the applet, you either need to JAR the file, or execute it with HTML. But, since this is only a one-file applet, we will use HTML:

[code]

Helloworld.html by Danny

Wait for the application to load below...

[/code]This loads the main class and executes it on the page.

Thanks for reading this tut!
