I’m looking for a few suggestions on an authentication method, I am considering doing something similar to amazon payments where a user has a key and a secret that they use in the applications api for a payment gateway.

What I have is a server that continuously grabs the Runescape grand exchange data for current prices of items and have it setup for users to submit recent grand exchange transactions buy/sell requests so on. I considered many forms of authentication but I don’t know if choosing the key,secret method is best for this project.

What I want: the ability to have both public access and private access sessions with public access session being throttled and limited to what they can do (for example a public session should not be able to submit data due to risk of poisoning)

What I don’t want to use for authentication methods:

  1. proprietary account - this seems like a waste of time to setup and maintain just for something so simple.
  2. Forum account - Although I would probably attach private sessions to an account for banning and identification purposes I couldn’t use this method for both public and private sessions
  3. toolbar_hash - I considered accepting the runescape toolbar hash and username but I have no way of reversing this information so really does not provide any authentication at all and can be easily spoofed. I also want to keep this information private to the user as it is confidential with the users runescape account.
