Javacisnotrecognized July 30, 2016, 4:36am #13

It’s similar to sending data packets over the network; theres a header, then the main data, then a footer (?). What the compressor puts in the header is information about what bytes have been excluded, the length of some part of a compressed part divided by some number, and then the position of the next header. When the compresser compresses it, it decides which bytes appear the least, and replaces them with a number mentioned in the header. So like, it could replace all the occurances of ‘255’ then ‘42’ with ‘1’, and all of ‘54’ and ‘32’ with ‘2’, and so on. That’s stored in the headers.
