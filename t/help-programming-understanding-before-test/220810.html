While loops are not faster than for loops. At least, not in any self-respecting compiler.

If your language is C++, then you shouldn’t use stdlib.h or stdio.h, but cstdio and cstdlib.

To test whether the machine is big or little endian is extremely easy.

unsigned n = 1;
char * buf = reinterpret_cast<char*>(&n);
if(!buf[0]) printf("Little endian") else printf("Big endian");

In fact, it is possible to do it in two lines, but no point. Also, if you want to use C (which btw is dead and is fugly), feel free to substitute the reinterpret_cast for a C cast.

About your code, it is kinda ugly. And if it is supposed to be C++, there are indeed errors on it. If it is supposed to be plain old C, then maybe not. If you wanna test your code, get a free compiler such as Bloodshed Dev C++.

Although in most contexts it is right to call mingw a compiler, if you want us to get technical, you might want to know that mingw is NOT a compiler, but a collection of tools. It includes several compilers, linkers, and other tools. The actual compiler is g++, and if you check your bin folder in your local Bloodshed Dev C++ installation folder you will probably find that there is indeed a file called precisely g++.exe. Also note that when you use mingw you are using GCC, because mingw is precisely the windows port of the GNU Compiler Collection (That is, GCC).
