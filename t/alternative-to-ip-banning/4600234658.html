There are a lot of reasons you might want to manually set your MAC address for your network card. I won’t ask you what your reason is.

To change this setting, we’ll need to edit the /etc/network/interfaces file. You can choose to use a different editor if you’d like.

You should see the line for your network interface, which is usually eth0. If you have dhcp enabled, it will look like this:

auto eth0 iface eth0 inet dhcp

Just add another line below it to make it look something like this:

auto eth0 iface eth0 inet dhcp hwaddress ether 01:02:03:04:05:06

Obviously you would want to choose something else for the MAC address, but it needs to be in the same format.

You will need to restart networking or reboot to take effect.

Just some ideas:

MAC Address
Process: Client determines current network interface address, and sends it to the server
Pro: Requires more effort for the user to get around
Con: Tools exist to easily change the MAC Address, and change it randomly based on time

Subnet
Process: Server adds last octet (subnet) of the connecting IP to the blacklist
Pro: Changing your IP at the router level doesn’t affect the subnet
Con: There are a limited number of IPv4 addresses, and people in the same neighborhood may have the same subnet address provided by the ISP

Client UID
Process: Randomly generated, or preset number set by the server operator, is sent to the server when a login attempt is made.
Pro: Allows the server operator more control over who can connect.
Con: Very easy to change, not much security.

Client UID - Checksum
Process: Generate numbers that are put into an algorithm, and always return a specified result (look at ISBN numbers)
Pro: In addition to the previous pro, any random tinkering will produce an invalid result.
Con: If the user knows the method you’re using, they have more knowledge, and can fine-tune the tweaking to produce valid results.

Online Registration
Process: Have an online form for user creation, required a captcha/other form of “human-checking”, sending an email to the user, finish verifaction there.
Pro: More effort is required, making people less willing to bypass the ban.
Con: Requires more effort from the user to get to play the game.
