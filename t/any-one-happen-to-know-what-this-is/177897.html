And that is because a packet was mis-read by the server and the offset of the buffer thrown off. So, when it reads the next byte as a packet type and applies the decryption, it gets a negative value.
And with the line of code in the parseIncomingPackets method: packetSize = packetSizes[pktType];
With pktType being negative throws an ArrayIndexOutOfBoundsException. :rolleyes:
Mopar servers these days… anyway:

This is unfixable unless you track down which packet is being mis-read. (Probably wrong packet size)
So just ignore them for now.
