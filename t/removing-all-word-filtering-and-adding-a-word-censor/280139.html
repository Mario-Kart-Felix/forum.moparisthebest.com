Welcome to my shitty tutorial on making it so you can type like this (Got spazzed out)

Or add a word censor. I believe it’s been done before but I’ve never seen. My way takes little effort and uses the replace method.

My way removes all of the word filtering, meaning nothing you type will be changed when sent.

First off, you must comment out two lines. These lines are located in client.java and are what actually filters the text you send.
Note: Skip this part if you don’t want to remove the text formatting.

Find

aString887 = Class35.method527(aString887, 0); aString887 = Class34.method497(aString887, 0);and comment it out (// before each line).

This part is where you add the censor. At the bottom of your client.java, declare a static string array called censorList. It should look like this.

public static string censorList[];

Next, set its value to all of the words you want to filter out. If I wanted to filter out “fuck” like RuneScape does, I’d set it to something like this.

{"fuck","fuc","fuk","fk","fhk","fhc"}So overall, the code would look like this.

public static string filterList[] = {"fuck","shit","fuc","fuk","fk","fhk","fhc"};

Next we’re going to actually filter the text sent.

Back under where you commented out the two lines, add this.

for(int number = 0; number < censorList.length; number++) { aString887 = aString887.replace(censorList[number], "****"); }Now I’ll explain what each thing does.

The first line initiates the start of a for loop, which will make it loop through the censor list. The next line makes it check for and replace the censored word it’s currently looking for. If it finds the censored word at any time, it will replace it with a ****. You can change this to whatever.

Also note that you can have multiple censor lists or make it so you don’t have a list at all. If you decide to not have a censor list, you’ll just do this.

aString887 = aString887.replace("word", "****");But you’ll have to do it for each word. That’s why I had you do it my way.

Anyway, thanks for viewing this. I was in the mood for some client hacking and wanted to remove the word filtering and actually learned a lot from messing with Class35.java, which is how I found this.
