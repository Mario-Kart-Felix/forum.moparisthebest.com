what CFF said.

StorkPK as far as i remember, is dual threaded and not dynamic, which isn’t that hard, however when you add the ability to do x amount of threads you need to do things differently. Writing a server from the ground up isn’t that hard to do with multi threading, but turning the current RSCD sources to having dynamic threading is a different story, not only do you need to change code in hundreds of areas, you need to find them as you play.

At projectrsc, someone else put the multi threading in, and whilst it might have worked well with 1 person online, it was a different story with lots of others, andi wasn’t going to run around patching up each part of code when the server went live, as it would have felt like a disgustingly unfinished server with bugs left right and center.

the performance gain wasn’t worth the amount of time, or crashes/bugs the server would face, not to mention people want the server out now, not next year.
