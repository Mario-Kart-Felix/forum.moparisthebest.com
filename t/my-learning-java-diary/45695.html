Week 1: Day 1:

Got the book, and though “Holy crap, it’s massive”, but then I opened it, and it seemed understandable, and the author seemed to have very good communication skills. I had a look to the introduction of the book, and it explained all about the past of Java. What it was originally created for, how it developed and when, and why by. All the usual mumbo jumbo.

I decided to delve straight in on Day 1. The first few pages were all about what Java can be used for, and why it is used. It also explained about the JDK, which isn’t necessary, but very much recommended. I just decided to use TextPad.

It then went on to explain what Object-Orientated Programming was, and why it was used. It used an example of a Stereo player. All the components are designed to interact with eachother using input and output.

Next it talked about Objects and Classes. A class is a template used to create an object. Every object created from the same class has similar, if not, identical, features. The example used for this was a modem. The objects were different types of modems, all doing pretty much the same job.

Using this, the book showed how to create your first application. I was pretty impressed it wasn’t a ‘Hello World’ app. It was a very lightly based Volcano Robot simulator. I first had to create the Volcano Robot, which was, ‘set to default settings’. I then had to ‘create’ the robot called dante.

Week 1: Day 2:

This chapter was called ‘The ABCs of Programming’. This chapter was pretty helpful, as it got me well known with statements, expressions and operators. The example used for this, was the Russian matryoshka dolls.

For the chapter, we left Classes, Methods and objects alone, and focused on the basic things you can do in a single line of Java.

The first loads of pages was all about what to name the variables and the types of data they are and what-not. byte, short, int and long for the signed types, and then after it went to floating-point numbers.

It then went on to explain about Class types. This was helpful, as it gave me some tips on how to make things quite a lot more easier for me. It then went on to talk about constants, which weren’t used in the examples.

The book then gave a simple example below:

[code=java]public class Variables {

	System.out.println("Level: " + initialLevel);
	System.out.println("Up: " + UP);

This was to get me familiar with System.out.println. This was only a short couple of paragraphs.

Then, the book talked about Comments (Simple), Literals, and Number Literals, as well as Boolean Literals and Character Literals. There was a list of character escape codes. Then there was some talk on String Literals.

Next came the arithmetic and Expressions. Most of it was pretty simple stuff that I grasped and memorized fairly easily. To put me to the text, the book provided this example, to get me even more familiar with how to incoperate the arithmetic in to a script.

As usual, I took the 3 question quiz, and got all of them right without cheating. I also answered the question, that would appear on a Java Certification test, correct.

The script had to calculate how much a $14,000 investment would be worth if it increased in value by 40% in the first year, lost $1,500 in the second year, and increased in value by 12% in the third. This was the script:

[code=java]public class Investment {

public static void main(String[] arguments) {
int investment = 14000;
int origInvest = 14000;

System.out.println("Original Investment: $" + origInvest);
investment = (investment / 2) / 10;
investment = investment * 4;
investment = investment + 14000;
System.out.println("40% increase in value after 1 year: $" + investment);
investment = investment - 1500;
System.out.println("$1,500 loss in value in the second year: $" + investment);
investment = (investment / 10) / 10;
investment = investment * 2;
investment = investment + 1530;
investment = investment + 15300;
System.out.println("12% increase in value in the second year: $" + investment);
System.out.println("After 3 years, the total value of your investment is: $" + investment);
investment = investment - origInvest;
System.out.println("Your total profit is: $" + investment);

}
}[/code]

Week 1: Day 3:

Hmph. By far the hardest day so far. I’m sure a few people would of found it easy, but the books wording was confusing. It was the first day I didn’t have to type any code out.

Day 3 was called ‘Working with Objects’. The topics in this day covered were:

  • Creating Objects
  • Testing and modifying class and instance variables in those objects.
  • Calling an object’s methods.
  • Converting objects and other types of data from one class to another.

The first bit of teaching was about using ‘new’. It was teaching what it was used for, what it does, how to use it. When the ‘new’ operator is used, the new instance of the given class is created, memory is allocated for it, and a special method defined in the given class is called. A constructor.

Next came Accessing and Setting Class and Instance Variables. Most of this bit was about using the dot notation as a way to refer to an object’s instance variables and methods. Next came changing values, which I was fairly confident with. The demo was based on the java.awt.Point package. After came class variables which you define by using the ‘static’ keyword. This gave a lot of examples, and I am fairly confident with this, as well.

Now came calling methods. It was a bit confusing at first, as the demo seemed a bit complicated, but it all was good after the description of each line, and I’m now pretty confident with that.

References to objects came next, and the book made it out to be very hard, but once I had seen the example, it looked pretty easy.

Now came the hard bit for me…Casting and Converting objects and primitive data types. Some of it I got my head around, other bits I had to re-read a few times, until I could semi-figure out what was going on. After came Comparing Object Values and Classes. This was easy stuff to get my head around, and the demos were very helpful. It was how to compare using the “==” and “!=” operators, and using a method “equals()” could easily produce different values.

The book then started talking about using the “instanceof” operator. There was just a couple of paragraphs on this with an example, and it seemed useful and easy to use.

Today I learned how to deal with objects: creating them, reading their values and changing them, and calling their methods. I also, kind of :P, learned how to cast objects from one class to another and class to and from primitive data types. There was a little bit on autoboxing and unboxing, but it will be more deeply explained later in the book.

As usual, I did the quiz, but only got 2/3 questions right this time I got the Java Certification question answer correct
The exercise was worded badly (in my opinion) and I didn’t really know what to do. What I thought you had to do was way too simple, which I could have done within 3 minutes.

End of day 3

Week 1: Day 4:

This chapter was called Lists, Logic and Loops. It was a very easy, understandable chapter. It covered:

  • Making a Java program repeat itself by using loops.
  • How to make a program decide whether to do something based on logic.
  • How to organize groups of the same class or data type in to lists called Arrays.

The first subject was about arrays. This was pretty easy, but the thing I might have trouble remembering, is that the array numbers start at 0. The rest of the array section was all about them, what they do, why they are useful, how to use them ect.

Next was block statements. Only a short bit on this because it’s easy. Just a reminder, as such, of using braces and where/when.

Then all the conditions came next. if, switch (There were 4-5 pages on this), for loops, while and do loops and do-while loops. All of which are easy enough to understand, and don’t pose any real trouble to me. They are also used a lot, so it’s helpful to understand them as early as possible.

Next I learnt how to break out of a loop, by using break; and how to label loops. Labelling loops took up one paragraph, and one example. Easy.

The last bit was the conditional operator, which again, was easy to understand.

Overall, it was an easy chapter that helped me learn a lot of important things which I will, in no doubt, be using later on.

I learned how to declare an array variable, assign an object to it and access and change elements of the array. With the if and switch conditional statements, I can branch to different parts of the program based on a Boolean test. I also learnt about the for, while and do loops and that each enables a portion of a program to be repeated until a given condition is met.

I did the questions, but only got a 2/3 again I also couldn’t answer the certification question as it was too hard

End of day 4
