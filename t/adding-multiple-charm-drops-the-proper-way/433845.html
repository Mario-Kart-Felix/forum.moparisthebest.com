Ok, so my players were recently saying it is too hard to obtain charms on my server.

They suggested i add a similar system to RuneScape where higher monsters drop multiple charms, for example, nex drops 20 and corp drops 13.

So i made this system. Its pretty basicy but does exactly what it does on the tin.

Ok so in npchandler.java add a switch statement. Like this

[code=java] public int multiCharmDrops(int i) {
switch(npcs[i].npcType) {
case 5001:
return 20;

			default:
			  return 1;
		}
	}[/code]

Basically what this does is make a switch statement instead of doing if (npcs[i].npcType == #### || npcs[i].npcType == ####) ect. so its a shorter code with easier functionability.

The case 5001: is my NPC is for nex, and the return value is the amount of charms it will drop, in this case 20.
The default is any npc not specified within the switch statement, making them only drop 1 charm.

Now, in your npchandler.java find something similar to this

you will see something like this.

if(npcs[i].npcType > 0) { int random2 = Misc.random(8); if(random2 == 4) { Server.itemHandler.createGroundItem(c, 12158, npcs[i].absX, npcs[i].absY, 1, c.playerId); } if(random2 == 3) { Server.itemHandler.createGroundItem(c, 12159, npcs[i].absX, npcs[i].absY, 1, c.playerId); } if(random2 == 2) { Server.itemHandler.createGroundItem(c, 12160, npcs[i].absX, npcs[i].absY, 1, c.playerId); } if(random2 == 1) { Server.itemHandler.createGroundItem(c, 12163, npcs[i].absX, npcs[i].absY, 1, c.playerId); } }

replace that whole statement with this

if(npcs[i].npcType > 0) { int random2 = Misc.random(8); if(random2 == 4) { Server.itemHandler.createGroundItem(c, 12158, npcs[i].absX, npcs[i].absY, multiCharmDrops(i), c.playerId); } if(random2 == 3) { Server.itemHandler.createGroundItem(c, 12159, npcs[i].absX, npcs[i].absY, multiCharmDrops(i), c.playerId); } if(random2 == 2) { Server.itemHandler.createGroundItem(c, 12160, npcs[i].absX, npcs[i].absY, multiCharmDrops(i), c.playerId); } if(random2 == 1) { Server.itemHandler.createGroundItem(c, 12163, npcs[i].absX, npcs[i].absY, multiCharmDrops(i), c.playerId); } }

basically this makes it so that instead of dropping 1 charm for every npc, it contacts the switch statement we earlier defined and retruns the value depending on what npc is killed.
