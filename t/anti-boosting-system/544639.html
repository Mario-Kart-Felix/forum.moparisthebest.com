in reality, there will be no way to fully stop boosting. you will need systems in place to check all possibilities, but you will still have to monitor it yourself.

[ul][li]ip addresses can easily be changed.[/li][li]accounts can be created over and over again[/li][li]simply checking the ‘last’ killed, - players can alternate accounts[/li][/ul]

simple example:
[players A, B, C]

[A] kills [B] gets pk points
[A] kills [B] gets nothing
[A] kills [C] gets pk points
[A] kills [B] gets pk points
[A] kills [C] gets pk points

etc…

checks for the address, players can simply connect to another client.

things i would do:
log all ip addresses that every account connects from. this will be useful in many other things.
cross check all ips and accounts before doing logic of the pk points.

log all users killed by you, and all accounts you’ve killed.
i wouldn’t give them points if you’ve killed them within x amount of time
i wouldn’t give them points if they’ve killed them more than they’ve won fights.

kdr system.
if your kdr is too horrible, then you cannot ‘gain’ pk points from this account ?
public viewing, so if it’s ‘too perfect’ you’re noticed.

you could also do something a little unique:
make pk points in the server, dependent on how many players are in the server.

each player starts with [x] pk points.
[A] kills [B]
[A]'s points +1
[B]'s points -1

this will make boosting more obvious / pk points harder to get / kdr more valuable.

if you do this system, you should also create another method of obtaining some pk points so the 'economy' isn't dry due to lack of players. such as voting for some. a quest. a boss. anything.

just don’t allow players to continually +1 every time they kill a player.
