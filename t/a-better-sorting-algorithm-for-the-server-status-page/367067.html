A better sorting algorithm for the server status page

General Discussion
system August 1, 2016, 12:45am #1

I decided to think of a fair yet effective algorithm as to how the list should be sorted (not strictly vote based, not strictly percentage based).

Uptime is highly valued, but people who vote for a server should be valued more - they are human beings with valid opinions and reasoning behind their opinions. However, server reliability should also be valued.

The algorithm I came up with is:

n = p * 0.2d + 2v

n = overall score (higher score means higher on the list compared to other scores)
p = average uptime percentage
d = age (in days)
v = total vote score (positive or negative depending on total upvotes or downvotes)

This algorithm assures that voters have a lot of weight, but server uptime percentage also is very important (servers with 100% uptime will have 100 points right off the bat). Each uptime percentage is worth 1 point, and each vote (positive or negative!) is worth 2 points (negative is -2 points).

Both uptime percentage (server reliability) and player votes (the peoples opinion) are taken into account. I think that this is a pretty good algorithm. What do you guys think?

I think a factor such as “reliability” should be derived from uptime and lifetime. Another factor, “popularity,” would be derived from uptime and votes. A cumulative score would be derived from the reliability and popularity, creating a balanced system.

More explained: uptime is already calculated from lifetime, but it still represents a unique, significant quantity. The higher the uptime in the shorter lifetime, the more meaningless the uptime (thus, a lower reliability). A lower uptime over a greater lifetime is also a lower reliability. Only if uptime can stay consistently high over an extended lifetime will the reliability be high. Relative maximums or some sort of ratio system would need to be used for fairness.

Popularity is a bit more evident, I think. The more votes over the less lifetime is a higher probability (an inverse usage of uptime as from reliability). The less votes over a greater lifetime is obviously a less popular server. Some line of equilibrium would have to be established I think, in order to fairly judge popularity (could be done with averages, I suppose).

These factors could be combined as simply as an average, or they can be scaled. I guess it depends on whether players are more interested in reliability or in popularity (I think they are pretty close to being even, but I have no clue). Another quantitative measurement to consider is a more accurate form of popularity, such as recording the number of times a server profile is viewed, or how many times the play button is clicked.

My two cents, I guess…
