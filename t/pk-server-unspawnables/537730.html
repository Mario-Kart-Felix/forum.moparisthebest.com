There are two real ways you can go about this. (That I find easy enough) Firstly, you can add a return statement in the part that checks the item ID. Or, you can make it seperate.

Example:
Return statement

if(command.startsWith("item")) { //The beginning part of the code that checks the items and such int[] unspawnables = {14484,11694}; for(int i = 0; i < unspawnables.length; i++) { if(unspawnables[i] == /*itemId code*/) { sendMessage("You cant spawn this... blah blah"); return; } } //Rest of the spawn code }

WIth a boolean.

if(command.startsWith("item")) { //The beginning part of the code that checks the items and such int[] unspawnables = {14484,11694}; boolean canSpawn = true; for(int i = 0; i < unspawnables.length; i++) { if(unspawnables[i] == /*itemId code*/) { canSpawn = false; } } if(canSpawn) { //Rest of the spawn code } else { sendMessage("You cant spawn this... blah blah"); } }

I personally recommend the return statement, it looks a lot cleaner in my opinion, but it’s a personal preference if it’s your server.
