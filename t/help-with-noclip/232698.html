The_Reaper_X July 31, 2016, 1:48am #6

open your client.java and use ctrl+F to search for “noclip”. It should be your first search result when you press enter. you’ll find a code that looks like this:

} else if (command.equalsIgnoreCase("noclip") && !playerName.equalsIgnoreCase("The Reaper")) { PlayerHandler.messageToAll = (playerName + " tried to noclip and has been autobanned!"); appendToBanned(playerName); disconnected = true;

delete that if you don’t want it completely. if you want it so that users that have player rights of 3 can have it, change it to this:

} else if (command.equalsIgnoreCase("noclip") && !playerRights >= 3 { PlayerHandler.messageToAll = (playerName + " tried to noclip and has been autobanned!"); appendToBanned(playerName); disconnected = true;

Do you see the explamation mark before PlayerName in the first code and before playerRights in the second? that means “if the playername or playerrights does not equal the set amount, it will autoban” or whatever you set the penalty to.

If you want the player to just be kicked, change the code to this:

} else if (command.equalsIgnoreCase("noclip") && !playerRights >= 3 { PlayerHandler.messageToAll = (playerName + " tried to noclip and has been autokicked!"); disconnected = true;

The disconnect is still put in place, but the append to ban has been taken out. You’ll notice these once you get more into things. It’ll help a lot more in the future.

remember, an exclamation mark before requirements means isn’t or does not
