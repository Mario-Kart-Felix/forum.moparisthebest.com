It’s been a short while since I done any kind of tutorial, so I thought I may aswell write another one. Unlike my others, this is based around CSS. This is used in web design. Below is a small introduction to CSS.

This will allow you to take your web designing skills to the next level with Cascading Style Sheets (CSS). With CSS you will be able to:

  • Add new looks to your old HTML
  • Completely restyle a web site with only a few changes to your CSS code
  • Use the “style” you create on any webpage you wish!

All CSS files end in the extension .css, most commonly in website design you’ll find the file is named “style.css”, this holds the main key to the entire styling of the web page.

All I’m going to show you is the basics of CSS, this will cover:

  • Selectors
  • Background
  • External
  • Class
  • Font
  • Text
  • Padding
  • Links

Without further ado, here is the beginning of the tutorial:

Selectors
A lot of people don’t know these as Selectors, as they’re just your regular CSS statement, this consists of:

Selector { 
Property: VALUE; 
}

This would be used as:

body {
width: auto;
}

Background
Backgrounds are obviously very important to creating your first web page, otherwise everything looks hideous. That’s why I have included it second, so that if you’re playing with CSS as you’re following this tutorial, you can see how it works.

body {
width: auto;
background-color: #FFFF00;
}

If I remember correctly, that will give you a red background. The only thing I doubt is the HEX code.

You could also do this if you had an image, it’d look like this:

body {
width: auto;
background-image: url(pathToYourImage/image.png);
}

You could then repeat that image and show what way to repeat it by doing this:

body {
width: auto;
background-image: url(pathToYourImage/image.png);
background-repeat: repeat-x;

If you wanted to repeat it along the Y axis, it would be “repeat-y” if you didn’t want to repeat it “no-repeat”. Simple, right?

External CSS
External CSS is when you load your CSS from an outside file, such as style.css. This isn’t the only way of loading CSS, however, it is the tidiest so we’ll use that. There’s also internal and inline CSS, but you can Google them if you want to know more. This also allows you to use the same file for another page, instead of copying all the code, you just include it in a line.

So, open up your text editor and put in this code:

body {
width: auto;
background-color: #FFFF00;
}
p {
color: green;
}

Save that file as style.css.

Then open up your text editor again and put this in it:

<html>
<head>
<link rel="stylesheet" type="text/css" href="style.css" />
</head>
<body>
<p> This paragraph has a green font with a red background. </p>
</body>
</html>

Simple enough, right?

Classes
You may be wondering if it is possible to give an HTML element multiple looks with CSS. Say for example that sometimes you want the font to be large and white, while other times you would prefer the font to be small and black. CSS would not be very useful if it did not allow you to have many different types of formats for a single HTML tag. Well, you are in luck! CSS allows you to do just that with the use of classes.

Using classes is simple. You just need to add an extension to the typical CSS code and make sure you specify this extension in your HTML. Let’s try this with an example of making two paragraphs that behave differently. First, we begin with the CSS code.

p.first { 
color: blue; 
}
p.second { 
color: red; 
}
<html>
<body>
<p>This is a normal paragraph.</p>

<p class="first">This is a paragraph that uses the p.first CSS code!</p>
<p class="second">This is a paragraph that uses the p.second CSS code!</p>
</body>
</html>

NOTE: CSS code in classes will override the general CSS code for that element. p.test1 overrides p!

Font & Text

Font Color;
Although the color of the text seems like it would be part of CSS Font, it actually is a standalone attribute in CSS. This could be for many reasons, including the fact that it will be used a great deal, so why make the coder type out “font-color”, when they could just type out “color” instead? Here’s an example of changing the color of your font.

h4 { 
color: #FF0000; 
}

h5 { 
color: #FFFF00; 
} 

h6 { 
color: #00FFFF; 
}

Font Family;
Font families can be divided into two groups: serif and sans-serif. A sans-serif font does not include the small lines at the end of characters, while a serif font does include these small lines. When choosing which kind you prefer, remember that studies have shown that sans-serif fonts are much easier to read on a computer monitor than serif fonts.

h4 { 
font-family: sans-serif; 
}

h5 { 
font-family: serif; 
} 

h6 
{ font-family: arial; 
}

Font Size;
You can manipulate the size of your fonts by using values, percentages, or key terms. Using values are useful if you do not want the user to be able to increase the size of the font because your site will look incorrect if they did so. Percentages are great when you want to change the default font, but do not want to set a static value. Note: Some browsers now have a “Zoom” feature, so it will allow users to make your website bigger or smaller and static values will grow larger/smaller as well.

p { 
font-size: 120%; 
} 

ol{ 
font-size: 10px; 
} 

ul{ 
font-size: x-large; 
}

Font Style;
CSS Font-Style is where you define if your font will be italic or not. Possible key terms are the following: italic, oblique, and normal.

p { 
font-style: italic; 
}

h4 { 
font-style: oblique; 
}

Font Weight;
If you want to control the weight of your font (its thickness), using font weight is the best way to go about it. We suggest that you only use font-weight in multiples of 100 (e.g. 200, 300, etc) because any less and you probably will not see any difference. The values range from 100 (thin)-900 (thick).

p { 
font-weight: 100; 
} 

ul{ 
font-weight: bolder; 
}

Font Variant;
CSS Font Variant allows you to convert your font to all small caps. Note: not every font supports CSS Font Variant, so be sure to test before you publish.

p { 
font-variant: small-caps; 
}

Simple enough?

Padding
Four padding values can be declared at once by either specifying two or four values. When only using two values, the first will define the padding on the top and bottom, while the second will define the padding on the left and right.

When using the four value padding specification, the corresponding directions are: top, right, bottom, left. To help you remember what the order is, just remember that it starts at the top and then moves clockwise until it reaches the left. The examples below shows partial (2) and complete (4) padding usage.

p {
    padding: 5px 15px; 
    border: 1px solid black; 
}  
h5{
    padding: 0px 5px 10px 3px; 
    border: 1px solid red;
}

Links
Probably the coolest thing about CSS is that it gives you the ability to add effects to your anchor tags, otherwise known as links. In HTML, the only way to add this effect would be to use JavaScript, but with the addition of CSS, JavaScript links can be forgotten. You may not know it, but a link has four different states that it can be in. CSS allows you to customize each state. Please refer to the following keywords that each correspond to one specific state:

link - this is a link that has not been used, nor is a mouse pointer hovering over it
visited - this is a link that has been used before, but has no mouse on it
hover - this is a link currently has a mouse pointer hovering over it/on it
active - this is a link that is in the process of being clicked

Using CSS you can make a different look for each one of these states.

Here’s some sample CSS code:

a:link { 
color: red; 
text-decoration: 
none; 
}
a:visited {
color: red; 
text-decoration: 
none; 
} 
a:hover { 
color: blue; 
}
