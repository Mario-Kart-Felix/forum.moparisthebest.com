I got in contact with a few people about this idea and while it seems like its not worth the time and effort, slow progression will be made at making a new platform for the future. All source code will be on github.

To whoever is interested,

I would like to “update” Moparclassic to become the next EasyRSC. It is one of the initial goals of the project to create a platform for the future. The updated version will be strictly Java8+ which means we will not be using scala or another language for scripting.

Server UI
Unlike EasyRSC, we will not be using a UI when starting a server but maybe we can use one to allow people to choose their content? The ScriptManager I wrote is a UI which allows for scripts to be activated/deactivated.

Server Core
The server core will be Elysium : https://github.com/Lothy/elysium-single-threaded

Server Scripting
The server content will be completely scripted and each script will be its own Jar File. Scripts will be downloaded by future hosters either from the MoparScape website or our own website, maybe even github? Each piece of content will have its own file, for example Flax, Bowstring etcetcetcetc. It might seem excessive but is appropriate as we have seen plenty of different kinds of servers.

Server Client
TBD

Possible additions that need to be discussed:
RSCDEF area unpacker/packer. If an area was not being used, the data for that area would not be loaded until it was activated and then removed out of the memory after a set amount of time without players. P2P places that lack player traffic would benefit from this.

Anti-Botting technology? It would come down to server owners using their own clients. I think we should just focus on offering a core technology.

Database. Should we allow users to choose their DB implementation or should we have a set standard?

Website/Registration system, we should probably port a simple secure platform that server owners can build on.

Rodgerwilco August 4, 2016, 3:13pm #16

What people fail to realize is that it already has happened. MoparClassic is an aspiration for active development. While the community seems ‘dead’ and ‘inactive’, WE still have servers actively running (RSCE, RSCR, Karamja, ClassicKBD, insert server here) and that is all based on what the community (over the last almost 10 years) has done. Individualism does not exist and it is all a common goal.

Viewing the communities history thread and pre moparscape forums you can draw a tree of what people have done and how we have slowly progressed and how it turned into the RSCDv25 era with Reines producing some top notch shit back in the day. All servers hosted today are influenced from RSCD. The term “MoparClassic” doesn’t mean an individual server for everybody to play but the community that you seem to have an active forum account on.

Nobody can make a claim to runescape classic (even Andrew himself) because he used a wheel someone else made. (Binary, Machine Code, OS, file formats, JVM etc etc) If you wrote everyone on a list who helped RSC exist then you would probably have over a million people. One of the last conversations I had with Kryptix was about the release of RSCE. Hell, he even posted the thought about it on these forums because he himself recognized the common goal and love for the game we all share. It’s not about what we can produce today, but where the game will be in 2020 or 2025.

Last note, some of the small 2 week servers over the years were the best servers ever and that was because we allowed for rsc to be accessible to someone who didnt know two cents about the jvm. KCCO
