sdouble July 30, 2016, 10:24pm #8

Anyone can learn how to code. The trick of programming is knowing how to get from point A to point B. Having a problem and then developing a solution that solves that problem is what matters. You said you’ve managed to take the code from them and edit it to be added to another program. Digging around in a bag of tricks for the answer will not help you in the long run, yet planning something out and then implementing it will. The hardest part of programming isn’t writing the code, it’s figuring out what you need to do to make that code effective. With someone giving you the answer this time, you’ll likely never have this EXACT problem again. Because someone told you how to do it. But what if you have a problem you’ve never experienced before? You dig around and there is no answer. Here’s an example: You want to write a chat client/server that allows many different clients. You ask people how to go about doing this, they tell you. You take what they said then write the code. Now you want to change the chat client/server into a card game that tracks everything through a database. What now? There is a lot to do. But thankfully, you know how to assign IDs to clients. But there is all of this other logic you have to figure out. You have to handle the database connection, the server for the clients to connect to, tracking and dealing the cards, chat with a possibility of private messages, scoring, rules of the games, etc. Again, writing the code for each is easy. You connect to the database and do this. You check the cards that are dealt and do that. But how do you put it all together? When do you do each? How do you manage multiple games being played?

Asking someone to tell you how to do it this time won’t help you in the future when you have something else to figure out. But figuring out how to piece together your projects from the beginning will make it easier than having a complicated project and not knowing how to plan it out and piece it together because you’ve never had to do that yet. Simple problems = simple solutions. But if you’ve never had to figure out your own solutions for the simple problems, when you encounter a difficult problem that requires a difficult solution, you’ll be up the creek because you haven’t developed the experience with solving these problems.

Anyway, an answer to your question: “Assigning an ID” should be no problem. Think of what all of the clients have in common: the server. So naturally, the server will be assigning IDs. It will know which IDs are already used. When the client connects, make it ask for an ID. The server says “Your ID is Boat1” and the client says “Thanks, piss off” and now you have your ID. Client2 connects and says “What the hell is my ID?” And the server says “Your ID is Boat2” Then the client says “Thanks, piss off” Now you have 2 clients and a server. Clients Boat1 and Boat2.
