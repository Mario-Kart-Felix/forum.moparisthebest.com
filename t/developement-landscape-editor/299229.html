Hello, I’m working on a landscape editor, and I’d like your opinion of it, as well as any suggestions you might have.

This is the current GUI:

Now, I know, that doesn’t look this much; And it isn’t, the GUI contains but one component, a JPanel, a thread prints the current state of the currently selected sector at 45 FPS.

What I’ve got so far:
The landscape file is loaded from the .jar files directory on startup.
All sectors are loaded into an array on startup.
Switch sectors with the arrow keys.
Click/drag mouse to paint tiles.

What I’m planning:
Ctrl+Z to undo changes to a sector.
Save on close, overwriting the old landscape file.(I might create a file dialog box to save the file instead.)
Parsing XML to load object positions.
Flood painting tiles.
Drawing shapes by dragging the mouse.(Rectangles/Squares/Circles)
Drawing walls.
Adding NPCs/Objects/Item spawns with a tool.
Saving.(Didn’t feel like doing it tonight, I’ll do it when I get up.)

I can’t think of anything else right now, I’m open to suggestions.
I’ll be releasing the program once I’m done with it, along with the source code.
