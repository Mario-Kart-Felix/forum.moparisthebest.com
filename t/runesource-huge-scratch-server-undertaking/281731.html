Graham July 31, 2016, 8:11am #16
_ikram:

Reading this again, how are you planning to up the entire system and managing the drivers during runtime.

It’s possible to dynamically load classes at runtime using reflection which is what we already have the beginnings of.

For loading plugins:

Class<?> clazz = Class.forName(name);
plugin = (Plugin) clazz.newInstance();

For loading platforms:

Class<?> clazz = Class.forName(name);
Constructor<?> constructor = clazz.getConstructor(InetSocketAddress.class);
platform = (Platform) constructor.newInstance(addr);

Obviously we’ve still got quite a while until we get the whole, nice pluggable system running but we’ve already got a good start for everything.
