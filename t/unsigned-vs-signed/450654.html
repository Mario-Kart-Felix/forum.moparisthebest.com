musicmasta August 3, 2016, 5:19pm #4

You are not required to sign a jar file that is started from a file that actually exists on your computer. When launching a jar file as a java web-start application, you are required to have the jar file signed, if the file is not signed and run as a web start application, it will restrict it from accessing your computer or network. Signing your jar file is kind of like signing your name on to a sheet of paper, it’s certifying it, giving it the proper authority to run.

To summarize:
Running it as a java web-start application requires the jar to be signed, or it will be restricted from running.
Loading it on your computer will not require the jar to be signed.
