The_Web_Guy August 3, 2016, 2:38am #4

Best way is to use the jarred client file that you’d use for the webclient, it’s easily universal for any Java version on any Computer Operating System. However, if you don’t want to do that or you currently don’t have that set up yet, here are a few files that will work for x86 (64-bit) computers and standard (32-bit) computers. (batch files ; for running)

32-bit (Recommended save as ‘Run.bat’)

@echo off title YOURCLIENT "C:\Program Files\Java\jre6\bin\java.exe" -Xmx500m Gui 10 0 highmem members 32 pause

if that one doesn’t work, try this.

@echo on "C:\Program Files\Java\jre6\bin\java.exe" -Xmx500m -cp .;Theme.jar Gui pause

You’ll have to look at your already existing code to see what it’s like and find out which one is best with you, i’m sure you’ll be able to compare some of the patterns and find out which one is best, you may have to change or alter the code a bit.

64-bit / x86 (Recommended save as "Run (x86).bat’)

@Echo off ECHO You may now close this window Cd bin "C:\Program Files (x86)\Java\jre6\bin\Javaw.exe" -Xmx254m -classpath .;Theme.jar Gui 0 0 lowmem members 32 exit

The above ‘Run (x86).bat’ is for clients that have a /bin folder in them.

By looking at your already existing run.bat files, and the three that I have provided you, you should be able to get a universal one working fairly soon. If you can’t after a few minutes, i’ll check back and post the code for your existing run.bat and i’ll create some run.bat files for you.

color 0c
cd locandclass

:Start
echo IGNORE ALL ‘CANNOT FUND PATH SPECIFIED’ ERRORS.
