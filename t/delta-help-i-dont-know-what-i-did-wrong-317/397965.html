WhyPlayLegit August 1, 2016, 12:13pm #3

From what you’ve shown, it looks like case 21 is orphaned, which means it is not in a switch statement.
If you do not understand what I am saying, let me give you an example.
Let’s say we have an integer named “var1”. We can make a switch statement over “var1” that will look like this:

[code]int var1 = 1;

switch(var1){}[/code]

Now, we can make cases in this switch statement, so, for an example. If we want to output text when var1 = 2, we will do this.

[code]int var1 = 1;

switch(var1){
case 2:
System.out.println(“var1 currently equals 2.”);
break;
}[/code]

So, what I’m basically trying to say is, CTRL+F for case 21, and make sure it’s inside a switch statement.

Note: If any other programmers see that my reply was incorrect, please let me know, I’ve been currently coding with C++ and I cannot remember if the logic is different from Java with switch statements. (Even though I’m quite sure they are pretty much identical)
