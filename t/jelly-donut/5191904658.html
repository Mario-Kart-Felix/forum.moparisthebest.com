Doesn’t matter, thread got bumped.

I’m going to try and format the threads in such a way where updates are listed in a more appropriate manner. Its always hard to keep track of what changes, and often times there aren’t anything really significant to note that changed.

Recent Updates

  • It’s now possible to configure the rate at which the ondemand service writes to a client, in bytes per cycle. A cycle delay is determined by the server engine pulse rate which currently stands at 50 milliseconds. A total of 97kb will be wrote to the client per second in perfect conditions. Note ‘perfect conditions’.

  • Fixed some network logic bugs where if the amount of packets parsed were throttled it would not parse anymore without further input from the client.

  • Cleaned up some fields to be more ‘readable’.

  • Started to remove logging because its an eyesore and not really needed as of this moment.

  • For anything that needed state checking on cycling, exported it to a separate method for cleanliness.

  • Began thoughts and design on how logging in will work. Got lazy, played LoL with Newty and never really finished this.

Things I (may or may not) want to Finish

  • I don’t really want to write a cache reading package, but I have to.

  • Logging in and game service design.

  • Maven building crap. How does it work? That’s the goal to find out.

Check out the github, give me some feedback, bump the damn thread, send me money, anything. Please.
