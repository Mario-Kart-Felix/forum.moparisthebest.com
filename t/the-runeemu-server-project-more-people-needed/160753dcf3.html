University actually. Getting RuneEmu to be a match for, say, the Runescape official server, would indeed take more time than I have or I’m willing to spend. The project is been going very slow lately, because me and all other programmers have been too busy to be really active. We think indeed that there is no point in releasing an incomplete project in binary form, especially because as of now it is interesting not because of its runescape functionality, but because of its design and code. The compiler / IDE we are using is Visual Studio .NET 2008, though we are writing using standard C++ and cross-platform libraries (especially boost) so it is possible to port it to any standard-compliant compiler easily.
Seeing how RuneEmu is Open Source, it wouldn’t really be a problem that .NET DLLs were compatible with it. However, we’re using native C++, which, unlike C++/CLI, does not make use of the CLR, so we don’t have to worry about that.
