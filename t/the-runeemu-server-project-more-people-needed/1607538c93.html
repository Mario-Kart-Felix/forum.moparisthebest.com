zstars July 30, 2016, 6:50pm #256

People, take flame wars to PMs.

To clarify, some facts (If someone is curious about one, I can prove / explain it).

-Java is designed to be simpler and cleaner than C++ (As a result, it is easier to learn, and it is harder for a newbie to mess it up terribly, but less powerful, and, if you know both, not necessarily easier to use).

-An optimal Java program is by design less efficient than an optimal C++ program. (Though the difference is / can be small and in many cases it is not significant)

-C++ doesn’t have implicit memory leaks. You can leak memory with C++ easily, but it can be avoided (and C++ can support GC via libraries, if you weren’t confident on your skills to do so). Memory can also be leaked with Java (By not getting rid of references that you should have removed).

-C++ is an ISO / ANSI standard. Proper C++ source is compilable on any standard-compliant compiler, on any machine. Java is a propietary language, and a trademark (Hence the “tm” that it is often accompanied with), owned by Sun Microsystems. A program compiled with C++ is (usually) dependant on its platform. A Java program is (usually) dependant on the Sun Microsystem’s Java Virtual Machine.

-RuneScript seems to be the name that Jagex has given to the Scripting language they are using at their server side. Whether their server core is in Java or C++, it is certainly not in RuneScript. RuneScript is to it as Python to RuneEmu, a scripting language to customize the server / add content without having to deal with the general-purpose core language.
