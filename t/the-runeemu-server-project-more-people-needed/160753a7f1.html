zstars July 30, 2016, 7:04pm #277

Yes, more or less, we started with blank files. RuneEmu is at the moment divided in five sub-projects. One of them is the Shared library, which is a collection of general-purpose utility classes. We started with it, creating some important classes (such as the RuneBuffer class) and bringing some from other projects of mine (such as SafeQueue or BThread classes). Now the Shared Library project is mostly complete, so we are now focusing on the RuneEmuServer sub-project, which could be considered the main one.

Of course, we don’t do everything from scratch, we are using a few libraries (Boost, Asio, Python headers).
