About server features, you might want to know, that the purpose of RuneEmu is not to get as many features as possible in the shortest time we can. After all, there are loads of GenericScapes servers which have already many more features than RuneEmu and your server together.

The aims of the RuneEmu project are, first, to make a proper MMORPG server, with a good, as professional as possible, modular and reusable design. And second, to gain knowledge and experience in the process.

I can say that we, the RuneEmu team, are following these aims nicely.

Just as a curiosity, at the moment the RuneEmu project counts with exactly 76 classes, and 9648 lines of code. All split up in five different projects. Two of them (A Shared library and an Asynchronous server base) are almost entirely reusable.

It might finally be noteworthy that besides the common RS features, such as walking or zoning, which are entirely runescape-binded, and thus not particularly interesting, RuneEmu has been supporting Python for a while, allowing things such as handling packets to be done through Python scripts.

On another note, you might want to know that your server (if what you say is true) has already two design issues:

First, loading all information from a database at startup is, for obvious reasons, extremelly inefficient for large ammounts of data.

Second, non-asynchronous networking is, by itself, inefficient for a medium to large ammount of connections. If, additionally, you happen to use one thread per client, it immediately becomes a slow and non-scalable server.

If you want to make a decent Java server, you should probably use a good base, such as Varek’s rs2d.
