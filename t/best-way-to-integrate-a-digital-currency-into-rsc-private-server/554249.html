Games

sin August 4, 2016, 5:12pm #1

This all stemmed from an idea i had about a year ago to create a digital currency like bitcoin but specifically for use as an in-game currency and called it GoldPieces/GP. One idea led to another and here I am, setting up a classic server to hopefully successfully integrate a digital currency (https://bittrex.com/Market/Index?MarketName=BTC-GP) into and run well, though I still have much to do and learn.

I used this server: https://github.com/Lothy/MoparClassic/

So here’s my idea of how to go about this thus far

VPS 1: GP Server Software
VPS 2: RSC Server
Webserver For Website

In the bank of the game there would be a generate address button, This would query VPS1 for a deposit address and send a callback when funds were received and confirmed, then to be credited to the account.

The only thing i’m worried about is drops/selling in shops etc. My solutions are to make 1GP equal to 1000coins or to remove coin drops/ ways for people to get coins other than player to player trading etc.

So what I would need to know is how to make sure people cannot get coins from the game that aren’t from other people basically for now to keep the economy somewhat workable.

silabsoft August 4, 2016, 5:13pm #4

The RPC protocol for bitcoin is rather shit to use if you are going to have multiple people working on the server. Someone can easily send undesired commands from the server without careful security.

I have also considered extensively on how to use BTC in a private server if you were going to use it as a currency I would simply have the website doing X amount of coins for Y amount of BTC and credit it accordingly instead of having the bank ingame generate a deposit address. Next for withdrawal address again I would use a web interface rather than creating some shitty system in game due to the lack of copy and paste for the game client.

However pay to win is highly undesired as Lothy said. I considered for a brief period of time having users pay to create accounts and then that GP is allocated to a server pooled gp amount which is then spread as rewards for killing high level creatures and performing certain tasks. However if you do not sustain the flow of new players paying to create a character or come up with another solution to keep the btc incoming in the pool will dry up quickly if you are not careful.
