[size=20pt]Easier, cleaner & better stair system for delta.[/size]

[b]The way delta handled stairs bothered me so much that I today, decided to write a new one.
You simply have to store your stair-Ids in the right array.

  • Supports messages (go up or down)
  • no more use of process and unneeded variables
  • Clean, not handled in 20 spots through your files.[/b][/center]

[size=18pt]Step 3:[/size]
Now create a new java file and call it: StairHandler.java
Paste this code into the file:

/**
 * @Author: RandQm
 * @Date: 23 April 2012
 * @Info: Handling stairs. (without process and ridiculous amounts of code like it originally was..delta)
 */

public class StairHandler {
	
	client c;
	
	public StairHandler (client c) {
		this.c = c;
	}
	  
	/*
	 * All stair IDs.
	 * stair1up: stairs where the player should raise 1 height level
	 * stair1down: stairs where the player should decrease a height level
	 * stairChoice: stairs where the player should get an option sent if he wants to go up or down.
	 */
	private int[] stair1up = {1747};
	private int[] stair1down = {1746};
	private int[] stairChoice = {1748};
	
	/*
	 * The method to handle the action when clicking a stair from the arrays above.
	 */
	public void handleStairs (int stairId, int stairX, int stairY) {
		if (c.GoodDistance(stairX, stairY, c.absX, c.absY, 1)) { //If the player is maximum 1 tile away of the stairs the actions will be executed.
			for (int up1 : stair1up) {
				if (stairId == up1) { //if the stairId is coming from our "stair1up" array, then...
					c.setAnimation(828); //stair animation gets executed
					c.toX = c.absX; //the player gets set at the same coordinates
					c.toY = c.absY; 
					c.heightLevel ++; //but increasing a heightlevel
				}
			}
			for (int down1 : stair1down) { //same here but for going down
				if (stairId == down1) {
					c.setAnimation(828);
					c.toX = c.absX;
					c.toY = c.absY;
					c.heightLevel --;
				}
			}
			for (int choice : stairChoice) { //if the stairId is in our options array,
				if (stairId == choice) {
					c.NpcDialogue = 124; //the option gets send
					c.NpcDialogueSend = false;
				}
			}
		
			/*
			 * For stairs with special requirements, e.g.: 
			 * - a specific variable amount is needed (donator ect...)
			 * - Specific stair-coordinates are needed.
			 */
			switch (stairId) {
			case 3028: //stair id
				c.toX = 3088;
				c.toY = 3120;
				c.heightLevel ++;
				c.setAnimation(828);
				break;
			default:
				break;
			}
		}
	}
	
}

Explanation: In the code itself.
