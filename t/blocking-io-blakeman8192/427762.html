Alright. Let me explain blocking IO. When you write to a socket like this.

OutputStream out = socket.getOutputStream();
out.write(10);

The code does not coninue till the data is received by the client or server. That’s why it’s called blocking IO, it blocks till the data is received. Imagine that example is a server, and the following code in the client.

InputStream in = socket.getInputStream();
int data = in.read();

Now your code will continue becuase the data is received and it doesn’t block anymore.

But with NIO, the data will just be send without blocking.

ByteBuffer buffer = ByteBuffer.allocate(8);
buffer.put(10);
channel.write(buffer);

Here, the buffer, which is holding the data, is send to the client. Even though the client doesn’t have the code to read it.

@Native^: That’s not entirely correct. When you call write(), it blocks until all bytes are written to the stream, not necessarily until it’s received. In the case of read(), it blocks until at least 1 byte is read.
