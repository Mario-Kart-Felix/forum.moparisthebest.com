[center]Alot of people have had problems with the auto-cachedownloader in deathlypvpz, even i did at one point but i figured out a solution and so far its worked everytime…

-Uploading your cache-
We need to upload your cache before we can add an auto-cachedownloader, so first thing you need to do is open My Computer > C: drive. Open your Deathlypvpz v4 cache, “or whatever your cache is called” Right click & select all of the files located inside the cache, click add to archive. you should see this menu.
Change the name of the archive to your desired cache name.
click ok.

Now go to www.fileden.com to upload your cache, i recommend using fileden because its a lot easier to access your files once uploaded, you may also use uppit, or dropbox. i do not recommend using dropbox, it is horrible sometimes.

Once your cache is uploaded, you should see 2 links, one has [img] around it, the other has [url], copy the top url, so copy this http://www.fileden.com/files/2013/7/4/3412173/YourCache.zip

-Setting up your AutoCacheDownloader-
Now open AutoCacheDownloader, located inside, Client/JavaFiles.
You should see this near the top,

change everything inside the parenthesis to your cache location, if you used fileden you should have something similar to this http://www.fileden.com/files/2013/7/4/3412173/YourCache.zip
Replace the old link with the new link to your cache, save your AutoCacheDownloader file.

-Setting up Signlink-
Now open signlink.java located inside your clientfolder, java files, sign
ctrl + f this code, public static String findcachedir() { String as[] = {"C:/"}; if(storeid < 32 || storeid > 34) storeid = 32; String s =
you should see "Your old cache name, replace it with the name of your new uploaded cache, so for example, If your .zip file is named "GayScape.zip, you need to enter String s = “GayScape” with the parenthesis, it is case sensitive so make sure you type it just as it appears.

-Compiling-
Now open Client & click compiler, you should not receive any errors, once you have done that, click run, your client should say Downloading Cache 0%, if it does then you have successfully fixed your cachedownloader.[/center]
