I would have posted it earlier but I wasn’t at home so didn’t have it handy :p. This is what I wrote to be used on the Emps-Scape highscores:

SET @rownum := 0;
SELECT * FROM ( SELECT @rownum := @rownum + 1 AS rank, username FROM highscores WHERE skill = ? ORDER BY level DESC, exp DESC LIMIT 1000 ) AS ranked_highscores WHERE username = ?;

It only reports ranks if you are in the top 1000, you can change that by changing the LIMIT clause.

The trouble with your code is that it’ll cause a lot of traffic between PHP and MySQL since the entire table has to be sent between them. Also, PHP is usually configured to run in small amounts of memory (perhaps 8 - 32 MB in general) and any more than that will cause errors. With a small private server this isn’t an issue. However, the high scores table is around 25 MB on Emps so we’d be getting close to that barrier (and 25 MB going between PHP and MySQL each time a user uses the highscores would probably be a big problem).

EDIT: in fact in this case you probably won’t need a nested query. But you do need it to find someone’s rank given their username
