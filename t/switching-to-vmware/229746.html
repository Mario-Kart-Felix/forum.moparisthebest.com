Switching to VMWare

Technology
Unhandled July 31, 2016, 1:25am #1

I’m really sick of the shit windows has been giving me. I haven’t been able to get more than 6 months without a total reformat in the last 3 years. Now that visual studio has stopped working, (will require a fresh OS install to fix), I’m really really sick of the shit.

My plan is to move to Linux (probably Debian), and run multiple VMWares with XP installed.

One VM for development, one for other windows only shit, etc. I will save a copy of the VMWare image after setting up windows and all my programs, so as soon as something goes bad within the VM, I can swap in the fresh image and not have to waste a weekend reinstalling.

With a REAL OS running underneath, I will never be stuck when something crashes, etc.

My question is about the performance hit. I currently have 2gb of ram, and an AMD Athlon 64 3500+, which should be enough to run one VM with 1.5gb of virtual ram. Of course multiple cores would be ideal, but I don’t feel like replacing my CPU at the moment. Games are basically out of the question, but there isn’t there still some 3D acceleration within a VM? (so I can do OpenGL development etc)

What do you think? Any obvious flaws?

inb4 obvious flaw: running windows at all.
