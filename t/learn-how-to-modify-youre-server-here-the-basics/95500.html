Learn How To Modify You’re Server Here (The Basics)
Welcome! Here, I’m going to try and explain to you, how to modify/add simple things to you’re source. Once you’ve read all of this, you should be able to create a half decent server. Now, I’m not going to go into the meanings of things, such as booleans, voids and why they are here, but I’ll show you how to add them/modify them and other things too!

Requirments
A source file and some patience.

Starting Off!
[i]Ok, to start off, simply double-click you’re source file, be it Testscape, Pimpscape, whatever. You should see a bunch of files, some .txt and some .cfg, whereas some are just documents. There are .class files aswell but we won’t get into those.

Now, the .txt files are named .Java files, these are easy to recognise, just a small notepad icon.

The .CFG files are named .CFG files, these are very easy to recognise, they have 4 small squares for an icon, blue, yellow, red and green.

The documents are usually used for things that are directed through the server, into a file, such as the Character file, once a character logs in, their file automatically saved to the Character file.

The .class files are files that you cannot access, these have been created after the .java files have been compiled. Just ignore those.

Now, when you first download a server you wan’t to rename it, this is one of the most simple jobs of all. You will understand how to do this by reading the next chapter[/i] : Modifying .Java Files

Modifying .Java Files
[i]In this chapter, I’ll show you how to add/modify basic things in .Java files, this can be tricky, so be carefull.

Now, the source I like to use, is the Phonescape source, (V1) it’s what I’ve been working with since day 1, it’s important that you use a source that you’re comfertable with. Now, this may be different from you’re source but nevertheless, you should pay attention. I’ll explain to my best ability what each .Java files are for and how to modify/add things to them.

First off, antilag.Java.

If you go into you’re antilag.Java, you should see something like this.[/i]

[code]// This resets server every 30 mins to stop lag - xerozcheez
import java.io.*;

public class antilag
{

public int ResetTimer = 0;

public void resetserver()
{
misc.println(“RESETING SERVER!!!”);
misc.println(“Saving all games…”);
PlayerHandler.kickAllPlayers = true;
misc.println(“GAME SUCCESSFULLY SAVED FOR ALL PLAYERS”);
ResetTimer = 0;
closeListener();
runserver();
}
public void process()
{
ResetTimer += 1;
if(ResetTimer >= 999999)
{
resetserver();
}
}

public void runserver()
{
try
{
String run = “runserver.bat”;
String xstr = “./”+run;
Runtime.getRuntime().exec(xstr);
}
catch (IOException ioe)
{
ioe.printStackTrace();
}
}

public void closeListener()
{
try {
server.shutdownClientHandler = true;
if(server.clientListener != null) server.clientListener.close();
server.clientListener = null;
}
catch(java.lang.Exception __ex)
{
__ex.printStackTrace();
}
}
}[/code]

Now, this is basically for reseting you’re server, to prevent lag. (The longer you’re server has been running, the more lag it will have) To make use of it, simply change the timer in this part of the void.

if(ResetTimer >= 999999)

The smaller the number, the more often you’re server will reset/restart. This gets rid of most of the lag for the players and in some cases, you. I’m not too sure, but I think the time is based on milliseconds. So every 1000 (1 Thousand) is one second. I’m not sure about this though. You can also change the message it sends before and after the reset/restart.

misc.println("RESETING SERVER!!!"); misc.println("Saving all games..."); PlayerHandler.kickAllPlayers = true; misc.println("GAME SUCCESSFULLY SAVED FOR ALL PLAYERS");

You could change it to something like this.

misc.println("The server is : Restarting, please wait."); misc.println("Saving all games..."); PlayerHandler.kickAllPlayers = true; misc.println("You're account has been saved succesfully, thank-you for you're patience.");

A small window should pop up, you can now use this to search for things you wan’t to modify/delete.

Like I said above, you can usually modify the server name by searching for it, so why don’t you? Just type it in. These are the things you will usually find in sources.[/i]

Now that we’re done with client.Java, we should probably move on to graphicshandler.Java.

graphicshandler.Java is basically used for things such as spells, arrow projectiles, the distance they are from you’re enemy, from yourself, etc. I don’t know much about this file, so I’m sorry about that. But atleast you know the basics. :)[/i]

[i]Now, lets move on to some important .Java files, the Item.Java’s. There may be more than one of these, (Item1, Item2, Item3, etc) don’t worry, that’s normal. It’s just so everything isn’t clogged up into the one file, making it very full and messy.

These are used for things such as, NPC drops, randomItems (These are used for Mini-games and such), etc. I’ll show you the basic layout for these things and how to use them.

First off, we’ll do an NPC drop. Lets say, we wanted a Goblin to drop a Santa Hat, we’d use this layout.[/i]

public static int goblin[] = {1050};

    public static int goblin()
    {
    	return goblin[(int)(Math.random()*goblin.length)];
    }[/code]

[i]Now, the 'public static int' is the type of statement, there are others, such as 'public boolean', 'public void', etc.  Don't change that, we'll only be focusing on the name and the item ID.  The item ID part is here.[/i]

[code]public static int goblin[] = {1050};[/code]

[i]The item ID is the 1050 (Santa Hat), if you were to leave just 1 item in there, that NPC would drop the item 100%.  To add another item, simply do this, I'll use a noted Santa Hat for example.[/i]

[code]public static int goblin[] = {1050,1051};[/code]

[i]So basically, all you're doing is adding a comma, then another ID.  This would mean that you have 50% now of getting the Santa Hat, one half.  Vice verca with the noted one.  Now, the 'goblin' part, is the name of the NPC which drops the item, this has to be the same one used in npchandler.Java, so that the server recognises which NPC actually drops these items.  You don't neccaseraly have to call a Goblin a 'Goblin' just because that's it's real name, you could call it 'Yayatepownzor' and it would work fine.  Just remember, you need to use that same name in npchandler.Java and in the Item.java.  I'll show you where the name needs to go.[/i]

[code]
public static int HERE[] = {1050};

    public static int HERE()
    {
    	return HERE[(int)(Math.random()*HERE.length)];
    }[/code]

[i]Now, with a follow up from the NPC drops, lets go to npchandler.Java.  First off, I'll show you how to finish the drop.  You'll need to add the drop using this format.[/i]

[code]if(npcs[NPCID].npcType == NPC ID HERE) 
{
ItemHandler.addItem(Item2.NPC NAME HERE(), npcs[NPCID].absX, npcs[NPCID].absY, 1, GetNpcKiller(NPCID), false);
}
[/code]

[i]See the parts where it says, 'NPC NAME HERE'?  That's where you need to add the NPC's name that you used in the Item.Java file.  In my case, it was 'Goblin', so I'd put Goblin and for where it says 'NPC ID HERE', I'd put the NPC's ID, again, in my case it was a Goblin, so I'd put the Goblin's ID like so.[/i]

[code]if(npcs[NPCID].npcType == 101) 
{
ItemHandler.addItem(Item2.goblin(), npcs[NPCID].absX, npcs[NPCID].absY, 1, GetNpcKiller(NPCID), false);
}

[i]We’re done! The drop will now work. If it doesn’t, then you have to fix the ‘Item2.’ part, if you added the NPC drop in Item3.Java for example, you’d put Item3 in there.

Now, I’m gonna explain what npchandler.Java is for. Basically, it handles all the important things related to NPC’s. Drops, HP, textUpdate’s and MUCH, MUCH more. To edit what an NPC says, you would simply login to the server and see what it says, then you would use the search feature (as explained before) and search for what you saw in-game. E.G - Say an NPC said, “Ownageserver is teh pawnzer!!!”, you would search for it in npchandler.Java. You would see something like this.[/i]

if (npcs[i].npcType == 535) { // description for easy searching goes here if (misc.random2(30) == 1) { npcs[i].updateRequired = true; npcs[i].textUpdateRequired = true; npcs[i].textUpdate = "Ownageserver is teh pawnzer!!!"; } }

To change what the NPC says, you would simply change what’s in the inverted comma’s, remember not to add the colours again, it won’t work.
