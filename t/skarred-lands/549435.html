[anchor=about] [size=18pt]About Skarred[/size][/anchor]
Skarred-Lands is a server that I consider as ‘mine’. However, I did not create the base for the source. It’s base is a PI server. The client is a refactored/renamed 317 client that is loading higher content, such as: 602 items, ~562 gameframe, and 474 or 508 maps… I have personally put in a lot of work to this base, regarding writing skills and new content. I have not finished re-writing all skills, as I keep going back and re-writing a skill I wrote earlier. I’ve decided to go with a concept of loading skill data (Ex: mining rocks, or exp/anims/log ids for firemaking) from JSON because of how fast and flexible JSON is with GSON. And finally, I plan to make Skarred-Lands into my version of RuneScape; Not yours.

I’ve just finished moving my prayer data into my json system. Again, the reason I’m using json to load the core skilling data is because I want the server to be dynamic. I want to be able to “update” the server without it ever going offline. While it wouldn’t be a full update, it would be able to update things like:
NPCs - Spawns, Definitions, Drops.
Items - Definitions and prices.
Shops - Everything related to a shop can be updated dynamically. Except a new form of currency.
Staff list - I have a built in staff manager (cannot remember if I put it in the OP or not) that essentially prevents anyone not on the list to gain or use staff powers.
Player messages - Login messages and the messages on the [INFO] ticker.
Skills - Right now it’s only Prayer, Thieving, Mining, and Herblore data. But eventually it will be nearly, if not, all skills. Example for Herblore I can add in new potion combinations or remove them, or even change an existing one up.
