SilentCJ August 4, 2016, 8:57am #4

Even if your syntax was right it wouldn’t work I’m afraid
You don’t really need a banned column as such, unless you explicitly wish to ban individual IP’s. Simply specify the IP’s that you want allowed in the allowed column and you’re all set.

Not tested or anything but should be fine. I know you have no concern for bad code but I advise using mysqli functions over mysql, they will be removed completely within a few years.

    $con = mysqli_connect('localhost', 'root', 'password', 'database');
    $remote_ip = $_SERVER['REMOTE_ADDR'];
    $query = "SELECT * FROM ips WHERE allowed = '$remote_ip'";
    $result = mysqli_query($con, $query);
    if(mysqli_num_rows($result) > 0) {
    	echo 'Allowed';
    }
    else {
    	header('Location: www.example.com');
    	/*
			Could also set a session variable to store a message to display
			to the user on the new page
    	*/
    }
    mysqli_close($con);

[code=php] $query = "SELECT * FROM ips WHERE allowed = '$remote_ip'"; $result = mysqli_query($con, $query); if(mysqli_num_rows($result) > 0) {[/code]
If I remember correctly, it would be faster to use: [code=sql]SELECT COUNT(*) FROM ips WHERE allowed = "127.0.0.1"[/code] As a query, then simply use mysqli_result in PHP to check if it's greater than one, rather than using PHP to check the row count. If you're dealing with a query that could potentially return a lot of large results, it's really wasteful for MySQL to send them to PHP just to have PHP count them. Not only that but the SQL engine can make further optimizations and just serve its own indexed size that it computes internally.
