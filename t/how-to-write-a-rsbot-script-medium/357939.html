BEFORE FOLLOWING THIS TUTORIAL - PLEASE FOLLOW THIS ONE: Starting RSBot Scripting

This tutorial will be straight forward, (somewhat) easy, and help you on your way to making your own scripts. If you did not read the tutorial above, DO SO NOW.

Please have some basic Java knowledge before attempting this tutorial.

Here are some of the things that this tutorial will cover:

[ul][li]Preparing our basic script to be put to use[/li]
[li]Clicking on objects[/li]
[li]Adding more to your paint[/li]
[li]Getting user input before they start your script[/li][/ul]

So lets get started! Open your test script java file now.

[size=15pt]1. Preparing our script[/size]

The first thing we need to do to make sure our script will work, is make it compatible with RSBots’ login script. To do so is just one easy conditional inside our loop() method. Find your loop method, and place this code inside it:

if (!isLoggedIn()){ wait(random(3000, 5000)); }What this does is make sure we don’t keep the loop going while the login script is logging the character into RuneScape. REMEMBER: This conditional should ALWAYS be at the top of your loop method

After we do this, we need to change some of our ScriptManifests so that we know what this script will do. Scroll up to the top of your script, and see if you can figure out what to do. First change the Author, the Category, the Name, and the Version to what you want. We will be doing more with the description later.

[size=15pt]2. Clicking on objects[/size]

One of the coolest built in features of the RSBot client is that it can click on things for you, one of which is objects. And it’s easy, too! The method for clicking on an object is:

atObject(RSObject object, String option); //RSObject is a object entity interface, more on that later

Before we can use it though, we need to define the RSObject. We do that like so:

RSObject object = getNearestObjectByID(objectId);

However, if the client cant find the object, the RSObject will be a null, and we will have some Null Exceptions in there. So here’s the whole process:

final RSObject bankChest = getNearestObjectByID(42192); //42192 is the object ID for the bank chest at Soul Wars if(bankChest != null) { atObject(bankChest, "Use"); //This will click the 'Use' option of the bank chest }

That’s it! You just change 42192 to the Object Id of your choice and “Use” to whatever the option is. You can put this anywhere, but let’s test it in the loop method. (Remember that everything you put in the loop method goes below the ‘isLoggedIn’ conditional!) If we just copy and paste that code in there, you will get some pretty fun lag. Let’s add a few conditionals so that it will only click it when we want it to.

if(inventoryContainsOneOf(995)) { //New method! inventoryContainsOneOf(itemId) is used a lot in scripts. Now the code will only happen if you have at least 1gp in your inventory. final RSObject bankChest = getNearestObjectByID(42192); //42192 is the object ID for the bank chest at Soul Wars if(bankChest != null) { atObject(bankChest, "Use"); //This will click the 'Use' option of the bank chest wait(5000); //New method! Pauses the script for a certain amount of milliseconds. In this case, it stops it from spam clicking the chest } }Put that code in, head over to the soul wars bank booth, run the script with nothing in your inventory and see what happens. Restart the script with at least one coin, and wallah!

[size=15pt]3. Adding more to your paint[/size]

Adding more information to your paint is extremely easy. Let me post the code we made last time:

g.setColor(new Color(0, 0, 0, 50) ); g.fillRoundRect(3, 180, 155, 130, 5, 5); g.setColor( Color.WHITE ); g.drawString("Run time: " + sH + ":" + sM + ":" + sS, 12, 200);
Let me explain which parts you change. The first method sets the color of the background of your GUI. In this case, it’s black (0, 0, 0) with a low opacity (50). This makes it a mostly see-through background. If you wanted it to be more red, try (255, 0, 0, 50)
The next part (fill round rect) draws and fills a rectangle, using the color you set earlier, with the coordinates you give as arguments, and size.
Then we set the text color, in this case, it’s white.
Now we write text onto the GUI, like this: g.drawString(String text, int x, int y);

Too add another line, we could add this below the first ‘drawString’:

g.drawString("Script by Davidi2", 12, 216); //Note that I added 16 to the y value, because that makes it 16 pixels lower

[size=15pt]4. Getting user input before they start your script[/size]

Getting input from the user can be important. You might ask for an item id, or if you’re making an auto-talker, the text to say. If you remember from the first tutorial, I said that you can put HTML into your script description. And that’s what we’re going to do. We’re going to use HTML to create a text box. So back up in your Manifests, let’s make an example autotalker.

@ScriptManifest(authors ="Davidi2", category = "Misc", name = "Auto Talker", version = 1.0, description = "<center><h1>Auto Talker</h1><br /><h3>By: <u>Davidi2</u></h3></center><br />What text do you want to say? <input name='textInput' type='text' size='100' maxlength='100' value='Text Here' />")
Now as you can see, I changed my description to some HTML. Now I suggest you read up on HTML input types if you want the full functionality, but remember this: the name (textInput) will be important later. Make sure it relates to what you want to accomplish.
Ok, now scroll down to your onStart() method. The Map<String, String> is actually the input you put into the description! So we can use that Map to get the user input:

String textInput = args.get("textInput"); log(textInput); Add that below startTime = … , save, compile, and run your script. See what happens!
