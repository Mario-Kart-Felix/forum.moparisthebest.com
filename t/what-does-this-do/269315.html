xpatx July 31, 2016, 6:26am #3

Break means the end of a case. The reason you add a break is because if you don’t it will overflow. So, yes it means the end of a case in a switch statement.

[code=java]
public void example(int i)
switch(i) {

case 1: //Works
System.out.println(“This is a message that case one is being outputted on data.”);
break;

case 2: //Fails
System out.println(“This is case two.”);

case 3: //Works
System.out.println(“This is case three”);
break;

}[/code]

Case 2 and 3 would cause an error because case 2 needs to return something, which would be the execution the case needs. So basically, the break causes it from stop overflowing.
