One thing you should also consider, is that even if you port forward, chances are that you’re required to input your local ip address (10.0.0.20, 192.168.1.175, etc). What a lot of people forget is that, when they set the open port, they specify their current IP address. However, most of the time, their machine is still set to DHCP, so the next time they restart their computer, they may not have the same address. You should specify that your machine get a static address.

There’s two solutions to that (afaik). One, on windows (Unsure about OSX/*Nix) you can go into the network and sharing center and specify a static IP in there, and windows will try to “reserve” that IP. But that isn’t a proper solution, as the computer doesn’t really have a say in the IP address. The most it can do is try. The other solution, and the correct way, would be to reserve the IP address in the router settings. Each router handles this a bit differently, so I can’t say how it would go exactly.
