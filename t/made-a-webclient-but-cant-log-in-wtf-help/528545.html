Vain1 August 4, 2016, 10:15am #6

it has nothing to do with the web page markup, please stop posting it.
I never said anything about the server being on or offline(I’m sure you can debug that yourself.)

a client (in any situation) connects to a server of some sort. the server binds itself to a specified port, and listens for all incoming connections of the proper protocol (tcp/udp/etc)
the client itself will try to connect to a specified IP address, of the desired server. if given the right port, and follows all other guidelines, a connection will be made.

if you’re hosting the server on the same system (computer, vps, etc) as the connection is being made (same where the client is) then you must connect to a local address (localhost, 127.0.0.1, 0.0.0.0, etc)
if you are hosting the server on the same network, but a different system (on a separate computer in your household or network area) then you must connect through an IPv4 address. (you can get this via the command ipconfig in command prompt)
if you’re connecting to the server from an outside network (other players connecting to you) they will need your full IP address or dynamic ip from something like no-ip or dyndns.
