Wintercanon August 1, 2016, 7:17am #2

During login the client and server exchange a protocol of various packets, verifying that it is indeed a Runescape client with the correct revision etc. If you look at this process in both the client and server you will see the different numbers sent back and forth. What you could do is set the client to send a packet with a hidden ID after it confirms the revision is correct, while also having your server expecting it. That way your server will reject any connections that don’t send this.

relmitos August 1, 2016, 7:17am #4

Client-side open the Client class and find the UID variable in the run() method, give that a nice value that only your client uses, let’s say 01312011.

Then server-side, open the ‘other’ Client class and find the var UID in the run() method.
below that place this:

if (UID != 01312011) {
    disconnected = true;
    returnCode = 7;
}
