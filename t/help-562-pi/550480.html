It could be that he changed interfaces (I’ve seen 10001 tutorials that “replace” the bank interface) and now it’s no longer loading.

He could also be sending the wrong packet data as well.

Because, again, I’ve seen tutorials that change the interfaces, and when they do you need to update the packet to go with the new interface. And if you don’t, you get this issue.

Here’s an example:

if (c.getOutStream() != null && c != null) { c.banking = true; c.isBanking = true; c.getItems().resetItems(5064); c.getItems().rearrangeBank(); c.getItems().resetBank(); c.getItems().resetTempItems(); c.getOutStream().createFrame(248); c.getOutStream().writeWordA(23000); c.getOutStream().writeWord(5063); c.flushOutStream(); }
This is specifically for my bank.
Now, you see the 23000? On a standard PI, and most 317s, that would be 5064.
The reason it’s not is because I edited the bank and put some new sprites on it’s interface, and of course it was cheaphacked so it’s just a new interface using some of the old content.
If I didn’t change the 23000, I’d have similar issues to what we’re seeing in OP’s post.
