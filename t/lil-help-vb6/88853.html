Well, what you need, is a winsock protocol in which connects to a remote server and sends the crc of the current program… And the server should do a check to see if it matches the “real” version, if it doesn’t it should redistribute packages immediately at which point the client computer restarts the update service and attempts a checksum match once again…

It’s not a copy and paste sort of thing, so stop looking for that.

zstars July 30, 2016, 8:34am #5

It is a bit tricky, you can use, as they said, a CRC, but if you don’t want to deal with CRCs you can use a version number too. Connect to a website to get the last version number, compare to your version number, if it matches, do nothing, if it doesn’t, download the new version into a temp folder. Now you just gotta install it over the old version. There are more ways, but a separate updater program which removes the old prog and replaces it with the new one is probably the most popular.

If that hasnt help heres something simular to what you need just for vb.net though so it will need a tiny bit of tweeking.

        Dim CurrentVer As Integer
        Dim NeedUpdate As Boolean

Function getLatestVer(ByVal url As String)
        Dim src As String
        Dim net As New Net.WebClient()
        Dim Result As Integer

        src = net.DownloadString(url)
        Result = Integer.Parse(src)
        If Result > CurrentVer Then
            NeedUpdate = True
        End If

    End Function
