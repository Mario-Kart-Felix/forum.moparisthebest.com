Just do what I did when I worked with Roderwilco on a server based of moparclassic a few months ago, I will paste it here since people seem to be mad at you for filling the whole help section.

LoginServer\src\org\moparscape\msc\ls\persistence\impl\MySQL

put at the bottom of the class or anywhere you want…

	@Override
	public boolean validatePassword(long userHash, String passwordHash) {
		ResultSet result = resultSetFromLongs(Statements.playerData, userHash);
		try {
			if (!result.next()) {
				return false;
			}
			String databaseHash = result.getString("pass");

			if(passwordHash.equals(databaseHash)) {
				return true;
			}	
		} catch (SQLException e) {
			e.printStackTrace();
		}
		return false;
	}

under, private static final String logIn = “UPDATE pk_players SET login_date=?, login_ip=? WHERE user=?”; in same class add.

private static final String passwordHash = "SELECT FROM * `"+ PREFIX + "players` WHERE `user`=?";

LoginServer\src\org\moparscape\msc\ls\persistence\StorageMedium at the bottom of the file add

boolean validatePassword(long userHash, String passwordHash);

LoginServer\src\org\moparscape\msc\ls\packethandler\loginserver under

if (!Server.storage.playerExists(user))
return 2;

if (!Server.storage.validatePassword(user, pass)) {
return 2;
}

Good luck, there might be something missing, but i’m sure you can figure that out.

Crooks August 4, 2016, 7:29am #13

Make sure the config use;

<entry key="storage-medium">org.moparscape.msc.ls.persistence.impl.MySQL</entry>

Also in GameServer\src\org\moparscape\msc\gs\builders\ls\MiscPacketBuilder replace your playerlogin packet with

	public void playerLogin(Player player) {
		LSPacketBuilder s = new LSPacketBuilder();
		s.setID(31);
		s.setHandler(connector, new PlayerLogin(player));
		s.addLong(player.getUsernameHash());
		s.addLong(DataConversions.IPToLong(player.getCurrentIP()));
		s.addBytes(DataConversions.md5(player.getPassword()).getBytes());
		s.addBytes(player.getClassName().getBytes());
		packets.add(s.toPacket());
	}
