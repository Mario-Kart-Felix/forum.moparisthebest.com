[center][size=18pt]Rune Revolution[/size][/center]

 Rune Revolution is a new input output framework that was originally named Styx. This is a different version than Galaxy which I recently released. They're both based off Styx but I developed this one much further. This is not a complete server, it's just a speedy little framework. I'll list the features below.

[size=14pt]Features[/size]

[ul][li]Login protocol decoding completed.[/li]
[li]Packet protocol decoding nearly completed.[/li]
[li]Incredibly basic player updating. Should be re-written.[/li]
[li]Packet loading via XML[/li]
[li]Uses NIO for fast asynchronous actions.[/li]
[li]Primitive task system, functions as needed.[/li]
[li]Entity based player system.[/li]
[li]Multiple connection handling.[/li]
[li]Flexible networking system.[/li]
[li]Multi-Threaded.[/li]
[li]Easy to follow packaging.[/li]
[li]Outgoing packets base written.[/li]
[li]Support for new task’s being added to the Game Engine[/li]
[li]No deadlocks.[/li]
[li]Non-Thread per client approach.[/li]
[li]Update Flags implementation.[/li]
[li]No outside libraries utilized.[/li][/ul]

I plan to continue this and wanted some opinions and suggestions. Im releasing this now because soon i'm going to be very busy preparing for school so I wanted to pump a second version out while I had time. Credits are included in the source download. Any questions, suggestions, or bug reports can be left as a comment on this thread and I will get back to you with a fix or response as soon as I can. This server is semi-class heavy but as a whole it functions beautifully.

This server contains the same amount of [b]in-game[/b] content as the original winter-love but it will crush any IO server in performance. I hope someone can learn something from this as I have. Finally I'd like to reiterate one more time, like I said before I will not be quitting this. I am actually very excited to continue, I just wanted some opinions and suggestions on the current design.
