Hi there, my name is t4 and I have noticed many of you obsess over SQL for data storage. Most of you opt into using something like MySQL installed via XAMPP. This is just plain stupid. Wouldn’t you love to use some sort of sensible structure for storing data, but don’t want the hassle of having external dependencies? Well there’s a solution for you! SQLite!

The best part? There’s a SQLite library written in Java and has easy-to-use functions for querying so you don’t have to learn much SQL!

You can pick up this wonderful library here:
http://www.sqljet.com/

Now, for the tutorial since this is a tutorial section! (Disclaimer: This is copy and pasted directly from the tutorial on sqljet’s website - I found it adequate)

Before we start, here are some constants:

    private static final String DB_NAME = "db.sqlite";
    private static final String TABLE_NAME = "employees";

    private static final String FIRST_NAME_FIELD = "first_name";
    private static final String SECOND_NAME_FIELD = "second_name";
    private static final String DOB_FIELD = "date_of_birth";
    private static final String FULL_NAME_INDEX = "full_name_index";
    private static final String DOB_INDEX = "dob_index";

Let’s start off with creating a database:

File dbFile = new File(DB_NAME);
dbFile.delete();
        
SqlJetDb db = SqlJetDb.open(dbFile, true);
db.getOptions().setAutovacuum(true);
db.beginTransaction(SqlJetTransactionMode.WRITE);
try {
    db.getOptions().setUserVersion(1);
} finally {
    db.commit();
}

Holy twirling dicks, t4! That was easy!

Okay, you want to make a simple table with that stores a first name, last name and a date of birth:

        db.beginTransaction(SqlJetTransactionMode.WRITE);
        try {            
            String createTableQuery = "CREATE TABLE " + TABLE_NAME + " (" + SECOND_NAME_FIELD + " TEXT NOT NULL PRIMARY KEY , " + FIRST_NAME_FIELD + " TEXT NOT NULL, " + DOB_FIELD + " INTEGER NOT NULL)";
            String createFirstNameIndexQuery = "CREATE INDEX " + FULL_NAME_INDEX + " ON " + TABLE_NAME + "(" +  FIRST_NAME_FIELD + "," + SECOND_NAME_FIELD + ")";
            String createDateIndexQuery = "CREATE INDEX " + DOB_INDEX + " ON " + TABLE_NAME + "(" +  DOB_FIELD + ")";
         
            db.createTable(createTableQuery);
            db.createIndex(createFirstNameIndexQuery);
            db.createIndex(createDateIndexQuery);
        } finally {
            db.commit();
        }

Woah, dude that looks pretty touch, placing entities in the database must be worse

Nope, it’s very simple:

  Calendar calendar = Calendar.getInstance();
  calendar.clear();
 
  db.beginTransaction(SqlJetTransactionMode.WRITE);
  try {
    ISqlJetTable table = db.getTable(TABLE_NAME);
    calendar.set(1991, 4, 19);
    table.insert("Prochaskova", "Elena", calendar.getTimeInMillis());
    calendar.set(1967, 5, 19);
    table.insert("Scherbina", "Sergei", calendar.getTimeInMillis());
    calendar.set(1987, 6, 19);
    table.insert("Vadishev", "Semen", calendar.getTimeInMillis());
    calendar.set(1982, 7, 19);
    table.insert("Sinjushkin", "Alexander", calendar.getTimeInMillis());
    calendar.set(1979, 8, 19);
    table.insert("Stadnik", "Dmitry", calendar.getTimeInMillis());
    calendar.set(1977, 9, 19);
    table.insert("Kitaev", "Alexander", calendar.getTimeInMillis());
  } finally {
    db.commit();
  }

Okay, how to I query the database for entities with first names of “Alexander”?

  db.beginTransaction(SqlJetTransactionMode.READ_ONLY);
  try {
    ISqlJetCursor cursor = table.lookup("full_name_index", "Alexander");
    while(!cursor.eof()) {
        System.out.println(cursor.getRowId() + " : " + 
                            cursor.getString(FIRST_NAME_FIELD) + " " + 
                            cursor.getString(SECOND_NAME_FIELD) + " was born on " + 
                            formatDate(cursor.getInteger(DOB_FIELD)));
        cursor = cursor.next();
  }
  } finally {
    db.commit();
  }

Wtf is a ISqlJetCursor? Just an interative structure that allows access to a one or more database entries! You can access the encapsulated data as seen above.

There’s quite a bit more in the tutorial on their site, hopefully I’ve made some of you aware of a great alternative.
