Krichevskoy July 30, 2016, 4:24am #15

This is turning very stupid very quickly. In the spirit of actually addressing the topic:

I’ll admit that I don’t like Visual Basic. The code is ugly, it’s non-intuitive to me, it’s tied to a specific IDE (which I hate), it’s slow, it’s buggy, it’s easy to crack, and running programs on a computer that didn’t compile them is a one-way ticket to DLL hell.

Those things said, I think it does more good than harm. First off - however corrupt, it is a flavour of BASIC. It’s impossible for me, whose first programming was done on a Texas Instruments calculator, to bash BASIC. BASIC of any kind is a good introduction to fundamental programming concepts. Maybe not the most current fad/paradigm and maybe not the best way of doing any particular thing, but you can learn those things later. Another advantage is that a school (remember our target demographic here) is more likely to teach a BASIC-centered intro programming class than anything else.

The thing that makes Visual Basic stand out over ‘normal’ BASIC is that it’s a RAD (visual) language. That gives it all kinds of advantages for the beginning programmer. A beginner isn’t going to be thrilled by “Hello World” printing once in a command line - but what if your program has a big red button that pops up a message box that you can press over and over again? Seeing very clearly that their program does something in the way they’re used to (i.e., graphically) is proof that something is coming out of their work.

With Java, making this program is a pain in the ass. As a teacher, you have the choice between teaching them to be dependent on a builder from day one or explaining not only the concepts of objects and instances, but also the vagaries of about 5 Swing classes, modal interfaces, and the event-based programming paradigm. Joy.

With Visual Basic, you show them how to drag a button onto the grey box, double-click it, and make a single API call.

I even use Visual Basic myself when I just want to throw something together quickly. Some of my dinky hack projects (read: key generators and game trainers) use RAD languages. They’re also good for UI prototyping (not that anyone here cares).
