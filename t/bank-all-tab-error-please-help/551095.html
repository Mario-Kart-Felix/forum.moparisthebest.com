sk8rdude461 August 4, 2016, 3:03pm #2

Pretty simple on as to what causes the error, actually.

The way most people design the “bank all” button is they use a for loop to loop the entire inventory, and deposits each item/stack accordingly.

What you need to do to fix it:
Find the loop, and add in an if statement to check if the bank is full before inserting the item. If full, then stop the for loop (Use a “break;”).
You may also want to add in a check to see if the item stack is “full”. Because that also will say “Bank full” when attempting to deposit an item.

One server I first delta server I developed when I picked it up always gave null all the time of clicking it. It’s array was wrong, it checked your inventory from 1-29 instead of 0-28, might also be worth while to check.

depending on your methods you could do something like this

    public boolean bankHasSpace(int itemToCheck) {
        for (int i = 0; i < playerBankSize; i++) {
            if (bankItems[i] == -1 || bankItems[i] == itemToCheck) {
                return true;
            }
        }
        return false;
    }
    public void bankAll() {
        for (int i = 0; i < playerItems.length; i++) {
            if (playerItems[i] != -1) {
                if (bankHasSpace()) {
                    bankItem(playerItems[i], i, playerItemsN[i]);
                } else {
                    break;
                }
            }
        }
    }

Or you can modify your bankItem method and make sure it declines to put it in the bank if you don’t have room. Not sure how yours is written.

The more correct method might be to create a new bankItem that will handle an array of arrays one array for each item (itemid, and slot) of the items from your inventory and then you don’t need to do a bankHasSpace() for every item, more correct, harder to do, up to you.
