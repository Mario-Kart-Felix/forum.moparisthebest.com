Project_Realm July 31, 2016, 7:16pm #8

Alright, well there can be a few reasons why it didn’t work.

  1. Your ISP is blocking 43594, which is highly unlikely.
  2. Your router is blocking 43594, which means you have to port forward (go to portforward.com and follow the instructions for your given router - pretty self explanatory)
  3. Your firewall is blocking 43594, in which case you have to tell it to stop blocking that port (it all depends on what kind of firewall you have)
  4. Your server isn’t listening to 43594, in which case you have to edit the code to make it listen to that port.

I would start with #2 (port forwarding), then #3 (firewall forwarding), then try to do the test again WHILE running the server. If it still doesn’t work, post on here again.
