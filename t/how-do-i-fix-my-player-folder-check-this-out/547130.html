The characters are saved in a .dat format more than likely. Typically, you can’t read .dat files with text editors, as they come up with a bunch of weird shapes. But some data may be readable…

If you want to make it readable, you’re going to have to change the save format from a .dat to a .txt, which will require you to change most of the saving system.

Otherwise, you’ll have to find a way to add the code in to change/add the permissions you want.

Since when do people save player data in dat formats?

What your looking at is a form of binary saving. This basically encrypts the data into a ‘.bin’ file (unless you are using 718+ which people normally program Seriazable saving, which in some cases is much more efficient. There is no way you can edit the credentials in the encrypted file for security reasons, hence why the developers program Binary or Seriazable saving methods. So to make your self owner, you have to go to the method that loads the encrypted file (normally found in the player class) and add (if you have this method)

if (getUsername().equalsIgnoreCase("Your name")) {
player.setRights(2);
}

If you have any other questions feel free to ask.
