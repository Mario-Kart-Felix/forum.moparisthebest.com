Well, with my curiousity, I ended up going back there and looking at how you guys program things. (The following is specifically about SparkWeb as that is all I looked at).

byte[] ip = socket.getInetAddress().getAddress();
String IP = ip[0]+"."+ip[1]+"."+ip[2]+"."+ip[3];

Can be replaced with:

String IP = socket.getInetAddress().getHostAddress();

Then I’m not sure if this is how you program, but I realized this:

while(true){
	try {

Most of the time you guys did statement{ and other times you did statement{. Personally (and I think by convention), I believe it should be statement {.

I also realized the HUGE gap between the class statement and the next method, like so:

public class Listener implements Runnable {

					/**
					* Accepts/denies connections.
					*/
					public void run(){

I’m not sure if you guys like it that way, but that is improper indenting.

All in all, it seems to me like you guys rushed this project. E.g.:

System.out.println("Failed to find to port 80.");
				/**
				* Blah.
				*/
				public LogHandler(){
					try {
						access = new BufferedWriter(new FileWriter(logroot+"accesslog.txt", true));
					} catch(Exception e){}
				}

In the latter you just used “Blah” as the comment and then you never handled the exception if one was thrown.

Sorry for getting all on your case for this, but I’m picky and dont like little things like this left alone. This also inspires me to create my own webserver (to learn even more NIO or MINA), but I was already planning this after finishing my proxy server. Good job.

P.S. Yes, I know this is webdesign and that you didn’t ask about this. whistles But hey, I’m picky and dont leave things like this alone.
