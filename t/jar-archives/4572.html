White_Castle July 29, 2016, 8:34pm #2

I’ve posted this before:

First make a text file called manifest.txt:

[code]Main-Class: Test

[/code]

Note: the enter after Main-Class is essential!

Then make a bat file:

@echo off jar cvfm manifest.txt Test.jar *.class pause

And then run the bat file![br][size=1]Posted at: July 09, 2006, 06:13:07 PM[/size][hr]Oh and to unpack just use WinRar.

It will take any files you like You don’t have to specify a manifest file if you aren’t planning on running directly from the jar. For example, if you wanted to put everything from a folder into a jar, you could do:

jar -cf myJar.jar *.*
White_Castle July 29, 2016, 8:34pm #7

Yeah, he’s right. I was only assuming that you wanted the same deployment as, say, mopar’s private server client. “jar cvfm manifest.txt Test.jar *.class” puts all of the class files in the folder you’re using into the jar. . puts every thing else as well, and I think that’d either make the jar too big or expose your source code. However you can do whatever you like because it’s your stuff. Specifying a main class in the manifest only makes the jar executable.

You type in the manifest “Main-Class:<your main class, the one with main(String args[]) in it>” + ENTER, and pack with the cvfm. Cvfm stores a manifest (excecutable), cf doesn’t (not executable).
