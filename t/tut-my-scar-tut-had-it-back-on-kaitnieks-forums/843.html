XXII.)Putting it all together[br][size=1]Posted at: March 27, 2006, 04:39:41 am[/size][hr]I.) What is SCAR?
SCAR is a scriptable macroing device made by Kaitnieks(whom some may refer to as “CakeMix”.) With this program you can
do almost anything(on a computer, of course.)

II.)Program line
When you open up scar you will see three lines which say

Program new; begin end.

The first line is the Program line and this is a requirement for all scripts. Some people choose not to make it,
I’m not sure if it makes a difference, but this line is basically the title of your script. For instance…

Program AutoMiner; begin blahblahblah; blahblah; end.

This let’s you know that it is in fact, an automining script. Cool, huh?

III.)begin
The second line when you open up scar says “begin”. This tells scar to do something. “What does it do???”
you may say. I’m going to tell you, right now. It tells SCAR to begin the script. It is a MUST for ALL scripts.
There’s not really too much explaining to this one, so I’ll go ahead and move on to the next one.

IV.)end.
This is the third line you see when you open up SCAR. It is always the last line of every script.
{Off Topic: I really like using the bracket things…hehe} Once again, there’s not much explaining to this one.
Do not get this confused with the other end; That just means that it is the end of the command or procedure. Moving on.

V.)MoveMouse
This function tells scar to move the mouse to the selected coordinates. The coordinates are in the bottom lefthand
corner of your SCAR window. However, you don’t see the mouse move there in this one, it just goes from one and then
magically disappears and reappears in the coordinates you typed in. (Ex.)

Program AutoMiner; begin MoveMouse(100,500); end.

That would tell scar to move the mouse directly to the coordinates 100,500 . Easy enough.

{SideNote}:To know the coordinates of the certain window you must drag the crosshairs over to the window you want.[br][size=1]Posted at: March 27, 2006, 04:40:55 am[/size][hr]VI.)MoveMouseSmooth
This is about the same thing as the MoveMouse function, except it moves the mouse in a smooth way to the coordinates,
as if a human were moving it. Not much explaining here, I explained it in the last part.

VII.)ClickMouse
Here we have the ClickMouse function. This will click the mouse! blashphemy, isn’t it? There’s another step to do when
you’re making the clickmouse part in the script though. It has to know if it needs to left click or to right click
the mouse. Such as

Program AutoMiner; begin MoveMouse(100,500); ClickMouse(100,500,true); end.

Ok, here we are telling it to move the mouse too to coords 100,500 then click on the coords 100,500.
After the second comma it says true. That means it will left click on those coords. To make it right click,
you would obviously set it to false. As in…ClickMouse(x,y,false); Enough said.

VIII.)HoldMouse
This one i didn’t figure out until not too long ago, it will make scar hold down the mouse, you might use something
like this to drag an icon or something over to a different spot on your desktop, games like “Spank the Monkey”, etc.
Here’s how it works.

Program Autominer; begin MoveMouseSmooth(100,500); HoldMouse(100,500,true); end.

Once again, we put true in there so they know you use a left click to hold it.
But this doesn’t really make sence unless you want to release the mouse, of course. Which is in the next part.

VIX.)ReleaseMouse
Obviously used with the HoldMouse function, this releases the mouse so you let go of whatever you were holding or moving.
Here’s an example:

Program Autominer; begin MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); end.

Still using the true part. NEXT!

X.)Wait
Using Wait will make it wait, as to not make everything happen at once, because things can and will go wrong if you don’t
make it wait. This program uses milliseconds so you will have to convert the seconds you want to wait, and then convert it
into milliseconds to write in there.

Program Autominer; begin MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); Wait(1000); end.

Tells it to do all that, and then wait 1 second.
[br][size=1]Posted at: March 27, 2006, 04:43:19 am[/size][hr]XI.)SendKeys
Here we have the SendKeys function. This is a favorite for autoing programs because it makes people less suspicious
about you autoing if you’re actually talking. An example may be:

XI.)SendKeys
Here we have the SendKeys function. This is a favorite for autoing programs because it makes people less suspicious
about you autoing if you’re actually talking. An example may be:

Program Autominer; begin MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); Wait(1000); SendKeys('Hello, how are you doing on this fine day?'); end.

That will do all of the above then type in that. The quotations make it so it types a sentence and not a command,
think of it as SCAR quoting what you said then typing it in. There’s only one problem. It doesn’t hit enter just yet.
So here’s how we fix it. To the computer, enter is the 13th character on the keyboard. So to hit enter when you’re wanting
it to talk, an example would be…

Program Autominer; begin MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); Wait(1000); SendKeys('Hello, how are you doing on this fine day?' + chr(13)); end.

Here it will say that, then hit enter for you so you’re actually talking. For some odd reason, if you just wanted to
hit enter and not say anything, the script would be like:

Program HitEnter; begin SendKeys(chr(13)); end.

XII.)SendKeysSilent
aside from the normal SendKeys, this will send keys to the window you selected by dragging the crosshairs from SCAR
on to that window. So even if you have other programs up, if you have it selected it will still say it to only that window
and not the ones you have on top of it. I basically explained it in part XI, so I’m going to move on.

XIII.)repeating
Another widely used command in SCAR is the repeat function. It is used to…well…repeat the script. However, this
command can not work alone because SCAR won’t know how long to repeat it, so you have to make it repeat until a certain
thing happens. For instance…

Program AutoMiner; begin repeat MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); Wait(1000); SendKeys('Hello, how are you doing on this fine day?' + chr(13)); until(false) end.

the until(false); part of it tells it to repeat that script forever. You can make it repeat a script until a certain
thing happens, which I’ll probably explain later on in here.

XIV.)constants
constants are harder for me to explain. All I can say about them is that they’re like a memory system for SCAR to use
so you don’t have to spend all that extra time typing. Cool huh. Here’s how it works.

Program AutoMiner; const Talk='Hello, how are you doing on this fine day?'; begin repeat MoveMouseSmooth(100,500); HoldMouse(100,500,true); MoveMouseSmooth(500,750); ReleaseMouse(500,750,true); Wait(1000); SendKeys(Talk + chr(13)); until(false) end.

Easy isn’t it, it just basically stores the information that telling it to Talk will tell it to say
‘Hello, how are you doing on this fine day?’ .

integer variables. There are also boolean varirables, string variables, array variables, etc… but I have no clue as to
how they work. So here’s how integer variables work.
[br][size=1]Posted at: March 27, 2006, 04:46:48 am[/size][hr]XVI.)Writeln
Writeln is pretty cool, it do you see the bottom window in scar where when you push play it will say
“successfully compiled” and “successfully executed” ? That’s where it will appear. To use it in a script it would be
something like…

Program AutoMiner; var x,x2,y,y2: integer; const Talk='Hello, how are you doing on this fine day?'; begin x:=100; y:=500; x2:=500; y2:=750; Writeln('You are now using the Auto Miner.'); repeat MoveMouseSmooth(x,y); HoldMouse(x,y,true); MoveMouseSmooth(x2,y2); ReleaseMouse(x2,y2,true); Wait(1000); SendKeys(Talk + chr(13)); until(false) end.

It will make it say “You are now using the Auto Miner.” in the bottom window part. Brilliant!

XVII.)Readln
This here function will bring up a window asking the question you typed in for the Readln, like:

Program AutoMiner; var x,x2,y,y2: integer; const Talk='Hello, how are you doing on this fine day?'; begin x:=100; y:=500; x2:=500; y2:=750; Writeln('You are now using the Auto Miner.'); Readln('Push ok to move on to the script.'); repeat MoveMouseSmooth(x,y); HoldMouse(x,y,true); MoveMouseSmooth(x2,y2); ReleaseMouse(x2,y2,true); Wait(1000); SendKeys(Talk + chr(13)); until(false) end.

If you haven’t done this by now, go ahead and copy and paste the script we have been building into the SCAR window.
It won’t actually automining, it’s just an example, but it works. Put it into SCAR, and hit run and
you can see how it works.

XVIII.)FindColor
This is one of the more complicated parts to some of the beginners. This is a kinda algebra type thing, because it
has if’s and then’s and or’s , ETC. Kind of like:

Program AutoMiner; var x,x1,x2,y,y1,y2: integer; const Talk='Hello, how are you doing on this fine day?'; begin x1:=100; y1:=500; x2:=500; y2:=750; Writeln('You are now using the Auto Miner.'); Readln('Push ok to move on to the script.'); repeat if(FindColor(x,y,564738,0,0,500,500))then begin MoveMouseSmooth(x1,y1); HoldMouse(x1,y1,true); MoveMouseSmooth(x2,y2); ReleaseMouse(x2,y2,true); Wait(1000); SendKeys(Talk + chr(13)); end; until(false) end.

Now I will explain how it works. In here I made new variable declarations, I changed the x and y variables to x1 and y1
because FindColor needed x and y. When you are using the FindColor function, you don’t need to define x and y
because it’s saying if it finds the color of 564738 in between the coordinates of 0,0 and 500,500 then to do the rest
of that script. The 564738 is the color, you can select the color that you need to get using the color picker, which
is right beside the magnifying glass. the 0,0 and 500,500 are known as the xs,ys and xe,ye coordinates.
To find these …well i’ll explain in picture form.

http://img512.imageshack.us/img512/2043/findcolor7nv.jpg

[br][size=1]Posted at: March 27, 2006, 04:51:45 am[/size][hr]XX.)includes
An include is a script with a bunch of procedures that has been premade and saved by a very generous person so you
don’t have to do all that writing yourself. To make SCAR use an include you would put {.include includename.txt}
right after the program line. Like this:

[code]Program AutoMiner;
{.include Randoms.txt}
var
x,x1,x2,y,y1,y2: integer;
const
Talk=‘Hello, how are you doing on this fine day?’;

procedure MineTheStuff;
begin
x1:=100;
y1:=500;
x2:=500;
y2:=750;
Writeln(‘You are now using the Auto Miner.’);
Readln(‘Push ok to move on to the script.’);
if(FindColor(x,y,564738,0,0,500,500))then
begin
MoveMouseSmooth(x1,y1);
HoldMouse(x1,y1,true);
MoveMouseSmooth(x2,y2);
ReleaseMouse(x2,y2,true);
Wait(1000);
SendKeys(Talk + chr(13));
end;

begin
repeat
MineTheStuff;
RandomHandler
until(false)
end.[/code]

See how it works? It tells it that the include it’s using’s name is called “Randoms”. To my knowledge, Randoms is not
an actual include, I made it up, so it won’t actually work. But here is a script with a real include

XXI.)comments
comments are things you put before, in the middle of(I wouldn’t know why), or after the script. Most people put them
before the script so you will notice them, but you know, whatever floats your boat. :rolleyes: They go before the program
line and to make them you can either put //then the message you want, which looks messy if you’re making a lot of them.
Most people use the //message part after the line that they want to explain. Another type is the {message} comment part.
It’s been made so you can say what you want to say without it interfering with the script. Putting your knowledge of
all that you’ve learned so far, along with these, could look like:
