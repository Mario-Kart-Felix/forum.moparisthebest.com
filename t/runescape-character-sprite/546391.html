RSCShadow August 4, 2016, 1:43pm #6

RSC sprites have a specific order they have to go in depending on the angle of a character. Otherwise, you might get torsos overlapping chainmails, etc. Anyway, they work similar to how Limits described. There are about 1300 sprites in RSC, and a few hundred animations. An individule animation described each angle and the colour of a set of sprites to be used for NPCs, equipement or players. Each sprite has a width, height and offsets. The offsets determine where to place the sprite on a 2D plane.

The_Web_Guy August 4, 2016, 1:44pm #7

Here is an example.

Character File

character-head = 1
character-torso = 8
character-legs = 4
character-facing = 4

character-facing 4 would be SE. (North = 1, East = 3, South = 5, West = 7, etc). Then you can interpret this file. Say head has actually 8 files associated with it. So if your head is 1, and you’re facing SE which is 4, then read it as pulling head_14 sprite. And if your head is 2 and you’re facing north, head_21 sprite. If you have 10 different head options, 10 torso options, 10 leg options, there are going to be 240 sprites, and 8,000 possible combinations.

Not sure if that makes sense, but that’s the way I would do it. And I’m pretty sure RSC has a similar method.
