I’m guessing you(the reader) know how scar works and would like to start scripting. Here’s a little guide for you to follow.

Lets look at some needed procedures for this program:

SendKeys(S: String); Chr(13);

That’s all you need! But, these two procedures alone will probally get you banned from runescape for autoing, since in the modern day and age, RuneScape have grown smart of autoers and have created traps to ban us. Below are some procedures we could use:

KeyDown(key : byte); KeyUp(key : byte); Wait(ms: Integer); Random(Range: Integer); ReadLn(question: string);

We will also make use of the case of and arrays. We will make our own procedures and go through them to see what they do. To get started, lets make a simple one with the first set of procedures, just to get the basic idea of an auto-talker.

Program AutoTalker; - We don’t need this, but as Kaitnieks said, “It’s a good style to specify program name at the beginning”.

[code]begin

end.[/code] - This is what we call “the main line”, it’s where we slap procedures and scar dose it in the order you want it to.

So, using the main line we can build our program, here’s a little example:

[code]Program AutoTalkerBeta;

procedure Hello(text : string);
begin
WriteLn(text);
end;

begin
Writeln(‘The procedure Hello won’‘t be used because it isn’‘t declared in the mainline.’);
end.[/code]

As you can tell, Hello isn’t called in the main-line. Lets make it so it is!

[code]procedure Hello(text : string);
begin
Writeln(text);
end;

begin
Hello('Now Hello is used, with the string(witch is this, and is declared with '‘s.) This may be confusing but you’‘ll get it later on in the tutorial.’);
end.[/code]

Right now we’re just using WriteLn, to write data to the Debug box. So, lets get on with our first, basic auto-talker.

[code]Program AutoTalkerSimple;

procedure Send(text : string);
begin
SendKeys(text + Chr(13));
end;

begin
Send(‘Hello. This is an Auto-Talker.’);
end.[/code]

Chr(13) Basically sends enter. SendKeys well… Sends the keys. You can use this in runescape, but it will probally get you banned. Lets take a look at a bit more advanced one.

[code]program AutoTalkerMedium;

const
Word1 = ‘This is our “first” string to send.’;
Word2 = ‘This is our “second” string to send.’;

procedure SendString(text : string);
var
TempInt, TempInt1 : Integer;
TempChar : Char;
begin
for TempInt := 1 to Length(text) do
begin
TempChar := StrGet(text, TempInt);
TempInt1 := GetKeyCode(TempChar);
KeyDown(TempInt1);
Wait(25 + Random(30));
KeyUp(TempInt1);
Wait(40 + Random(30));
end;
SendKeysVB(’{ENTER}’, True);
end;

begin
SendString(Word1);
SendString(Word2);
end.[/code]

Now, it may look complicated but I’ll explain it in parts.

Lets start with the const, the const is like variables, you can store date in them, but you can’t change it. There like a way to store settings for your program. In this case we’re storing data(strings) that we will use without changing them. We’ve got our const declared as strings, but you can use Number = 5; for example as a an Integer.

“procedure SendString(text : string);”

Basically, declaring a procedure. “text : string” Tells you text is a string and will probally be used in the procedure. Doing this allows us to use SendString(‘Text is here, it’s a variable.’); I used text witch was a bad example, but it’s our variable to write.

“var
TempInt, TempInt1 : Integer;
TempChar : Char;”

These are our vars, we call these locoal vars, ONLY this procedure can use them. Global vars are declared at the top of programs.

“for TempInt := 1 to Length(text) do
begin
blablala
end;”

This is basically a repeat. What the first line is telling scar is probally “Ok, we repeat what’s below until TempInt reaches the length of the variable text.” for … to … do ALWAYS goes up in 1. Look at Length(text), remember we declared text at the top of the procedure? (The reson I split this part up is because it’s a bit to go through at once, and it’s best to do it in the parts that matter).

"TempChar := StrGet(text, TempInt);
TempInt1 := GetKeyCode(TempChar);
KeyDown(TempInt1);
Wait(45 + Random(30));
KeyUp(TempInt1);"

On the first line, the procedure StrGet gets the string(text), character at the position(TempInt). We used for … to … do, so if TempInt is on it’s 3rd round it will be 3, so it will pic the 3rd character in the variable text. Second line we are doing somthing similar, we are saying that TempInt1 is now the key code of TempChar, to explain this better, say the 3rd character of text was ‘g’, we then get the key code of ‘g’ and say TempInt1 is it. Say ‘g’ keycode is 5, we are saying TempInt1 is 5. KeyDown simulates pressing a key, the key must be a byte, it accepts integers so we are basically getting the byte code of ‘g’ and pressing it. We then wait 45 miliseconds with a random 30 added, then we bring the key ‘g’ up. This may be confusing but experiment on each of the procedures.

Now we come to the main line, we basically use our new procedure, SendString, to …send strings! SendString presses diffrent keys in a guarenteed 45 to 75 miliseconds, it’s too human like to be bannable but everyone that uses this auto-talker would be a fast typer. It types WAY TOO FAST for me, and I make ALOT of typing errors, and often I correct them. You can make a more advanced one, witch uses arrays of characters that need shift pressed, and in your SendString procedure do somthing like “for i := 1 to 21 do begin if Char = Array[ i ] then …” and whatever.

Good luck, and thanks for reading! (If you need further assistance, pm me.)

A few final notes, about my style of coding. You can use somthing like “if(findcolor(…))then” or “if FindColor(…) then”, or “until(False)”, “until False”, it dosen’t really matter but some things like “if Color = 1 then” might not work under some circumstances.

procedure TypeSend(Text: string); var i: Integer; Shift: Boolean; begin for i := 1 to Length(Text) do begin Shift := (Text[i] = ':') or (Text[i] = '!') or (Text[i] = '+') or (Text[i] = '?') or (Text[i] = '<') or (Text[i] = '>') or (Text[i] = '@') or (Text[i] = '#') or (Text[i] = '$') or (Text[i] = '&') or (Text[i] = '%') or (Text[i] = '*'); if (Shift) then begin KeyDown(VK_Shift); Wait(5 + Random(20)); end; TypeByte(GetKeyCode(Text[i])); if (Shift) then begin KeyUp(VK_Shift); Wait(5 + Random(20)); end; Wait(50 + Random(120)); end; Wait(200 + Random(500)); TypeByte(13); end;
