well, the thing with adding more packet handlers is that…

the client comes from the runescape site, of course after that it is deobfuscated and most of the variables/classes etc. are renamed. after that it is changed to connect to a custom ip (the private server). the private server is created in relation to what packets the client sends and which ones it receives. the packets are the little tidbits of data sent from the client to the server (and vice-versa) that tell each other what to do and keep them up to speed on variable changes. this way, the client and server can operate individually while still remaining in sync. Naturally, if one of the packets is not handled by the server, it does not send the next packet back to the client telling it to make graphical changes, and that change is not reflected on the server either, as it is an unrecognized packet. So, for example, say the item on item packet is not handled, if you tried to use an item on another item in your inventory then nothing would happen. If it was handled, however, the packet would be processed by the server, the appropriate graphical changes would be made on the client, and the changes would be reflected in your inventory server-side and client-side.
