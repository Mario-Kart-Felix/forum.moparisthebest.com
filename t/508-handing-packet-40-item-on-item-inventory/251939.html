Well, guys, I’ve put a lot of effort into this today. I first starting with handling item on npc, but that didn’t go too well (Trouble finding what was what) and I eventually got this. It took a while of experimenting to find the 2 item things. I’m only releasing because I want to help people who couldn’t do this themselves, just like Pali released his 508 server. Enjoy!

You need to know how to declare things, create new classes, what an array is (I guess for when changing the packet size), and… Well, I guess that’s it…

Difficulty: Easy (Hard if you’re a beginner)
Copy and Paste: Yes, in the ItemOnItem class.
Classes modified: Packets.java, PacketManager.java, ItemOnItem.java (Will be added later). All of these are in the IO folder.

Okay. First off, packet 40 is by default not handled in a blank 508 server. Handling it is actually very easy, but making it useful is sorta the hard part…

Step 1: Changing packet 40’s size
Well, First off, we should change the packet size in Packets.java. Scroll down where is assigns packet 40’s size. The size should be -3. Change the -3 to 16, the packet 40 size.

Step 2: Handling the packet
This is done in PacketManager.java. First, you must declare a new variable. The variable’s type should be ItemOnItem. Assign it to be a new ItemOnItem. It’s like this.

Type Variable = AssignedThing

You should know where this goes from the previous declared things in this file.

Step 3: Making the packet stuff happen from a file like the other ones.
Scroll down a bit to find switch (packetId) { and create a new case under it. The case ID should be 40, obviously. Inside it, put…

itemOnItem.handlePacket(p, packetId, packetSize);I honestly don’t think I needed to put that, as it was in a lot of the other cases. Now, you’re done with PacketManager.java.

Step 4: The part that was hard for me! The file!
Okay. In the IO/Packets folder, create a new class like the others and make sure it’s called ItemOnItem. Inside it, it should contain (Since it’s a packet handling class) this.

public void handlePacket(Player p, int packetId, int packetSize) { if (p == null || p.stream == null) { return; } }That should be right under public class ItemOnItem implements Packet {.

Well, here’s the most important part for this. Without it, you wouldn’t be able to check the item ids or slots.
You need to read the data for this packet. I won’t give you easy C+P version of it… Or maybe I will. Here it is. It goes under the } under the return;

int usedWith = p.stream.readUnsignedWordBigEndian(); int itemUsed = p.stream.readUnsignedWordA(); int uselessStuff = p.stream.readUnsignedWord(); uselessStuff = p.stream.readUnsignedWord(); uselessStuff = p.stream.readUnsignedWord(); uselessStuff = p.stream.readUnsignedWord(); int usedWithSlot = p.stream.readUnsignedWord() - 128; int usedSlot = p.stream.readUnsignedWord() - 128;Now since that is copy and paste, I shall explain it all (Sign… :()

int usedWith = p.stream.readUnsignedWordBigEndian(); - It’s what the selected item is used with. The first 2 bytes are the item ID, so that’s what the readUnsignedWordBigEndian does. Though, I’m not why you just don’t read the 2 bytes. I’m not smart enough to define Big Endian =\

int itemUsed = p.stream.readUnsignedWordA(); - That’s the item that you first selected. The ReadUnsignedWordA reads 2 bytes, but subtracts 128 from the second one.

int uselessStuff = p.stream.readUnsignedWord(); - This is just to increase the offset to read from by 2. It’s repeated 4 times as you can see. Basically, if you want to read the 13th byte like we do, but the offset is set to read the 5th, you read bytes so the offset increases until you’re at the one you need.

int usedWithSlot = p.stream.readUnsignedWord() - 128; & int usedSlot = p.stream.readUnsignedWord() - 128; - It’s the item slots, the one you use and the one you use it with. I subtract 128 from it because the slot is actually slot + 127 if you just read the unsigned word. But, why did I subtract 128? Because you never check an index starting at 1, always 0. That means an item in the first slot’s slot will be 0.

Step 5: Last need part!
Well, now that that terribly boring part of this tutorial I doubt anyone here will actually read except mods because they’re checking to see if this tutorial doesn’t break the rules is over, I can tell you how it works. That was a long run on sentence.

Well, you need to start the case checking (Or not. I recommend it). You start it by using Switch (int) {. In this case, the int will be itemUsed. [b]Don’t forget to add the bracket after that line!!

Now, it works like this.

case *First used item id*: if(usedWith == *Item used with*){ //Actions } break;And voila!

Step 6 (Optional): An example.
This is an example, and you’ll need this for item changing (Not the example).

  1. Import palidino76.rs2.players.items.PlayerItems;. It makes it so you don’t have to give a path when using the file.
    2. PlayerItems pi = new PlayerItems();. You add that for when editing your items. Like, pi.addItem(stuff).

Those 2 lines I got from the ItemOption1 file.

Now, the example. Using a short bow on bowstring…

case 50: //Unstrung shortbow if(usedWith == 1777){ p.frames.sendMessage(p, "You string the unstrung bow."); pi.deleteItem(p, 50, usedSlot, 1); pi.deleteItem(p, 1777, usedWithSlot, 1); pi.addItem(p, 841, 1); p.addSkillXP(100 * p.skillLvl[9], 9); }That’s an example.
