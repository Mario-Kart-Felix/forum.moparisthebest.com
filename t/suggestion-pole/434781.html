Andrew_Ferguson August 2, 2016, 4:16pm #1

Hi everyone, lately I’ve been thinking about how the system works on the site. If we want or need a topic to be closed or locked we have to contact a moderator or report our own post to ask. I think it would be a lot simpler if we could lock our own topics, that was if we see the topic is going in the wrong direction, or we simply are ending a project/question, we can just have it die off and not be bumped later on by a random person.

Simple version: Users able to lock own topics.
Simple expected results: Less grave digging, and topics getting way off topic.

/Opinions, revisions, or idea below.

mod_taharok August 2, 2016, 4:16pm #2

This one comes up very occasionally. The two sides of the argument are basically on one side, a user gets complete freedom over their topics and can lock/open them as they see fit, whereas on the other a topic should only be locked under special circumstances, of which nearly only moderators are aware (hence the many strings of posts wanting a topic locked, but the moderator not actually complying).

Then there’s the question of whether a user can unlock their topic if a moderator locked it. I’m fairly certain this is not possible. Given that, I’m partially for this idea. I like the idea of development-oriented sections being allowed this right, and the same with help sections (although we would still encourage topics to stay open for any possible relevant discussion).

I actually think the development teams board was set up like that at one point…
