Let me explain it to you:
Your ::master command is in the administratorCommands section. There is also a playerCommands, moderatorCommands, and ownerCommands section.

By doing what I told you, you have moved the command from the administratorCommands section to the playerCommands section of all of your commands.

These sections are grouped together so that the coder only has to type out the playerRights == 2 ONCE and then take all of the commands from that group and make them require that they are administrators.

So every command within the administratorCommands section requires administrator rights, and every command inside the playerCommands section requires no rights.

You should have something like this in your commands file:

if (c.playerRights >= 0) playerCommands(c, playerCommand); if (c.playerRights == 1 || c.playerRights == 2 || c.playerRights == 3) moderatorCommands(c, playerCommand); if (c.playerRights == 2 || c.playerRights == 3) administratorCommands(c, playerCommand); if (c.playerRights == 3) ownerCommands(c, playerCommand);

That means that if you’re an owner (playerRights == 3) then you have access to the ownerCommands section of the commands. This also means that only people with the rights level of 3 can access these commands.

It also means that if you’re NOT an owner (playerRights == 3) all of the commands in that section become unavailable to you.
