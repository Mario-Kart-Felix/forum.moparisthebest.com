Does anyone find it a hassle to have to go look up an item on an ID list just to spawn something if you don’t have the ID memorized?
I have, so I decided to make a few commands to make your spawning easier.

I made these commands out of 100% scrap.
Credits to anyone else are not needed.

I’m sorry if my conventions aren’t up to your standards.
I did my best to perfect the indentions, and that’s good enough for me.

First, open up server.model.players.Player.java, and declare these variables:

public boolean ibnInitialized = false; public boolean quantityRequest_LIST = false; public boolean quantityRequest_SINGLE = false; public int ibnSRHolder = -1;

NOTE: On my server, these variables are actually declared in Client.java.
If you have any issues with these four variables, try moving them to Client.java.

All four of these variables are used if you want to input a quantity for the spawned items.

ibnInitialized keeps the server from throwing a NullPointerException.
I included this because I haven’t worked with arrays initialized with “new int” before.

quantityRequested becomes true if you used ::ibnx or ::ibnlistx.
quantityRequested_SINGLE for ::ibnx, quantityRequested_LIST for ::ibnlistx.

ibnSRHolder holds the item ID of the clicked item while you input the desired quantity.

Differences between commands:

::ibnall

c.getItems().addItem(Server.itemHandler.ItemList[i].itemId, 1); c.sendMessage("Returned item: "+Misc.optimizeText(listedName)+""); results += 1; if (c.getItems().freeSlots() == 0) { c.sendMessage("Your inventory is full!"); c.sendMessage(""+results+" results were returned."); return; }
This is called when the item is a valid result. Instead of ending the method, the loop continues its search for more results. If the inventory is full, then the loop breaks.

::ibnlist

for(int z = 0; z < c.questListTextID.length; z++) { c.ibnResult[z] = -1; c.getPA().sendFrame126("", c.questListTextID[z]); } c.ibnInitialized = true; c.getPA().sendFrame126("Results", 663);
Clears all lines in the quest list, and resets the result placeholders. Switches the ibnInitialized to true, which is called in ClickingButtons. Displays the text “Results” at the top of the quest list.

Resets the spot variable.

This is called when the item is a valid result. The item’s ID is added to the current spot in the result placeholder array, the item’s name is added to the current line in the quest menu, and the spot and results variables increases by one.
