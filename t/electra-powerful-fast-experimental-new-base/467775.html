[SIZE=5]Introduction[/SIZE]

Electra is a server that is created to be intuitive, well designed, and fast. It’s currently under development and many things have yet to be implemented. Electra isn’t designed to be a new base that will work with other clients. I’m not going to limit myself to strictly the work that Jagex did 7 years ago, and neither should you. I plan on actually maintaining and constantly improving the server, so it’s not just a one-off base release like literally every other base out there has been so far.

[SIZE=4]Features[/SIZE]

[ul][li]Math parser and evaluator, used in the compiler for converting expressions into bytecode for array lengths but can also be used in your projects as well. You can translate infix (the notation humans use) to postfix notation. From postfix notation you can evaluate the expression.[/li]
[li]Efficient networking via Java’s NIO API.[/li]
[li]An improved and more usable event/handler system.[/li]
[li]An extremely efficient and ‘coder’-friendly network event (packet) system.[/li]
[li]Support for reconnecting instead of just forcing the old player to disconnect and reload the same player as a different instance. Only the client that opened the original connection can reconnect to a session.[/li]
[li]Simple and non-redundant naming conventions for variable and method names. You may dislike this, but it’s too late to change now. It’ll grow on you![/li]
[li]The login block is encrypted using RSA. This means that every session is secure as long as the private key is unknown. Change the private/public key set if you’re going to use this publicly.[/li]
[li]Cache file and cache archive reading.[/li]
[li]JAGGRAB and OnDemand file server for cache updating.[/li][/ul]

[ul][li]More than just walking around ;)[/li]

[li]Porting over my server-sided clipped walking/following code from a C# server I wrote.[/li]
[li]All the general RuneScape content-related stuff like skills and stuff.[/li]
[li]In-game map editor (this is probably going to take a long time and a lot of work)[/li]
[li]Checksum checking on login (easy, just keep putting it off for no reason)[/li]
[li]Figuring out why the client gets confused when I send too much data to it over a short amount of time. This really just applies to JAGGRAB and OnDemand freaking out when all the data is being sent/read all at the same time.[/li]
[li]Something better than a file associating eventhandlers to events.[/li]
[li]Benchmarks and core improvements based on if the server can handle 2k players without cycle desyncing. I’m sure I can make it work with just one thread, to keep the design simple.[/li][/ul]

[SIZE=3]Dependencies[/SIZE]

[ul][li]SnakeYAML - YAML parser and emitter for configuration files[/li]
[li]Apache Commons BCEL - Bytecode Engineering Library for event ‘compiler’[/li]
[li]Apache Commons Compress - For BZip2 compression/decompression in the cache[/li][/ul]

[SIZE=3]Compatibility notes[/SIZE]
I’ve taken it upon myself to simplify and modify the client. The modifications I’ve made are to make the design of the server simpler. I’ve simplified the protocol (no random data types), removed unused anti-cheat packets (which are useless now anyway), etc.; and the enabling of RSA, JAGGRAB, and OnDemand makes this client incompatible with any other servers and this server incompatible with other clients.
