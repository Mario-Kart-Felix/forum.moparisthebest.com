semaphore July 30, 2016, 2:58pm #8
Moparisthebest:

…and since today you can’t get an undetectable deob…

Why do you think that?

It’s fairly trivial to run a deob without any more detectability than any other modified client. It requires a couple changes to the code where the reflection checks are made as well as a deob name mapping but other than that, there are really no other real obstacles to running a deob with no more detection than any other modified client.

I downplayed the term mapping. To explain, the mapping must be able to successfully return visibility modifiers on any field/method of the original codebase as well as original field/method names.
The simplest way that I can think of to accomplish this involves using zkm for renaming because it generates a flat-file change log. Parse the change log and you have a perfect mapping of the original -> deob names/visibilities.
