How to make a WebClient after security update (And Java 8) - DogeSign

Patrick_Le_Star August 4, 2016, 11:52am #1

[CENTER][/CENTER]

As some of you may know, A (fairly) recent Java update has made it extremely awkward for users to play RuneScape Private Server Webclients with self-signed certificates. To allow users to play, you will need to pay sometimes between $150 to $200, be a registered company AND provide your actual information in order to get an overpriced certificate.
At the moment, this may be fixed by simply changing the security level to medium. Whilst this works, Java 8 will remove the capability to run self-signed code fully.
But fear not! Patrick Star has the solution!

DogeSign is a centralized certificate authority, signing certificates for free in order to ensure continuity of webclients for the foreseeable future!
Below is an informative paragraph about how to import CA’s to your Java installation.
[SPOILER=much informative]
[B][SIZE=5]Method 1: Easy Installer[/SIZE][/B]

  1. Download https://dl.dropboxusercontent.com/u/45427687/dogesign.jar (Source code available here: https://dl.dropboxusercontent.com/u/45427687/setup-src.zip )
  2. Run
  3. Input Java’s Keystore Password (defaults to changeit. If you don’t know what this is, you should probably leave it as changeit)
  4. Done!
    [B][SIZE=5]Method 2: Manually Adding[/SIZE][/B]
  5. Download the certificate for the CA. DogeSign’s CRT can be found here: http://puu.sh/7lAeI.crt
  6. For Windows users, Open control panel and search for Java


    Open the appropriate Java configuration.
  7. Go to the security tab
  8. Click Manage Certificates, and select Signer CA in the dropdown.
  9. Click Import, change file type to All Files and select the certificate. Press Open.
  10. Select “Secure Site CA” in the dropdown box, and repeat step 5.
  11. Press Close, Apply, OK.
    You’ve now added a certificate authority!
    [/SPOILER]

If you’re the owner of a private server, and would like to get a signed certificate for your web browser, Follow the instructions below.
[SPOILER=so success]
Create a certificate and private key pair. This can be done using any tool of your choice or the below command line:

keytool -genkey -dname "CN=SuperFunScape, OU=Uniqueness, O=Old School Remake Super Fun Unique Best Server, ST=Many Server, C=RS" -validity 365 -alias ServerName -keypass XXZZXX -keystore ServerName.ks -storepass XXYYXX

It’s important that the country code is set to RS. All of the other fields are irrelevant, and should reflect your server. Validity should only be 365 days.
Create a certificate sign request for the generated certificate / keypair

keytool -certreq -alias ServerName -file ServerName_csr.pem -keypass XXZZXX -keystore ServerName.ks -storepass XXYYXX

And PM me the ServerName_csr.pem file.
When I get around to it, I will sign this and respond with a signed one. You can then use this just like any other self-signed certificate.
Feel free to copy the instructions in the other spoiler to instruct your users how to trust our certificate.
[/SPOILER]

[SPOILER=very responsibility]
If a webclient has a backdoor in any way using one of DogeSign’s signed certificates, Download the jar file and PM me it. I will then verify the backdoor and revoke the certificate. This will block all users from being able to load their client.
[/SPOILER]

[B]What’s the point of this? Why not just get users to add a certificate per server?[/B]

Because that wastes the users time. An application could easily be made to automate this process, but it’s still more effort than adding one certificate one time. Additionally, this provides a level of security for the users, as it allows me to manually block backdoored clients.
