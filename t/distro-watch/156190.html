Well, this section doesn’t get alot of stuff other than questions and help topics, I thought we could bring some intelligent discussion to this board since generally only people with a brain will be able to use Linux based operating systems.

I thought we could have our own little distro watch and discussion thread. Spice it up with a bit of a debate, but no ridiculous flame. Should be interesting to see what other people use and why it’s the right thing for them. New users will find it helpful if they’re looking for a new OS and current users will find some hopefully well reviewed alternatives. We’re not looking for HUGE posts, just say what needs to be said. Feel free to link to images of your desktop, but don’t post the full image into this thread, only a link- please.

Personally, I have tried a variety of *unix operating systems. I wanted something simple and is easy to maintain, I wasn’t bothered about too much customization but I wanted it to look just right. I’ve tried all the *ubuntu except the new Gobuntu. I’ve also used openSuse, freeBSD and simplyMepis. I didn’t particularly like any except for Kubuntu. Now, Thug-life pointed me in the direction of Sabayon a little while ago, it’s a very large gentoo based OS that is filled with huge amounts of preloaded software and games. (It comes on a 3.7GB DVD). For me, it was too bloated, I haven’t tried the mini version but I probably will do so soon. The thing about Sabayon was it seemed to concentrated on eyecandy and not enough on functionality.

I now use Kubuntu on my main computer (with XP as dual boot) and Arch Linux on my old computer. Kubuntu has everything needed for new users to easily understand and for more experienced uses to customize. I Have Arch Linux because there is no fancy easy to use GUIs and everything must be done properly often on the command line - I wanted to try something “real” Linux users use (from Kamu). It’s more difficult and requires experience, but with the help of Mopman, I got it running quite quick and found it to be very slick. I would not however recommend it at all for anyone with less than a few months experience of Linux Operating systems.

I have been using Arch Linux full-time for the past two years, and it has worked perfectly for me. The Arch philosophy follows KISS, but not in the form that it provides easy ways to do everything. In this sense, KISS more means that it gets rid of all the easy-to-use front-ends. This essentially makes it easier to get into the actual configuration files and modify things to do exactly what you want them to do.

When you first install Arch Linux, you’ll probably only be confronted with a blank terminal, which allows you to build up your system to be capable of exactly what you want it to do, and nothing more. This seems to be quite the opposite of Sabayon, which, as you pointed out, installs everything automatically, seemingly bloating your system. This, in my opinion, speeds up productivity and overall performance.

The Arch package manager, pacman, follows the same philosophy as the rest of the system. Although there are 3rd party GUI front-ends which a few people prefer to use, pacman itself is a terminal based application. This contributes to the Arch philosophy that users should have the freedom of choice. By only providing the very baseline of what you need to install and remove packages easily, it allows you to build up what you really want it to do.
