Alright. So for a day I’ve been working on a scripting language for RSPS. The compiler is being written in C# and the interpreter will be in Java (duh). Now, I’m posting this here because there’s no certain section for RSPS tool development, and we’re all friends, right?

Anyway, for all you Linux users out there who would want to use this, look up MONO. It’s something for Linux and I suppose Mac users that allows them to run .NET applications, which the compiler is.

As for screenshots, I can’t really give any. I suppose I could of a batch files giving errors… Anywho, here’s a sample shit script and what its compiled output is.

[code]Script:
;testClass
;-For testing scripts
Script ends on “end”.
Extention: rss-;

testClass is ending
var i is string
var j is string ;Both strings
compare (i, j) is “lolol”
;Right now, the compare statement is useless… >.>

var i2 is int
var j2 is int ;Both integers
compare(i2,j2) is 111

end

Result(bytes):
09 74 65 73 74 43 6C 61 73 73 00 03 01 01 01 69 01 01 01 6A 02 01 01 69 01 6A 01 07 22 6C 6F 6C 6F 6C 22 01 00 02 69 32 01 00 02 6A 32 02 00 02 69 32 02 6A 32 00 00 00 00 6F

Result(text):
.testClass…i…j…i.j…“lolol”…i2…j2…i2.j2…o[/code]

The syntax may be ugly, and not the most flexible, but it’s easy to get used to and work with. Here’s the same script above that I fucked up, followed by the output.

[code];testClass
;-For testing scripts
Script ends on “end”.
Extention: rss-;

testClass is shitplz
var i is string
var i is int ;Both strings
compare (i, s) is 111111
;Right now, the compare statement is useless… >.>

var a is int
var d is int ;Both integers
compare (a, d) is "k"

end[/code]

You’re probably wondering why it only showed 4 out of however many errors. This is because I have it so if there’s certain errors, it stops checking for other ones. If I continued this, there would be a high chance of the compiler crashing.

Oh, and finally, it works like how you compile Java files. Make a batch file and execute the file “scriptrs.exe” with the arguments as the files.

So that’s all I have for you. This project won’t take too long (Atleast I’m hoping. The interpreter will be a pain in the ass too because it’s Java…), and the compiler should be done maybe next week depending on how much I slack

Update
Well, I’ve added if statements, variable assigning, the boolean type, and fixed some bugs, such as commentary ones. I have some work to do with the formatting, such as remove unnecessary spaces (Shouldn’t be too hard, but it needs to be added ASAP). Not only that, but I’ve added a decompiler. Note that scripts weren’t being compiled to protect their source, but to make it easier for the interpreter and not take up as much CPU. This is for if you want to edit a script if the source isn’t given.

Update (20 minutes after the previous one)
I’m glad I got this update out of the way, which is USR (Unneeded Space Removal). Basically, it makes it so you can have as many spaces as you want and it will remove them and compile like they weren’t even there. If you’re confused, it makes something like this compile fine:

testClass is ending var i is int var j is int compare( i , j) is 11 ;Actions here end var s is string var b is bool if (s ) is "lol" ;-More actions here More commentary Closing the commentary-; assign s "Ohai guiz!1" end assign b false end

~Lin
