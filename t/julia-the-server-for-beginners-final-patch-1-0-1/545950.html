[size=15pt]Introduction:[/size]
[spoiler]
Easily create the server you want without touching a line of code.
The purpose of this project is creating your own base as beginner as simple as possible.
There are a whole bunch of editors (see media and features) that make modifying this server
to your liking very easy WITHOUT touching a single line of actual java code.

If you’re a beginner in the rsps scene or just somebody who wants to make a base quickly without
having to actual program, then this is for you.
[/spoiler]

[size=15pt]Features:[/size]
[spoiler]
I’m not going to drop a wall of text here as nobody reads it anyways.
Just check out the media and you’ll see what features are available.

Ok RandQm, but what is different in this version compared to the previous ones?
[spoiler=this]

[ul][li]A whole bunch of bugs have been fixed from the previous versions.[/li]
[li]Some beauty work:[/li]
[list]
[li] I added themes.[/li]
[li]You can’t open multiple instances of the same window anymore, it will jump to the front.[/li]
[/list]
[li]The code behind the search frame now works, so you can search items/shops/npcs that are in your server.[/li]
[li]The shops editor has been expanded on supporting level requirements and points.
(Though the points is kinda useless as you should write code on how to get the points, didn’t think that one through.)[/li]
[li]Scroll panes have been added to easily go through data in lists.[/li]
[li]Readability has been improved e.g. ‘ID 1333 - AMOUNT: 25’ became ‘Rune scimitar(1333), amount: 25’[/li]
[li]Player manager has been reworked (codewise) for better updating, ban/mute etc… didn’t work correctly either which has been rewritten… but that belongs more under bug fixes. I guess.[/li]
[li]About the data importer, though it’s impossible to transfer data from somewhere else 100% correctly to this server, I optimized it a lot.[/li][/ul]

It’s pretty much the final product of the goals I had for this project when I started it.
[/spoiler]
[/spoiler]

[Spoiler=How to patch?]
[I]I downloaded the source from my download link and let the patch run. It works fine, so if you have any issues you are doing something wrong![/I]

Step 1: Extract the CONTENTS of ‘Julia patch #.rar’ into the MAIN directory of Julia. (This is the directory which contains the run.bat)
Step 2: Run ‘Julia #.jar’ and wait for a screen to pop up which says that Julia has been patched.
And you’re done! You can now delete the ‘patch’ folder and the ‘Julia patch #’ jar.

[SIZE=7pt]note: # stands for the patch number.[/SIZE]
