speedster is right, however he explained it pretty noobish. (no offence), he gave a very simplistic / general idea of what it actually means, which is ok because it means he understands it, but not enough to be able to explain it in great context to anybody else.

So, heres my attempt: When you declare a method in a class static you do not need an instance of the class.
You can use static methods to access other static members of the class, however, you cannot use the static method to access any non-static variables or methods. A class containing static members is called as a static class, also the private modifier specifies that the member can only be accessed in its own class.

in your case, you would probably be better assigning the field and method final aswell, this means that you cannot write to that variable at runtime, this is useful for static members as they are often set and not written to again (unless it is bad code), it also saves room in the jvm as for normal string it puts memory aside for it incase it needs editing and changing later on.

however, you are only a beginner and its best not to over-flow you with information. my advice would be to read the online sun java tutorials (via google), and that should teach you most of what java is about.
