yes you are

if (error = true) {

the “=” operator is used for assignment, you are making error equal to true
the “==” operator is what you are looking for, it tests for the value equality
replace it with this

if (error == true) {

or just

if (error) {

this reminds me of the very amusing bugs which can come up if you use this trick with the assert keyword,
if you dont already know, assert is a useful way of checking if your assumptions are true, you can run a program with assertion enabled to detect bugs, or disable so it runs faster or whatnot

but try this, and your program will work fine with assert enabled, but mysteriously bug when they are disabled.

assert (id = 1);

up there, you’ve set id to 1, but only if assert is enabled
