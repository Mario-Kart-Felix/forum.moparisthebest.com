[u][b]Integers/INTS, BYTE, SHORT, LONG[/b][/u]

Java stores integers in binary numbers (int, byte, short, long)

In Java, the 'int' type is a primitive , whereas the 'Integer' type is an object. The differences between objects and primitives are somewhat beyond the my noob intelligence, but to summarize, objects provide facilities for polymorphism, are passed by reference (or more accurately have references passed by value), and are allocated from the heap. Conversely, primitives are immutable types that are passed by value and are often allocated from the stack.

[code=java]switch(variable) {

       System.out.println("Hello MOPAR!");
      break;
    case 2:
        System.out.println("Hello World!");
      break;
    }
}[/code]

Switch statement is a tidier way of doing if/else if statements!

Here's a chunk of code quickly made so you guys can relate it to your RSPS.

[code=java]private void commands(Player player, String command) {
    if (command[0].equals("update")) {
        // send update
    }
    if (command[0].equals("master")) {
        // give player max stats
    }
}[/code]

That code can also be written like so:

[code=java]private void commands(Player player, String command) {
    if (command[0].equals("update")) {
        // send update
    } else if (command[0].equals("master")) {
        // give player max stats
    }
}[/code]

To make a switch statement you put the crap that would go in within the IF statement and put it into a switch statement like this:

[code]switch (command) {
    case"update":
      //send update
      break;
    case"master":
      //give player max stats
      break;
}[/code]

You must make sure you have a colon after the case (case"test":) and don't forget to add [b]break; [/b]

Another cool thing about switch statements are you can stack things example:

[b]case"master":
case"max":
maxstats();
 break;[/b]

What that does is if the the command is called master or max it will still give execute the same command without needing to retype it or do any special operations.

What is a Enum? 
Enum is type like class and interface and can be used to define a set of Enum constants. Enum constants are implicitly static and final and you can not change there value once created. Enum in Java provides type-safety and can be used inside switch statment like int variables.

Here's an example:

[u][b]Parameters[/b][/u] - reference from mindprod.org
A parameter is a variable that you pass in to a function. Kinda like an argument...only not as loud and frustrating.

Parameters are always passed by value not by reference. The called method gets a copy of each primitive and a copy of the pointer to each object. The called method can therefore not change any local variables in the caller. However it can change the contents of objects (which includes strings and arrays) passed to it. Some people may argue that counts as call by reference, however a calle cannot make the callers variables point to different objects, just change the contents of objects. Objects are always accessed by reference, (literally via a reference) even when parameters are not involved. You can kludge call by reference by passing values back to your caller inside an array or by returning a freshly minted object full of values.

[code=java]/**
 * add two numbers
 * @param a first addend
 * @param b second addend
 * @return sum of a+b
 */
static int add( int a, int b )
   {
   // a and b are 'parameters', the variables the callee uses to accept inputs.
   return a + b;
   }

It’s sort of similar to having a variable declared but in this case its for a sole method only.
