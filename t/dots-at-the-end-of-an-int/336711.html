system July 31, 2016, 3:53pm #2

A dot means the number will be treated as a double (as opposed to an int, which is the usual case). However, it is pointless since the double is converted back to an int in each such assignments. Actually, in Java, an int cast is required so that src is probably bs.

One reason you would use a dot is fractions:
double d = 5.0/9.0; (5/9 would yield 0 because int division truncates)

By the way, don’t ever use a dot by itself like that. “3.0” is much more legible than “3.”, yet yields the same value and properties.
