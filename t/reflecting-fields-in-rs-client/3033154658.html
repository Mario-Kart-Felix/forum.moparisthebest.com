1- to reflect private fields use getDeclaredFields instead of getFields():

Field[] cVars = slClass.getDeclaredFields();

2- you have no problem reflecting statics fields that is a field.get(null) , but to reflect a non-static field (not a client field), you will have to reflect its value from an object that carries an instance of it, for example lets say you want to to get the name of your player.
To do that, you’d have to reflect the getPlayerName non-static from the MyPlayer static field wich will look like:

Object myPlayer = myPlayerField.get(null); //since its static String myPlayerName = playerNameField.getString(myPlayer)// will get a player's name from //myPlayer's object wich will in turn , return my player's name //and since its a name (String), that is why we've used the getString(Object o) function.

3-To know what fields are responsible for what, you can just look @ previous Updaters output and learn how to detect and maintain them with some bytecode editor library such as BCEL,and write an updater for them, you can thank the old people who initially discovered them by hardwork,
other than that, to discover some new fields, just read deobs to find out some methods that use fields and that can be useful to know whats the purpose of the field, and/or use a reflection explorer as you mentioned to detect variations during game play, but there will be thousands of them, so try customizing the explorer and limit the fields ammount.

Hope that helped ^^
