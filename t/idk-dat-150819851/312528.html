File Header:
A short that contains the number of character design parts.(84 by default in a 317 cache)

Reading of the IDK.DAT header.

    public static void unpackConfig(StreamLoader streamLoader)
    {
        Stream stream = new Stream(streamLoader.getDataForName("idk.dat"));
        count = stream.readUnsignedWord();
        if(cache == null)
            cache = new IDK[count];
        for(int j = 0; j < count; j++)
        {
            if(cache[j] == null)
                cache[j] = new IDK();
            cache[j].readValues(stream);
        }
    }

Reading of the character design part data:

   private void readValues(Stream stream)
    {
        do
        {
            int opcode = stream.getByte();
            if(opcode == 0)
                return;
            if(opcode == 1)
                partID = stream.getByte();
            else
            if(opcode == 2)
            {
                int modelCount = stream.getByte();
                models = new int[modelCount];
                for(int k = 0; k < modelCount; k++){
                    models[k] = stream.getShort();
                }

            } else
            if(opcode == 3)
                noInterface = true;
            else
            if(opcode >= 40 && opcode < 50)
                originalModelColor[opcode - 40] = stream.getShort();
            else
            if(opcode >= 50 && opcode < 60) {
                reassignedModelColor[opcode - 50] = stream.getShort();
            }
            else
            if(opcode >= 60 && opcode < 70) {
                headModels[opcode - 60] = stream.getShort();
            }
            else
                System.out.println("Error unrecognised config code: " + opcode);
        } while(true);
    }

opcode:

[ul][]0 - ends of part config
[]1 - value 0 - 13 identify which part the particular Character Design data is using

PART IDS:
[list]
[]0 - Male head
[]1 - Male jaw
[]2 - Male torso
[]3 - Male arms
[]4 - Male hands
[]5 - Male legs
[]6 - Male feet
[]7 - Female head
[]8 - Female jaw
[]9 - Female Torso
[]10 - Female Arms
[]11 - Female hands
[]12 - Female legs
[]13 - Female feet
[/list]

[*]2 - Models: first byte is a model count (this means a character part can have a MAX of 256 models) then you loop and read short value of the model ids[/ul]

[ul][]3 - I defined this as “noInterface” when the character design interface is up the client will ignore this part. However the server can still send this part to the client as the characters appearance. (Example: set your characters leg design to 83 and your legs will become a cart. but this cart does not show up in the character design interface.)
[]40 - 49 - gets a short value of a color on the original model (used for model recoloring)
[]50 - 59 - gets a short value of a color to replace the original model color (used for model recoloring)
[]60 - 69 - short value of head model ids[/ul]
