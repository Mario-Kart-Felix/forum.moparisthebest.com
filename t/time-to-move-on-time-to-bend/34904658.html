Are you sure there is one?
If I remember correctly (keeping in mind that I haven’t looked at RS2 protocol in a long time), the way it works is:
client sends, basically, “actionX click(objectX,objectY)”
The server decides that an object at (X,Y) responds to actionX. Let’s say it’s a rock. The server receives the action and changes the rock’s state to “mined” in the next object-state update. Then it adds another piece of coal to your next inventory update. It might send you a text message that says “You have mined coal” but it could just as easily be a message that says “Welcome to RuneScape!”

There isn’t a “OK you mined coal successfully!” packet, just the related effects. You could detect by checking for any of them.
