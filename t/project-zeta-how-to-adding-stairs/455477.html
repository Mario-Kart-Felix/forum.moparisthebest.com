Super simple, but I just thought about mentioning it.

Go to package org.randqm.model.player.packet. You should see StairHandler. Open it.

You should see

	/*
	 * The stair Ids if you should increase 1 height level.
	 */
	private int[] stair1up = {1747, 1738};
	
	/*
	 * The stairIds where you should decrease a height level.
	 */
	private int[] stair1down = {1746, 1740};
	
	/*
	 * The stairIds where you should have the choice to increase or decrease your height level.
	 */
	public int[] stairChoice = {1748, 1739};

Notice stair1down, stair1up, and stairChoice. Here’s the translation.

stair1up - Makes the player go up.
stair1downp - Makes the player go down.
stairChoice - Sends a choice beofre going up.
[center]
How is this possible?[/center]

RandQm made it where stair1up and stair1down gets PosX and PosY and then sets the height level to 1. Or if it’s already 1, 2. Same with stair1down. Gets PosX and PosY and decreases height level by 1.
