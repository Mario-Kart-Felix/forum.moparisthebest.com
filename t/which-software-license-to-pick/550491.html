Which software license to pick

General Programming
zyle1992 August 4, 2016, 2:57pm #1

For my current project I’m working on, I want the initial public release to have a specific license, so that all updates going forward follow the same guidelines. But I’m not sure which license to pick. I’ve looked at the Unlicense, GPLv3, AGPLv3, CC-BY. They’re all kind of similar, but I’m just a little bit blurry on some details.

People can take my source code and modify it, distribute it, for commercial or non-commercial use. They attribute the original source to me, with (maybe) a link to the original repo. They don’t have to release their source code if they don’t want to. If they use someone else’s derived work that’s licensed, then they have to state what they’re using, as well as a link to their code, with attribution for them.

For example, my project , Bob takes , makes his changes to it, and distributes it under and open-source license and releases it publicly as . Joe, takes , modifies it for his server, but doesn’t want to make his source code public, because he’s got a feature that no other server has, and uses it to draw attention. However, Bob requires attribution for , so, while Joe doesn’t need to release his code, he does need to say “I got my code from Zymus’s , and using version 3 of Bob’s plugin”.

Is there something that kind of matches with this? Or is there some exclusive term that I need to modify.

EDIT: Actually, there’s not reason I should restrict how Bob can use his code, so disregard that part.

I personally prefer MIT/BSD licences over GPL/viral licences. Whatever you’re doing probably isn’t going to see that much use anyway, so why limit the interest further by applying something like the AGPL licence?

If you use GPL code in a server you do not have to release the source, only when you make your project available to the public to run for themselves.

The GPL boils down to, if people use GPL code in their projects, and release those projects to the public, the GPL code [1] must be released also.

[1](and any additions/modifications to the GPL code - depends on what you’re doing but licensing can get complicated quickly with multiple licenses)
