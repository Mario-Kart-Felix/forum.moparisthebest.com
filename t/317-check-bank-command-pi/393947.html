This command is set up to check banks of any existing player, regardless of whether or not they are online. It does so by checking players added to the PlayerHandler’s players array. (online players) If the player is online, it will simply grab their bank and toss it at you, and if they are offline, then it will load their bank from their character file.

First up open Player.java and search for this:

public int bankItems[] = new int[Config.BANK_SIZE]; public int bankItemsN[] = new int[Config.BANK_SIZE];

Below it add this:

public int oBankItems[] = new int[Config.BANK_SIZE]; public int oBankItemsN[] = new int[Config.BANK_SIZE];

This will allow you to temporarily store someone’s bank so that you can view it later.

You can close that up, we won’t be needing it any further. Now open Client.java and make sure you have java’s File and Scanner classes imported. They look like this:

import java.io.File; import java.util.Scanner;

If you don’t have it, just toss it in with the rest of you imports. Then add the following method:

public boolean loadCharBank(String name){ Scanner in = null; try{ File acc = new File("Data/characters/" + name + ".txt"); in = new Scanner(acc); while(in.hasNextLine()){ String nextLine = in.nextLine(); if(nextLine.contains("character-bank")){ String line = nextLine.split("=")[1]; line = line.trim(); String stuff[] = line.split(" "); int key = Integer.parseInt(stuff[0].trim()); int id = Integer.parseInt(stuff[1].trim()); int amount = Integer.parseInt(stuff[2].trim()); this.oBankItems[key] = id; this.oBankItemsN[key] = amount; }else if(nextLine.equalsIgnoreCase("[friends]")) break; } }catch(Exception e){ return false; } return true; }

What this does is use Java’s File class to load up your character’s file. Of course if you’ve changed the path to the character files, you should reflect that change in the path above. If the character file does not exist, Java will throw an exception, which will be caught by the catch method, obviously, which will return false. The boolean returns will be explained later. If the file exists, then we’ll use the Scanner class to read the file.

Now that we’re reading the file, we’re going to use what we know to read the relevant data we want. if we’re looking at a line with “character-bank” we must be looking at their bank, so we’ll get that data in. The code is pretty self explanatory at this point. We split the line by the = sign. What this does is return an array of strings where each index in the array is a part of the string that was split. For example, if we have
I went = to the = mall.
then split that by “=” we would get an array as such:
{"I went ", " to the “, " mall.”}

In our case, there’s only one equal sign, so there’s only two array indexes. We only need the second, as that’s the data we want, so we only store the second. Now as we also know, data in our char files are separated by the “space” you get by pressing tab, which is also /t in most operating systems. I used the space itself to avoid any problems. Once you split by that space, you get an array containing the key, or slot in the bank, the item id and the item amount. we’ll parse those to integers, and store them in our temporary bank. Once all that is finish, we return true.

Ok, now go to Commands.java and add this command (I’m fairly certain you know how to add commands if you’re looking at this tut)

if (playerCommand.startsWith("checkbank")){ boolean isOnline = false; String name = ""; try{ name = playerCommand.substring(10); }catch(Exception e){ c.sendMessage("You didn't enter a name."); return; } if(name.isEmpty()){ c.sendMessage("You didn't enter a name."); return; } Client c2 = null; for(Player a : PlayerHandler.players){ if(a != null){ c2 = (Client)a; if(c2.playerName.equalsIgnoreCase(name)){ isOnline = true; break; } } } if(isOnline){ c.oBankItems = c2.bankItems.clone(); c.oBankItemsN = c2.bankItemsN.clone(); c2 = null; }else{ if(!c.loadCharBank(name)){ c.sendMessage("Character not found."); return; } } int tempItems[] = c.bankItems; int tempItemsN[] = c.bankItemsN; c.bankItems = c.oBankItems; c.bankItemsN = c.oBankItemsN; c.getPA().openUpBank(); c.bankItems = tempItems; c.bankItemsN = tempItemsN; c.oBankItems = new int[Config.BANK_SIZE]; c.oBankItemsN = new int[Config.BANK_SIZE]; }

Ok so here we start off checking to make sure the inputted name is not blank. If it is, we’re pretty much done. Next thing to do is declare a new client, we’ll probably need it later. Now we start looping through each player. If that player is not null, then we cast it to our c2 client. Now we check that player’s name, versus the name given to us from the command. If they match, our player is online, so we can break out of the loop, and best of all, we won’t have to load it from the file later on. It’s also a good idea to have a boolean to know if we found him online or not for later =P

Ok, so now we check if the player is online, based on our boolean variable that we set earlier. If it’s true, it’s because we found him online, so we take our c2 Client (this is why we needed to declare the client outside of the loop, so that it could stay within scope for this moment) and we copy the bank array to our temporary copy bank arrays. Be careful when doing this, because simply using the assignment variable to do this is not enough. You MUST invoke the clone() method on any object that you want to copy in this manner (in this case, our array) because otherwise, all you end up copying is the reference to memory, meaning instead of editing temporary values, you’ll be editing the REAL thing. I’m sure you can see how that would be a problem in our specific example

Assuming our player was not online, we’re going to load the bank up using our method from before. If it returns false, it means the character wasn’t found most likely, so we let the user know. Perhaps he typed the name in wrong =P

Now that we have our temporary bank array loaded up, we make another temporary array, and assign our own real bank to it. In this case, we don’t need to clone() because we actually do want to pass the memory reference instead of a copy of the object itself. Doing this allows us to give our own real bank array its reference back later on. Now we give our real bank array the reference to the memory of the other player’s fake bank. Now that that is done, we open up the bank the way we would normally, and the server handles it as it normally would, except this time it’s loading the other person’s items, instead of your own

As soon as the open bank method is done, we’ll give our own real bank its original memory reference back, using our temp arrays from earlier, and we’ll completely destroy the fake bank by assigning it new memory. Fear not, Java’s garbage collection will take care of the crap we tossed out by doing this

That’s basically it. The only thing you might be wondering is, what if I try to withdraw something? Nothing happens, unless you have the same item as that person in your own bank, in which case, it will just withdraw it from your bank. Depositing will also deposit to your own bank.
