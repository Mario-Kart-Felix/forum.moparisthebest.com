Setsuna:

No clickingmost file. So I’m guessing they’re in the client.

Okay, then find case 185, and get the IDs of the emotes. Or as an easier way to debug the IDs, add this to case 185.

sendMessage("Action button ID: "+actionButtonId+".");

This will tell you what the ID of that button is, whenever you want to add a new ID, you would add a new case under the switch statement. As an example…

case 1: teleportToX = *; teleportToY = *; break;

Now replace the one with the emote button ID and the * with your desired coordinates.
