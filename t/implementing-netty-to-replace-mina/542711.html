cleako August 4, 2016, 1:06pm #20

Finally was able to play around with PRSC and see what makes it different from stock DarkQuest. It includes netty in place of mina, uses python for scripts instead of java, bsh, or scala, it has a connection throttle filter and automatic IP banning, utilizes SHA512 password encryption as well as a private salt for database security, and has a pretty startup within console for the LS and GS. Client has a built in hash checker for cache updates when not running localhost and added fonts for capcha. Active tile was removed. Everything else is content from what I see. Response time running localhost compared to stock DarkQuest is nearly identical. Start up time is considerably longer for the GS in PRSC with most diverted to the landscape and python scripts.
