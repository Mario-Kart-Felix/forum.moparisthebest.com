austinsobes August 2, 2016, 4:43pm #16

I recommend that you don’t start with any .NET languages. You’ll get used to it and then you’ll realize they are slow and garbage. You’re obviously knew. Grab a book on Java, C++, or Python. Java and Python are recommended languages to start with. C++ should be attempted after you know some good concepts.

Deitel’s books on C++ and Java are really good. I do not recommend Wrox. They say they are for beginners but use terminology that will lose you at sentence one if you are new. I have a book or two on Python, though I haven’t read them and cannot vouch for them. I don’t plan to read them either since Python is a beginners language(I’m aware that it’s a powerful scripting language. but C’mon. CAMOONNNNNNNEEEEE) I personally recommend Deitel’s book on Java. Java is a very powerful language that’s widely used and it’s multiplatform(Can be used all all OS’s). And that being said, it’s still a beginners language so it will be relatively easy for you to learn. it’s also based off of C++, which is based directly from C. So switching between these core languages is very simple and easy in the future.

Here are a few examples of the Hello World! application. C# is the only .NET language I included. If you honestly need to use Visual Studio(I do not recommend it whatsoever) then use C#.*

Java

class helloWorld { public static void main(String[] args) { System.out.println("Hello World"); } }

C++

[code]
#include

int main()
{
std::cout << “Hello World!” << std::endl;
}[/code]

Visual Basic (Might be incorrect)

Module helloWorld sub main Console.WriteLine("Hello World!") end sub end Module

C#(C Sharp)(Not including pre-generated using directives/declarations)

class helloWorld { public void Main(String[] Args) { Console.WriteLine("Hello World!"); } }

*I do not recommend Visual Studio because for one, it’s not multiplatform. .NET languages only work for Windows and only work if the person using your application has the .NET framework. While an IDE is recommended, I say don’t use one until you need to. It’s good practice to not use an IDE for small projects, this way you don’t rely on help. Use an IDE for huge projects like games and such. My personal favorite application to program in is either NotePad++ for windows, or Gedit for Linux.

I hope this helps. Email me or Private message or simply respond in this thread. Always happy to help people new to coding. Though I’m not an expert in all of these languages. I can set you on the right path to learning.
