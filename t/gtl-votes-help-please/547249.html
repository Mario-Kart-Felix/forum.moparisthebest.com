sk8rdude461 August 4, 2016, 1:52pm #2

Have you heard of MySQL? Yeah, that’s what you’re using.
Chances are your website has phpmyadmin and all that installed on the same server as your website, so the “Database address” will be “localhost”.
The database, you must setup yourself.

So in cPanel, find phpMyAdmin, there should be some database tools near it:

Open the first application. It’ll take you to a page with some (depending) databases and etc.
There will be a gray box that says "create a new database"
put a name in for the database, and then hit create.

After it creates, go back to that page. Scroll down.

There’s a section to "Add new MySQL users"
Create one.
remember the information, as you’ll be using it shortly.

Next, after the user is made. Add it to the database. On the same page, a bit under where you created the user is the “Add User To Database” section. First, select your user in the first drop down. In the second dropdown, select the database you just made.
Hit add, it’ll take you to a new page to select what permissions to give the user. Just give it all permissions.

Now you’ve got your database.
Time to use it.

Database username is the user you just created (Most webhosts add some form of a name extension to the beginning of the usernames/databases. Ex: my site adds “canyourc_” to the start of each, so if i made the account “admin” it would be “canyourc_admin”).
The password, is the pass for the user.

And of course, the database name is the database we just made.

Image issue:
Probably an issue with your callback links. Double check that they’re correct.

::claim issue:
Right now, your current error is because you didn’t edit the SQL information on your server. In Server.java, where you have the vote constructor (Something like “public static MainLoader vote = new MainLoader(DATABASE_IP, DATABASE_USER, USER_PASS, DATABASE_NAME);”) is your error.
You need to replace the information with the information you just got from your host (like localhost and etc, all the stuff we used for: http://imagizer.imageshack.us/v2/xq90/537/6KK3gL.png ).
EXCEPT the IP address will not be “localhost”.

This is why you need remote MySQL. And to answer your question on that. You buy the feature from your webhost. It is not a feature you can just download or install. They need to activate/allow it.
I can’t say how much it costs or anything, as it varies per host. But it will typically be included in any monthly plan.
