runescape_sucks August 4, 2016, 3:02pm #11

Here’s a snippet on a way it can work. However, this is limited by where your clipped, so you can’t click through walls you couldn’t already walk through. Also note, this is all server-sided.

Edit: if it was partially client-sided, you could probably get it to clip through walls.

[code=java]
int currentX = player.getLocalX();
int currentY = player.getLocalY();
int lastX = player.getNewWalkCmdX()[player.newWalkCmdSteps-1];
int lastY = player.getNewWalkCmdY()[player.newWalkCmdSteps-1];

		int teleToX = player.absX + (lastX - currentX);
		int teleToY = player.absY + (lastY - currentY);
		
		//put teleport method here

		player.newWalkCmdSteps = 0;
		player.getNewWalkCmdX()[0] = player.getNewWalkCmdY()[0] = 0;[/code]

That would go in your walking packet handler, at the end of where the walking is handled. If you want to limit it to just minimap clicking, check if it’s packet ID is 248.
