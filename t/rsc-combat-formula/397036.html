Here is a simulation of the combat formula that I have had for a while. I believe you can distinguish the parts of the formula from the parts of the simulation on your own.

public final class Combat {
     private static final int YOUR_ATTACK_LEVEL = 99;
     private static final int YOUR_DEFENCE_LEVEL = 99;
     private static final int YOUR_STRENGTH_LEVEL = 99;
     private static final int YOUR_HITS_LEVEL = 99;
     private static final int YOUR_AIM = 81;
     private static final int YOUR_ARMOUR = 200;
     private static final int YOUR_POWER = 81;
     private static final int YOUR_OPPONENTS_ATTACK_LEVEL = 99;
     private static final int YOUR_OPPONENTS_DEFENCE_LEVEL = 99;
     private static final int YOUR_OPPONENTS_STRENGTH_LEVEL = 99;
     private static final int YOUR_OPPONENTS_HITS_LEVEL = 99;
     private static final int YOUR_OPPONENTS_AIM = 81;
     private static final int YOUR_OPPONENTS_ARMOUR = 200;
     private static final int YOUR_OPPONENTS_POWER = 81;
     private static final int[] FIGHT_MODES = {1, 3, 0, 0};
     private int yourHits = YOUR_HITS_LEVEL;
     private int yourOpponentsHits = YOUR_OPPONENTS_HITS_LEVEL;
     
     private Combat() {
          
     }
     
     private void simulate() {
          while(yourHits > 0 && yourOpponentsHits > 0) {
               System.out.println("You hit " + getYourNextHit() + " on your opponent! Your opponent now has " + yourOpponentsHits + " Hits left.");
               if(yourOpponentsHits <= 0) {
                    break;
               }
               System.out.println("Your opponent hits " + getYourOpponentsNextHit() + " on you! You now have " + yourHits + " Hits left.");
          }
     }
     
     private int getYourNextHit() {
          return yourOpponentsHits - (yourOpponentsHits -= getNextHit(YOUR_STRENGTH_LEVEL, YOUR_POWER, YOUR_ATTACK_LEVEL, YOUR_AIM, YOUR_OPPONENTS_DEFENCE_LEVEL, YOUR_OPPONENTS_ARMOUR));
     }
     
     private int getYourOpponentsNextHit() {
          return yourHits - (yourHits -= getNextHit(YOUR_OPPONENTS_STRENGTH_LEVEL, YOUR_OPPONENTS_POWER, YOUR_OPPONENTS_ATTACK_LEVEL, YOUR_OPPONENTS_AIM, YOUR_DEFENCE_LEVEL, YOUR_ARMOUR));
     }
     
     private int getNextHit(int strength, int power, int attack, int aim, int defence, int armour) {
          return isNextHitValid(attack, aim, defence, armour) ? (int)(Math.random() * getMaxHit(strength, power)) : 0;
     }
     
     private int getMaxHit(int strength, int power) {
          return (int)Math.ceil(((1D * strength) + FIGHT_MODES[0]) * ((power * 0.00175) + 0.1) + 1.05);
     }
     
     private boolean isNextHitValid(int attack, int aim, int defence, int armour) {
          double percent1 = (attack + aim) / ((attack + aim + defence + armour) / 100D);
          double percent2 = (defence + armour) / ((attack + aim + defence + armour) / 100D);
          double random = Math.random() * 100D;
          return percent1 > percent2 ? (random > percent2 ? true : false) : (random > percent1 ? false : true);
     }
     
     public static void main(String[] args) {
          new Combat().simulate();
     }
}

Protosstribe August 1, 2016, 12:09pm #17

I believe the system is built in two steps. The first step validates whether your next hit will succeed or not. The second step calculates the actual hit. The actual hit may actually be totally random (or as random as the PRNG itself can produce). But the validation for each round is not random. It is based on percentage. This percentage must take into account the two opponents skill levels, equipment, prayer and potions (potions are just a multiplier on the strength level, but I’m not sure about prayer).

The “randomness” that people talk about, is because the calculation for the percentage does not take into account the whole fight. Only one round at a time. If you have a 50% chance to do thing “A” and a 50% chance to do thing “B”, you could still do thing “A” twice in a row without doing thing “B” once. Based on those observations, we would have a 100% chance to do thing “A” instead. But that’s not true.

In my combat formula (posted on the previous page), I made this second step produce a hit with a value of 0 or greater (which in fact would mean you hit, but your hit is so weak that you do no damage). It could be 1 or greater aswell. This is just a matter of taste.

My formula is based on a few principles, however. They are as follows:

  1. It should be as simple as possible. I doubt Jagex made an extremely advanced and complex formula just for this. It would probably slow it all down aswell (this may not even be a problem).
  2. Everyone should be able to hit everyone else, no matter their levels and equipment. I know for a fact that a level 3 character hit a 1 on my level 122 character once. So yes, I have seen it. This level 3 character would even be able to hit a 1 on my level 122 character twice in a row. The chance for that is incredibly low, but it is still possible.
  3. The more equal the two opponents total skill levels and equipment etc. are, the more equal chance they will also have. For instance, 50 attack against 50 defence would mean the same chance to hit for the attacker as it would be to block for the defender (provided that everything else is equal aswell).

Because it would make absolutely no sense if (for example) your opponents attack level were taken into consideration while calculating your next hit on him (or her for that matter). Prayer may of course also be used in the calculation. There are equal prayers for attack and defence, so they are opposites aswell.

Attack and weapon aim are opposites to defence and armour. Strength and weapon power are used while calculating your max hit (along with the strength multiplier for potions and something similar with strength-based prayers).

99 99 99 vs. 1 99 1 and 99 99 99 vs. 99 99 99: The first one (you) in both cases would have an equal chance to hit your opponents (if your equipment statistics were equal to your opponents aswell). The second one in the first case would obviously not hit very often on you, while the second one in the second case would probably be very equal to yourself.
