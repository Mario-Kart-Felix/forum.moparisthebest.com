sk8rdude461 August 4, 2016, 2:56pm #6

Do you only want a maximum of two people being connected to each other at a time?

If so, you could make a list of some sort that would hold the connected clients, and each connected client would have a “paring ID” which would be the ID of the person they’re chatting with. Set it to like -1 on default.

Then when a new client connects in, check for any other clients that have the paring id of -1, and if so, connect them.
If there is no one else available, send them into their own “lobby” to wait for a second person.
