Ben July 31, 2016, 10:48am #16

I don’t see the problem with setting the regex as an attribute of an XML element (apart from that I didn’t implement it as a feature within the namespace, but that’s a different topic.)

If someone wishes to change it so that it doesn’t check for email address validity, so be it – the server side script will block it anyway.
The UI features are for the users only, whatever happens when he sends the form for processing is not related to the UI whatsoever.

Look at it this way:
Human <–> Browser <–> Server

The human user is associating with the browser, and the browser is associating with the server.
When the human user attempts to enter an invalid email address, the browser notifies the human user and does not proceed with sending the data, knowing the server won’t accept it.
If the human user wishes to remove this check from the browser, so be it.

When he sends an invalid email address, the server will not accept it and throw him back to the same form he has entered.

This check happens at browser level, and then again at the server level.

Doing this is perfectly OK.
