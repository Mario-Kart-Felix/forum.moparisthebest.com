GalkonsPride July 31, 2016, 4:40am #24

Okay. Several flaws in this project of yours.

Animations: importing HD animations to a 317 client will be nearly impossible without rewriting the animation class. Each animation/frame is held in its own format, much like the 3d models in the client. They are all held in data files, but they are Jagex’s own format, therefor, we have nothing but our own minds to work with them.

Models: the models are written in a new model format commonly labeled as .mdl, short for model. It is easy to get a client to load the new model header, as it has been released. If you don’t have the release, just rewrite it yourself so it loads the polygons[triangles], colors, etc.

NPC’s: good luck with getting HD NPC’s working. 100% of the new NPC’s have animations not supported by the client, which IMPORTING the new animations would cause the client to crash. They also will be invisible without the new model header.

Items: about 90% of the items will not work in game because of several factors. One, is that if you don’t get the new model header working, you won’t get any items working. Another, is the animations they are given are different from the standard 317 animations. Also, almost ALL the models would be fucked if you are aiming for an HD client. Wanna know why?

Because HD loads new textures and details onto each individual model, making the detail of much higher quality. There are several classes in a 508+ client devoted to this factor only. A 317 client will not have the ability to load new textures of these details onto over 15000 models[more if you’re aiming for more models].

Between the factors of textures, animations, formats, you’re basically screwed unless you rebuild the client itself, which basically has no point.
