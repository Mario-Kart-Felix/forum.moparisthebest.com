Ecko August 4, 2016, 12:46am #5

The user in the error has over 11 characters, which is more than what is displayed when using the int(11) data type. Change 11 to a reasonable number like 30. Lots of people use varchar(255) to avoid these problems.

As for you hashing question, do you mean encode? Because there is no way to ‘decode’ or ‘decrypt’ md5(). If you want it to be stored in the database a sha-1, change the part in your registration script that uses md5() on the password to sha1().
