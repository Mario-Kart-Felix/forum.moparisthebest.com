QuestMaster_2K10 August 1, 2016, 12:18am #7

Okay listen up.

  1. The IP Address that the client should connect to should be programmed into the client. Doing this, you must find “Applet_sub1.java” and “client.java” in an rs2 client. The code can be changed, or added in those files so the client will know which IP the client will connect to.

  2. It will connect to the server, until it reaches the update server. Once it’s reached the update server, then it will know which deob the server is (ex: 317, or 508), if the deob is not the same as the client, then the client will will reach 2% on the loading screen and will no longer load but crash.

  3. If the server’s deob is the same as the client, then the client will load, bringing you to the login screen for you to input a username and password.

  4. Once you log in with a valid username or password, then the server will read the data being sent to the server, such as Connection IP, deob type, login type, and ecryption keys.

  5. If the ecryption keys, deob, and login type are satisfactory, then the player will then be successfully logged into the server.

That’s how connecting to a rs2-base server would be. For RSC however, the IP is programmed into the client, but must be accepted as a valid ip on the server. This is for security purposes for if someoone hacks their client, it would only be possible to connect to their server instead of others.
