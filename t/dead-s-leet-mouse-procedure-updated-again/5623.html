400 lines of code for a mouse movement is kinda nuts, but hey, it works, and it looks as realistic as anything. It accelerates, decelerates, changes direction slightly every so often, and gets pulled towards a random point to produce a curve. I have a feeling this scipt may be almost unbannable for mouse movements alone. Linear regression shouldnt work on it as its just too random and curves enough to evade it, and it changes speeds so a simple distance check to see if the distance between each movement is constant also wont work. As it is, i think all the bugs are fixed, but i could be wrong, plz tell me if you do find any.

btw, i want to make this an include, could someone tell me how to do that.

edit, ok i figured out how to make it an include, so pop it into your include directory and its deadmouse(targx,targy,speed,acceleration,turnspeed : integer; finishingtouch : boolean);

edit, ive now removed all the global varables, so it should be easy to implement into any script now. It still has no dependancies other than itself (lol), so everything should run seamlessly. ive also changed finishing touch to move mouse smooth as suggested, and added a random interval between each mosue movement. I have one last thing to add, and thats overshoot/waypoints. instead of going strait to the target, the mouse will stop once along the way, and also an overshoot procedure will be added which will cause the mouse to aim at a point within a box from the target, stop for a split second, and re aim back at the target.

edit, i found a bug which would sometimes cause the direction of the mouse to change in the wrong direction. After fixing this, i had a new bug, which wasnt really a bug, just a problem. On smaller mouse movements, the cursor would sometimes spiral the target. Ive fixed this as well. Ive also made two global variables, finishingtouch and overshoot. setting finishing touch to true will make sure that the mouse always hits the target exactly (not within 10 pixels), and overshoot causes the mouse to over shoot the target, stop, and reaim. It only happens 2 in 3 times, because sometimes a human would get it right, but usually they miss the target the first time.
