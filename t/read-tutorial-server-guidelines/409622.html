What this tutorial is meant to teach:
This tutorial is here to teach users on how to properly submit a useful tutorial. It has become the norm to simply post Copy & Paste snippets with small information for the users who are using the tutorial to “learn”. To put an end to this, I hope this tutorial can teach people on how to construct a proper tutorial for this section.

Please note that this tutorial/server guidelines only applies to RSPS only and should not be applied to anything else.

Before starting your tutorial please read this topic [size=32pt]HERE[/size] and the use the links within that thread and continue further reading.

What?! Why do I have to read so much stuff?
The answer lies simply in the fact that it will help you comprehend how your tutorial should be laid out, what it should contain, and how it should be written. If you cannot read a few simple topics, how could you even produce your own topic in a way that would allow others to gain knowledge or ideas from so they can understand it?

I don’t want to write a tutorial but I can’t find a tutorial for a specific source that I’m using?
Frankly, you shouldn’t be looking for tutorials on how to do something for a RuneScape Private Server if you can’t understand the JAVA programming language and how it works. Programming takes studying which means more reading!

I’d rather just take snippets of codes and bring up my server so I can start collecting donations
In all honesty, that kind of thinking is shunned by the more intelligent programmers around the world. Although it costs money to keep a dedicated server online, nothing should be done for the pure purpose of profit. However it is not illegal (per say) but it will not make you a better person overall and you won’t be learning anything out of it and you will continue to be as ignorant as you started. Not good right?

So if I want to write a useful tutorial or program a server, what should I do?
The number one thing you should do is find a programming language which you find is suitable for you. You can find one here but note that not all languages is meant for programming a server. If you want to narrow down the list of languages to research I can name a few that has been used to program a RuneScape Private Server over the years:

[ul][li]ruby[/li]
[li]JAVA[/li]
[li]C#[/li]
[li]C++[/li][/ul]

[size=5pt]note: I have only listed those that I’ve seen so far, if you know any more possible languages in which COULD/HAVE be/been used to program a server, tell me.[/size]

After you find the language you like, start learning everything you can and don’t stop!

I feel like I know as much as I can to start programming or writing tutorials, what should I do?
Well first should take a look this thread HERE. If you don’t understand anything written there, you probably shouldn’t start writing tutorials or programming a server yet.

I understood what was in the link, I started programming a server and it’s gone by well so far. I want to write a tutorial to contribute to the community, how do I do so?
Well if you’ve gotten this far, you’ll know that tutorials shouldn’t be written by the source you use. Everything you program (which is usually JAVA on this site) can be implemented into other sources and even other languages. If you don’t know how, you should stop programming right now and continue studying how to program with the language you chose and/or the language that the other source uses. If you do, then you should know that a tutorial is not only meant to teach someone how to do something but give someone and idea using the method you approached with that tutorial. The tutorial should NOT teach how to program something within a specific source. It should be detailed and documented regardless of method/variable names.

I think that sums it up, if there’s anything you think I should add to the tutorial, just tell me and I’ll see if it should go here.
