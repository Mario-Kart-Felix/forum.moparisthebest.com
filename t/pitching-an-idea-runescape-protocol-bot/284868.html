unlimitedorb July 31, 2016, 8:42am #13

Just because they modified the client and didn’t update the version, doesn’t mean the update was passive (“ninja update”). Users would still have to download the latest client. A packet bot would easily be able to detect whether or not it’s out of date.

A passive update requires that classes be defined and then loaded into the JVM or for classes to be dynamically redefined.

java.lang.instrument gives agents the ability to redefine classes on the fly, but we know the client doesn’t use agents so that’s a dead end (class redefinition is an optional JVM feature anyway).

The other option is for the client to load it using defineClass… they don’t do that either.

Autorune died because the authors didn’t review changes after an update carefully enough (or they didn’t fully understand the extent of the protocol in the first place)… not because of some nonexistent passive update feature.
