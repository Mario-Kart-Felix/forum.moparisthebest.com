[anchor=clickme]I. What is Data Encapsulation[/anchor]

Alright, this is kind of hard to explain. Data encapsulation, often reffered too as a wrapper or container, is pretty much a way to encapsulate your data. It’s pretty much packaging all of your seperated data into one Object. Data encapsulation is one of the four fundamental OOP concepts, and its ironic because data encapsulation was the first thing that I learned about Object Orientation. In chapter three I’m going to show you specific examples in winterLove where data encapsulation could be (should be…) utilized.

[anchor=clickme2]II. Why you’re better off using Data Encapsulation[/anchor]

[ul][li]Maintainability[/li]
[li]The fields of a class can be made read-only or write-only[/li]
[li]A class can have total control over what is stored in its fields[/li]
[li]Flexibility[/li]
[li]Store data in a more manageable way[/li]
[li]The users of a class do not know how the class stores its data. A class can change the data type of a field, and users of the class do not need to change any of their code.[/li][/ul]

[anchor=clickme3]III. Examples of how Data Encapsulation can be Utilized[/anchor]

Example one, global items:

public static int[] globalItemAmount = new int[15001]; public static int[] globalItemController = new int[15001]; public static int[] globalItemID = new int[15001]; public static boolean[] globalItemStatic = new boolean[15001]; public static int[] globalItemTicks = new int[15001]; public static int[] globalItemX = new int[15001]; public static int[] globalItemY = new int[15001];

globalItemController[i] = itemController; globalItemID[i] = itemID; globalItemX[i] = itemX; globalItemY[i] = itemY; globalItemAmount[i] = itemAmount; globalItemStatic[i] = itemStatic; globalItemTicks[i] = 0;

Seriously? That looks pretty redundant to me. Now, this is what we can do if we make a container class.

globalItems[i].setItemOwner(itemController); globalItems[i].setID(itemID); globalItems[i].setPosition(position); globalItems[i].setAmount(itemAmount); globalItems[i].setStatic(itemStatic); globalItems[i].setTicks(0);

A lot better… wouldn’t you agree? Don’t you think the object orientated way looks more… maintainable? We can edit a bunch of different values with one field!

Example two, player positions:

public int absX; public int absY; public int heightLevel;

This can pretty much be simplified into one field using data encapsulation…

[anchor=clickme5]V. Our very first Container Class[/anchor]

Lets find another specific example in winterlove where encapsulation should be used… how about items? What does one item consist of?

An item ID

And an item amount

So, create a new class and name it Item. Inside put

[code=java]/**

  • Holds data for an item

  • @author lare96
    */
    public class Item {

    /**

    • The item id
      */
      private int id;

    /**

    • The amount
      */
      private int amount;
      }[/code]

Now we need the constructor, and the getters n’ setters. This setters will help us set the variables in the new instance we create, and they will also help have total control over what is stored in our fields. You’ll see.

Add this constructor. This will basically allow us to instantiate the Item class and set the variables

/** * Constructs a new container with the specified data */ public Item(int id, int amount) { this.setId(id); this.setAmount(amount); }

Now for the setters… see how we can manage the input data? If someone inputs an id/amount below one, or over the max integer limit, it will change the data to something that won’t cause any errors.

[code=java] /**
* @param item
* the item to set
*/
public void setId(int id) {
if (id < 1) {
this.id = 1;
} else if (id > Integer.MAX_VALUE) {
this.id = Integer.MAX_VALUE;
}

    this.id = id;
}

/**
 * @param amount
 *            the amount to set
 */
public void setAmount(int amount) {
    if (amount < 1) {
        this.amount = 1;
    } else if (amount > Integer.MAX_VALUE) {
        this.amount = Integer.MAX_VALUE;
    }

    this.amount = amount;
}[/code]

Now for the getters, so we can retrieve the data!

[code=java] /**
* @return the item
*/
public int getId() {
return id;
}

/**
 * @return the amount
 */
public int getAmount() {
	return amount;
}[/code]

And that’s it…! Extremely simple. Your class should now look like:
