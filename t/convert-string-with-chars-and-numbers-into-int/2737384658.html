TeleNubby July 31, 2016, 7:01am #28

@Fireeater93, I’m assuming you don’t fully understand how the try&catch block works. So I’ll explain it to you:

The method [tt]parseInt(java.lang.String)[/tt] in the Integer class attempts to parse an integer value from the String provided. Now, you see, occasionally the input provided will not follow the expectations of a standard integer. Additional characters (i.e. ‘a’, ‘b’, ‘%’, "^’) will be thrown in rendering the integer invalid. When this occurs, an exception is thrown by the name of [tt]java.lang.NumberFormatException[/tt]; I’m sure the name is self-explanatory. Using the aforementioned process, one is able to identify the occasional invalid integer and return some sort of response if wanted.
