Cres August 4, 2016, 5:52pm #8

For players, the list of players to update is changed when a client sends the region change packet. I’m still kind of confused because I lack some information about regions and player updating in general.

A region is a sort of temporary data container for entities (GroundItems, TemporaryObjects, Players, NPCs)
If a region no longer has any entities in it, it may destruct (unless it is persistent, which means that the region holds data such as static object spawns or tile events which we don’t want to load/unload every time a region is created)

EDIT: as for the 4x4 chunks that make up the region, all the entities are stored per chunk. A chunk’s only purpose is to allow for controlled updating of entities when there’s too much load on the region. It would first start with the chunk the player is in, then adjacent chunks, then diagonal chunks, etc…

Wrote a quick implementation of a Quad Tree to see what the results would be and I’m kind of amazed.

The task was to retrieve all the other objects around ‘our’ object in a 32 tile square around it.
2000 objects were randomly generated in a (2^14)^2 2d area (to simulate the player updating process on a full server)

The time to lookup all objects in range is almost always around 100ms less for the quad tree.
Not only that, the standard way to lookup objects in range takes a whopping ~125ms on average to complete.
This is approx 1/5 of the cycle time of a rs2 server.

[code=java]import java.awt.Rectangle;
import java.util.ArrayList;
import java.util.Random;

import quad.QuadTree;
import quad.QuadTreeObject;

public class Main {

public static void main(String[] args) {
	final int WIDTH = 16384;
	Random r = new Random();
	//construct the population
	QuadTreeObject[] objects = new QuadTreeObject[2000];
	for (int i = 0; i < objects.length; i++) {
		objects[i] = new QuadTreeObject(r.nextInt(WIDTH), r.nextInt(WIDTH), new Object());
	}
	QuadTree tree = new QuadTree(16, 100, WIDTH);
	for (QuadTreeObject o : objects) {
		tree.insert(o);
	}
	Timer timer = new Timer();
	//quad test
	for (QuadTreeObject o : objects) {
		tree.retrieve(new Rectangle(o.getX()-16, o.getY()-16, 32, 32));
	}
	long el1 = timer.elapsed();
	timer.reset();
	//normal test
	for (QuadTreeObject o : objects) {
		ArrayList<QuadTreeObject> results = new ArrayList<QuadTreeObject>();
		Rectangle search = new Rectangle(o.getX()-16, o.getY()-16, 32, 32);
		for (QuadTreeObject x : objects) {
			if (search.contains(x.getX(), x.getY())) {
				results.add(x);
			}
		}
	}
	long el2 = timer.elapsed();
	
	System.out.println("Quad speed: "+el1);
	System.out.println("Normal speed: "+el2);
}

}[/code]
