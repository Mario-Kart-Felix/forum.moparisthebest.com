Well, there is an easier way of doing that. You can use java.lang.reflect.Method to store declared methods you wish to invoke. But unless you store the instances of classes you wish to call instance methods from, you need the invoked methods to be static.

For example, you could do the following:

import java.lang.reflect.Method;
import java.util.Hashtable;

public class ButtonExecutor {

	private ButtonExecutor() { // allow no instances of this class
	}

	private static Hashtable<Integer, Method> assignments;

	public static void registerButton(int buttonId, String methodName, Class<?> methodParent) throws Exception {
		Method method = methodParent.getDeclaredMethod(methodName, new Class<?>[0]);
		assignments.put(buttonId, method);
	}

	public static boolean executeButton(int buttonId) throws Exception {
		if (!assignments.containsKey(buttonId)) {
			System.err.println("No assignment for button " + buttonId + ".");
			return false;
		}
		assignments.get(buttonId).invoke(null, new Object[0]);
		return true;
	}

	static {
		assignments = new Hashtable<Integer, Method>();
	}
}

An example of using this class:

public void handleButtons() {
	int buttonId;
	// ...
	try {
		boolean status = ButtonExecutor.executeButton(buttonId);
		// ...
	} catch (Exception e) {
		e.printStackTrace();
	}
}

public void registerButton(int buttonId) {
	ButtonExecutor.registerButton(buttonId, "handleButton", ButtonHandler.class);
}

ButtonHandler.java:

public class ButtonHandler {

	public static void handleButton() { // must be static and have no arguments; return type does not matter
		// ...
	}
}

java.lang.reflect.Method represents a physical method inside of a class file, such as my[tt]handleButton()[/tt] above. Calling ButtonHandler.class accesses the public java.lang.Class of the specified class file. java.lang.Class is a reflective representation of a Java class file. You can use java.lang.Class to get the declared fields and methods of a Java class file, as well as access their values and invoke them.

When accessing a reflective java.lang.reflect.Field or java.lang.reflect.Method, you must provide an instance for it, unless it‚Äôs static. Take a look at the following Java code:

ButtonHandler.handleButton();

The above code can be achieved by doing the following:

Class<?> buttonHandlerClass = ButtonHandler.class; // represents the ButtonHandler class
Class<?>[] argumentTypes = new Class<?>[0]; // this represents the Java class types of the arguments of the method, such as String.class; this array is empty because our method has no arguments
Method handleButton = buttonHandlerClass.getDeclaredMethod("handleButton", argumentTypes); // retrieves the 'handleButton' reflective representation for the button handler class
Object[] arguments = new Object[0]; // these are the respective arguments to the argument types specified above. For example, if the method had a String argument, then this array must contain a String object respective to the String.class type in the argumentTypes array. Note that this array is also empty because our method has no arguments.
Object returnVal = handleButton.invoke(null, arguments); // calls/invokes handleButton() from a static context (the null argument denotes a static context). If you would like to access a method from a virtual context, you can do so by providing an instance to that class in place of null (an instance such as 'Object someObject = new Object()', where someObject is the instance
// note that 'returnVal' is the value returned by handleButton. A value is always returned, even if the method invoked has the return type of void
