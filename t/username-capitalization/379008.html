Everytime a new player logs in, a new character file is created. But, every character that was suppossed to be capitalized is lower case.

For example: “Mod Hershey” is shown as “mod hershey” in the character file.

So, when that player uses the clan chat or yell, it shows the name in lower case and looks bad.

Does anyone know how to get the server to save character names as they were originally typed in when they first logged in?

EDIT: I went into the character file and changed the name of the file and the “username” part of the file to the capitalized version, “Mod Hershey”. But when I log in, it still says “mod hershey has logged in.” and any other messages or anywhere it uses the username its lower case even though the character file is capitalized.

rova1 August 1, 2016, 7:02am #2

Um, if you’re using PI then open Rs2LoginProtocolDecoder.java (…PI\src\server\net) and import this if it isn’t there already:

import server.util.Misc;

Then find:

cl.playerName

and modify like this:

cl.playerName = Misc.optimizeText(name);
