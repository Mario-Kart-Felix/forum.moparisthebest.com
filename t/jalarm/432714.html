JAlarm is an alarm application written in Java. The application utilizes JLayer for mp3 integration and audio support. All other content written or used within the application are designed solely by the Arsenal Development Team, lead programmer Emily (myself). The program uses a decent usage of multi-threading along with both basic script, and sound design script support.

Download Link: JAlarm.rar

Application Design Script:
The script is very basic and limited to RGB color values in an external file.
The line numbers in your script file should pertain as follows:
Index Listings:
(Line 1 = First line of File)
(Format for colors is “0,0,0” (ignore quotes))

1 - Description/Title of Script
2 - Frame Background
3 - Month box background
4 - Day box background
5 - Hour box background
6 - Minute box background
7 - AM/PM box background
8 - Year box background
9 - View Button Background
10 - Informational Button Background
11 - Alarm Button Background
12 - Reminder Text Field Background
13 - Current Hour Box A Background
14 - Current Hour Box B Background
15 - Current Minute Box A Background
16 - Current Minute Box B Background
17 - Current AM/PM Box Background
18 - Color Symbol Box Background
19 - Current Hour Box A Text Color
20 - Current Hour Box B Text Color
21 - Current Minute Box A Text Color
22 - Current Minute Box B Text Color
23 - Current AM/PM Box Text Color
24 - Colon Box Text Color

The alarm supports multiple alarm tasks but you should not abuse it and try not to keep the application running for long periods such as weeks, months, years. The application is in a @version 1.0.0 release, and needs users to submit bug and glitch reports. Task cleanup was removed on concurrent processing problem, so memory usage consumed by growing HashMap is notable.

Do not solely rely on this program as your alarm application. It has not been tested on a level to provide an accurate judgement on its reliability

The application will (is supposed to) open the alarm, regardless as to if the frame is minimized or not. The application must be running for the alarm to work.

Application Initial Launch Appearance (Using BlackSkin)

Please remember to click close and do not exit with the red X.
NOTE Do not use the remove function or you will get concurrent processing errors.

Both dismiss and snooze functions should work appropriately.
