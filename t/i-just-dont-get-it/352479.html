Ok, A source is the actual files to the game like where object and npc are placed in the game and where they are placed and what they do.

Client is the thing you use to play the game on, it has all the customs (things originally not in that version of RuneScape or not even in RuneScape at all) and with out the client your game could be all glitched up and wont be able to see items that will be a little dot called a Null which on the ground is a big head.

The compile saves everything you changed in the .java files and puts them into the .class files which your source/client read the .class files. Example you add a new object in the game using client.java and save it, if you don’t compile it will not be changed in game. Don’t edit the class files unless you know what you’re doing.

When you run the server by using Run.bat or Runserver.bat you can play on it with your client without making it online, only people on your computer can play on it as long as you set the ip to 0.0.0.0 or 127.0.0.1 by changing this in the Gui.java in the client files and then compile.

To get a nice source and client go to server downloads and find a nice server you’ll like to spend your time playing.

Hope this helps you!
