This tutorial will teach you how to add almost full Runecrafting to your server. This doesn’t include the portals to get to the altars, or the abyss. It includes full rune multipliers for which depend on what Runecrafting level you have (E.G. 11 runecrafting enables double air runes).

NOTE: I haven’t tested this yet, though it compiles with no errors for me in my server, so I doubt it will create any errors for you (Unless you add anything incorrectly).

Most of the things explained in this thread are already commented within the Runecrafting class.

Difficulty: 1/10 - If you follow the tutorial step by step, reading each note, you will be fine.

Let’s get started.

Step one

This is the Java class which contains most of the code which will enable Runecrafting for your server. Save it as ‘RuneCrafting.java’.

[code]package net.iClarity.model.Skills;

import net.iClarity.model.ClientManager;

/**

  • Class to manage the skill: RuneCrafting.

  • @author Primadude
    */
    public class RuneCrafting {

     /**
  * Rune essence ID constant.
  */
 private static final int RUNE_ESS = 1436;

 /**
  * Pure essence ID constant.
  */
 private static final int PURE_ESS = 7936;

 /**
  * An array containing the rune item numbers.
  */
 public int[] runes = {
         556, 558, 555, 557, 554, 559, 564, 562, 561, 563, 560, 565
 };

 /**
  * An array containing the object IDs of the runecrafting altars.
  */
 public int[] altarID = {
         2478, 2479, 2480, 2481, 2482, 2483, 2484, 2487, 2486, 2485, 2488, 2489
 };

 /**
  * 2D Array containing the levels required to craft the specific rune.
  */
 public int[][] craftLevelReq = {
         {556, 1}, {558, 2}, {555, 5}, {557, 9}, {554, 14}, {559, 20},
         {564, 27}, {562, 35}, {561, 44}, {563, 54}, {560, 65}, {565, 77}
 };

 /**
  * 2D Array containing the levels that you can craft multiple runes.
  */
 public int[][] multipleRunes = {
         {11, 22, 33, 44, 55, 66, 77, 88, 99},
         {14, 28, 42, 56, 70, 84, 98},
         {19, 38, 57, 76, 95},
         {26, 52, 78},
         {35, 70},
         {46, 92},
         {59},
         {74},
         {91}
 };

 /**
  * Checks through all 28 item inventory slots for the specified item.
  */
 public boolean itemInInv(ClientManager p, int itemID) {
         for (int i = 0; i < 28; i++) {
                 if (p.playerItems[i] == itemID + 1) {
                         return true;
                 }
         }
         return false;
 }

 /**
  * Replaces essence in the inventory with the specified rune.
  */
 public void replaceEssence(ClientManager p, int essType, int runeID, int multiplier) {
         for (int i = 0; i < 28; i++) {
                 if (itemInInv(p, essType)) {
                         p.deleteItem(essType, i, 1);
                         p.addItem(runeID, 1 * multiplier);
                 }
         }
 }

 /**
  * Crafts the specific rune.
  */
 public void craftRunes(ClientManager p, int altarID) {
         if (System.currentTimeMillis() - p.runeCraftDelay < 1000) {
                 return;
         }
         int runeID = 0;

         for (int i = 0; i < this.altarID.length; i++) {
                 if (altarID == this.altarID[i]) {
                         runeID = runes[i];
                 }
         }
         for (int i = 0; i < craftLevelReq.length; i++) {
                 if (runeID == runes[i]) {
                         if (p.playerLevel[20] > craftLevelReq[i][1]) {
                                 if (p.playerHasItem(RUNE_ESS) || p.playerHasItem(PURE_ESS)) {
                                         int multiplier = 0;

                                         for (int j = 0; j < multipleRunes[i].length; j++) {
                                                 if (p.playerLevel[20] > multipleRunes[i][j]) {
                                                         multiplier = multipleRunes[i][j] + 2;
                                                 } else {
                                                         multiplier = 1;
                                                 }
                                         }
                                         replaceEssence(p, itemInInv(p, RUNE_ESS) ? RUNE_ESS : PURE_ESS, runeID, multiplier);
                                         p.runeCraftDelay = System.currentTimeMillis();
                                         p.startAnimation(791);
                                         p.frame174(481, 0, 0);
                                         p.gfx100(186);
                                         return;
                                 }
                                 p.sendMessage("You need to have essence to craft runes!");
                                 return;
                         }
                         p.sendMessage("You need a Runecrafting level of "+ craftLevelReq[i][1] +" to craft this rune.");
                 }
         }
 }

}[/code]

Add this class in with your other classes. You will have to edit the packaging to your own server’s custom packaging. If your server is packaged, then remove the package declaration completely. The import of the ‘ClientManager’ class is the equal to the ‘client’ class in most Winterlove-based servers, so change that to whatever yours is named. Again, if your server isn’t packaged, then remove the import.

Step two

I will now explain each part of this class to give you a better understanding of what is contained in this class.

Firstly, the class is packaged, other classes are imported (ClientManager) and the class is declared. This doesn’t really need any in-depth explanation; you can research what each of those terms mean if you do not understand (Package, import and declaration), though, you should know what they mean if you are serious about programming your private server/other Java applications.

Our first encounter within this class is two constant variables:

[code] /**
* Rune essence ID constant.
*/
private static final int RUNE_ESS = 1436;

    /**
     * Pure essence ID constant.
     */
    private static final int PURE_ESS = 7936;[/code]

These are basically an easy way to declare the integers ‘1436’ and ‘7936’, because you can remember ‘RUNE_ESS’ and ‘PURE_ESS’ a lot easier. They are declared final which means these variables cannot be modified.

The next thing we notice in this class is two arrays:

These contain IDs which are used further on in this class/tutorial. The first array, ‘runes’, contains the IDs of the runes that are available to be runecrafted. The second array, ‘altarID’ simply holds the object IDs for the altars that you use to craft the desired runes.

2D arrays are used when you want to hold more than one aspect in an array. In this case, the first array, ‘craftLevelReq’ holds the rune ID data and the levels required to craft those runes. The second array contains only the multiple rune levels, but they are set out in the order of the ‘runes’ array, which makes it a lot easier to use.

The first method, ‘itemInInv’, checks through all 28 slots in the player’s inventory for the specified item (in this case, essence).
The second method, ‘replaceEssence’, replaces all essence in the inventory with the specified rune. The amount of runes given depends on what the multiplier is (the multiplier depends on your runecrafting level).

This method uses all the other variables, arrays and methods to do it’s job; to craft the specified runes. The following happens when you runecraft: the player displays the runecrafting graphics; the player begins the runecrafting animation; the runecrafting sound is played to the player through the client; exp is added to the player’s runecrafting skill; the essence is replaced with the rune they are crafting.

Step three

Firstly, we need to add a Singleton to prevent us making the methods within the Runecrafting class, static, and to use Java conventions. In your Client class (or equivalent class), add this variable and object:

[code]/**

  • The RuneCrafting instance.
    */
    private RuneCrafting runeCrafting = new RuneCrafting();

/**

  • The RuneCrafting Singleton.
    */
    public RuneCrafting getRuneCrafting() {
    return runeCrafting;
    }[/code]

We now need to make it so when you click on an altar, is crafts the runes.

For those who have their packets in separate classes, add this there and change it to make it work for you.

In the ‘parseIncomingPackets’ method in your Client class, or the equivalent class, under:

int objectY = p.inStream.readUnsignedWordA();

Add this piece of code:

for (int i = 0; i < getRuneCrafting().altarID.length; i++) { if (objectID == getRuneCrafting().altarID[i]) { getRuneCrafting().craftRunes(this, objectID); } }
