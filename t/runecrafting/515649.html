Normally it’s structured something like so:

if(c.hasReqs){
     //Do this
}else { //This means if they don't meet the requirements above
     c.sendMessage("Some sort of fail printout here.");
}

But before it gets to that, there’s usually a check to see if they player has the item.

if(c.getItems().playerHasItem(RUNE_ESS_ID)){
     //Do stuff
}else { //Again, only happens if they don't meet reqs above
     c.sendMessage("You're missing item XXXX");
}

All together:

if(c.getItems().playerHasItem(RUNE_ESS_ID)){
     if(c.hasReqs){

     }else { //This means if they don't meet the requirements above
          c.sendMessage("Some sort of fail printout here.");
     }
}else { //This shows when you don't meet the main res (not having item(s))
     c.sendMessage("You don't have item XXXX.");
}
