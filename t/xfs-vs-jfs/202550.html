Here’s what I’ve found out:
XFS:

  • Only Linux filesystem with online defrag tools.
  • Very fast. Good with large files. Bad for small files.
  • Can’t shrink filesystem.

JFS:

  • Like XFS, but less CPU intensive.
  • No defrag tools.

http://www.tldp.org/LDP/sag/html/filesystems.html:

Modern Linux filesystem keep fragmentation at a minimum by keeping all blocks in a file close together, even if they can’t be stored in consecutive sectors. Some filesystems, like ext3, effectively allocate the free block that is nearest to other blocks in a file. Therefore it is not necessary to worry about fragmentation in a Linux system.

ZFS > Most

I use ZFS on a open solaris box.

Good:
Awesome feature set, multiple expandable file system by just adding drives, self-healing
gzip compression option on file systems.
Mature Solaris Kernel
Free
Cheap alternative to other similar hardware options
load balances devices

bad:
Not a mature file system, relativly new.
Hard to interactive with from a Windows Machine at this time.
Not many users.
Clustering not as good as it can be.
Eats alot of RAM.
Uses alot of CPU. not so much imo
No marketing.
Needs to be Scrubed sorta like

Has alot other features / support that are more for techies but not for me
cache devices,
usb thumbdrives / hard drives
seperate logging devices
