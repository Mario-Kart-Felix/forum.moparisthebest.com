try and remember that game development isn’t just something you can fly straight in to, a good game developer usually knows a fair amount of maths as well as programming related things. i’d recommend you start off learning/reminding yourself of things like trigonometry as it’s basically essential to figuring out things like collision and distancing etc. teach yourself to use a good library for graphics and learn about software design and data structures (basic ones as well as sorting and (graph?) searching algorithms) it helps to learn about graphics too, ie. what an alpha level is, rotation algorithms, etc.

when you’re ready to move into 3D you’ll want to learn good amounts of linear algebra, calculus and even more comp-sci/programming things but with the latter it’s more a case of “you’ll learn it when you need it” rather than a bunch of prerequisites however knowing how matrices work and such is going to be a definite prerequisite in the 3D world.

Ah yes, I’m very familiar with DGIJ. I’ve reworked with several copies of his software engine, eventually building my own with some elements very closely based on his book. Nevertheless, I would not recommend it because it teaches people how to build a software 3d engine, and does a terrible job at doing so. His engine is rather slow and there is little practical use nowadays for a software engine.

It’s great for learning purposes, but not really for game development. I’m definitely planning on posting my own software engine in the future on these boards, but as a mere learning exercise. If someone wants to make a serious 3d game in Java, I would recommend using JOGL or LWJGL for starters, and maybe even a 3d engine like Xith3D or JMonkeyEngine.
