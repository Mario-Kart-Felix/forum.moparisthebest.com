[code=java]import java.util.Arrays;

public enum BarrowsItem {

DHAROKS_HELM(25000, 0000, 0001, 0002, 0003, 0004),
DHAROKS_BODY(25000, 0000, 0001, 0002, 0003, 0004),
DHAROKS_LEGS(25000, 0000, 0001, 0002, 0003, 0004),
DHAROKS_AXE(25000, 0000, 0001, 0002, 0003, 0004);

private int costPerLevel, fixedForm;
private int[] degradedForms;

BarrowsItem(int costPerLevel, int fixedForm, int... degradedForms) {
	this.costPerLevel = costPerLevel;
	this.fixedForm = fixedForm;
	this.degradedForms = degradedForms;
}

public BarrowsItem getForDegrade(int id) {
	for(BarrowsItem b : BarrowsItem.values()) {
		if(b.degradedForms != null &&
				Arrays.asList(b.degradedForms).contains(id)) {
			return b;
		}
	}
	return null;
}

}[/code]

The enum would store all the armour ids, the prices, and the fixed versions. You grab the data based on the id the player used (Whether they used it on him, or they went through a dialogue or whatever), then multiply the cost based on the (index + 1) of the id they used (So if I used 0000 it’d be 25k, if I used 0004 it’d be 100k).
