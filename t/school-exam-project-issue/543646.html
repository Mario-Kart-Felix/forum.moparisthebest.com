lothy August 4, 2016, 1:13pm #2

The solution to this can range from extremely simple (i.e., a two table solution that maps a student to zero-to-many answers) to comparatively complex (i.e., a larger database solution with tables that allow many students to sit many exams, with each question being defined in the database, each answer for those questions, etc).

Look up ER (entity relationship) diagrams on wikipedia and start thinking about the entities that you would need for your solution. Do yourself a favour and use crow’s foot notation - Chen notation takes up too much space.

And when you’ve done that, look into the concept of normal form. Brotip: nine in ten databases used in production software (i.e., real software with real users) will generally only go as high as 3NF. So if you aren’t overly interested in the university-level academic side of things yet then you can stop reading at 3NF.

Oh, and if you want a diagramming tool that’s free then I suggest yED (found here: http://www.yworks.com/en/products_yed_about.html).

This is how I (personally) would accomplish this issue. If the results are going to be read by you or school administrators, here’s one option…

After all the information has been inputted, on the process page you build a long paragraph containing all the answers.

An example of this would be a database structure like this…
id - information
1 - First Name: JohnLast Name: DoeQuestion 1: Yes

Etc. So when you’re later pulling the information, you can just select the one column as a whole and output it. This wouldn’t be ideal if you want to access the answers individually or pull the answers.

Another option would be building an array from the database, which isn’t really recommended. An example:
id - results
john, doe, yes […]

Then use PHP to separate values.

Without knowing whether or not the questions are just yes/no, involve multiple options, or include text, I wouldn’t be able to provide much support.
