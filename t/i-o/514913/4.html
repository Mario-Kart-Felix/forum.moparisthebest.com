Yesteryear August 4, 2016, 6:42am #4

IO stands for Input/Output. With IO you can read or write bytes, shorts, int or longs from or to streams. A (data)stream is a connection to a file or a (networking) socket.

A byte is a primitive data type that is made up of 8 bits. A short is 16 bits, an integer is 32 bits and a long is 64 bits.

Java has a lot of classes to use IO. For example a FileInputStream is an IO class to read from a file. With FileOutputStream you can write to a file.

The methods you posted are methods that write to a buffer in the same way OutputStreams writes to a stream. After you completed the data packet (a message over network) you write the buffer using the flush() method. (note that these methods are from the RSPS, and not from the Java api)
