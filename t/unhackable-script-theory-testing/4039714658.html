unlimitedorb August 1, 2016, 1:13pm #34

Any program stored on the client side is “hackable.”

Jar file is stored on the server? Not hackable. Why? Because you don’t have access to it. It’s possible to gain access to it by exploiting a vulnerability in the server setup, though.

Client requests a jar file using the latest asymmetric military grade encryption? Hackable. Why? As soon as it’s sent over the wire and arrives on your computer it’s on the client side and, therefore, it’s hackable. Just look for a call to defineClass or however else it could be loaded into memory. Even if it’s using a PKI, the client will have the public key and, with it, the ability to decrypt anything the server encrypted. Will it take time to do this? Yes. Is it worth the hacker’s time? Maybe. Is there anyway a client side program can be made completely secure? Not that I know of.
