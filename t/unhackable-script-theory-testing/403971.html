lothy August 1, 2016, 1:09pm #11

At first I read this and thought you were literally talking about being a third party and still obtaining the encrypted data, but then I realised that you were talking about compromising an application running on your own system.

For the record, if you were trying to identify the paths as a true third party using the man-in-the-middle attack, it would be futile.
With a key exchange algorithm (diffie-hellman, or the equivalent elliptic curve algorithm) the data would be secure enough from the likes of virtually anyone; if he hardened it further so that each path was sent with its own key then he would also achieve perfect forward secrecy – so cracking the key for one of the encrypted paths would take ages, but that key would not compromise the security of every other path.

tl;dr: Your friend may be naive, but you’ve oversimplified the process yourself and aren’t much better.
