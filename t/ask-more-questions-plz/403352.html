Jython August 1, 2016, 1:03pm #8
T4_:

What is the point in having custom data types (SMART, specialA/C, etc)?

smart is interesting
the way it works is by using a size flag bit to determine the bit width, either a 7- or 15-bits
the flag is the MSB and if isn’t set it reads 7-bits but if it is it reads 15-bits
the - 32768 is to strip the flag bit from the output (100000000000000 = 2^15 = 32768)
the reason one version uses - 64 and - 49152 is because it uses a two’s complement system for signed integers where the MSB of the 7- or 15-bit integer is a sign flag

for the other “custom data types” (- 128, 128 -, and -) it’s most likely a form of obfuscation
