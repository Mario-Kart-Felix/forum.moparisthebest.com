An “entity system” should use a single (abstract) entity and could contain abstract data. You would extend the actual entity (ex: an NPC) from the abstract entity class.

Example:

    public abstract class Entity {
     
            protected final String name;
            protected float x, y;

            public Entity(String name, float x, float y) {
                this.name = name;
                this.x = x;
                this.y = y;
            }

            /*
             * Gets x coordinate of entity.
             */
            public float getX() {
                    return x;
            }
     
            /*
             * Sets x coordinate of entity.
             */
            public void setX(float x) {
                    this.x = x;
            }
     
            /*
             * Gets y coordinate of entity.
             */
            public float getY() {
                    return y;
            }
     
            /*
             * Sets y coordinate of entity.
             */
            public void setY(float y) {
                    this.y = y;
            }
     
            /*
             * Gets name of entity.
             */
            public String getName() {
                    return name;
            }
     
            /*
             * Sets name of entity.
             */
            public void setName(String name) {
                    this.name = name;
            }
    }
public class Player extends Entity {

    public Player(String name, float x, float y) {
        super(name, x, y);
        onLogin();
    }

    // login stuff or something..
    private onLogin() {
        // you can access the variables from the Entity class
        System.out.println("Entity '"+getName()+"' spawned at x="+getX()+", y="+getY());
    }

}

Hope this helps. Good luck with your project(s)
