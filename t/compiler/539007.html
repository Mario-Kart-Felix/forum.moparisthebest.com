sk8rdude461 August 4, 2016, 11:33am #4

I can’t possibly wonder why a compiler tailored for a packaged source wouldn’t work on a client. Maybe because the client has no packaging, and that it has like 2 folders with java files.

@echo off title Building Files javac -d ./bin/ src/*.java src/sign/*.java title DONE pause
Make sure all your .java files are in a folder called “src” and then that in src there is a folder named “sign” and in said folder is signlink.java
Then make sure you have a “bin” folder in the same directory as the src folder (don’t put the bin folder in the src folder).
Lastly make sure your compiler is in the same directory as the other 2 folders.

@echo off Title Client cd ./bin/ START javaw -Xmx1024m client 30 0 highmem members 32 exit
There’s a run for your client as well.
