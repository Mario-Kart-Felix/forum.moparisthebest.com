I think it would come down to complexity then. You would need to rasterize the model with the graphics approach in order to know the region in which a mouse click could occur. I’d imagine it would be a lot easier to just make opengl do this implicitly, and use a basic screen color searcher for the actual action. There are also subtle differences between opengl implementations as to exactly how the rasterization is going to occur. You can never be certain if a custom rasterization method will produce the exact same region as opengl’s (and, of course, if it doesn’t, you have the potential for eventually missing the target in some instances).
