Your code would have to look like this:

if (command.equalsIgnoreCase("100kills")) {
if (c.killCount >= 100) {
c.sendMessage("Congratulations on your 100th kill!");
c.getItems().addItem(itemId, amount);
} else {
c.sendMessage("You need at least 100 kills to use this command!");
return;
}

As for the variable killCount, you will need to make sure the variable is being declared and initialized properly; for example, you need to make sure that you find the NPC death method and set the variable to increase (c.killCount++) every time an NPC is killed, so the variable is being stored. You need to also make sure you are saving the variable’s value to the player file so that it can be initialized back into the previous value as before once the server is restarted.

Edit: If you have done the following, I can only suggest you check the control flow of your if-statement, because if it isn’t being done properly it might not check for that variable’s value until AFTER the command has been executed, which sounds highly improbable due to the example you provided, but it is the only thing I can consider a possibility.

i also messed up my code, by not including equals operators.

however, useful things to know for commands:

equalsIgnoreCase for all commands unless parameters are required. do not use equals, its stupid.

do not use unnecessary loops, and don’t execute the code inside the loops.

loop through actual players/data - not random numbers of iterations.

always check for space before adding items.
always check for items before deleting them.
always check for deleting the item before modifying data.
always check for data before adding items.

etc etc.
