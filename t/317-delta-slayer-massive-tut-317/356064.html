Adding the slayer tasks.

Go to client.java
do ctrl + f and search lowtask OR (if that doesn’t work) task
wait until you see something like this:

 public int[] lowTask = { 90, 1616};
 public int[] mediumTask = { 90, 1624};
 public int[] highTask = {1624, 1615};
 public int taskID = -1;
 public int taskAmount = -1;

Explanation:
the tasks are basically what the player gets set…pretty simple…there are 3 different task types NORMALLY but you can create your own. I will show you how to at the end (in the extras).

Now as you probably guessed the numbers in the { } bits are the creatures you get set as slayer tasks.

public int[] lowTask = { 90, 1616};
public int[] mediumTask = { 90, 1624};
public int[] highTask = {1624, 1615};
public int taskID = -1;
public int taskAmount = -1;

this means that you can get skeletons and basilisks for low tasks, skeletons and dust devils for mediumtasks and dus devils and abyssal demons for hightasks.

to choose your own monsters just put their id’s in the { } bits and then save your client but don’t close it…yet.

Setting levels.

So you may want to set a level…like people with level 10 slayer getting set Abyssal Demons for slayer tasks wouldn’t be good. To do this you need to search for the following:

 public void giveTask

This is normally AFTER the part where you set your creatures.
Now in my source this is how its set up:

 public void giveTask() 
 {
  if (playerLevel[18] >= 1 && playerLevel[18] <= 65) {
   taskID = randomLowTask();
   taskAmount = 15;
   talk2("Your task is to kill "+taskAmount+" "+GetNpcName(taskID)+"s.", "", 1599);
  }
  if (playerLevel[18] >= 65 && playerLevel[18] <= 80) {
   taskID = randomMediumTask();
   taskAmount = 20;
   talk2("Your task is to kill "+taskAmount+" "+GetNpcName(taskID)+"s.", "", 1599);
  }
  if (playerLevel[18] >= 80) {
   taskID = randomHighTask();
   taskAmount = 30;
   talk2("Your task is to kill "+taskAmount+" "+GetNpcName(taskID)+"s.", "", 1599);
  }
 }

Explained:
The first bit you see [if (playerLevel[18] >=1] is basically saying you have to have a slayer level higher than one to recieve this task but then the following [&& playerLevel[18] <= 65)] means that the max slayer level a player can have to recieve these tasks is 65. Simple code used pretty much all over the source-code.

The next bit [ taskID = ] is basically telling the client that it has to use that particular slayer task set. e.g. randomhightask would link back to the creature ID’s for hightask and so on.

The next bit [taskAmount = ] is saying how many you can get in that particular task. There are other ways of doing this. For example you could put in taskAmount = 10 + misc.random(40); This would tell the client that it has to have a minimum of 10 for a task but a maximum of 40 for a task…so basically a random number of creatures for a task. By changing the (40) you could set that to whatever you want…for example:
taskAmount = 100 + misc.random(400);
That would be minimum 100 and maximum 400. You get the point?

The final bit of code [talk2("Your task is to kill “+taskAmount+” "+GetNpcName(taskID)+“s.”, “”, 1599);] is telling the player what their task is. “+taskAmount+” is…well…their task amount. and the “+GetNpcName(taskID)+” is the name of the NPC or creature.
