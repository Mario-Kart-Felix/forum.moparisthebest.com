Advocatus August 1, 2016, 11:08pm #6
SnowBoarder7:

it’s so easy to decompile Java and change the if statements, the URL connections, etc…

Use a bytecode obfuscator. I can guarantee that it will not be easy to decompile it and change those basic things… Take a look at the current rs client following the latest bot busting update. Decompile that, and then show me how easy it is.

My advice.
Have a seperate applet program to act as a classloader and generate a numerical value from the machine info, the day, week, month, and year and also some other variables. Do some numerical operations on this value, and then send it to a server where it is checked. If valid, load the real application jar (could even do a custom compression method on the jar itself).

Even if you did that and they used reflection to dump the jar during runtime, the obfuscation would be enough to be pretty damn daunting.
