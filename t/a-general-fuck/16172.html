BobDole81 July 29, 2016, 10:18pm #2

I gave you the initial solution yesterday, man. When writing the size of the content pool, use an integer instead of byte…the size of the content pool is 256 bytes and a byte will write that as 0, whether you read it signed or unsigned since the greatest actual value of an unsigned byte is 255 (don’t forget zero, which makes 256 values).

Another possible error is that you’re trying to decompress loader.jar when you should be decompressing loader.nvk
