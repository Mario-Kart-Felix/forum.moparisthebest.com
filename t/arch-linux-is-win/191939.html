Look, I’ve used both before. The difference is not noticeable. If you want “speed”, then you have to be prepared to edit configuration files. You can always rebuild your own vanilla kernel with all the flags you want, but I HIGHLY doubt you are better able to optimize a Linux kernel for i686 than the crew over there at Arch Linux. Your perception of speed comes from how snappy the system feels, in which case, you can improve it yourself. Just edit some configuration files, select what starts at boot-up, pick lightweight apps and build your system. Arch Linux is all about keeping it simple and building your system the way you want it. It’s easy to build a “slow” system by selecting a million daemons to boot up, picking the most heavy-weight software and so on… If you want speed, use things like fluxbox, or XFCE and things like that. Make use of Python, it’s fast. Ask on the Arch Linux forums; there’s a lot of knowledgeable people over there.
