moparisthebest July 31, 2016, 8:32am #10

As I thought, most of the RAM (almost 8GB) is cached, and so not really used, just saved for quick use in the future (so in reality, 2.2GB is used, and the rest is free, not too shabby for a 64bit OS, running lighttpd, MySQL, postfix, sshd, and all the other services required for a server):

$ free -m total used free shared buffers cached Mem: 16078 10638 5440 0 485 7894 -/+ buffers/cache: 2258 13820 Swap: 30517 0 30517

I’ve now had more time to configure and set up everything like I want it (I think!) and after adjusting to the far different, almost script-like, configuration file format I have to say I think I like it much better than apache2.

edit:
I have now taken the time to ‘tune’ the configuration and hopefully the work will result in a super-speedy web server that won’t get bogged down. Time will tell I suppose.
