[quote="xxxtilllyxxx, post:18, topic:453461"]When would reflection be needed / used? What is the "transient, volatile, and protected" modifiers used for?[/quote]

[ul][li]Reflection is used to examine / modify the run time behavior of applications on the JVM. In other words, you can use it to see (or change) things at run time that you normally wouldn't be able to. This is useful for debuggers, IDEs, or whatever else you'd like to do. It has some performance overhead.[/li]
[li][i]transient[/i] means that a field's value is not written to the serialized representation of the object upon serialization. Upon deserialization (reading the object back into memory), each transient field is initialized to its default value.[/li]
[li][i]volatile[/i] is used by the programmer to indicate that a field will be modified by multiple threads. It is used by the JVM to ensure that all reads and writes to [s]transient[/s] [i]volatile[/i] fields are done correctly. I am not too familiar with it past that.[/li]
[li][i]protected[/i] is an access modifier - it determines what level of access other classes have to fields and methods. [i]protected[/i] members are accessible by the class, package, and subclass. A stranger example is when [i]no modifier[/i] is given, which grants access to only the class and package (but not subclasses).[/li][/ul]

Vrunk, your description of volatile isn’t right. It doesn’t describe behavior of accessing transient variables, it describes accessing volatile variables, wherein a variable is expected to be read and written to at the same time. Volatile ensures that the variable is stored in memory shared between all threads, rather than being locally copied for each there (and therefore, if a thread decides to access and change that variable, it must wait until another thread has finished writing/reading it first).

Your use of the word transient is wrong and confusing in that description.

[quote=“The Enforcer, post:31, topic:453461”]I have a question.

Some masterminds say that C++ is the king of all programming languages, the superior language. I, as a dedicated java enthusiast, would like to know if it’s maybe time to abandon java and start writing the clearly superior C++? Could you express your thoughts on this?[/quote]

C++ has tons of features. I do not have much experience with it.

That being said, # of features != superiority. You can produce very efficient code with C++, but is execution speed your greatest concern for all areas of your project? My guess is probably not.

There are many other things to consider - community support, standard libraries, package management, language syntax, the purpose of your project, the size of your team, the number of mistakes your team will make with the language, the rate at which your requirements change, how enjoyable the language is to work with, etc.

I tend to develop with a scripting language (Ruby, Python) and only optimize the parts of the application that need it the most. You can do that in a systems language - both Python and Ruby have decent support for C extensions. If you are interested in learning a systems language, give C a shot. D and Go are also interesting.
