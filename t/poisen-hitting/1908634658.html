Exactly what I said. In order for us to do this correctly, we need to have the server differentiate between three damage types:

Physical - incoming, up-front damage. Applied once on damage taken assuming we do not have a miss.
Poison/periodic - delayed, taken over time. Timer initiated on begin, one timer per applier (a poisoned dagger will not poison the same target 15 times simultaneously this way, but will apply/refresh only one timer). Should take in a set amount of damage and a period of time, in seconds, to apply it over. Let’s say damagePeriodic(args: dmg, time).
Special - applied with the orange “splat”. Special damage - I believe this takes effect on all stats, reducing each stat’s temporary value by x.

In this way, we tell the server to apply damage of a certain type, and the rest is handled for us. For poison, a timer is set from a specific source - say, DDS+ held by player PlayerOne - and we do not create more timers on each application from PlayerOne’s DDS+, but instead refresh the amount of time left on the original timer. We’re not switching back and forth between different icons, but instead applying damage in its native type.
