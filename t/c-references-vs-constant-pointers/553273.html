Tree August 4, 2016, 3:59pm #3

You’re getting bogged down in things you don’t need to be bogged down in. Learn about pointers and how they work. Write a program that sends addresses flying around and output a few so you know what’s happening.

An excellent place to start is C, as you’re almost forced to learn these concepts. Also, this isn’t exactly relevant but it’s something to be mindful of. Don’t become too obsessed with performance and how efficient something is. Write for readability and good design. Then if it has trouble, modify it. Too many people try writing code that’s pre-optimised and it turns into a mess. Optimisation requires a lot of understanding about what’s happening in the background and when you focus on this too early, you get lost down the rabbit hole.

Focus on basics. Focus on readability. Nail the design. Then build from there. You may want to get everything perfect before showing people but everyone starts somewhere.

a reference is a const pointer, not to be confused to a pointer to a const, that is basically syntactical sugar. it allows you to use the dot accessor notation (myObject.func()) instead of dereference-access (myObject->fun()).
