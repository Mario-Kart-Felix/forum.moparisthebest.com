The RuneScape client is also redundant in functionality. The portions of their native code to handle interfacing with OpenGL and DirectX aren’t needed, since they have their own 3D software renderer. Their 3D software renderer in native code is redundantly backed up with their pure-java software renderer. Thus, you can still run the RS client on any platform, just not 100% effectively on every platform.

There are advantages and disadvantages of using JNI, and sometimes it’s even necessary. I agree with the idea you should only use it if you absolutely need to, and if you only use it correctly. It’s pretty much pointless to write 95% of an application in Java, and the remaining 5% in C++ with OS dependencies that completely contradict the platform independent nature of the other 95% of the program. If you are going to make it platform dependent, why chose Java? There are generally better choices for applications only written in Windows, Mac, or other systems, if you are writing solely for that system.
