[center][size=18pt]Using the Java Native Environment[/size][/center]
This tutorial will teach you the basics of using the Java Native Interface (JNI), coupled with C++, to do a lot more with your programs. This will not be an explanation of the JNI and what it is; if you want that, click here This will be the ‘dummies’ guide, for people that have never used C++, and I will include pictures of what you will be doing in the IDE.

[anchor=javacode]Writing the Java Code
The first thing you do when using the JNI is make a normal Java class. This class will be the one that loads the Dynamic-link library (DLL), which you will be making later. Here is the example class we will be using in this tutorial, it is important to read the comments:

[code=java]/**
*

  • @author David
    */

public class Tutorial {

static {
System.loadLibrary(“TutorialDLL”); //Load the library we will be creating in C++
}

public native void nativeVoidInitialize(); // Native methods are methods that are handled in the C++ code library.
public native void helloWorld(); //They can still be called in the Java program, as you will see below, but the library will handle them

void noParameters() { //These next three methods look like normal methods, and that’s because they are.
System.out.println(“No parameters :)”);
}

void getNativeParameters(int i) { //They can be called in the normal Java program, but they are also loaded into the JNI and can be called via the library
System.out.println("Integer: " + i);
}

void getTwoNativeParameters(int i, boolean b) { //There is a special way to load method ID’s into the C++ code, which I will show you how to do
if(b) {
System.out.println("Boolean true, integer " + i);
} else {
System.out.println("Boolean false, integer " + i);
}
}

public static void main(String args[]) {
Tutorial tut = new Tutorial(); //Makes an instance of this Tutorial class
tut.nativeVoidInitialize(); //These are the native methods that will be handled by the C++ code, look up to see how they are declared
tut.helloWorld(); //You can have as many native methods as you like!
tut.noParameters(); //This is just showing that you can call these methods from inside the Java code, and in the C++ code (later)
}
[/code]

That’s it! That’s all the Java code you will be writing for this example.

[anchor=makingheader]Making the C++ Header File
Now that we have that, we need to make a header file. Header files are used to declare things for other C++ files to use. You can read more about them here.
The great thing about these header files is, you don’t need to write them! In fact, they are generated for you using a simple batch script, and editing them is not suggested. Here’s the batch script (make sure that the .bat is in the same place as your .java file)@echo off echo JNI Tutorial - Compile and Create Header File - By Davidi2 title Compiling... "C:\Program Files\Java\jdk1.6.0_18\bin\javac.exe" Tutorial.java echo Compiled! title Creating header file... "C:\Program Files\Java\jdk1.6.0_18\bin\javah.exe" -jni Tutorial echo Tutorial.h header file created! pause
This will create the Tutorial header file in the same folder as your other files. Do not open it yet, just leave it alone.

[anchor=finally]Writing the C++ code
Phew! That was a lot of work, and we didn’t even start writing the code yet! Don’t worry, it gets a little more fun now.

If you looked at TutorialDLL.cpp, you saw that it already had an import (“stdafx.h”) Right below that line, add

#include "Tutorial.h"This line is very important. Not only is it our declarations for our Java methods, it also contains the JNI include line.

So now we can get started! Open up your Tutorial.java again. See the native voids? These are the methods that we need to cover. The way we write a Java method (for JNI) is like this:
JNIEXPORT type JNICALL Java_ClassName_MethodName(JNIEnv *env, jobject obj) { }
The bolded things are the ones we change, the normal ones we leave alone.

Type - The type of method it is. void, String, integer etc
ClassName -> Method name - Self-explanatory, they are the class names and method names
JNIEnv and jobject you don’t need to worry about at the moment.
So here’s how we would write the method for “public native void nativeVoidInitialize();”

JNIEXPORT void JNICALL Java_Tutorial_nativeVoidInitialize(JNIEnv *env, jobject obj) { }

It’s that easy! So now, every time you call nativeVoidInitialize in your Java program, it will be handled here! This mean that you can do C++ things, from a Java program. Feel free to experiment around with this, but I’m going to continue the tutorial.

If you look back in your Java file, you see the normal methods, like getNativeParameters. The way you call these is not the same (because they are not native) In order to call a normal method, you need to have a JNI Environment, which you saw in JNIEnv *env.

Let’s use this to our advantage.

JNIEXPORT void JNICALL Java_Tutorial_nativeVoidInitialize(JNIEnv *env, jobject obj) { jclass tutorial = env->FindClass("Tutorial"); //Get the JNI class instance jmethodID gnp = env->GetMethodID(tutorial, "getNativeParameters", "(I)V"); //Finds the Tutorial method 'getNativeParameters'. env->CallVoidMethod(obj, gnp, 6); //This calls the method that we found in the previos line. All of the parts after the method ID are the methods parameters jmethodID gtnp = env->GetMethodID(tutorial, "getTwoNativeParameters", "(IZ)V"); //Finds the Tutorial method 'getTwoNativeParameters'. env->CallVoidMethod(obj, gtnp, 10, false); //This calls the method that we found in the previos line. All of the parts after the method ID are the methods parameters }You can get the gist of what each line does from the comments, but if you don’t know C++, don’t worry about it too much. Just try to memorize how to use the JNI specific methods.
Even still, I will go a little more in depth into GetMethodID, because it is important.

GetMethodID(jclass, methodName, methodTypeAndParams);
The first two are self-explanatory, but the second one, not so much. The method type goes like so:
“(Paramtypes)Methodtype”
for example, (IZ)V indicates a void, with the params int, and then boolean.
Here is a table of the basic representations:
Z boolean
B byte
C char
S short
I int
J long
F float
D double
So let’s summarize what the code does so far. We run Tutorial.java, which will call the native method nVI. This points to the DLL, which executes the code above. The code above gets an instance of the Tutorial class, then gets the method ID’s for two different methods, and executes them with the parameters.

Here’s the entire cpp code:

[code=cpp]// TutorialDLL.cpp : Defines the exported functions for the DLL application.
//

#include “stdafx.h”
#include “Tutorial.h”

JNIEXPORT void JNICALL Java_Tutorial_nativeVoidInitialize(JNIEnv *env, jobject obj) {
jclass tutorial = env->FindClass(“Tutorial”); //Get the JNI class instance
jmethodID gnp = env->GetMethodID(tutorial, “getNativeParameters”, “(I)V”); //Finds the Tutorial method ‘getNativeParameters’.
env->CallVoidMethod(obj, gnp, 6); //This calls the method that we found in the previous line. All of the parts after the method ID are the methods parameters
jmethodID gtnp = env->GetMethodID(tutorial, “getTwoNativeParameters”, “(IZ)V”); //Finds the Tutorial method ‘getTwoNativeParameters’.
env->CallVoidMethod(obj, gtnp, 10, false); //This calls the method that we found in the previous line. All of the parts after the method ID are the methods parameters
jmethodID np = env->GetMethodID(tutorial, “noParameters”, “()V”); //Finds the Tutorial method ‘noParameters’.
env->CallVoidMethod(obj, np); //This calls the method that we found in the previous line. All of the parts after the method ID are the methods parameters (none, in this case)
}

JNIEXPORT void JNICALL Java_Tutorial_helloWorld(JNIEnv *env, jobject obj) { //Handles the native method helloWorld
printf(“Hello World!\n”); //Equiv to System.out.print();
}
[/code]
