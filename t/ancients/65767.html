Have variables. This isn’t a Tut so I don’t know how well it’ll work. At the top where there’s variables type public boolean candoancients = false;

Then (this is asuming you already have the quest made) right before you close off the quest void, type candoancients = true;

Now since I don’t know what the first ancient is, search for it’s name. It’ll say

if(spellID == ####){ //(ancients name here)

After that line type if(candoancients = true) {

And on the line before the last } type

}else{ sendMessage("You have to do (Quest name here) before you can use ancients."); }

And repeat for the rest of the ancients. I don’t know why I thought of this, but if you want it to kill them if they try change it to

]}else{ sendMessage("You have to do (Quest name here) before you can use ancients."); ApplyDead()}
And that’ll kill em! You can use variables to make it warn them several times before killing them.
