[anchor=tldr]TL;DR - The irony is this is before the wall of text
Some members have stated that a wall of text can look menacing, and that a lot of people would skip it. I’ve decided to try and sum up all the sections (besides the google one, and the link one) into this one brief summary. If you don’t like how it’s explained, read the full section.

 [u]Posting[/u] - This one is hard to "TL;DR" because of how specific it can get sometimes. Basically, it's no biggie if you post into the incorrect section. We won't get mad, but you'll probably get a member or two telling you where it belongs, and it might get moved by a moderator. Again, nothing bad. If you have an error in your server console, it's probably server sided. If you have an error in your client console (or it freezes), it's more than likely client sided. NPC's, Items, Objects, and Maps are all done client sided in most 317s, and therefore you should post there. If you have any questions on this, read the more detailed version (it's worth it).

 [u]Revision[/u] - The importance of telling us the revision of your server is to essentially help us know the packet and code structure of your server. This is based on each time RuneScape made an update to the game.

 [u]Server Base[/u] - This is one of the most crucial pieces of information because there are millions of servers released on so many bases, knowing what base will provide us with a starting point to get you the proper code you need. Usually the person that released it will have the base in the release, but don't always. If they don't, I'd compare some code in it to code released on sites (like this one) for tutorials and such.

 [u]Informative Titles[/u] - Think of Server Help as a candy shop. If your topic has a good title, it would taste better to the helper, and therefore they will be more inclined to test it (a.k.a read your topic). Thus increasing your chance at having your problem solved. A great title format would be: [Revision Number] Server Base - Simple description of error.

Example:
[317] PI - Cannot combine Sara brews when using them on each other

  [u]Multiple Topics[/u] - It's highly recommended that you do not post multiple topics for each one of your errors or issues. Please remember that it's against the rules to post multiple topics that are duplicates or near duplicates. Instead, List each of your problems in one topic, and pick the most important one to you as the thread title.

I hope this TL;DR sums it up nicely, please leave questions pertaining any section (read the actual section first pl0x) of this post below.

[anchor=Revision]Revision - What’s so important about it?
To start, I’ll explain where we get the revision from. The revision number comes from RuneScape, each time they make an update the revision number is changed. Usually it’s just up by one, but I believe it depends on how they’re feeling lol. While I don’t know the exact reason why we care to know it, I’ll explain why I ask for it. I ask because it helps in finding the packet structure and all that good stuff. I personally try to use the RSWiki as much as possible before asking packet based questions.

[anchor=Base]Server Base - Don’t say the wrong one
This is a lot more important than telling us your revision. Why? Because you can modify client data to make it look like a newer revision, and a lot of people base their servers revision off of that and that alone. When I say “Don’t say the wrong one” I mean to say the actual base of your server. There are thousands (dare I say millions) of servers out there. That doesn’t mean the server’s base you downloaded, “LeechScape v101.2 by CrappyDev”, is “LeechScape”. The base is a tad bit harder to get than that (Unless it was a framework). Usually they’ll include the base in the title, along with the surname they decided to give it. So instead of it being, “LeechScape v101.2 by CrappyDev”, it’s, “[Project Insanity] LeechScape v101.2 by CrappyDev”, or even, “[Delta] LeechScape v101.2 by CrappyDev”. That would make the server’s base Project Insanity, or delta (respectively). Meaning when you reference the base, call it a PI (Project insanity) or Delta. Knowing the base will help us 100% in helping you because we will have a great understanding of the coding structure used, packets, and all that general stuff. It is one of the most important pieces of information you could give us, besides your issue and your code.
What if it doesn’t say the base? - Great question. There are a few reasons why it may not, including (but not limited to): The dev forgot to say, the dev won’t say, it’s in the post and not the title, it’s a framework (basically a base), or you’re not looking hard enough. To find the base you’ll have to look in the code (or replies on the thread, lol). I cannot give you examples of all coding structures in every base ever made, but I’ll give you a couple from Delta (from memory) and PI, since I used them as examples.
Delta:

sM("This is a send message in delta!");

Project Insanity:

c.sendMessage("This is a send message in PI!");

While those are super basic examples (and method names can be changed super easily), there are a few minor differences. (Remember that I haven’t touched a delta base since Bluur’s client was popular) In the PI, you have the client object before the sendMessage, unlike delta. However, if you’re in the client class of PI you wouldn’t either.
Anyone reading, if you have a better example feel free to post it.

[anchor=Titles]Titles - Get your help faster
The first thing a potential helper sees about your post is the title. If your title doesn’t give much information, some people won’t even bother to even click your post. I know when I first started being a major contributor to this section, my first “round” in the posts was me clicking topics that had the base in the title because I knew how to help with those ones the most. Only after that would I go look at the others, and maybe not even then.
Put yourself in my shoes, if you can, and go into this section. Look at the topic titles. (At the time of posting) Here is what I see:

Click me for full size
While it might be ironic that the topic named “LOOK AT THIS” has nearly the most posts in the picture, that’s not the point here. The one topic I replied to (at the time, I started this post soon after) is titled “How to find custom item ids? 317”. What’s so good about this topic that I went to it first? His title. It tells me exactly what he needs help with, and his revision. While I would rather him have put the base there, the revision helps.
Now lets get back to mr. LOOK AT THIS. The all caps suggests he is either frustrated or annoyed (these are the 2 emotions I usually get off the people posting all caps titles I’ve worked with in the past). Do you really think I want to work with some random person on the internet that is irritated? No. They make my ‘job’ difficult. When you’re upset, you aren’t level headed and you most likely aren’t thinking clearly. So when we ask you to do something, try something, or post something. It can end up with you just getting more upset at us, and no one needs that. Do not post if you’re upset. Take a breather first. Seriously, you might even find that you knew the answer before you post because you weren’t thinking straight earlier.
What do you require in a title - While not an official requirement here at moparscape. I’d like to see a title in this format:
[Revision] Base - Issue
Examples:
[317] PI - Cannot cut any trees with hatchet
[718] Matrix - Cannot give myself admin rank
[317] Delta - Packet 126 isn’t working, no interface text updates when I use it.

[anchor=MultiTopics]Multiple topics - They won’t get you faster help
If you looked in the picture above (Link) you would have noticed some users have made more than one topic. Depending on the contents of said topics, it could or could not be against the rules. Moparscape doesn’t allow you to post multiple topics trying to accomplish the same thing. However, you can post as many (as long as they aren’t spam) as you want if they’re about different stuff. But in Server Help, that may not be a good idea. If you list your issues into one post, that will not diversify your viewers into other topics of yours. Meaning instead of having one topic with 11 views, another with 40, and then a third with 500, you could have one giant topic with all 500 views (because it might be the same people reading the other topics and whatnot). That would give you a higher chance of all your issues being solved. Here is an example on how I’d make a topic with multiple issues.
spoiler=Example [317] PI - 5 issues. Issue One: Cannot enter altars if wearing a tiara.
(Body)
Hello faithful members of Server Help. I have a lot of issues today. And I thank you all for taking the time to read my topic.
The first issue is most important to me, so I’d like it if you helped me with that first.
1. Cannot enter altars if wearing a tiara.
Whenever I am wearing a tiara, and try to click the mysterious ruins (object 12345), it will not let me in. But if I do the same thing with a talisman in my inventory, it works.
Here is my code:
ActionHandler (For clicking the object)

case 12345:
     c.getRunecrafting().enterAltar();
     break;

RuneCrafting.java (for the enterAltar method)

public void enterAltar() { if(c.getItems().playerHasItem(4444)) { c.getPA().movePlayer(1, 2, 0); } }
2.Other issue
Other data about issue
(Topic End)
And so forth. It still gives you the informative title, and you still have all your questions, there’s just less link surfing to get to them.
[/spoiler]

[anchor=goog]Googling - Getting useful search results
You should always google your issue before you come here, so here’s some tips to help you get some better results.
Google loves key terms. Never enter a sentence. You’ll get obscured results that way.
Example:
How do I add new attack animations to a project insanity RuneScape Private Server? - My question
Project Insanity Adding Attack Animations - Same question, less words.

Google also has lots of search features. I’m not going to highlight all of them, but here’s some I use:
“word or phrase” - If you quote a word or phrase, google will make sure the page contains those exact words (should be in that order as well)
site:http://moparisthebest.com/smf - You can add this to search one site and one site alone.
filetype:zip - Searching for specific file types.
-Term - Excluding a search term. Good for filtering.
