Ok so I’ve seen posts where people try to compile things and they get like 100 errors or something. Well, a very easy way to fix this is to actually quit being lazy and read the errors! First it will say the file name that the error occurred in. Then it will say the line that the error was in. Then a brief description of the error. An example:

client.java:7100: cannot find symbol
symbol : variable signlink
location: class client
signlink.errorname = s;

So what you need to do is first open up the file the error was in. Then go to the line the error was in. This is tricky and can be EXTREMELY time consuming, unless you have this application called Xcode. Xcode can be downloaded when installing developer. You can buy it here…
http://developer.apple.com/programs/
But I’m sure 99% of you DON’T want to pay $99 for it. So google it or something looking for free download link. I don’t really know any free ones, but you can give it a shot. Anyway, whenever you get Xcode, you’re going to want to open up client.java or whatever file it was. BUT OPEN IT WITH XCODE!!! DO NOT RIGHT CLICK AND SELECT EDIT!!! OPEN! WITH! XCODE! Once you have it you should have your file. What’s cool about Xcode is that it is programmed for the Java language, unlike notepad. Now here’s a quick summary of the lettering colors that you will see when using Xcode.

Green = will not appear as script, basically used as a note. Does not effect file at all.
Pink = Basically just java language like “if, else, end, break, case” etc. Pretty much starting words for lines.
Blue = Numbers.
Black = Actions. Telling whats going to happen, comes after pink words.
Red = Text that will appear, like sendmessage and stuff.

Okay that should help you identify problems much much easier than looking through a bunch of black text. But that’s not the biggest advantage of this. Probably the biggest advantage of Xcode (at least for debugging) is the thing that tells you what line number you’re on. All you have to do is click on any line in the file and up in the top left hand corner it will say the name of the file and what line your blinking cursor thing is on. This is helpful cause it’s WAY easier to find the line your error was on than if you were to go on notepad and count the lines one by one, especially when there’s thousands and thousands of lines.

So once you’ve found the line that your error was on, you should be able to see what was wrong. If you can’t figure it out, post a reply here and a picture of the file with the line that’s screwed up, and be sure to tell me which one it is. I realize it can still take a long time to find and fix 100 errors, but at least you can do it quicker, as well as having a more understandable application to edit your java files with.
