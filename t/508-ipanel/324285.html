It’s very easy to make these ‘Panels’ for any server.

Here’s a base of how to make GUI’s.

[code]
package yourserver.yourserverfolderetc;

import javax.;
import java.awt.;
import java.awt.event.;
import javax.swing.;

public class Gui extends JFrame {
private JLabel textt;
private JButton button1;

public Gui() {
	super("Control Panel");
	setLayout(new FlowLayout());

	textt = new JLabel("Heres a button -->");
	textt.setToolTipText("Where is the button!?");		

	button1 = new JButton("Button");
	button1.setToolTipText("Button.");
	
	add(textt);
	add(button1);
	button1.addActionListener(new Action());
}

static class Action implements ActionListener {  
   public void actionPerformed (ActionEvent e) {
                System.out.println("A button was clicked on the Control Panel.");
	} 

}

}[/code]

To make the GUI start up when you run the server, go to your Engine class and import:

import javax.swing.JFrame;

Then declare:

public void startGUI() { Gui gui = new Gui(); gui.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE); gui.setSize(275, 180); gui.setVisible(true); }

Then find the run method. In it call:

startGUI();

That will start up that GUI. So yeah, that’s a base of making GUI’s for your server. You’ll have to build up from there and actually make it so when you press buttons, it actually does something. Very easy, tbh. Good luck. Do some more research about GUI’s if this didn’t help.
