TeleNubby July 30, 2016, 11:21pm #6

Well, to see if the message comes, you can create a quick packet sniffer in Java. There’s even a good public API if you want one (http://netresearch.ics.uci.edu/kfujii/jpcap/doc/).

Next, you’ll want to read about the MSN protocol to find out what triggers the client to identify the incoming command as a ‘message’ (a good documention on the protocol can be found here: http://www.hypothetic.org/docs/msn/)

So now, you can run your program to sniff out the sessions between the messenger and server, and if a message is received, you’ll be able to know that.

According to you, the next step would be to play a sound and set focus if the window is not already focused. I’m afraid I do not know of anything that would get a Window instance (or anything similar) of a Window that YOU did not create in Java (so anything not in Java) using the Java API off the top of my head. But I also do doubt there is anything directly there for you to use. You’ve probably got a better chance using Native API. If I find anything, I’ll tell you.
