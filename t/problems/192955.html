I agree with Unhandled. You are an idiot if you think can simply assign a instance of Robot to a Colour array. Look at the API of Robot and find the right method. I’m not going to look at it. What you will probably get is an Image or an array of colours. In either case you need to call a method.

Now for the nitpicking: Why didn’t you use a char[] rather than a string[]? It would be far more efficient. In fact from a coding perspective you could make a String with all those chars and then call string.charAt(). You also might want to abstract the string decoding. Choose an algorithm; for example process in 4 char chunks, swap char 0 and char 3, then xor by 0x23. Something like that can both encode and decode as it is symmetrical. It also looks like you coded it by hand. For increased randomness you might want to generate it randomly.
