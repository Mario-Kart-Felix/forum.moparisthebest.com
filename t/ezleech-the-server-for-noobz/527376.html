[size=18pt]about the project:[/size]
ezleech is going to be designed as a ‘core’ server. you will be able to modify everything to your liking outside the core server. the core will be represented as a jar, and all other data needed will be separated into appropriate directories (similar to minecraft)

there will be a control panel which allows access to all of the server settings. a toolkit for modifying data (accounts, packets, quests, etc) though some features will not be added for a long time.

i plan to use rs2hd and will currently only do the 377 revision, however may make higher revision ‘cores’ later.

data: im more than likely going to support xml&json for data, but for now just xml. i may even get in to adding sqlite or some other data storage. for xml, im considering to not be using native bindings, or xstream. i’m leaning towards jibx.

[size=18pt]current ideas (not implemented anything, still using standard rs2hd #377)[/size]
quest (support to define quests)
character (support to define new variables to save per player file)
command (support to define new commands with specific actions)
packet (define packet id and which class to user via the core (for use to change packets if you want) as well as handle packet data such as clicknpc)
