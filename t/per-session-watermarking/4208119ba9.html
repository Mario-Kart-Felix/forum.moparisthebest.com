I noticed that iagex implemented a new loader system in rs. The jar file is grabbed from the server. The jar contains the loader and an encrypted ‘inner.pack.gz’ file which contains the rs classes. To decrypt the file you need the ‘0’ parameter value and the ‘-1’ parameter value. The first one is the ‘key’ used to decypher the file and the second is the ivParameterSpec.

So if you want to get the rs classes you need to get those 2 parameters including the gamepack which coresponds with those parameters. Don’t use another gamepack.jar file from them with the 2 parameters from another rs instance as it won’t work.
