Well I got a nice burst of motivation to fix up my questing…

I practically redid the API, so each quest doesn’t need to use as many methods and had each quest load for the player (seperate table in the database for quests!). And I commented everything for it, so when it’s finished I can give out the Java Docs and explaining each method is already done.

As of this moment I have finished the rest of quest requirements/ etc for when you haven’t even started it

Oh and the last thing I did yesterday was adding a lot to the site, bridging forums and site to work as one, and also created images that will be used on the site. Fixed up xhtml validation, etc. Lots of fun!


The Quest shown in the picture is the following, this is just for people who are wondering how the quests are going to be done. Note that this isn’t a finished version and anything can change!
The Quest Log and messages are stored in a HashMap.

package net.arcanum.server.quest.quests;

import net.arcanum.server.quest.Quest;

public class ExampleQuest extends Quest {
	
	public String getName() {
		return "Example Quest";
	}

	public int getQID() {
		return 9247;
	}
	
	public void init() {
		getRequirements().setAttack(70);
		getRequirements().setStrength(99);
		getRequirements().setDefence(50);
		getRequirements().setStart(4);
		getRequirements().setDefault();
		
		insertQuestMessage(0.0, "Hello!");
		insertQuestLog(0.1, "A dwarf down near Edgeville bank is looking for");
		insertQuestLog(0.2, "a brave warrior.");
		
		
	}
	
	public void processing() {
		
		
		
	}
}
