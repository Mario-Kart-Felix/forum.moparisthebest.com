[ul][size=12pt]What is it?[/size]
Arcanum is my new server which has great potential. It is f2p-based and will remain so for a long while most likely.
It is a 317 client and I may go up to 377 if I ever require it…

[size=12pt]What about features?[/size]

At the moment? It’s pretty basic. It will be rapidly created, so it may not look like a lot at the moment but I am getting a solid base down first, before I go on to better front end features and then create a sink hole for myself later when I need to convert it to a new system.
No other client will be supported than my own if it is ever released or hosted because of the extra use of features which require other data to be sent to the client unless you modify another client to support it.

All code for the following features were been written by me.
[list][li]Loading/saving players via MySQL.[/li]
[li]Player look [on first login can design your character]. It will cost a fee through something like a make-over mage (not exactly) to redesign.[/li]
[li]Item config, NPC config, NPCSpawn and Shops are loaded from SQL, not config files[/li]
[li]Rights / staff is handled via MySQL as well to allow for dynamically changing rights without logging out (also seperate from user tables)[/li]
[li][User/ip] Bans/mutes are handled via MySQL and work correctly.[/li]
[li]For bans/mutes, IF they expire, a PHP script runs via cron to auto expire bans that need to be expired.[/li]
[li]Displaying ban reason to user when they try to login if they happen to be banned.[/li]
[li]Expired bans will NOT be deleted but rather flagged inactive; in the first picture this is not depicted in sql snapshot because it was added just before I posted this.[/li]
[li]Accounts require registration VIA client or site before they can log in. Also requires email activation (can’t login until activated!!)[/li]
[li]Change your password or view account information while you are logged in via the menu bar![/li]
[li]Server-wide chat to replace ::yell, includes player list to see who is in the chat![/li]
[li]Energy & weight for each item. The more weight you have, the faster energy degrades while running. Energy degrades at nearly the exact rate of RuneScape and regenerates at the same rate as if you had 1 agility or were in a non-members world.[/li]
[li]PvP/PvE. All three forms of combat, but I still have yet to create a better way to generate max hits for range and mage.[/li]
[li]Website and forums are bridged so only one account is needed for both (register on forums redirects to the site’s register, anyway).[/li]
[li]Pages via the site for Admins and Mods to ban or mute users (mods can’t ban!)[/li]
[li]Bans are automatically expired if they are past the expiry date or meet it. This is done by the server (CRONw was acting shittily, will probably move the server over to linux when the beta actually comes.)[/li]
[li]Advanced Quest system that allows for anyone to submit quests and a whole quest is managed within one file, allowing for any quests to be added or remove instantly without a whole lot of work.[/li]
[li]Fully working Report Abuse in game, and moderators have the ability to mute with it. Note: This is actual 48 four mutes, this isn’t some half assed report abuse (it was so easy because my current ban system supported any length of ban/mute! :3)[/li]
[li]Pretty much everything is logged to MySQL along with timestamps. Chat logs, server-wide chat logs, trades, mod logs and admin logs. :3[/li]
[li]If you are under an NPC/player and you go to attack it, you would not move. It now finds a spot for you to move to (if there’s an object at the default move to place it will find an open spot, and if it doesn’t find a place… well sad times, because you can’t attack :-X. This should NEVER happen, though).[/li]
[li]Before, running and retaliate settings were saved, but, when you logged in if you were running it said walk… Wait, walk? no… I’m running! Well, it’ll now switch to run and not be default to walk. Also, brightness levels are now also saved and your saved brightness will be applied to you when you log in :D[/li]
[li]Moving fishing spots (to help prevent long afking[/li]
[/list]

Skills at the moment: (all listing are fully working to my knowledge)
[list]
[li]Combat (attack/strength/defence, range and magic)[/li]
[li]Prayer[/li]
[li]Woodcutting[/li]
[li]Fletching[/li]
[li]Mining[/li]
[li]Runecrafting[/li]
[li]Fishing[/li]
[li]Cooking[/li]
[li]Some unmentioned skills are planned for the beta and definitely for the version afterwards[/li]
[/list]

Planned features?
[list]
[li]A player killing “point” system, simply to award the user with small things here and there for pking a lot (not like 100 points = some overpowered weapon…)[/li]
[li]IP history and password history in MySQL to assist staff in recovering user’s accounts if they have ever changed their password. Maybe even recoveries with recov history, too![/li]
[li]And more![/li]
[/list]

As you can probably tell I like databases… This looks like a big project, but then again it isn’t that big.
