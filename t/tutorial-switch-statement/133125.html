Purpose: To stop the confusion of ‘cases’.

Explanation of what a switch is
A switch statement is a control flow statement. And are similar to ‘if’ statements. In switch statements there are cases. The cases within a switch statement are basicly like ‘if’ statements. When you use the switch statement you specify a number storing data type, normally an int. The usage of the statement is:

switch(a) {

a - The name of the interger type.

So for example I could have this:

switch(nirvanasInt) {

Under the statement you then add cases, followed by the break statement. You can if you wish make a ‘default’. You can then catch missed values (When the interger is not equalled to any of the cases).
Usage for adding a case:

case ####:

#### - The value of the switched interger. Switch statements are somewhat like if statements. Its basicly saying that if the switched interger is equlled it will perform the action of the content within the case.

So heres an example:

case 57:

Each case must be ended with the break statement.

break;

So putting that all together we’d getting something that would look somewhat like this:

[code]switch(nirvanasInt) {
case 57:
System.out.println(“Omg the number shwas 57!”);
break;

default:
System.out.println(“The number wasnt 57.”);
System.out.println("It was " + nirvanasInt);
}[/code]

You can almost make numerous cases do one thing. Example:

switch(nirvanasInt) {
case 57:
case 10:
case 78:
case 23:
case 99:
System.out.println("It was one of teh lucky numbers!");
break;

case 61:
System.out.println("Oh whoot it is teh 61!!!");
break;

default:
System.out.println("Awwwwwwwwwww");
}

Thats about it. They are normally used where you would have numerous ‘if’ statements of performing actions if an interger was equlled to a certain value.
