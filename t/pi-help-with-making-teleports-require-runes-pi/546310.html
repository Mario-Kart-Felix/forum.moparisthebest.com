sk8rdude461 August 4, 2016, 1:42pm #2

First, collect data for all the teleports. The data you’ll probably need:
Required level,
What runes, and how many,
X, Y, Z coords for each teleport.
Clicking button id for each teleport

The last one is bolded because it is important.

Now, the next part is up to you.
You need to figure out how you want to store this data.
I personally would use an enum.
Ex:

[code=java]

enum TeleportData {
LUMBRIDGE(01, 10, new Item[] { new Item(563, 1), new Item(555, 5) }, new Position( 0000, 1111, 2));
private int buttonId, int levelReq;
private Position teleTo;
private Item[] runesReq;
}[/code]

And then you’d make getters for the data appropriately.

Then you need to make a method to check if they have all the required items, along with it make sure they can teleport (like make sure if they aren’t in a duel, etc).
Hint: make it return a boolean

private boolean canTeleport() { if(!doesntHaveRunes() { sendMessage("Yo get the runes"); return false; } if(inBadArea()) { sendMessage("You cannot teleport here"); return false; } return true; }

Don’t forget to make the method handle the actual clicking…

[code=java]
public void clickTeleport(int buttonClicked, Client player) {
if(!validButton())// Just in case the buttonClicked id is NOT one in the enum…
return;
if(canTeleport()) {
player.getPA().teleport(THE_POSITION_DATA_FROM_THE_ENUM);
player.sendMessage(“Omg you teleported!”);
}

}[/code]

 private Item[] runesReq;

player.getPA().teleport(THE_POSITION_DATA_FROM_THE_ENUM);

}

}

return false;

sendMessage(“You cannot teleport here”);
return false;
}
return true;
}
