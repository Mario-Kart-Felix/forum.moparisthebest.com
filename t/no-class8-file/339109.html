tobias222 July 31, 2016, 4:18pm #9

The JDK compiler (javac) converts .java to .class … we can’t understand the code in .class files but the computer can, so we code in the .java files then compile. Someone has deleted your class8.java because they don’t want you easily getting the ints. And because running the client only requires .class files, (because the computer reads them), it will still work fine without the java file. Decompile the class8.class file, and save it as class8.java, then any errors you get use a class8.java from another client to copy and paste the correct code. (This is because a decompiler will output messy code sometimes which can often contain errors, mainly because of syntax)
Good luck
