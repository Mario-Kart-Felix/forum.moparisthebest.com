hi I have been programming in C for about a 2 year now (eventually moved to ++ after I bought a book a few months later), and for some reason, a few months ago I jumped right into game development because I was so determined to do so. well I have been creating failing projects for awhile and like two days ago I finally decided to try and mock the most simplest game mechanic ever thought of - a youtube video gave me the idea.

it’s a game about a 32x32px block-like guy slides around picking up 32x32px blocks of some type of brown object - preferably brownies, anyways; this is one of my very many attempts at trying to implement my not-so-well-thought-of OO design, so if you have any pointers you could throw I would appreciate it greatly.

there are too many files in the directory so I choose not to post them here in separate tags to save thread space.
i am on a linux distro but the library i am using is not platform dependent so you should be able to run/compile it on any OS.

if you want to use my shitty code go right ahead. no credits necessary:

project folder build1: http://uppit.com/jzrppp57vshk/build1.tar.gz (deprecated) (memory leaks)
project folder build2: http://uppit.com/eolmr8oqihac/build2.tar.gz
libraries: SDL, SDL_image, SDL_ttf, SDL_mixer
you can download sdl at: libsdl.org

modifications/add-ons:

new state system (paused, unpaused, menu, title)

menu buttons (quit, resume)
new title screen
when paused music pauses and when unpaused the music resumes
new logo
you can now toggle the music on/off at the title screen
two people can now play the game together (use wasd to control player 2 on 2-player mode)
