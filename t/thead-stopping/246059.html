I take it you just want to basically ‘pause’ the Thread until noted? Couldn’t you just use Thread.yield(), which will yield the current Thread and allow other threads to execute.

About the restarting a thread part, you can’t legally restart a Thread. It was made explicit in the API. If the Thread object’s run() method has returned, it cannot be restarted. So all you need to do is pause it at the end of run() and restart at a later point of your choice.

You need a semaphore.

In java, an easy way to implement one is with synchronized methods to access a boolean value.

boolean running = true;

public synchronized void stopRunning(){
running = false;
}

public synchronized void startRunning(){
running = true;
}

public synchronized boolean getRunning(){
return running;
}

public void run(){
//assuming you want it to loop?
while(true){
if(!getRunning()){
Thread.sleep(somenumberofmilliseconds);
continue;
}
doStuff();
}
}

Don’t use deprecated methods, they aren’t deprecated because they work. They are deprecated because they won’t work as expected, they are broken.
