system July 30, 2016, 5:04pm #6

You can get the mouse coordinates from outside your application. Here are some examples:

[code]public static final int getX() {
PointerInfo pointerInfo = MouseInfo.getPointerInfo();
return new Double(pointerInfo.getLocation().getX()).intValue();
}

public static final int getY() {
PointerInfo pointerInfo = MouseInfo.getPointerInfo();
return new Double(pointerInfo.getLocation().getY()).intValue();
}

public static final Point getPoint() {
PointerInfo pointerInfo = MouseInfo.getPointerInfo();
return pointerInfo.getLocation();
}[/code]

You need to import the following:

import java.awt.MouseInfo; import java.awt.Point; import java.awt.PointerInfo;

mod_taharok July 30, 2016, 5:26pm #18

If you are clicking inside of a JFrame, you can use the MouseListener interface to track the coordinates of the mouse inside the JFrame itself. You can add the location of the JFrame on the screen to those coordinates to get the mouse coordinates on the screen itself.

If you are moving the mouse and clicking outside of the Window, then you would have to already know the coordinates because you need to use Robot for that.

However, it appears that you are trying to track the mouse coordinates when the JFrame is not doing the moving/clicking. You could always try making a full screen Window that was not being painted and then just receive Mouse Events. I am sure it is possible to do that, but I do not know if it would work.
