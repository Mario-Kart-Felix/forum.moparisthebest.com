stCky August 4, 2016, 2:54pm #9

Dreamweaver, as great as it might seem it will at times insert it’s own bloat code (for legacy and cross browser support etc) and is generally a bit clunky, as Ruby mentioned you should be doing all your styling in a separate stylesheet and stay away from inline styles (<div style=“xxx”) as much as possible. You might find using a lighter editor such as Notepad++ or Sublime Text a bit easier once you have the basics of html/css down - I’ve never once opted for a full blown IDE for any php/mysql/javascript/html/css and we were recommended to move away from Dreamweaver as fast as possible where I studied.

It’s common for people to create a base list of common classes to avoid “quickly adding styles into html”, some examples below:

.left{float: left;}
.right{float: right;}
.clear{clear: both;}

.aligncenter{text-align: center;}
.alignleft{text-align: left;}
.alignright{text-align: right;}

.orange{text-color: orange;} /* easily set text to orange, a common colour for your design possibly */

Other than that and the mentioned broken tags - great work for your first site, you indicate clearly where the user is currently viewing the site and the sections all make sense.
