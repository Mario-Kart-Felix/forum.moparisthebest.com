ExtremeX-Scape August 3, 2016, 3:09am #2

One will not simply not work for ‘everyone.’

There are people who use Mac, and Linux based machines, they cannot run a Windows Batch File. (Unless they have WINE)

You could create a JAR (Java Application Resource) file, and release it for the public to download. That will work for everyone as long as they have Java, because Java is cross-platform.

[quote=“Kidpaparoach, post:6, topic:444811”]Uhhmm, intresting. What would be a good way for us noobs to make it possible for mac users and linux users play any client that we provide?[/quote]say we had a batch file

@echo off java ./Client pause
what you have there is pretty much of the linux/mac version, the main difference is removing any DOS specific directives/commands and throwing a shebang (interpreter-location directive) line at the top. certain commands are a bit redundant too ie. pause since one can just execute in the terminal.
the loose equivalent of the former in linux/mac would be

#!/usr/bin/env bash java ./Client
(the shebang may be variable, using /usr/bin/env should work for os x and most linux boxes)
