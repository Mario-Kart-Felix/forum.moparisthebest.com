unlimitedorb December 27, 2018, 12:54pm #7

It’s not his system. It’s a refactored version of the client’s system.

From JSR-200:

Classes are represented in a form which suppresses their individual constant pools, in favor of a large constant pool that serves the entire archive.

References like the above, sprinkled throughout JSR-200 (The pack200 specification), fit the description in your first post pretty well…

I couldn’t find an open source implementation of unpack200, but a light browse through JSR-200 reveals that you may have refactored a slightly modified version of unpack200. You might be able to locate an open source implementation in something like Apache Harmony if you really feel like you need to see if you wasted your time refactoring “createclassfile”. That being said, I think this is a very nice and thorough effort.

Edit:

This is NOT simply pack200. This is the format of a reference table (the parser calls it ‘JS5’; throw new RuntimeException("Incorrect JS5 protocol number: " + protocolNum); ) where the classes are stored in the pack200-ed format.
