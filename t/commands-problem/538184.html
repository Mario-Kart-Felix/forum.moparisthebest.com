Dan0194 August 4, 2016, 11:24am #4

You can also make an interface to seperate the moderator, normal, admin, owner commands.

So for example if you made a new class named CommandHandler and put this inside

[code=java]public interface CommandHandler {

/**
 * handles all the commands
 */
void handleCommand(Player player, String command);

}[/code]

And then make a new class named like NormalCommands

And put this inside

[code=java]public class NormalCommands implements CommandHandler {

@Override
public void handleCommand(Player player, String command) {
	//All commands
}

}[/code]

And then make a class to manage your commands so put this in

[code=java]public class CommandManager {

            private enum CommandRights {
	NORMAL(0, "NormalCommands"),
	MODERATOR(1, "ModeratorCommands"),
	ADMIN(2, "AdminCommands");
            OWNER(3, "OwnerCommands");
	
	private int rights;
	private String object;
	
	private CommandRights(int rights, String object) {
		this.object = object;
		this.rights = rights;
	}
	
	public String getObject() {
		return object;
	}
	
	public int getRights() {
		return rights;
	}
	
}

    private static CommandsManager commandsManager = new CommandsManager();

public CommandManager getCommand(Player player, String command) throws ClassNotFoundException, InstantiationException, IllegalAccessException  {
	for(CommandRights c : CommandRights.values()) {
		if(player.getRight() <= c.getRights()) {
			Class<?> handleObject = Class.forName(c.getObject());
			Object handleInstance = handleObject.newInstance();
			CommandHandler command = (CommandHandler) handleInstance;
		        command.handleCommand(player, command);
			}
                     return this;
		}

                    public CommandManager getCommandManager() {
                    return commandManager;
                  }
	}[/code]

And then whereever you handle your packet for commands put

You would have to make the classes for AdminCommands, OwnerCommands and ModeratorCommands and just follow the same guideline I used for NormalCommands.

You obviously don’t need to do this but it would be a lot better and cleaner.
