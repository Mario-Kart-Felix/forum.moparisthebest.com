ripzone August 1, 2016, 2:25pm #2

You seem to be confused about APIs. You don’t really “distribute” either OpenGL or DirectX. That term is used to mean the sharing of some kind of implementation.

The implementation is provided by Windows (in form of DLLs), DirectX Redistributables, or your graphics card driver. You just link with the libraries.

If you check the Windows SDK license:

Installation and Use. One user may install and use any number of copies of the software on your devices to design, develop and test your programs that run on a Microsoft Windows operating system.

Distributable Code. The software contains code that you are permitted to distribute in programs you develop if you comply with the terms below …

The software is licensed, not sold. This agreement only gives you some rights to use the software. Microsoft reserves all other rights.

Because this software is “as is,” we may not provide support services for it.

You don’t exactly understand what triton is saying. OpenGL is an abstract library, not something you actually use to do anything. It is the implementations of OpenGL which do the rendering, etc. The library is a concept, the implementation is the code. It doesn’t matter if you credit OpenGL because the implementation you use for it is going to be different between running the program on Mac OSX versus Windows 7.

Now if you are making a game in Java or any other higher-level language, on the other hand, it does start to matter. Java requires JNI bindings in order to communicate with opengl, and these bindings are code, not just abstract libraries. They generally do have licenses that you must abide by in order to incorporate them in your project, but they are easy to follow. Refer to JOGL or LWJGL for more information about how they license their bindings.

Mopman August 1, 2016, 3:07pm #6

The answer to your question is no, because the licences of OpenGL and DirectX permit use in commercial applications. I’m not sure why it was overcomplicated.

It’s spelled out explicitly on the OpenGL licencing page:

Applications developers DO NOT need to license OpenGL.

If you are a software or applications developer, you do not need to license OpenGL. Generally, hardware vendors that are creating binaries to ship with their hardware, or software developers that write an OpenGL driver, are the only developers that need to have a license. If an application developer wants to use the OpenGL API, the developer needs to obtain copies of a linkable OpenGL library for a particular hardware device or machine. Those OpenGL libraries may be bundled with the development and/or run-time options or may be purchased from a third-party software vendor without licensing the source code or use of the OpenGL trademark.
