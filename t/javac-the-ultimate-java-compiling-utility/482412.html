This tool can be used to compile packaged applications without the need to specify every directory within the application to be invoked from the command-line (command-line compiler, Win batch files, etc. for those of you who do not use an IDE.)

What this means is that if you don’t like to use an integrated development environment (IDE) and you prefer the old-school command-line initialization and compiling, this tool is definitively for you, simply because if you download a server that has been repackaged and it does not come with a compiler ready for you, you can implement this tool with a few easy steps.

Say you were programming Hyperion, and you did not wish to use an IDE, and you’re using javac++, this is what your batch file would look like:

java -jar javac++.jar src lib bin pause

There are three parameters that you need to figure out.

  1. Where the source is (the .java files, usually named “src”)
  2. If you have any dependencies (libraries, usually named “dep”, “deps”, “lib” or “libs”)
  3. And finally the output directory. (usually named “bin” or “build”)

Another thing is that you do not need to specify your directory to your jdk, which is sometimes frustrating I can imagine.

Some of you who know about setting the environmental variables know that this can be done much more simple, but not everyone knows how / cares to do so.
