system July 31, 2016, 3:53am #1

I’m releasing v1 of the the RuneCore framework.
This framework is built using non-blocking IO, and is very scalable.
It is (almost) 100% documented for your convenience.

Features:

  • A framework based on multiplexing non-blocking IO operations.
  • Full player handling/etc.
  • Packet skeleton.
  • Login.

Work to do for v2:

  • Player/map rendering and updating.
  • Walking/other primary packets.
  • Content (like NPCs, combat, etc) reserved for v3.

This is only useful to those who can build upon it, nobody can “play” it yet, although it is a server with login.

RuneCore is capable of running circles around the RS2D framework in terms of scalability, although RS2D is more complete, so the comparison isn’t 100% fair. It hasn’t been tested with tons of players yet, as we’re still in the alpha stages of creation. This release is for those who wish to look into building their own scratch servers, or who want to take RuneCore and build onto it.
I only ask that you release any source you make from RuneCore when you are done using it. I can’t enforce this, it is simply a request to keep the open-source spirit of this community alive.

Before you claim leech, know that I am Stormrider.

Authors: Stormrider, Graham.
