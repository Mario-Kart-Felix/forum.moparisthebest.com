Our Aim
Software developers always set goals. Our goal - a revolution. People want to do tasks now. Not wait. And because people want tasks done fast, the robotics industry has been growing at “Far more than the average” (61% demand) Us at FlareSoft have been working hard on our simple communication server “ComServer”. ComServer was designed to be flawless, low memory consuming, and simple at the same time. First we implemented an easy to use loader - where the user could dynamically change what port the server is on, run the server on more than one port, and request administrative actions overboard (non-LAN). Then we created a simple configuration document, for easy changes (such as name changing) on the server. We are still continuing development to make ComServer as simple as possible for users.

What is it?
Robotics can move mouse, type keys, scroll, and more. Unlike other robotic software, ours is a server, meaning others [across non-LAN] can execute commands for the robot to do. It’s a simple solution for remote setup, account creations, and more. You can create queues of actions for the robot to perform and wait for specified command details are provided. There are millions of options to do with your robotic server.
