Milk Bone Framework, is basically the Original Winterlove Framework, with modifications, such as, packaging, class indentation fixes, and a few modifications, works just about the same as the Original Winterlove Framework. I made this so people could read the damned source; Milk Bone Framework was made using netbeans 7.2.1.

First of all, ClientHandler subclass of Server.java file, was Renamed to UserHandler, UserHandler does not contain code in the run() method, from UserHandler. Instead, I created a new class that implements the UserHandler class, since the UserHandler still contains its static fields. The UserHandlerImpl class contains the the run() methods code, to shorten up the class “UserHandler”. Soon in the next version, the Cryption class, might get renamed to “Key” or “KeyCodec”, or “CodecKeyGen”; the login field will have a new class of its own, whilst being reworked. The “stream” class has been renamed to “Stream” for grammar corrections. The “MilkBone” class will serve as a replacement class for the main method to be used in. Soon, I will modify the “Server” class’s constructor to “Server(boolean membersOnly, int port)” which will allow to set whether the server is a Members Only server, and allowing the server to be hosted on different ports while running the one simple class.
