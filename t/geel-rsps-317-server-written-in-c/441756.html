Hey, so, I’ve been working on a server written in C# recently. This may be the first (I’m pretty sure it’s not) to be written in C#, and I’ve been making a lot of headway with it. It uses .NET 4.0 and SQLite.

Reason to use C# over Java:
-VS2010 is easily 10X better than Eclipse
-Reflection
-Anonymous methods (lambdas). This may not seem important, but I absolutely love anonymous methods. In order to write an “anonymous” method in java, you need to create an entire interface and instantiate it. In C#, it’s so much easier. It will make the job a lot easier.
-Speed (debatable)
-I have much more knowledge in C# than Java
-Writing it in C# means I can’t just copy paste entire classes; I will be forced to entirely understand the protocol.

Expected features:
-External assembly support: This will allow somebody to write their own assembly (DLL) and stick it in their folder and the game will run it. This will allow them to manage minigames, skills, NPCs, etc. OUTSIDE of the “main game.”

-Dialogue system: This will be very easy to use, and will make heavy use of anonymous methods (see?). In just a few lines, one can write entire conversations.

-“Individual” NPCs/objects: Not very “revolutionary,” I imagine, but each NPC/Object (that needs to be) will be its own class–no massive switch statements for no reason.

-SQLite player saving: Saves players in a database instead of individual text files; makes it easy to perform batch operations on players.

-Unique method of cache editing:This is more of a client feature, but it allows me to modify the cache without actually touching the cache files themselves.

What’s done so far?

-Logging in
-Moving around
-Player update loop (also see other players)
-Chat
-Logging out
-Player saving
-Item actions
-NPC management
-NPC update loop
-Dialogue
-Item cache editing
-NPC cache editing
-Object cache editing
-Object management

I recently added Item cache editing. It’s not true cache editing, as I don’t actually touch the cache files. Instead, I have a separate file that the client loads all item definitions from (as well as from the cache). When the client tries to get an item, I look to see if the item is in the new cache file. If it is, I return it.

This also allows me to create NEW items very easily.
