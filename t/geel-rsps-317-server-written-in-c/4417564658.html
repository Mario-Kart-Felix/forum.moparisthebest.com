In Java you can use java.util.ServiceLoader<?> to find and load all implementations of the given generic type, provided that they’re in the classpath (and their constructors are public with no parameter arguments - well suited for an Abstract Factory). So you don’t have to use Reflection directly (only indirectly via that class). This approach is usually called SPI (short for Service Provider Interface), and it supports reloading on the fly (although I doubt it uses HotSwap).

Edit: I forgot to mention that you need to have it in a Jar file, with a META-INF/services/[INSERT_FILE_WITH_NAME_OF_GENERIC_TYPE_HERE_AND_NO_EXTENSION]. In that file you need to provide the implementing classes fully-qualified class-names, one per line (comments are possible too).
