just an update, its a 525 server and ive looked in ALL folders and it doesnt have a config.jar

Include all details within your original post otherwise you’ll get a thread looking like this. The details you should include with any help thread are:

The server revision, the base you’re using, a detailed description of what you have done to solve the problem and the errors that occurred.

Anyways, in a 525, depending on packaging, you will have to do some investigative work. I’ve never worked with a higher revision than #317. My suggestion is to get a program capable of doing something known as a call hierarchy. What that means is you can search methods and when they’re used. This would be useful for you because you could search the method that adds the experience to the players skill and found out when it is specifically being used in combative situations. You could then hardcode the changes into the file or you could do something similar to what these lower revision servers are and set-up an EXP rate within a configuration class, so everything can easily be changed at a later date.
