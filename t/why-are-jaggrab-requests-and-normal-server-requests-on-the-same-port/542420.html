They aren’t. The client will try to connect to the http server (port 80) to fetch the files. If that fails, it switches to port 43595 (and if that fails, it switches back, and so on). For “on demand” files, it uses the game port 43594, and those requests aren’t JAGGRAB.

The client will first try to connect to the server using port 80. If it’s successful, the client will then process requests using the HTTP protocol. If the HTTP service is unavailable, it will then connect to the server on port 43595, and requests will be made using the JAGGRAB protocol.
