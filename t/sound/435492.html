Blubbers August 2, 2016, 4:51pm #6

After some research i found out that it is possible to capture the soundoutput of the soundcard via native code. This is very system and hardware specific, so i wouldn’t suggest that.
Another option would be intercepting the waveIn/waveOut calls of the applicitation (in this case the JVM) via C++/C# which also is very system specific.
The sound analysis in both options should best be done in native code, but it think it’s just not worth the effort because it is not very portable.
