Wintercanon August 1, 2016, 7:38am #54

@Rix_ originally I thought his plan was to not include the server side to the game with the client. So that clients would connect to the server but still experience a single player game. The advantage of this method is player data could be stored remotely to prevent character modification, as well as game updates that do not require you to re-download the game.

What I was suggesting was, if he were to use that method, instead of building a player list and looping through the updating of each player, create an instance of the world for each individual player. That world would implement Runnable and just handle local updating and npcs for the user the world is instanced to.

About the dynamic content of his server. If it is going to be single player it will need a lot more depth. More random actions occurring, as well as a world more interactive, that possibly changes slightly with player decisions. To implement this you would need to have event listeners and a fair few threaded processes running asynchronously so that the game can have a wide array of actions ticking at once.

  1. No emulator uses any of Jagex’s code(with the exception of the stream class, but blakeman8192 fixed that(RSStream)).
  2. No one here distributes the client. They upload it to another site, and have that site distribute it. That is totally legal. But if one were to host the client on their personal computer, and have users download it from their computer, that wouldn’t be legal. Basically, it’s like if you search “child porn” on Google, and a link to http://www.childprawns.net/ or whatever showed up. Google wouldn’t receive any sort of legal issues, because they’re just providing a link to the content, not providing the content itself.
  3. Our goal here isn’t to “rebel against Jagex”. In making a RuneScape Emulator, you’re doing nothing against Jagex. Those who develop/play RuneScape Emulators either currently play RuneScape, or have ceased to play RuneScape. The goal of this site was to allow programmers to create a RuneScape Emulator where they could do whatever they wanted. Things they couldn’t do normally(Note: The original site, mitb, was created for RuneScape cheating. Emulators were created to provide an environment in which people could test their bots, SCAR scripts, etc.). Emulator development was then branched off into it’s own location.
