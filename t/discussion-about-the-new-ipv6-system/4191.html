[quote=“David, post:6, topic:4191”]IPv6 is overkill IMO.

What’s wrong with 000:000:000:000:PORT ?[/quote]
RFC 1883, which, AFAIK, is one of the earliest, if not the earliest, specifications for IPv6, was released in 1995. Today, we do not expect to have IPv6 completely unrolled for at least a couple more years, and probably longer. Meaning, complete time from base specification to full implementation is going to border on 15 years, as a fairly conservative estimate.

They’re overshooting on purpose. They have no idea how rapidly connectivity is going to expand. Reimplementing the protocol which transmits damned-near-everything used on the internet today is not a task anyone wants to have to deal with again any time soon. If IPv6 is still alive and well in 50 years, that’s probably a good thing.

Read about IPv6 here: http://playground.sun.com/ipv6/INET-IPng-Paper.html
The current ip is in the form 00.00.00.00 (or 000.000.000.000 in decimal) <- each block is 0…FF = 2^8
An IPv6 address is 0000.0000.0000.0000.0000.0000.0000.0000, each block is 0…FFFF = 2^16
Old address 32 bits, new 128 bits.
BTW, IPv6 isn’t exactly new though
