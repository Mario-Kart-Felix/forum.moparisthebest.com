ExtremeX-Scape August 4, 2016, 1:47pm #16

The map index file (within the map archive, 4) contains attributes of a terrain:

  • coordinate hash code
  • object file id
  • terrain file id
  • preload flag

Knowing this information you can parse terrains and object information for every position on the map (that isn’t hard-coded, obviously)

  • terrain parsing:

    • reads attribute ids from the terrain file that denote when that terrain should be marked (0 and 1) using your traversal map (this is used for clipping), you can mark blocked tiles and bridges using this information.

  • object parsing:

    • reads an unsigned byte which is packed with information about the object (type, orientation), this can be used to mark walls, larger objects etc not traversable.

The client contains all of this information and how this data should be parsed, this wa s a very basic rundown of how it works. If you have any other questions feel free to ask.
