Bad Standards!

Try that code. As you can see, it works, but if you ever wanted to change something in it, then it would be a hassle. That is just a very small peice of code I concocked very quickly, but you should see other people’s cose. 300 or 400 lines of code, and it doesn’t have good coding standards. Here is a good example of the script I just showed you, with good standards.

Good Standards!

It’s very simple. Here are the rules.

[ul][*] Tree your code AKA display the code in a neat format. Example…[/ul]

Begin
 Repeat
  WriteLn('wow');
 Until(False)
End.

Another Example only this time we used Tab to tree the code:

Begin
     Repeat
           WriteLn('wow');
     Until(False)
End.

[ul][] Try to comment most parts of it, that way it’s easier to find where you left off from and it’s easier for someone else to debug your code.
[] Try to make everything user friendly. Take the first code’s instructions on the Const setup. It wasn’t too descriptive.
[] Use procedures instead of putting all your code right in between the Begin and End. commands.
[] Use ; at the end of each command such as… WriteLn(‘asdkjaskldj’); and SendKeys(‘asdasdasds’);
[*] Try to keep functions and procedures in the proper case. Such as the procedure WriteLn. Don’t write it like writeln. Same with IntToStr and StrToInt and the 1000s of others.[/ul]

I hope you learned something.

~Ron

PS: Yeah I know the Const doesn’t affect the script at all. It was just there to show you, how you can make it neat and show you how to setup the instructions for other people.
