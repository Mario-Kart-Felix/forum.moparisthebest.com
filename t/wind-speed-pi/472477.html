the initial idea of this was: every height level increased, the more damage you would be dealt (depending on wind speed). the higher your height level the faster the wind speed, causing more damage. i was going to add rewards, etcetera. but to be quite frank, i couldn’t be bothered.
the good thing about this is, that it comes with an interface that’s already in the RuneScape cache.

[code]package rs2.model.content;

import java.util.HashMap;
import java.util.Map;

import rs2.model.players.Client;

/**
*

  • @author Rhubarb/Int Bauk

*/
public class WindSpeed {

    public enum Wind {
            SLOW("Slow", 0, 0),
            MEDIUM("Medium", 1, 1),
            FAST("Fast", 2, 2),
            EXTREME("Extreme", 3, 3);
    
            String windType;
            int windSpeed, heightLevel;
            
            Wind(String windType, int windSpeed, int heightLevel) {
                    this.windType = windType;
                    this.windSpeed = windSpeed;
                    this.heightLevel = heightLevel;
            }        
            
            public String getWindType() {
                    return windType;
            }
            
            public int getWindSpeed() {
                    return windSpeed;
            }
            
            public int getHeightLevel() {
                    return heightLevel;
            }
            
        	public static final Map<Integer, Wind> wind = new HashMap<Integer, Wind>();
        
    	public static Wind forId(int id) {
        		return wind.get(id);
    	}

    	static {
        		for (Wind en : Wind.values()) {
            	wind.put(en.getWindSpeed(), en);
        		}
    	}
    }
    
    /**
     * The interface used for the windspeed.
     */
    public static int INTERFACE_ID = 12278;
    
    /**
     * The disaster that occurs when facing the wind, dependant on height. HORRIBLE SWITCH.
     * @param player
     * @param speed
     */
    public void handleDisaster(Client player, int speed) {
            Wind wind = Wind.forId(speed);
            if(wind.getHeightLevel() == player.heightLevel && player.heightLevel == 1) {
                    player.dealDamage(1);
        		player.handleHitMask(1);
        		player.getPA().refreshSkill(3);  
            } else if(wind.getHeightLevel() == player.heightLevel && player.heightLevel == 2) {
        		player.dealDamage(2);
        		player.handleHitMask(2);
        		player.getPA().refreshSkill(3); 
            } else if(wind.getHeightLevel() == player.heightLevel && player.heightLevel == 3) {
        		player.dealDamage(3);
        		player.handleHitMask(3);
        		player.getPA().refreshSkill(3); 
            }
            player.sendMessage("Well done! You've managed to withdraw the wind that hit you!");
    }        

    /**
     * Handles the wind speed.
     * @param player
     * @param speed
     */
    public void handleWindSpeed(Client player, int speed) {
            Wind wind = Wind.forId(speed);
            updateInterface(player, speed);
            player.sendMessage("The wind speed that just hit you was " 
            + wind.getWindSpeed() +"/" + wind.getWindType() + ".");
            handleDisaster(player, speed);
    }
    
    /**
     * Updates interface.
     * @param player
     * @param speed
     */
    public void updateInterface(Client player, int speed) {
            Wind wind = Wind.forId(speed);
            player.getPA().walkableInterface(INTERFACE_ID);
            player.getPA().sendFrame126(wind.getWindType(), INTERFACE_ID);        
    }

}[/code]

Seems quite alright, although I keep seeing people write the enum name again as a param for the object…This is just unnecessary, if you name the enum properly you can just do toString().toLowerCase().replaceAll("_" " ");

Also if you’re going to keep using this pattern

SLOW(0, 0),
MEDIUM(1, 1), //etc

then you can just skip that, but keep the enum if you would like more of an OO approach, you would do

private enum Type {
SLOW,
MEDIUM,
FAST,
EXTREMELY_FAST;
}

then to get the windSpeed and heightLevel you would do

enum.ordinal()
