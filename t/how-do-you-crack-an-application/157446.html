I know how, but it’s pretty hard.
EVERY serial has it’s own binary sequence or key sequence. The serials have binary code in them, so each key stands for something. When a Key Generator community figures it out, they make a generator for it.
ex:

7-character key:
A4C3-76E

Each of the characters mean something, or have a specific order.

Get it?

I can’t really explain it, it’s kind of hard to explain.

Hoped I helped…

EDIT:
For instance:

Serial:
A4C3-76E

Product Key:
1G0T-AE8

Then, out of those keys, they make something out of it, such as:

First character of the serial(A, first letter of the alphabet ) + first character of product key(1): B

Second character of serial(4) + second character of product key(G, 6th letter of the alphabet) - 3: 7

Third character of the serial(C, 3rd letter of the alphabet) + third character of product key(0) + 3: C

Fourth character of the serial(3) + fourth character of product key(T, 20th letter of the alphabet) x 2: 46

Fifth character of the serial(7) + fifth character of product key(A, 1st letter of the alphabet) + 5: 13

Sixth character of the serial(6) + sixth character of product key(E, 5th letter of the alphabet) x 3: 33

Seventh character of the serial(E, 5th letter of the alphabet) + seventh character of product key(8): M

So:
B7C4-613-33M

learn to read assembly
attach a debugger to your program you want to crack such as ollydbg (good debugger) not as hard as softice.
http://www.ollydbg.de/

Set a breakpoint at the invalid string such as “incorrect serial please try again”.
Then when you start the debugging process enter some serial and press Okay or w\e button
Then it stops at some breakpoint address
Then from there just learn what every opcode does
Finally you will get the mathematical formula to create a keygen.

But most keygen makers just rip the complete code starting where the registers set in like your name or w\e you put in to get a serial generated and all the way down to where the function stops and just call it in their C++ application using something like

int Keygen(inputNameCombinationOrSomething) {
  __asm{
      the ripped code goes here
  }
  return GeneratedCode
}

If you are interested I suggest you learn assembly you would also become one of the best programmers ever if you master it.
