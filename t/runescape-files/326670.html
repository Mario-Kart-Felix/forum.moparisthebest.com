yes they do. pack200 is a gzipped pack200 version of the client jar. that is why once you unpack it, the classes etc are different… it uses a sorting algorithm on constant pool and fields and stuff afaik (oh and the gzip magic number is stripped).
js5 is a cute little format they use based on their ‘js5 metadata tables’. the actual file is optionally compressed because it uses their homebrew container format. the file starts with the metadata table and then has trailing data behind it, which includes the ‘packaged’ class data and etc. it’s pretty much their own alternative for pack200 for pre-j2se-5 users
