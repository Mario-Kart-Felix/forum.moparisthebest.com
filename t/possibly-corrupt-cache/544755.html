You need to generate a new 2048 bit public and private key pair, the public key goes in the client (typically in the stream class) and the private key pair goes in the server (typically the RS2LoginProtocolDecoder class)

Nikki released a tool which generates these keys and pretty-prints them, you can find it on Rune-Server.

sk8rdude461 August 4, 2016, 1:24pm #10

Every little error you get you instantly assume the cache is corrupt. It’s funny.

If you want to test if the cache is corrupt, load it in Tom’s Cache Suite and open everything 1 by 1 (from the initial menu, you don’t need to literally open all items). If something doesn’t load, it’s corrupt. If everything loads, you’re fine.

neitiznot.cache.clip.region.ObjectDef.getObjectDef(ObjectDef.java:9)

Is your error. Looks like it’s trying to get the object information from the maps and can’t because for some reason something is null that shouldn’t be.

Or, better yet, post your actionhandler, I’ve seen a similar error before.
