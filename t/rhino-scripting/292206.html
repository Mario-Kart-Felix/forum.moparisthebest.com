BAsh July 31, 2016, 9:46am #17

I built an ASM based code transformer (think: not remover / shift simplifier / etc) with javascript executed by Rhino for condition checking and to do the actual transformations. I had around 15 transforms each with an average of 3 or 4 javascript based conditions and a 15 line block for the transform itself.

It takes about 100-300ms to compile all of that javascript (only short pieces of code) but reduced execution time for each transform from 4 or 5 secs to sub 100ms.

Bottom line:

  • It’s no where near 0.5secs to compile one script.
  • Subsequent runs when the classes are already loaded / JIT compiled are much faster.
  • Unsure about how Rhino works but the scripts are probably compiled anyway or are parsed the same.
  • The cost of caching an instance of a runtime generated class is minimal.
