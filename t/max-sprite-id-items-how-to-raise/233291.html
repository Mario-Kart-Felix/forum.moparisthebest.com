It’s not really sprites or they inventory items, it’s a sprite id for calling settings from animation.xml for one or multiple items. Settings about what color it is, which model to load, which gender it is to.

Example:

  <AnimationDef>
    <name>partyhat</name>
    <charColour>16755200</charColour>
    <genderModel>0</genderModel>
    <hasA>true</hasA>
    <hasF>false</hasF>
    <number>0</number>
  </AnimationDef>

Let’s say one item is a party hat, to get a new color instead of one of the other already made colors, it will load the settings for “sprite” id 254 (which is the settings for a orange party hat) from animation.xml.

If i add one that has id 256+ it will simply restart and load from the buttom, as if it were loading the “sprite” id like this: id requested - max id value (256 in this case).

So it’s actually loading the settings for a model and not a sprite.

As I said, just change the data-type to short or int… Then you should be able to load many more. A byte is made out of 8 bits, while a short is made out of 16 bits (2 bytes), and an int is made out of 32 bits (4 bytes or 2 shorts).

What I meant with sprite, I of course meant the ID as reference to them in the cache. I also believe every ID is just a pointer to a set of sprites, which all is determined by the current direction your character is pointing to.
