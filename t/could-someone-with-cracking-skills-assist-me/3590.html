yetanothercheater July 29, 2016, 8:21pm #10

Most programs are compiled. This means that whatever code you wrote has been converted into machine language that the computer processor actually understands. It’s possible to disassemble programs, which means getting the assembly language version of the source code. However, to get the source code as the programmer wrote it (unless he used assembly) is generally hard or impossible. There are decompilers for some languages, which do this job to some degree, but it’s much harder than disassembling. For example, it’s unlikely that a decompiler will produce the variable names that the original programmer used, since the compiler just changes them to memory locations when making the executable.

Some programs are not compiled. Rather, the source code is combined with an interpreter that basically converts the code to machine language as you’re running the program. This is usually slower than compiling, so fewer programming languages use this method. It might be possible to retrieve the original source from interpreted programs, but I don’t think it would be a trivial task.

I’m not sure why you’re so worried about someone being able to retrieve the source code of your program. Crackers usually use the assembly language output of a disassembler, along with a debugger, to modify programs. Converting the code to a higher-level language is pointless if you know assembly, as you’d be doing much more work than necessary.

I’ve done a few quick searches for Blitz3D decompilers and found nothing but a fake one. This doesn’t mean that one does not exist, but I think it would have popped up in some of the results by now. Have you thought of asking on the official forum (if there is one)? The people there would probably know.
