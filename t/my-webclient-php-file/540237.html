Just a brief question, you do understand that you can include HTML on a PHP page, right? If you must have the HTML code mixed with the PHP code, for whatever reason, here is a better option.

[code]<?php
$webclient = ’

Applet failed to run. No Java plug-in was found. '; ?> Title <?php if(isset($logged)) { echo $webclient; } else { echo "You cannot see the webclient! Please Login."; } ?> [/code]

However, the best option, by far, would be to just keep the PHP and HTML separated…

[code]<?php
// All of your PHP crap.
?>

[/code]

No point in using PHP for outputting HTML information, only use PHP when needed, also, escaping characters in the HTML code like that is redundant. Also, you can take any page with .HTML extension, and make it .PHP and it’ll give the same output. Ex:

index.html changes to index.php --> Nothing changes except file name, and the file can now support PHP code.
