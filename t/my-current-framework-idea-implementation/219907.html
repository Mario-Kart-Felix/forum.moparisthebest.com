I suppose the other alternative is to make some form of EventFactory that maps Strings to Event implementations.
EventFactory.getEvent(“Eventname”); would return a new instance of the correct Event.

So that we can avoid reflection, we can have developers add a static method to each implementation that EventManager can call - this method can return a new instance of the particular event based on whatever arguments.
