At my computer the time between each mouseMoved event was 8 ms. This was an average over 30 seconds of moving the mouse fast around. It is the same for going slower, but obviously ifyou go too slow it will be longer between each mouse event.

I assume its the same with MouseDragged events…

KBot PRO’s mouse uses splines to generate magnification points. Each point has a random magnification level and the end point (target) has a big magnification which gets smaller the closer you get. The mouse then follows the splines major curves, but does rarely follow the spline. The mouse randomly do a drop effect as well (simulating lifting the mouse and replacing it another place on the mouse pad).

I still use bezier splines for this. The start point remains the same but the end point can move under mouse move. The bot can therefor follow/hover over a moving object in game
