Features so far:

OBJ Viewer (View OBJ models)
JAG -> OB3 Converter (Extracts the OB3 models (Jagex’s RSC model format) from the .jag cache file)
OB3 -> OBJ Converter (Converts jagex’s model format into a popular model format (Wavefront OBJ)) - however this won’t come with this tool to make it harder for people stealing each others models
OBJ -> OB3 Converter (Converts the editable model format back into jagex’s one)
OB3 -> JAG (Import the OB3 models back into the jagex cache)

Basically everything you need to import your own models. Popular modeling tools can convert many different model formats into OBJ format (meaning you can import WoW, rs2, quake models into the RSC client etc)

However from what little research on this modeling stuff i have done (yes i’m new to this) OBJ does not directly contain textures/colors, and does that externally. Right now everything works except proper colors/textures on the models you create. The green zelda above has been temporary hardcoded to be green, whilst i still work out how jagex’s model color/textures are stored.

If worst comes to worst ill scrap the OBJ format and convert OB3 to another model format that includes color, and/or get the RSC client to load a different format (not jagex’s OB3)

Ill reply to this thread with what progress i make.
