Hey everyone, I just finished my C# Client updater. Why is mine better then the ones built into the clients? Because, it has to connect to the server EVERYTIME. Which means, a longer loading time, people get angry. Basically, if your client is outdated, you just click the auto-updater, and bam. It downloads the client file and your done.

What it exactly does:
It compares the current version (an online text file you have uploaded) with a local one (settings.txt). If the local one is less than the current one, it updates, if its equal too, it simply outputs a message saying “client is up-to date”

How-to use:
Alright so go to the settings (File > Settings) and from there a little box should popup with 4 text fields. Download path, file name, Version path, current version. Simply fill out the information. Download path, is where you would download the file offline. File name, is the name of the file. Version path, is the online text file that contains the current client version. Version, whats the current version of the client you are making/releasing?

Help menu:
Click on one of the buttons under “Help buttons” and text will be displayed.

For Client owners (for online text file): Upload your copy of settings.txt to your version directory. That IS the online text file. It will automatically update the text file version number. And before all of that, make sure you did all of the above!
