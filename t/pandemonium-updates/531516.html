 Pandemonium is a RuneScape 317 emulator I've been writing for about a week now. I plan to develop this to the point of being able to implement content. Pandemonium is based off of Fox which I posted in the download section a few weeks ago. The same principals apply here, it will be a light weight framework that makes use of the NIO reactor pattern. I'll try to update this thread whenever I make a step in the development. The framework is multi-threaded and is based around the idea of keeping it simple and easy to follow without sacrificing performance in the process. I'd like someone to be able to pick up this server and within twenty minutes understand how the internals work without running through 50 polymorphic classes.

Content:

[ul][li]Multi-Threaded Backing[/li]
[li]Entity Based[/li]
[li]Console[/li]
[li]Player Updating[/li]
[li]Player Movement[/li]
[li]Packet Obfuscation & Deobfuscation[/li]
[li]Concurrent Approach[/li]
[li]Selector Reactor Pattern Implementation[/li][/ul]

 When writing in an IDE i've often found it a little hard to actually keep track of whats being debugged and printed since it is usually crammed in a small console in the bottom corner. For this purpose I decided to write my own basic one.

In my opinion this gives a better view when debugging code or just trying to keep track of any run-time attributes.

I’ve recently made it to the point of global login and player updating, by this I mean that everyone in your region will see you when your added into the world and can interact with you as such. The release I posted lacked this feature.

Object managing is practically finished like been stated above by Vix.
It uses object based updating.
Objects hold a list of players that have been updated so it doesn’t update every time a player loads the region.
This way objects like tree stumps will be visible for all players logging in, entering the region or already being at the region.
Timed objects are also supported for e.g. tree respawning.
