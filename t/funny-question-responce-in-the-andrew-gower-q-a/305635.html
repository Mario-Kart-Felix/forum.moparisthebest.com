a few months ago, while i was making my pokemon client in java, in which the walking is done through arrow keys, i realized that there is a downside, which may appear as a bug, but isnt. it also happens in the current runescape client with the ‘map rotation’. if you load the runescape client through a frame (ie using main(), or a loader, etc), login, hit any arrow key and leave it pressed then click the window title bar (while still pressing the arrow key), then release the arrow key, and release the click, the client will keep on spinning as if you were holding the arrow key. that is because java has no control over the window’s title bar, and does not receive events from it. therefore, if they implement walking in it, this will also happen (as it did in my pokemon). even though it is not a bug, it sure seems like one, and i find it annoying
