This is a nice simple loader derived from the work me and SilverLining did on Generibot. It’s a multi-tabbed runescape loader, and that’s about it xD

Features:

[ul][li]Multiple runescapes open at once, without using an unsigned client[/li]
[li]Auto-updating, it checks for runescape updates at startup by querying the runescape servers[/li]
[li]Plugginable interface for things such as combat calculators, etc.[/li][/ul]

Plans for the future:

[ul][li]A Plugginable interface for things such as combat calculators and other such rubbish[/li]
[li]More control from the config file[/li]
[li]Clean the source![/li][/ul]

So yeah, this is my first release in a while, feel free to post any criticisms as long as they’re constructive.

PS: Forgot to mention, loader is modified so that it downloads runescape.jar and loads from that, this of course means that you can modify the runescape jar prior to loading; But I’d discourage that unless you know what you’re doing.

UPDATE: Updated to version 0.21, now includes plugginable interface, and some bugfixes.

package com.rsloader.plugin;

import com.rsloader.control.Controller;

/**
 * <p>The Plugin interface allows people to write plugins that can modify the loader or add extra functionality, there are no limitations apart from the limitations of the JVM</p>
 * @author newbiehacker
 */
public interface Plugin {
    /**
     * <p>Returns this plugins name so that controllers can display it to the user in some way</p>
     * @return This plugins name
     */
    String getName();

    /**
     * <p>Requests this plugin to initialize and do whatever it needs to before being attached to the Controller</p>
     * @param c the controller that we will be attching to
     * @return Whether we could initialize successfully or not
     */
    boolean init(Controller c);

    /**
     * <p>Requests this plugin to clear all resources it owns before it is detached from the controller</p>
     */
    void stop();
}
