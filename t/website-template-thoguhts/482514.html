Some serious problems with your code:

[ul][li] tags, deprecated in HTML5, kill them. Use CSS property text-align instead.[/li]
[li]Opening and closing of tags, there are numerous unclosed tags in your code, if you indented correctly you should be able to see, there are a couple of center tags (which shouldn’t be there in the first place) that have no ending tag, and a random there too. I say random because you don’t appear to have any of the other HTML5 dividers? , , etc.[/li]
[li]Self closing tags, your

's, and 's should be closed as written, with a succeeding space and forward slash. See here for a full list of self closing tags.[/li]
[li]List items not nested within a list object such as
    or
      .[/li]
      [li]Image alt attribute not used, it is a WCAG requirement.[/li][/ul]

      Run your code through a HTML validator and you’ll likely see most of these errors crop up.

      Content wise, you seem to have the same paragraph repeated in the operating system section for download? If you’re going to include the same paragraph three times you should probably present it differently.

      Also you may want to fix the nav bar, it seems to be wrapping on my screen (1440x900 res.) to the next line.

      Regarding the footer, I’d just stick a 20px padding on the bottom and it’ll already be 100% better.

      RSCShadow August 4, 2016, 2:43am #3

      Your site width is way too wide. You should look into responsive design or stick with a consistent width less than 1024px.

      Self closing tags, your
      's, and 's should be closed as written, with a succeeding space and forward slash. See here for a full list of self closing tags.
      They don't actually need the forward slash, unless you're doing XHTML. In HTML5 is perfectly valid. Though I personally use self-closing tags as it makes more sense to me.

      EDIT: I noticed you’re using more than one CSS framework (bootstrap, 960). I recommend Bootstrap as it has nice grid / responsive classes you could make use of. I don’t even see you utilizing either’s grid system so I don’t know why you even included 960.

      http://twitter.github.com/bootstrap/scaffolding.html

      Also I recommend making use of a CSS spritesheet for the social media icons below your header.
