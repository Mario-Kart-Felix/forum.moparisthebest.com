This is just data for easy use of equipment slots, rather than manually adding in the item ids and slots to your server to equip into a different slot.

The equipment data file: equipDefinitions.xml

You’ll have to load the XML file and store the data in your server yourself, cbf to explain fully how to do it. I use XStream to load my xml files.

You can add more items to the file easily, just add another:

<equipDefinition> <id>#</id> <slot>#</slot> </equipDefinition>

The slots:

HEAD: 0 CAPE: 1 AMULET: 2 WEAPON: 3 CHEST: 4 SHIELD: 5 LEGS: 7 HANDS: 9 FEET: 10 RING: 12 ARROWS: 13

The EquipDefinition class:

[code=java]/**

  • Represents an equip definition.

  • @author PrimaDude.
    */
    public class EquipDefinition {

    /**

    • The item id of the equipment.
      */
      private int id;

    /**

    • The slot to equip the item in.
      */
      private int slot;

    /**

    • Gets the item id of the equipment.
    • @return The item id.
      */
      public int getId() {
      return id;
      }

    /**

    • Gets the slot to equip the item in.
    • @return The slot.
      */
      public int getSlot() {
      return slot
      }

}[/code]

The EquipDefinitions class:

[code=java]package org.iClarity.rs2.util.items;

import java.io.FileInputStream;
import java.io.FileNotFoundException;
import java.util.logging.Logger;
import java.util.List;

import org.iClarity.rs2.util.XStreamConfig;

/**

  • Represents an equip definition.

  • @author PrimaDude.
    */
    public class EquipDefinitions {

    /**

    • A list of the equip definitions.
      */
      public static List equipDefinitions;

    /**

    • Logger instance.
      */
      private static final Logger logger = Logger.getLogger(EquipDefinitions.class.getName());

    /**

    • Loads the equip definitions.
    • @throws FileNotFoundException If the file cannot be found.
      */
      @SuppressWarnings(“unchecked”)
      public static void loadEquipDefinitions() throws FileNotFoundException {
      logger.info(“Loading equip definitions…”);
      equipDefinitions = (List)XStreamConfig.getXStream().fromXML(new FileInputStream("./data/equipDefinitions.xml"));
      logger.info(“Loaded “+ equipDefinitions.size() +” definitions.”);
      }

    /**

    • Gets the specified equip definition.
    • @param id The item id.
    • @return The equipment definition.
      */
      public EquipDefinition get(int id) {
      return equipDefinitions.get(id);
      }

}[/code]

Call the loadEquipDefinitions() method upon server start-up, and change the directory that the method checks for the equipDefinitions.xml file if need be.

Usage:

EquipDefinition equipDefinition = EquipDefinitions.get(itemId); if (equipDefinition == null) { return; } itemEqupMethod(itemId, equipDefinition.getSlot());
