Okay, first I will explain the whitelist:

A list of usernames. If a username is on the list then they can log onto the server. If the username is not on the list they will be unable to log onto the server.

Advantages
You can play with only selected people. This stops random players coming in and trashing/griefing other people’s work. Useful for small servers that don’t want “outsiders” joining their game.

Disadvantages
Whenever a new player wants to join, the host of the server must open and edit the file whitelist.txt and add the players name. This can be a pain if the host wishes the server to be open to other players. This is also a problem when removing players. We would have to wait for the host to remove the name from the whitelist or wait until they are banned.

Is sense made?
