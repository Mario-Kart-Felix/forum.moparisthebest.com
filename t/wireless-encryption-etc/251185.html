moparisthebest July 31, 2016, 4:06am #3

Secure sites that use SSL should be secure even over an unsecured network. Wireless encryption actually restricts access and encrypts the data you send, problem is most encryption mechanisms are very easily cracked. Someone with the right card and one of many sniffing/cracking programs that abound for free on the internet can have access to your network within a few minutes.

If you really want the packets you send flying through the air to be truly safe, do what I do. Route all of your traffic through SSH to your router (assuming your router has an ssh daemon running, if not any server that you can log into with ssh will work). This is much easier than you would think, with the proper command ssh can start up a socks5 proxy on your local machine. Set your browser to use that as your proxy, and your traffic is automatically encrypted with the uncrackable (unless you are the NSA) SSL encryption and perfectly safe in between your computer and wherever you had ssh connect to.

I have a script written for linux that runs every time I log in to automatically set up this connection for you that I can post later when I’m on my laptop if anyone is interested.

Meanwhile, refer here for more info:
http://wiki.freaks-unidos.net/weblogs/azul/firefox-ssh-tunnel

In windows you can use Putty to set up the socks5 proxy through an ssh server. Google for a tutorial.

Here is my script, with personal details removed (if you wonder why I keep trying, it’s because when you connect wirelessly you obviously have to set up the connection to the router before you can ssh into a server):

#!/bin/sh
killall ssh;
killall ssh;
echo "starting";
while true; do
	# make sure you use an IP instead of a domain if you don't want to do a DNS lookup in the open
        # also, if you run this automatically you have to set up ssh to not require a password, google that as well
	ssh -fND localhost:1080 user@ip > /dev/null 2> /dev/null;
	if [ -n "$(netstat -na | grep "LISTEN" | grep ":1080")" ]
	then
		break;
	fi
	echo "failed";
	sleep 5;
done
echo "success";
