I’m not sure if I understand the question, but if you get the InputStream from a Process p and read it blocks (waits to return) until there is input to read. So if you take this approach there shouldn’t be a problem with the delay.

Process p = ....;
InputStream i = p.getInputStream();
// use i to read

Edit: you’d also have to grab the output stream and write individual moves to it in a separate thread. You’d have to synchronize this properly.

slavemaster August 2, 2016, 4:55pm #4

If I understand your question correctly, you should want to use a thread dedicated to this I/O to block on reading from the process output to accommodate for the delay. Processing it could be delegated back to whatever thread you deal with logic through.

Note: it might be beneficial, presuming Houdini emits text, to use a Reader implementation e.g. BufferedReader over the stream acquired via Process.getInputStream().
