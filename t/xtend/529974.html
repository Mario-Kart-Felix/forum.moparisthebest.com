Xtend is a general-purpose high-level programming language for the Java Virtual Machine. Syntactically and semantically Xtend has its roots in the Java programming language but focuses on a more concise syntax and some additional functionality such as type inference, extension methods, and operator overloading. Being primarily an object-oriented language, it also integrates features known from functional programming, e.g. lambda expressions. Xtend is statically typed and uses Java’s type system without modifications. It is compiled to Java code and thereby seamlessly integrates with all existing Java libraries.

Xtend supplies an easier and more organized approach to the development of Java integrated object oriented programming. Ill show you a primitive example when constructing a new class.

package com.flare.net;

import org.jboss.netty.buffer.ChannelBuffer

@Data class ByteVector {
	
	@Property var ChannelBuffer backing
	
	def readLong() {
		backing.readLong()
	}
	
	def readInt() {
		backing.readInt()
	}
	
	def readShort() {
		backing.readShort()
	}
	
	def readByte() {
		backing.readByte()
	}
	
	def writeByte(byte b) {
		backing.writeByte(b)
	}
	
	def writeShort(short s) {
		backing.writeShort(s)
	}
	
	def writeLong(long l) {
		backing.writeLong(l)
	}
	
	def writeInt(int i) {
		backing.writeInt(i)
	}
}

As demonstrated above you can see that with the aid of Xtend you will be able to save time when actually developing the code itself through the use annotated modifiers, as well as a good deal of other features you can view in the website link below.

Will explain the principals of the dummy code I provided: http://www.eclipse.org/xtend/documentation.html#activeAnnotations

If your actually interested in this you can install the software in eclipse via this link:

http://download.eclipse.org/modeling/tmf/xtext/updates/composite/releases/

Once it has installed you will be able to create Xtend files and utilize them in your java application. You won’t need any sort of compiler since Xtend code will automatically compile into java source code.

http://www.eclipse.org/xtend/documentation.html Will supply you with everything you need to get started. Once again I would like to stress that there is so much more to this language than the single block of code I provided as a demonstration.

Also to show that this can be used into a networking environment and perform any operation similar to Java here is a Netty implementation for example. This example is limited but it’s just proof of concept for the time being.

package com.flare.net;

import org.jboss.netty.channel.ChannelPipelineFactory
import org.jboss.netty.channel.Channels
import org.jboss.netty.channel.ChannelPipeline
import com.flare.net.codec.decode.LoginProtocolDecoder
import com.flare.net.codec.encode.BackEndEncoder

public class PipelineRegistrey implements ChannelPipelineFactory {
	
class PipelineRegistrey implements ChannelPipelineFactory {
        
        override getPipeline() {
                val pipeline = Channels.pipeline
                pipeline.addLast('encoder', new BackEndEncoder)
                pipeline.addLast('decoder', new LoginProtocolDecoder)
                pipeline
        }
}

It is very similar to Java and with a little studying it should be able to be implemented into RuneScape private servers just as easily as any other scripting system of third party library.

package oremtius.beer

class BottlesOfBeer {

def static void main(String[] args) {
	println(start(99));
}

def static start(int amount) '''
	«FOR i : amount .. 1»
		«i.bottles» of beer on the wall, «amount.bottles» of beer.
		Take one down and pass it around, «(i - 1).bottles» of beer on the wall.
	«ENDFOR»
	
	No more bottles of beer on the wall, no more bottles of beer.
	Go to the store and buy some more, «amount.bottles» of beer on the wall.
'''

def static bottles(int amount) {
	switch amount {
		case 0:
			'no more bottles'
		case 1:
			'one bottle'
		default: '''
			«amount» bottles
		'''
	}.toString.toFirstUpper
}

}
