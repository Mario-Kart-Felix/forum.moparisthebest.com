Programming competition?

Community Suggestions
Ambokile August 4, 2016, 11:18am #1

What would you think about having a programming competition? We could give them a complex rule and they must make a program that we can run and then verify if their program meets the rule for a number of cases.

Example:
Most systems prevent users from registering with simple passwords, in order to ensure that their users are correctly protected. One rule to detect “simple” passwords is to reject the password if it contains any sequence of characters followed immediately followed by the same sentence. Make a program to check a password (1 to 10 uppercase letters), and print “Rejected” if the password is invalid, or “Accepted” if the password is valid.

APPLE - Rejected (P is repeated)
NONONSENSE - Rejected (NO is repeated)
PASWORD - Accepted
ONION - Accepted (ON is not immediately repeated)
