FYI:

Ternary isn’t restricted to two values for example:

int x = 5;
int y = x > 0 ? x : x < 0 ? (x*-1) : 0;

which is the same as

int x = 5;
int y = 0;
if(x > 0)
y = x;
else if(x < 0)
y = x*-1;
else
y = 0;

and you can keep adding more conditions to your ternary in that fashion…

Ah ok, but they don’t “go through the same exact process”. Like I said, a ternary statement is treated as a single sequence by the compiler. It is semantically different. Using a ternary expression can actually produce lengthier bytecode. For instance, if you want to increment a value. See: http://stackoverflow.com/questions/2317614/java-bytecode-iconst-0-iadd-sequence. Point is that it pushes a constant and then adds. With a if-else statement, iinc would be used instead, since the compile treats it as two separate sequences.

@Chassy13 Don’t go with Davidi2’s suggestion. Simply looping through an array would be faster. If you want something efficient, perhaps use a HashMap, but it may turn out to be worse if the collection isn’t that big.
