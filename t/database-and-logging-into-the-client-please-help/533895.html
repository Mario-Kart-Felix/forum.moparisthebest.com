lothy August 4, 2016, 10:48am #12

I wasn’t really talking about the collation of the database table so much as what’s actually passing into the hash function.

For example, you might be hashing a UTF-8 string in Java but perhaps you’re hashing a ISO-8859 string in PHP (I’m assuming that’s the default character encoding used in PHP, having read this: http://stackoverflow.com/questions/9351694/setting-php-default-encoding-to-utf-8).

Anyway, a hash is ultimately preformed on the byte array, not the resulting string. This means that two different byte arrays, when processed with two different encodings, can output the same textual string - but the resulting hashes wouldn’t match up.
