And… the “main” is the method called by default by the JVM when you use a class file. The methods work in Java is that they are defined by their name and the syntax of their arguments. [quote=“Javacisnotrecognized, post:20, topic:26370”]You don’t have to write “Tester = Tester++”, you can just use Tester++.
The exact way it works is that writing “Tester++;” increases the value of Tester by 1, and also returns the value of Tester++. Like writing

int I = Tester++;

Is valid, and would result in I equalling the value Tester had after it was ++ 'ed.
On the other hand,

int I = ++Tester;

Would make the value of Tester increase by one, but I would equal the value of Tester before it was ++ 'ed.[/quote]

You got it mixed up java

Prefixing with ++ increments any other operations are done, where as suffixing with ++ increments afterwords.

int i = 0;

// The follow are identical, as there are no other operations taking place.
i++;
++i; // i == 2 after this operation

// The follow are different, however
int j = ++i;  // j == 3 after this operation, and i == 3
int k = i++; // k == 3 after this operation, but i == 4

This concept applies to all operations in java, both arithmetic and bitwise (I believe).
