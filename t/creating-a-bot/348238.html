moparisthebest July 31, 2016, 5:41pm #2

I feel like I just answered this question not too long ago:

[quote=“Moparisthebest, post:2, topic:346150”]Asking a question like this is just going to get you a different answer from every person, they all have their upsides and downsides, and in the end it all boils down to opinion and what you like best and are most comfortable with. Sort of like the linux KDE vs Gnome vs Xfce argument, or the windows vs linux vs mac.

My opinion about the subject though would be to write the bot in Java, using either bytecode injection (faster) or reflection (potentially less detectable), always using only the unsigned client. Give your bot access to the code by either loading the applet like a browser would, writing it as a java agent, or attaching to the running VM. Then give it input by generating mouse and keyboard events silently but exactly how a real human giving the bot real events would look like. (you can use the event generating classes from cherokee)

Good enough answer?[/quote]

Not exactly, but close enough. You have to get the information to act on some way, and your options are limited. Basically reflection, interface/method injection, or renaming are your options. You do have other options, such as a Man In The Middle approach (proxy bot), and looking directly at the memory, but there are reasons no one currently uses those, because they are way too hard to keep updated each revision, I would call trying to do it that way a nightmare.

Your real choice comes in the way you get access to the code, which is covered in my quote.

edit: Another option I didn’t really think of before (because it’s so bad) would be to modify rt.jar, or even modify and compile your own VM for the applet to run in (using OpenJDK).

[SIZE=“4”]Usage:[/SIZE]

1. Set your gfx to the following: (After chaging these settings, you need to close down RS and start it again)

24v1rp2.png740×489

[list]

3. Then, in this exact order:

[LIST=1]
[]Close down SCAR and any instance of RS you have open (i.e. your browser).
[]Start up RS (normal client, not SMART, but in IE/firefox) in HD mode. Wait for login screen to appear.
[]Start up SCAR.
[]Create a SCAR script, e.g. below.
[*]Run the script! If you try and start SCAR before RS has fully loaded it will probably crash out. This will be fixed later on.

Example SCAR script. Moves the mouse to a banker in East Varrock. Will probably work for other bankers too. IDs of each model is the number that floats above in game.
[scar]
program New;
{.include srl/srl.scar}
var
x,y: integer;
begin
SetupSRL;
if(FindModelByID(x,y, 588752808)) then
begin
MMouse(x, y, 0, 0);
Writeln(‘x=’ + IntToStr(x) + ‘, y=’ + IntToStr(y));
end else
begin
Writeln(‘Model not found.’);
end;
end.
[/scar]

FindModelByID prototype:
[scar]function FindModelByID(var x, y: integer; id: integer): boolean;[/scar]
Returns true if model with ID specified is found and false if not found. If found, x,y coords placed in x and y. Else contents of x,y should be ignored.

Finding IDs of models:
[LIST=1]
[]Press numpad 5 to enable overlays
[]Press numpad 7, 8, 9 and nine to choose different strides. Don’t worry what a stride is, this just breaks the models up into groups so it’s less cluttered on screen.
[*]If the ID of the model you want isn’t displayed, try a different stride.[/ul]
