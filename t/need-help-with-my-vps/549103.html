Yeah, 9/10 problems with a VPS are/will be solved by the VPS hosting company.

You need to ask them (NOT US) to make sure port 43594 is open for both inbound and outbound connections.
And you can probably ask them to open your windows firewall (if you are on windows and it is enabled).

The rest is cake.

Run No-IP (so it acts as a DNS).
Run your server.
Change the IP in the client to the No-IP if not already.
Jar the client
Distribute it
