[size=18pt]Introduction[/size]
[anchor=introduction]

Okay, in the introduction I’m going to try and get as much done as possible. This will include going over computer specs and player count. So, first of all, computer specs and what resources would be required. This also contains other general information. Before I go into what people say are the facts I’m going to share with you my take on the situation. Many people claim you need a ton of RAM (Random access memory) to host a minecraft server but with my personal experience this simply is not the case. I’ve ran Linux with 1GB RAM, 3.00 GHZ 1mb up and it’s handled quite a few players lagg free - so please don’t get sucked up into ‘you can’t host the server without 10GB RAM’ talk, it simply is not true.

But saying that, this doesn’t mean you will get away all the time and host hundreds of players. That wasn’t the message I was trying to send out :). Now that we’ve gotten that cleared up we can move onto the factual areas of it.

What you really should have

[ul][li]Preferable a 1mb up + connection (This would be best in my view thought that may differ from person to person!)[/li][li]1GB + RAM (Again, personal preference)[/li][li]2.00GHZ + (Again, personal preference)[/li][li]A Computer[/li][/ul]

As you’ve probably noticed a lot of this is personal preference, I run fine on these specz so I’m comfortable with them. You also have to note I said computer and NOT laptop. Never should you host a server which requires a lot of resources at times be hosted 24/7 on a laptop - this is not good.

The laptop will suffer a constant strain and the heat will potentially cause damage to the machine. Get a computer if you’re hosting 24/7. Once your laptop reaches a certain point (heat wise) it will automatically shut down to stop components dieing. Now we’ve got that done let’s move onto Minecraft versions.

http://www.minecraftwiki.net/wiki/Version_history

This is the version history for Minecraft - I’m not going to mention this a lot but I think it’s worth reading

[size=18pt]Setting up a server - Linux Ubuntu[/size]
[anchor=settingupserver]

And, now we move onto what you’ve all been waiting for (The joy) setting the actual server up with all our downloads. I don’t want this guide to be quite as long as my other one so I’ll try and make this section as short and quick as possible. But before I do that I’m going to recommend you use Linux! What is Linux you may ask?

Definitions of Linux on the Web: an open-source version of the UNIX operating system

That answers it. It is superior over windows in many ways. It’s also best for hosting the server. You can find Linux (Ubuntu if you’re new) here:

http://www.ubuntu.com/desktop/get-ubuntu/download

Just follow the instructions to install it. First of all I’m going to cover setting up on linux Ubuntu - to do this I’m going to be using SSH. So you will need to download Putty/ or use terminal. If your server is in another location then use Putty.

http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html

Let’s begin. First of all some basic commands.

cd /dirname/etc/etc - Goes to the specific directory
ls - displays all files in the directory you're currently in
java - once Java is installed, shows a prompt of useful information and reminders
passwd - changes the root password
nano textdocname - allows you to edit/read text files
ctrl + c - acts as a cancel feature
sudo - logs into admin priv
apt-get install package- install command
wget URL - downloads files from a direct URL
Java filename - Runs a Java app i.e .jar

First of all we’re going to download Minecraft_server.jar using the wget function, I’ve found a direct link (Which wget requires) for this tutorial. So you do

wget directlink

So we’re going to do

wget s3.amazonaws.com/MinecraftDownload/launcher/minecraft_server.jar?v=1298922308461

This will download Minecraft_server.jar, when it has downloaded it should either go to Downloads or your current directory. This is where basic commands will come in such as ‘cd’ and ‘ls’. Perform

ls

To check what is in your current directory, while

cd directory

Will take you to the specified directory, e.g cd /bin/. We will end up using this later one, once you’ve located the Minecraft_server.jar take note and move onto the next stage. Now we need to install Java, this is a very easy part. We’re going to be using the apt-get install function. I’m going to do some explaining.

‘apt-get’ - selects the package while the actual ‘install’ part of it tells it what to do. e.g apt-get install nano - this would install nano if the package name was correct.

‘sudo’ - allows the user to do administration (Gives them admin privileges).

Now that we’ve got that cleared up, we’re going to install Java. We simply run the following command in terminal.

sudo apt-get install openjdk-6-jre

This will install the package we’ve stated. Once that is installed go back to your old directory you took note of (Preferable make it in a file to prevent a ton of mess). We will go back by doing

cd /your/dir/here

So if mine was in /bin/ I would do

cd /bin/

Then we’re going to actually run Minecraft_server.jar.

java -Xmx1024M -Xms1024M -jar minecraft_server.jar nogui

Allow me to explain this code in more depth, the '1024mb’s are the amounts of RAM the JVM (Java virtual machine) can use. The ‘minecraft_server.jar’ is obviously the target file and the ‘nogui’ is specifying no Graphical user interface. If you want to edit the actual files we’re going to install a text editor called ‘nano’.

There are other methods but I like nano (For no apparent reason at all). We’re going to use an old command ‘sudo apt-get install’

sudo apt-get install nano

This will install ‘nano’ which a text editor we can use to read/write to files. We will use it like this.

nano filename.type

e.g

nano whitelist.txt

With this installed you will be able to edit configuration files for your Minecraft server and various other things, this will also help us with our plugins later on in this guide. And that’s it! I’m just going to redirect you to a topic regarding port forwarding s there are so many things which people do wrong and so many various types, it’s just not worth typing up.

http://portforward.com/

That site covers everything you’ll need to know to get your server started in regards to port forwarding.

[size=18pt]Bukkit - Plugins[/size]
[anchor=bukkitplugins]

Bukkit adapts your server for multiplayer use, without Bukkit multiplayer would be very different. It provides great mods such as big brother (BB) which are an anti-griefeing tool used by countless servers. In this part I’m going to show you how to set it up. (On Linux)

First of all you’re going to need to download it.
http://wiki.bukkit.org/Setting_up_a_server

wget URL

To download the file, then

ls

To check what’s in the directory. And finally to actually run Bukkit we’re going to be using a different run code from normal

#! /bin/sh 
BINDIR="$(dirname "$(readlink -fn "$0")")"
cd "$BINDIR"
java -Xincgc -Xmx1G -jar craftbukkit-0.0.1-SNAPSHOT.jar

We’re going to use nano to do this. Type

nano

Then we’re going to edit it with the run code, then do ctrl + o to save it. Then we have to do chmod to give it permissions to execute the code. We simply do

chmod +x Minecraft.sh

This will give it +x which is executable, then we simply execute it everytime you want to launch Bukkit. And now for some basic information regarding plugins, if you want plugins then you can find them all at

http://plugins.bukkit.org/

That contains a vast library of plugins you can use, based on version etc again, if you want to download one, go to the ‘plugins’ directory and then

wget directlink

It will then download it. If it’s a .jar simply run your server as I showed you before, if it is a zip you can use methods such as

unzip file

and then run your server - if a plugin is complex it will most likely contain instructions on setting things up such as SQL.

[size=18pt]Setting up your own server - Windows![/size]
[anchor=settingupserverwindows]
Most of you will know what Windows is, so I doubt I’ll need to put an explanation. I’m going to make this as short as possible and avoid all room for speculation about the Operating system itself.

List of things you need

[ul][li]WinRar - http://www.rarlab.com/download.htm[/li][/ul]

[ul][li]Java - http://www.java.com/en/download/[/li][/ul]

[ul][li]Minecraft - http://www.minecraft.net/download.jsp[/li][/ul]

[ul][li]Bukkit - http://wiki.bukkit.org/Setting_up_a_server[/li][/ul]

This time we’re going to download Minecraft_server.exe which can also be found at http://www.minecraft.net/download.jsp Once you’ve downloaded it create a new folder called ‘minecraft’ in a specified directory.

Then simply run the EXE file, it will then bring up a GUI (Graphical User Interface) - It will also make multiple files.

I’m now going to explain what each of those individual files are responsible for, and what functions you can edit. This can be used across all OS - I just include it here.

[i] Files

Banned-ips - contains all banned ips

banned-players - contains all banned playerNames

ops - contains the name of all servers OPS

Server - a log of everything printed out on the GUI

white-list - a list of names which are allowed to play your server if the whitelist function is enabled in properties.
[/i]

Now I’ve left out properties because I want to go into more detail on it. Properties has a ton of stuff you can edit such as save name etc

#Minecraft server properties #Sat Apr 16 21:23:33 BST 2011 level-name=world // save name hellworld=false spawn-monsters=true // spawn mobs? online-mode=true // if this is true hacked clients won't be able to connect spawn-animals=true // spawn things like pigs etc max-players=20 // maximum players on your server at one time server-ip= // ip of server pvp=true // is playervsplayer on? level-seed= // seed i.e Tom_BETA server-port=25565 // server port number IP:PORTNUMBER white-list=false // does it use the whitelist function stated above?

That’s a brief overview of the properties file. Now that you’ve ran the EXE the server is listening for connections on the specific port in the properties file, so remember to change that! If you need to portforward then simply use the following site.

http://portforward.com/

It will cover everything in regards to portforwarding your router.
