anyways, what you should do is have a char[26], one character for each letter of the alphabet. this will reduce the execution time for your application through each addition of an incorrect/used character and also for retrieving the value of a certain character (to see if it has been used or not), since it would be O(1), rather than like O(26 / 2) (if using a list or such)

anyways, here’s pseudocode:

char[] occupied = new char[26]; for (int i = 0; i < 26; i++) { occupied[i] = -1; } byte c = Character.toLowerCase(/*user tries a character*/) - 97; //assuming you are using utf-8 if (occupied[c] == -1) { /*not in use*/ occupied[c] = c + 97; }
