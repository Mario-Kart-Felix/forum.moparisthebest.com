zyle1992 August 1, 2016, 12:56pm #6

Are you sure you’re not accidentally starting the server in two places? Otherwise there’s another service running on port 43594.

The easiest way to test is replace the 43594 with a random number. Like 38752. If it works, then there’s another server running on port 43594 on your computer. If it doesn’t work, try another random port. If it still doesn’t work, then somewhere in your server, a server socket is being opened, and bound twice.
