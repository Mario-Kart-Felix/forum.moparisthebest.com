My "cfg" parser API

General Programming
RegeX July 29, 2016, 8:41pm #1

I’ve written a small API to parse “cfg” files like:

[font=Courier][i]NPC City_Guard {
name => “Injured Guard”;
level => 20;
isAggressive => false;
}

NPC Wilderness_DeathStalker {
name => “Deathstalker”;
level => 99;
isAggressive => true;
}[/i][/font]
And turn them into objects for me so I can do with them what I want.

The example app I wrote and uploaded at the API site transposes the declarations into the appropriate classes, and then transposes the assignments into the fields. Fun:

[font=Courier]D:\devel\cfg_parser\example>java -cp .;CfgParser.jar; NpcParser
NPC(Deathstalker) [Level-99,Aggressive]
NPC(Injured Guard) [Level-20,Passive][/font]

Not bad for literally 3 lines of actual code lol.

Basically for the api, docs, and example, head to http://sieg-heil.org/cfg_parser/ and http://sieg-heil.org/cfg_parser/example/

It’s pretty nifty, you privateserverpeople should stop using raw string functions and use this. The app shouldn’t bitch too much about space formatting etc, just don’t go sticking it on a single line etc =.

Oh yeah, and the => possible types must match “string”, 1 (int), or true (boolean). Because I’m a type-strict bitch. And I’m unloved.
And I didn’t use a package for my classes. Burn.
