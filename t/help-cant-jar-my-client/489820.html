If you’re on about just jaring it, that’s extremely easy. Being unable to open it means you didn’t add all of the files to the jar correctly, you might be missing the signlink class or the manifest. That’s usually the problem for me.

You can use the command prompt to create a jar and I find it easier than a jarmaker.

jar cvfm [NAME] [MANIFEST] [CLASSES] [OTHER FILES]

Here’s an example.

jar cvfm client.jar META-INF/MANIFEST.mf *.class sign/*.class *.dat *.gif

In your bin folder of jdk, there is an application called jar, you can use this to create a jar through the command prompt.

Copy the jar.exe, jli.dll and you might need msvcr100.dll, paste them into your client directory.

Copy the directory of your client, then go to command prompt.

Type:

cd (paste directory here)

Then you can work straight from the client directory. From there you’d use the following to jar your client.

jar cvfm [CLIENT NAME] [MANIFEST] [CLASSES] [OTHER FILES]

jar cvfm Client.jar META-INF/MANIFEST.mf *.class *.dat

*.class = all of the files that end in .class, rather then typing all of them out.
