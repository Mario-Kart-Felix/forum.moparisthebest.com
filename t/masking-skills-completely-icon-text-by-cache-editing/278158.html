Masking Skills Completely (Icon & Text) By Cache Editing!

Games
Runescape
Lin July 31, 2016, 7:35am #1

Welcome to a tutorial on masking skills. I figured this out today with little effort and some experimenting, and I liked the results so I thought I’d share. It replaces the skill icon and the text. It’s basic interface editing, but it indeed does not require hardcoding.

The Downloads
Tom’s Cache Suite - For dumping & replacing interface.jag. You can’t download it coming from this site, so Google it. I’ve Googled it and successfully found a download.

Cygnus Hex Editor - Only if you don’t have a hex editor. You must use a Hex Editor to prevent corruption.

The Icon
Search the skill icon you want on Google Images (For example, “runescape construction icon”). It’s normally the first image. Copy the 32x32 one (Or the one near it), and paste it into an image editor. Replace of all of the seemingly white with magenta (Google image the color :)). Magenta is the transparency color for images. Make sure the image starts as far left and as far high as they can. Make sure you save as a GIF, JPG, or PNG.

Replacing The Image
This part is easy, and might take some time to get perfect. Open the Cache Suite and load your client’s cache. Click on “View/Edit Image Archives” and select “Media”. Next, on the left list, find “staticons.dat” or “staticons2.dat”. If you’re replacing the last row of skills, they’ll be in staticons2.

After selecting the file, you’re going to select the image index. After finding the one you want, click “Import” and select the new stat icon. Make sure you don’t see the magenta part. Set the Offset X/Y to 0, and click “Save Archive”. If it asks to rebuild, click OK. Start up your client and see if the icon is in the right spot. If it’s not, change the Offset X/Y to something above 0. If you go below 0, the icon won’t show.

Changing The Text
This is probably the part that most people will get stuck on, but it just like editing a text file in a complicated way. In the Cache Suite, click “View/Edit Jagex Archives” and select “interface.jag”. Once it’s loaded, selected “data” on the left and click on “Dump File” and save as “interface.jag” in your My Documents.

Open up Cygnus Hex Editor and open up interface.jag from your My Documents folder. I’m not going to explain what you see, as I don’t feel like it. Press Ctrl+F to find, and click on the right part of the textbox so you’re typing in a string. Type in the old skill (Eg. Farming) and click Find Next. If it can’t find it, click “Ignore Case” and try again.

What you should find is “Advance skillname”. Now here’s where people will screw up. On the right, click before the skill name, so your cursor is at “Advance |”. On the bottom right, make sure it says “OVR”. If not, press Insert. You’re now writing over the text.

Start typing the new skill name until there’s no more text from the old skill. DO NOT GO OVER THE EXISTING DATA! If I was replacing Herblore with Summoning, it’d look like “Summonin”. From this point, press Insert again so on the bottom right, it says “INS”. Then you can finish typing the skill.

You must do that 2 more times; one for the “View stat Guide” and the other for “* XP:”. When you’re finished, don’t forget to save it.

Go back to the Cache Suite and back on where you dumped the file, click “Replace File” and select the interface.jag that you just edited, and then click “Repack Archive”.

Fixing the XP Spot
Sadly, with my experiments, I couldn’t find the data that controlled where the XP label was positioned. However, you can still do this server sided. Open up your server’s client.java and find the method “refreshSkills”, or just look for “sendQuest(”" + playerXP[20] + “”, 4157);" or another skill.

Next, find the line that writes the XP for the skill you replaced. For example, 18 would be slayer. In the “” before the line, put in spaces. Do this until the XP value comes AFTER the “* XP:”. Here’s my replaced Slayer with Construction.

sendQuest(" " + playerXP[18] + "", 12171);I know that this is a terrible way to do it, but until someone finds how to edit label positions through the cache, it will be done this way.

Thanks for reading this, and I hope it helped. Cheers!
