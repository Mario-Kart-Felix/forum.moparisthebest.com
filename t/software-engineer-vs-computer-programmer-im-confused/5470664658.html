There’s a story behind software engineering. Software development as a profession has an awful reputation. Statistically speaking most projects fail - they go over budget, they run over deadlines, or the software isn’t fit for purpose.
The idea with software engineering is that it’s supposed to add engineering rigour to the software development process, thereby mitigating these failures.

Has it worked though? Not really.
