if you feel like trying just a tiny bit, you can fix this with ease.

using jbe (i’ve built a fixed version here), look through the constant pool of each class you see in the stack trace, starting from Sprite (that is, first check Sprite, then RSInterface, then client, then RSApplet).
check for any windows-specific constants like “C:/” or even the whole path of “C:/WS2_cache_32”, and change the constant to something non-windows specific.
replacing “C:/” with “./” will probably work, provided you put the cache in the working directory of the client (as in, if you run the client from /home/eddw/client/, you should put the cache in /home/eddw/client/WS2_cache_32/ or whatever you change it to).

once you think you’ve found everything, save all the classes you modified and try again.
