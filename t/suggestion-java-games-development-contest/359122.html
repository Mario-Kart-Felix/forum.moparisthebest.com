Suggestion: Java Games Development Contest

General Programming
Speljohan July 31, 2016, 7:09pm #1

I like competitions, people like competitions, everyone likes competitions!

My suggestion is that we have some sort of Java Games Dev competition where the goal is to create a game based on a specific theme in a given short timeframe (a few days max).

Suggested rules would be:

[ul][li]The theme must be abstract to promote creative thinking [/li]
[li]Only allowed to use code written by yourself, no snippets from anywhere else[/li]
[li]All art assets must be created by the contestant[/li]
[li]Only the JRE standard API is allowed, no third party libraries[/li]
[li]For art assets, any tool may be used (if you want music/sound effects, you create them yourself)[/li]
[li]Live streaming (http://www.livestream.com/) of the project development process is a plus, since it simplifies enforcing the rules[/li]
[li]Source code must be released as open source when completed, and all resources such as art/music must become public domain[/li][/ul]

The purpose of this would be for one, to create and be creative, secondly to learn from both your own experiences and from other people’s projects, and more importantly, have fun!

A prize for the winner isn’t neccessary, but if people are interested it can be arranged in one way or another (perhaps something as simple as a custom badge?)

So, if anyone else thinks this is a good idea, tell me!
