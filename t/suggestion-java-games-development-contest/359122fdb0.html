Fat_Cookies August 1, 2016, 1:18am #70

Since this is 9999 dead mainly because nobody has the time to work 2 days straight I suggest we change the competition into one of those java 4k challenges in which all participants must create a game under 4kb after being compressed into a jar executable.

The rules would be relatively simple:

  1. The jar must be under 4k (hurrrrrrrrr)
  2. Cannot use any external libraries or resources (so getting gfx from a web server is forbidden).
  3. Must be able to run offline (similar to 2.)

Unlike the current contest this could run over a longer period of time: perhaps 2 months. I personally believe this is a true challenge of ones programming ability since you have to cram a lot of shit into a really small space. In my opinion, the 48 hour challenge is more of a “who is saddest/can stay up for 48 hours” contest.

It’s truly amazing what some people have achieved with 4kb using java which has quite a big overhead. I was really impressed by FortessFall4k which is basically a simplified version of angry birds.
