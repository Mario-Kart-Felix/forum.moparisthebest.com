I never figured that this was a problem, but during my first use of Netbeans for the wonderful GUI editor I came across a warning message:

Leaking this in constructor
The code is this: [code] private static Palette instance = null; 
public Palette() {
	instance = this;
}[/code]

I never figured this was a problem – and I still don’t know if it really is – but is there a better way to do this? It’s never really occurred to me to do it any differently.

I suppose I could keep track of all instances of classes in one area, such as the engine. What do you guys think?

HcoJustin August 4, 2016, 11:20am #2

This causes a problem if you create a second instance of Palette. It updates the global instance causing some problems. If you absolutely need one and only one instance, use the singleton pattern
