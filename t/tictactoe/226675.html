Methods that are called often should not be totally cluttered, but should call several lesser methods that do the actions the original method did. Also, by separating many actions into several methods, you help better-organize your code.

What I mean is, make a class that loops as long as the game is running, whose loop does the updating and calls the rendering up the render pipeline.

The render pipeline is a series of tasks that are done before the final product is rendered onto the screen. The first stage of the render pipeline, in this game, would be to process user input. The second stage is to swap the buffers (read below). And the final stage is to draw the on-screen buffer onto the game canvas.

Off-screen and On-screen buffers. An off-screen buffer is an image that stores the next frame of the game. This image is created during the render loop, and is meant to sit in storage until the next loop where it will be rendered onto the screen. The on-screen buffer is an image which has already been created and just needs to be drawn onto the screen.

The purpose of using off-screen and on-screen buffers, also called double buffering, is to enhance the overall graphics of the application. In cases where a game’s render loop is pushing a good 50-80 fps (frames per second), then the game will flicker very badly. By using double buffering, you remove most, if not all, of this flickering. And, if optional, you can also do triple buffering.

Why use a render loop? This is just good game design. A render loop does all the processing, rendering of the buffers, swapping of the buffers, and call to the paint method (to render the on-screen buffer onto the canvas). In cases where you don’t use a render loop, the following things can happen:

-Without double buffering, flickering can occur
-Without a loop calling the repainting of a canvas, no animations are possible
-Without a loop calling the repainting of a canvas, a frame going on top of the game frame or applet can distort the rendered image, aka, make it ‘dirty’

Well, there’s my 3-cents, hope you can improve the game!
