I’ve been thinking about this concept and realised that I’ve probably been thinking about it incorrectly. I’ve been trying to think of a way to make something like this work:

PascalScript Executer Object -> native, separate program

But I doubt that is or ever will be possible without far more effort than I would want to put into it.

So I’ve tried to think along the same lines as Kaitnieks might have.
1 - SCAR uses a 3rd party Pascal interpreter that could just as easily be used in any other program.
2 - I’ve just finished a custom encryption protocol specifically for use with said interpreter in the context of SCAR
3 - I have access to all of my own function source, which could also be used in any other program.
4 - I already have a mechanism in place for evaluating scripts and doing something to them (certification).

Along those lines I’ve come up with another idea - ‘compile’ the scripts by means of a sort of mini-SCAR with the encrypted script hardcoded into the program. All Kaitnieks would have had to do is make a template for himself and copy + paste the script into a string when he was done certifying/encrypting it.

Of course, in order to make SCAR-lite someone would need to have all of the scripting functions from SCAR. I have an extremely old copy of the source, but it would be useless for this. Did Kaitnieks give anyone the most recent source before leaving?
