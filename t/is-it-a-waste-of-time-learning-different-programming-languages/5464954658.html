Ashi:

I still don’t know the language deep enough to not need google every 5 mins.

You don’t need to know every method of every class in every API ever made. What you do need to know how to do is put it all together into one place with a smooth flow. A very strong understanding of algorithms, design patterns, etc are all universal concepts that can be applied in almost any language.

If you are going to focus on Java, then you’ll definitely want to know the ins and outs of some portions of the stdlib, but you definitely don’t need to memorize it. Like knowing the main differences between an ArrayList vs LinkedList, or when to use HashMap vs TreeMap and their differences. You’ll also want to remember that Java’s Cloneable interface is a shallow copy, and you’ll want to know when/why a deep clone is needed.

Every language has its little quirks like this. Just make sure you figure out what they are
