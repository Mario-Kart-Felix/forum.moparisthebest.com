Ok so today I am going to teach you a little bit about characters also from here on out called “char” because that is how it is defined in Java.

First lets give you some information on char. Java uses this thing called a “Unicode”. What it does is it allows you to define fulling international character set that can represent all the characters in all of the human languages (Latin, Greek, etc). For this purpose it makes the chars a 16-bit type. The range of “chars” are from 0 to 65,536, but NO negative chars allowed. The standard set of chars are from 0 - 127 and they are called ASCII.

Now that you know some information about chars lets start.

please try to read the tutorial and do it before you see the finish product, or else you will learn nothing.

lets start with opening it up with a class by doing this

[code]/*

  • test by gods army
    */
    class CharDemo {
    }[/code]

now lets add a void into that by doing this

public static void main(String[] args) { }

we will put the above void right between class CharDemo and the }

looking like this

[code]/*

  • test by gods army
    */
    class CharDemo {
    public static void main(String[] args) {
    }
    }[/code]

now lets identify the chars by adding it right under public static void main(String[] args) {

use this code to identify.

	char ch1, ch2;

now that we have our chars identified lets find their values.

add this under the code above

ch1 = 88; // code for X ch2 = 'Y';

so now it should look like this

[code]/*

  • test by gods army
    */
    class CharDemo {
    public static void main(String[] args) {
    char ch1, ch2;

    ch1 = 88; // code for X
    ch2 = ‘Y’;
    }
    }[/code]

So what do you think will happen if you try to run it? nothing? well your right. It has nothing to print out anywhere, just the values that it holds but nothing more. so now to finish our program lets top it off with a print :).

we’re going to do this

	System.out.print("ch1 and ch2:");

this will will now print out “ch1 and ch2:”. so lets give it the values it requires.

by adding this

	System.out.println(ch1 + " " + ch2);

by adding the ch1 + it will read the value without having to be in the quote signs but still print out a value/number or letter.

remember to name it “CharDemo.Java” exactally (when using a notepad and not an IDE).

so now lets compile. oh, you dont know how to compile? well lets make a new file and name it Compile.bat and put this in it.

@echo off echo Made for Char tutorial by Gods Army echo compiling javac CharDemo.Java pause

now compile and you should get a new file. it will be called CharDemo.class

yay your done! almost. lets make it run now. make a new file and name it Run.bat and put this into it.

@echo off echo Made by Gods Army for Char tutorial echo compiling. java CharDemo pause

Good job. If everything went properly you should get this output in your CMD box when clicking the run.bat file

ch1 and ch2: X Y

It comes out like that because X is equal 88 in Unicode (remember that?) actually means X. and ch2 was the value ‘Y’.

Yay your done!

I hope this helped you learn a little more about char values.
