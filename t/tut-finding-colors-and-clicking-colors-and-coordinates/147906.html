The ‚ÄúColor Picked:number goes here‚Äù means the color that you clicked on a coordinate.There are different colors all over the world,and there are also different colors all over the monitor screen,so it is most likely,the colors are always different.If you want it to click colors then add these on SCAR:

[code]program New;
{.include SRL/SRL.scar}

var
x,y,Ax,Ay:integer;

const
Color = 16248815;
Tolerance = 5;

begin
SetUpSRL;
GetClientDimensions(Ax,Ay);
if FindColorTolerance(x,y,Color,0,0,Ax,Ay,Tolerance)then

Now I will explain everything thoroughly step by step,so you can understand.You see,program is there for the name it‚Äôs suppose to be called.You can replace ‚ÄúNew;‚Äù with any other thing you want,like this program Tutorial;
See, I named mine tutorial,like I said before you can name it anything you want to.

Now for SRL.{.include SRL/SRL.scar} is probably the best include ever.It has whatever you might possibly need,like Mouse(which clicks the coordinates/colors you want it to)it clicks more human-like than Clickmouse,which we will discuss later in this tutorial, so it is less detectable.

Var is short for variable.You put all your variables you want in it, you can name them whatever you want,so SCAR can be sometimes fun!I put Ax, Ay, x, and y for my variables.

Now for const,const stands for constant because they always stay the same.Like I said before, you can name it anything you want,I name mine Color,for color,and Tolerance for tolerance.Tolerance is the objects tolerance.These two things(var and const) are very helpful to SCAR scripters.You will need these throughout your career.

Begin is for the script to start.It starts and does everything you tell it to do with you magic keyboard.

SetUpSRL setups SRL, you must have this in your first thing in your main loop;this sets up SRL.

GetClientDimensions gets the dimensions of the object you want it to.See for instance I wanted it to get the dimensions of Ax and Ay.It would get them for me,it is hardly ever used,so you might want to forget about it if you have to.

if FindColorTolerance(x,y,Color,0,0,Ax,Ay,Tolerance) then Mouse(x,y,true);
This function is used in runescape alot(and tutorials).It is a very nice command.If it finds the color that you want it to,it clicks it.‚ÄúIf‚Äù and ‚Äúthen‚Äù are used alot.If it does something then something will happen,right?FindColorTolerance find the tolerance of the object,like I said before.X, and y are very common and useful variables because x is the horizontal axis and y is the vertical axis,so SCAR is like math.They can be anything you want it to be, or you don‚Äôt have to make it stand for anything,so it‚Äôs like nothing if you don‚Äôt want it to be anything.Color is the color it finds also.0 and 0 are just the axis number it will find.Ax and Ay are just variables too.Tolerance is the tolerance,like I stated before.

End makes the script end.End; makes the script end in a procedure,which we will talk about later.

Now I will teach you how to click coordinates,it‚Äôs really simple.Click the eyedropper and click anywhere.The debug image will show something like this:

Forget about the "Color Picked: 16248815, we are going to talk about the ‚Äúat (152, 464)‚Äù
Ok now add this:

[code]program Tutorial;
{.include SRL/SRL.scar}

begin
SetUpSRL;
Mouse(271,497,1,1,true);
end.[/code]

Ok this is what the Mouse(271,497,1,1,true); stands for.Mouse clicks the object.271 and 497 are the coordinates.1 and 1 are the tolerane numbers.True clicks it directly while false right clicks it,usually showing some options.

Now you are done!!
