its not called a handler its called a class…

and why would you not check for nulls, knowing there’s nulls and then print an error that you know is going to print? That doesn’t make any sense.

Try-catch is there for you to help you check for unexpected errors during the runtime of your code. Try-catch is only needed when your code might throw an error (during processing, task systems, etc). If your code isn’t throwing an error if you check for nulls, why check for the error…?

So yes, check for nulls… and read up on what try-catch is. It doesn’t fix your errors.
