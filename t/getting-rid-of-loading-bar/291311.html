What this tutorial will do is get rid of the loading bar and just leave the text, you could also remove the text too and it’s very easy to figure that out. SO

Find this in your client class.

aClass15_1109.method237(0);

You should see a block of code similar to this

aClass15_1109.method237(0); char c = '\u0168'; char c1 = '\310'; byte byte1 = 20; aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, "Loading -Please Wait.", 23693, c1 / 2 - 26 - byte1, c / 2); int j = c1 / 2 - 18 - byte1; Class30_Sub2_Sub1.method337(c / 2 - 152, 304, 34, 0x8c1111, j, true); Class30_Sub2_Sub1.method337(c / 2 - 151, 302, 32, 0, j + 1, true); Class30_Sub2_Sub1.method336(30, j + 2, c / 2 - 150, 0x8c1111, i * 3, 0); Class30_Sub2_Sub1.method336(30, j + 2, (c / 2 - 150) + i * 3, 0, 300 - i * 3, 0); aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, s, 23693, (c1 / 2 + 5) - byte1, c / 2); aClass15_1109.method238(171, 23680, super.aGraphics12, 202);

Now I’m going to break this down.

These lines tell the text where to print out.

char c = '\u0168';
char c1 = '\310';

This line ,obviously, prints out the text

aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, "Loading -Please Wait.", 23693, c1 / 2 - 26 - byte1, c / 2);

These five lines are what create the loading bar.

Class30_Sub2_Sub1.method337(c / 2 - 152, 304, 34, 0x8c1111, j, true); Class30_Sub2_Sub1.method337(c / 2 - 151, 302, 32, 0, j + 1, true); Class30_Sub2_Sub1.method336(30, j + 2, c / 2 - 150, 0x8c1111, i * 3, 0); Class30_Sub2_Sub1.method336(30, j + 2, (c / 2 - 150) + i * 3, 0, 300 - i * 3, 0); aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, s, 23693, (c1 / 2 + 5) - byte1, c / 2);

To remove any of those things just simply comment out the lines you’d like to not have so for example.

aClass15_1109.method237(0); char c = '\u0168'; char c1 = '\310'; byte byte1 = 20; aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, "Loading -Please Wait.", 23693, c1 / 2 - 26 - byte1, c / 2); int j = c1 / 2 - 18 - byte1; //Class30_Sub2_Sub1.method337(c / 2 - 152, 304, 34, 0x8c1111, j, true); //Class30_Sub2_Sub1.method337(c / 2 - 151, 302, 32, 0, j + 1, true); //Class30_Sub2_Sub1.method336(30, j + 2, c / 2 - 150, 0x8c1111, i * 3, 0); //Class30_Sub2_Sub1.method336(30, j + 2, (c / 2 - 150) + i * 3, 0, 300 - i * 3, 0); //aClass30_Sub2_Sub1_Sub4_1272.method381(0xffffff, s, 23693, (c1 / 2 + 5) - byte1, c / 2); aClass15_1109.method238(171, 23680, super.aGraphics12, 202);
This would be not having a loading bar.
