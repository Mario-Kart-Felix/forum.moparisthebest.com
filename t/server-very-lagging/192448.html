If you’ve got a 64-bit PC, you could potentially handle up to 17179869184 GB of RAM (2^64/1024/1024/1024), as opposed to 4GB (2^32/1024/1024/1024) with a 32-bit PC.

Also, another thing that can help. In lots of servers there is a “System.gc()” called in the server loop (in server.java). I would remove this, or only call it every 100 or 1000 loops. Garbage collecting in the main loop of a program (even if there are threads) can have an impact on performance. In fact, it’s best to only garbage collect after you have done something where you have allocated a lot of memory and no longer need to use it. Also, if you do have 18GB of RAM it will be a less important operation ;).
