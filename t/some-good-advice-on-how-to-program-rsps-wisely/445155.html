[center][size=14pt]Keeping code clean
[/size][/center]
In PI sources, you will always see dirty code. There are ways to clean up this code with not much trouble and quick learning.

First, let’s go over what is dirty code and what is clean.

[size=10pt]Dirty:[/size]

	sendFrame126("Levels: " + totalLevel, 13983);
    }

Code like this is very ugly and should never ever turn into a habit. This code could be simplified into a few lines, seeing as there is a certain pattern to all the incrementing. If there is ever a constant pattern that you can very well simplify, then you should always do so.

It could simplify into this easily.

Clean:

    public void totallevelsupdate() {
	int total = 0;
	for(int slot = 0; slot < 22; slot++) {
	    total += getLevelForXP(c.playerXP[slot]);
	}
	sendFrame126("Levels: " + total, 13983);
    }

You should always keep in mind that another programmer, someday, will always see your code and you want to impress that programmer as much as you can as well as keeping the same outcome of the code. Conventions are also very very important to other programmers, or newcomers. I suggest you read this link under if you do not understand what conventions are completely.
http://www.oracle.com/technetwork/java/javase/documentation/codeconvtoc-136057.html

There are many more examples that I can explain about, but at the moment I cannot think of any others.

[center][size=14pt]Bad programming habits:[/size][/center]

I will always see a ton of code in one method piled up like crazy. If you can, you should always try to use an array to store some of that data to make it less confusing. You also should always try to stay away from unnecessary looping, or if your code can be stopped right then and there, but continues throughout the method, you should avoid this and always stop if you really do not need to keep going. You always want your code to not put so much stress on the VM.
