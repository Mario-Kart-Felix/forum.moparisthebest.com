public void saveImage(String name, BufferedImage image) {
	try {
		File imageFile = new File(name + ".png");
		ImageIO.write(image, "png", imageFile);
	} catch (IOException e) {
		e.printStackTrace();
	}
}

obviously none of you have a clue on how the rs client works

i didn’t say that what T4_ posted was inaccurate because of whatever the fuck you guys think, it would work properly if the client did all of its painting in the canvas’ overriden paint method – but it doesn’t. it makes calls to the ‘getGraphics’ function and draws onto that. therefore, the best way to do this is to hack the canvas and override ‘getGraphics’ and have it draw onto a cached image and return a copy of that whenever the ‘getCanvasImage’ function is called

the only other (relatively) efficient alternative i can think of if you want to keep it in pure java is to use reflection and access the client’s offscreen buffer (it’s an int[] that stores the value of each px)

of course though, hacking or reflecting the client kind of defeats the purpose of an image recognition bot (is there any purpose at all really…?)

Find the canvas class, find the getGraphics() method. Then either:

  • inject something that will set the local bot copy of the Graphics object,
  • override the method by changing the name and superclass to your own canvas class then implement your own getGraphics

or use reflection/inject a getter to get the int[] pixel buffer (best method)
