The_Web_Guy August 4, 2016, 12:13am #10

How to PortForward, the easiest way…

  1. Open up any Web Browser, other than Internet Explorer; Computer may crash.
  2. In the URL bar type in ‘192.168.1.1’, if that doesn’t work go to ‘192.168.0.1’
  3. Login to your router settings, if you haven’t defined a router password, the default username is ‘username’ and password is ‘password’. Sometimes the username can be ‘admin’ and the password can be ‘admin’ as well. Try a couple combinations if it isn’t working the first time.
  4. Navigate to Port-Forwarding options, not Port-Triggering. If you don’t see a clear link that says port-forwarding, look for one that says advanced or networking.
  5. Go to Start > Run
  6. In the small window that pops up, type ‘cmd’
  7. In the black window that comes up after clicking ‘OK’ type in ‘ipconfig’ …
  8. You’ll see something like ‘IPV4 Address . . . . . . . 192.168.1.###’
  9. Take that number (the 192.168.1.###) the ‘#’ should be something different for you, either one, two, or three digits long.
  10. Go back to your router settings, for the service name type in “RSPS” or whatever you’d like, it doesn’t matter.
  11. For where to direct it to put in the 192.168.1.### (Whatever you took note of from ipconfig)
  12. Select TCP.
  13. Enter in starting port and ending port (should be the same). This is whatever the port is that your server is listening to, generally 43594, but sometimes can be 5555.
  14. Save router settings and close the window.

There you go, it isn’t too hard.
