l0l. No. Thread-per-client is incredibly unscalable. No one likes 100s of threads for a single process.

Example of tpc: (Note this is NOT from any WinterLove base server)

new Thread() {
    @Override
    public void run() {
        try {
            int packetId = -1;
            while ((packetId = clientSocket.getInputStream().read()) >= 0) {
                  byte[] payload = new byte[256];
		  clientSocket.getInputStream().read(payload);
	     }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}.start();

You’re creating a new thread for every new client that is connected.
