[quote=“Kryptix, post:71, topic:306490”]My point is, stuff that can be handled by the client should be handled by the client. When creating a MMORPG it’s all about spreading the load out to make it as efficient as possible. I know it doesn’t really matter on a private server with 0-500 players online at any time but it’s good practise to always do things the best they can be done.

If you have 2,000 players online and have them all talking, it could be over 1,000-2,000 messages-per-second and to have the server scan all of those to filter the bad words is unnecessary when you can just let the client do it for the chat messages it can actually see. I know it’s not so important now but back in the day when Java was less efficient running on poverty-spec servers it was a huge thing. I guess Andrew learnt that from other games and made RuneScape as efficient as he knew how. You’re right though, it’s not required now but as I said I find it good practise to do things the best way I know how.

In my opinion, the server should handle the bare essentials. It should only handle things that it has to process or verify. For example, what’s the point of using a R2H on a R2H? It sends a packet to the server, the server checks and realises it does nothing and sends a packet back. What Zilent and I started doing was doing some of these checks client-side to avoid unnecessary data transfer. I know it’s really not needed but we were at one stage trying to make it as efficient as we possibly knew how. There’s literally thousands of things in the RSCD source that can be converted to be client-sided only, even simple things like the “Welcome to RuneScape!” message being sent every time you login. Any thing that you can take away from the server and let the client handle without any problems occurring is the way to go IMO.

We even went as far to making a language list on the client such as “You swing your axe at the tree…” and instead of sending a string we’d send a INT, the client would know that INT related to that string and display it.[/quote]
