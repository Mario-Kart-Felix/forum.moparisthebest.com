public class SmithingScreen {
	
	public static void createScreen(mudclient mc) {
		/**
		 * Greates a new interface with a 450x250 panel
		 */
		InterfaceGUI i = new InterfaceGUI(450,250,mc);
		i.titlebar.label = "Smithing menu, pick an item to smith!";
		i.titlebar.x = 100;
		/**
		 * Adds the interface to a list
		 */
		InterfaceHandler.guis.add(i);
		/**
		 * Called when the label is clicked
		 */
		Callback labelcallback = new Callback(){
			@Override
			public void run(InterfaceGUI gui, RSGuiModels id) {
				gui.isVisible = false;
				System.out.println("called");
			}
		};
		/**
		 * Creates a label (text) and adds it to the interface at specific coordinates (the label can be altered at runtime)
		 * Supports: Hover, click
		 */
		final ILabel l = i.addLabel("",220, 215, labelcallback, 1);
		/**
		 * Called when the sprite is clicked
		 */
		Callback spritecallback = new Callback(){
			@Override
			public void run(InterfaceGUI gui, RSGuiModels id) {
				ISprite sprite = (ISprite)id;
				System.out.println("Clicked sprite with id: " + sprite.spriteID);
				}	
		};
		HoverCallback spritehover = new HoverCallback() {

			@Override
			public void hoverin(RSGuiModels c) {
				ISprite sprite = ((ISprite)c);
				l.label = getBarLine(sprite.spriteID);
				
			}
			@Override
			public void hoverout(RSGuiModels c) {
				l.label = "";
			}
			
		};
		HoverCallback panelhover = new HoverCallback() {

			@Override
			public void hoverin(RSGuiModels c) {
				if(c instanceof IPanel) {
					((IPanel)c).borderColor = 0x6b5e53;
				}
				
			}
			@Override
			public void hoverout(RSGuiModels c) {
				if(c instanceof IPanel) {
					((IPanel)c).borderColor = 0x463d2e;
				}
				
			}
			
		};

The code I posted is just the smithing screen code. It will not work without the actual system.

As for the actual system, it’s not that hard to write it yourself. Basically all it requires is a bit of math and some knowledge on OOP.

Idea is simple:
Create an object called an interface.
Add components to it.

How to paint the interface/components.
Have a function that is called every 100milliseconds.
In this function, you call the function “paintMe()” on the interface, which will then paint itself onto the screen and then call the function “paintMe()” on all of its components, making them paint themselves on the screen.

As for the callbacks, they’re just interfaces. Read up on this: http://leepoint.net/notes-java/oop/interfaces/interfaces.html
