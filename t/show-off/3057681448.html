I looked around a bit more and I think the structure of a model is something like this:
[Short]VertexCount
[Short]FaceCount
[Short * VertexCount]VertexY array
[Short * VertexCount]VertexZ array
[Short * VertexCount]VertexX array
[Byte * FaceCount]The number of vertex in the face. (Usually 3 or 4, triangle or quad)
[Short * FaceCount]Face backside color/texture. (Color is negative, textures are sprite file offset)
[Short * FaceCount]Face frontside color/texture. (Color is negative, textures are sprite file offset)
[Byte * FaceCount]Either 0 or 12345678, my guess is a boolean, something to do with lighting/shading.

The backside/frontside short somehow relates to a RGB color, I can’t find where, but mapping the short to a color manually is possible; I’ve done a few:

[code]-1:Null

-20769:Chair Brown:new Color(158, 71, 0)
-16913:Anvil Grey:new Color(127,127,127)
-13742:Barrel Light Grey:new Color(103,103,103)
-10571:Barrel Dark Grey:new Color(79,79,79)
-8457:Torch Base:new Color(63,63,63)
3:Wood Texture[/code]
