I have also been working on something similar, however not an editor. the RSC client has all it’s ‘interfaces’ hard-coded, its a mess. I’ve been writing an RSC component framework & have abstracted events such as rendering, keyboard/mouse events etc. This allows easy access to implement any sort of visual interface & use them with ease. Finished 1 Component already (a dialog, with text & options)

That is called simply like this, and easy to process events.

static String bankpin = ""; public static void test() { RSCOptionDialog option = new RSCOptionDialog() { public void onButtonClicked(RSCOptionDialogButton button) { bankpin+=button.text; if(bankpin.length() == 4) { bankpin = ""; destroy(); System.out.println("Bankpin: " + bankpin); } else { setTitle2("@red@" + bankpin + "*"); } } }; option.setTitle("Enter your Bank pin"); option.setTitle2("@red@*"); option.setOptions("1", "2", "3", "4", "5", "6", "7", "8", "9", "0"); }
