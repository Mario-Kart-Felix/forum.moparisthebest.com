QuestMaster_2K10 August 1, 2016, 12:18am #1

Well, my recent attempt to glitch into the RuneScape client be injecting it with malformed code, didn’t work out so well. Well here’s some newer ideas i’ve came up with, I could use a hand with ideas, and if they go well, i’ll release the steps for the entire community so they can do it themselves, under the staff’s permission.

Accessing Hidden Worlds

Yes, there are hidden worlds in RuneScape. But no, Jagex wants to remain it as a secret and not tell anyone. Just recently there was a glitch, that somebody logged into world 901! This was a hidden world that was never meant to appear visible on the world selection screen. They we’re using this world to test updates for the new Playable demo feature. They later said that world 901 was the demo world. I was thinking of the possibilities of loggin into a staff-only world (Yes they exist). I havn’t an idea though how to implement this idea.

Using Commands on RuneScape

Well this was a newer idea i’ve came up with. Not too long ago, Jagex removed the “::” command input, but instead made an interface in the client to type the command over. I can the only reason this was added, was because one of the J’Mods was attempting to type in a command. But he accidentlly put an ’ ; ’ for the last colon instead of a ’ : ', so it appeared as a chat text, and everyone seen the text, revealing that commands really did exist on RuneScape. I was thinking that if we can make the client think you’re an Admin, although you really we’rent, I may beable to access the command interface in which can be accessed by typing ’ ~ '. Although I’m not sure if the commands would actually work.

Bypassing the Chat Filter

I’ve been studying a RuneScape Private-Server i’ve made. And i’ve reviewed a chat glitch that I have on my chat filtering system. The letter y can’t be said on the server, because the stream methods for the packet are incorrect. But the server registered it as an Alt-Code that was being inputed, and as a reaction logged in the player out. (Of course, I fixed it). But if we can figure some sort of sequence of text, or an Alt-Code that can be used, we can register it to the chat filter on RuneScape, causing a glitch similiar that happened to me. It will log everybody out in the surrounding area, once the text would be said. Only those who have chat turned OFF won’t be logged out, because it won’t appear on their client. This would be a good advantage to use if you were in a Wilderness world, and are about to die, you would be able to log everybody out and kill them for free!

RuneScape No-Clip

I had an idea to walk through a seam in the RuneScape map. I’ve been viewing alot latley, and this glitch may work in several other locations. If I can find just the right area to do this, then I can possibly make it possible for the player to walk through thousands of coordinates worth of mapspace before becoming stuck and having to find a way out of it. I’ll be using Cheat Engine, to find a development key to track the steps and glitch the player modeling with the object assets near the seam, it may cause a glitch in the client to re position the player, and may move him or her off the map further into an object, like a wall. This it will appear solid to walk on, if the the areas are walkable on.

Loggin into a Member world as a Free user

Well this glitch HAS worked in the past by two players last year. There’s no information on how they did it because Jagex feared it may happen again. The players were thought of doing it again but Jagex did not release any information until further time. I have a major Idea on how this may work. Although the bad part is, it may require a different account to be a member. Here’s what i’m thinking: A member logs into a member world, while using Cheat Engine to the value of the World that has been logged into. Then log out of the account after about 10 seconds. Then go to a different member world, with the last digits the same as the one logged into the last one. Then type a username and password. Then login, right as it starts to login, use Cheat Engine to track the value of the world, and then freeze it. Then change the value again, to world 6. Then minimize it and it should take in effect making the client think that you’re still a member confusing the server. The way it confuses the server is, the server thinks that you’re a free member loggin in. But the client thinks you’re a member loggin in. Using Cheat Engine will make the client think to login to a member world, while the server is thinking the player would be in a free world. So the player will log into the member world although the player is an f2p.

Although…Any options on an NPC that requires the member variable be higher than 0 (which the default value would be if you weren’t a member), then it wouldn’t work. Although I think this was removed, but instead use world requests instead.

Interface Parsing

Well, If we can choose three different interfaces, and can find their Interface IDs, then we may be able to glitch the Quest tab. This will make it for when you click a Quest in the quest tab, instead of displaying the quest interface it will show a custom interface which can be selected through Cheat Engine. I can explain how this works if you wish to find out about them.

[b]All of the other main and maybe major glitch that can be used in RuneScape have already been found and removed, which makes it hard to find a newer way. I’ll be thinking of some new ones, but I could really use some ideas or a discussion.
