Alright, so I’m trying to determine a better way of handling events, that could have millions of different permutations. An example of this would be the Item-on-Item handling on a private server. Now traditionally, people would program all of the different permutations into one block of code, with hundreds of conditionals. Now, while it may work, it’s not very nice, and maintenance is a pain. Another way that has been gaining popularity is using scripts to handle it, which reduces the maintainability problem, but requires the use of another API to work, which I would like to avoid.

So what are your thoughts on a better way of handling such events?

if you don’t want to use a “Script” you could implement an event system, have the default event at the bottom of the chain and then special events on top for example for Item on Item it would be something like

{0} - ItemEvent(Item,Item) - do some hammer on bar event
{1} - ItemEvent(Item,Item) - DEFAULT EVENT - “Nothing interesting happens”

personally I would have items extend WorldEntity as they can be placed on the map and just have a general event for everything

EntityInteractionEvent(Entity,Entity)

this would cover everything from item on NPC item on player player on player ect.

Downfall to this system is you would have to have strict checking as you would cycle all events in the chain until the correct one is preformed.
