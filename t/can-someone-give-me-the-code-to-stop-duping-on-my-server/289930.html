You shouldn’t use variables for timers, it’s not a great way to do it.
Instead you could create something along the lines of a timer class.
An example of this is as follows.

/**
 * A timer.
 * @author beta_
 */
public class Timer {

	/**
	 * The start time.
	 */
    private long time = System.currentTimeMillis();
    
    /**
     * Reset the timer.
     */
    public void reset() {
        time = System.currentTimeMillis();
    }
    
    /**
     * Get the elapsed time.
     * @return milliseconds elapsed.
     */
    public long elapsed() {
        return System.currentTimeMillis() - time;
    }

}

Then to use it you would use.

private Timer tradeTimer = new Timer(); // Create a new Timer instance.
if(tradeTimer.elapsed() >= 60000) { // Execute the following code if 60000 milliseconds or 1 minute has passed.
    tradeTimer.reset();
    // Trade crap
} else { // Otherwise...
   sendMessage("You must wait " + (60000 - tradeTimer.elapsed()) + " milliseconds before logging out."); // Send the message
}
