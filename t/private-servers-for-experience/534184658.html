Javacisnotrecognized July 30, 2016, 4:37am #28

Well, you could move all the player processing out of the main thread. It’s kind of like this:

Thread main: forever loops through all the clients processing what is going on

Thread-0: Listening on a port. When it gets a connection, puts another client in the main threads update cycle

client threads: They spend their time waiting with a buffer of bytes for the server to notify() them. Then they swap the buffer for whatever is on the socket.

Thread-misc: Some thread which does nothing useful and causes OutOfMemoryErrors.
