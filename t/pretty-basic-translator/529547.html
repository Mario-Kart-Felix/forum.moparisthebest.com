[code=java]
import java.io.BufferedReader;
import java.io.File;
import java.io.FileReader;
import java.io.IOException;
import java.io.InputStreamReader;

public class Translator {

public static void main(String[] arguments) throws IOException {
	// buffered reader for user input
	BufferedReader inputStreamReader = new BufferedReader(new InputStreamReader(System.in));
	
	// nice little message to tell the user to enter a word
	System.out.print("Please type an English word to translate to Spanish: ");
	
	// read the line that the user typed.
	String inputtedLine = inputStreamReader.readLine();
	
	// print the translated string.
	// note: since translate is now a string, we can simply output the entire method.
	System.out.println(translate(inputtedLine));

	// buffered reader for file input
	BufferedReader fileReader = new BufferedReader(new FileReader(new File("./directory/", "file.extension")));
	// represents each line within the file
	String readLine;
	// while we can read the line and the line that we are reading is _not_ null
	while ((readLine = fileReader.readLine()) != null) {
		// we can print the translated line
		System.out.println(translate(readLine));
	}
	// close the reader to prevent them nasty little resource leaks
	fileReader.close();
}

/**
 * Translates a <code>String</code> to that of another <code>String</code>
 * @param string	The <code>String</code> to translate.
 * @return	The translated <code>String</code>
 */
public static String translate(String string) {
	// switch the string to lower case, to avoid casing issues.
	switch (string.toLowerCase()) {
	// if the string equals car
	case "car":
		// return the translated string, which is coche
		return "coche";
	// if there is no match found, return this message
	default:
		return "Sorry, but " + string + " does not exist in our database.";
	}
}

}[/code]

Rather than returning a String with an error message, throw an IllegalArgumentException (or some other unchecked Exception). That makes it easier to distinguish an error from a translated word. Otherwise you’d have to match the word against the error message for each and every word, in order to handle it somehow.

The following is perhaps a little off-topic, depending on how you see it…

The reason the method should return a String, and not print it, is best described as SoC (Separation of Concerns). Every method, class, package, what have you, should preferrably have as little responsibility as possible, but no less. However, that does not necessarily mean one responsibility, as with packages for example. Packages may have many classes and interfaces, each with one or a few different responsibilit(y|ies). But all the classes and interfaces should preferrably have something in common. To be useful in a set of use cases, or whatever.

With that said, getting the result and printing the result are two distinct responsibilities. Both of which may be part of one or a few class(es), depending on the situation. But they should preferrably not be in the same method.
