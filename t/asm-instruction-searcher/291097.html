Well, since there is hardly any info on ASM updaters around while it’s so much better then using BCEL, I decided to release my instruction searcher.
This makes the switch from BCEL (instruction list driven) to ASM (visitor driven) a lot easier.
Obviously there are cases you should use the visitor based way, because of performs reasons.
But this doesn’t work on every occasion, so an instruction searcher is vital for a successful ASM updater.

First how do you use it:
Well just like this:

		for(ClassReader cr : classes)
		{
			ClassNode cn = new ClassNode();
			cr.accept(cn, 0);
			
			//Sadly they didn't parameterized their arraylists
			for(Iterator it = cn.methods.iterator(); it.hasNext(); )
			{
				QIS s = new QIS((MethodNode) it.next());
				if(!s.hasInstructions()) continue;
				//Do what ever you want, like s.next() etc
			}
		}
