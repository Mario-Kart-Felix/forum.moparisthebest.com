moparisthebest August 1, 2016, 12:18pm #6

I’m sure there is probably a way to load the client into a C# form, although it will still have to run a JVM, and there is probably a way to push fake keyboard and mouse events to the client. The question here is how you plan on getting data from the client? You can’t easily modify it with C# code (no pre-written libraries, though you could roll your own), you certainly can’t perform reflection on the client in pure C#, without so much glue code written in Java (and probably C) that it would be much easier just to write the whole thing in Java. You can certainly react to colors and such, but then you might as well go with a pre-written solution like Simba.

So basically you need to ask yourself what exactly you wish to accomplish and why you want to write it in C#? If it’s because you know C# better, you will have to write enough Java code to learn it as well, so you might as well do the entire thing in Java, if it’s just to see if you can, go for it.
