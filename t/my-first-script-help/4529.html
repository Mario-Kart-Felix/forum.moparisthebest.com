aznph8playa July 29, 2016, 8:34pm #4

Do you have the java jdk/bin folder recognized as a path by your windows command prompt? To be sure that you do, go to control pannel, advanced, Environmental Variables, scroll down to check the Path one. Each of the semicolons seperates a different path, if you dont see one that goes to your jdk/bin folder. Open up a window and search for it. It’ll be something similar to C:/Program Files/Java/jdk.something/bin. Now go back to your Environmental Variables set up. After adding a semicolon to the end of the current paths, copy and paste the path to the jdk from your folders window. This will have set a path for java.

If this doesn’t help you or you have already done this, attempt to set a classpath. A classpath will tell the java program where it is you want it to compile. You may or maynot have to do this with your java program. It is possible to compile and execute java classes by simply being in the directory when in cmd prompt. However, if this is not the case for you as it wasn’t for me, go to the folder you’ve saved your java file in, copy and paste the extension. Go to control panel, create a new environment variable Named: CLASSPATH with contents: <your folder’s extension>

Hopefully this helped.

Good call Ron, also… meters need to be initialized, right now it doesn’t know what it is multiplying 39.37 with. Try this:

class InchToMetTable {
	public static void main(String args[]) {
		double inches, meters;
		int counter;
		counter = 0;
		meters = 1.0;
		for(inches = 1; inches <= 100; inches++) {
			inches = meters * 39.37;
			System.out.println(inches + " inches is " + meters + " meters.");
			counter++;
			if(counter == 10) { // If the counter is 10 it should output a black line....I hope xD
				System.out.println();
				counter = 0; // Should reset the line counter
			}
		}
	}
}

However, right now the program isn’t doing anything that you’d wish it to do. You see, by stating inches = meters * 39.37, you’re just resetting the variable inches each time you run the loop and since the value of inches is what determines if the loop ends, the loop will go on forever.
If you want inches to go up consistently in the loop, you can’t redefine it in the loop. I think you meant to have
meters = inches * 39.37; (or some constant that I don’t know).
If that be the case then the code would look more like this:

class InchToMetTable {
	public static void main(String args[]) {
		double inches, meters;
		int counter;
		counter = 0;
		meters = 1.0;
		for(inches = 1; inches <= 100; inches++) {
			meters = inches * 39.37; //Constant maybe different
			System.out.println(inches + " inches is " + meters + " meters.");
			counter++;
			if(counter == 10) { // If the counter is 10 it should output a black line....I hope xD
				System.out.println();
				counter = 0; // Should reset the line counter
			}
		}
	}
}
