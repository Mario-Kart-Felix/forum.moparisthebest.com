Cavendish School of Computer Science 
Department of Software Engineering 
Coding Standard
Introduction
This document specifies some rules which must be followed when writing C++ programs. The aim of insisting on adherence to a coding standard is to give students experience of the discipline required to do this, and give practical experience of the advantages to be gained by such standardisation.

Module leaders may extend the standard for their module. In such cases the module leader will specify what the additional rules are.

The areas that are addressed by this document are:

   1. Identifiers
   2. Program layout
   3. Commenting

For program characteristics which are not specified by this standard, programmers are free to devise their own style but this style must be consistently adopted.
1. Identifiers
This section relates mainly to the names used for constants, variables, functions, parameters, classes, and other objects in a program. In addition, a rule relating to global variables and rules for use of constants are also included here.

   1. Global Variables must not be used.


      ie. variables whose declaration is not inside any block must not be used.
   2. Identifier names should be meaningful and clearly identify the object named.


      eg. ``sum'' or ``minutes'' rather than ``s'' or ``m''. Single letter identifier names are only acceptable when used for the control variable in loops.
   3. Constants should be used wherever possible rather than specific literal values.


      For example, a const declaration should be used to declare constants such as PI, and this name should be used within the program code rather than use the literal value (eg. 3.14159) at each point.
   4. Constant declarations are preferred to use of #define.


      ie. use
          const int STACK_SIZE = 20;


      rather than
          #define STACK_SIZE 20 Constants should be in upper case characters. If constant identifiers are multi-word names then words should be separated by the underscore character (`_').
   5.


      eg. PI, STACK_SIZE
   6. New data types (introduced by typedef, enum, struct or class declarations) should start with an Upper-case character and be followed by lower-case characters. If such identifiers are multi-word names then the first character in each word should be in upper case.


      eg. Queue, BankAccount

      All other identifiers should be in lower case, except if they are multi-word names when the second and subsequent word should start with an upper case character.

      eg. variable names ``minutes'', ``hoursWorked''; function names ``print'' or ``getInput''

2. Program layout
This relates to the physical layout of code on the page. This layout should reflect the logical structure of the code.

   1. There should be no more than one statement per line.
   2. Indentation by n spaces should be used to indicate blocks of code, eg. within a function, a loop, a conditional statement, a declaration, etc. The value of n should be in the range 2 to 8 inclusive and it should be consistently used throughout the program.
   3. Binary operators should be separated from their operands by a space.
   4. The opening brace ( `{' ) of a block should be placed on the line following the preceding text. Both the opening brace and the closing brace ( `}' ) should be placed on lines by themselves, and they should be vertically alligned. This rule applies in all cases (eg. function definitions, class declarations, control structures, etc.) with the single exception of the do ... while control structure (where `while' should appear on the same line as the closing brace - see below).


      An example illustrating use of this rule is given by the program in Section 4 (for function definitions) and by the Queue class declaration under Rule 8 of this section, shown below.

   5. Opening and closing braces ( `{' and `}' ) should always be used to specify blocks of code which are part of conditional statements or loops, even when the block of code contains 1 statement. An optional exception to this rule is the layout of the if ... else if ... form of the if statement. This can be layed out as:

   6. There must be at least 2 blank lines immediately after a function definition.
   7. When declaring a pointer variable, wherever possible the `*' symbol should appear next to the data type.


      eg. int* ptrToInt;
   8. When declaring C++ classes, declarations within the public, protected, and private categories should be indented by n spaces (where n is the same value as adopted for Rule 2 of this section). These categories (when they exist) should appear in the order public, then protected, then private. The keyword private should always appear if the class contains a private category.

3. Commenting
Comment statements should be used wherever it is felt necessary to provide additional documentation to clarify the code. The following are only minimal rules for including comments:

   1. Every program should have a comment at the top of the program giving the filename in which the program is stored, the purpose of the program, the author, the date the program was created, and the date the program was last modified.
   2. Every class declaration should have a preceding comment describing the purpose of the class.
   3. Public members (data members and member functions) of class declarations should be documented, using comments where it is felt necessary.


      Rules 2 and 3 of this section are illustrated in the `Queue' class declaration in the previous section.
   4. Where it is felt necessary function definitions should have a preceding comment describing what the function does (unless this comment has already been supplied for rule 3 above).
   5. The comment at the top of the program (required by Rule 1 of this section), and the comment preceding function definitions (required by Rule 4 of this section), should be of the /* ... */ variety. In all other cases /* ... */ should be used for multi-line comments and // ... should be used for single line comments.

4. A simple example program which conforms to the standard

/* demo.cc   : inputs 10 integers and outputs them in reverse order
   Written by: ...
   Creation date         :  5/11/97 
   Last modification date: 12/11/97 
*/

   
#include <iostream.h>

/* read data into array */
void readData( int size, int data[] ) 
{
   for ( int i = 0; i < size; i++ ) 
   {
      cout << "Enter number " << i + 1 << "...";
      cin >> data[ i ];
   }
}


/* output data from array in reverse order */
void writeDataReversed( int size, const int data[] ) 
{
   for ( int i = size - 1; i >= 0; i-- ) 
   {
      cout << data[ i ] << endl;
   }
}


void main() 
{
   const int ARRAY_SIZE = 10;
   int input[ ARRAY_SIZE ];

   readData( ARRAY_SIZE, input );
   writeDataReversed( ARRAY_SIZE, input );
}
