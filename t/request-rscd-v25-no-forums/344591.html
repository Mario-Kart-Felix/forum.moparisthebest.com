Zilent July 31, 2016, 5:10pm #7
dimebagx:

Is there anyway to run these servers without xamplite?

The only reason for the database is to load and save data from when players log in and out. You can modify the code to load a player from the server configuration XML file if you wanted to. It would take a little bit to chop and hack the log-in server out of the source but it’s entirely doable.

You’ll find a PlayerLogin handler in the server section that will fire when the client sends a login packet. You can simply find what happens when the login server should have responded and emulate that manually as it’s only String and numeric data, really. A similar tactic can be done with the client itself in order to just walk around on the client without having a server. It’s just cutting out the middleman

Client -> Login Packet -> Server -> Login Server -> validate login -> Return Packet -> Server -> Client
-to-
Client -> Login Packet (won’t be validated) -> Server -> Client
