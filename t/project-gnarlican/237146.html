[center]Hello, and welcome to the Gnarlican development topic! Here you can find out all about my new server Gnarlican The point of this server is to revolutionize the way you administrate and run your server. Using a fully custom GUI that controls nearly everything that goes on, on the server. Lots of performance updates such as Grahams singled threaded tutorial, EventManager, ConnectionFilter, Rebuilt server, making it more stable and unseen to hackers. With a 99% Anti-DoS/SYI or any other sort of nulling program. While playing this server you wont get any sort of lag, disconnections, un-fair diseplin. This server will be fully auto-mated to Ban/Kick/IPBan/ any player that wishes to break the rules, in other words the server will know if your not following the rules all by it’s self. Their will still be a Report Abuse and you still must use it normally because not always will the auto-mated bot work, and also it will help you on getting any staff positions in your future. The main point of playing this server is to just have fun and staying safe while playing.
Note: This project will take some time, now that i only do this kind of stuff for a little bit of time now. About an hour or two a day.[/center]

Features:

[ul][]1) Using packages heavily.
[]2) Rebuilt hole server.
[]3) Faster IO that runs every 30ms instead of 500ms.
[]4) Commands now work through a hashmap - should be faster and more reliable. And a lot easier than a huge if statement for all of them.
[]5) A flexible banning/muting system, allowing bans by username, UID and IP address. You can mute and un-mute as well.
[]6) Garbage collector will be called once every minute, and when a player is removed.
[]7) Moved the client methods to the ActionAssistant and ContainerAssistant classes. Helps to split things up for readability.
[]8) PvN(Player vs. Npc) - Perfect realistic and complete.
[]9) Packets are handled through a hashmap, so are commands, for greater speed and flexibility.
[]10) The server/client communication has been switched to using NIO instead of polling every 500ms. This gives a much faster response time and the selector used instead means there is much less load on the CPU during idle periods.
[]11) All the input/output is now done through the NIO API! Should be faster, and the selector (as opposed to polling) means that it is a lot less strenuous on the CPU when the server is idle.
[]12) The Stream class will now automatically create a larger buffer if it is needed.
[]13) NPC handling has been rewritten from scratch for greater stability and speed. So has dialogue managemen, item spawns, shop management and object management.
[]14) Server GUI - Can control pretty much anything on the server with this gui.
[]15) EventManager - A better type of processing, timers, ect.
[]16) Single threaded - Removed thread-per-client so i don’t rape my CPU while running the server.
[]17) ActionFloodProtector - Stops mass clicking for good.
[]18) Improved on the synchronization to fix threading issues
[]19) Using constants on all the main things.
[]20) Dumped well over 500 null packets and fixed them, making the server A LOT more stable, and faster.
[]21) Complete debuging - Console prints nearly everything.
[]22) Rebuilt the processing method - Still at 500ms but it’s not a loop in a loop, it runs on its own thread. Now it’s stable and fast
[]23) Built a IPManager - Manages bad IPs to be blocked from connecting by default. Retrieve ip from a hostname/hostname from ip.
[]24) Added prayer - Perfect switching, Draining, Level required, and Dialogs.
[*]25) Finished perfect herblore (After 2 days) Everything works, you need correct levels to make potions and identify them.[/ul]

Winterlove:
As many of you know, the server Winterlove(Blakescape) is a very unstable server with thread per client, the processing being very unreliable as it was a for loop inside another for loop. In gnarlican these minor errors have been fixed and will not effect the server no more. Rebuilding the processing method running it as a event, making it a lot more stable and reliable.
