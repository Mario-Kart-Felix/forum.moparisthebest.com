[quote=“semaphore, post:24, topic:309992”]You’re right. Learning c just to screw around with JVMTI will leave you confused and frustrated.

You should learn c to better understand the ultimate foundation of java. There’s an old quote that resurfaces every now and again at the client sites I work with. It seems that there’s a generation of coders whose first language is java. This is somewhat troubling because of just how much one misses by only learning java.

Anyway here is the quote:

If you know c, java is simple.
If you only know java, good luck with c.

I’m sure there are exceptions to the rule but I’ve worked with hundreds of developers. I’ve never found an exception. The only people that have ever disagreed with me on this are those that learned java first. I’ve even had people that learned java first say ‘Oh I had no problem learning c!’, but when given a task… say… create a small app that reads from shared memory and forks a new process on write to the memory and notifies the parent process by writing to a predefined shared block of memory… deer in headlights. In fact most of them don’t even know how to fork a new process.

If you learned C first, you would agree with me that java is morbidly simple in comparison. (TBH I look at java in much the same way most java developers look at javascript).[/quote]
