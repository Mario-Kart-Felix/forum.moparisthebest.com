Why Aren't Web Documents Compiled?

General Programming
Lin August 4, 2016, 6:46am #1

Hey guys. I’m not a web developer by any means. Hell, I can barely make an HTML document that displays text! But I do know that standard web documents and php files and such are sent as plain text. Why?

It confuses me because if HTML documents (we’ll just use them as our primary example from now on) were compiled to byte code to be interpreted by the browser, not only would speed increase when constructing webpages, but their size would decrease and make viewing (especially on mobile devices) them much faster.

So help me out here. Give me the advantages that outweigh the pros I listed above, because I can’t think of any other than easy source viewing.

~Lin

davidi2 August 4, 2016, 6:46am #2

For the same reason XML is not compiled. HTML is just a markup language. You could create a new file format to create webpages, but because HTML is the standard and easy to write people just decided to not do it.

RSCShadow August 4, 2016, 6:46am #3

It’s too late at this point, really. Every browser vendor has come up with their own way to actually construct the DOM and run the Javascript. When HTML was designed, they had no idea it would become this popular.

Either way, a few website are just doing the entire thing in Javascript and CSS and then minifiying it, doing away with HTML completely.

Since HTML has been covered, PHP actually has caching mechanisms which cache PHP opcodes/bytecode. Some popular ones are APC, XCache, etc.

en.wikipedia.org

PHP accelerator

A PHP accelerator is a PHP extension designed to improve the performance of software applications written in the PHP programming language. Most PHP accelerators work by caching the compiled opcode/bytecode of PHP representation of php files to avoid the overhead of parsing and compiling source code on each request (some or even most of which may never be executed).
