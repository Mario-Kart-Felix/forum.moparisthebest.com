mod_taharok July 30, 2016, 1:45pm #13

Java Archives can be run as executables using the Java Standard Binary Edition 2 (javaw.exe). It reads the manifest file inside the jar to locate the main class and execute it. Inside of your Server jar file, the Manifest points to Main.class. Main.class is the main class, so the Jar will auto-run. No problems there.

However, inside of your Client jar file, the manifest directs the jvm to also run Main.class. Now, the problem is, your client’s main file is Main2.class. Therefore, javaw.exe will output an error when trying to auto-run the jar file.

Understand now?
