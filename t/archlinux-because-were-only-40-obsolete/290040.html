moparisthebest July 31, 2016, 9:25am #3

Cutting-edge isn’t always a good thing, open source is all about release early, release often. That means that every release isn’t stable, in fact most early releases aren’t stable. I would bet that a linux distro that freshly compiled all programs in it’s distro as soon as they were released would not even be remotely usable, because the bugs in early releases would cause it to crash all the time.

There is a fine line to walk between cutting-edge and stable, obviously when you install debian stable you aren’t looking for a system that will be cutting-edge, but instead you want one that will be perfectly stable, and since they spent 2 years working up to the release with thousands testing it and are only doing bug and security fixes without trying anything new chances are it will be stable. Distros like Arch have their place too, but since it is a ‘rolling release’ and they release packages when and as they feel, things have a tendency to break more and you don’t want your network or ssh breaking down when you are running a server.
