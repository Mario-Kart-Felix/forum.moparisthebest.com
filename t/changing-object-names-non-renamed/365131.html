This is client side on Non Renamed so go ahead and open up your client’s class46.java,

There should be no renamed objects in there ATM which unlike class8.java makes it harder for newbies to understand because they don’t have a bunch of models they can follow, anyways look for this

        class46.method582(true, aClass30_Sub2_Sub2_753);

underneath it you should see this

        return class46;

now under

        class46.method582(true, aClass30_Sub2_Sub2_753);

put this

switch(i) { }

now you can put any renamed objects inside the body of that switch statement.

so lets say I wanted to rename an object you guys can use this format

case ####: class46.aStringArray786 = new String[5]; class46.aStringArray786[1] = "NEWOPTION"; class46.aByteArray777 = "NEW EXAMINE.".getBytes(); class46.aString739 = "NEW OBJECT NAME"; break;

now you can just fill in the spots, so the case number is w/e the object ID is like 410 is an ancient altar.
the NEWOPTION is an option that would replace the first option of the object so with an ancient altar it would replace the “pray-at” or w/e it said. The NEW EXAMINE replaces what it said when you examine the object. And finally the NEW OBJECT NAME is the new object name, you can rename it w.e you want.

if your compiler throws 100 errors at you than you probably didn’t put it inside the switch body so make sure it looks like this

switch(i) { case ####: class46.aStringArray786 = new String[5]; class46.aStringArray786[1] = "NEWOPTION"; class46.aByteArray777 = "NEW EXAMINE.".getBytes(); class46.aString739 = "NEW OBJECT NAME"; break; }

That’s about it.
