most servers fixed validation for item spawning/spoofing. the can however force the client to believe you have an item when you don’t (great for those pesky servers that handle click items to give items without actually checking whether you have the item or not)

you can use the click_object packet to trick the client to clicking on any given object (project insanity for instance, doesn’t check to see if the object exists at those given coordinates) and use such things as ladders/portals etc. to teleport amongst the game (great for teleporting during pking as most users “oh look, portal at home: dont need to check if user is in wild/combat/etc”)

you can also do this to the click_npc / click_item packet for any instance of teleporting

i do believe method46 can spawn an npc in the location (not sure about it. just going off memory - don’t have my client at hand)

you can attempt to change the position (haven’t tried it in this manner) using the Entity.setPos() method.
