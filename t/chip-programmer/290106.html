sorry peterbjornx, but that is the worst explained idea in history, please elaborate how exactly counters connected to a parallel port would allow you to flash an EPROM chip

Like the data bus on the parralel port is connected to the Q0…7 I/O’s on the chip, something needs to supply adresses, so you could hook up a couple of interconnected counters to the address lines and drive it by one of the status lines on the parport, so like

JAVA example

for (int i = 0; i < data.length;i++){
parport.resetBit(SOME_STATUS_LINE2);//#CE low, enable chip
parport.setByte(data);
parport.setBit(SOME_STATUS_LINE2);//#CE high, disable chip
parport.setBit(SOME_STATUS_LINE);//addr++
}
