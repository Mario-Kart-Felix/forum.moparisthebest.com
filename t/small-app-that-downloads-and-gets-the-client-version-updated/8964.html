Nerice July 29, 2016, 9:12pm #1

Version finder now updates itself, to show how easy and powerful bcel is to directly change classes, and save them. Very useful for bots used in RS to update themselves, you write your code once, let the bot change and write what needs to be updated itself after a RS update!

Its left at the previous RS version, so the first time you run it, it will say RS has been updated, say its saving its updated self, and the current version of RS. The next time you run it, it will just say the version of RS (Until the next time RS updates…)

The downloader can be used to download any file off a url. First parameter is where to save it, and the filename. Second is the url to download from. Defaults to getting runescape.jar if ran with no arugements.
bcel.jar and regexp.jar is included, as they are required and not sure how many people would have it. Classes and source is included.

Intended as examples for people that like to look, and be able to figure out what is being done.

Tried to make it simplistic to understand, and used the most basic classes. All heavily commented. All java.

Three classes, as the main program is depenant upon the other two, but the downloader can be ran standalone, and the version finder can be used in a different app, just give it the client class. Up to the user to put in a getter method if they would like to integrate into something else to determine if there has been a update. Just takes one method added that returns the variable…
