Welcome to iClarity.

iClarity is back - better than before. iClarity was a PKing-based Runescape private server which I hosted for a short period, and then begun to release in versions. Now, I am going to put most of my spare time into this (apart from CoD4 gaming), however, I have school and homework etc so I won’t have as much free time as I did during the Summer.

Over the past two months, those in which I have been in-active (Not been creating a private server), I have learnt a great deal more about the Java language. I have been focussing on certain aspects; one of which was UI’s.
So far, I have been working on this for a week, and this time, I plan to get a lot more done. Last time, I focussed mainly on the PKing and combat-based aspects, but this time, I plan to focus a lot more on other features such as skills, minigmes and quests. Combat will still play a big role in the server.

This time I plan to host this, and not release it, however, it will probably end up being released anyway.

[size=12pt]What is this server based off?[/size]

This is based off iClarity V1.05, though, I am going to totally re-write almost every feature of iClarity, as they were shit. This time, I plan to get stability at it’s optimum position before I begin creating any big in-game feature such as a minigame etc.

[size=12pt]What will the server contain?[/size]

iClarity is going to be a full-world based private server. All features, aspects, npcs, minigames and quests are going to be spread across almost the full Gielinor world. I plan to re-write the previously made minigames such as duel-arena, fight cave and barrows. Other new minigames will also be created. iClarity will also include high-level monsters from which you can gain rare drops from. These include the dragon chain from the kalphite queen etc. The server will have a optimum economy.

[size=12pt]Feature log[/size]

12th October

  • Combat is now working, however the delays in-between attacks aren’t working yet.

11th October
Sorry for 7 days of no updates; I have been busy IRL.

  • Melee max hit calculation is now Perfect.
  • Combat has been worked on a little bit.
  • New saving method tested and it works (Saves when client is closed, player disconnects, an exception is thrown, or the player logs out).
  • Fixed a bug - when you equipped a shield when you had a two-handed weapon equipped, an exception was thrown.

4th October

  • Characters do not save every 5 seconds now; they save every 5 minutes. However, when you disconnect, logout, or get updated, your character will be saved. This is much more efficient.
  • The skill of Runecrafting is beginning to be written.

2nd October

  • Started to re-write combat; no timers are being used this time.
  • The logging in glitch where you get a black screen is FIXED.

1st October:

  • Completely removed PvP combat and magic. Weapon speeds, and weapon animations are now contained in 2D arrays rather than within switch-statements.
  • Skill ‘timers’ such as fishing and cooking now use events instead of processing. The same will apply to the combat ‘timers’ when I re-add and re-write combat.
  • Server has been re-packaged slightly.

30th September:

  • Frames are now situated in a separate class, and methods and parameters, renamed. To call them the server uses a singleton to avoid the frame methods being static.
  • Commands are now managed via an interface (One child class per command) and the amount of commands there are have been reduced greatly.

29th September:

  • The way in-coming connections are handled have been re-written.
  • The server only allows one connection per IP using an arraylist to contain the current-connections.

Thanks, Primadude.[/CENTER]
