Working on a Java Disassembler

General Programming
Kattoor August 4, 2016, 5:13pm #1

Let me start with saying: yes, I know we have javap for this.

I wanted to do something about my lack of knowledge of concepts such as decompiling, disassembling, bytecode, etc. That’s why I started off by making a simple HEX viewer which allowed me to browse the HEX of Java Class files.

As it seems rather intriguing to be able to at least read bytecode, I started reading small parts of this document, which provided me with the necessary insight to start building my own Java Disassembler.

I have been working on it for a solid three to four hours (whilst continuously learning) and came up with a basic version able to read the…
… Class file’s magic number (0xCAFEBABE)
… Major + Minor version
… Constant Pool Count
… Constant Pool
… Access Flags
… Class’ and super class’ names
… Interface Count
… Implemented interface’s names
… Fields count
… Fields: access flags, name index, descriptor index and attributes count
… Fields’ attributes: attribute name index, attribute length and info for attribute
