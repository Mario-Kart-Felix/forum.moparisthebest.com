Enabling Quests in MoparClassic

Took me a while to figure out how to get quests working in MoparClassic, so I thought I’d share how I did it. It’s actually really simple and shouldn’t have taken me so long to figure out. The org.moparscape.msc.gs.quest.QuestManager class looks for the quest classes in a jar file in whatever directory you are in. This tutorial assumes you are running your server from the GameServer folder. So here’s what you need to do:

Hard Way (If you’ve made quests that aren’t already included.)

[ol][li]Build the server with the command, “ant build”. This compiles the quest source files into class files.[/li]
[li]Create a new folder in the GameServer folder called “org” and inside that create “moparscape” and inside that create “msc” and inside that create “gs” and inside that create “plugins” and inside that create “quests”. The final directory structure should be “org/moparscape/msc/gs/plugins/quests”[/li]
[li]Now go back to the GameServer folder, and then go to “bin/moparscape/msc/gs/plugins/quests” and copy every class file in there[/li]
[li]Go back to the GameServer folder and paste all the class files into the “org/moparscape/msc/gs/plugins/quests” folder you made[/li]
[li]Go back to the GameServer folder again, and compress the whole “org” folder into a zip file.[/li]
[li]Change the file extension from “.zip” to “.jar”[/li]
[li]Start the server, and one of the messages it displays should be “Loaded 7 quests successfully (0ms)”[/li][/ol]

Easy Way:

[ol][li]Download This.[/li]
[li]Put it in your GameServer folder[/li]
[li]Start the server, and one of the messages it displays should be “Loaded 7 quests successfully (0ms)”[/li][/ol]

If anyone knows a better way to get it to recognize your quests, let me know and I’ll edit accordingly.
