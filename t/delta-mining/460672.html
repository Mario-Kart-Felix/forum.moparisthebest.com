zyle1992 August 4, 2016, 12:13am #2

Normally it helps to break the process down into smaller and smaller steps, until you can put those steps into code. The process in this case would be

First iteration:
Add code to the server so that when a player clicks an ore object, it mines it.

But we can’t really convert that into code very easily, so we break it down again.

Add code to the Mining Handler for the specified rocks, at which point will perform an animation, and give the player ore, replacing the rock with an empty one.

Which is a bit better, but still not quite there, because we need to know which animation, for how long to play it, which ore to give the player, and experience to give the player, and replacing the object with an empty rock, depending on the type of the filled one.

Luckily for you, most of that information is in other servers, and you can pick and choose which parts you want, and change it to your needs.
