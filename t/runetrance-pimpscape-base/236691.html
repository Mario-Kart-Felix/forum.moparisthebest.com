This is going to be fun.
Welcome to the RuneTrance project thread!

Some information you should know:

  • RuneTrance is based off the original pimpscape, by xerocheez
  • Will be cleaned to death, stabilized, and handled left and right.

What I have done:

  • Removed thread-per-client (Thank you Graham!)
  • Added EventManager! (Thank you Graham!)
  • Added awesome teleports using EventManager for precise timing.
  • Added DialogueManager class, holds methods for dialogues (Thanks to Graham for a few of the interfaces :whistle:- Uses methods like
    ns1(string message1, int npcID); ~ Which is npc says “message” and the npcID is for the id for the moving head on the frame…
  • Added CommandManager class! Holds every command and loads them in the client when called upon. Cleared out customCommand() method. (Do you have any idea how many commands are in pimpscape?)
  • Added FrameManager class, going to contain all frame#() methods and possibly more
  • Finished FrameManager class, going to do renaming for frames I know.
  • Added AnimationHandler class… handles all animations.

What’s going to come to RuneTrance?!

  • Specials
  • Good timers for pking

^^ That’s it as of now, I can’t say for sure what yet.

Why pimpscape? Why wouldn’t you use deltascape?
This is for a lot of good practice! I get to practice starting new classes, constructors, instantiating, and there will be a reward in the end! The entire source will be following correct code conventions and I will release the cleaned version (I think)!

Will this source be packaged? (files in folders)
No. Unfortunately, a lot of this community doesn’t understand simple things like a folder, and I want this to be a stable source for everyone to use that can’t understand Shard Revolutions.

I will try to update this. (Not totally often, I’ll try.)
