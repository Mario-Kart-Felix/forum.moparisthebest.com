The Wanderer:

I’m not network guru, but 0 ms update time for 2000 players all sending chat packets seems a bit…farfeched? I mean, for the value to be truncated down to a zero it would have to be 1 two thousandth of a second, which is insane. Could you perhaps provide more precise data, perhaps in the form of nanoseconds. (if it says 0 ns, im sorry but im calling bullshit.I love this project, but theres no way you could update 2000 players in 1/ 1 x 10^9 of a second).

Well it does just up to 1/3 and higher values some of the times that we do the command. To be honest it actually surprised me too, but I’ll let you in on a couple secrets that may explain how we keep it so low. First, networking and player updating are handled in two separate threads, and the packet reading and writing in the player updating thread is nonblocking, so read and write operations don’t really take much time because write operations fill a buffer, and read operations don’t occur until a whole packet is ready, and all they do is drain a buffer. Then as far as the player updating code goes, we have various optimizations put in place to made it faster, and this test was running on my desktop computer with a Quad Core i5 @ 3.3 GHz using around 80-90% CPU.

But using nanoseconds is actually a really interesting idea, I’ll add that and made another video for you! I HIGHLY doubt that it would say 0 ns because even I don’t think that would be possible.
