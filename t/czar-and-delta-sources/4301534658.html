if you think of it Hyperion isn’t the greatest either, the cons of Hyperion is the highly abstracted design which by default will slow the server down since the content you do add will have to go through so many sets to even be processed, on the other hand a far less abstracted design like RuneSource will have far more performance then Hyperion or Apollo, but suffers from the WinterLove design which means you will have to add a lot of patch code to fix problems up, things that wouldn’t need to be patched up like in a design with Hyperion or Apollo. So really it’s up to what the person is looking for, from a programmers perspective a Hyperion or Apollo design would be a better choose as content development would be easier, from a less knowledgeable programmer or a person who uses a WinterLove design server, RuneSource would appeal more to them, since a lot of the people who started Java programming here started on the WinterLove base meaning they will find it easier to program then a Hyperion or Apollo server.
