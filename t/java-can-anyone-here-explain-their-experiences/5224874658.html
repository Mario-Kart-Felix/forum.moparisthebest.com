Protosstribe August 4, 2016, 8:57am #30

I buy a lot of books and read the parts I am currently interested in, saving the rest for later. Then I usually apply what I learn from them in practice. But, I usually never write the examples from the books that have any, because I prefer to read the theory and think on my own. Before I write any code, I usually think about sound architecture and design. That also includes following any of the well known principles (for instance KISS (Keep It Simple, Stupid), low coupling and high cohesion, SoC (Separation of Concerns), IoC (Inversion of Control) and SOLID (Single responsibility principle, Open-closed principle, Liskov substitution principle, Interface segregation principle and Dependency inversion principle)).

All the above might not be relevant to you right now. But if you continue and practice you’ll eventually find it relevant at some point in the future. However, I can recommend a book called Introduction to Java Programming (9:th Edition) by Y. Daniel Liang. I have the 7:th Edition. It’s a thick book that covers all you need to know right now.

Currently I have 76 books in programming languages (and other kinds of natural languages for that matter) in my shelf. All ranging from primer books to advanced ones (compiler theory (and language transformation in general), program analysis, neural networks, reverse engineering, computer forensics and malware analysis, OS design and general program architecture and design). If you are interested in any specific area, perhaps I could recommend some more books. I have books within “most” areas.
