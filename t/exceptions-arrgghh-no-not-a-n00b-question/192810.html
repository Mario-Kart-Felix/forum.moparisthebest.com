semaphore July 30, 2016, 9:04pm #2

Do you use the built in class-tooling to intercept/modify bytecode before it gets loaded or do you use a 3rd party suite to do so? Either way this is likely thrown when the class definition has changed, say a method signature is different, but classes that call that method have not been changed. For example:

Class A:
public static void doSomething(String str);

changed externally to

Class A:
public static void doSomething(String str, int i);

and Class B calls doSomething(“here”). This would give you that error when the jvm tries loading the classes.
