sk8rdude461 August 4, 2016, 3:00pm #4

Definitely is a packet issue.

You’re not reading the interface ID from the client, so all the values are off.

https://rswiki.moparisthebest.com/index.php?title=317:Item_action_1 If you read here. The client is sending 3 values. So your server should be expecting… 3 values.

But looking in the process packet method:

public void processPacket(Client c, int packetType, int packetSize) { int itemSlot = c.getInStream().readUnsignedWordA(); int itemId = c.getInStream().readUnsignedWordBigEndian();

You’re only reading two of them.

The datatype of the frame (interface id) is a “Little Endian Short Special A” (So says the rs wiki I linked above), so lets read it like so:

Make sure to have that BEFORE the itemSlot integer, but AFTER the start of the process packet method.
