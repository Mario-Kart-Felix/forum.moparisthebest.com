This is the showoff section so I just thought I’d showoff something I just got working for the official server. I have no idea if this has been done before, but considering how far advanced some 317’s are I wouldn’t be surprised. Maybe even it’s commonplace. I’m out of the loop.

Basically, I can edit anything about a definition (or add new definitions) completely server sided and have them go out to any client that connects to the server.

The core behind this process is Graham/Discarded’s cache system, as well as Graham’s update server that is in ScapeEmulator. I simply added the ItemDefinition encode method (which I know has been done before) and wrote the simple 50 line editor.

Also worth noting that because the official server uses the cache definitions to populate its item information, there is no issue worrying about inconsistencies between the client and server. Anything that uses the items name (or any other value) server sided will update as well

Example (no client editing done)

All I do is change the item id at the top and add any modifications I want into the modify method, like so:

[code=java]public final class ItemEditor {

private static final int ITEM_ID = 4151;

private static void modifyDefinition(ItemDefinition def) {
    def.name = "My big black whip";
    def.originalModelColors = new short[] { 528 };
    def.modifiedModelColors = new short[] { 800 };
}

public static void main(String[] args) {
    try {
        System.out.println("Loading cache and finding item entry... ");
        Cache cache = new Cache(FileStore.open("data/game/cache"));
        Container tableContainer = Container.decode(cache.getStore().read(255, 19));
        ReferenceTable table = ReferenceTable.decode(tableContainer.getData());
        int fileId = ITEM_ID / 256;
        ReferenceTable.Entry entry = table.getEntry(fileId);

        if (entry == null) {
            System.out.println("No entry found!");
            cache.close();
            return;
        }

        Archive archive = Archive.decode(cache.read(19, fileId).getData(), entry.size());

        int archiveMember = 0;
        for (int member = 0; member < entry.capacity(); member++) {
            ReferenceTable.ChildEntry childEntry = entry.getEntry(member);
            if (childEntry == null) {
                continue;
            }
            int id = fileId * 256 + member;
            if (id == ITEM_ID) {
                ItemDefinition definition = ItemDefinition.decode(id, archive.getEntry(archiveMember));
                System.out.println("Item definition found and decoded, modifying...");
                modifyDefinition(definition);
                System.out.println("Writing changes to cache...");
                cache.write(19, fileId, archiveMember, definition.encode());
                break;
            }
            archiveMember++;
        }
        System.out.println("Cache item update complete!");
        cache.close();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

}[/code]

Running the program will modify the cache, including updating any CRC/version numbers.
