moparisthebest July 31, 2016, 5:50pm #6

It all depends on what you are doing, and where and how you are doing it. Most of the time it boils down to preference.

But if you want to be a programmer, you should probably get experience in different types of programming languages. These are just examples and there are multiple languages you could use for each category.

Object Oriented - C++, Java, Python
Native language - C++, C
Procedural - C, Pascal
Functional - Lisp, Scheme (neat as well, EVERYTHING is done via recursion)
Logical - Prolog (this is neat, it may just blow your mind)
Declarative - SQL, such as MySQL (you just say what you want to do, don’t care about how it’s done)

I’ve used all of the above at one time or another, and it is interesting to see how different the same program can be in two different languages. Sometimes the same program will be 5 lines in Prolog, and 100+ in C, or vice versa, depending on what you want to do. It will give you a good base for the future so you can use the right tool for the job.

Cheese_Police July 31, 2016, 5:51pm #7

While you are mostly right your categories are just paradigms. Most of the languages you listed are multi-paradigm, i.e. they support several styles of programming.

OO - C++, Java, Python, Common Lisp, Delphi, C#, VB.NET, F#, OCaml, … ?
Native - C++, C. Lots of languages can be compiled to native code, like some dialects of Lisp, OCaml, Delphi, others…
Procedural/Imperative - All of the above to a degree.
Functional - Varies. Haskell is pure functional, Lisps support functional styles very well, but most aren’t strictly functional. Python is decent here as well.

Moparisthebest:

Functional - Lisp, Scheme (neat as well, EVERYTHING is done via recursion)

Nitpicking here, ‘Lisp’ (or LISP, though this usually implies McCarthy’s original dialect) is quite a broad generalization. Scheme itself is a Lisp, as is Common Lisp, Clojure, and a bunch of others I don’t care to list. Lisp refers to the family of languages that are Lisp dialects. Saying that everything is done via recursion is simply not true. In fact, Common Lisp was the first standardized OOP language (CLOS). That on its own does not break away from being purely functional, but OOP tends to lean towards imperative programming, of which Common Lisp has always allowed for, along with most others. Clojure, for example, is nearly pure functional in the sense that all data structures are immutable, however there are mutable references (atom, agent, ref), and naturally being on the JVM imperative code is possible (though usually hackish).

If you want to see a language where ‘everything’ is done via recursion, take a stab at Haskell…
