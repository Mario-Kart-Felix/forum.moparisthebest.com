Mopman July 31, 2016, 6:18pm #49
Moparisthebest:

I was trying to keep it simple for him. To get technical about it, 4GB is the max amount of RAM any x86 (32-bit intel) operating system can directly support at any one given time. Linux can support more with a 32-bit OS, but it basically just swaps which sticks it uses at any given time and hides this hackery from the rest of the system. So it’s still best just to use a 64-bit OS when wanting to use over 4gb of RAM.

No, 4gb is (roughly) the amount that can exist on one page using 32-bit addressing. Each process can still use a different page, so having over 4gb is still perfectly useful (with PAE) as long as no /one process/ wants more than 4gb of ram, which is pretty rare really isn’t it.

For some reason Microsoft in their infinite wisdom limited most of the consumer editions of Windows to 4gb, though. Linux is fine, as are the server editions (even Win2k supports 128gb, iirc :/)

EDIT: Ok Win2k was 32, 2003 was 128.
