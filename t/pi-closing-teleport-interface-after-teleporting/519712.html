Whatsup guys,

I’m not sure if this whas already released but i just created it for my own project today.

What does this do? It closes your Teleport Interface after teleporting.
I thought about this because its verry annoying when u are using a Pking teleport and you are bridding you need to close the tab first before u can attack someone.

Ok so here we go:

Find your button ID
For Example ID: 176146

Go to your Clickingbuttons and find that Button and it will look like this:

case 176146: c.getPA().spellTeleport(3366, 3266, 0); break;

So you can see it doesnt do anything, it only teleports you to that location but it doesnt close the Interface.

Under the coords add this:

What this do: It checks what spellbook the player is using> 0,1 or 2. So when you clicked on the teleport it gets you back to your spellbook.

So your button will not look like this:

		case 177215:
		c.getPA().spellTeleport(3303, 9375, 0);
		c.setSidebarInterface(6, c.playerMagicBook == 0 ? 1151 : c.playerMagicBook == 1 ? 12855 : c.playerMagicBook == 2 ? 16640 : 12855);
		break;
