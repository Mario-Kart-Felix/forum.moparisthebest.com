Well, the green text is known as comments. They are ignored by SCAR, and can be used either as information to the user(instructions on how to use the script), or for plain decoration. You should skim through this in case there is any instructions specifically for the script. The pink text, now that is called a “string”, in a nutshell, it’s considered text to the script. But generally, you should copy everything provided in “code tags” into SCAR, just for a reference of what it is. I’m not sure what links you visited, but if you could link to the script your using, or post it here(in code tags, I’ll explain further down), we could help you set it up. Scripts are always different, so knowing some basic things about SCAR coding and SRL will help with setting it up, usually it’s just a case of changing a few strings and replacing False with True, but it depends.

Code tags -

You use them like this:

And you can put say, a script inside them:

program CodeTagExample; begin Writeln('Code tags are a bit more useful on large scripts'); end.

And it will appear like:

program CodeTagExample; begin Writeln('Code tags are a bit more useful on large scripts'); end.
