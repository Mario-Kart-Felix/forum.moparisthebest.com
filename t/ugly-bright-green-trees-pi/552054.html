sk8rdude461 August 4, 2016, 3:13pm #8

As far as I know, my cache is corrupt as well.
Here are some possible solutions:

  1. gzip the 1570.dat, edit the name to be “1570.gz” and then use galkons index repacking method to pack the model (I did this, and my trees are find and dandy now)
  2. Place the 1570.dat into a “Raw” or “Models” folder, and preload the model.
  3. Use ModelCompression to load the model from a model.dat/idx file, instead of the raw folder (Faster than using the folders).

Links:

  1. Index Repacking
  2. Preloading models
  3. Model Compressor

Yes it will as long as you load the preloaded models after the models from the cache.

It’s just like custom items…
You can put the model in the cache (The right way)
or you can preload it.

It really just depends when/where the model is loaded.
If you load the new tree model first, it won’t work because the cache will load the old ugly broken model after.
If you load the old ugly model from the cache first, it will work because the new tree model overwrites the old.
