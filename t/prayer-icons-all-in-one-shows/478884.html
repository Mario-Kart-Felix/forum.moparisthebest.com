zyle1992 August 4, 2016, 2:16am #8

My best idea would be that the “normal” position is setting the prayerIcon to -1, instead of 0.

The prayers are read client-side as a byte, with each bit being a flag. That’s why the “setPrayerIcon” are powers of 2.

Protect from Magic is 4, which in binary is

0000 0100

Redemption is 32, which is

0010 0000

Which brings us to my thoughts. It’s not uncommon for -1 to be the “no value” or “null value”. -1 in binary is

1111 1111

Which would make the client render all of them at once.

Also, you should consider refactoring the code to make it a bit cleaner.
