zyle1992 August 1, 2016, 11:13am #17
lesterknome:

Why make a new class for every Monster? Why not just define all of them in your initial Enum Monsters, Add them to a hashmap. Then use get(room) to return the Monster object then get the rest of the data from there?

I’m not sure if that would even be necessary. I suppose you could just have a Monster class, and maybe just an enum MonsterType, that has like spider, horned fly, etc. Probably something like this:

public final class Monster {
    public enum MonsterType {
        HORNED_FLY("Horned fly");

        private String name_;

        private MonsterType(String name) {
            name_ = name;
        }

        public String getName() {
            return name_;
        }
    }

    private MonsterType type_;

    public Monster(MonsterType type) {
        type_ = type;
    }

    public MonsterType getType() {
        return type_;
    }
}

and then just do

room.addMonster(new Monster(MonsterType.HORNED_FLY));

You could also, as an alternative, create a MonsterDefinition class, so that you could possibly load it from an external source, instead of having to recompile every time.
