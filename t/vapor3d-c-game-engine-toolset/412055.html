Hi! I’m a Computer Science student and wannabe Game Developer from Portugal.

I’ve been developing a 3D engine and game development toolset for some years now and would like to share it with you! It’s called vapor3D. I started because I felt a lack of good open-source 3D game engine (we already have some nice rendering engines, like OGRE).

The end goal is to develop a (indie!) game with with it so I would love to join/assemble a team, have some help with ideas, team up with artists, etc.

I’ve still not released a pre-compiled package because I feel it’s still not ready for prime time, but programmers can get the source and compile it. Also those who want to try can ask me for it.

Source (Subversion): http://svn.assembla.com/svn/vapor_engine/

Website: http://www.vapor3d.org/

[size=12pt]Features[/size]

Core:

[ul][li]Clean C to facilitate bindings to scripting languages[/li]
[li]Cross-Platform Platform Layer (Threading, Timers, Windows)[/li]
[li]C++ Reflection and Serialization (binary and JSON)[/li]
[li]Task-based concurrency system[/li]
[li]HTML-based Logger[/li][/ul]

Rendering:

[ul][li]OpenGL Rendering Backend (Direct3D possible in the future)[/li]
[li]Modern Shader-based design (no OpenGL deprecated features)[/li]
[li]Support for Forward rendering (Deferred might come later)[/li]
[li]Fixed and Shader-based Pipelines[/li]
[li]Object Properties are defined via Materials (like OGRE)[/li][/ul]

Scene:

[ul][li]Component-based Scene and Game Objects (similiar to XNA and Unity)[/li]
[li]Hierarchical View Frustum Culling
[/li][li]Existing Components: Transform, Geometry, Billboard, Camera, Grid, Light, Model, Projector, Dynamic Sky Dome, Listener, Audio, Water.[/li]
[li]Seamless saving and loading to files[/li][/ul]

Resources:

[ul][li]Virtual File System (via ZIP, directories, and HTTP servers)[/li]
[li]Asynchronous and Threaded loading done in the background[/li]
[li]Asynchronoous Resource hot/live updating (just hit save, and it updates in-game/editor)[/li]
[li]Milkshape3D (MS3D) meshes, OGG audio, PNG and JPEG images, Bitmap and TrueType (TTF) fonts, Lua scripts, GLSL shaders.[/li][/ul]

Animation:

[ul][li]Skeletal Animation via Milkshape3D[/li]
[li]Animation Blending (CPU and GPU)[/li]
[li]Attachment of objects to bones[/li][/ul]

Scripting:

[ul][li]Scripts via Lua programming language (used in WoW and other games)
[/li][li]Bindings to Lua via SWIG binding generator[/li]
[li]Possible to use co-routines for easy game logic[/li][/ul]

Terrains:

[ul][li]Height-map based terrains[/li]
[li]Editable in real-time[/li]
[li]Paintable in real-time[/li]
[li]Integrates with paging for seamless worlds[/li][/ul]

Physics:

[ul][li]Bullet-based simulation[/li]
[li]Support for convex-hull generation (soon)[/li]
[li]Triggers for game events[/li][/ul]

Controllers:

[ul][li]First-Person Camera Controller[/li]
[li]Third-Person Camera Controller[/li]
[li]Script Controller[/li]
[li]Kinematic Character Controller (via Physics)[/li][/ul]

Networking:

[ul][li]UDP-based communication[/li]
[li]Remote Object Replication support[/li]
[li]Class fields are serialized in binary with efficient encoding (based on Google Protocol Buffers scheme)[/li]
[li]Plugin-based server with authentication, compression, and user management[/li][/ul]

Editor:

[ul][li]Cross-platform wxWidgets-based GUI[/li]
[li]Plugin-based functionality[/li]
[li]Full Undo/Redo support[/li]
[li]Easy placement of entities with manipulators[/li]
[li]Load scenes in engine with a single call[/li]
[li]Terrain editing support[/li]
[li]Resource browser with automatic resource previews[/li]
[li]Lots of other stuff I can’t be bothered to type[/li][/ul]

I’ve been working lately on the networking support, based it on the concepts that the Source engine by Valve uses. At the moment automatic object replication is fully working, you annotate your objects (using macros, C++ doesn’t have them like Java… :() and the engine does everything for you.

With that I added real-time collaborative editing to the editor, and it is awesome. Still needs some work though, like resource synchronization between editor instances.

I’ve also fixed the saving/loading of scene files in the editor. The JSON serializer had lost some backward compatibility, which I think is really important, so I re-added compatiblity (hacks) so it works with earlier versions of stuff I saved.

Right now, my priority is fixing any design mistakes I have, and stabilize everything. There’s a lot of features that I have built over time, but stopped working for various reasons.
