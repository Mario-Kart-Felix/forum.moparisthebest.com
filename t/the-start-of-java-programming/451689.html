JavaTC August 3, 2016, 7:45pm #6

Look at code snippets, both in the source and on here from tutorials. I personally can’t explain how to observe code, you just do it.

I learned by just looking at code snippets, adding them in my source, and then I got curious to see what something actually did when I added it.

For example, look at this

if (playerCommand.equalsIgnoreCase("slots")){ int hasitem = 25000000; int slot = (Misc.random(100)); int risked = (hasitem * 2); if (!c.getItems().playerHasItem(995, hasitem)) { c.sendMessage("You havent got above "+hasitem+" gp in your inventory."); return; } if (slot == 11 || slot == 22 || slot == 33 || slot == 44 || slot == 55 || slot == 66 || slot == 77 || slot == 88 || slot == 99) { c.sendMessage("You pulled slotmachine and got "+slot+" You won "+(risked/1000000)+"M."); c.getItems().addItem(995, risked); } else { c.sendMessage("You pulled slotmachine and got "+slot+" You lost 25m."); c.getItems().deleteItem(995, c.getItems().getItemSlot(995), hasitem); } }
This is a command that adds a bare-bones ‘slot machine’ though commands.
Knowing that the item code for gold is 995, you can infer that any parts of the code with 995 in it have something to do with adding or removing gold.
