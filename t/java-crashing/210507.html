yakman July 30, 2016, 11:09pm #4

dont call paint(getGraphics());

use repaint() instead

also, dont override paint() for swing components, which includes JFrame
override paintComponent() instead

also, dont draw on the JFrame, but draw on your own JPanel, and put it on the JFrame with setContentPane()
cause you see, you’re drawing on the entire JFrame, but swing supports more then that, with your method, you cant add a menu bar, or other stuff,
so use setContentPane()

JFrame has a JRootPane which contains its Content Pane, its JMenuBar, and its JLayeredPane,
when you override paint() you are removing all that functionality

kind of like this

public class MyImagePanel extends JPanel {

    @Override
    public void paintComponent(Graphics g) {
        //draw stuff
    }
}

/////
/////

JFrame frame = new JFrame("Display Image");
//other JFrame stuff, ie. setDefaultCloseOperation()
frame.setContentPane(new MyImagePanel());
frame.setVisible(); //should always be the last thing
