Clawz_fury August 4, 2016, 11:23am #4

you should be checking/loading global items every time the region changes

when the region changes, if the player is on the z coordinate that the item is supposed to be on, then spawn the item. if not, don’t spawn the item (since the server -> client packet for spawning items doesn’t include the height).

“how can i check if the player is on the proper z coordinate” you should have some sort of list of all the active global items. if not then you’re out of luck and you’re probably gonna end up hardcoding your way out of this

“im not loading the z coordinate” use data encapsulation, you should be loading the x, y, and z coordinates

if you’re doing all of this already then i don’t know what the problem is
