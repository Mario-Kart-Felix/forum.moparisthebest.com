The exact same thing happened to me. Here’s what I found:

Basically, the script goes through the files in the “Equipment” folder looking for .png files and loads any of them that it finds. This error means that the data entry in the Config.ini file for the item directly following (alphabetically by filename in the “Equipment” folder) the last one successfully loaded is messed up or just plain missing. If you look in the “Equipment” folder, right after the graphic file for “Potion of Attack” (which is “AttackPotion.png”) you’ll see the graphic file named “BowofFeyMagic.png” and this item has no “Autograb” entry. I went through and put an “Autograb=1” after every item entry that didn’t have it and ran the script again.

This time it went farther but stopped at “Bow of Innocent Blood”. The file directly after the graphic file for that entry is “CloakofGreaterSpeed.PNG” which is NOT listed in the Config.ini file at all.
