the simba program uses a hard coded number on where to search for items, and where to drop items.
For example, my screen resolution is 1280x720.
It initially searched for in the area (610,500) x (785,595). In other words, it starts scanning for items at the pixel x = 610,y = 500 and ends at x = 785, y = 595.
But my monitor resolution is different. On my monitor, the flash program is located in the center of the screen. In order for it to scan correctly, it needs to be (810,600) x (1000,719). So I changed the ScanItem code.

if FindBitmapToleranceIn(item.Image,x,y,610,500,785,595,25) then

to (810,600) and (100,719)

if FindBitmapToleranceIn(item.Image, x, y, 810, 600, 1000, 719, 25) then

Now Simba was able to find items properly, but it wasn’t able to move it to my inventory. Since your inventory is about 100 pixels higher, I changed the GrabItem procedure code to the values (800,500) x (1000,630).

if FindBitmapToleranceIn(SlotList.List[i].Image,a,b,800,500,1000,630,25) then

Finally, one other factor. You can’t play on fullscreen (i.e. you can’t press F11 to fullscreen the browser). Fullscreen = will not work.

You’re getting a compiling error because you aren’t loading the includes correctly. Didn’t think this would be necessary, but I guess it is.

Step 1: Download the attachment from the original post
Step 2: Extract the files. You should see 2 folders, one called “Scripts”, the other “Includes”.
Step 3: Open the extracted “Includes” folder. You will see a folder called “ROTMG”. Cut and Paste this into your Simba “Includes” folder.
Step 4: Open the extracted “Scripts” folder. You will see a file called “ROTMGAutoGrabber.simba”. Cut and Paste this into your Simba “Scripts” folder.
Step 5: Now open up Simba, go to File -> Open, and open up ROTMGAutoGrabber.simba. Select the game window using the crosshairs button, and press Run.

There you go, how to run ROTMGAutoGrabber for idiots. Keep in mind that it might still have issues, it sounds like the game sprites have been changed since this script was released.
