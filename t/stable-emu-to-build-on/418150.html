cleako August 1, 2016, 3:34pm #7

Things here and there. The e-mail alerts system is in place but only has two test events that will trigger it. The Moparscape forum authentication looks done but I have not yet figured out how to test it with a SMF installation. There is a lot of room for plugins to be added. DarkQuest has a great deal in terms of listeners (action and executive), minigames, quests, and skills. I am not yet sure if they are compatible or at least similar yet. The npchandler folder contains duplicates of many of the NPC BSH scripts from looking at it, maybe they interact with them, I don’t know yet. Those could be either completely BSH or made into NPC plugins that interact with the listeners as they are in DarkQuest. The skills plugin folder contains the mining skill but there are several more that can be added. DarkQuest has 9 plugin skills total. MoparRSC still utilizes XML for the game server NPC/Item defs and locs, objects, doors, and shops. DarkQuest moved all of that into MySQL and being able to manipulate the data with a website has utility.

Overall the additions that you contributed to MoparRSC are spot on but the RSCA source feels a whole lot more rough compared to the changes Peeter made to it with DarkQuest. If you were to take DarkQuest (pre-revision 70’s points system) and made the same changes… rawr.
