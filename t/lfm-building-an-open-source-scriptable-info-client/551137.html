robhol August 4, 2016, 3:03pm #1

Basically, I’ve found out that the OSBuddy people aren’t planning on implementing scripting anytime soon. As a result I’ve been wanting to roll my own client, but having no knowledge on (Java-specific) reflection and the internal workings of Runescape, I’ve kind of lost hope that this is something I can do by myself, particularly given the general difficulty of finding up-to-date information. I would be able to learn, and the “mechanics” don’t seem too complicated, but it would take me too long to get the Runescape-specific know-how on my own.

I basically want to create - initially - a bare-bones Runescape “container”. From there, I’d like to build and gradually expand support for reading and interpreting game state, and start designing an API. I’ve looked at a few bots, most of which use more Java for their “scripting” needs. I’d prefer to operate with a different language, though I’m not quite sure which one.

The project would be open-source, not only because it makes it easier to extend and improve, but to serve as an example - based on my own googling, it definitely couldn’t hurt.

Does this seem interesting to anyone?
