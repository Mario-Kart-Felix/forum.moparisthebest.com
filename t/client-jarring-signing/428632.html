Yes, while signing the jar allows the SecurityManager to do more, it also allows you more flexibility. Such as referencing the cache location via

System.getProperty("user.home") + "/.file_store_32/";

The “user.home” property is restriced in unsigned applets, but without signing it, you have to manually set the cache location based on the operating system.
