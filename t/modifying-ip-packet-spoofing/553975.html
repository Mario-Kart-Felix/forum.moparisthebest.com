Yes. Happens all the time - many networks (your ISP for example) however won’t let the packet out of their edge, or their upstream won’t, if the IP isn’t one they should be originating traffic for (see ‘BCP38’ for more information).

I was also wondering if there's some sort of pattern that people use to confirm if the IP is real (like regex)? Or can I send a string as my IP so it would display as "hello world" or something?

No. It’s a 32 bit integer, see sini’s link.
