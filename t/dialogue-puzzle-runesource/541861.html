This is going with the npc cases that is above the actual switch that then converts to case 1-whatever # you have.

Let me give you some info about using cases.

Example 1:
[spoiler]
This will make all cases 1-4 (or how many) use the functions in case 4…

case 1: case 2: case 3: case 4: //functions here break;[/spoiler]

Example 2:
[spoiler]whereas, this portion of code will have its own thing for each case.

case 1: //functions break; case 2: //functions break; case 2: //functions break; case and so on: //functions break;[/spoiler]

What are breaks and why would it matter where I put it?
Take the bottom code as an example:

[spoiler]

[code=java]case 1:
system.out.println(“case 1”);
break;
case 2:
system.out.println(“case 2”); //because the ‘break’ is in case 1, it won’t execute this case and print “case 2”.
break;

case 3:
case 4:
case 5://so for this, case 3, 4 and 5 will print this because ‘break’ is not in case 3 nor 4. If it was then only case 5 will print this code.
system.out.println(“So now every case has its own function and when it executes that function it ‘breaks’ from the case.”);
break;[/code][/spoiler]

If this does not answer your problem, well then you may have learned something about using cases. Idk
