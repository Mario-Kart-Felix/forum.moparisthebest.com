Most of tutorials for webclient focus on windows platform, bascially there is no much difference for linux, because java is cross-platform. Here I only list somethings may be missed in this forum.

  1. Follow Ric’s webclient instrucation to make all changes on client.java, signlink.java… so on. I suggest import all original web client into netbean or eclipse before changing.

  2. search through webclient project, assume your project name is webclient, for “./”, replace it to signlink.getCacheDir(), because “./” means application try to find dependence on your current directory instead of your cache folder.

  3. Don’t forget signer your jar file, in Linux, no need to download any jar and signer tools, because they are supposedly build-in command. Assume you already FTp your webclient.jar into server, forexample: /home/rune/webclient/webclient.jar.

cd /home/rune/webclient keytool -genkey -alias webclientKey jarsigner webclient.jar webclientKey

  1. Open java console to trace the log, d better press “5” to see maximum details. Insert debugging code in suspicious place with method13(), we can’t expect success at the first time try. we have to debug, compile, ftp, polish up our application.

My successful case is located at http://gangliu.ca/index.php/runescape, welcome to test it.
