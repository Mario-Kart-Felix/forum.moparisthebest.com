I personally use Java’s Calendar, however System.currentTimeMillis(); should work just fine for what you’re doing

So what you’re going to want to do, since they can only do the command every 2 hours, is when they type the command to check the current time, and get the players last time received (like the last time they claimed their reward) and make sure there is at least 2 hours worth of time in between.

Note: The following code was not tested on a server. However, with very minimal modification it should work perfectly.

[code=java]

public class TimeModificationTest
{
public static void main(String [] args)
{
/*
* I stored System.currentTimeMillis(); in the long currentTime in case it happen to switch right over (it’s super rare, but possible)
* as it goes through the three ints; seconds, minutes, hours.
* From there I put all the ints (your time) into a string that is easier to read than just 3 numbers.
* I also created the method addTime( int timeToAdd, String type ) so you can see what the System.currentTimeMillis(); would look like
* with 2 hours added.
/
long currentTime = System.currentTimeMillis();
int seconds = (int) (currentTime / 1000) % 60 ;
int minutes = (int) ((currentTime / (100060)) % 60);
int hours = (int) ((currentTime / (10006060)) % 24);
String formattedTime = "Hours: " + hours + " Minutes: " + minutes + " Seconds: " + seconds;
System.out.println("Time: " + formattedTime);
addTime(2, “hour”);
}

public static void addTime( int timeToAdd, String type)
{
	long currentTime = System.currentTimeMillis();
	
	/* 
	 * If you didn't know, you can use a switch statement on strings.
	 * I added the other variables, second and minute, just in case you had some use for this method.
	 */
	switch( type ) {
	
		case "second":
			currentTime += (timeToAdd * 1000);
			break;
		
		case "minute":
			currentTime += (timeToAdd * 1000) * 60;
			break;
			
		case "hour":
			currentTime += ( (timeToAdd * 1000) * 60 ) * 60;
			break;
		default:
			System.out.println("Unknown time format.");
			break;
	}
	
	/* Reusing the code from above, because if it ain't broke, don't fix it. */
	int seconds = (int) (currentTime / 1000) % 60 ;
	int minutes = (int) ((currentTime / (1000*60)) % 60);
	int hours   = (int) ((currentTime / (1000*60*60)) % 24);
	String formattedTime = "Hours: " + hours + " Minutes: " + minutes + " Seconds: " + seconds;
	System.out.println("Time: " + formattedTime);
}

}[/code]

The above code works perfectly (I believe).
It’s just an example on how to modify and retrieve the time in a “readable” format instead of a long like:1349333576093
So if I were you, I’d store the readable format, or you could just store it in the long form and access it that way.
It’s up to you, what you do next. But if it were me this is what I’d do:

  1. Create a method to read the time, and have it return a string. Something like public String methodName(long timeToRead) would be perfect.
  2. Create the variable you’d like to save. And make sure to save it.
  3. Create your command, make it check the current time, and the time from last confirmation. Compare them, make sure its at least 2 hours away, and if so reward them and then make sure to update the players last check to the current time.
