Most of the code that gives you skill xp is written in individual spots, i’m not really an expert and i’m not sure if thats clear enough. But i know that when i did this on my source i had to change many different numbers individually. I don’t know if it differs on your source but here’s some examples of things i changed on mine to fix this.

Prayer

if(UsedOnObjectID == 10638){ if(ItemID == 526){ deleteItem(526, 1); addSkillXP(50*getLevelForXP(playerXP[5]), 5); //<------Change the "50" sM("You receive some prayer experience."); setAnimation(894); lowGFX(247,0); }

When you use bones on an alter a number of xp is multiplied by your prayer level, determining how much xp you receive. Change that number for each of the different types of bones. Also edit the amount of xp given when they are buried, probably in potions.java

Herblore

if ((itemUsed == 267) && (useWith == 227) || (itemUsed == 227) && (useWith == 267)) { if (playerLevel[15] >= 20) { deleteItem(267, getItemSlot(267), 1); deleteItem(227, getItemSlot(227), 1); addItem(2444, 1); addSkillXP(1500, 15); //<----Change here setAnimation(1652); } else { sM("You need 20 herblore to make this potion."); } }

The same thing goes for herblore and many of the other skills like crafting, runecrafting, farming, mining, smithing, fishing, firemaking, etc…

My point is i’m pretty sure there’s not just some kind of rate you can change like you can with the combat skills, i think you need to change the numbers individually. There may be a much easier way but i know this works even if it takes time.
