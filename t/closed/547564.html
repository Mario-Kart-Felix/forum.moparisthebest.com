sk8rdude461 August 4, 2016, 1:56pm #11

If you delete the method, you will get an error.
The error will be because the method is called somewhere during startup (hence how it makes the prices.txt).
You can remove that too.

The reason this prices.txt is generated, is because in the cache there are item definitions.
These item definitions contain data like wield anims, colors, names, descriptions, All the stuff you use when making custom items in ItemDefs.
Also in the cache with these item definitions, are the item prices.
Yes. The item prices are in the cache files.
And so someones solution to it was to just dump the prices from the cache into a text file, and use it server sided.

So yes, you can remove it if you don’t want it. It won’t harm anything.
