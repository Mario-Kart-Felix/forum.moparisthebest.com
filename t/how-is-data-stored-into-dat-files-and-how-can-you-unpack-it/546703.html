matzie August 4, 2016, 1:50pm #6

Just like at the lowest level there only needs to exist a format to read and write the data.

At the binary level, bit patterns mean different operations. While programming, you must only concern yourself with the order at which you read/write the values required to represent the 3D object.
It is always nice to use byte-size pieces of data (int = 4bytes) you just write the values in the same order you read them. Hope this makes sense.
example:
vertX:double
vertY:double
vertZ:double
pathPoints:int
pathX:double[]
pathY:double[]
pathZ:double[]
color:Color

if you were to write vertX to the file first, the you best read vertX first as well, because it is the first in the file.

You are in control of the complexity of the 3D object, and the scheme in which you save it.
Smart ass answer: in binary

To do it yourself you just need to create an instance of a FileWriter object, and you may begin writing bytes to it. It is typically best to use a wrapper class that can handle writing Integers, doubles and other things to the file as well for convenience (like the Stream class in clients/servers)

This is an example of packing the data from the listed files (filesToPack) to the file to pack to (packToFile). You can use things like RSA or DSA for encryption or key authorization, but this is just a small example of packing multiple files into one.

final File packToFile = new File("./Cache.dat");
try (FileOutputStream output = new FileOutputStream(packToFile)) {
     byte[] buf = new byte[1024];
     File[] filesToPack = new File("./cacheFiles").listFiles();
     for (File file : filesToPack) {
          try (FileInputStream input = new FileInputStream(file)) {
               while (in.read(buf) > 0)
                    output.write(buf);
          }
     }
}

Now, I would use a ByteArrayInputStream to parse the packed data.
