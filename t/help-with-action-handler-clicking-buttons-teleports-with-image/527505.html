Why do people still do this shit:
c.getDH().sendOption5 (“monster1”, “monster2”, “monster3”, “monster4”, “monster5”);

The point of the dialogue handler is to handle dialogues
Thus why those statements had private access in that class.

In the dialogue handler, you want to add something like this into your switch statement (make sure the case ID isn’t being used).
(Also make sure to remove existing code and replace with mine…

case 666:
sendOption5 ("Rock Crabs", "Cyclopes", "Frost Dragons", "Brimhaven Dungeon", "More..");
c.teleAction = 1001;
c.nextChat = 0;
break;
case 667:
sendOption5 ("Monster", "Monster", "Monster", "Monster", "Monster");
c.teleAction = 1002;
c.nextChat = 0;
break;

And in action handler:

case 13615: c.getDH().sendDialogues (666, 0); break;
Then in your clicking button for the 5th option of the dialogue:

if(c.teleAction == 1001)
c.getDH().sendDialogues(667, 0);
