justaguy August 3, 2016, 3:01pm #15

Seperate a website into two components: a font-end and a back-end.

The front-end is the parts of a website you see infront of you, such as the actual template and dynamic features. This goes under HTML/XHTML/HTML5/etc…, CSS, and Javascript. The HTML is the page markup, this basically sets up the DOM, which can be manipulated by Javascript using your browser’s DOM API. With this capability, Javascript can be used to add extra, dynamic content to a seemingly dull page. CSS is basically the style to an HTML element (such as an anchor, image, and so on) you can apply to make your front-end look more appealing, and so everything fits together seamlessly on the page.

The back-end is what goes on server-side or “internally” to get a webpage running. Although it isn’t actually required to have a viewable webpage, you can use a language such as PHP, ASP, or web application frameworks like Ruby on Rails or Django as server-side languages and frameworks. The back-end process webpages before it is sent to a user. To more complete the list, commonly SQL is a language used in the back-end to query with a database, but this isn’t required in all back-ends.
