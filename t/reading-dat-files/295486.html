@xeez`: Maybe you should take a look into the signlink class. This is a nifty class JAGeX use that is used in both the loader and the client. The loader‚Äôs version is run in a privileged environment, and the client‚Äôs is unfortunately not. What the loader does is load the client, and pass its signlink to the client. The client then uses this signlink. But whats that do? Simple, the signlink from the loader is still privileged, so it lets the client request privileged actions through it. Signlink is just a service that links the client‚Äôs unsigned environment with the loader‚Äôs signed environment, which should be pretty easily deduced from the name. Anyway, back on subject, the signlink class has a method with a local String[] variable that contains a list of possible cache home directories. Visit each and find your very own, and if you‚Äôve ever played runescape, there should be a random.dat inside the folder (the cache folder is named after the game that uses it, e.g. runescape, arcanists, etc and is a child of a folder following the format of .jagex_store_<store#>, or .file_store_<store#>; this folder is in turn a child of the cache home :p). This contains your client session UID, which is sent during login.
