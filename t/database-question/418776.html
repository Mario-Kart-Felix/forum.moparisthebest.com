What would be the best way to store a colour value in a MySQL database? Should I convert the hexadecimal code into r,g,b values and store them in a column each or should I store the hex code directly in a char(6) ? Or is there another way?

CodeForFame August 1, 2016, 3:42pm #2

What are you using the values for?

If it’s just to persist them, and you’re not doing anything with them then I’d say use a numerical datatype, not char.

If you’re going to be making queries that depend on particular values you should probably have different columns for their red, green and blue values (again using a numerical data type).

I really don’t see why you would ever use char.

Also, take this with a grain of salt, because optimizing queries and such isn’t my thing.
