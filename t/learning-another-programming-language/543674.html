lothy August 4, 2016, 1:13pm #4

Java and C++ are in many ways very different. I think it’s quite wrong to give people the impression that they’re overly similar.
Syntactically, somewhat.
But there’s a great deal of extra responsibility that the programmer needs to exercise when working in C++. For example, resource management in C++ requires special attention. You don’t have environment features, such as the JVM garbage cleaner, doing the dirty work for you.

If you know one programming language well and you have some knowledge about control flow in general (for computers using the von Neumann architecture), then learning a new programming language is a breeze. It’s practically just syntactic sugar. Yz85Racer August 4, 2016, 1:14pm #19

Truthfully, you’d be better off learning a language like C. But more importantly, you’d be better off learning the concepts behind programming, rather than focusing entirely on syntax. Once you’ve got a general grasp on certain concepts, it just becomes a matter of applying and remembering the syntax for different languages.

Obviously this isn’t ENTIRELY all there is too programming, given how different languages handle things like OOP, resource management, etc; so it’s a bit of general advice. Don’t focus too hard on particular language’s syntax, as opposed to the concept behind it.
