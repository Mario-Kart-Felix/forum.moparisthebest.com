DFM Form Parser by Ron - Fix Up Your Forms! Works with Divi!

Screenshot:

Description:
When you make a form using the form editor in scar and save the file. You create a DFM file. This DFM file can be translated into SCAR code using the SCAR’s “load form” option, but compared to this, it’s obsolete. Check out the features!

If you still don’t find this useful, I can totally clear it up for you. This script in essence can make any doosh bag make a perfect form! Here are the steps to use this.

  • First, go into scar’s Form Editor. Tools > Form Editor.
  • Now just make a totally random form. Throw components on there like crazy and do whatever.
  • Once you’re done with the form editor, click the Save Icon and save the file somewhere like in My Documents or something.
  • After that is saved, open up my script, and type in the exact location of the file. Check some options if you like.
  • Click on Generate Code. There should be code now in the memo box. Now either, save the file to a .scar file using that button, or click on Copy to Clipboard and paste the code into a new scar window.
  • Finally, when the newly generated code is in the scar window, Run the script, and viola! You have just made a PERFECT FORM!

Features:

  • Automatically create the entire form’s code with safe statements!
  • FreeForm() to dissolve any memory leaks.
  • Change the form name in the DFM form easily in case you made a mistake.
  • Able to make your form popup in the center of your screen.
  • For those advanced users, you can see how long it takes for your form to compile.
  • To improve performance and decrease the size of your scar file, you can convert the statements into a WITH Statement!
  • If you don’t like how the SCAR window stays in the background when your form pops up, no worries, you can set it to generate the code to minimize it and then display your form!
  • Able to copy code to clipboard and paste it somewhere else.
  • Resize window to better see the code.
  • Saves all settings including height and width of form.
  • Disable resize.
  • Disable maximize/minimize.
  • Disable system menu.
  • Force the form to stay on top of all other windows.
  • Able to see only safe procedures and vars code.
  • Try/Except error checking option.
  • Able to see how many lines of code and how many characters total.
  • ReadOnly ability.
  • Variables shown as Label1, Label2, Label3 : TLabel; and the like.
  • Replace width and height with clientwidth and clientheight.
  • Browse button for easy file location.
  • Save the code to a .scar file.
  • Break down long variable lists of the same type.
  • StatusBar to show you detailed information about the generated code.
