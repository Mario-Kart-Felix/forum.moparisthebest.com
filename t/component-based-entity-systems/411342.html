A typical entity system in a modern game uses complex object-oriented inheritance to compose behavior for entities. Usually this entails using public-import multiple inheritance in a way that almost looks appears as a has-a relationship.

A rift, however, in such designs has emerged in recent years with a new /data-driven, aspect-oriented/ design pattern in which entities are composed by components which describe types of behaviors an entity possesses.
