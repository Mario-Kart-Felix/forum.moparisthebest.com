https://www.paypal.com/cgi-bin/webscr?cmd=p/xcl/rec/ipn-intro-outside&

IPN.

The basic idea is that paypal will make a POST request to your server with a hidden form containing any variables you wish (the customers name/address, a unique id you may wish to assign them, the transaction id, etc). You will need a script that catches this POST, makes any nessecary security checks and flags the customer as paid in your database. Then, the link page simply checks if they’re allowed it or not (and ideally, generates them a one-use unique link so they can’t share it with others).

This is not difficult if you know PHP or another programming languages, however if not I recommend you look for a commercial solution (won’t be too expensive). Attempting to hack it together yourself if you don’t know a programming language would probably result in something that works but is easy as hell to exploit.
