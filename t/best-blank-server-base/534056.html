Compiled a list while reading through the original and a more recent one at the same time…

  1. Makes no use of the attachment function of the Selector, instead creates a Map?

  2. Lack of documentation throughout the entire server.

  3. All of it’s update flags are stored as booleans.

  4. It’s PacketManager/Client class? Dear lord…

  5. Threads should be executed with the ScheduledExecutorService to avoid manually fixing them.

  6. Poor design flow on the entire entity system.

  7. Parses the cycle rate from the command line? Should be a fixed constant.

  8. It’s buffer allocations are way off.

  9. Nested both StreamBuffer implementations in the same class as the super, really weird.

  10. Misc class, self explainitory.

  11. No packet validation.

  12. Not user friendly stream implementation.

  13. Cheap haxed way for finishing a packet read.

  14. Handling packets is very dogdy.

  15. Inbound packet structure is garbage.

  16. Global state.

  17. Dodgy NIO implementation as a whole.

  18. Position’s data encapsulation sucks.

  19. WinterLove style updating.

  20. Horrific design, looks very thrown together.

  21. Player specific movement handler.

  22. Equipment slots…? (Look at Misc for info).

  23. HostGateway should be using an AtomicInteger to ensure thread safety.
