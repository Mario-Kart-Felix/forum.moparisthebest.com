byte July 30, 2016, 5:27am #4

A hook, simply said, is an event handler. Hooks are implemented through callback functions. For a bot’s purpose we are interested in two hook types, on field mutation and on function entry/exit. These are called application hooks, system hooks also exist where by events such as mouse move, key type, new message (win32) are handled/filtered before processing.

In a bot’s scope, a field mutation and function entry hook could be implemented as such:

//bot code
class Hooks {
 private int i;
 public static void notifyIMutation(int i) {
  //filter if necessary
  this.i = i;
 }
 public static void notifyYEntry() {
  //handle
 }
}

//client code
class client {
 private int i;
 public void y(int a) {
  Hooks.notifyYEntry();
  i = a * 2;
  Hooks.notifyIMutation(i);
}

The obvious advantage is speed. In this case all we are doing is calling our notify function instead of going through the reflection rountines simply to retrieve the data of a newly mutated field.
