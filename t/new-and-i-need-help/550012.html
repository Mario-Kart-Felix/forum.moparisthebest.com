This is simple, OP…

Your issue is because you’re trying to connect to yourself… And you can’t.

When you are on the same computer as the server - Set the client IP to “localhost” or “127.0.0.1”. They do the same thing.
When you are on the same network as the server - Set the client IP to the local IP address (The one you used to portforward in your router)
When you are on a different network as the server - Set the client IP to either your No-IP (Assuming the No-IP DUC is running) or your IP address (Found at http://whatismyip.com)

No-IP is “old technology” and hardly needs to be used. The reason people used it in the past was because their IP addresses were dynamic, so if they used their normal address it would eventually change, thus rendering the client useless. However, in this day and age, hardly anyone has a dynamic IP (That doesn’t mean there aren’t people who do), so it’s not needed.
