interface ISorter { int[] Sort(int array[]); }

[size=17pt]Bubble Sort[/size]

Description: Bubble Sort is a very simple Algorithm, It basically searches through a list until it finds something out of place, It then swaps it with the next in line. And it continues to do so, until a loop through the list is completed without a swap. This runs well when a list is Nearly sorted. But worst when a list is Random. Its worst-case is O(n^2).

BubbleSort.java

[code]
public class BubbleSort implements ISorter
{
public int[]
Sort (int[] array)
{
boolean sorted = false;

  while (!sorted)
  {
     sorted = true;

     for(int i = 0; i < array.length -1; i++)
     {
        if(array[i] > array[i+1])
        {
           int temp = array[i+1];
           array[i+1] = array[i];
           array[i] = temp;

           sorted = false;
        }
     }
  }

  return array;
}

}[/code]

[size=17pt]Insertion Sort[/size]

Description: Insertion Sort Iterates through a list, and removes an element from that list, then puts it into an already sorted list. Its worst case is O(n^2).

InsertionSort.java

[code] public class InsertionSort implements ISorter
{
public int[]
Sort(int[] array)
{
int select = 0;
int temp = 0;

  for (int i = 0; i < array.length; i++)
  {
     for (select = 0; select < i && array[i] > array[select]; select++);
     
     temp = array[i];
     for(int s = i; s > select; s--)
     {
        array[s] = array[s-1];
     }
     array[select] = temp;
  }
  return array;

}
}[/code]

[size=17pt]Selection Sort[/size]

Description: Selection Sort Iterates through a list, and finds the smallest value, It then swaps the smallest value with the starting position. It also uses the idea of Partitioning. Its worst case is O(n^2).

SelectionSort.java

[code] public class SelectionSort implements ISorter
{

public int[]
Sort(int[] array)
{
int select = 0;
int temp = 0;

  for(int i = 0; i < array.length-1; i++)
  {
     select = i;
     for(int c = i+1; c < array.length; c++)
     {
        if(array[select] > array[c])
           select = c;
     }
     temp = array[select];
     array[select] = array[i];
     array[i] = temp;
  }
  return array;

}

} [/code]

[size=17pt]Quick Sort[/size]

Description: Quick Sort picks some value in a list, and uses it as a pivot. It then re-organizes the list so that anything less than that pivot is on the left, anything larger than it is on the right. It then Recursively calls Itself until the list is sorted. Its worst case is O(n^2), but on average its O(nlogn). Results vary depending on the Pivot chosen, and how you chose the pivot.

QuickSort.java

[code] public class QuickSort implements ISorter
{
public int[]
Sort (int[] array)
{
return QuickSorter(array, 0, array.length-1);
}

private int[]
QuickSorter (int[] array, int low, int high)
{
if(high <= low)
return array;

  int rHigh = high;
  int pivit = high;
  high = high-1;
  int temp = 0;
  int rLow = low;
  
  while(low <= high)
  {
  
     while (low < rHigh && array[low] <= array[pivit])
        low++;
        
     while (high >= rLow && array[high] > array[pivit])
        high--;
     
     if(low < high)
     {
        temp = array[high];
        array[high] = array[low];
        array[low] = temp;
     }
  }
  temp = array[pivit];
  array[pivit] = array[low];
  array[low] = temp;
  pivit = low;
  QuickSorter(array, rLow, pivit-1);
  QuickSorter(array, pivit+1, rHigh);
  return array;

}
}[/code]

[size=17pt]Merge Sort[/size]

Description: Merge Sort, this is my favorite sorting algorithm. It goes like so, Check if list length is 1, if so, its sorted. If its not, then divide the list into 2 lists, Then recursively call Merge Sort from there on each list. After that, actually MERGE the lists together. Its worst case is O(nlogn).

MergeSort.java

[code] public class MergeSort implements ISorter
{
public int[]
Sort (int[] array)
{
return MergeSorter(array);
}

private int[]
MergeSorter (int[] array)
{
if(array.length == 1)
return array;

  int[] left = new int[array.length / 2];
  
  for(int i = 0; i < (array.length / 2); i++)
     left[i] = array[i];
  
  int[] right = new int[array.length - left.length];
     
  int it = 0;
  for(int i = (array.length / 2); i < array.length; i++)
     right[it++] = array[i];
  
  array = Merge(MergeSorter(left), MergeSorter(right));
  
  return array;

}

public int[]
Merge (int[] left, int[] right)
{
int len = left.length + right.length;
int[] toReturn = new int[len];
int leftIter = 0;
int rightIter = 0;

  for(int i = 0; i < toReturn.length; i++)
  {
     if(leftIter < left.length && rightIter < right.length)
     {
        if(left[leftIter] < right[rightIter])
           toReturn[i] = left[leftIter++];
        else
           toReturn[i] = right[rightIter++];
     }
     else if(leftIter < left.length)
     {
        toReturn[i] = left[leftIter++];
     }
     else
     {
        toReturn[i] = right[rightIter++];
     }
  }
  return toReturn;

}
}[/code]

In a bytecode-compiled system, source code is translated to an intermediate representation known as bytecode. Bytecode is not the machine code for any particular computer, and may be portable among computer architectures. The bytecode may then be interpreted, or run, on a virtual machine.
