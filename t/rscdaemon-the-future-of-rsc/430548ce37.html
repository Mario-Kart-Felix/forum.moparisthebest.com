@Peeter

There are multiple server code bases being kept in-sync with each other (C++11 / Java7) mainly because I am curious to benchmark them against eachother and because it is somewhat trivial to obtain the same behavior in each code base. Each facilitates what you are describing in their standard libraries (as I am sure that you were aware). If further flexibility is required, then a custom system would have to be either extended from the existing facilities, or otherwise drafted up. If the standard implementations prove to be a bottleneck, then a more sophisticated approach is required, although I do not foresee that to be the case. Remember that premature optimization is the root of all evil.

When I first started thinking about this project, closures were still on the menu for Java 7, so I had decided to remain with a single thread per world instance. That is of course, subject to change at my discretion, so the direct answer to your question is that as of now a very simplistic event model is used, but may change at any time, for any reason.

I had hoped that this project would have been under the radar for at least an additional 5 to 6 months since the time its existence surfaced so it would have much more substantial progress to show and because I would have more concrete answers to the questions that are being raised.
