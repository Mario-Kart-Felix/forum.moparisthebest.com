I thought I’d make a thread to show progress on our latest project.

RSCDaemon is an event driven new server base for RSC private servers. The client has been partially rewritten in a modern, object oriented approach.

The server core is pretty much done. It’s now just a case of writing all of the plug-ins. All the listeners are done too except a handful but they will be perfected as the content is written.

A new server from Staminus has been reserved, it’s their most expensive box available with the highest scale (20gB/s) DDoS protection available too. The box is being setup, secured and maintained by an ex work colleague of mine with over 20 years of experience in administrating boxes (mainly for Fujitsu).

The website is coming along well and should be able to be put online soon to start inviting members. It will be invite only like Reines’ RSCD as for a while it will be in BETA. We want 1 account per person for now.

The website is HTML5/CSS3 compliant and will only work in modern web browsers. If an old web browser is detected it simply asks them up update (I can’t be bothered to include little bodge-job repairs to fix IE7 and stuff).

Java will still launch the client so the end user shouldn’t notice anything different. The C++ client plays as the Java client does with the bonus of it being a lot harder to play around with. This should really cause a headache for bot makers.

Anyway, there’s still tons more to do before it will be considered playable but I thought I’d post up a teaser.

If anyone wants to contact me the IRC details are in my signature.

C++11 support is bag because when the gc flushes out the bytes from the byte buffer stream it creates a lock on the thread and you get synchronization issues. you could create a wrapper around the string and allow it to flush without all the commas but then it would just act like a variable and the compiler would not know which byte code to deob.

Best way in my opinion is to just compile the runtime processor on the fly, instead of having to wait for the event to call the pull() method. also benefits by doing this is the push() uses too much comments that are not filtered for C++11 but more java and python. since java was made for runescape only it is not optimised for bots you see. but c++ was made for high end programmers to create bots for games like world of warcraft so it should come with some hacking abilities embedded into it.
