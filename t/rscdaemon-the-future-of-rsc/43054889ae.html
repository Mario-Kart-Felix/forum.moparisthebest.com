There’s a reason why the path-generation is taken care of by the client. It would cause major load on the server-side, should it handle all players path-generation requests. It would be easy to crash the server if you sent new requests over and over to new random locations. So, the server should not generate a path. It should just validate the path generated by the client. You can fake it all you want client-side, but the server should just reject the request by stopping the player if it found out.
