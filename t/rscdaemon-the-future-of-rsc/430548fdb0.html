Zilent August 2, 2016, 4:01pm #67
The Enforcer:

How much of the client is going to be written in native code? If it’s over 50%, why not go all the way?

Time constraints. The only parts that I’m rewriting, in order of importance, are parts that:

A) Need to be changed to accommodate for an event driven server
B) Are useful for maintaining the client (OO menu system with future XML support)
C) Are useful to bot-makers. Let’s face it. This client isn’t for script kiddies.

That, unfortunately encompasses the whole entity mangement package, the main client classes (GameWindowMiddleMan? / mudclient / Applet / Frame / menu / audio (deprecation) / parts of the terrain management / part of the camera as well as others that have been modified and/or removed.

One last note, projectiles have been synchronized so damage appears at the correct time. Projectile speed can also be adjusted on the server-side. Projectiles can be now also be fired from NPCs.

std::shared_ptr<Timer> RSCDaemon::GameState::addProjectile(std::shared_ptr<Mob> origin, std::shared_ptr<Mob> target, ProjectileType type, unsigned int speed = DEFAULT_SPEED);
