If you took 5 seconds to read what I said, I’m pretty sure anyone with a brain could understand what I meant.

Here’s a basic timer used with the walking method to demonstrate how they work.

public int disablemovement;

This needs to be given a value somewhere.

if (disablemovement > 0) { //This should be under process
disablemovement--;
}

Why process? I believe it’s called every 500ms, so it will constantly check to see if this variable is above 0. If it is, it will reduce it until it’s at 0.

Now find the walking method

		if(c.disablemovement > 0 && packetType != 98) {
                        c.sendMessage("You're movement is disabled");
			c.playerIndex = 0;
			return;
		}

Checks if the value is above 0, if it is, you can’t walk until the value reaches 0 again. It’s that simple.

So all you need to do is change that to work with your cooking method. When it starts to cook, set a small timer before it moves on to the next item. Use if statements to check whether the timer has reached 0 or not.
