So if you figure this out, there will no longer be a need to completely annoy your players when making either big or small updates while your server is up, and its also instant which can really help with working on what needs to be done instead of waiting and rerunning. IF your unlucky and the status checkers check your server while making an update(hence being offline to rerun), you can lose uptime. With this, you are always online and making updates!

Im sure many of you have wondered how to do such a thing, and the best part is while you follow these directions carefully, you will make it through. Its not hard infact, its quite simple.

The key to this tutorial is a simple tool named J(ava) Rebel. This nifty tool is he main base for what we are trying to achieve throughout this tutorial, it does all the behind the scenes work while I teach you how to implement it into your source. This will require simple editting of an XML file, so dont get scared if it looks too different, because its not. To download jRebel, proceed here,

zeroturnaround.com

Download JRebel | Faster Java Application Development | Save Time

Download JRebel and start skipping those annoying Java redeploys | Save hours of development time every week | No more redeploys, ever!

After downloading it run the installer and remember were it installed. After installing an application will popup, this is used to help you configure jRebel. If you may, you can look over the entire thing and try to figure it out, and if you cant dont worry because I will get right to the point (note I didnt have an application like that to help, I downloaded the JAR).

Lets begin, reviewing the install path mine was,

C:\Program Files (x86)\ZeroTurnaround\JRebel\jrebel.jar

On my computer and probably many others, newly installed applications go into the Program Files (x86) folder, however yours might be different.

Now we are actually going to do something. Go into the source you wish to use jRebel with, and create a new XML file named rebel.xml. Next off open up the new XML file and place the following inside:

[CODE]<?xml version="1.0" encoding="ISO-8859-1"?>

[/CODE]

This is where you need to do some editting. Where it says PATHTOSOURCE is obviously where you need to place the complete path to your server source. Heres an example:

[CODE]<?xml version="1.0" encoding="ISO-8859-1"?>

[/CODE] Easy huh.

The, (or similar)

java -cp . -Xmx500m Server

in your run batch file gets replaced by,

java -noverify -javaagent:"C:\Program Files\ZeroTurnaround\JRebel" server

But wait! Dont get too confident. Remember when I asked you to remember the path to where jRebel installed to? Well this is where you need to recall where it installed to. You need to also provide the complete path to the jrebel.jar file. Like so,

C:\Program Files (x86)\ZeroTurnaround\JRebel\jrebel.jar

Remember that is just an example and your will/might be different.

To actually use it just turn on your newly editted batch file (I mean runserver.bat or run.bat, etc.) and whenever you make changes simply open up your compiler and jRebel will automatically detect changes and apply them without having to restart anything. Heres a small video,

http://img72.imageshack.us/img72/4470/clip0037.mp4

It reloads the classes in your source after they are compiled, so obviously you cant go around expecting it works with CFG. Anyhow, I really hope you enjoy this!
