Hahahahaha this is funny. Pretty sure no-ip was originally used with the DUC for people with dynamic IPs to host nice little wL sources.

“slows down your connection”

Here’s why that isn’t true:

When you register a domain with No-IP an A record (or AAAA if you decide you want to use IPv6) is added to their domain. An A record is cached by either your DNS server or your own computer (in case of Windows or you’re using dnsmasq on Linux). After that initial connection you’re connected directly through the IP of the server (iirc rsps’ are tcp) and every connection to the domain after will be to directly to the IP until the TTL set on the A record.

If you’re on Linux you can use dig to test this. Your first request’s query time will be significantly higher than the second request’s query time.
