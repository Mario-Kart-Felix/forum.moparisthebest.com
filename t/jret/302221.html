I’ve recently started a new project (because I was bored as usual). Named it JRET (for now). Which stands for Java Reverse Engineering Tool. Basically the idea is to build a pascal/VB scriptable updater… Well, I’m writing a wrapper for BCEL trough JNI with some of my own classes and such added which will then be accessible trough a script engine. I’ll also add in a bytecode viewer and such into the program.

Little test:

That’s be this:

var i, j: Integer; JarIO: TJarIO; Classes: TClassGenArray; Fields: TFieldArray; begin Memo1.Clear; JarIO := TJarIO.Create('runescape-551.jar'); Classes := JarIO.GetClasses; for i := 0 to Length(Classes) - 1 do begin Memo1.Lines.Add('Class: ' + Classes[i].ClassName); Memo1.Lines.Add('Superclass: ' + Classes[i].SuperClassName); if Classes[i].HasFields then begin Memo1.Lines.Add('Fields:'); Fields := Classes[i].GetFields; for j := 0 to Length(Fields) - 1 do Memo1.Lines.Add(' - ' + Fields[j].Name + ' : ' + Fields[j].Signature); end; Memo1.Lines.Add(''); end; JarIO.Free; end;

Isn’t much, but then again, I’ve only worked on this since yesterday, luckily I have experience with JNI or I doubt I’d have gotten this far, the Delphi wrapper for JNI is a wreck, using my own updated copy (So far it’s even memory leak free, it’s a bitch to manage memory when your classes have to behave like java classes, lol)
