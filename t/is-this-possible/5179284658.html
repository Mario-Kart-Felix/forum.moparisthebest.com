[quote=“A.L.P.O.H, post:1, topic:517928”]I have been learning how to code in C# and was wondering if I should rewrite my server in C#.
people tell me that C# is a more powerful language, so would this improve the capabilities of my server?[/quote]

‘More powerful’ usually indicates more features, not necessarily greater performance. C# definitely has more features than Java, but its speed comparisons are usually questionable. The runtime environments also work a lot differently. Java uses a Just-in-Time (JiT) native compiler to compile the most/longest used methods from bytecode to native code to enhance their speed, whereas .NET uses an Ahead of Time (AoT) compiler, which compiles most or all of the CIL bytecode to native code before executing. I’m not sure if .NET support JiT compilation or not. Which of these is better is actually quite a debate. It’s more complicated than is immediately obvious. There are pros and cons to both.

So if the runtime environments aren’t immediately comparable, then you have other considerations. Are the library functions more or less efficient than the other? More than likely they are equivalent, though you do want to consider whether the Java runtime environment you’re using binds a lot of its core API features to native code (which Oracle’s does). Microsoft’s .NET likely does the exact same. So decide first if you are using a lot of API-dependent code and, secondly, whether the equivalent code in C# is equally or more accelerated than the code you’re using in Java.

Next, do the API or language features in C# allow you to optimize your pipeline greater than that in Java? In some situations, this may be the case. It depends what I/O libraries you’re using and whether they require specific features of one language over the other (for all intended purposes, C# probably contains all the features of Java, but the converse is certainly not true).

Finally, you could decide to just accelerate the core code yourself by writing your own native libraries that directly communicate with POSIX or Win32 I/O functions. If that’s the case, there’s no need to even question whether to change languages because both C# and Java support native binding with C.

Only switch if you can deterministically know it will enhance the performance. If you’re not sure, port only the core code and write a test driver to benchmark the speeds between the C# and Java cores. Pick whichever is faster and stick with that language.
