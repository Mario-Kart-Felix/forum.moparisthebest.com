IMHO, for the first language, best approach is to get a good book/ebook with exercises. Speed doesn’t matter much, as long as you advance at a steady rate, learning the concepts first and doing exercises to practise them just after. Any side-project for practise will probably also help. However, risk of that is getting attached to bad coding practises, so make sure you are willing to consult the book frequently and to refactor it as you learn.

Once you know one programming language, however, it will be much easier to learn other languages (especially if they use the same paradigm), so you can then take a much more informal approach.

zstars July 31, 2016, 11:08am #33

I don’t really agree with the whole videos thing. Videos might be a ok as a complement, but it is a very bad idea to base your learning on them when learning a language such as C++ (as he mentioned). Videos make sense only for visual technologies, be that a 3D model editor, a RAD tool, or Photoshop. The more complex a language is the higher the need for a book. For references, I invite you to ask about the topic in freenode’s ##c++, where among the 500+ users there are often experts and college teachers.

When you are learning a first language you not only need to learn programming concepts and how they are implemented in the particular language, but also, you need to develop the skill required to use these concepts in the appropiate times by yourself. Best way for this is simply to practise. Just like you wouldn’t hope to learn how to derivate / integrate in maths without actually doing some exercises, generally as you learn different integration methods or different types of integrals, you can’t reasonably hope to be able to resolve programming problems by yourself without practise, which is also best provided by a proper book with exercises. Especially because though much less boring, programming is significantly more complex than integrating.
