But VB7’s way does work. But if you also wanted to set the positions of each button manually, etc., you can use reflection for just the adding part. (The below is tested and does work.)

	private void addButtons()
	{
		try
		{
			Class<?> thisClass = this.getClass();
			Field[] allFields = thisClass.getFields();
			for(Field curField : allFields)
			{
				if(curField.getType().getName().equals("javax.swing.JButton"))
					theFrame.add((JButton)curField.get(this));	
			}
		}
		catch(Exception e) { e.printStackTrace(); }
	}

But that’s pretty pointless. Reflection is slow and you shouldn’t use it if your just too lazy to type out add() a few times. Not to mention, the getFields() method only gets public-access fields.

If someone would just explain the for-each loop…

Basically, for-each takes an array, and puts it into objects.

So say:

int[] integerArray = new int[] { 1, 2, 3 };
// There is your array. Now lets loop it…:
for (int i : integerArray) {
System.out.println(i);
}
// Output:
// 1
// 2
// 3

It takes all of the items in the array, piece by piece, and assigns it to the variable ‘i’.
Then you can use the variable as you’d like.
