Object Orientation
[hr]
Java is inherently object-oriented, which means that Java programs are made up of programming elements called objects. Simply put (don’t you just love the word, simply?), an object is a programming entity that represents either some real-world object or an abstract concept.

All objects have two basic characteristics;

[ul][li]Objects have data, also known as state. For example, an object that represents a book has data such as the book’s title, author, and publisher.[/li][/ul]

[ul][li]Objects also have behaviour, which means that they can perform certain tasks. In Java, these taks are called methods. For example, an object that represents a car might have methods such as start, stop, drive, or crash. Some methods simply allow you to access the object’s data. For example, a book object might have a getTitle method that tells you the books title.[/li][/ul]

Classes are closely related to objects. A class is the program code you write to create objects. The class describes the data and method that define the the object’s state and behaviour. Then, when the program exectues, classes are used to create objects.

For example, suppose your writing a payroll program. This program probably needs objects to represents the company’s employees. So, the program includes a class (probably named Employee) that defines the data and methods for each employee object. Then, when your program runs, it uses this class to create an object for each of your company’s employees.
