2 - It’s best to just make an inWilderness boolean so you don’t have to add a new ‘Safe’ area everytime you make a new area, instead everything that isn’t inside the coordinates will become ‘Safe’, usage;

public boolean inWilderness() { if (absX >= 2940 && absX <= 3400 && absY >= 3524 && absY <= 3970 || absX >= 2375 && absX <= 2418 && absY >= 5129 && absY <= 5167) { return true; } return false; }

You would have to edit your attacking methods to fit this, which is very simple though.
