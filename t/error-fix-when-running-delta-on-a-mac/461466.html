Have you ever had this error?

This is how to fix it.
I’ve saw a lot of people asking how to do it.

So first, lets take “item.cfg” for example, it will say “item.cfg: file not found.”
You need to open up the folder “config” and take out “item.cfg” and put it into the main file.

Now open up “ItemHanlder.java” and search for “config\” and there could be more than one “config\” so double search it, then replace it with “./”

[size=24pt]Make sure to take npc.cfg, npcdrops.cfg, autospawn.cfg, doors.cfg, shops.cfg, item.cfg, globaldrops.cfg out of your config folder and put them in your main file. [/size]

Look no more “item.cfg: file not found.”

Save and compile and run.
Congratulations, you now have a working delta source for mac. [size=24pt](After you do it with the other ones)[/size]

When it’s finished!

[size=24pt]Locations:[/size]

npc.cfg
npcdrops.cfg
autospawn.cfg

are all in “NPCHandler.java”
so there will be three “config\”

doors.cfg

is located in DoorHandler.java
there is only one “config\”

shops.cfg

is located in ShopHandler.java
there is only one “config\”

item.cfg and globaldrops.cfg

are located in ItemHandler.java
there is two “config\”
