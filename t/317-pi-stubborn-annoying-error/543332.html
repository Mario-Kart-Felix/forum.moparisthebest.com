[quote=“xX4m4zingXx, post:2, topic:543332”]There is an interface(or something) called 868685
and that number is too big.[/quote]
Now expand on that and explain how to solve it.

  1. You can make the ID of the interface lower. Search that ID in RSInterface.java then lower it. Make sure you change all instances of that number because they’re most likely for the same element.

  2. Extend the interface/sprite amount. I’m not exactly sure which one you have too many of, but it’s probably interfaces. I’ll show both just in case.
    Find:

		spriteNodes = new MRUNodes(50000);

Should be right underneath the method start.
Change the 50000 to a value higher than the number that’s giving the error (868685).

Find:

		interfaceCache = new RSInterface[j + 31000];

Change the 31000 (yours may not be 31000, look for the variable interfaceCache) to a higher value like before.

I highly recommend going with solution #1 instead of #2 because #2 can drastically change the loading time (in a bad way).
