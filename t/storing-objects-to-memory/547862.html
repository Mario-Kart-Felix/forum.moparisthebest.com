HcoJustin August 4, 2016, 1:58pm #4

Storing something in memory has the benefit of having insanely fast lookup speeds, versus having to read it from lets say a file on the hard drive every time it is needed. In Java, everything is created and stored in memory until it can no longer be reached by a reference. Of course there are still a few exceptions where the GC won’t clean it up even after it should.
