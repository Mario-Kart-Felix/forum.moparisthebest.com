Just looking for some feedback. A while ago, I made a post on MITB about writing a custom kernel for the PSP (school assignment). But, my teacher didn’t give the class enough information to get anywhere close. As it turns out, he just wanted to see how far we could get, so there wasn’t really a “score”.

However, I’m still interested in creating a custom kernel for the PSP. However due to it being my only one, I was wondering if it would be a safer (and more applicable?) to create a “modchip” instead of flashing the NAND. There are a few reasons as to why I would prefer to do this:

[ol][li]By flashing the NAND, I would be erasing all of Sony’s code (including the bootloader). So if I mess something up even once, I’m left with a fancy looking paperweight.[/li]
[li]Since the kernel would be stored on a separate modchip (probably an EEPROM), I can easily reprogram it to my needs. Whether it be to add a new feature, or to fix runtime errors.[/li]
[li]It would not only give me experience working with electronics, it would also help me become a better programmer. It would most likely be written in C/++, probably with some ASM if needed.[/li]
[li]I just think it would be a fun thing to do[/li][/ol]

So if anyone here has any experience/resources with something like this, I would very much appreciate any help you can give.
