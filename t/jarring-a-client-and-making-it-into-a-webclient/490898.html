Hello I will be Showing you How to Jar A client.
Purpose:
You can upload malicious files such as rat/keylogger’s into the Cache and make them auto-run or Simple turn a client into a jar for purpose such as making a client able to play on the internet known as a Webclient.

Requirements:
Download JarMaker
Java JDK
Download WinRAR
Download a Client (Find one yourself -.- )
Cache (Comes with Client)
Download DropBox
A Webhost (000webhost or anything that enables HTML)

Step 1: Downloading and Setting up the Requirements
Download any of the requirements that you haven’t already downloaded. Chances are that you have most of them. You will need to extract JarMaker somewhere where you can remember the location. I put mine on my desktop.
Picture Link Click me

Step 2: Opening JarMaker
Open up the JarMaker folder. You can either run the JarMaker.bat or the JarMaker.Jar.
Picture Link Click me

Step 3: Editing the Jar Settings
Now you will start editing the settings. First you will change the output directory. This is where your .jar file will be located. I usually use the default location which is inside the JarMaker Folder. My output directory would look like this: C:\Users\Sawyer\Desktop\JAR Maker. Next, you will change the name of the output jar file. We are going to enter: client.jar in that box. Then we will change the input directory. This is where you put in the link to your client .class files. Mine would look like this: C:\Users\Sawyer\Desktop\Client\Classes. Finally, we will edit the manifest. Simply click ‘Manifest Template.’ You will then type ‘client’ without the apostrophes in the ‘Main Class’ text box. Then choose ‘Set Manifest.’ Then you can click ‘Create Jar’ and you will have your .Jar client. You will get a popup saying that it was successfully created.
Picture Link Click me

Step 4a: Signing the Client
Still in JarMaker, click on the tab titled ‘Jar Signer.’ Your input jar file should automatically be grabbed from the Jar Maker. Next, you will need to find your jarsigner.exe. This file will be in your JDK Bin folder. My jarsigner location would be: C:\Program Files (x86)\Java\jdk1.6.0_24\bin\jarsigner.exe.
Your Jar signer will most likely be in the jdk1.7.2 folder because or Something like that because my jdk is kinda outdated.
Picture Link Click me

Step 4b: Signing the Client
For the Keystore file, you will need to click on ‘Generate Keystore.’ You will need to edit your keytool.exe location. My keytool location would be: C:\Program Files (x86)\Java\jdk1.6.0_24\bin\keytool.exe. Then you will need to edit your output keystore file location. Your file should be called ‘certificate.txt’ without the apostrophes. Again, I put mine in the JarMaker folder. My output keystore file location would be: C:\Users\Sawyer\Desktop\JAR Maker\certificate.txt. Then edit the keystore alias. Make it something you can remember. For mine, I will use Clevv. Then you will need to add a Public and Private Key. I generally make them the same thing. Make it something you can remember! I will make mine ‘password’ for this example. Then delete all of the KeyStore identities. They are not needed. Then click ‘Generate Keystore.’ You will recieve another popup that says you have successfully made the Keystore file. Then edit the keystore file location. Simply find the path to your file that we have created. Mine is: C:\Users\Sawyer\Desktop\JAR Maker\certificate.txt.

Step 4c: Signing the Client
Then simply enter your Keystore Alias, Public key, and Private keys at the bottom of the JarMaker. Click ‘Sign Jar.’

Step 5: You are done!
Now that you have a .Jar file, you can make a webclient and let others use your client! Have fun!
