It would be nice if the code followed some conventions.

It’s unnecessary to have a no-arg constructor, if the only way you’re instantiating the class is by passing a client to it.

You could move the items into an enum

public static enum Logs {
    NORMAL(item, experience, requiredLevel);

    private final int item;
    private final int experience;
    private final int requiredLevel;
}

Creating a new instance of the class is unnecessary, as it would be easier to pass the client as a variable

Firemaking.lightLog(client, logId);
