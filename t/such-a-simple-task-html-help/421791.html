Well, if your image is a background (which it is judging from the name), why not do something like this:

<div style="position: absolute; left: 0px; top: 0px; width: 100%; height: 0%; background-image: url('/Images/background-1.jpg');">

</div>

The style tag in your code fragment seems unneccesary, and at one point you are using single quotes then double quotes.
Also it would probably be a good idea to put your css into a seperate stylesheet so you can use classes and re-use your styles. It makes web development alot easier in the long run.

I’m pretty new to web development since it’s not really my thing unless I absolutely have to, but I fail to see why what I gave above wouldn’t work unless the image location is incorrect. If you want to repeat or scale your background image.

Also, if your background needs a to repeat or stretch you can use the CSS properties “background-repeat” or “background-size”.

Try to avoid inline styles, they’re untidy and inefficient.

Use this:

* {
    margin: 0;
    padding: 0;
}

body {
    background-image: url(../Images/background-1.jpg);
    background-repeat: repeat-x repeat-y;
}

That’ll work fine providing your image is tile-able, if it isn’t, then… you probably shouldn’t use it.

Oh and the top part is just a html reset to set the default padding and margins to 0 for all IDs, classes and other tags.

If you don’t want to use an external style sheet then just put this in the head and paste my code between the style tags.

<style type="text/css">

</style>

Just that a few of your style attributes are unnecessary, pretty much all of them in fact, just nice to keep it clean.

Absolute position is irrelevant as it is a background image. It already defaults to top left anyway, width set to a liquid background is never good either, plus also unnecessary because all divs default to 100% of the page width (or container width).

Also I defined the attributes under the body tag through css, which is better than creating a new div and assigning that as one container to contain everything else, using it under the body will ensure it is applied to everywhere which has no other background image.
