JVMTI has many functions to interface with the jvm. Probably of most interest to us is functions to get field data (http://java.sun.com/j2se/1.5.0/docs/guide/jvmti/jvmti.html#SetFieldModificationWatch). As you probably know JVMTI is an event based tool meaning you set a few callback functions and they are called when say a field is modified or a method of interest is entered. Processing goes from there. It’s a debugging/profiling tool after all but it allows great flexibility for such a task as creating a bot for rs. All other bot<->client interaction can be done through JNI (mouse events anyone?).

Of course you’ll need a second program/script to update your field database when the client updates. Implementable in an infinite amount of ways, personally I use beloved sed.

However this requires C knowledge and to no offence to anyone here, it doesn’t look like that many people possess it.
