surfer247 July 29, 2016, 9:13pm #7

I don’t know if you still need this, but I think this can help:

if(!fsockopen($domain, 25, $errno, $errstr, 30)){ $error='Invalid domain'; }

This was part of an email check that I modified for this. It checked to make sure the domain that the user entered after the ‘@’ was a valid one and in use by trying to open a socket to it. It prevented people from entering something like “fjkdhfdjsh@fjsdkhf.djsfkg” as their email. It should work fine with subdomains, but I’m not sure about folders inside domains (ie www.mysite.com/section/). Note that the domain has to be in use, not just registered, so if you are looking for something that just checks syntax, this isn’t what you want. Oh, and don’t worry about the $errno and $errstr, they are just for more accurate error reporting. If you want more information on the function, check it out here: http://us3.php.net/fsockopen
