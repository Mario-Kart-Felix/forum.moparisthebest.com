No no…

See, the reason you keep getting errors when you fix another, is because it’s in issue further back in the client. Take a look for things above the messed up parts.

IE>

[code]
public void blabla(){
sendMessage(“blabla”);

public void blabla1(){
sendMessage(“blabla”);
}

public void blabla2(){
sendMessage(“blabla”);
}

public void blabla3(){
sendMessage(“blabla”);
}[/code]
Once blabla1 is taken out, blabla2 is the only thing seen. Basically, blabla1 = blabla2. blabla1 = error. In the end, blabla2 = error. So, fix what came before blabla (or the void you had issues with (same with ints booleans etc)), and it’ll fix everything else. Also take the /* and */ from what you tried to fix away.
