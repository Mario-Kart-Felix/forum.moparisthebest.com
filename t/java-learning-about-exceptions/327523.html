Hello, and welcome to my Java tutorial on learning about exceptions.

What are Exceptions?
An exception is an error that occurs in your Java runtime Applet, (Or in this case, your server). The exception is usually caused by a code that is inproperly added.

What are some examples?
An example of an Exception that occurs is shown below:

[code]public class ExceptionExample {

                 public static void Main(String[] args) {
                     System.out.println("System Now Started.");
                     int[] Array = new int[50];
                     Array[100] = 1;
                 }

}[/code]

This is an example class, with a basic error. The error is, that I created a new Array of ints, which can hold up to 51 different integers (Including 0). However, during the Main method, I made it so “Array” number 100, would equal to 1. Why is this a problem? Because! I didn’t make it so there could be up to 100 different integers in the Array, I only made it so 51 different integers could!

Why is this a problem?

This is a problem because there are no known existing arrays that could hold up to 100 diffetent integers. Although I assign to be otherwise. This is called the ArrayIndexOutOfBoundsException.

Why is it important to try and prevent Exceptions?
Because, an Exception is an Error. If your server has alot of exceptions, then it has alot of errors. Someimes, when you have errors, your server will not work properly, or will not work at all, until the exception is handled.

How can I handle an Exception?
Simply place this code in the class that has the exception you are wanting to handle.

try { } catch (Exception e) { System.out.println("Error on Exeption: " +e); }

This code will Catch any exceptions that happen to come accross. When you specify it to be exception e, it will handle all Incomming Exceptions, instead of a Specfic one.
No, this will Not handle the exception, but it will make an output into your Command Prompt explaining the information. This is much easier creating your own line of output, because when you do so happen to recieve the exception, you would know where it’s at, for the fact that you created your own output at the excact spot that it’s at.
