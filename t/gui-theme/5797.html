BobDole81 July 29, 2016, 8:47pm #3

For the windows look and feel, you can use this:

        try {
            javax.swing.JFrame.setDefaultLookAndFeelDecorated(false);
            javax.swing.UIManager.setLookAndFeel(javax.swing.UIManager.getSystemLookAndFeelClassName());
        } catch(Exception e) {
            javax.swing.JFrame.setDefaultLookAndFeelDecorated(true);
        }

Of course, though, the System look and feel will be different whether your on windows, a mac, or linux

BobDole81 July 29, 2016, 8:48pm #18

You don’t have to switch anything…the client is set up by jagex to run both as an applet an application - when you run it as an application, only 1 method is called by default: main(). As an applet, 3 methods are called by default: client(), init(), start(). That will control everything in the applet really, and since thats all that the browser’s java plugin calls, that’s all you have to call.
