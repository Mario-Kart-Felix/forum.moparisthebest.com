[center][size=14pt]1) Recognizing that your system is infected (+ reporting)[/size][/center]

In order to remove an infected file you must first be aware that you have one. I’m going to create a list of the main give aways for your computer being infected. I’ll number them so that they’re easy to follow. If a scan doesn’t pick up anything, you can tell by the below.

[list type=decimal]
[li]You notice a considerable decrease in your computers performance, this is a tell tale sign as these files are very draining on your computers resources.[/li]
[li]You notice that your computer is taking a while to startup, the infected file has to startup also, it could also make modifications to your system (which also need to startup).[/li]
[li]You notice UAC has been disabled, this is a key trick a lot of people use to limit your control over administrative tasks on your PC[/li]
[li]You notice that you cannot access task manager, this again is always used, this way you cannot end the process, or remove the file.[/li]
[/list]

These are just a few I thought are the most common nowadays, though there are more. Check out the links below for a compilation of other signs.

[center][size=14pt]2) Identifying infected pages (SSP/Forum + more)[/size][/center]

If you can identify the page which is infected before you get infected then you’ll be saving yourself a lot of hassle. Most people who attempt to store infected files on your computer are lazy and lack genuine knowledge which would be required to be a ‘good hacker’.

SSP:
[list type=decimal]
[li]You notice a lack of effort in the advertising of the page, things like ‘jsdfjsgg’. This is a tiny give away but not always true. It shows the person has no interest to advertise the server, only to infect you.[/li]
[li]You notice repeats with the same images/information. In 99% of cases this will be somebody attempting to mass infect users. [/li]
[/list]

On the SSP, if unsure you should always use a link scanner, these will scan potentially malicious links and give you information on them. If a link is found out to be malicious you should report it immediately. Here are some usefull tools to use while browsing the SSP.

[list type=decimal]
[li]Link scanner - This scans links to ensure they’re safe. Good examples of link scanners include: http://linkscanner.avg.com/ and http://linkscanner.explabs.com/linkscanner/default.aspx - be sure to always use a few link scanners as one of them may be incorrect, I’ll be posting some more at the end of this post.[/li][/list]
[li]Source code check - This is vital when playing on a webclient, most tactics used on the SSP are Java drive bys (http://en.wikipedia.org/wiki/Drive-by_download). Checking the source code can almost complete eliminate this threat. A normal applet will look like this (Differs obviously):

[/li][/ul]

<applet name="Webclient_Name" width="765" height="503" archive="client.jar" code="client.class">
<param name="java_arguments" value="-Xmx1024m"></applet>

[ul]Where as that only downloads client.jar, a drive by would redirect you to another index (index2.html in most cases) so doesn’t have the applet code in it’s main index. So, it wouldn’t have the code above, just a code to fake what it would look like until it can redirect you to the page which downloads the infected file. (if that makes sense).
[list][li]You notice that the webclient page looks like this: http://imageshack.us/photo/my-images/268/jdblooklike.jpg/ That is what most java drive bys will look like, if it looks like that you need to check it out and be careful about it.[/li]
[/list]

Take note that all SSP related ones count on the forum also.

[anchor=three]
[center][size=14pt]RAT[/size][/center]

RAT stands for remote administration tool, it goes by its name. It can control your computer and has an array of built in functions to make your life generally problematic. This can range from crashers, to messing with webcams and your computers sound system. I’m going to go over what I myself would do if I had a RAT.

[list type=decimal]
[li]Cover all webcams, and disconnect sound/pen drives. This means the person is limited to how he can affect you/ see you physically. [/li]
[li]Clear all private data (passwords etc) from all programs and your browser, this means the person cannot grab them and use them. [/li]
[li]Download an anti virus, then boot into safe mode and scan. This way the RAT won’t be able to harm you. You could also try to look for unusual processes, end them and then delete the file.[/li]
[/list]

RATs will slow down your system immensely because they come with a huge amount of monitoring. If you’re confident, you can find the file, end the process and remove it yourself but I don’t recommend that.
[hr]
[anchor=four]
[center][size=14pt]keylogger[/size][/center]

It goes by its name, it logs keys, this is mainly used for stealing passwords and your personal data, this one cannot control your PC and the logs of what you’ve typed are usually sent to an email.

[list type=decimal]
[li]Use onscreen keyboard, if you really HAVE to do something use the onscreen keyboard. It will not log what you type there. [/li]
[li]Check for unusual processes, as always, this can be a good way to remove something.[/li]
[li]Download an anti virus, then boot into safe mode and scan. This way the RAT won’t be able to harm you. You could also try to look for unusual processes, end them and then delete the file.[/li]
[/list]
[hr]
[anchor=five]
[center][size=14pt]Trojan[/size][/center]

This pretends to be something it’s not, it derives its name from the old trojan horse story. It can take the shape of an anti-virus program, or a windows application, anything. The window which it appears in will be distorted, and it will over exaggerate a lot. (e.g you have 6000000000000 viruses, buy now or ur gonna die)

[list type=decimal]
[li]There isn’t really a need to protect yourself much here, but it will create a lot of lagg so be prepared. [/li]
[li]Never trust it, no matter how legit it seems, just don’t follow it. [/li]
[li]Download an anti-virus and scan, you will ALWAYS be able to notice the virus process in this case because they are so obvious (size wise compared to other processes).[/li]
[/list]
So that’s it for my 3 examples.
