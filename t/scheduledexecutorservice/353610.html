Introduction
Before you read on, this isn’t solely a Java tutorial. It is knowing and applying the ScheduledExecutorService class for uses on a common server.

ScheduledExecutorService
What is this? This class is available in the [tt]java.util.concurrent[/tt] library and is a subclass of a ExecutorService that basically can schedule commands to run after a given delay, or to execute periodically. Read more…

Example
This is an example since JDK 1.5:

[code=Java] import static java.util.concurrent.TimeUnit.*;
class BeeperControl {
private final ScheduledExecutorService scheduler =
Executors.newScheduledThreadPool(1);

public void beepForAnHour() {
    final Runnable beeper = new Runnable() {
            public void run() { System.out.println("beep"); }
        };
    final ScheduledFuture<?> beeperHandle = 
        scheduler.scheduleAtFixedRate(beeper, 10, 10, SECONDS);
    scheduler.schedule(new Runnable() {
            public void run() { beeperHandle.cancel(true); }
        }, 60 * 60, SECONDS);
}

}[/code]

Application
Now see how applying it to be a simple timed announcement works! I’m comparing it against using the normal [tt]process()[/tt] method.

[code=java]import java.util.concurrent.*;

/**

  • See how the timed announcement works using

  • the ScheduledExecutorService.

  • (This is actually based on BeeperControl)

  • @author Rix
    */
    public class AnnouncementSchedule {

    /* Schedule instance. */
    private static ScheduledExecutorService scheduler =
    Executors.newScheduledThreadPool(1);

    /**

    • The main method (duh…)
    • @param args The arguments.
      */
      public static void main(String[] args) {
      // something to schedule at a rate
      // it will print out “ANNOUNCEMENT!”
      final Runnable announcement = new Runnable() {
      public void run() {
      System.out.println(“ANNOUNCEMENT!”);
      }
      };
      // this schedules printing out “ANNOUNCEMENT!”
      // at a fixed rate
      final ScheduledFuture shout =
      // delay, interval, unit
      scheduler.scheduleAtFixedRate(announcement, 500, 500, TimeUnit.MILLISECONDS);
      }

}[/code]

[code=java]/**

  • A timed announcement using process()

  • like how it’s used in winterLove servers.

  • @author Rix
    */
    public class AnnouncementProcess {

    /* A flag to specify if the pseudoserver is offline */
    public boolean offline;

    /* The constructor. */
    public AnnouncementProcess() {
    while (!offline) {
    // the NanoTimer class is used as a timer (apparently)
    for (NanoTimer n = new NanoTimer(); { // processes every 500ms
    for (int t = n.time((long) 500 * 1000000L); --t != -1; process());
    }
    }
    }

    /**

    • The main method (duh…)
    • @param args The arguments.
      */
      public static void main(String[] args) {
      AnnouncementProcess ap = new AnnouncementProcess(); // instantiates the constructor
      }

    /* The method called every 500ms */
    public void process() {
    System.out.println(“ANNOUNCEMENT!”); //prints “ANNOUNCEMENT!”
    }
    }[/code]

These programs’ outputs would be, every 500ms:
[tt]ANNOUNCEMENT![/tt]

More importantly, we would want to know which uses more CPU. Running AnnouncementProcess averagely uses 0.9% of the CPU (see this picture for proof) but… AnnouncementSchedule uses 0.0% of the CPU (picture here) (by the way, I was using Java VisualVM if you were wondering)!!! You may want to try this on your own. Download the folder from http://uppit.com/ykhyvmtpk013/schedule.zip and see the results.

Conclusion
ScheduledExecutorService > Usual processing (although you can use event handlers… but IMO this is better for simpler tasks)

Hopefully you have learnt something from this tutorial. If you have any questions, feel free to ask me.
