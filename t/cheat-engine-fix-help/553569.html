sk8rdude461 August 4, 2016, 4:33pm #3

There is no permanent client sided fix.

You may be able to prevent cheat client from working with your client (Which I highly doubt), but then another program (eventually if not already) will be able to do the same thing.

It’s a never ending battle, really. You develop an anti hack system, someone develops a new hack. Rinse and repeat.

X-J-K-X August 4, 2016, 4:33pm #4

Validate everything server sided and you wouldn’t have this problem.

There are ways to block it, but there are also ways to bypass the block, so as said above it’s kinda impossible.

Get the list of processes, scan through them, find the original file path, match the file executable’s bytes with a list of blocked executables or at least a pattern, then block accordingly. Might work, might not, there’s always a way around.

Welliton_inc August 4, 2016, 4:33pm #5

You can never make your server 100% secure. There are always people finding ways to bypass security. All people do is just add extra layers of security in order to make the process of exploiting longer and harder, but not impossible.

Think of it like breaking into a house, you have a lock, a door, an alarm. But the thief can still break into the house, but the more things needed to break into, the longer and more effort is required to do so.
