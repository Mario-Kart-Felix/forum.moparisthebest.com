So because my aus host has had a few problems with dos attacks and power outages (mfw) I’m going to be moving the server and services over to a new linode vps in the US (California). The new server is now all setup and I’ve just tried redirecting everyone to the new server through the RPL_REDIR numeric but that had two problems. Firstly, some people’s clients flat out ignored it and secondly it seemed to trigger the connection flood settings on my new server when it received 30 connects all at the same time. Testing changes on “production/live” systems are great!

So my plan is thus,

  • Tomorrow I’ll link the new server to the old server and move the services over as soon as I link it
  • Once the servers are linked I’ll update the dns to point to the new server only and I’ll leave the old server up for a few days to allow dns propogation
  • Then I’ll set the redirect numeric for new connections only to point to the new server and either leave it for another day or so or randomly start disconnecting people until there are no more clients on that server
  • When the server is empty I’ll kill the link from the new server <-> old server
  • Will still continue running bopm from my home connection, if only to not piss off linode with the port scans it does (not a critical service so it doesn’t really matter where it runs from)

Should only take a few days with minimal interruption if I feel like taking it slow.

Unfortunately(?) this will mean zachera’s server will also be isolated, however I’m also reconsidering even having a second server in the first place (yeah yeah, I know I know).

I doubt anyone really knows or cares, but yeah. This move should bring us back to being stable again.

Migration is effectively complete, all that needs to happen is for existing users to reconnect to irc.moparisthebest.xxx
The old AU server will be shutdown tomorrow (sometime after 24 hours from this post)
