Tech

Attributes
Our attributes system was made to prevent unorganized variables clogging up entity-extension objects. It’s fast, reliable, and easy to use! Our first type of attributes system can replace boolean variables. It’s an attribute tag system that is constructed with an array list-iteration of a String. We can easily add/remove temporary attributes that store data with the more advanced attributes system. This system can replace int(integer) variables. Constructed with a hash map-interaction of String for the attribute key, and Integer for the attribute value it’s a simple solution to storing data that we can serialize to a player’s game file. These attributes save thousands of pages in memory and used the right way lead to a more organized server!

Plugins
We have setup many plugin systems all around the server. From teleportation plugins, to projectile based spell plugins, the plugins are simple abstract classes that serve as interfaces for a true-plugin. A true-plugin has pre-made data and few construction arguments making content-development far more elegant. The plugin system is based around the OOP (object-oriented programming) style, where there are many branches off other objects.
