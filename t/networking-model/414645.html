Ok, so I designed a networking model for a MMO project I’m currently working on.
I’d like to get some critism on this model and what could be done better or if this is mad.

So basicly, I’m planning to let the users host my servers.
As in, I host the Master Server, and the users host the actual servers that do the logic and all the handling.
I have a small budget so hence I’m using this model.
I’m integrating server hosting protocols in the client, so that players can host directly from their client by simply clicking create server, and then by setting it up. Then that gets registered in the Master Server and other players can now join a game. This would mean though that a server can’t have more than 64 players at the same time.

Now here are my concerns, this is going to be an MMO, and letting the players host the servers is kindoff dangerous. Because I’m planning to let all those servers connect to the same databases that have information about each player (name, password, ip, information about a custom place he built,…).

So, if you know any games using this method please mention, if you think this is completely idiotic because it is too insecure and people can obtain database information please mention that aswell.
If this is actually a good model that is possible, then mention that aswell.

Or, how would you do this?
