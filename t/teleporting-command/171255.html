If you want to teleport by just talking, then you would do this in client.java, case 155 (first click NPC):

if (NPCID == 400) { try { teleportToX = 5412; teleportToY = 8123; sendMessage ("You have been teleported to wherever!"); } catch(Exception e) { sendMessage("Error with teleporting to wherever!"); } }

Just replace 400 with the NPC id, 5142 with the X coordinate of where to teleport to, 8123 with the Y coordinate of where to teleport to, and wherever to where you are teleporting them to. You don’t need the catch(Exception e) part, and there is pretty much no possible way for it to not teleport you (unless you deleted the teleportToX and teleportToY ints), but I just added it anyways. Click on the picture of a person running icon to see your coordinates.

If you want to have them talk to them and give an option of Yes and No and if they say Yes to teleport them somewhere, then look for a tutorial. It is too long to explain.

A normal code for a teleportation command would be the following (add under the barrows command):

else if (command.equalsIgnoreCase("commandname") { try { teleportToX = 5142; teleportToY = 8123; sendMessage("You have been teleported to whereever! } catch(Exception e) { sendMessage("Error with teleporting to wherever!") } }

Just replace commandname with what you would want them to say to execute the command in-game (::commandname if you didn’t change it), 5142 with the X coordinate of where you want to teleport them to, 8123 with the Y coordinate you want to teleport them to, and wherever to where you are teleporting them to. Click on the picture of a person running icon to see your coordinates.
