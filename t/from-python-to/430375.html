uh, general ‘stuff’ usually applies to any OO language. java is a pretty common language in the market, so if that’s a goal of yours i’d become pretty familiar with it. STL in c++ is sort of similar to java’s, how do i say this… common(?) libraries. c++ by default doesn’t have support for threading or networking or anything really fancy of that sort. STL basically provides basic data structures like iterators, strings (not to be confused with cstrings), maps, vectors (basically ArrayList in java), etc.

if your question is ‘should you take java seriously, not just as a stepping stone’ then the answer is yes.

Web Applets are the main thing that Java has over the other two. Also more people have the JVM installed than the python runtime, which you would generally have to bundle with your program on platforms like windows. Lastly, with C++ you have to be very careful with, and sometimes even write the same method twice to make it work the same under multiple compilers and/or platforms, then you have to distribute multiple binaries, one for each platfrom. With Java all of that is taken care of for you, and you can ‘compile once, run everywhere’.
