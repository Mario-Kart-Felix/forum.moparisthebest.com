Class File Manipulation Library

This has been long in the works, and I work on it whenever I get bored. I suppose some people’d like to see it (maybe), and it’s not entirely complete, but it works. Just wanted to throw it up here…

Design choices:

Constants are generalized. ConstantUTF8 is the only specific one, having a string associated with it. LengthConstant is a constant that has an operand of 2, 4, or 8 bytes in length (ConstantClass, ConstantString, ConstantInteger, ConstantFloat, ConstantLong, ConstantDouble). ConstantTwoShort is a constant that has two u2 operands (ConstantFieldref, ConstantMethodref, ConstantInterfaceMethodref, ConstantNameAndType).

Instruction types are generalized, i.e. OpInsn is an instruction that consists of only an opcode, and the majority fall into this category (*aload, the math instructions, etc.). OperandInsn is, obviously, an instruction that has a u1/2 operand (bipush, sipush). The rest is self explanitory. I stole this scheme from ASM.

If you post questions, I will answer them. There’s a test class in there (Test) that shows how the Code attribute must be used for changes to be reflected in the written class file.

Please, PLEASE, use this and tell me if you spot any bugs. Thanks.
