sspoke August 4, 2016, 11:01am #2

byte-code is ran by java.exe which re-translates byte-code back to assembly.

assembly is hard-coded into the CPU that’s the language of the CPU. Although the CPU uses the assembly as just a bunch of binary data people made it easier to read with words like PUSH, ADD, POP, JMP, CALL.

assembly binary data is like a stream class in runescape private servers one bad byte and the bytes won’t be translated properly after that.
