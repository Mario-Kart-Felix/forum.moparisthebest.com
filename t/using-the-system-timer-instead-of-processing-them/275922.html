Hey everyone, today I am going to teach you how to use ‘timers’ more efficiently by using the system’s current millisecond time. It can be used in codes in which you want to restrict something being done in a certain amount of time.

Everyday I see tutorials/servers containing codes in the process method to restrict something from happening, such as the delay in which you are able to eat a food item/drink a potion. There is a much efficient way to do it: the System timer.

The ‘System.currentTimeMillis’ is a long datatype and is equal to the current time in milliseconds. The long data type is a 64-bit datatype, and it is used when you want to have a larger scale of number(s), instead of an int.

If, for example, you wanted to restrict players using an object for … let’s say 10 seconds, you would do this:

[code]long integer;

if (objectID == ####) {
if (System.currentTimeMillis() - integer < 10000) {
sendMessage(“You can only use this object once every 10 seconds.”);
return;
}
integer = System.currentTimeMillis();
/*
* Whatever the object does goes here.
*/
}[/code]

10000 milliseconds is equal to 10 seconds, hence the “integer < 10000”.
The long variable will equal to the current system’s time in milliseconds if 10 seconds have passed - Remember this. The system’s current time will constantly be raising while the ‘integer’ variable will not, until 10000 milliseconds (10 seconds) have passed, then it will be equalled to the system’s time again.

The System’s time is constantly going up in milliseconds, just like the real world time.

if (System.currentTimeMillis() - integer < 10000) {

This line means that if the System’s current time minus the ‘integer’ variable (which was equalled to the system’s time), does not exceed 10000 milliseconds, then you cannot gain what the object does/gives to the character until it does exceed 10000 milliseconds, and the method will end by using the ‘return;’ code.

As I said at the beginning, it can also be used for food item eating. Here is an example of that:

[code]case 385:
long healTimer;

if (System.currentTimeMillis() - healTimer >= 2000) {
	healTimer = System.currentTimeMillis();
	deleteItem(385, Slot, 1);
	startAnimation(829);
	updateHp(20, true);
}
break;[/code]

For this food item (Shark) you have to wait 2000 milliseconds (2 seconds) in between eating each one.

I hope you all learned something ;).
