Linguistic vs. Programmatic Method Names

General Programming
zyle1992 August 2, 2016, 4:32pm #1

So recently, I’ve been seeing how different people name their methods/functions. The two most common types are Lingusistic (which is more natural), and Programmatic (which can improve understanding). Here’s an example of each

Linguistic:

public interface Plugin {
    public void onEnable();
    public void onDisable();
}

Programmatic:

public interface Plugin {
    public void enable();
    public void disable();
}

Now, which one of these is “correct”? Before you say “well it depends on the programmer”. Here’s the difference between the two (at least in my understanding)
Linguistic is more “natural”. So in this case it would be “on enable, do this”; whereas with Programmatic, it’s more “enable the plugin”. To me, Linguistic seems to cross from the past-tense to the present-tense (“when the plugin is enabled, this happens”) which I’ve always been told to avoid, and to keep everything in the present (“load the plugin”).

However, it seems that Linguistic is by far more popular (Android, HTML/Javascript, and Bukkit just name a few).
