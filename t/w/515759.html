Lin August 4, 2016, 6:49am #10

Because of the way networking works, packets - groups of data with headers as to what exactly the groups of data contain - consist of bytes. When you click an item in the client, it creates a packet with the bytes containing the values to make up an integer (4 bytes combined). However, to make packet ‘sniffing’ and cheating harder, the client adds a value like 128 to the item ID and when the server reads it, it simply subtracts the 128, which creates the original un-touched item ID. Also, your “readSignedWordA” would work just fine as “readUnsignedWordA”.
