I’m a little confused here.

Edit2: I guess what we are doing is “dynamically linking” to other code. See here: http://en.wikipedia.org/wiki/GNU_General_Public_License under “GPL in Court”. Yes I know it is unlikely we would actually be able to enforce this but lets just have a hypothetical discussion.

Edit3: Looking at http://blogs.zdnet.com/BTL/index.php?p=3937 it appears that java isn’t under the GPL but under a modified version. What that means is that you can link to it but if you write a program under the (full) GPL, the ball game changes.

Edit4: http://www.gnu.org/licenses/lgpl-java.html With LGPL inheritance and “linking” are the same thing. So that means that if calling a library’s constructor/method are allowed then they are allowed to extend/inherit. It would make sense that this similarity would be the same for the GPL so the question is are we allowed to “link” (call constructors/methods) from closed-source code with GPL’d code?

Edit5: Since extending is the same as implementing in the eyes of the GPL and according to this http://www.gnu.org/licenses/gpl-faq.html#OOPLang all Aryan scripts must be released under the GPL. These scripts extend the abstract class Script which is under the GPL and therefore they are part of the GPL. This however does not make sense as I’m sure that someone has released closed-source modules for applications like eclipse.
