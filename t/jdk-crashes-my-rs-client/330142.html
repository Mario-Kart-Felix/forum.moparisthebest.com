mod_taharok July 31, 2016, 2:52pm #6

The solution to this method really should be stickied. It’s starting to be asked a lot.

To put it frankly, the 64 bit JIT compiler inside the VM has troubles compiling the walking algorithm to native code, hence the reason the client crashes when you start walking. I don’t know if there are equal problems in the other distributions of the 64 bit runtime environment-- I’ve never been interested enough to check.

I don’t know enough about the VM to speculate as to exactly what the root cause is, but the solution is simple. Use a 32 bit VM instead.
