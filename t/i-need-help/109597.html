Shops:
Step One
Open up shops.cfg and add a new shop in the same format is one above it, only make the number one more, so if you have 1-42 shops, make shop 43.
This is the format description
the number, “shop = #” is the wanneshop number. Remember it.
the 1 1, or 2 2 is the shops buying capacity. 1 1 means it buys everything (general store) 2 2 means it buys only what it sells
after that are items and their quantities
item id, quantity, item id, quantity
example:

4151 10 4153 10

that would be 10 abyssal whips (4151) followed by 10 granite mauls (4153)
Step Two
Now, pick an npc. If it’s tradable, (has a trade-with option) continue on, if it isn’t, skip this next part

Step Three
now open up autospawn.cfg and spawn a new npc, this is the format description
spawn = #

is the id of the npc

the next two numbers are the X and Y coordinates coorospondingly.
the next is the height level, to find these, go where you want it to spawn on your server and type in ::mypos
it will give you all you need
the next are walk ranges, walk type and description
for ranges, youll see
x1 y1 x2 y2
just replace it with the coordinates of the spawn, it wont move at all. or if you’re feeling clever, try to figure it out, for walktype put 0, and for description just put anything

Voids:
Voids are basically a place to store something for repeated use. example

public void ihatetotype() {
sendMessage("I hate typing alot");
addSkillXP((100*playerLevel[1]), 1);
teleportToX = 1337;
teleportToY = 1337;

now, say you have 3 commands you want to do the same thing, instead of putting

sendMessage("I hate typing alot");
addSkillXP((100*playerLevel[1]), 1);
teleportToX = 1337;
teleportToY = 1337;

in all three commands, you can just put

ihatetyping();

and it will take the same effect. to add a void, search for this

public void

just add it above another.

Booleans:
booleans are just a piece of information, which can be used as a true or false. example:

public boolean modhouse() {
if((absX >= 3249 && absX <= 3258 && absY >= 3438 && absY <=3431))   
heightLevel == 1)
return true;
else				
return false;			
}

now, say you want a command only to work if your in the a mod house, like ::godownstairs, for the coordinates you would put the coordinates of your house. absX >= is the lowest x coord, absY >= is the lowest y coord, absX <= is the highest x coord, absY <= is the highest y coord of the area. there are more things you can do with a boolean then just locations, for example
inCombat();
now in the command you would have

if (command.equalsIgnoreCase("godownstairs") && modhouse() == true) {
heightLevel -= 1;
teleportToX = 1320; //fake coord
teleportToY = 4545; //fake coord
sendMessage("welcome downstairs"); //yay for downstairs
} else if (modhouse() == false) {
sendMessage("youre not in the mod house"); //aww so sad
}

also a boolean can just be something for personal use…example
making it so you have to have used an object recently to open a door
you would have to declare

public boolean opengate = false;

you could declare it = true; but what you declare it as is what it starts as when you log on, in the case you may have something like

case 12345:
{
opengate = true;
}
break;

then the case for opening gate case

case 5434: //gate
if (opengate == true) {
teleportToX = 2144; //fake coord
teleportToY = 3243; //fake coord
sendMessage("welcome"); //yay
} else if (opengate == false) {
sendMessage("you cannot open this fool"); // aww so sad
}

when you’re using it in an “if” or “else if” statement, you use == true or == false, when your setting it, like in the case makes it true, you use = true; or = false;, and in an information boolean such as coords, make sure to use the () after it.
to add a boolean search for

public boolean

and add it above it
