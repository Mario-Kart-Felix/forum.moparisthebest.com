abraham2119 July 31, 2016, 7:10pm #14

The game is fun, but the code is horrible. As super_ said, try to abstract levels/worlds. Defining levels through a configuration system would be best.

I see a tremendous amount of hard-coding. You should always try to make an application as dynamic and as easy to develop as possible. All of the resource-loading, drawing, event handling, and basically the rest of the engine processing is hard-coded. Try to buy a book on game developing, or on general application design. It should help.

Also, swing provides double buffering support by default. This option can be toggled via the ‘JComponent.setDoubleBuffered(boolean)’ function. So no need for your custom back buffering method. Also, painting on top of a JFrame is a horrible idea. Instead, paint on a swing container such as a JPanel.

You should also look into archiving all of your external resources (sprites, etc) into the same JAR used as a codebase since this provides ease of usability. Look here for more info.

But overall good job, especially as it is your first game. I remember my first game was equitable, if not worst, to how yours is written. You will progressively improve, though.

If I have time, I’ll write my own version of pacman along with a simple level editor so you can learn from it.
