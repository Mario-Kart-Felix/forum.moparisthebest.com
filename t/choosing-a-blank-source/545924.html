P.S: What is a RS2Protocol?

rs2 protocol = runescape 2 protocol

protocol: a set of rules governing the exchange or transmission of data between devices.

so basically the protocol is the way that data is sent to and recieved from the client. one thing that you NEED to understand is the basis of how runescape servers function is by SENDING (output/writing) data to and RECIEVING (input/reading) data from the client.

whenever you see people talking about “I(input)/O(output)” all they’re referring to is communication between the client and server. also note that a “packet” is data that’s being sent or received

another example is you might see these numbers a lot in the runescape servers part of the forums 317, 319, 377, 459, 474, 508, 525, etc. those are all different revisions and each of them have a different protocol (different packet structure, different way of doing things)

last example, god you better be reading all of this lol.

  • when you pickup an item, the client sends a packet to the server
  • when you drop an item, the client sends a packet to the server
  • when you want to send a message to the player in the chatbox, the server sends a packet to the client
  • when you want to display an object, the server sends a packet to the client

this will all start to make sense once you start playing around with the server, if you decide to use a good server that is. once you understand this you will already be ahead of most of the community. you are not going to become a programming genius after 1 or two months. this isn’t something you can just jump into (if you want to actually become good, fast). study the internals, learn how everything works and you should be fine bro!

once again, don’t feel like you’re being annoying. any questions you have, post em here
