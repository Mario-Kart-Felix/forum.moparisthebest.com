King_JC August 4, 2016, 10:48am #20

I submit a new task/tick to registerGlobalEvents() for things like my in game clock, water temperature variable (re-calculates every 5 minutes) etc., and I have no performance issues.

I keep all new instances of Objects local/inside if-else statements so I don’t create any unnecessary Objects if the criteria isn’t met.

I understand though that long term this may use up more memory as Objects won’t be cleaned, but nothing a weekly restart won’t fix. In Hyperion there is no other way to do things like this without reinventing the process() method.
