Make the client do its fair share of the computing.
All the client does these days is connect to the server, record mouse & key events, and display the game.
That really isn’t much considering the server and the client both run on the standard household computers of the people running & and playing the game.

 Yes, thats really it. Just make a sever with the most basic things possible:

A playerhandler,
A socket,
(Perhaps) A system timer synchronised with the client.
A table full of booleans player stats & other things,
System for updating files when logged out,

Thats it.
Now, I know. It makes another problem: Client editing. However; note that all the game upgrading will be upgrading the client, which means the client will have to be re-downloaded every time the most up to date version of the game needs to be played. In essence, it is all client editing. And note; if someone had already created an edited client for the current way servers are set up, they would already be able to control it. Since nobody has devoted any time to it, making one wouldn’t be any easier, and only a total jackass would bother to hack a private server, I don’t think its really a big problem.

Just think about it
