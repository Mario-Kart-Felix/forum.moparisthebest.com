sk8rdude461 August 4, 2016, 3:00pm #4

What object owns the “IsDead” variable?
Is there an “IsDead” variable for NPCs?

I believe there is an NPC class (NPC.java) somewhere in your server… Could you look in there and see if it has an IsDead boolean, or something of similar sort?

If it does… You could change the code to this:

if (server.npcHandler.npcs[attacknpc].npcType == 1645 && server.npcHandler.npcs[attacknpc].IsDead) { PlayerHandler.messageToAll = ("The xxxx has been slain by " + playerName); }

The reason it doesn’t work currently is because the “IsDead” variable is probably related to the Client object (A.K.A, the player). So that boolean is only true when the player dies, or false when they’re currently alive. Hence why it constantly sends as you attack the monster.

Have you tried dying while attacking said monster? I bet (with the current code) it will probably send your message.
