Well, I haven’t played the game this is made for, but I have a couple tips that will make your scripts a lot more user friendly.

First of all, you’re using coordinates for the clicks, and for some people, the coords may be different. I suggest using bitmaps, there’s a few tutorials for bitmaps in the Scar FAQ’s and Tutorials sections.

Another thing to keep in mind would be to name your procedures after what they do. So if that procedure a you have shoots or something, you could name the procedure shootStuff;. This is just one way to remind yourself of what the procedure does when you look back at it a while from now.

The last tip I have is indenting. It will not only help you in the future, but other people when they look through your script.

For example, I indented your script here…
[scar]////////////////////////////////////////////////////
// By:SSlleeaakkiinngg //
// Type:Itchy Invasion //
// This is my second script. //
// Please comment on this. //
// Hope you enjoy it! //
// Works on any. //
// //
////////////////////////////////////////////////////
program itchy;

procedure a;
begin
clickmouse(437,415,true);//hits any single petpetpet and robot
sleep(300);//the reason why they are so long because you wont lose ammo
clickmouse(422,384,true);
sleep(300);
clickmouse(648,358,true);
sleep(300);
clickmouse(666,404,true);
sleep(300);
clickmouse(650,438,true);
sleep(300);
clickmouse(387,235,true);
sleep(30);
clickmouse(387,197,true);
wait(350);
end;

begin
repeat
a;
until(false);
end.[/scar]

This way the script has a much nicer look to it and is spaced out some more so that its easier to debug or look through.

But, a very good 2nd script from what I can see, keep it up!

-Weibs.
