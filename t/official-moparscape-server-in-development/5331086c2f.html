lothy August 4, 2016, 1:32pm #811

Tabbed banking shouldn’t be that hard.
The bank table in simplest form can be represented with the following relation:

PlayerID
ItemID
Amount
PRIMARY KEY(PlayerID, ItemID)

Add an extra field, TabNumber, and store the tab that any given item is in (or null, if not sorted into a tab).
Then you simply apply filtering and ordering when it’s time to display the items in the bank panel.
