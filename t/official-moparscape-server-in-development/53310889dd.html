This is some code i wrote using sini’s lua integration. With it, I extended his API a little further. It doesn’t allow scripts direct access to anything. Everything in the script environment is sandboxed, in a way.

[code=lua]game.bind(‘command’, ‘axp’, function(args)
local xp = tonumber(args[1]) or 0

addxp(attack, xp)
end)

game.bind(‘level-up’, attack, function(level)
gfx(fireworks_gfx, 100)
skillbox(attack, level)
intclose()

local steel_long = 1295
local black_long = 1297
local mith_long = 1299
local addy_long = 1301
local rune_long = 1303
local dragon_long = 1305

if level == 5 then objbox(steel_long, ‘You can now use @dbl@steel @bla@weapons’)
elseif level == 10 then objbox(black_long, ‘You can now use @dbl@black @bla@weapons’)
elseif level == 20 then objbox(mith_long, ‘You can now use @dbl@mithril @bla@weapons’)
elseif level == 30 then objbox(addy_long, ‘You can now use @dbl@adamant @bla@weapons’)
elseif level == 40 then objbox(rune_long, ‘You can now use @dbl@rune @bla@weapons’)
elseif level == 60 then objbox(dragon_long, ‘@dre@Members @bla@can now use @dbl@dragon @bla@weapons’)
end

– testing api functions
if level == 99 then
if skill_lvl(strength) == 1 then msgbox(’@red@WOW! @bla@Level 99 attack with only 1 strength?’) end
say(‘W00T’)
delay(2)
anim(human_dance)
end
end)
[/code]

Like I said previously, I can understand not wanting to use a scripting language based on not knowing it. But man, lua is as simple as it gets, the only learning curve would be API calls.
