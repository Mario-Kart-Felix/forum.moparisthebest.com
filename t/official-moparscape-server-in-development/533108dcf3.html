public static Position bestDummyPosition(Position cur, Position next, int size) { if (cur.equals(next)) { return null; } int deltaX = next.getX() - cur.getX(); int deltaY = next.getY() - cur.getY(); switch (Direction.between(cur, next)) { case NONE: return null; case NORTH: return new Position(cur.getX(), cur.getY() + 1); case SOUTH: return new Position(cur.getX(), cur.getY() - 1); case EAST: return new Position(cur.getX() + 1, cur.getY()); case WEST: return new Position(cur.getX() - 1, cur.getY()); case NORTH_EAST: if (Direction.isTraversable(cur, Direction.NORTH_EAST, size)) { return new Position(cur.getX() + 1, cur.getY() + 1); } if (Math.abs(deltaX) > Math.abs(deltaY)) { if (Direction.isTraversable(cur, Direction.EAST, size)) { return new Position(cur.getX() + 1, cur.getY()); Basically, it’s dummy following method. The position is checked for blocking outside this method, so don’t need to mention that for N/S/E/W. For NE/SE/NW etc, the reason its messy (and the thing I want to see if I can clean up), is that it follows a specific priority.

  1. Checks for an exact diagonal match
  2. If not found, find the preferred direction (for example, if the target is 5 tiles north and 2 tiles west, it will check for the north position first and then the west)

It works properly in terms of mechanics (as far as Im aware at least), but I just get the feeling Im missing something painfully obvious to make it cleaner.
