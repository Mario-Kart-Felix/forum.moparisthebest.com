sinisoul August 4, 2016, 11:07am #318

No the way they interact with each other is through dependencies. The use of the dependencies hash key in the info method of a plugin now is used to assure that plugins load in the correct order, not to assure that they were in fact loaded. If you extend the base class in the plugin module now it will automatically add the plugin to be loaded at runtime.

class TestPlugin < Plugin::Base
  info :authors => [ 'david_African', 'sini_African' ], :dependencies => 'blah', :description => 'huehuehue'

  def load
    bind :btn, :action => :say_hi, :id => 505
  end

  def say_hi
    puts hi
  end
end

class Blah < Plugin::Base
end

# Both are registered
# Blah is loaded when test is loaded, if blah is loaded before test blah is not reloaded :P
