Step 1 - obtaining the jar:
With the iDungeon script, they give you a launcher. This launcher follows a protection method that is explained here(http://www.powerbot.org/vb/showthread.php?t=527049) with a custom en/decoding method. The launcher I used can be obtained here(http://shadowscripting.wordpress.com/download-idungeon/).

So open up ur decompiler. open the launcher jar in that program. look in the main class.

find the line where the URLClassLoader is declared (for this script it is in the loop method). below isnt what u will see but it will be close(for idungeon unless other script writers use the same encoding method).

notice the i.l method you will need to include that method, to en/decode the strings in the project u create in ur IDE or folder of sources.

id advise u look at the tutorial that it looks like they used from the powerbot forums b/c if the scripter uses what they suggest and u mess up whoever bought the account will no longer have one… good thing for me this scripter didnt do that, bad for them since now i have their code they tried to keep guarded. the iDungeon site checks to see where the request is coming from, if its not java then it redirects u somewhere. this is what i got from decoding using the method mentioned: “shadowmoose.com/iDungeon/New_Loader/createJar.php?name=user&pass=pass”(not exactly the same but i added stuff to try to speed this up).
