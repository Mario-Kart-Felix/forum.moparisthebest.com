Yeah… No.

Try actually reading the API. java.awt.event.ActionListener inherits source, a field in java.util.EventObject. Invoking getSource() will return ‘source’ which is the Object of which the event initially occurred. What you want to do is invoke getActionCommand(). It will return the String identifying the particular command for this event.

So, I fixed your code up for you.

[code]import java.awt.BorderLayout;
import java.awt.event.ActionListener;
import java.awt.event.ActionEvent;
import javax.swing.JFrame;
import javax.swing.JButton;
import javax.swing.SwingConstants;
import javax.swing.JOptionPane;

public class Test extends JFrame implements ActionListener { // Personally I don’t like the K&R style of code format.
public static void main(String args[]) {
Test frame = new Test();
boolean isResizable = false; // ?
frame.setDefaultCloseOperation(EXIT_ON_CLOSE);
frame.setTitle(“Adrian’s GUI”);
frame.setSize(400, 400);
frame.setResizable(false); // Is this what you wanted?
frame.setVisible(true);
}

private Test() {
	setLayout(new BorderLayout());
	JButton myButton = new JButton("Press me!");
	super.add(myButton, BorderLayout.EAST);
	myButton.addActionListener(this);
}


public void actionPerformed(ActionEvent e) {
	if(e.getActionCommand().equals("Press me!")) {
		JOptionPane.showMessageDialog(null, "You pressed my button.");
	}
}

}[/code]
