As DeathsChaos9 states, it’s just a warning. Just ignore it, it’s not a big deal.

If you really want to know…

“client.java uses or override depreciated API” - means, as far as I know, in client.java is using code or overriding code which is deprecated.
Basically…lets say someone creates code which people then use. Then create something much better. Instead of deleting the old code, which breaks every program using it, the old code gets tagged deprecated to let people know not to use it. Deprecated-list, every class should have it stated in the java-documentation as well, if it is deprecated. Also, (some/most) IDE’s tend to point it out to you too when you try using a deprecated class/method.
