xx_matty_xx August 4, 2016, 11:04am #11

You’re not really being very specific about what you’re actually implementing in these languages. If you’re just talking about the backend then you can do everything over HTTP using PHP and you’d need a custom client that communicates with HTTP instead of sockets. It wouldn’t be great because HTTP is stateless and it’s only a request-response thing, so you’d be making a lot of requests and causing a lot of network and cpu overhead. One advantage I guess is that you don’t need to hold sockets open so maybe the server would use less memory.

As for the client, yes I see no reason why you can’t implement it with HTML/JS using WebGL. You might have a few problems working out how to make network connections because of sandboxing etc, but it should be possible provided that everything’s on the same domain name.
