The_Enforcer August 1, 2016, 1:22pm #12

RSCD actually has tons of publically unknown vulnerabilities that can be used to crash the server. Most of them can be abused by sending specially crafted packets that cause exceptions in code parts that have no exception handlers. These can of course be easily fixed by reading the stack trace. However there a few that cause the exception indirectly, in this case the actual cause is difficult to find and adding a try block wouldn’t work since the game engine would already be corrupt - these are the useful ones.
