I had this few weeks back. IT was SO frustrating, the only way I fixed it is, I do a backup of my source before I edit any file that could possibly glitch up. So I copied all the NPC Drop files, and pasted them onto my good source, and it worked like a charm.

Yes, if you want to edit drops you use the .TSM file, but don’t forget, once you add a new item, you don’t press (SPACE), you press (TAB). From what I see, yours is not a TAB, it is about 3 TABS? Idk

example: let’s use your man file.

#Man drops 1:1(TAB)995:100(TAB)554:100(TAB)4151:100 #Guard drops 9:1(TAB)995:200(TAB)555:100 #general graardor 2550:40(TAB)829:1(TAB)391:2(TAB)385:2(TAB)564:50

With the tabs it should look something like: although i can’t press tab on here, im doing 3 spaces, to represent a tab. SO DONT GET CONFUSED. Also remember, the 1:1 at the begging is, 1 being the NPC code, and the :1 is being how many kills it takes to drop something constantly I believe. And the space between (1:1) and 995:100 should be MUCH greater than the space between (995:100) and (554:100) although it is still a tab, it separates it farther than the rest. So keep that in mind

#Man drops 1:1 995:100 554:100 4151:100 #Guard drops 9:1 995:200 555:100 #general graardor 2550:40 829:1 391:2 385:2 564:50

Hope I helped. Goodluck!
