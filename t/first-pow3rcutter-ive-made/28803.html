A WC script, good choice, my first script was a WC script, this was in Kaitnieks time so we didn’t have SRL then.

Anyway, a few suggestions, if you want to chop a specific tree, instead of checking the uptext for chop, make a bitmap(MAKE SURE YOU PUT BLACK WHERE IT’S NEEDED*) of the tree and use something like “if(FindBitmapIn(…))then”, and use the bitmap in of the top-right corner, make sure it has space to find it so be generous with the space you allow.

Instead of FindColor(**), you could use a FindObj, witch basically checks the UpText.

  • = Black is a wildcard in bitmaps, in other words SCAR won’t search for black in the bitmap, this allows text-finding.
    ** = Although this is against my first suggestion, it’s useful when there are say only plain trees, or only oaks, or yews around so it won’t chop another tree by accident. You can mod a FindObj to use bitmaps if you would like.
