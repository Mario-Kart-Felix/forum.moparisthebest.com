Protosstribe August 4, 2016, 3:20pm #5

vortex: Yes, it’s a virez that will destroy your GPU. Edit: By the way, what graphics card do you have? Do you have OpenCL 1.1 or later?

Taharok & Eyeownyew: I’m using a library called Aparapi. It compiles the Java bytecode in the run() method and all run-reachable methods (as they call it) on the fly, to C99 code, which OpenCL is using. So there are quite a few limitations to how you can write your Java code.

Edit:

I made some changes to how you build the project and fixed some issues with running it. I had not tested to clone, build and run before. But now I have. It didn’t work, but now it does (at least for me).

I also updated your starting position in the scene, so you should hopefully see spheres when you start it, unlike before, when you had to walk around to find them.

A few new controls have been added. Press ESC to exit (it may require a few presses if you’re running in JTP (Java Thread Pool) execution mode, as it’s pretty unresponsive then). Press E to display your current execution mode to the standard output. They can be GPU or JTP (Java Thread Pool) at this time. Press T to toggle between execution modes.
