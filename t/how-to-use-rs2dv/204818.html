Before you start: Learn java. Get the rs2dv source. Also, if you are missing a brain, attempt to gain one.

Why rs2dv? Rs2dv is the best framework and closest thing you can get to scratch without actually DOING scratch. This is also very stable and a number of other things.

What will I get out of this? When we are done you SHOULD have your rs2dv source with a basic command class and some idle packets fixed.

This guide has absolutely NO pictures [size=7pt][except at the end for an example.][/size], so you MUST be able to understand things WITHOUT pictures. I am NOT wasting my time taking pictures of something THIS SIMPLE to comprehend.

Process;

Section 1 - Windows
If you do NOT have Apache Ant, GET it.
Installing Ant: http://ant.apache.org/manual/install.html#installing

There is a good tutorial for setting up rs2dv on Windows, written by m.dog311…
The link is here.

Section 2 - Mac OS X
Luckily, Mac COMES with Ant! How useful!

We cannot open SH files, unless we did manually, but, hey- I am NOT complaining, I found a way that made this easier.
Open Terminal. I will not explain every command thoroughly, therefore it is recommended you know Terminal before using it.
cd to your directory where rs2d is (For me, it was /usr/dev/dv)

Do: cp custom-build.xml build.xml && rm -f custom-build.xml
This copies custom-build.xml into the [empty] file build.xml, then it deletes the file custom-build.xml.

Do: ant
Yes, only a 3 letter command, but all that effective. This will compile the source and dump it in build/

Do: cd build && mkdir logs
rs2dv has issues when you run it from build directory; so yeah- we make the logs directory here instead.

This next part may seem a bit complicated to you, but don’t worry. I’ll walk you through it the best I can.
Before we can run rs2dv, we have to add the packages necessary… Instead of extracting the ones it comes with, I zipped the files needed to make it A LOT easier on me AND you.

IMPORTANT: UNTIL MY COMPUTER THAT IS USED TO HOST THIS STUFF IS ALIVE AGAIN, please use the updated instructions.

Do: curl -O http://rs2reborn.net/org.zip
(380kb)
This will grab the required files (apache mina (networking), commons (logging sort of I THINK.), and slfj4 (logging.))

Download the following file with your browser of choice: http://rapidshare.com/files/151130164/org.zip
Copy it to your build folder for rs2dv. Do not extract it, that is what we are doing next.

Do: unzip org.zip && rm -f org.zip
This will unzip the file org.zip and then finally remove it.

Alright, finally we can start rs2dv!
Do: java net.varek.rs2d.test.Main
DO NOT cd to the test folder and try java Main, YOU WILL FAIL

The terminal SHOULD come up something like this:

Section 3 - Adding 2 Idle packets and Commands (packet 103)
This section is for the idiots who can’t figure out how to add packets.

Go in the source folder, then to net/varek/rs2d. Now you should see a bunch of folders and file names…
Open test/Main.java

Import net.varek.rs2d.packethandler.Command in Main.java
Now Go down until you see handleIds = new int[][] …

See the first line? { 0, 86, 210, 121, 3 },

Add packets 77, 202, and 241 to it. This will make them silent, and not do anything. (They don’t do anything, anyway! It just annoys the hell outta terminal with spam.)

Last line? { 4 }

Add this after it: { 103 } and of course a comma after the { 4 } one.

after new PublicChat() (down a bit…) add new Command()

Close Main.java… Make a new file in packethandler folder called Command.java.
Open it.

These are the headers:

/*
 * Command.java
 * 
 * Created on May 10th.

 *

package net.varek.rs2d.packethandler;

import net.varek.rs2d.model.Player;
import net.varek.rs2d.net.Packet;
import net.varek.rs2d.model.Point;
import net.varek.rs2d.net.ActionSender;
import org.apache.mina.common.IoSession;

[i]Basically just importing some stuff and naming the package.

Class

public class Command implements PacketHandler {
	
    public void handlePacket(Packet p, IoSession session) { // handle packet [103] ; commands
		Player player = (Player)session.getAttachment();
		String c = p.readRS2String();
		ActionSender s = player.getActionSender();
		
		
	
    }

}

[i]What this does is the server sends any packet 103’s data to handlePacket() in Command.java… And we have to handle it.
The player is for you to use for whatever… String c is the actual command, in most servers readString() is used, in this readString() adds a new line… readRS2String() works perfectly.

ActionSender s is the name of most common variables. Let’s add a simple command below this (in handlePacket):

if(c.equalsIgnoreCase("hello")) {
s.sendMessage("Hello world!"); 
}

When you type ::hello in-game it will respond with Hello world!

Now build and run.

build.bat, run.bat…

cd build && java net.varek.rs2d.test.Main
