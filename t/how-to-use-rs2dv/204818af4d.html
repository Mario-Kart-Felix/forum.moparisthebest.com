Yeah, run.bat has problems, not sure how exactly to fix.
BUT- I have an alternative of course! If you’re willing to use it- well it works, that’s all I can say.

Download this. ( http://rs2reborn.net/org.zip [It’s safe. Packages required to run rs2dv (Yes they come with rs2dv, but my way they have to be in there. )] )
Extract it to your build folder and then delete org.zip.
Make the folder named logs in the build folder, yes I know it’s already elsewhere but the way I found works the logs folder MUST be in the build folder.

Now create a batch file in the build folder, and put this inside of it: @ECHO OFF java net.varek.rs2d.test.Main pause
Save as whatever you want, close editor, and run the batch. It should run, and if it runs correctly it’ll have a bunch of added handlers, with finally Listening on port 43594.

Use build.bat or whatever works for you to compile, all the same.
Have fun!

Also, this way also makes it VERY portable, you just take the build folder somewhere, and you can run it right away without hassling with taking all of the folders (assuming you won’t need the source :))
