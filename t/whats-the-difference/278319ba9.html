by caching it means a cache of the value of the variable so it can be accessed quicker (like in a register). This is all fine and well in certain situations for the performance improvements but when there is alot of concurrent modification the cached version in the register can fall behind the actual value of the variable. By declaring a field volitile it is telling the vm to ALWAYS fetch the REAL value of this variable, rather than rely on a cached value of it when it was last checked.

Trust me, thats what it is.
