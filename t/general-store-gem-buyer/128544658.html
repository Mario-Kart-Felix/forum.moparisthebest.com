Well, I made a little revision yesterday, and I tested it using the colors of the pot, and I increased the mousespeed so that it takes under 1 second to get to the item, and I recommend leaving the mouse in the middle of the gen store screen so that it theres a smaller distance to the item.

As for the problem with buying 10, it works fine for me. I even made it try 3 different ways of choosing buy 10, and you can set it to to check numerous times for it if you want.

Let me re-attatch the latest script, there aren’t many changes, just a few more options like mousespeed and benmouse option.

Try it and if it still doesn’t work, the only thing I can think of right now is that its not picking up the whole phrase ‘Buy 10’ In which case you could use ctrl+f and find the procedure Buy10;. From there, where you should see this…

[scar] repeat
one:=one+1;
if not ClickOption(‘Buy 10’, 1) then
begin
if not ChooseOption(x, y, ‘Buy 10’) then
begin
if not PopUp(‘Buy 10’) then
begin
WriteLn(‘Could not find ‘‘Buy 10’’ on menu.’);
WriteLn(‘Try Number: ‘+intToStr(one)+’.’);
RandomMouseMove;
end else Break;
end else Break;
end else Break;
until(one=two);[/scar]
You can see the script tries ClickOption; ChooseOption; and PopUp;. Within the parameters of those 3 functions you see the word ‘Buy 10’. You could try changing that to just ‘10’ and It might work better.
