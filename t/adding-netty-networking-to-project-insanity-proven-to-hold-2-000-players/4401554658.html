Does it matter? We’re talking about servers with 50 people. Not 2500503945809452.[/quote]then why are you using netty in the first place, considering the exact job of netty is to allow you to have 2500503945809452 different people/connections. if you’ve ever delved into the beautiful code that is the connection handling in the majority of netty/MINA based servers you will find that it’s almost always state machine based, and the logic is always cooped up into tiny blocks (states) which constantly require you to jump around the code if you want to understand a certain bit. this convolution is a fucking pain in the arse. modelling the server through actors however would make your code easier to follow and you can model the game world much more naturally (process per entity, or whatever), so it’ll be easier to write a server and much easier to read the code of one.

case study: winterlove. notice how the code for the login block is much easier to follow. the general linearity of the process as expressed in pseudocode is:

void login() { int someData = read(socket); send(socket, randomData); long moreData = readLong(socket); send(socket, 12435); String username = readString(socket); String password = readString(socket); send(socket, "OK we got ur pass!!"); }
where everything is very linear, very concise. you can deduce exactly how it works just by reading this.

meanwhile using Netty/MINA will end up in a state machine (more pseudocode):

[code=java]
enum LoginState {
BEGIN,
READ_SOME_DATA,
READ_MORE_DATA,
READ_USER_AND_PASS
};

LoginState state = LoginState.BEGIN;

void updateLogin() {
case (state) {
LoginState.BEGIN:
int someData = read(socket);
send(socket, randomData);
state = LoginState.SENT_SOME_DATA;
break;
LoginState.READ_SOME_DATA:
long moreData = readLong(socket);
send(socket, 12435);
state = LoginState.READ_MORE_DATA;
break;
LoginState.READ_MORE_DATA:
String username = readString(socket);
String password = readString(socket);
send(socket, “OK we got ur pass!!”);
state = LoginState.READ_USER_AND_PASS;
break;
LoginState.READ_USER_AND_PASS:
// player is now logged in.
doSomething();
}
}[/code]
what a pile of huge fucking hack. using actors will allow the former linear style to be used, with the performance and scalability of the latter as well as some big extra benefits to make your life easier.

so yes, it does matter, if you want to have a fast AND comprehensible emulator.
