system July 31, 2016, 1:25am #20

adu: It is not really that hard…

1). Normally you have to examine the client for its outgoing packets (which packets; according to their ID, when; what action that sends them and how; what ‘data-types’ they use)…

2). Then you need to make a new handler for them (a ‘PacketHandler’ in this case), that reads the proper values in the correct order (referred to as ‘how;…’ in the text above).

3). In the last and final step (before testing), you should make the server reference the handlers for the packets somehow. In most RSC private servers they use an XML file (usually called ‘PacketHandlers.xml’). In this file you should write the fully-qualified package-name (the package-name + class-name put together, and separated with a . (dot)), aswell as the ID (or IDs) of the packets you wish to reference that packet-handler with, together, and follow the used format (there should be others written, so just follow their format).

Note: The first step is not needed anymore (unless you really wish to learn more than C&P), because the ‘protocol’ has already been almost fully documented (the documentation can be found in one of the stickies; at an external site). What I mentioned was more just how you do it in general… A. k. a. ‘reverse-engineering’ (even though reverse-engineering usually involves alot more than that).

I hope that explained enough for you to understand… Good luck!
