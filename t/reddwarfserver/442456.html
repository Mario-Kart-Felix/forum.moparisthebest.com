RedDwarf Server is a horizontally scalable application server for low latency applications such as online games, virtual worlds, and social networking applications. It is the official community fork for Project Darkstar, previously sponsored by Sun Microsystems, Inc. RedDwarf is a server side technology written in Java that radically simplifies the development of online games and lowers the barrier to entry into the MMOG market.

Why are you forking Project Darkstar?

Sun’s (now Oracle’s) investment in Project Darkstar was discontinued immediately upon the close of the Oracle-Sun acquisition. This left Project Darkstar in an awkward, limbo state where contributions, development, and releases were controlled by Oracle, but there were no resources devoted to managing this effort. Rather than leaving Project Darkstar to whither and splinter off in many directions, we have organized a centralized community fork to continue development where Oracle left off.

What are the goals for RedDwarf?

From a technical standpoint, RedDwarf’s objectives remain similar to that laid out for Project Darkstar. The “version 1.0” milestone will be a complete multi-node implementation that provides true transparent, horizontal scalability to support large virtual worlds, thousands of simultaneous connections, and intelligent load balancing.

What’s “different” about RedDwarf as opposed to other network game frameworks?

Unlike most network game frameworks, RedDwarf is more than just an infrastructure for communications. It is a completely multi-threaded, event-driven transactional system that provides automatic data persistence. Much like other Java application containers, RedDwarf maintains data consistency and integrity for your server side application state while providing a persistent database backend that protects against server crashes and network failures. However, unlike most application containers (such as J2EE containers), RedDwarf is engineered and tuned to ensure low-latency response time, a must for games.

How do I build a game with RedDwarf?

RedDwarf is a server side technology. Your server side game state is completely managed and executed in the RedDwarf environment when you develop against the RedDwarf server API and deploy and boot up your game in the RedDwarf server. The server must be written completely in Java. On the client side, however, RedDwarf is completely language agnostic. There are a number of available client APIs that provide the communications endpoint to communicate with a RedDwarf server. RedDwarf does not provide any other client side tools (such as graphics engines, physics engines, etc), but it is flexible enough to be integrated with just about anything that you want.

What license is RedDwarf available under?

RedDwarf is continuing with an identical licensing scheme as Project Darkstar. RedDwarf server is distributed under GPLv2 with the server APIs being made specially available under GPLv2 + classpath exception. The Java client and the C client APIs available as part of the RedDwarf project are distributed under a BSD license.
