Believe it or not, runescape doesn’t keep the open door on the same tile as the closed door, or vice versa. I figured this out when I was working on doors for my POH.

The proper way to make doors work is like so:
Get the door’s object id, and position (x, y, z).
If the door is already open, the id it requires should be is 1 less than the door’s ID. If the door is closed, it should need the ID that’s 1 above the door’s ID. However, in some special cases this isn’t true.
Once a player clicks a door, you move it 1 tile off (This depends on what way the door opens/closes), and then replace it with the new ID from above.
