junior July 30, 2016, 12:13pm #2

That means that there is a procedure, function, or even a variable named “HeadColor” that is used twice in the script. If you dont see it twice in the script, that variable, procedure, function, or whatever it is is already being used by one of your includes that you have included, and im guessing its WoodCutting.SCAR. Look through the script and try deleting it, blank it out, or name it something different… But if you name it something different, all the original identifiers named “HeadColor” will have to be changed to what it was changed too.
ex: Lets say you change the identifier to “HeadColor1”, all the other “HeadColor” identifiers that are in that current script will have to be changed to “HeadColor1”.

EDIT: And dont think about running a script all night on a single player, gather up an army instead. Its much safer and you will get better results… If your combat levels on each account are good enough, not level 3’s.
