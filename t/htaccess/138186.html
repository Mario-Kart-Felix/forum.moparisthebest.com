HTACCESS
HTACCESS is a file that sits in the root of a directory, and Handles Things to do with:

[ul][li]Site Files[/li]
[li]Error Documents[/li]
[li]Access[/li]
[li]Bot Index Controls[/li]
[li]And More.[/li][/ul]

Some Basics

[ul][li]Only One ‘Command’ Per Line[/li]
[li]Not All Hosts Allow[/li]
[li]Its FULL name is .HTACCESS , always.[/li][/ul]

Changing Error Documents Through HTACCESS
First, You need the htaccess file, where allowed. Some hosts dont allow HTACCESS, simply due to the control it can give you.
To Change an error Document, use this format

ErrorDocument ERRORNO /PATH/DOCUMENT.EXTENSION

so for a custom error 404 document its:

ErrorDocument 404 /errors/404.html

Redirecting
To redirect, theres one simple line to add, Redirect /olddirectory/oldfile.html http://newsite.com/newdirect
So, To stop people going to http://yoursite.com, and make them go to http://www.yoursite.com (for Google PageRank or Preffered URL.) use Redirect http://yoursite.com http://www.yoursite.com/

Indexing (Enabling/Disabling, Cloaking)
With indexing you can, Hide certain file types, certain file names, or Even Every File.
To Stop All files being indexed, Use Options -Indexes or to Make files all indexed Options +Indexes (some hosts disable indexing by default.)
To Hide a certain type of File: IndexIgnore *.TYPE, and to stop indexing a singe file, IndexIgnore NAME.EXTENSION

Default Page shown
When You type yoursite.com/directory , the first thing to be shown is normally index.html or .php, To change this use DirectoryIndex FILENAME.EXTENSION

Password Encrypting Directories
Use This tool http://www.kxs.net/support/htaccess_pw.html to encrypt passwords.

First, In Your HTACCESS File, put:[code]AuthUserFile /.htpasswd location
AuthGroupFile /DIRECTORY TO PROTECT
AuthName NAME OF AREA TO BE PROTECTED
AuthType Basic

require valid-user[/code]
In a file called .HTPASSWD, add USERNAME:ENCRYPTEDPASS
