Vain_ beat me to it, but the best I’ve used yet for designing GUI applications is the Window Builder add-on to Eclipse. It’s ridiculously easy to use, it actually parses your code so you can change the code directly and it won’t break the visual forms, it hasn’t messed up for me yet in my hundreds of hours of using it, and it works very effectively for the entire Swing package. I’m not sure if I have ever been more pleased about any other piece of software than I am with this one.

I would also recommend you to never program GUIs by hand if you have an option to use an editor. There’s only so much you can do without visualizing it, not to mention there are things designed in Swing specifically for WYSIWYG editors, such as GroupLayout which is really complex to code with by hand, but is very simple to use with a builder. Mind you, I’ve found that almost every problem in GUI design can be solved with a GroupLayout. There are a few cases of misalignment that are easier solved with some grid-based layouts.

Hope that helps.
