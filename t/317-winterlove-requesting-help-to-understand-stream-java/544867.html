It has nothing to do with java, it has to do with networking.
The stream class methods are used to create frames and packets so we can communicate over the network.
http://en.wikipedia.org/wiki/OSI_model

RSPS related, every client will have a connection with the server.
The communication between those happens via network streams, so every player will both client and server sided have a stream
that reads or writes data.

When we e.g. use stream.writeByte(); we send a byte to the client/server.
stream.createFrame(26) will obviously created a frame.
And when we combine those we can create packets.
again: http://en.wikipedia.org/wiki/OSI_model

there’s nothing more you really need to know about a buffer… its basically a container for data of a specific primitive type, in this case bytes. for example, if you need to mail something to your friend, you package it and ship it off. then your friend can ship something back to you.

same thing for rsps, you put whatever data needs to be sent to the client inside a buffer and send it off to the client using the socket channel (outgoing packets: sending a message, sending hint icons, opening an interface, etc.)

the client can then send data back the same way (incoming packets: equipping an item, picking up an item, clicking an object, clicking an npc, etc.)
