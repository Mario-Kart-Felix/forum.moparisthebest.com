Edit: I wrote one really quick that only supports one login/password. This shouldn't be used to protect mutiple pages etc as it doesn't have any PHP sessions or cookie support. It should work as I finally tested it a second ago. If you have any questions about it please let me know :).

<?php
//Code by http://www.RSHumor.Net
//Do not remove these lines
//This *should work*. However was programmed at midnight in about 4 minutes.
//Any questions please contact RSHumor.Net
//Username they are supposed to use
$validusername = 'USERNAME';
//Password they are supposed to use
$validpassword = 'PASSWORD';
if ($_POST) {
	$username = addslashes($_POST['username']);
	$password = addslashes($_POST['password']);
	//Submitted the login form
	if ($username == $validusername && $password == $validpassword) {
	//They logged in
	echo "Page here? / Maybe navigation to other pages?";
	} else {
		echo "Login failed!";
		form();
	}
} else {
	form();
}
function form()
{
	echo"<form method=\"POST\" action=\"filename.php\">
    Username:<input type=\"text\" name=\"username\" size=\"20\"><br>
  Password:<input type=\"text\" name=\"password\" size=\"20\"><br>
  <input type=\"submit\" value=\"Submit\" name=\"B1\"><input type=\"reset\" value=\"Reset\" name=\"B2\"></p>
</form>";
}
?>

BobDole81 July 29, 2016, 7:58pm #4

MySQL is a type of database server, which, in this situation, would be used to store the usernames, passwords, and other details of all of the users that would have to login with this script.

As this is a pretty basic script, it would only be used for a single page, but as Rawrness said, you could add session and cookie support to it for it to be secure for several pages.

If you don't have any knowledge about php (and don't want to use one of the countless premade scripts out there), I would suggest checking out apache authentication (assuming your server is running apache). It's a much easier, and pretty nice, method to password protect an entire website or even just specific files.
