system July 30, 2016, 4:43pm #13

Not quite everything else, pizza pie. If you are declaring a constant, you usually first of all use “private static final” or any other modifier (where modifier means “private” in this case (could be “public” and “protected” aswell)). Then when naming it, you capitalize all letters in it, and you also must initialize its value. A constant should be constant, and not be changable.

Here is an example:

private static final String LOCAL_INTERNET_PROTOCOL = "127.0.0.1";

As you can see, I also put underscores between each and every word in the name, that is also mentioned in the Java naming conventions.

There is another thing I can point out in your source code, -Chaos-. You don’t really need the variable named “int3”.

Now, just do it like this instead:

System.out.println("Your answer is " + (int1 + int2));

It does take up less memory, since the variable “int3” would create a new reference within the memory, pointing towards both “int1” and “int2” as one summation.
