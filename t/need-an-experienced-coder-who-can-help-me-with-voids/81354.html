I don’t like the word “voids” by definition, Void means a vast open space filled by nothing, which is what most Voids are nowadays :(. They are actually methods that anything at any given time can call upon it to do THIS.

Void’s are usually used to call upon certain small amounts of information not huge chunks that would usually be in a Handler. (if you don’t shove everything in Client.JAVA you’re going to be much better off in the long run). Here’s a basic set up for a method with a switch method in it.

public void test(int ke) { switch(ke) { case 1: sendMessage("Congrats, you have just learned how a switch\nType works, now to move on to more complicated things."); break; case 2: sendMessage("Switch with an Integer in the brackets can be used to call upon a selection of a method rather than the whole method"); break; case 3: sendMessage("For example, after this method I will show you how to call upon it with another method."); break } } public void calltest() { test(1); sendMessage("You have just called upon switchID 1 of method Test."); }
