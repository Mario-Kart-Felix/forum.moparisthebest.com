Neither of these will be an issue if the leaders of the development are mature enough. For example, I would plan on having a founder/collaborator, someone who merely runs the project, a community manager if you will. An unbiased person who just focuses on making sure progress is happening and assigns all developers their respective tasks to stop overlapping, this person doesn’t have to have programming knowledge, regardless of language choices.

I would then have 3 possibly 4 very respected, very active, focused, mature Senior Developers. These developers will have extensive knowledge of the Java language, must be able to work smoothly in a team no matter who the other members are and must have the desire to leave something behind to the other members of the Mopar communities when they leave. These developers would preferably be badged Java Gurus. These developers also control the GitHub account, all requests must run past this team, even the Junior Developers must run their ideas and what they’re programming into the base past at least one member of this team.

I would then have 6 possibly 7 respected, focused and mature Junior Developers. These developers must also have extensive knowledge of Java and must be able to work smoothly with co-workers, no matter their differences. I realise all programmers have different skills and traits, but there must be certain things that will be agreed on, no matter the developer. These developers must have the desire to achieve and learn more and more within the Java language, as we all know, there’s always room for improvement. These members must be well-known around the MoparScape community and will be focused solely around this project.

After that, I don’t believe we need any other named programmers, the rest would be the community who would want to leave their piece of their code behind, and leave a part of them with this base for the future MoparScapers to use and see. This project, if it goes ahead, will be around MoparScape for years, this will be the first fully developed MoparScape source and we’d be the proud developers and collaborators of this project.
