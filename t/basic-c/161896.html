zstars July 30, 2016, 5:25pm #17

Don’t use char pointers unless you have a good reason. Use std::string.

C-strings, pointers, C-functions, C-Casts, all these are evil and should be avoided.

In short:
C++ > C

If there is a C++ way, use it.

Anyway, your code has indeed quite a few errors, and so does the code of quite a few people who have responded.

-strcmp() does indeed return 0 if the two strings match (And you should not use !, because it does not have a LOGICAL_NOT meaning)

-Your char buffers are only 4 characters long, and you do no checks at all, which means that they are extremelly likely to overflow, and lead to unexpected behaviour (most likely crashes).

-Someone who responded uses uninitialized char pointers as if they were buffers. That of course is wrong, as a char pointer is exactly that, a pointer to a char. NOT a char, NOT a char array.

-Note also that, that re-making a standard library function is not a good idea, especially if you change its actual behaviour. The real strcmp does a lexicographical compare. Also, if you really wanted to, you should do it right. strcmp() does not modify the char buffers it takes as parameters, so they should be const (char const *).
