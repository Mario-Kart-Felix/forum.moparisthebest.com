moparisthebest August 1, 2016, 3:18pm #15

There is no downside, but if they have javascript disabled, it will be transmitted to your server in plain-text anyway (you can’t do anything about that, except HTTPS), so you have to be able to handle that if it happens. (you can check if it’s a hash or not with the length)

A quick google shows me this:
http://jssha.sourceforge.net/

But of course you don’t have to just do a single hash, you could sha-1 the password with the username as a salt in the browser, then perform your sha-512 on it once it reaches the server, for a complete hash that looks something like this:

hash = sha512(512salt+sha1(username+password));

Either way is good.

Also, think about HTTPS, I pay $10-$15 per year per domain for cheap but well supported SSL certificates, and it’s very easy to set up server-side.
