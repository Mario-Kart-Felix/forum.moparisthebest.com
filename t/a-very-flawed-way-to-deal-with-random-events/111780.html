A very flawed way to deal with random events

General Discussion
Brazen_Bull July 30, 2016, 11:03am #1

I realise this is a wall of text, but I promise this eventually leads to a very unique idea. Also, I should mention I haven’t played the game for a long time, so there’s gaps in my knowledge.

We all know it would be useful if we could get humans to respond to random events for us. My first thoughts were that a sleepwalker system (the one where we used to type out each other’s security words for accounts with 100% fatigue) would be cool, but impossible for security reasons. For someone to address random events for you, you’d have to give them almost complete control over your account, which they could abuse somehow.

But what if we instead made use of all the abusers and thieves?

If a random event completely disables an account, then to steal your items, he would have to address the random event for you. Now what if you have no items worth stealing? Couldn’t, in theory at least, giving your password to hackers be a useful way to deal with the random event, assuming that you can (A) manage to recover the account, and (B) manage to return to whatever you were originally automating?

Obviously this isn’t a realistic solution, but it does have a lot going for it in theory.

What makes this idea almost feasible is the predictability of hackers; they will always take you to the bank. Ofcourse you will run into the occasional jerk who, after visiting the bank, takes your account on a joy ride then abandons it in a cabbage patch, but then you simply need to give the password away again.

Since there’s a finite number of banks, and which bank you are at can be identified using the minimap, you can simply have a scripted response to being left in each of the different banks. This might sound like a daunting task, but with a bit of ingenuity, the difficulty could be reduce. Here’s a short story illustrating one idea for a solution:

  1. Script detects you are in the varrock bank. It then walks you to a point that players can teleport to.
  2. Script logs in a second helper account containing runes. This helper account teleports to the where the main account is located. If this helper account encounteres a random event, then the script logs out, and switches to another helper account in a list of available helper accounts.
  3. Script transfers a few runes from the helper account to the main account. The main account teleports to a location from which it can more easily return to work.

This also has problems, for instance it requires the accounts to have a certain magic level, but the point is that there could be a solution out there.

Also, you’d need to address the issue of how to advertise your password. On the one hand, spamming IRC would stop jagex from proving that you’re giving away your account, but on the other hand, this would be harder to implement, and wouldn’t scale well for a large number of cheaters. Simply advertising the account in game would be a very easy solution, but this would draw Jagex’s attention to you.

I can’t comment on how you reclaim the account. I’m kind of assuming it’s as easy as filling out some recovery questions, as it was in my days.

The idea of manipulating the average punny RS thief appeals to me. Who knew these minebags could actually serve some useful purpose - even if they’re totally unaware of it?

Post your thoughts.
