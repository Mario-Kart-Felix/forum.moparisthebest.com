So I was bored today and figured I should do a basic/slimmed down version of RS’s Grand Exchange.

This uses MySQL to store all the items being sold, and any completed offers waiting for the buyer to get the GP.

Of course it isn’t fully featured like RS’s, nor does mine control the prices (you throw them on the GE for whatever price you want). Someday if I do a lot more work on it I may make it so it controls the prices.
If I do make a new version after this, it will use interfaces no doubt. Right now it just uses command and a web interface for viewing what is on the market.

Only items that are sellable on the REAL RUNESCAPE GE are sellable on here as well. It’s pretty simple.

At the moment it’s pretty basic. Three commands are used:
::collect (used by the seller to collect GP when someone buys one of their items)
::sell (sell an item. Syntax: ::sell ITEMID AMOUNT PRICE)
::buy (buy an item. Syntax: ::buy OFFERID AMOUNT. You can find the ID of the offer on the site)

Picture (doesn’t show much)

If the following isn’t allowed, then oh well, go ahead and remove it. I don’t see why not, there is no advertising to other pages / the root directory is a blank page pretty much. So this is in no way advertising another community/forum.
A demo can be found here: http://24.137.82.221/ge/

It just shows the items currently on the GE, and of course would show any new items if someone sold one and remove any items if someone bought one.

The login boxes on the left aren’t implemented yet but in any future updates it will be used to log in (for sellers, in the future buyers if you put in a buying offer. Right now is just selling offers) and track your offers, if it sold, etc.

I do not plan on releasing this. I mainly made this to show how easy a system like this is to create. Plus I have never seen a GE using MySQL yet, so why not?
