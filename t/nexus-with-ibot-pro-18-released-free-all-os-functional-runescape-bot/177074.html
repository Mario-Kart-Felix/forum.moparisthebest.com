We currently have 1 platform dependency issue. Apple has released only a beta Java 6 64 bit JVM, no 32 bit one.
SWT for mac is only 32 bit right now. Apple is behind on Java compatibility.

My current options i belive are:
Tell mac users to try this:

Compile nexus in a Java 5 version that Apple has and instruct mac users to instead install a old Apple Java 5 JVM.

Frankly, it makes me hate Apple.

The un-intended benifits from using SWT have been:
All gui data structures can only be modified from one thread. May not seem like a benifit but it is, i can assume many of data structures i have will be only modified from one thread. Operations in the bot are guarnteed to be serial not parallel. I moved most of the nexus level api into the gui thread which other threads pile jobs on to.

Original reason for chooing SWT

[ul][li]lower cpu/ram usuage[/li][li]easier for the users to understand the gui, familar look and feel[/li][li]More out-of-box widgets to use, less buggy advance widgets[/li][li]smaller widget size so i can fit more widgets on the screen.[/li][/ul]
