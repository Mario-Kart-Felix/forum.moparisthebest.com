array(‘username’=>‘username1’, ‘password’=>‘password1’),
array(‘username’=>‘username2’, ‘password’=>‘password2’),

);

if ( isset($_POST[‘username’], $_POST[‘password’]) ) {

		$_SESSION['username'] = $user['username'];

	}

	echo 'Wrong username or password.';

';

Unhandled July 31, 2016, 12:27pm #20
eczema3:

IIRC, sessions in PHP are stored server side, not set as client cookies.

PHP Documentation:

A visitor accessing your web site is assigned a unique id, the so-called session id. This is either stored in a cookie on the user side or is propagated in the URL.

Yes, the session data itself is stored on the server and is referenced by the user’s session id, which is most commonly stored in a cookie.
