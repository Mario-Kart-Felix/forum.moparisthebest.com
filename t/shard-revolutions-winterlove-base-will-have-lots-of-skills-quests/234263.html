About the server

Basically, I want to write a server that is stable, secure, fast and reliable. Many servers are like this, but most of you find them hard to use (e.g. RS2D) and they have few features.

So, I’ve decided to write a Winterlove-based server, apply all the stability fixes I can throw at it, and come up with something decent for you all, with some quests and skills while I’m at it!

I’m not going to put a stupid percent meter or anything, since how do you judge it? I’m just going to post the updates as I make them, maybe add a few screenshots every now and again, etc.

This is going to be open source, but I haven’t added it to the ‘Downloads’ section since it is far from complete. Maybe when it is a mod could move the topic or I will just make another one.

Oh, and if someone would make a logo that would be nice cos I’m a programmer, not a graphics designer.

More technical details of it

It’s going to use my EventManager instead of hundreds of process() timers, so it should run very smoothly.

It has SYIpkpker protection code (3 connections per host), a huge Client object is not made on connection (less RAM usage) and is non threaded. This should make a decent impact on performance, especially on large servers.

I’m adding javadoc comments as much as possible, so when I do generate doc files it should be really easy to find out how it all works and fits together!

It is packaged, and it does not come with a build/run files. I didn’t add a build/run batch files since the packaging may change when I refactor anything, and I don’t want to mess around with adding them. Maybe when I do a release build I will add them, but for now, I reccomend you use a Java IDE (like Eclipse).

Alpha 7 URL (for the current stable source - reccomended!):

https://www.grahamedgecombe.com/svn/...ns/tags/alpha7

The username is shardrevolutions. There is no password. This will give you read only access to the repository.

(If you think that link is advertising then you are an idiot. My website contains nothing about private servers, apart from the Subversion for this project. If an admin/mod wants me to remove the link then I happily will, but I believe that people should be able to checkout the current source if they want.)

A good subversion client to use is TortoiseSVN (http://tortoisesvn.net/). Once you have downloaded it, create a new folder for this, right-click it and select SVN Checkout. Enter the URL. It will then prompt you for the username and password (which are above), and probably give an error about an invalid certificate. This is ok, as the certificate is self signed. Now you should have the source code! When you want to update the source code to the latest version, just right click -> TortoiseSVN -> SVN Update.

I will not be providing a virus scan. If you think a virus scanner can detect a virus in a source file, you a very wrong and I suggest you go and learn about how they work.
