lothy August 3, 2016, 2:57pm #2

You would have to profile that. Using JNI adds some overhead so if you do go down that path then you should reserve native calls for computationally expensive operations only.

Have you considered looking at C/C++ and OpenCV? I don’t know why your running time for such an operation is so high, but obviously 1/3rd of a second for the calculation of one point is not going to work.
