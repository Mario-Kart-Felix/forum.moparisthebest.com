IYAOYAS August 4, 2016, 11:45am #222

The action lock system is modeled after the concurrent locks design but does not have the same contract as a concurrent lock. The similar aspect is that it latches onto an entities state which acts as the “resource”. A lock can either latch onto the state in a restrictive manner where it does not let the state be utilized by other interested parties or it can be a passive lock where it allows other parties to latch onto a resource in parallel (not in a concurrent context!). I chose to go this route instead of employing a bulky action/event listener pattern because it is less taxing and provides a greater measure of flexibility.
