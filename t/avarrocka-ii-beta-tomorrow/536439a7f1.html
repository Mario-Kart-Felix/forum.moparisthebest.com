IYAOYAS August 4, 2016, 11:50am #268

I’ve been working on a low level scripting language whose construct is based on Java’s labeled code blocks since before I joined Avarrocka. A variant of this had been under development for about a week before I joined Avarrocka, but was, and still (slowly) is being developed for the RuneLocus starter pack which will be released totally open source. I still have a lot of testing to do in regards to the interpreter and full scale functionality, etc… but unfortunately with the deadlines we have recent progress has been slow at best so it will be a while before we start using this for actual features. Here is an example of an “Avarrocka Script” (based on RL script) at the present moment:

Multiple line code comment:

[code][#

This is a multiple line code comment in Avarrocka Script. You can put really

Whatever you want on however many lines you want so long as you stay within the

square brackets. A code comment can be started with an open square bracket followed

by a ‘#’ symbol and ended in the reverse fashion.

Everything after the the comment is opened is totally ignored because the only thing the
syntax interpreter is looking for at this point is “#]” meaning it is not mandatory for you to
prefix each line with the ‘#’ symbol but is conventionally correct (conventions outlined
in the README text document).

A single line code comment can be initiated with the ‘#’ symbol or two forward slashes.
Conventionally speaking the ‘#’ symbol is preferred for this language but not mandatory.

#][/code]

Example of an actual script:

[CODE][#

Prints “Hello” to the command line but does not start a new line and then

flags the interpreter to start parsing the world_block.

#]
hello_block: {
# all of the PrintStream methods are included by default in scripts
print("Hello ")

# notice how we skip right over the exclamation_block and go right to the world_block
goto world_block

}

[#

Prints the exclamation point at the end of the “Hello World” statement

and then halts execution of this script.

#]
exclamation_block: {
println("!")

# halts execution of the entire script!
end

}

[#

Prints “World” to the command line and flags the interpreter to start

parsing the exclamation_block.

#]
world_block: {
print(“World”)

# we can use goto to parse the script in a backwards fashion
goto exclamation_block

}[/CODE]

The syntax and script interpreters are separate but linked. When the syntax interpreter finishes its work it flags the script interpreter via a wakeup() method and vise versa when the script interpreter has finished its work. I plan on being able to actually define variables within scripts, (maybe) block constructors (eg: say_something (string: value): {) as well as more higher level functions but things like that wont happen for some time.

Notes: A more elegant, and better thought out programming construct needs to be implemented if I want to work on adding higher level functionality. I thought multiple line code comments were essential to the language because if I decide to release it to the public there needs to be a way for people to license their scripts without tons of single line comments. I will be adding support for Java multiple line code comments in the future but it is not a priority at the present moment.

EDIT: Until AvarrockaScript is finished we decided to use http://www.beanshell.org/ for higher level scripting functions. Once AvarrockaScript is complete we will use AvarrockaScript for lightweight scripting and BeanShell for more demanding content.

I would appreciate input as far as the overall appearance and design of the scripting language.
