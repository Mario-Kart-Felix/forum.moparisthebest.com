Code convention has been established for this project to maintain great looking ‘code’.

[code]/ – CODE CONVENTION --/

Each class should respect this code convention. Each classes must be revised to support this code convention.

  • Beginning Comments:

/*

  • Avarrocka (Client/Server) Development
  • @author information
  • @see if needed
  • @version (update by 1# at each update)

*/

Example:

/**

  • Avarrocka Client Development
  • @author Artem Batutin artembatutin@gmail.com
  • dec 4, 2013 Main.java
  • @see java.lang.Object
  • @version FINAL
    */
  • Code convention:

  1. Class (static) variables
    First the public class variables, then the protected, then package level (no access modifier), and then the private.

  2. Instance variables
    First public, then protected, then package level (no access modifier), and then private.

  3. Constructors

  4. Methods
    grouped by functionality: the goal is to make reading and understanding the code easier.[/code]
