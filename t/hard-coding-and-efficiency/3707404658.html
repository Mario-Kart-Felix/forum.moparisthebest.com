Fat_Cookies August 1, 2016, 1:14am #25

Avoiding hardcoding is rather easy:
You load stuff from files instead of sticking them in the source code and use clear,
Use multi functional methods instead of sticking everything into the same if or switch statement.

In my personal opinion efficient programming isn’t something you learn from a book, it’s just common logic sense most of the time and a basic understanding of what a computer has to process. There are a couple of features in a programming language that you might have to learn like bitshifts, bitwise operators or the “condition ? true : false” thing in java (i forgot the name ) but those are things you can teach yourself.

Take the example of filling 2 arrays with numbers 0 to 9
A Linear approach would be:

        int[] uno = new int[10];
        int[] dos = new int[10];

        for (int i = 0; i < 10; i++) {
            uno[i] = i;
        }
        for (int i = 0; i < 10; i++) {
            dos[i] = i;
        }

A more efficient way is just common sense:

        int[] uno = new int[10];
        int[] dos = new int[10];

        for (int i = 0; i < 10; i++) {
            uno[i] = i;
            dos[i] = i;
        }

Or you could even use the Arrays.copyOf(int[] array, int length) for an even more efficient way!

The first snippet has to iterate 20 times whereas the second only has to iterate 10 times.
