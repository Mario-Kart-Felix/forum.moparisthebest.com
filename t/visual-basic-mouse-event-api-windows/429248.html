I’m using the mouse_event API from the user32 dll to simulate a mouseclick.

To simulate the click:

mouse_event(&H2, 0, 0, 0, 0) mouse_event(&H4, 0, 0, 0, 0)

But I don’t quite understand the &H2 and &H4.
I know the &H2 means left mouse down and &H4 means left mouse up.

I think those values get written somewhere in the memory, get read by windows --> Click.

Am I right?

Can someone explain this further?

Thank you (:

_s11 August 2, 2016, 3:17pm #2

from reading the docs of mouse_event &H2 is just hexadecimal two so 0x0002 which is also MOUSEEVENTF_LEFTDOWN, &H4 is 0x0004 hence MOUSEEVENTF_LEFTUP

therefore

&H1 - MOUSEEVENTF_MOVE
&H2 - MOUSEEVENTF_LEFTDOWN,
&H4 - MOUSEEVENTF_LEFTUP
&H20 - MOUSEEVENTF_MIDDLEDOWN
&H40 - MOUSEEVENTF_MIDDLEUP
and so on, just convert the 0x notation hexadecimals from the docs to the &H notation used in Visual Basic
