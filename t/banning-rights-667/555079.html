It’s been done plenty in the past, and is a terrible solution to a vulnerable server.
Players shouldn’t just be able to “obtain rights” that you don’t want them to have. At the most they should be able to edit their client so it shows a crown. But because that’d only be client sided, their crown would be literally useless.

You should also have some form of a staff list as is… Not just for security, but for management.

As for implementation, I don’t work with 667 so I’m not very much help.

The list could be as simple as an ArrayList or even just an array.
You can use Strings to hold player names in it. On server start have that list populate with player names (Whether by external file or not).
If a player logs in and they have staff, but they’re not in that list, remove their rights or ban them or whatever you want.

You could also use a HashMap where the key would be the player name, and the value is their rights. This is a bit more secure as you can force specific players on certain rights, and if they change them; the server can force it back.
I.e:

[code=java]
private HashMap<String, Integer> staffList = new HashMap<String, Integer>();

private static void test() {
staffList.put(“dillo1000”, 12);
}

public static void run() {
test();
int testRights= 10;
int listRights = staffList.get(“dillo1000”);
if(testRights != listRights) {
testRights = listRights;
}
}[/code]

That’s just an example though, you’d have to make it work how you wanted.
