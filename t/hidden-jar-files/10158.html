BobDole81 July 29, 2016, 9:23pm #2

There’s no way to do it - a jar file is simply a zip file. There are such applications out there that can package the jar into a binary executable (exe), either the executable will be just be an archive as well or it will have to extract the classes to the system temporary directory to run. There’s no way to totally hide the classes.

semaphore July 29, 2016, 9:25pm #11

If it’s obfuscation you want, you chose the wrong language to develop with. You should be learning c and asm.

For the safest product, develop a subscription based system where the service is remote and no sensitive code resides on the client. The client is ‘dumb’ and it wouldn’t particularly matter much how ‘hacked’ the client was, if the token is invalid, no service is rendered.

If you want a very good example of client-side protection, take a look at the recent-ish versions of Mousepad’s maphack for d2. It was brilliant and incredibly effective.

The bane of java is that it is trivial to ‘hack’. Even know-it-all (know-nothing) kids trying to hack a third or fourth class game have some success.
