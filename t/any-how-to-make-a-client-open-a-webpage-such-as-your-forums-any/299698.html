[center][size=17pt]How to Open a Webpage when you Start a Client![/size]
By: Davidi2[/center]

Difficulty:
0.5/10: Follow the tutorial and you should have no trouble.

Client Tested On:
DavidScape 508 v4.3 Client

[center][size=13pt]Intro[/size]
Have you ever wanted to have a person open your client, then it opens a webpage? Such as voting, your forums ect? Well by adding a class you can make that happen very fast.

[size=13pt]Instructions[/size][/center]

1. Where all of your Client Files are located, create a new text document. Open it and paste this: [code=java]
import java.lang.reflect.Method;
import javax.swing.JOptionPane;
import java.util.Arrays;

public class browserLaunch {

static final String[] browsers = { “firefox”, “opera”, “konqueror”, “epiphany”,
“seamonkey”, “galeon”, “kazehakase”, “mozilla”, “netscape” };

/**
* Opens the specified web page in a web browser
* @param url A web address (URL) of a web page (ex: “http://www.google.com/”)
*/
public static void openURL(String url) {
String osName = System.getProperty(“os.name”);
try {
if (osName.startsWith(“Mac OS”)) {
Class<?> fileMgr = Class.forName(“com.apple.eio.FileManager”);
Method openURL = fileMgr.getDeclaredMethod(“openURL”,
new Class[] {String.class});
openURL.invoke(null, new Object[] {url});
}
else if (osName.startsWith(“Windows”))
Runtime.getRuntime().exec("rundll32 url.dll,FileProtocolHandler " + url);
else { //assume Unix or Linux
boolean found = false;
for (String browser : browsers)
if (!found) {
found = Runtime.getRuntime().exec(
new String[] {“which”, browser}).waitFor() == 0;
if (found)
Runtime.getRuntime().exec(new String[] {browser, url});
}
if (!found)
throw new Exception(Arrays.toString(browsers));
}
}
catch (Exception e) {
JOptionPane.showMessageDialog(null,
“Error attempting to launch web browser\n” + e.toString());
}
}

}
[/code]

Ok, Ill explain that later.

2. Now, Goto File->Save As, and where it says

Save as type: Text Document
change it to “All Files.” Now, change “File Name” to browserLaunch.java, and click save.

Simple Right?

Now here’s what you do next for 508:

3a. Open Client.java and find: var_client Under that line, add: browserLaunch.openURL("YOUR WEBSITE HERE");

What that does is that once the client starts, it calls upon browserLaunch’s void “openURL” to open the website you decide to put in.

Your Done! Pretty Easy.

Now here’s what you do next for 317:

3b. Open Applet_Sub1.java and find: run() Under that line, add: browserLaunch.openURL("YOUR WEBSITE HERE");

What that does is that once the client starts, it calls upon browserLaunch’s void “openURL” to open the website you decide to put in. Its almost exactly like the 508 version.

Explaination:
So now that your done, Ill explain browserLaunch.java, section by section.

Sec1.

[code=java]import java.lang.reflect.Method;
import javax.swing.JOptionPane;
import java.util.Arrays;

public class browserLaunch {

static final String[] browsers = { “firefox”, “opera”, “konqueror”, “epiphany”,
“seamonkey”, “galeon”, “kazehakase”, “mozilla”, “netscape” };[/code]
As most people know, the first few lines are the imports. This tells the class what files from your downloaded java it needs to use in order to complete its task. Then, the official class starts by declaring that it is public browserLaunch. The static string array is a list of possible browsers the person might use. If it only had IE, then there would be a java error when someone with firefox tried to run client.

Sec2. Big section, hold on…

public static void openURL(String url) { String osName = System.getProperty("os.name"); try { if (osName.startsWith("Mac OS")) { Class<?> fileMgr = Class.forName("com.apple.eio.FileManager"); Method openURL = fileMgr.getDeclaredMethod("openURL", new Class[] {String.class}); openURL.invoke(null, new Object[] {url}); } else if (osName.startsWith("Windows")) Runtime.getRuntime().exec("rundll32 url.dll,FileProtocolHandler " + url); else { //assume Unix or Linux boolean found = false; for (String browser : browsers) if (!found) { found = Runtime.getRuntime().exec( new String[] {"which", browser}).waitFor() == 0; if (found) Runtime.getRuntime().exec(new String[] {browser, url}); } if (!found) throw new Exception(Arrays.toString(browsers)); } } catch (Exception e) { JOptionPane.showMessageDialog(null, "Error attempting to launch web browser\n" + e.toString()); } }
Ok. I couldnt really explain this while trying to break it up so I guess ill just do it all.
The first line declared the void public, so that your other files can read and use it. Then String osName = System.getProperty("os.name"); gets the type of Operating System you are running, so that it can find how to execute your browser. After that, if (osName.startsWith("Mac OS")) { Class<?> fileMgr = Class.forName("com.apple.eio.FileManager"); Method openURL = fileMgr.getDeclaredMethod("openURL", new Class[] {String.class}); openURL.invoke(null, new Object[] {url}); }Is if your running a Mac, it uses the com files (in your java version) to declare the method in the Mac File Manager that opens the browser. That method is invoke. Still with me? Ok, if you are: else if (osName.startsWith("Windows")) Runtime.getRuntime().exec("rundll32 url.dll,FileProtocolHandler " + url); This is for if your running windows. This was the only time I preferred Windows to Ubuntu… Its just using the Runtime environment’s rundll’s in C:/Windows to execute the file protocol handler and open the browser. Last but not least, Linux/Unix & the ending. else { //assume Unix or Linux boolean found = false; for (String browser : browsers) if (!found) { found = Runtime.getRuntime().exec( new String[] {"which", browser}).waitFor() == 0; if (found) Runtime.getRuntime().exec(new String[] {browser, url}); } if (!found) throw new Exception(Arrays.toString(browsers)); } } catch (Exception e) { JOptionPane.showMessageDialog(null, "Error attempting to launch web browser\n" + e.toString()); }

It couldnt find an exact OS, so its assuming your using Unix or Linux. It then declares a new boolean called found to false. We start a for loop and start it, declaring a new string browser. The loops condition is !found, or found = false. It then searches the OS Runtime for an executable browser that is in the Array. If it is, it uses the Runtime’s exec() method to execute the browser and set its url to URL. Last but not least, if it cant find any browser, it shows a popup displaying the error.
