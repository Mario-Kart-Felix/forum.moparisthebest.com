I’d like to add to ollie123’s post. In these mid-level programming languages. Almost 80+% use = as an Assignment operator, and == as a comparison operator. That being said, When programming always write as you talk.

scan.nextLine() = funds;
scan dot next line(input) is now equal to funds;

That’s not what you want is it?

funds = scan.nextLine();
funds is now equal to scan dot next line(input)

Basically, you want it to be be said in your head as, “Variable Funds is going to be set as User Input”.

Programming almost always works as a ‘this’ to ‘that’. Everything on the left is being assigned or compared to what is on the right. Also, check your MSN.
