Javacisnotrecognized July 30, 2016, 6:31am #15

Once started, the JVM never terminates, unless you forcibly destroyed each thread and make it call (*vm)->DestroyJavaVM(); from C. If you call System.exit(), somewhere in the VMs code it is calling the C function exit(int), which ends the calling process. The reason it kills the whole browser is because the VM doesn’t compile into an application, so it can’t be started as a seperate process, so is running as the same process as the browser.
