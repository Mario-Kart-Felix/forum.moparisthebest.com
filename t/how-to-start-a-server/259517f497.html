Okay, to add some diversity, I thought I’d add how it looks on my Belkin router. First off, you’re going to have to figure out what your local IP address is (it can be different than this format, but Belkin uses the format of 192.168.2.xxx where xxx is the identifier to a specific computer on the network.)

First, you’re going to need to go to your router (probably located at 192.168.2.1, but can be found using ipconfig as seen in the first post of this thread.) If you look on the left hand side, you should see a link to “DHCP Client List”. Here, we will figure out what your IP address is. Click on this and if you have a password, type it in, if you don’t leave the field blank, as the default password is blank. Now, you should come to a page that looks like this (there will be a row for each computer connected to the router):

When I installed Windows, I named my computer John-Laptop, which is why the hostname is John-Laptop. (If you don’t know yours, you can find it by going to Start, right clicking on (My) Computer and looking for where it says Computer Name, or something of that idea.)

Now that I have found my local IP address is 192.168.2.2, I’m going to go to the link that says “Virtual Servers”. Here’s what the page looks like (I have blurred out my other entries):

You can put anything you want in the “Description” field, this is just for you to know what it’s for later. Make sure you check the Enabled box, and enter the port you would like to start the server on (the default is 43594) in both of the Inbound port boxes. Also make sure you have TCP (Transmission Control Protocol) selected as the type, NOT UDP. As you can see, I entered the last number of my local IP address to finish off the IP address box. This tells the router that whenever someone tries to access this port, to send it to your computer. Lastly, I put the same numbers in the private port boxes as I did in the inbound port boxes. This is just so you can access it from the computer running the server.

Now just scroll down to the bottom and hit “Apply Changes”. It may take up to 60 seconds for the page to load, and your internet may stop working for a few seconds. This is normal. Once this is done, you have successfully forwarded the port.
