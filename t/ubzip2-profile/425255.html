I’m failing to understand why you’re actually working directly on the BZIP2 implementation. The algorithm itself is slow, not the code Apache provided. Unrolling the algorithm doesn’t necessarily make it faster; if anything, it’ll be slower because it’s a larger method that needs to be interpreted/stored in the processor cache/RAM. You’re giving the processor more instructions to execute at one time, to perform the same function.
