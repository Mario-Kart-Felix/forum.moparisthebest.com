With this there isn’t much to think about. You set up a place for users to communicate with each other off game, you set up emails/contacts where they can get ahold of you to alert you about bugs and other issues, you obviously point the domain at your webhosting, you set up a few toplist pages and ask that your users vote for you, and that’s really it.

Java-RSPS-Noob August 4, 2016, 10:04am #7
  1. Buy a domain with real hosting for a webhost
  2. Make an actually nice website/forums, prevent bots from entering
  3. Before you publicly release literally do all skills etc extensively. This way you need to keep a “log” that all players can view to show that you as the owner have already took actions to find most bugs and are currently in the process of fixing them
  4. Start the server up after most of the bugs have been fixed and ask them to find ALL bugs you have missed no matter how big/small.
  5. Keep the bug log on forums/whatever updated at all times
  6. communication is KEY literally. Your players need to see that you ARE updating that you ARE fixing the bugs. That is how the server respect is earned!
  7. Develop new content. Try to keep content as frequent as possible but not too frequent cause that makes it look like it’s shabby work.
  8. Another point is to always remember to keep yourself happy.

Notes*

I didn’t want to include this in the main section here but this is VERY important.

The economy … if you let too many items get flooded at once you WILL fail!. Have money sinks, have item sinks, take items out of the stores, discontinue them, bring NEW items in temporarily. KEEP GAMBLING OUT. Make strict rules for the server about gambling/sharing, multi logging, muling, giving away items once a person quits, and etc.

Remember, they depend on YOU to design and keep the economy stable. They can provide suggestions yes, and I encourage them to provide suggestions. But it is YOU that must understand how an economy works and what is good/bad for it.
