You will NOT be able to make a MMO in them. You just won’t.

So. You want the right direction for MMO-city?
Game Maker/Construct is decent for beginners. You will at least learn some Game Logic and they have built in support for GML/Python so they will introduce you to some basic programming in an enviroment where things actually will happen on the screen.

The alternative is to just learn programming directly. Someone recomended C/C++. I do not. My university started with Java (with Slick2D or just LWJGL for gamedevelopment. But Haskell is a decent choice too (it’s not really game friendly though). Action Script 3 is a good starting point with the right library (Flixel or FlashPunk), but you won’t be able to do a non-farmville-like MMO with it. I’ve never touched Unity so I’m going to say:

Start with C# and XNA. It has decent performance, good documentation, an active community, and some good tutorials from Microsoft™. Visual Studio Express was free last time i checked, so thats a plus.

Just remember:
You will make shitty games for a lot of years. I started with C++ for like 7-10 years ago (I’ve never been dedicated/serious enough to do anything though) and I still make shitty games. Don’t start making your dream game. Make a lot of clones of simple games. Start with pong, go to snake, then breakout, Luna Lander, tetris, Super Mario etc.

I repeat. Your first games will suck without doubt, there is no point in wasting a good idea in a bad game.

And thats just the programming bit, then there is math…

You’ll need lot of math för a 3D game (you will do fine with some basic trigonometry for basic 2D).
