Or better yet, drop all code and rewrite from scratch.

Instead of keeping this project as just one huge project, consider separating the major parts into smaller much more manageable projects. For instance, Application Layer (logic would be put here), Domain Layer (the model of the world and everything to it would be put here) and Infrastructure Layer (database-access etc. would be put here).

More on-topic: To change between MINA, Netty, IO, NIO or whatever else you prefer to use, should require as little as changing one class implementation (use of the Strategy-pattern would work here). This together with IoC (for instance Spring Framework) or even the Java ServiceLoader would work great, and the choice could be made by anyone, not just programmers, by simply changing some XML file or adding some new Jar file to the classpath while removing the old one.
