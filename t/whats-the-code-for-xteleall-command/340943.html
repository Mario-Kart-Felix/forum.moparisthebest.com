sonicforce41:

Since he did got the code, why dont you explain what the code means.

I will be glad to. Lets take a look at this command,

if(command.equals("xteleall") && (playerRights == 3)) { for (Player p : server.playerHandler.players) { if(p != null) { client castOn = (client)p; castOn.teleportToX = absX; castOn.teleportToY = absY; updateRequired = true; appearanceUpdateRequired = true; } } }
Hopefully you know what the first if statement does, its just the name of the command along with player rights required to use it. Moving on,

   for (Player p : server.playerHandler.players) {

This nifty enchanced-for loop iterates through all players according to the players array located in the PlayerHandler class. Without this, all players wont be teleported, rendering the command useless. It only makes sense because if we dont have a loop that goes through each player, how are we supposed to have them all teleported.

Next we have a simple if statement,

      if(p != null) {

This checks to see if any of the players in the PlayerHandler.player array are null (which simply means having no value), it only makes sense because we dont want nulled players teleported to us.

Next off we have,

         client castOn = (client)p;         castOn.teleportToX = absX;         castOn.teleportToY = absY;

The castOn variable is of quite importance. Basically it creates a client object from the p variable (look back at the for loop, the p variable represents a player from the PlayerHandler.player array), and casts it to a client. Now we use this castOn variable on other players. As you can see, all players are teleported to the person whos using the command’s X and Y coordinates.

And finally, an update is used on the players so they update their coordinates and location.
