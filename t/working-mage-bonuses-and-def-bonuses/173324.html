LoL.

Ok, well I recommend making a new method for melee. Directly editing the CalculateMaxHit method would not be proper, since defense doesn’t reduce how much damage you could do, but reduces the likelyhood of you hitting higher. An int would be best in this case.

public int calculateAccuracy() {
return #;
}

You’d have to go from there. My suggestion would to have the return be a percentage value, like:

if (this) {
return 75;
}

That 75 would later be used as a percentage. You’re going to have to toy with the math though, dependant upon the defenders defense level/bonuses and the attackers attack level/bonuses. Remember, strength doesn’t effect how often you hit.

After you have the method down, you’re going to have to edit the parts where it declares damage, my suggestion would be looking for:

hitDiff

Also:

playerMaxHit

When you get to there, you’re going to have to use the return statement, here’s the setup I would advise.

if (misc.random(calculateAccuracy) !> 75) {
hitDiff = playerMaxHit;
}

Although keep in mind 75 is a very high number, because in this context the percentage is how often you hit your max hit. You could do it in another way too, like this:

if (misc.random(calculateAccuracy) !> 75) {
hitDiff = misc.random(playerMaxHit);
} else {
hitDiff = misc.random(playerMaxHit - 10);
}

These are just a few of my on-the-fly ideas. Good luck.

Now for mage, I would do the same but with a new method.

public int calculateMageAccuracy() {

You don’t have to make a new method, you could just put a seperate part in the calculateAccuracy method because all that’s really doing is giving percent accuracy. Again, do it dependant on the defenders levels/bonuses and the attackers levels/bonuses. RuneScape’s mage (defence wise) is based 20% on defense level and 80% on mage level. So someone with 90 mage and 25 defense will get hit less than someone with 25 mage 90 defense, even if they’re wearing the exact same things. Also work in the attack/defense bonuses on mage.
After that, you’re going to have to edit the spells where it declares hitDiff.

~Again, good luck.
