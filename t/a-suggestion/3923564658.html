Both your questions have the same answer, MoparScape 4 executes all clients inside a very restrictive SecurityManager, with only the permissions required to run the client. In fact, you can’t execute native code, you can’t use any reflection, the only website you can contact is the server MoparScape is currently pointed to, and the only file you have read or write access to is your own personal cache folder that is unique to your client. This means the one and only malicious thing a client could do would be to fill up a user’s hard drive.

With a configuration like that, any user can run any client in MoparScape 4 secure in the knowledge that no malicious code can be ran on their computer.

So that is the main reason for me requiring all users of the server status page to use MoparScape 4, so that no idiots infect users, and I don’t have to constantly check to make sure no server status page posts link to viruses.

Did that answer your questions or concerns?
