Beale you keep handing him the answer… And the wrong one at that.

If you want to loop an entire array, why not set the max value of the loop to the arrays length, and not a value you’re guessing?

int someArray[] = new int[25]; // Example 1: for(int i = 0; i < someArray.length; i++) { System.out.println("The value of index " + i + " is " + someArray[i]); someArray[i] = 125;// Setting to 125 for the next loop example. } // Example 2: for(int value : someArray) { System.out.println("The value is " + value); }

The problem with example 2 is, you don’t have an index to work with. So if you were going to use the int in the for loop for another purpose, then you would go with example 1. However, in the case of OP, example 2 should work fine. Assuming all skills in his skill array are being used…

If they’re not, then OP can A) Use example 1, and ignore the indexes not being used; B) Use example 1 or two, and set the array size to how many skills he is using; C) Define the specific number like you did.

The reason why I say not to define some random number, is if/when they decide to add a 26th skill, or they decide that there’s gonna be 20 skills instead of 25. Who wants to go edit a questionable amount of for loops to do change 1 single number? No one.

Agreeing with sk8r, just because something works, doesn’t mean it is the best way to do it. In fact, most of the time, in programming if something “just works”, it is usually badly written.

The point he is trying to make is that if he wanted to add another skill, say sailing, this loop would no longer include that skill. It would only include the 25 skills before it. This may seem arbitrary, but it’s better to be safe than sorry.

In common practice you don’t loop through an array by just guessing the size. A good reason for this is human error. Say if you did this

for(int skill = 0; skill < 15; skill++) { }
You would have an error that wouldn’t show up as an error in the code, but would definitely show up when it came to checking the skills.

However, if you only checked the length of the playerSkill array,

for(int skill = 0; skill < player.playerLvel.length; skill++) { }
If you were to mistype any part of the playerLevel part, it would show up as an error, and upon fixing it, it would still perform as needed.
