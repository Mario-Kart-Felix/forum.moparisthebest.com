Switch/case statements and writing things out are not inherently evil. If you mess around using character code formulas you’re asking for trouble because you’re duct-taping a solution using a pattern that does not actually fit. As for reflection, the same issue applies - it is an incomplete formula that is limited to the same range as the character code formulas. Parsing the constant name will be of no use for anything non-alphanumeric (not even stating that this is an incredibly hacky abuse of reflection).

Also, note that neither the reflection or character code formula method takes modifiers into account properly, as the code I pasted does.
