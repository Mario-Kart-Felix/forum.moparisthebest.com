Today I will show you how to add my simple Thieving base to your server. To my knowledge it can be added to most servers; I have tested it on two (Project Czar & Deltascape). I wrote this last night and it is the shortest thieving I have created so far. It is very simple to build off and add to.

I haven’t added the NPC animations for when you fail the pickpocketing; when I have them added, I will update this topic :).

Step 1:

You will firstly need to add my ThievingHandler class. Save it as ‘ThievingHandler.java’ in your server files. I will explain each bit/method within it.

public class ThievingHandler {

/**

public void ThiefStart(client p) {
	p.thiefNPC = false;

public void ThiefFinish(client p) {
	switch (p.thievingType) {

		case 1:
		case 2:
		case 3:
			finishPP(1, 30, 100, 1, "Man");
			break;

			break;

		case 9:
			finishPP(40, 900, 1200, 5, "Guard");
			break;


	}

/**
 * Start PickPocketing
 */
public void startPP(int lvlReq, String npcName) {
	if (System.currentTimeMillis() - p.stunDelay >= 5000) {
		if (System.currentTimeMillis() - p.thiefTimer >= 2000) {
			if (p.playerLevel[17] >= lvlReq) {
				p.thiefTimer = System.currentTimeMillis();
				p.thiefDelay = 3;
				p.startAnimation(881);
				p.sendMessage("You attempt to pickpocket the "+ npcName +"...");
			} else {
				p.sendMessage("You need to have level "+ lvlReq +" thieving to pickpocket this Npc.");
			}
		}
	} else {
		p.sendMessage("You are stunned!");
	}
}

/**
 * Finish PickPocketing
 */
public void finishPP(int lvlReq, int exp, int cashAmt, int npcHit, String name) {
    	NPC n = server.npcHandler.npcs[p.ThiefNPC];
	int chance = 3 + (p.playerLevel[17] - lvlReq);

	if (misc.random(chance) >= 3) {
		p.addItem(995, cashAmt);
		p.sendMessage("You pickpocket the "+ name +".");
		p.addSkillXP(exp, 17);
	} else {
		p.sendMessage("You fail to pickpocket the "+ name +".");
		p.stunDelay = System.currentTimeMillis();
		p.appendHit(misc.random(npcHit));
		p.gfx100(245);
		p.startAnimation(p.GetBlockAnim());
		n.requestText("What do you think you're doing?!");
		n.TurnNPCTo(p.absX, p.absY);
	}
}

client p;

I have already added some Npcs to the start and finishing methods of the pickpocketing process to start you off.

The ‘ThiefStart’ method enables your character to start thieving an NPC.
Firstly, it sets the ‘thiefNPC’ boolean to false, so it does not call the ‘ThiefStart’ method again until you thieve an NPC again.
Each ‘case’ in the switch-statement represents the ID of the Npc that is being theft from (the thievingType integer). Instead of calling each bit of ‘code’ again and again under each ‘case’ in the switch-statement such as ‘startAnimation(811);’ etc, I have put it into a short method (public void startPP) to shorten the ‘ThiefStart’ method.
Under each ‘case’ in the switch-statement contains this method called:

startPP(integer, "String");

The ‘integer’ is the level required to thief the NPC; if your thieving level does not meet this requirement, then it will send the message that you need the certain level required.
The ‘String’ is the name of the NPC that you are thieving, which will be used in the messages sent to your character such as “You attempt to pickpocket the ‘npc name’…”.

The same goes for the ‘ThiefFinish’ method; I have made a method to call each bit of code, but using less space.
Under each ‘case’ in the switch-statement contains this method called:

finishPP(1, 30, 100, 1, "Man");

The ‘1’ is the level required to thief the NPC - This is used in the calculation as to whether you succeed or fail in the pickpocketing.
The ‘30’ is the experience added to your thieving level if you succeed the pickpocketing.
The second ‘1’ is the maximum damage the npc will deal on you if you fail the pickpocketing.
Obviously, the “Man” string is the name of the NPC and will be used in sent messages to your character such as “You fail to pickpocket the ‘npc name’.”.

This method finishes the thieving and calls the method for the relevant NPC as to whether you succeed in the PickPocketing or not.

The ‘startPP’ method handles the starting pickpocketing codes into one method. It sets it so you can thieve every 2000 milliseconds (2 Seconds) with this:

if (System.currentTimeMillis() - p.thiefTimer >= 2000) {

And if you’ve been stunned it may only let you pickpocket after the 5-second-stunning is over with this:

if (System.currentTimeMillis() - p.stunDelay >= 5000) {

Using the System’s timer instead of ‘processing’ it in the ‘process()’ method saves space and is more efficient.
The rest of the method sets the delay in which the thieving-finish will occur, sends the message “You attempt to pickpocket the ‘npcName’…” and sets the pickpocketing animation.

The ‘finishPP’ method determines whether you succeed or fail in pickpocketing your target Npc. Firstly the method sets the variable ‘n’ so I can call from the ‘NPC’ class using, for example ‘n.hitDiff’ instead of doing ‘server.npcHandler.npcs[p.ThiefNPC].hitDiff’ each time:

NPC n = server.npcHandler.npcs[p.ThiefNPC];

Secondly, it sets the variable ‘chance’ to equal your chance in succeeding/failing by doing the equation ‘3 + (playerLevel[17] - the level Required to PP)’:

int chance = 3 + (p.playerLevel[17] - lvlReq);

If the random number of ‘chance’ is higher than 3, than the pickpocketing succeeds, and you receive the appropriate Exp and Money. If it doesn’t exceed 3, then the pickpocketing fails, you get hit by the NPC, you get stunned for 5 seconds and the NPC will force send the text above it’s head: “What do you think you’re doing?!”.

The ‘GetBlockAnim’ method simply sets your block animation for when you get hit by the Npc when failing a pickpocketing. In this method I use the ‘for’ statement to shorten codes such as:

if (id == 1000 || id == 1001 || id == 1002 || id == 1003 || id == 1004 || id == 1005 || id == 1006 || id == 1007 || id == 1008 || id == 1009 || id == 1010) { return ####; }

Which can be shortened to this:

for (int i = 1000; i < 1011; i++) { if (id == i) { return ####; } }

The ‘appendHit’ method is a shorter and simpler way to declare your character getting hit, and is used for when you get hit by the Npc when failing a pickpocketing. It sets the hitDiff variable to whatever the method’s integer is set to, and then updates the first player hit mask - “hitUpdateRequired = true”.

The ‘updateHP’ method is used in the ‘appendHit’ method, it is a simpler way to declare hp being depleted, and is my alternative than using the ‘healing’ boolean. The updateHP method may be used to both add and deplete HP with the ‘flag’ boolean declared within the method. If the ‘flag’ variable is set to true, then HP will be add onto the character’s current HP. If the ‘flag’ variable is set to false, then HP will be depleted from the character’s current HP. This method also updates the character’s HitPoints skill integer on the stats interface, to whatever the current HP is - “setSkillLevel(3, playerLevel[3], playerXP[3]);”.

Step 3:

Again, in client.java, in your ‘process’ method add these:

if (thievingType > 0 && thiefNPC) { if (GoodDistance(absX, absY, skillX, skillY, 1)) { ThievingHandler thief = new ThievingHandler(this); thief.ThiefStart(this); } } if (thiefDelay > 0) { thiefDelay--; } if (thiefDelay == 1) { ThievingHandler thief = new ThievingHandler(this); thief.ThiefFinish(this); }

The first of the three above calls the constructor within the ThievingHandler class to set the player’s id, then calls the method to start pickpocketing when the thievingType exceeds 0, the thiefNPC boolean is set to true, and when you are within the distance of 1 square of the Npc you are pickpocketing.
The second of the three above simple depletes the ‘thiefDelay’ long by 1 each time the process method is called.
The third of the three above makes it so when the ‘thiefDelay’ long is equal to 1, then the finishing of pickpocketing is called.

At the top of packet 17 in client.java (Search ‘case 17:’ in client.java to find it) add this below ‘case 17:’:

skillX = server.npcHandler.npcs[NPCSlot].absX; skillY = server.npcHandler.npcs[NPCSlot].absY; ThiefNPC = inStream.readUnsignedWordBigEndianA();

The ‘skillX’ and ‘skillY’ integers will equal to the Npc’s coordinates which is used earlier in this tutorial.
The ‘ThiefNPC’ integer will equal to the identification of the NPC.

Next, add this:

if (NPCID == 1) { thievingType = 1; thiefNPC = true; }

This is the pickpocketing set-off for the ‘Man’ npc. This will attempt to pickpocket the man.

In NPC.java, add these methods:

}

The ‘TurnNPCTo’ method sets the NPC’s facing destination to your absX and absY, then updates the focus mask which updates the focus destination of the NPC.

The ‘requestText’ method simply shortens the ‘code’ used in my Handler. This method will use the force text mask to show text above an NPC’s head, and in this case, it will be “What do you think you’re doing?!” when you fail the pickpocket.

Step 6:

To add another NPC to pickpocket, simply add another

if (NPCID == #) { thievingType = #; thiefNPC = true; }

And in the ThievingHandler class, in the ‘startPP’ method add another:

case NPCID: startPP(level Required, "Npc Name"); break;

And again in the ThievingHandler, in the ‘finishPP’ method add another:

case NPCID: finishPP(levelRequired, EXP, Money given, MAX hit from fail, "Npc name"); break;
