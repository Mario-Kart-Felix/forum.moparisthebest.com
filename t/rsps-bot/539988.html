Hey guys, just wanted to share a little project I’ve been working on. It’s not much, but I’m pretty proud with the way it’s turned out so far haha. So basically it’s a bot for a RSPS (Soulsplit, if that counts as advertising, I’ll remove it).

I don’t know what else to tell you about it haha.

• Written in C#.
• Can create custom scripts (Pretty basic).
• Computer ISN’T usable while botting. (I tried SendMessage and PostMessage to simulate mouse clicks in an inactive window, but it didn’t work, so I’m using mouse_event).
• Moves your actual mouse.

Notes:
• GUI is messy, but I’m trying to get the script loader/reader working 100% before I work on the GUI.
• The only reason this is for Soulsplit is because in the script you can move the camera (WriteProcessMemory), and the memory addresses/offsets would be different in other clients.

Features going to be added:
• Invalid player position (E.G if the bot messes up and clicks on something it wasn’t supposed to, it would detect that it is on a tile that wasn’t allowed, and restart the script/run).
• Try to get SendMessage/PostMessage to work so it can be inactive!
• Possible pixel color detection so you can see if an ore/tree is available/unavailable.
• Going to add more options like Mining different types of ore, Adding different types of trees for Woodcutting.
• Going to add scripts for most skills.
• Going to add the ability to load custom made scripts.

Example of a script:

x,500,500,none y,383,500,none 754,289,1000,left 601,386,1500,left 265,521,5000,left 450,120,5000,left 071,136,5000,left say,{F1},500,none 027,315,15000,left 113,350,15000,left 275,234,15000,left 201,273,15000,left 275,234,15000,left 201,273,15000,left 275,234,15000,left 201,273,15000,left 754,289,1000,left 601,386,1500,left 265,521,5000,left 300,208,2000,left 586,334,1000,right 583,404,2000,left 498,140,1000,left 242,309,1500,left 5000,5000,5000,4

Explanation:

Third argument is always how long you want to wait (in milliseconds) before executing the next action.

123,456,1000,left
First and second arguments are the X and Y mouse coordinates, respectively. Fourth argument is what type of mouse click is used (left/right)

x/y,500,500,none
First argument is what camera position you’ll be changing. Second argument is the value you want the camera to be at. Fourth argument is “none” when you are using this command.

say,::empty,1000,yes/no
First argument makes the user say something. Second argument is what you want to say. Fourth argument is whether you want to press Enter after it (May not be needed for switching to Inventory via F4)

5000,5000,5000,5
This line basically tells the program that the script is finished. Fourth argument tells the program where to start the next run at. E.G: With woodcutting, the script starts by moving your camera, teleports you to Draynor, walks to the trees, chops for a minute, then teleports back and deposits the logs, if you had argument four set to 0, it would move the camera and teleport again, but that isn’t necessary to do after every run, and you might waste 5 - 10 seconds.
