It’s not exactly the file system class… to find that class lookup seekablefile.seek() invoke and see if one arg is pushed like:

iload X sipush 512 imul
Or whatever. This is basically doing:

cursectorid * 512

This class is the indexed filesystem class. The class I pasted is the interface they use over it that makes it much easier to use. It lets you lookup entries by name, and have sub-entry abstraction, among other goodies like checksums and ‘versions’ (I’m not too sure on this field of the information table format [entry X inside the indexed file system 255 where X is the indexed file system # and X != 255. If X == 255, the file is the overall checksum file hard-coded in modern new-engine servers as ‘update keys’ or ‘uKeys’])
