Very confused as to what source you have now…

You should try to understand the errors that are appearing, as these are simple things to fix. Pasting the error and expecting someone to fix it without your source will be hard because we cannot browse your files for methods and how variables are used.

I noticed through a lot of those errors you had “SendMessage”. That’s no good because you should always begin method names with a lowercase letter and any sequential words should be capitalized… “sendMessage” would be the correct format, but for Delta (I’ve never used it before), isn’t sendMessage simply “sM”?

Is your server packaged? Look through your Command class and check other commands that have not been altered by you, and see how sendMessage is called. If it has a c.sM(“Message here.”); format, then use that.

If you look through your Client class you can check the methods and how they are declared.

For instance you might have:

public void sM(String str) {//sends a message
//packet shit here
}

This would mean you would use the method like so: “sM(“Welcome to RuneScape.”);”.

“Welcome to RuneScape.”; is a String in Java, and it is inside a method sM();. You can use this concept to look up all your teleport methods and see how they should be implemented.

If at the start of your Commands file you see “Client c” anywhere, then this means a new instance of the public class file “Client” has been created, and you can now access Client.java inside another file, like so:

public class Commands {
        public void playerCommand(Client c) {
                c.sM("Welcome to RuneScape");
        }
}

c. accesses the Client class, because sM(String); is a method in the Client class. Hopefully, now you can look around your files and understand what you are looking for instead of people telling you to add c.getPA().sendMessage(String); when that method does not even exist in your source.
