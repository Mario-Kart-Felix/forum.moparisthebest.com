Just added version checking to the getRS.sh script, and cringe all you want, but I did it with grep, as a bash one-liner. 

get_rs_version(){
	jar_file=$1
	# that's right, I'm parsing bytecode with grep, deal with it, explanations to follow:
	# javap: get the bytecode for client in text
	# egrep: only look for *push or *const, 3 digits, starting with 6 or 7 for now (will have to up that number later)
	# sed: delete everything before the number
	# egrep -v: we don't want certain numbers that appear in every client, like 640 and 765 for now
	# sort | uniq: remove duplicates
	# head -n1: we should be down to the version number anyhow, but just take the first in case it messes up and there are duplicates
	javap -classpath $jar_file -c client | egrep "(push|const)[[:space:]][6-7][0-9]{2}$" | sed "s/^.*[[:space:]]//" | egrep -v "(640|765)" | sort | uniq | head -n1
}

Anyone see anything wrong with that?
