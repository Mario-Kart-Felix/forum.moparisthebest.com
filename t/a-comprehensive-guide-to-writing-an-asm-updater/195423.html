[size=18pt]Part 1: The Updater Base[/size]
[size=16pt]1.1: Introduction[/size]
The Updater Base, or the Base of the Updater, is a compilation of the core classes of the Updater, without them the Updater would be nothing. The base to the updater is like the Linux kernel to, for example, Ubuntu. The base will, for now, contain only two classes: the main Updater class itself containing the main method, and the ‘Transformer’ class which contains all the methods required for transformation of classes.

[size=16pt]1.2: The Updater Class[/size]
[size=14pt]1.2.1: Introduction[/size]
The ‘Updater’ Class is the core class containing the main method used to run the Updater itself. I will start with this class mainly because without it none of the code in the classes to follow would make sense. So in you IDE of choice create a new project, I called mine ‘ASM_Updater’ but you can call yours whatever you want. In the projects source repository add a new Class ‘Updater’ (put it in a decent package). And now to begin.
[size=14pt]
1.2.2: Fields[/size]

The Updater class has several fields used throughout the Updater. These are as follows:

HashMap<String, String> CLASSES = new HashMap<String, String>();// This is where the transformers will put all the Defined classes e.g. add(“Node”, nd), where nd is the name of the class in the RS Source and Node is our name for it.

HashMap<String, String> FIELDS = new HashMap<String, String>(); // Same as CLASSES but for fields.

ClassReader[] loaded = null; // Where the Updater class will dump all the loaded ClassReaders, the transformer classes will then iterate through them to find classes to carry out transformations.

long rsBuildTime,pattenTime, fileTime, totalTime; // Used for debugging

[size=14pt]1.2.3: The Constructor[/size]
In updaters the constructor of the main Updater class is where all the main “action” is called from. This is the design we are going to use:

Parameters: String == the path to the RS class files.

Start the timer -> set the path to the path specified in the parameters -> load the classes into memory -> iterate through the class files until we find the “client” -> run that class through the version getting method -> make one call to each transformer -> stop the timer -> collect all timing information -> print out timing information at the very end.

If you did not understand that… don’t worry!

So currently our constructor look a bit like this:

public Updater(String path) {

	/* Code goes here */

}
So first things first we'll set the path variable we set up to the one specified by the user. After doing this, if you look back at our layout, we start the timer. So now you code should look something like this:

public Updater(String path) {

	long start = System.currentTimeMillis();

}
Then, we load all the classes. Somewhere below create a method with the name “loadClasses” and the parameters (String path), and then make a call to this method below the last line of the constructor. We will come back the the loadClasses method a little bit later on. When we call “loadClasses”, all the files that are found by the method “loadClasses” are then loaded into ClassReaders and then dumped into the variable loaded (which, as you know, is an array of ClassReaders). 

So now we need to find out which ClassReader contains the byte of the class “client” so we can pass it off to our method which finds the RS version number. To do this you will need to iterate through the ClassReaders until you find one with the name client. Now that you've done all of that you constructor should look somewhat like this:

public Updater(String path) {

	long start = System.currentTimeMillis();

	loadClasses(path);
	for(ClassReader cr : loaded) {
		if(cr.getClassName().equals(“client”) {
			/* When it gets here we have found the Client ClassReader */
		}
	}

}
Now we have acquired the client ClassReader we need to pass it off to a method that will examine it to get the version number. This method is getRSVersionNumber(ClassReader cr) and will then return the current RS version, we'll come back to this method later. In your IF clause insert a call to getRSVersionNumber and pass on the current ClassReader to it.
Since the  getRSVersionNumber returns the RS version number we can just stick it right inside a System.out.println() statement. Now we can jump right the way outside our FOR clause.
After the FOR clause is where we would put all the calls to the transformers, so for now just put a TODO comment in to show that we need to put the calls to our transformers here. So our constructor should look somewhat like this now:

public Updater(String path) {

	this.path = path;
	long start = System.currentTimeMillis();

	loadClasses(this.path);
	for(ClassReader cr : loaded) {
		if(cr.getClassName().equals(“client”) {
			System.out.println(“RS build is: ” + getRSVersionNumber(cr));
		}
	}
	
	/* TODO Insert calls to transformers */

}
The rest of the constructor is just used for getting the times and stuff, and since I'm going to “spoon feed” you, you'll have to work that out for yourself.

[size=14pt]1.2.4 Loading the classes[/size]

Now we are going to see how to load the class files into our ClassReader array, which isn't too hard. Skip this section if you already know how to load files into ClassReaders. Currently our loadClasses method looks like so:

public void loadClasses(String path) {

	/* Code goes here */

}
First things first we're going to need to create a instance of File so we can see what is in our specified directory and since the path variable will just be a folder you'll need to insert a call to  System.getProperty(“user.dir”) in the constructor of this File object. Now we need to initiate the “loaded” variable but the thing is: we don't know how many files we are going to need to load. So simple create an integer and then iterate through the files checking that they are not directories and incrediment that integer each time. Like so:

int files = 0;
for (String file : f.list()) {
		f (!new File(file).isDirectory()) {
		files++;
	}
}
Now we know how many files we are going to load we can initiate the “loaded” variable and then put a ClassReader object into each part of the array like so:

loaded = new ClassReader[files];
for(int i=0;i<files;i++){
	FileInputStream fis;
	fis = new FileInputStream(f.listFiles()[i]);
	loaded[i] = new ClassReader(fis);
}
Remember to insert the correct try/catch block where applicable. And there you have it. You have now loaded all the RS class files into ClassReaders. You can even put a little System.out.println at the end to specify how many files have been loaded.

[size=14pt]1.2.5: Getting the RS version number[/size]

This is the final method in the updater class, and likewise the final part in the updater section. As well as being the last, it is also the hardest out of all the methods in this class. So let us begin. You should have already created a code-less method for this and that will look like this:

public int getRSVersionNumber(ClassReader cr) {

	/* Code goes here */

}
For this method we use the ClassNode class of the tree package which is in a separate jar file from the core API that we have just been using so you may need to make some updates to your build path. This is how you do it:

public int getRSVersionNumber(ClassReader cr) {

	ClassNode cn = new ClassNode(); //Creates a new instance of the ClassNode
	cr.accept(cn, 0); //Make the ClassReader accept the ClassNode as its classVisitor
	List methods = cn.methods; //A list of all the methods in this class
	for(int i=0;i<methods.size;i++) {
		MethodNode mn = methods.get(i); //Get the current method from the list “methods”
		if(mn.name.contains(“main”)) { //Check to see if the method is the main method
			AbstractInsnNode[] ain = mn.instructions.toArray(); //Creates an array of AbstractInsnNodes which can be used to check out the different bytecode instructions
			for(int j=0;j<ain.length;j++) {
				AbstractInsnNode insn = ain[j];
				if(insn instanceof IntInsnNode) { //Checks to see if the current instruction is to do with ints, e.g. BIPUSH instructions
					IntInsnNode iin = insn;
					int operand = iin.operand; // Get the operand of that instruction
					if(operand != 503 && operand != 765
									&& (operand > 480 && operand < 500) { //Checks to see that we have not found the width, height or any other numbers that Jagex may put in there to throw us off
						return operand; //If it is none of these numbers then we have found the version number!
					}
				}
			}
		}
	}
}

}

Make sure to put in any try/catch clauses and class casts where necessary. You could add timers if you wanted, but I’m going to leave that out for now. If you have followed the tutorial correctly you should now have a working Updater class! Now for the TransformerBase class!

[size=14pt]1.3.1: Introduction[/size]

The TransformerBase class is the super class of all of you transformation classes, e.g. NodeTransformer. It contains all the necessary methods for the updating procedure to be carried out. So create you new TransformerBase class in you IDE of choice, typically this class has been put in the same package as the Updater class but it doesn’t matter too much.
