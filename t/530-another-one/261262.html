It wouldn’t increase memory usage, just load times as a result of when bits of the cache is loaded into memory, they are requested from the server instead of from your hard disk. The advantages are many however, because they can’t access your hard drive or local network at all, making it impossible to detect you cheating based on checking files, running processes or anything of the like, it basically runs in a sandbox and can only contact the server it originated from.

If you run it signed, the load times and network usage would be reduced, but you would need to transparently redirect File accesses so that each client used it’s own cache (possibly by putting one of your own classes in place of sun’s File, FileInputStream etc), also you have to worry about all the other things they could possibly do to detect your cheating or running more than one instance of the client.
