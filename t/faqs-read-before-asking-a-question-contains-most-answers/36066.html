Q7: Can I edit people’s stats?
A7: No you cannot. You cannot go into the character files and just edit them there. That will not work. You have to make a level up command. You can find this in the Tutorial Section. But with a level up command, it will bring your xp to a certain amount resulting in leveling you up.

Q8: When I try to log into MoparScape, it says I am already logged in.
A8: This means that the server you are trying to connect to suddenly logged you off. It still thinks that you are on. Either restart the client and try again to log in. Or ask the owner to restart the server which should log your guy off.

Q9: [i]When I try to log into MoparScape, it says “Error connecting to server, hit cancel and start server”?
A9: This simply means that the server that you are trying to connect to is down. Try going to the Server Status List and getting a different server ip. So don’t worry. Just keep trying and you will get on one of them eventually.

Q10: Can I edit HybridScape?
A10: No you cannot edit HybridScape because it is built into MoparScape. There is no compile.bat or client.java. HybridScape is for playing alone by yourself. No you cannot edit it. To get an server that you can edit, download any server from the Download’s Section.

Now these are some errors that you may run into frequently.

Error1: The path is not specified.
Answer: This means that you have to set the paths. First download java at www.java.com
[i]Then open up My Computer.

Click System Information

javac2.jpg794×595

Click the Advanced Tab

Click Enviromental Variables

Click New at the top

[i]Then for variable Name put CLASSPATH [i]and for the path put CLASSPATH=C:\Program Files\Java\jdk1.5.0_09\bin;%CLASSPATH%;.
Now click Ok and click New one more time put as the variable name Path and the path put C:\Program Files\Java\jdk1.5.0_09\bin

Now you should be able to runserver and compile.

Error 2: 'javac' is not recognised as a internal or external command
Answer: [i]This means that you have not installed java. You have to go to www.java.com and at the top, click Download Now. Then you wil be able to run your server and compile it.

Error 3: error: unable to startup listener on 43594 - port already in use
Answer: This means that you have another program running on that port. Close all programs using port 43595. Either that or make a .bat file and put this in it: tskill java /a Run that and it will close anything running on port 43594.
Creds: Itachi Uchiha for Idea
