rova0 August 3, 2016, 2:54pm #8

Navigate to : C:\Users[u]Your Name[/u]\VistexCache (it will be called something like that)

If there is a zip file in there called cache.zip (or similar) then you’re OK, otherwise you will need to highlight all files/folders in there and put them in a new zip file called cache.zip (make sure it’s .zip and not .rar)

Next you will need to upload cache.zip to anywhere you can hotlink it from (recommended to use a free webhost or dropbox for this), mainstream file hosts (such as uppit, mediafire, etc) do not allow hotlinking usually, unless you pay for a member account. You will need to obtain a direct link for the download, eg www.somewhere.com/cache.zip - it can’t be to a website where you have to click a button to manually start the download.

Finally open up CacheDownloader.java (client side) and find where it says: " private String cacheLink = " , replace the current link with your own. Then to test everything has gone OK, go back to the folder named VistexCache and either move or delete that entire folder.

Run the client and hopefully it should start to download the client from wherever you uploaded it to. It then unpacks it, give it a few minutes to do its thing and once done the cache is saved.

I hope these steps were clear, just give me a shout if you have more problems. Sorry I don’t like using TeamView and don’t feel like downloading Vistex V5 at the moment to check all this. But as far as I know from the previous versions, they have 99% of the work done for creating the auto-downloading cache. You just need to re-upload it yourself and alter the cache link appropriately.
