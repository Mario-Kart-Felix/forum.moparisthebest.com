get_attack_level(): 3.1753301620483E-6[/quote][/quote]Those results are most likely spoofed, unless you implemented the generic function horridly. The eval() function should, in theory, take longer than simplistic conditional statements and array-index lookups. Regardless of this, the minute incremental benefit in time should not be used as the deciding factor in this case. It is obvious that it would be far more convenient to you and to the user implementing this ‘library’ if you would replace all the skill-specific function naming with a generic one.

As I recommended before, have constants for each skill referencing their index in the array, and have constants referencing indices for “rank”, “level” and “xp”, in order so that the generic function could be invoked as easily as:

get_stat(StatGrabber.ATTACK, StatGrabber.LEVEL)
