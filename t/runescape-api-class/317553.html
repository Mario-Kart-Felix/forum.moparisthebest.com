The problem with the method that you posted is that it’s almost impossible to actually use in any feasible instance. For example, I write a program using your “api” that allows a user to look up a specific stat for a RuneScape player. Problem with your function is that in order to print out, for example, the user’s attack level, i have to have a large switch statement like this:

<?
$skill = $_GET['skill];
switch($skill){
   case 'attack':
      get_attack_level();
   break;
   case 'defence':
      get_defence_level();
   break;
   case 'strength':
      get_strength_level();
   break;
etc etc etc.
}
?>

If you use the method I posted, you could just do this:

$this->getStatInfo($_GET['skill'],"level");

To me, that’s a lot easier and a lot more efficient. If you’re still wanting to design a RuneScape API, I’d suggest reading up on API design (http://lcsd05.cs.tamu.edu/slides/keynote.pdf, for example). It’ll help.
