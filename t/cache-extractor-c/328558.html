This is something I made today in another attempt to succeed in extracting stuff from the 317 cache. It hit me to look in the startup method where it actually unpacks everything. It was all cake from there.

It’s in C#, meaning some things are different. For example, I had to do several changes to get the array belonging to x’s index in a 2D array (You can’t just do array[index] like in Java The way I created the 2D arrays was one of the two ways, which was the different-than-Java one). Other than that, the only thing that changed was the sprite extracting method, some syntax stuff, and IO classes.

I’m supplying the code. If you only download it to see what it does, it just allows you to open your cache, dump the obj.dat file, and display the compass image. This is being released for learning purposes. Unless, of course, you want to build onto it and make it repack (If you do this, share please >.>). Here’s a small screenshot I took first…

Download Source with EXE

Extracting stuff is pretty simple. In fact, it’s done the same way as the client does it. You should just be able to look at the code and figure it out from there. If not, then this isn’t something you want to look at.

Oh, and finally, a bunch of people are going to find this useless. But like I said, it’s for learning purposes so you don’t have to look at the client or other tools which do other things, or a base to build off of.
